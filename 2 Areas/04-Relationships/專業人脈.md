## 人脈分類

```
專業人脈
├── 核心圈（5–10 人）
│   ├── 職業導師
│   ├── 信任的同事
│   └── 長期合作夥伴
│
├── 重要聯繫（15–20 人）
│   ├── 行業領袖
│   ├── 潛在機會提供者
│   └── 技能互補者
│
├── 一般聯繫（50–100 人）
│   ├── 同事
│   ├── 行業同行
│   └── 供應商/客戶
│
└── 潛在人脈
    ├── 社群成員
    ├── 興趣相投者
    └── 社交媒體聯繫
```

| 等級 | 人數 | 定義 | 維護頻率 |
|------|------|------|---------|
| **核心圈** | 5–10 | 最信任、最密切 | 每週/每月 |
| **重要聯繫** | 15–20 | 有價值、需定期互動 | 每月/每季 |
| **一般聯繫** | 50–100 | 建立了基本關係 | 每季/半年 |
| **潛在人脈** | 100+ | 初步認識或弱聯繫 | 按需互動 |

### 拓展技巧

**價值先行原則**：

```
先給予，後獲取
    ├─► 幫助他人解決問題
    ├─► 提供機會和介紹
    └─► 建立信任和口碑
```

**有效網絡技巧**：

1. **做好準備** — 研究對方背景，準備有價值的話題，設定明確目標
2. **主動傾聽** — 70% 傾聽 / 30% 表達，提問深入，顯示真誠興趣
3. **提供價值** — 提供幫助或資源，介紹相關的人脈
4. **及時跟進** — 24 小時內發送感謝，定期保持聯繫

## 價值互換

### 可提供的價值

**技術價值**：
- [ ] 開源項目貢獻
- [ ] 代碼審查
- [ ] 技術諮詢
- [ ] Bug 修復幫助

**知識價值**：
- [ ] 最佳實踐總結
- [ ] 案例研究
- [ ] 技術分享/講座

**資源價值**：
- [ ] 優質項目介紹
- [ ] 人才推薦
- [ ] 工具資源庫

### 成功案例

**案例 1：開源貢獻帶來的合作**

- **背景**：貢獻到 LangChain 項目
- **過程**：發現 Bug → 提交 Issue → 提供修復 PR → 與維護者討論 → PR 合併
- **結果**：與維護者建立聯繫，獲得更多項目參與機會

**案例 2：內部分享建立跨部門合作**

- **背景**：在公司內進行技術分享
- **過程**：準備演示文稿 → 邀請跨部門同事 → 收集反饋 → 後續提供諮詢支持
- **結果**：與產品部門建立合作，獲得跨部門項目機會，被推薦給外部合作夥伴
