# Claudian 命令体系

> **核心理念**: 命令是 Claudian 的快捷方式——一次配置，永久复用，让复杂操作变成一条简单的指令。

## 命令是什么？

### 简单理解

**命令 = 快捷键 + 预设工作流**

就像你为电脑设置快捷键一样，命令让你用几个字就能触发复杂的操作：
- 输入 `/日报` → 扫描所有笔记，生成今日日报
- 输入 `/总结` → 总结当前笔记的核心观点
- 输入 `/思维导图` → 把当前文章画成思维导图

### 命令 vs 技能

| 维度 | 命令 (Command) | 技能 (Skill) |
|-----|----------------|-------------|
| 复杂度 | 简单操作 | 复杂工作流 |
| 结构 | 单个文件 | 文件夹（多文件） |
| 适用场景 | 快捷操作 | 完整任务 |
| 创建难度 | 低（几分钟） | 高（需规划） |

**建议**: 先用命令快速实现简单需求，复杂任务再用技能。

## 内置命令

Claudian 预置了一些常用命令，打开仓库就能用。

### PARA 工作流命令

#### `/para-整理收集`

**功能**
整理收集箱（`0 Personals/📥 00_InBox/`）里的内容，按 PARA 方法论分发到对应位置。

**使用时机**
- 每天晚上整理当日收集的内容
- 每周处理一周积累的笔记
- 收集箱内容过多时

**示例**
```bash
/para-整理收集

AI: 正在扫描收集箱...
发现 15 个文件需要整理：
- 5 个项目相关 → 移至 1 Projects
- 3 个领域相关 → 移至 2 Areas
- 2 个学习资源 → 移至 3 Resources
- 5 个已完成 → 移至 4 Archives

是否继续？
```

**自动分类规则**
- 包含项目关键词 → Projects
- 持续性任务 → Areas
- 学习资料 → Resources
- 已完成内容 → Archives

#### `/para-库概览`

**功能**
查看整个知识库的当前状态，包括各分类的文件数量、最近更新等。

**使用时机**
- 定期检查库状态
- 开始新任务前了解库的情况
- 回顾知识积累

**示例**
```bash
/para-库概览

AI: 📚 知识库概览

1 Projects (活跃项目)
├── Claudian 系统搭建 (2 个笔记)
├── 学习 Rust (8 个笔记)
└── 公众号运营 (5 个笔记)
共: 15 个项目，32 个笔记

2 Areas (长期责任)
├── 健康 (12 个笔记)
├── 职业发展 (8 个笔记)
└── 财务管理 (6 个笔记)
共: 26 个笔记

3 Resources (学习资源)
├── 编程 (45 个笔记)
├── 设计 (23 个笔记)
└── 阅读 (18 个笔记)
共: 86 个笔记

4 Archives (已完成)
共: 156 个笔记

最近更新:
- 2026-01-22: Claudian 系统搭建指南
- 2026-01-21: Rust 学习笔记 #3
...
```

#### `/para-刷新缓存`

**功能**
更新缓存文件，提升查询性能。

**使用时机**
- 新增大量笔记后
- 笔记移动频繁后
- 查询速度变慢时

**示例**
```bash
/para-刷新缓存

AI: 正在刷新缓存...
✓ 项目列表已更新
✓ 笔记索引已重建
✓ 链接关系已刷新
完成！缓存大小: 2.3MB
```

#### `/search`

**功能**
搜索 InBox 和 PARA 内容中的特定信息。

**使用时机**
- 查找特定主题的笔记
- 查找之前记录的想法
- 回顾某个知识点

**示例**
```bash
/search 搜索关于"Obsidian 插件"的所有笔记

AI: 找到 8 个相关笔记：

1. [[1 Projects/Claudian/Claudian 系统搭建指南.md]]
   提到: 插件配置、技能包、MCP 服务器

2. [[3 Resources/Obsidian/插件推荐.md]]
   提到: 20+ 实用插件列表

3. [[5 Zettels/📌 permanent/Obsidian 双链原理.md]]
   提到: 双链机制、知识网络

...
```

### Obsidian 相关命令

#### `/obsidian`

**功能**
智能选择 Obsidian 相关技能，自动处理 Markdown、数据库、可视化画布等任务。

**使用场景**
| 任务 | 自动调用技能 |
|-----|------------|
| 画思维导图、流程图 | json-canvas |
| 创建数据库视图、看板 | obsidian-bases |
| 处理 Markdown 笔记 | obsidian-markdown |

**示例**
```bash
/obsidian 帮我画一个项目进度的思维导图

AI: [自动调用 json-canvas 技能]
正在创建思维导图...
[思维导图图片]
```

### 工具命令

#### `/创建指令`

**功能**
创建新的 Command 指令，简化常用操作。

**使用时机**
- 发现有重复使用的指令
- 想快速调用某个功能
- 为团队创建标准工作流

**语法**
```bash
/创建指令 <指令名> <指令内容>
```

**示例**
```bash
/创建指令 日报 扫描今天编辑过的所有笔记，生成日报

AI: ✅ 命令「日报」已创建！
位置: .claude/commands/日报.md
下次输入 /日报 即可使用。
```

#### `/创建技能`

**功能**
创建新的 Skill 技能包。

**使用时机**
- 需要自动化复杂任务
- 有重复性工作流程
- 需要给团队共享工作方法

**详细说明**: 见 [[2 Areas/Claudian/Claudian 技能系统.md]]

#### `/claudian`

**功能**
PARA 助手菜单，提供知识管理相关的快捷操作。

**示例**

```bash
3. 刷新缓存 → /para-刷新缓存
4. 搜索内容 → /search

```

## 自定义命令

### 创建命令的步骤

#### 方法一: 使用 `/创建指令`（推荐）

**示例 1: 创建「日报」命令**
```bash
/创建指令 日报 `
扫描今天（2026-01-22）编辑过的所有笔记，
提取项目进度、学习内容、思考感悟，
按照以下格式生成日报：

## 今日完成
- [项目1] ...
- [项目2] ...

## 今日学习
- [主题] ...

## 今日思考
- ...
`
```

**示例 2: 创建「周报」命令**
```bash
/创建指令 周报 `
扫描本周（周一到今天）的笔记，
汇总项目进度、学习成果、关键决策，
生成周报。
`
```

#### 方法二: 手动创建命令文件

**步骤**
1. 打开 `.claude/commands/` 文件夹
2. 创建新文件 `命令名.md`
3. 编辑内容

**文件格式**
```markdown
---
name: 命令名
description: 命令描述
---

# 命令执行指令

在这里写具体的操作步骤...
```

**示例: `.claude/commands/总结.md`**
```markdown
---
name: 总结
description: 总结当前笔记的核心观点
---

# 笔记总结

请按照以下步骤总结当前笔记：

1. 读取当前笔记的完整内容
2. 提取核心观点（3-5 条）
3. 找出重要细节和例子
4. 生成结构化总结

输出格式：
- 核心观点：...
- 关键细节：...
- 可执行建议：...
```

### 命令最佳实践

#### 1. 命名清晰

✅ **好的命名**
- `/日报` - 清晰明了
- `/总结当前笔记` - 功能明确
- `/生成思维导图` - 知道会做什么

❌ **避免的命名**
- `/a` - 不知道什么意思
- `/test1` - 无意义
- `/命令123` - 难以记忆

#### 2. 指令具体

✅ **好的指令**
```
扫描今天编辑的笔记，提取项目进度，
生成 markdown 格式的日报，保存到日报文件夹。
```

❌ **模糊的指令**
```
处理今天的笔记。
```

#### 3. 考虑复用性

设计可以重复使用的命令：
```
# 通用总结命令
总结当前笔记的核心观点和关键细节。
```

这样无论打开哪个笔记，都可以用这个命令。

#### 4. 提供参数（高级）

命令可以接受参数：
```bash
/总结 5条  # 只总结 5 个要点
/搜索 Obsidian 指南  # 搜索包含"Obsidian"和"指南"的笔记
```

## 命令管理

### 查看所有命令

在输入框输入 `/`，会弹出所有可用命令列表。

### 删除命令

1. 打开 `.claude/commands/` 文件夹
2. 删除对应的 `.md` 文件
3. 重启 Obsidian 或重新加载插件

### 修改命令

1. 找到 `.claude/commands/命令名.md`
2. 编辑内容
3. 保存后重启生效

### 导出/导入命令

**导出**
```bash
# 复制整个 .claude/commands/ 文件夹
```

**导入**
```bash
# 粘贴到新的仓库的 .claude/commands/
# 重启生效
```

## 故障排除

### 命令不生效

**检查项**
1. 命令文件是否在 `.claude/commands/` 文件夹
2. 文件名是否正确（不含空格）
3. 是否重启了插件或 Obsidian
4. 命令格式是否正确

### 命令执行错误

**可能原因**
- 指令描述不够清晰
- 依赖的技能不存在
- 当前上下文不符合预期

**解决方案**
1. 重新编辑命令文件，提供更清晰的指令
2. 检查依赖的技能是否存在
3. 在对话中直接告诉 AI 需要什么

## 最佳实践总结

### ✅ 推荐做法

1. 从简单命令开始，逐步创建复杂命令
2. 为重复性操作创建命令
3. 定期清理不使用的命令
4. 为命令添加描述性注释
5. 与团队成员共享有用的命令

### ❌ 避免事项

1. 不要在一个命令里塞太多功能
2. 不要创建名称相似、功能不同的命令
3. 不要忘记重启插件后测试新命令
4. 不要在不必要时创建命令（直接对话可能更快）
