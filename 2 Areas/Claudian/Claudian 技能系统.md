# Claudian 技能系统

> **核心理念**: Skill 就像是给 AI 准备的「工作交接大礼包」——打包任务流程、工具用法、参考资料，让 AI 读取后照着干活。

## 什么是 Skill？

### 简单理解

**Skill = 你的数字员工**

你给它工作手册（SOP），它就能帮你干活。就像培训新同事一样，你把任务流程、工具用法、参考资料都打包好，AI 读取后就能照着执行。

### Skill 的价值

| 普通聊天 | 用 Skill |
|---------|---------|
| 每次都要解释「怎么做」 | 一次配置，永久复用 |
| AI 可能忘记规则 | AI 会按 SOP 干活 |
| 遇到问题临时编程 | 直接调用现成脚本 |

### 一个 Skill 的结构

```
skill-name/
├── SKILL.md           # 工作 SOP - 核心指令
├── scripts/           # 工具使用说明 - 可执行代码
├── references/        # 参考资料 - AI 需要时读取
└── assets/            # 工作素材包 - 模板、图片等
```

**各部分说明**

#### SKILL.md（核心）
- 告诉 AI 这个技能是干嘛的
- 如何使用这个技能
- 工作流程和规则

#### scripts/（工具）
- 可执行代码
- AI 可以直接调用
- 不用每次都重新编写

#### references/（参考）
- 参考文档
- AI 需要时才读取
- 避免信息过载

#### assets/（素材）
- 模板文件
- 图片资源
- 其他工作素材

## 本仓库预置的技能包

### 1. obsidian-markdown

**功能**: 让 AI 能正确处理 Obsidian 的特殊语法

**适用场景**
- 处理 Markdown 格式
- 解析 wikilinks `[[note]]`
- 处理 callouts、embeds 等 Obsidian 特有语法

**示例用法**
```
/obsidian 帮我整理这篇笔记的格式
```

### 2. obsidian-bases

**功能**: 让 AI 能创建数据库视图

**适用场景**
- 创建 .base 文件（Obsidian 数据库）
- 设计表格视图
- 创建看板视图

**示例用法**
```
/obsidian 创建一个项目管理看板
```

### 3. json-canvas

**功能**: 让 AI 能画思维导图和流程图

**适用场景**
- 创建 .canvas 文件（可视化画布）
- 绘制思维导图
- 设计流程图、架构图

**示例用法**
```
/obsidian 帮我把这篇文章画成思维导图
```

### 4. skill-creator

**功能**: 让 AI 帮你创建新的技能包

**适用场景**
- 开发自定义技能
- 重复性工作的自动化
- 复杂任务的标准化

**示例用法**
```
/创建技能 我想创建一个「日报生成」技能
```

## 如何使用 Skill

### 方法一: 直接对话

明确告诉 AI 使用哪个技能：

```
使用 obsidian-markdown 技能，帮我整理这个文件
```

### 方法二: 命令调用（推荐）

使用 `/创建指令` 为技能创建启动命令：

```
/创建指令 整理格式 使用 obsidian-markdown 技能整理当前笔记格式
```

创建后，直接输入命令即可：
```
/整理格式
```

### 方法三: 自动选择

使用 `/obsidian` 命令，AI 会根据你的需求自动选择合适的技能：

```
/obsidian 创建一个项目管理看板
```

AI 会自动调用 `obsidian-bases` 技能。

## 如何创建新技能

### 步骤 1: 规划技能

**思考问题**
- 这个技能要解决什么问题？
- 需要哪些工具或脚本？
- 有哪些参考资料？
- 工作流程是什么？

### 步骤 2: 让 AI 帮你创建

**命令方式**
```
/创建技能
```

然后告诉 AI 你的需求：
```
我想创建一个「日报生成」技能：
1. 扫描今天编辑过的所有笔记
2. 提取关键信息和进度
3. 按照特定格式生成日报
4. 可以保存到指定文件
```

**AI 会自动创建**
- `SKILL.md` - 工作流程
- `scripts/` - 必要的脚本
- `references/` - 相关文档
- `assets/` - 模板文件

### 步骤 3: 测试技能

1. 重启 Obsidian 或 Claudian 插件
2. 测试新技能是否工作正常
3. 根据需要调整 SKILL.md

### 步骤 4: 创建命令（可选）

```
/创建指令 日报 使用日报生成技能，扫描今天的笔记并生成日报
```

## Skill 最佳实践

### 1. 保持技能原子化

- ✅ 一个技能做一件事
- ❌ 不要在一个技能里塞太多功能

### 2. 提供清晰的 SOP

**好的 SKILL.md**
```markdown
# 日报生成技能

## 目标
自动扫描今日笔记，生成结构化日报

## 工作流程
1. 获取今日编辑的笔记列表
2. 提取每条笔记的关键信息
3. 按类别（项目、学习、思考）整理
4. 生成 markdown 格式日报
5. 保存到指定文件

## 注意事项
- 跳过没有实际内容的笔记
- 保留原笔记的链接
```

### 3. 合理使用 scripts

- 脚本应该是可复用的工具
- 避免硬编码配置
- 提供清晰的注释

### 4. references 按需设计

- 不要把所有资料都塞进去
- AI 需要时再读取
- 保持精简和聚焦

### 5. 定期维护

- 删除不再使用的技能
- 更新过时的脚本
- 优化工作流程

## 示例：创建「读书笔记」技能

**需求**
自动从书籍目录生成读书笔记模板

**SKILL.md 内容**
```markdown
# 读书笔记生成器

## 功能
根据书籍章节生成结构化读书笔记

## 输入
- 书籍标题
- 章节列表

## 输出
带章节标题的笔记模板，每个章节包含：
- 核心观点
- 金句摘录
- 个人思考

## 工作流程
1. 询问书籍标题和章节
2. 为每个章节创建标题
3. 在每个章节下添加固定模板
4. 保存到指定目录
```

**创建命令**
```
/创建指令 读书笔记 使用读书笔记技能，根据章节生成笔记模板
```

**使用**
```
/读书笔记

请提供：
- 书籍标题：《深入理解计算机系统》
- 章节列表：
  - 第一章：计算机系统漫游
  - 第二章：信息的表示和处理
  - ...
```

## 常见问题

### Q: 技能创建后不生效？
**A**: 需要重启 Obsidian 或重新加载 Claudian 插件

### Q: 如何删除不需要的技能？
**A**: 直接删除对应的文件夹，重启即可

### Q: 技能可以分享给他人吗？
**A**: 可以，整个技能文件夹可以打包分享

### Q: 一个任务可以组合多个技能吗？
**A**: 可以，在 SKILL.md 中说明如何调用其他技能
