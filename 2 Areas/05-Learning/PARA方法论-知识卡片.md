# PARA 方法论

## 📌 基本信息
- **中文名**: PARA 方法论
- **英文名**: PARA Method (Projects, Areas, Resources, Archives)
- **提出者**: Tiago Forte
- **首次提出**: 2017年
- **学科领域**: 知识管理、生产力

## 📖 定义

### 标准定义
> PARA 是一套信息组织系统，将所有信息和任务按照四个维度分类：Projects（项目）、Areas（领域）、Resources（资源）、Archives（归档）。

### 我的理解
PARA 不是简单的文件夹分类法，而是一种基于**行动性**的思维框架。它不是按"主题"分类，而是按"我现在能对它做什么"来分类。这解决了信息堆积无法有效使用的问题。

## 🔍 深入理解

### 核心特征
1. **行动导向**: 每个分类都对应明确的行动类型
2. **动态流动**: 内容会在四个类别间流动
3. **简化决策**: 减少整理时的犹豫和分类困难
4. **定期维护**: 通过归档保持系统清晰
5. **跨工具通用**: 可适用于任何笔记/任务工具

### 边界与限定

**适用情况**:
- 个人知识管理系统搭建
- 信息过载和整理困难
- 需要提升信息检索效率
- 想要系统化的知识管理

**不适用情况**:
- 纯粹的文件存储需求（如媒体库）
- 超大规模的企业级知识库
- 需要高度复杂的分类体系

**常见误解**:
- ❌ PARA 是文件夹结构（它是一种思维框架）
- ❌ PARA 不能改（可以根据工具调整）
- ❌ PARA 必须用英文（可以用中文或其他语言）

**相关但不同的概念**:
- 与 [[Zettelkasten]] 的区别: Zettelkasten 侧重知识卡片和关联，PARA 侧重信息分类和组织
- 与 [[GTD]] 的区别: GTD 侧重任务管理，PARA 侧重信息和任务的整体组织

### 举例说明

| 例子 | PARA 分类 | 理由 |
|------|----------|------|
| 准备下个月的演讲 | P - Projects | 有明确截止日期和可完成状态 |
| 保持身体健康 | A - Areas | 持续维护，无完成状态 |
| Python 编程学习笔记 | R - Resources | 兴趣主题，收集资料 |
| 2023 年度总结 | A - Archives | 已完成，不再活跃 |

## 📝 个人笔记

### 我的理解
PARA 的核心优势在于解决了"分类困难"的问题。传统的按主题分类经常让人犹豫不决，而 PARA 按"我现在能做什么"来分类，决策更加简单直接。

### 学习笔记
- **学习日期**: 2026-01-26
- **难度**: 简单
- **掌握程度**: 理解

### 实践应用
- **已应用**: 我的库已完全采用 PARA 结构
- **应用效果**: 信息组织清晰，检索效率提升
- **待改进**: 定期归档习惯需要加强

### 关键要点记录
1. **Projects = 能标记为"完成"的事情**
2. **Areas = 需要持续维持的标准**
3. **Resources = 有兴趣的主题资料**
4. **Archives = 不再活跃的内容**

### PARA 使用流程
```
收到新信息
    ↓
问自己：这对什么有用？
    ↓
├─ 推动某个项目 → Projects
├─ 维持某个领域 → Areas
├─ 兴趣或参考 → Resources
└─ 都不是 → 删除或 Archive

定期回顾
    ↓
├─ 完成 → Archive
├─ 不再需要 → Archive
└─ 发现新需求 → 创建新 P/A/R
```
