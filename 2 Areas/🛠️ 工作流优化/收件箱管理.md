## 📋 处理流程

### 每日处理（建议早晚各一次）

```
📥 打开收件箱
    ↓
📖 逐一查看每条笔记
    ↓
❓ 这条信息有用吗？
    ├─ 否 → 🗑️ 删除
    └─ 是 → 下一步行动是什么？
            ↓
🎯 有下一步行动？
    ├─ 是 → 是否需要他人？
    │       ├─ 是 → 📋 等待清单
    │       └─ 否 → 是否在 2 分钟内完成？
    │               ├─ 是 → ⚡ 立即执行
    │               └─ 否 → 📅 排入日程/放入项目
    └─ 否 → 有参考价值吗？
            ├─ 有 → 📚 放入资源
            └─ 无 → 🗑️ 删除/归档
```

### 2 分钟原则

如果一件事可以在 2 分钟内完成，**立即执行**，不要加入待办清单。

## ⚡ 快速处理技巧

### 批量处理

1. 设定 15-30 分钟专注时间
2. 只做分类，不深入处理
3. 标记待整理的内容

### 标签系统

| 标签 | 含义 | 处理优先级 |
|------|------|-----------|
| `#urgent` | 紧急 | 立即处理 |
| `#waiting` | 等待他人 | 放入等待清单 |
| `#someday` | 有空再说 | 放入 someday 清单 |
| `#reference` | 参考资料 | 移入资源 |

### 每周清空目标

- **理想状态**: InBox 始终为空
- **可接受状态**: < 10 条待处理
- **警告信号**: > 20 条堆积

## 📊 收件箱健康指标

```dataview
TABLE WITHOUT ID
  file.link as "笔记",
  dateformat(file.cday, "yyyy-MM-dd") as "创建日期",
  date(today) - date(file.cday) as "存放天数"
FROM "0 Personals/📥 00_InBox"
WHERE file.name != "README.md"
SORT file.cday DESC
LIMIT 20
```

### 健康标准

| 指标 | 目标 | 警告 |
|------|------|------|
| 笔记数量 | < 10 | > 20 |
| 平均存放天数 | < 3 天 | > 7 天 |
| 每周清空次数 | 7 次 | < 3 次 |

## 💡 最佳实践

### ✅ 应该做的

1. **随时捕获**: 有想法立即记录
2. **定期清理**: 每日至少一次
3. **快速分类**: 2 分钟内决定去向
4. **保持简洁**: 只记录必要信息

### ❌ 不该做的

1. **过度整理**: 捕获时不要花时间详细组织
2. **堆积如山**: 长期不处理
3. **什么都存**: 无用信息及时删除
4. **忘记清空**: 让收件箱成为垃圾堆
