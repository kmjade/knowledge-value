### 命令最佳实践

#### 1. 命名清晰

✅ **好的命名**
- `/日报` - 清晰明了
# 筆記
- `/生成思维导图` - 知道会做什么

❌ **避免的命名**
- `/a` - 不知道什么意思
- `/test1` - 无意义
- `/命令123` - 难以记忆

#### 2. 指令具体

✅ **好的指令**
```
# 專案
# 檔案
```

❌ **模糊的指令**
```
# 筆記
```

#### 3. 考虑复用性

# 設計
```
# 通用总结命令
# 筆記
```

# 筆記

#### 4. 提供参数（高级）

命令可以接受参数：
```bash
/总结 5条  # 只总结 5 个要点
# 指南
```
