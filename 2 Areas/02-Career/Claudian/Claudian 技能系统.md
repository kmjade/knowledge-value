## 什么是 Skill？

### 简单理解

**Skill = 你的数字员工**

# 讀取

### Skill 的价值

| 普通聊天 | 用 Skill |
|---------|---------|
# 配置
| AI 可能忘记规则 | AI 会按 SOP 干活 |
| 遇到问题临时编程 | 直接调用现成脚本 |

### 一个 Skill 的结构

```
skill-name/
├── SKILL.md           # 工作 SOP - 核心指令
├── scripts/           # 工具使用说明 - 可执行代码
# 讀取
└── assets/            # 工作素材包 - 模板、图片等
```

**各部分说明**

#### SKILL.md（核心）
- 告诉 AI 这个技能是干嘛的
- 如何使用这个技能
# 工作流

#### scripts/（工具）
- 可执行代码
- AI 可以直接调用
- 不用每次都重新编写

#### references/（参考）
# 文檔
# 讀取
# 資訊

#### assets/（素材）
# 檔案
# 資源
- 其他工作素材

## Skill 最佳实践

### 1. 保持技能原子化

- ✅ 一个技能做一件事
- ❌ 不要在一个技能里塞太多功能

### 2. 提供清晰的 SOP

**好的 SKILL.md**
```markdown
# 日报生成技能

## 目标
# 筆記

# 工作流
# 筆記
# 筆記
# 學習
4. 生成 markdown 格式日报
# 檔案

## 注意事项
# 筆記
# 筆記
```

### 3. 合理使用 scripts

- 脚本应该是可复用的工具
# 配置
- 提供清晰的注释

# 設計

- 不要把所有资料都塞进去
# 讀取
- 保持精简和聚焦

### 5. 定期维护

# 刪除
# 更新
# 工作流
