## 📋 核心原则

### PARA 四大分类

| 分类 | 定义 | 特征 | 示例 |
|------|------|------|------|
| **Projects** | 有期限的短期项目 | 有明确目标、有截止日期 | 写完一本书、开发一个产品 |
| **Areas** | 长期责任领域 | 持续维护、无截止日期 | 健康管理、财务管理 |
| **Resources** | 持续感兴趣的主题 | 参考性质、主题导向 | AI 学习、阅读笔记 |
| **Archives** | 已完成或不活跃内容 | 保存但不碍事 | 旧项目、过时资料 |

### 决策流程

```
📥 收件箱
    ↓
有下一步行动吗？
    ├─ 否 → 是否已完成？
    │       ├─ 是 → 📦 Archives
    │       └─ 否 → 🔍 参考价值？
    │               ├─ 有 → 📚 Resources
    │               └─ 无 → 🗑️ 删除
    └─ 是 → 有截止日期吗？
            ├─ 有 → 🎯 Projects
            └─ 无 → 🎯 Areas
```

## 🔄 每日工作流

### 早晨例程

1. **检查收件箱** (`/para-库概览`)
   - 查看昨日新增内容
   - 快速分类到对应位置

2. **查看项目状态**
   - 确认今日待办
   - 更新项目进展

### 工作时间

3. **快速捕获**
   - 灵感 → InBox
   - 任务 → InBox → Project
   - 参考 → InBox → Resource

### 晚间复盘

4. **整理收件箱** (`/para-整理收集`)
   - 分类新内容
   - 归档已完成任务

5. **更新记录**
   - 记录完成事项
   - 标记明日重点

## ⚠️ 常见问题

### Q1: 收件箱堆积太多怎么办？

**A**: 设置每日固定时间整理（如早晚各 15 分钟），使用 `/para-整理收集` 批量处理。

### Q2: 项目和领域的界限模糊？

**A**: 判断标准是有无截止日期。有明确完成日期 → Projects；持续进行 → Areas。

### Q3: 资源太多难以管理？

**A**: 使用标签细分，定期（每季度）回顾并清理低价值资源。

### Q4: 归档后还能找到吗？

**A**: 可以。归档不等于删除，使用搜索仍能找到归档内容，只是移出活跃工作区。
