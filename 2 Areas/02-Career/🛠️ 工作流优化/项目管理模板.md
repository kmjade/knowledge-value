### 任务分解技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| **动词开头** | 明确行动 | "制定计划" 而非 "关于计划" |
| **单一行动** | 每条一个任务 | 拆分为多条 |
| **可执行** | 明确可操作 | "打电话给张伟" 而非 "联系张伟" |
| **有时间限制** | 添加截止日期 | "3月15日前完成" |

## 🔄 项目状态流转

```
📝 规划 (Planning)
    ↓ 完成规划
🔄 进行中 (Active)
    ↓ 遇到阻碍
⏸️ 暂停 (On-Hold)
    ↓ 阻碍解决
🔄 进行中 (Active)
    ↓ 完成所有任务
✅ 完成 (Completed)
    ↓ 30 天后
📦 归档 (Archive)
```

## 📊 项目视图

### 使用 Dataview 查看项目

```dataview
TABLE
  file.link as "项目",
  status as "状态",
  priority as "优先级",
  due_date as "截止日期",
  date(today) - date(due_date) as "剩余天数"
FROM "1 Projects"
WHERE !contains(file.folder, "Archives")
SORT priority ASC, due_date ASC
```

### 按状态分类

```dataview
TABLE WITHOUT ID
  file.link as "项目名称"
FROM "1 Projects"
WHERE status = "active"
SORT file.mtime DESC
```

## 💡 最佳实践

### ✅ 应该做的

1. **明确目标**: 每个项目有清晰的完成标准
2. **分解任务**: 大任务拆分为小步骤
3. **定期更新**: 每周至少更新一次进度
4. **及时归档**: 完成后立即归档

### ❌ 不该做的

1. **目标模糊**: 无法判断是否完成
2. **任务过大**: 难以估计和执行
3. **忽视更新**: 项目信息过时
4. **拖延归档**: 完成的項目堆积
