# Algorithmic Trading (算法交易)

## 定义

算法交易是使用计算机程序按照预设规则自动执行交易决策的方法。交易规则可基于技术指标、统计模型、机器学习，甚至是简单的价格突破条件。从高频交易 (HFT) 到长线趋势跟踪，算法交易覆盖了几乎所有交易风格。

## 核心原理

### 算法交易的基本结构

```
                    ┌─────────────────┐
                    │   交易策略       │
                    │   (人类设计)     │
                    └────────┬────────┘
                             │
                             ▼
  ┌──────────────────────────────────────────────┐
  │              算法交易系统                      │
  │                                              │
  │  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
  │  │ 数据输入  │→│ 信号生成  │→│ 订单执行  │   │
  │  └──────────┘  └──────────┘  └──────────┘   │
  │       ↑                           │          │
  │       │         ┌──────────┐      │          │
  │       └─────────│ 风控管理  │←─────┘          │
  │                 └──────────┘                 │
  └──────────────────────────────────────────────┘
```

### 策略分类

| 类型 | 描述 | 典型持仓 |
|------|------|----------|
| **趋势跟踪** | 跟随市场方向交易 | 数天~数周 |
| **均值回归** | 价格回归均值的套利 | 数分钟~数小时 |
| **突破交易** | 关键价位突破时入场 | 数小时~数天 |
| **统计套利** | 利用资产间价差 | 数秒~数分钟 |
| **高频交易 (HFT)** | 极速订单执行套利 | 毫秒级 |
| **做市商策略** | 提供双向报价赚价差 | 瞬时 |

## 关键要点

1. **回测 ≠ 实盘** — 历史表现不能保证未来收益
2. **过拟合是杀手** — 策略越复杂，过拟合风险越高
3. **滑点和手续费** — 回测必须考虑真实交易成本
4. **风险控制** — 仓位管理比信号质量更重要
5. **持续优化** — 市场在变，策略也要迭代
