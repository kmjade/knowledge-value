# 09-数据伦理

> 统计可以欺骗——而且经常被用来欺骗。

## p-hacking

| **定义**: 反复分析数据直到得到显著结果——然后只报告这一次 |
|------|
| 做法: 收集更多数据后停止·变换变量定义·删除「不合作」的观测 |
| 后果: 假阳性泛滥——可重复性危机 |

## 多重比较问题

| 检验20个假设——即使全部无效——平均会有一个 p<0.05 (纯随机) |
|------|
| 对策: Bonferroni校正(α/检验次数)·FDR(Benjamini-Hochberg)·预注册 |

## 可重复性危机

| 2015: 100个心理学实验——只有36%可重复 (Open Science Collaboration) |
|------|
| 原因: p-hacking·小样本·发表偏差(显著结果更容易发表) |
| 解决: 预注册·开放数据·大样本·报告效应量而非仅p值 |

## 数据可视化伦理

| 操纵 | 例子 |
|------|------|
| **截断Y轴** | 从50开始而不是0——放大微小差异 |
| **樱桃挑选** | 只展示支持结论的时间段 |
| **3D饼图** | 扭曲比例——前面的扇区看起来更大 |
