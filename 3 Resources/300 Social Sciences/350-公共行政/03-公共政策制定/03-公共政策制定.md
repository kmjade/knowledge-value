# 03-公共政策制定

> 政策不是从天上掉下来的——它是多方博弈的结果。

## 政策循环

| 阶段 | 内容 |
|------|------|
| 议程设定 | 为什么这个问题「突然」被关注？——触发事件·倡导者·政治窗口 |
| 政策制定 | 提出方案——谁参与？——专家·利益集团·公众 |
| 决策 | 选择方案——政治交易·多数投票·精英共识 |
| 执行 | 把纸上的政策变成行动——最难的阶段 |
| 评估 | 政策有效吗？——反馈到议程设定——循环 |

## 议程设定

| Kingdon 的多源流模型: 问题流+政策流+政治流——在「政策窗口」打开时汇合 |
|------|
| 问题流: 事件·指标——「发生了什么？」 |
| 政策流: 专家社群——「有什么解决方案？」 |
| 政治流: 选举·民意·利益集团——「政治气候合适吗？」 |

## 政策工具

| 工具 | 例子 | 强制性 |
|------|------|:---:|
| **管制** (Regulation) | 环境标准·禁烟令 | 高 |
| **经济激励** | 补贴·税收减免·罚款 | 中 |
| **信息** | 警示标签·营养标签·公开排名 | 低 |
| **直接提供** | 公立学校·公立医院 | — |

## 执行失败

| 为什么好政策执行失败？ |
|------|
| 政策设计不清晰——「目标宏大——措施模糊」 |
| 资源不足——没有钱也没有人 |
| 街头官僚的抵抗——他们用脚投票 |
| 利益集团的破坏——在实施中削弱政策 |
