# 06 - GTD 执行（Engage）

## 选择行动的四个标准

当面对下一步行动清单时，按这个顺序选择：

```
1. 情境 → 我现在在哪儿？有什么工具？
2. 时间 → 我有多少时间可用？
3. 精力 → 我现在精力水平如何？
4. 优先级 → 在可做的里面，哪个最重要？
```

### 情境矩阵

| 情境 | 可做的 |
|------|--------|
| @电脑 + 60分钟 + 精力好 | 写报告、设计、编程 |
| @电脑 + 15分钟 + 一般 | 回邮件、整理文件 |
| @手机 + 随时 | 打电话、查消息、看资料 |
| @外出 + 30分钟 | 取快递、买文具 |
| @任何地方 + 低精力 | 阅读、听播客、整理清单 |

## 每日三层视角

```
      ┌──────────────────────────────┐
      │   日历上的硬性时间           │  ← 不可动摇
      │   ┌──────────────────────┐   │
      │   │   当天必须完成的      │   │  ← 优先级高
      │   │   ┌──────────────┐   │   │
      │   │   │ 清单上可做的  │   │   │  ← 有余力时做
      │   │   └──────────────┘   │   │
      │   └──────────────────────┘   │
      └──────────────────────────────┘
```

## 能量管理 > 时间管理

```
高能量时段（早晨）→ 做需要深度思考的
中能量时段（上午/下午）→ 会议、沟通、处理事务
低能量时段（午后/傍晚）→ 整理、归档、阅读
```

## 拖延破解法

| 拖延原因 | 破解 |
|----------|------|
| 任务太模糊 | 拆成具体的下一步行动 |
| 任务太大 | 拆成 25 分钟能完成的小块 |
| 不想做 | 先做 5 分钟试试（5 分钟法则） |
| 完美主义 | 先完成再完美 |
| 不知从何开始 | 明确第一步物理动作 |

## 批处理

```
回复邮件 → 集中 2 次/天（不要随时收）
处理消息 → 集中 3 次/天
打电话 → 集中时段一口气打
```

## 工作流状态

```
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│ 收集  │ → │ 理清  │ → │ 整理  │ → │ 执行  │
└──────┘    └──────┘    └──────┘    └──────┘
                  ↑                      │
                  └── 新事项 ─────────────┘
```
