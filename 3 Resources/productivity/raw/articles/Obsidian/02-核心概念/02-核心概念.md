# 02 - Obsidian 核心概念

## 仓库（Vault）

仓库 = 一个文件夹 + `.obsidian` 配置目录。

```
MyVault/
├── .obsidian/          ← 配置（主题、插件、设置）
│   ├── app.json
│   ├── themes/
│   └── plugins/
├── 笔记1.md
├── 笔记2.md
└── Attachments/        ← 图片/附件
```

> 一个 Obsidian 可以打开多个仓库，每个仓库独立配置。

## 笔记（Note）

**就是 `.md` 文件**。可以用任何编辑器打开。

```markdown
---
title: 笔记标题
tags: [tag1, tag2]
created: 2026-05-25
---
笔记正文...
```

### 笔记命名

- ✅ 支持中文名、空格
- ✅ `[[链接下来]]` 自动匹配
- ❌ 避免 `: * ? " < > |` 等特殊字符
- 💡 命名建议：**具体描述** > 简短代号

## Wikilink（双向链接）

```markdown
[[笔记名]]                → 创建链接
[[笔记名|显示文字]]        → 自定义显示
[[笔记名#标题]]            → 链接到具体段落
[[笔记名#^block-id]]      → 链接到具体块
```

**双向链接 = 我链接你 + 你知道我链接了你**

打开右侧边栏 → 反向链接面板看谁链了这篇笔记。

## 图谱视图（Graph）

`Ctrl+G` 打开。每个点是一篇笔记，每条线是一个链接。

- **局部图谱**：只看当前笔记的关联
- **全局图谱**：看整个知识库的关联结构
- **分组着色**：按文件夹/标签给节点上色

## 标签系统

```markdown
#标签             普通标签
#ai/ml            嵌套标签
```

标签在 YAML frontmatter 中也可以定义：
```yaml
tags: [ai, ml, research]
```

> 标签 vs 链接：标签用于**分类**，链接用于**关联**。

## 搜索

`Ctrl+Shift+F` 或左侧搜索图标。

**搜索语法**：

| 语法 | 示例 | 说明 |
|------|------|------|
| `word` | `obsidian` | 搜索包含 word 的内容 |
| `"phrase"` | `"知识管理"` | 精确短语 |
| `path:` | `path:Projects` | 限定文件夹 |
| `tag:` | `tag:#ai` | 限定标签 |
| `file:` | `file:学习` | 文件名匹配 |
| `line:` | `line:(TODO)` | 行内搜索 |

## 别名（Alias）

同一篇笔记可以用多个名字引用：

```yaml
---
aliases: [PKM, 知识管理, Personal Knowledge Management]
---
```

然后 `[[PKM]]` 和 `[[知识管理]]` 都会指向同一篇笔记。

## 附件管理

建议配置：设置 → 文件与链接 → 附件文件夹 → `Attachments`

这样所有图片/附件自动归到一个文件夹，保持仓库整洁。
