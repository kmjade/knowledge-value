# 06 - OKR 实施指南

## 季度全流程

```
Week -2: 准备
  □ CEO/管理层讨论下季度大方向
  □ 回顾上季度 OKR 结果

Week -1: 起草
  □ 各部门起草 OKR（自下而上 60% + 自上而下 40%）
  □ 跨部门沟通，消除冲突和盲区

Week 0: 对齐
  □ 全员 OKR 对齐会议
  □ 最终确认并发布

Week 1-12: 执行
  □ 每周一：更新 KR 进度
  □ 每周五：回顾本周进展
  □ 每月：月度检查会议（30 分钟）

Week 13: 复盘
  □ 评分 + 复盘会议
  □ 记录经验教训
  □ 输入到下季度
```

## 第一次实施 OKR

### 试点建议

```
不要全公司推行 → 选 1-2 个团队试点

试点团队特征：
✅ 团队负责人认可 OKR 理念
✅ 团队有可衡量的结果
✅ 团队规模 5-15 人
✅ 愿意接受新方法

试点周期：2-3 个季度
  Q1: 学习尝试
  Q2: 调整改进
  Q3: 验证效果 → 决定是否推广
```

### 第一个季度 OKR 建议

```
不要设太难的 OKR → 第一个季度的目标是"学会 OKR"

O：掌握 OKR 方法并建立有效的季度节奏

KR1：完成 3 次全员 OKR 会议（起草/对齐/复盘）
KR2：团队 OKR 完成度标准差 < 0.3（大家理解一致）
KR3：团队成员普遍认为 OKR 帮助了聚焦（调查评分 > 3.5/5）
```

## OKR 教练

```
每个团队需要一位「OKR 教练」：

职责：
□ 辅导 OKR 的撰写
□ 推动周检查和月回顾
□ 组织季度复盘
□ 确保 OKR 不与绩效挂钩

教练不一定是管理者 → 可以是任何理解 OKR 的成员
```

## 节奏与会议

| 会议 | 频率 | 时长 | 参与人 |
|------|------|------|--------|
| OKR 起草 | 季前 2 周 | 2 小时 | 部门负责人 |
| OKR 对齐 | 季初 | 2 小时 | 全员 |
| 周检查 | 每周 | 15 分钟 | 团队 |
| 月回顾 | 每月 | 30 分钟 | 团队 |
| 季度复盘 | 季末 | 90 分钟 | 全员 |
