# 04 - OKR 对齐与层级

## 三层对齐

```
┌──────────────┐
│  公司 OKR    │  ← 3-5 个 O，指明大方向
│  (Why)       │
├──────────────┤
│  团队 OKR    │  ← 3-4 个 O，对齐公司 + 自主空间
│  (What)      │
├──────────────┤
│  个人 OKR    │  ← 2-3 个 O，对齐团队 + 个人成长
│  (How)       │
└──────────────┘
```

## 对齐 ≠ 层层分解

```
❌ 机械分解：
公司 O：营收 1 亿
  → 团队 A：营收 3000 万
    → 个人：营收 100 万（没意义！）

✅ 有机对齐：
公司 O：成为市场第一
  → 产品团队 O：推出行业领先的功能
    → 工程师个人 O：掌握推荐算法并应用到产品
  → 销售团队 O：建立可复制的销售流程
    → 销售个人 O：成为 Top 3 销售
```

## 对齐方式

### 垂直对齐（层级）

```
公司 O
 ├── 部门 O（承接公司 O 中与本部门相关的部分）
 │     └── 个人 O（承接部门 O 中与自己相关的部分）
 │
 └── 部门 O（各部门有自己的解读和侧重）
```

### 水平对齐（跨团队）

```
产品团队 O ────┐
              ├──→ 协同 KR
工程团队 O ────┘

两个团队可以有共同的 KR：
"产品+工程 → KR：功能交付周期从 4 周降到 2 周"
```

## 对齐检查清单

```
写 OKR 时问自己：

□ 我的 O 是否指向公司/团队 O 的方向？
□ 我的 KR 是否对团队/公司 KR 有直接贡献？
□ 有没有部门的 OKR 是"孤岛"（无人对齐）？
□ 有没有重要目标无人承接？
□ 有没有重复 OKR（多人做同一件事未协调）？
```

## 个人 OKR 与绩效

```
OKR              ≠   绩效考核
追求挑战         ≠   追求达标
60-70% 完成即好  ≠   期望 100%
季度回顾         ≠   薪资调整
透明公开         ≠   排名比较
```

> 如果 OKR 用来考核，就不会有人设"挑战型"目标。两者必须解耦。
