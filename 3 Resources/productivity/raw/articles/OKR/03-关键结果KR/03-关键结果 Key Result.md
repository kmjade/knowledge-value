# 03 - 关键结果 Key Result（KR）

## KR 是什么？

> 关键结果 = 怎么知道目标达成了（定量的、可衡量的）

**KR 的三要素**：
1. **可衡量** — 有数字
2. **有起点和终点** — 从 X 到 Y
3. **是结果而非任务** — 衡量产出，不是动作

## 好 KR 的标准

### ✅ 结果是结果，不是任务

```
O：打造行业最好的移动端体验

✅ KR：应用商店评分从 3.8 提升到 4.5（结果）
❌ KR：重新设计 App 界面（任务）
❌ KR：收集 100 条用户反馈（任务）

任务 → 你做了什么
结果 → 你做到了什么
```

### ✅ 有起点和终点

```
✅ NPS 从 30 提升到 60
✅ 页面加载从 3s 降到 1.5s
✅ 客户数从 500 增长到 2000

❌ NPS 达到 60（缺少起点，不知道是否现实）
❌ 页面加载快（没有数字）
```

### ✅ 可验证

```
✅ 新产品线贡献 20% 营收（财务系统可查）
✅ 月度活跃用户达到 50 万（数据看板可查）

❌ 用户满意度大幅提升（无法验证）
❌ 团队协同效率明显改善（无法衡量）
```

## KR 的类型

| 类型 | 示例 |
|------|------|
| **增长型** | 月活用户从 10 万到 30 万 |
| **质量型** | 客诉率从 5% 降到 1% |
| **效率型** | 部署时间从 2 天降到 2 小时 |
| **收入型** | 月度经常性收入达到 ¥500 万 |
| **满意度型** | NPS 从 40 到 65 |
| **里程碑型** | 通过 SOC2 认证 |

## KR 数量

```
每个 O 配 3-5 个 KR

3 个最理想

5 个是上限

如果少于 3 → O 可能太简单
如果多于 5 → 需要拆分 O
```

## 完整示例

```
O：让用户在使用产品的第一周就感受到明显价值

KR1：注册→首次有效操作的时间从 15 分钟降到 3 分钟
KR2：Day 7 留存率从 20% 提升到 40%
KR3：新手引导完成率从 45% 提升到 80%
KR4：客服关于"怎么用"的工单减少 50%
```

## 信号型 KR vs 指标型 KR

| | 指标型 | 信号型 |
|---|--------|--------|
| 定义 | 精确数字 | 定性 + 证据 |
| 适用 | 有成熟数据体系的 | 新业务/探索性目标 |
| 示例 | 营收 100 万 | 获得 3 个付费客户的正向反馈 |
