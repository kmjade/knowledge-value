# 05 - OKR 评分与复盘

## Google 的评分系统

```
0.0 — 0.3  远未完成（红色 🔴）
0.4 — 0.6  有进展，未达预期（黄色 🟡）
0.7 — 1.0  完成/超额完成（绿色 🟢）
```

> Google 认为 0.6-0.7 是理想的完成度。1.0 意味着目标设得太低了。

## 如何评分

### 定量 KR

```
KR：月活用户从 1 万增长到 5 万
  实际：3.5 万
  完成度：(3.5-1)/(5-1) = 62.5% → 0.6

KR：NPS 从 30 提升到 60
  实际：55
  完成度：(55-30)/(60-30) = 83% → 0.8
```

### 里程碑 KR

```
KR：通过 SOC2 认证
  状态：已完成 → 1.0
  状态：进行中，审计阶段 → 0.7
  状态：刚开始调研 → 0.2
```

### O 的评分

```
O 不单独打分，O 的评分 = 其下 KR 的平均值

O：打造行业最好的移动端体验
  KR1: 0.8
  KR2: 0.6
  KR3: 0.7
  → O 评分 ≈ 0.7
```

## 季度复盘

### 复盘会议流程（90 分钟）

```
Part 1: 回顾 OKR（45 分钟）
  □ 逐条 KR 打分
  □ 讨论为什么是这些分数
  □ 哪些做对了？哪些没做对？

Part 2: 5 个为什么（30 分钟）
  □ 选 1-2 个完成不好的 KR
  □ 用 5 Whys 挖根因
  □ 记录关键发现

Part 3: 下季度 OKR（15 分钟）
  □ 这些发现对下季度 OKR 的影响？
  □ 有什么需要调整的方向？
```

### 复盘问题清单

```
对每个 O 讨论：

1. 这个 O 是否仍然重要？
2. 我们的 KR 是否正确衡量了进展？
3. 阻碍进展的最大因素是什么？
4. 如果重来一次，我们会怎么做？
5. 需要哪些资源或帮助？
```

## 周度检查

| 动作 | 时长 | 频率 |
|------|------|------|
| 更新 KR 进度（数字） | 5 分钟 | 每周一 |
| 写下本周要做的 3 件事 | 5 分钟 | 每周一 |
| 检查是否在推进 OKR | 1 分钟 | 每天 |
