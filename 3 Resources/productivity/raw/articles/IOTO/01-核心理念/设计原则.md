# 设计原则

> [!summary] 概述
> IOTO 的四大設計原則：知止有定、篤行有為、慎思致知、預定日新。

---

## 原則一：知止有定

> 項目驅動，主動設限

### 核心含義

項目是對未來的一種承諾，也是對當前的一種約束。我們只能在有限的時間和資源內，去實現對未來的承諾。

### 三個要點

| 要點 | 說明 |
|------|------|
| 慎重立項 | 每個項目都要經過深思熟慮，評估可行性 |
| 聚焦價值 | 只保留真正重要的項目，拒絕碎片化 |
| 認清自我 | 了解自己的能力和資源邊界 |

### 實踐方法

```
項目評估清單：
□ 這個項目對我的人生目標有何貢獻？
□ 我是否有足夠的時間和資源？
□ 這個項目是否值得長期投入？
□ 我是否願意為此犧牲其他機會？
```

## 原則二：篤行有為

> 成果導向，以終為始

### 核心含義

讓產出目標反向決定輸入內容，跳出「先存後看」的陷阱。在製造中梳理，在交付中優化。

### 三個要點

| 要點 | 說明 |
|------|------|
| 先完成再完善 | 交付最小可行成果，持續迭代 |
| 服務他人體現價值 | 成果要有受眾，避免自嗨 |
| 在製造中梳理 | 通過輸出倒逼輸入 |

### 實踐方法

```
成果導向思維：
1. 我想製造什麼成果？
2. 完成這個成果需要什麼輸入？
3. 這些輸入是否在我當前項目範圍內？
4. 如何快速完成第一版？
```

## 原則三：慎思致知

> 拒絕盲目，捍衛意圖

### 核心含義

你的精力和注意力是極其稀缺的資產，拒絕無意圖的信息侵入。嚴格區分「他人的已知」與「自我的真知」，守衛思維主權。

### 三個要點

| 要點 | 說明 |
|------|------|
| 意圖准入機制 | 無目的不收集，無訂單不進貨 |
| 構建認知防火牆 | 用項目邊界過濾信息 |
| 深度內化加工 | 將他人的已知轉化為自我的真知 |

### DIKW 層級

```
D (Data)     數據 - 原始素材
I (Information) 信息 - 有上下文的數據
K (Knowledge) 知識 - 內化的理解
W (Wisdom)   智慧 - 可遷移的洞見
```

**傳統做法**：搬運 D/I（他人的已知）
**IOTO 做法**：鍛造 K/W（自我的真知）

### 實踐方法

```
信息准入檢查：
□ 這個信息對哪個項目有用？
□ 我準備用它來製造什麼成果？
□ 這是否只是別人的觀點（D/I）？
□ 我如何將其轉化為自己的見解（K/W）？
```

## 原則四：預定日新

> 秩序先行，習慣為王

### 核心含義

好的秩序是高效的前提。讓大腦專注於創造，讓瑣事歸於系統。善用自動化，堅持數據驅動。

### 三個要點

| 要點 | 說明 |
|------|------|
| 習慣比工具重要 | 工具服務於習慣，而非相反 |
| 保持結構簡單 | 避免過度分類，聚焦核心流程 |
| 漸進增強 | 從最簡單開始，逐步優化 |

### 實踐方法

```
建立習慣循環：
提示 → 行動 → 獎勵

IOTO 習慣：
- Alt+1/2/3/4 快捷鍵肌肉記憶
- 每日生產時間固定
- 每週項目回顧
```

## 四原則關係

```
知止有定（定方向）
     ↓
篤行有為（出成果）
     ↓
慎思致知（守邊界）
     ↓
預定日新（建習慣）
     ↓
（循環持續）
```
