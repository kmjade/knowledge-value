# 文件夹架构

> [!summary] 概述
> IOTO 採用項目驅動的極簡文件夾結構，實現「輸入→輸出→任務→成果」的邏輯循環。

---

## 核心架構

### 目錄結構

```
📁 0-輔助/IOTO
├── 📁 Scripts          # 核心腳本
│   └── ioto-scripts.js
└── 📁 Templates        # 模板文件
    ├── 📁 OBIOTO       # 默認模板
    │   ├── 輸入-原始資料.md
    │   ├── 輸入-碎片筆記.md
    │   ├── 輸出-閃念筆記.md
    │   ├── 輸出-卡片筆記.md
    │   └── 成果-*.md
    └── 📁 MyIOTO       # 用戶自定義模板

📁 1-輸入
├── 📁 原始資料         # 外部收集的材料
└── 📁 碎片筆記         # 臨時想法、摘錄

📁 2-輸出
├── 📁 閃念筆記         # 靈感、初步思考
└── 📁 卡片筆記         # 結構化知識卡片

📁 3-任務
├── 📁 項目A            # 項目任務清單
│   ├── 任務清單.md
│   └── 日期任務/
└── 📁 項目B

📁 4-成果
├── 📁 項目A            # 項目成果沉澱
│   ├── 📁 文章
│   ├── 📁 總結
│   └── 📁 其他
└── 📁 項目B
```

---

## 四大核心目錄

### 1-輸入（Input）

**定位**：外部信息的入口

| 子目錄 | 用途 | 內容類型 |
|--------|------|---------|
| 原始資料 | 有明確用途的素材 | PDF、網頁剪藏、文檔 |
| 碎片筆記 | 臨時想法和摘錄 | 靈感、金句、待處理 |

**使用場景**：
- `Alt+1` → 選擇「原始資料」或「碎片筆記」
- 需要明確關聯的項目

### 2-輸出（Output）

**定位**：內化加工的中轉站

| 子目錄 | 用途 | 內容類型 |
|--------|------|---------|
| 閃念筆記 | 快速記錄靈感 | 想法、洞見、關聯 |
| 卡片筆記 | 結構化知識 | 概念、方法、原則 |

**使用場景**：
- `Alt+2` → 選擇「閃念筆記」或「卡片筆記」
- 對輸入材料進行加工

### 3-任務（Task）

**定位**：項目的執行軌跡

| 子目錄 | 用途 | 內容類型 |
|--------|------|---------|
| 項目文件夾 | 項目任務管理 | 任務清單、進度追蹤 |

**特點**：
- 項目在「任務」和「成果」文件夾下成對生長
- `Alt+3` 打開當前項目任務列表

### 4-成果（Outcome）

**定位**：價值沉澱的倉庫

| 子目錄 | 用途 | 內容類型 |
|--------|------|---------|
| 文章 | 完成的文章 | 博客、專欄文章 |
| 總結 | 項目總結 | 複盤、經驗總結 |
| 其他 | 其他交付物 | 視頻腳本、課件等 |

**使用場景**：
- `Alt+4` → 選擇成果類型
- 最終交付的產出物

## 邏輯循環

### 四象限循環

```
         📥 輸入
        ↗        ↘
   🧠 輸出 ←── 📅 任務
        ↘        ↗
         🎯 成果
```

### 流程說明

1. **輸入 → 輸出**：對原材料進行加工，產生知識
2. **輸出 → 任務**：將知識轉化為行動項
3. **任務 → 成果**：執行任務，交付成果
4. **成果 → 輸入**：成果可作為新項目的素材

---

## 項目的雙相映射

### 物理存儲

項目在「任務」和「成果」文件夾下成對生長：

```
3-任務/
├── AI學習/        ← 項目A 任務
└── IOTO研發/      ← 項目B 任務

4-成果/
├── AI學習/        ← 項目A 成果
└── IOTO研發/      ← 項目B 成果
```

### 邏輯映射

項目通過筆記的屬性自動關聯：

```yaml
---
project: AI學習
type: 輸入/原始資料
created: 2026-05-22
---
```

### 好處

- 項目不是死板的存儲容器
- 項目是無處不在的流向指標
- 分散配置體現數據驅動思維

---

## 實踐建議

### 初始化步驟

1. 運行 Framework Settings → 目錄初始化
2. 下載默認模板（IOTO Update）
3. 創建第一個項目
4. 開始使用快捷鍵創建筆記

### 維護原則

- 不要在根目錄創建額外文件夾
- 項目名稱要清晰明確
- 定期清理已完成項目（歸檔）
