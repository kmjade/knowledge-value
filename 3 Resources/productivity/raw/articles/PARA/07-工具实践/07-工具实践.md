# 07 - PARA 工具实践

## 工具选择

| 工具 | 优势 | PARA 体验 |
|------|------|-----------|
| **Obsidian** | 本地 MD、双向链接、插件 | ⭐⭐⭐⭐⭐ |
| **Notion** | 数据库、协作、模板 | ⭐⭐⭐⭐⭐ |
| **飞书** | 多维表格、文档、知识库 | ⭐⭐⭐⭐ |
| **文件夹** | 零成本、任何系统 | ⭐⭐⭐ |
| **Logseq** | 大纲式、开源 | ⭐⭐⭐⭐ |

## 方案 A：Obsidian

```
Vault/
├── 0 Inbox/              ← QuickAdd 一键捕获
├── 1 Projects/
│   ├── 01-Learning/
│   └── 02-Work/
├── 2 Areas/
│   ├── 01-Health/
│   └── 02-Career/
├── 3 Resources/
│   ├── 01-Tech/
│   └── 02-Books/
├── 4 Archives/
│   └── 2026/
├── _templates/
└── _meta/
```

### 项目模板

```yaml
---
title: "{{title}}"
tags: [project/active]
status: active
priority: high
due: "{{due_date}}"
created: "{{today}}"
---
```

### 插件配合

```
QuickAdd → 快速捕获到 0 Inbox
Dataview → 仪表盘查询
Folder Notes → 文件夹即笔记
Calendar → 时间视图
```

## 方案 B：Notion

### 数据库设计

```
Projects 数据库
├── 属性：名称、状态、优先级、截止日
├── 看板视图：按状态
└── 关联 → Areas 数据库

Areas 数据库
├── 属性：名称、描述
└── 汇总 → 活跃项目数

Resources 数据库
├── 属性：名称、主题、类型
└── 画廊视图
```

## 方案 C：纯文件夹

```
Home/
├── 0 Inbox/
├── 1 Projects/
├── 2 Areas/
├── 3 Resources/
└── 4 Archives/
```

> 最简单，全平台通用。配合 Alfred/Raycast 等启动器使用。

## 跨工具原则

```
不管用什么工具，PARA 的核心原则不变：

1. 四层结构
2. 可执行性驱动
3. 渐进整理
4. 定期归档
5. 信任搜索
```

## 工具迁移

```
Notion → Obsidian
  └── 导出 Markdown → 重建 wikilink

Obsidian → Notion  
  └── 批量导入 → 数据库重建

旧文件夹 → PARA
  └── 批量剪切粘贴 + 每周回顾修正
```
