# PARA 知识库总览

> PARA 是 Tiago Forte 在《构建第二大脑》中提出的数字信息组织方法。
> 将全部信息分成四个类别：**P**rojects（项目）、**A**reas（领域）、**R**esources（资源）、**A**rchives（归档）。

## PARA 四层结构

```
┌─────────────────────────────────────────────┐
│              P · Projects                    │
│         有明确目标和截止日期的任务             │
│         状态：活跃 → 完成 → 归档              │
├─────────────────────────────────────────────┤
│              A · Areas                       │
│         没有截止日期的长期责任                 │
│         持续维护，永不完全"完成"               │
├─────────────────────────────────────────────┤
│              R · Resources                   │
│         感兴趣的主题、参考资料                 │
│         可能有用，但不紧急                     │
├─────────────────────────────────────────────┤
│              A · Archives                    │
│         已完成/不活跃/过时的内容               │
│         冷藏而非删除                          │
└─────────────────────────────────────────────┘
```

## 为什么用 PARA？

| 问题 | PARA 的解法 |
|------|------------|
| 文件找不到 | 4 层就够了，不用记路径 |
| 不知道放哪里 | 从「可执行性」判断 |
| 文件夹太深 | 最多 4 层 |
| 到处重复 | 每样东西只放一个地方 |
| 知识沉淀不下来 | Resources → 长期积累 |

## 核心理念

> "组织信息的标准不是'它是什么'，而是'它有什么用'。" — Tiago Forte

**可执行性 > 主题分类**

```
❌ 传统：按主题分类（技术/生活/工作/学习）
✅ PARA：按行动需求分类（现在要做的/持续关注的/参考备查的/完成的）
```
