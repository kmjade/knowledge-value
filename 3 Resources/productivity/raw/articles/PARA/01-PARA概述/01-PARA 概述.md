# 01 - PARA 概述

## 来源

Tiago Forte，《构建第二大脑》（Building a Second Brain），2022 年出版。

## 核心定义

| 层级 | 定义 | 关键问题 | 示例 |
|------|------|----------|------|
| **P**rojects | 有目标和截止日期的任务组 | "我现在在做什么？" | Q2 报告、网站改版 |
| **A**reas | 需要长期维护的责任 | "我持续关注什么？" | 健康、财务、职业 |
| **R**esources | 感兴趣的话题 | "我可能用到什么？" | Python、摄影、AI |
| **A**rchives | 已完成或不活跃 | "我暂时不需要什么？" | 去年的项目 |

## 两个核心原则

### 1. 可执行性驱动

```
问自己：这个信息的"有效期"有多长？

现在要做 → Projects
一直关注 → Areas
以后可能用 → Resources
已经不用 → Archives
```

### 2. 四层就够

```
❌ 不要这样：
工作/部门/项目A/文档/2026/Q2/报告/草稿/v3

✅ 这样：
1 Projects/Q2 报告
```

## PARA 与 GTD 的关系

| | GTD | PARA |
|---|-----|------|
| 侧重 | 任务与行动 | 信息与知识 |
| 核心 | 下一步行动 | 可执行性分层 |
| 工具 | 待办清单 | 文件夹/标签 |
| 互补 | GTD 管理行动 | PARA 管理信息 |

> GTD 告诉你「下一步做什么」，PARA 告诉你「信息放在哪」。两者互补，不是替代。

## 信息流动

```
       新建/捕获
          │
   ┌──────┴──────┐
   │ 和某个项目有关？  │
   └──YES──┬──NO──┘
      Projects    │
           ┌──────┴──────┐
           │ 和长期领域有关？ │
           └──YES──┬──NO──┘
              Areas     │
                   ┌────┴────┐
                   │ 暂时有用？  │
                   └─YES─┬─NO─┘
                    Resources  Archives
```

## 同名歧义

```
PARA 在这里 = Projects / Areas / Resources / Archives
不是 = 西班牙语"为了"
不是 = 段落（paragraph）
```
