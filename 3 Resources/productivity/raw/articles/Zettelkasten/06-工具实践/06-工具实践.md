## 方案 A：Obsidian Zettelkasten

### 目录结构

```
Vault/
├── 0-Fleeting/           ← 闪念笔记
├── 1-Literature/         ← 文献笔记
├── 2-Permanent/          ← 永久笔记（核心）
│   ├── 2026-05-25 关于想法的笔记.md
│   ├── 2026-05-26 关于方法的笔记.md
│   └── ...
├── 3-Index/              ← 索引笔记/MOC
│   ├── MOC-Zettelkasten.md
│   └── MOC-认知科学.md
└── 4-Templates/
```

### 永久笔记模板

```markdown
---
tags: [core, {{topic}}]
created: {{date}}
source: {{source}}
---

# {{一个完整的陈述句标题}}

{{核心观点，1-3 句话}}

## 详细说明

{{展开论述}}

## 链接

- [[相关笔记1]] — 补充关系
- [[相关笔记2]] — 对比关系

## 问题

- 这个想法带来了什么新问题？
```

### 插件组合

```
Templater        → 新建笔记自动套模板
QuickAdd         → 快速捕获闪念笔记
Dataview         → 自动汇总/发现
Better Word Count→ 避免卡片太长
```

### Dataview 查询

```dataview
TABLE file.ctime as 创建时间, length(file.outlinks) as 链接数
FROM "2-Permanent"
WHERE contains(tags, "core")
SORT file.ctime DESC
LIMIT 20
```

## 方案 B：纸笔 Zettelkasten

### 材料

```
□ A6 卡片（或索引卡）
□ 卡片盒（2-3 个鞋盒也行）
□ 笔
□ 编号系统
```

### 工作流

```
1. 闪念笔记 → 口袋备忘录（1-2 天内处理）
2. 文献笔记 → A6 卡片，手写
3. 永久笔记 → A6 卡片，编号 + 链接
4. 索引卡片 → 主题入口（"参见卡片 xx"）
```

> Bob Doto 的方法：一本书，一叠卡片，每次只写一条。

## 方案 C：Logseq

```
Logseq 天然适合 Zettelkasten：
- 大纲式 = 每行一个想法（天然原子性）
- 双向链接自动建立
- 内置闪念笔记页面
```
