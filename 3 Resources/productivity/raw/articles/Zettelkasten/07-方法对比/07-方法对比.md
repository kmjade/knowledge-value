# 07 - Zettelkasten 与其他方法对比

## Zettelkasten vs GTD vs PARA

| 维度 | Zettelkasten | GTD | PARA |
|------|-------------|-----|------|
| 核心目标 | 思考和创造 | 完成行动 | 组织信息 |
| 单位 | 原子笔记（想法） | 下一步行动 | 文件夹/数据库 |
| 组织方式 | 网络链接 | 清单+情境 | 四层结构 |
| 输出 | 文章、书籍 | 完成的任务 | 知识库 |
| 时间跨度 | 数年积累 | 本周/本月 | 持续 |
| 适合谁 | 写作者、研究者 | 事务繁忙的人 | 信息量大的人 |

## 如何组合使用

```
最好的个人系统 = 组合，不是选一个

Zettelkasten → 管理想法和知识（长期）
GTD         → 管理行动和任务（短期）
PARA        → 管理文件和项目（中期）

三层协同：
  想法 → Zettelkasten（深度思考）
  行动 → GTD（执行追踪）
  文件 → PARA（组织存放）
```

## 在 Obsidian 中三合一

```
Vault/
├── 0 Inbox/              ← 闪念笔记 + GTD 收集篮
├── 1 Projects/           ← PARA Projects = GTD 项目
│   └── 各项目文件夹
├── 2 Areas/              ← PARA Areas = 责任领域
├── 3 Resources/          ← PARA Resources + ZK 文献笔记
├── 4 Archives/           ← PARA Archives
├── ZK-Permanent/         ← Zettelkasten 永久笔记（核心）
│   ├── 2026-05-25 笔记1.md
│   └── 2026-05-26 笔记2.md
├── ZK-Index/             ← ZK 索引笔记 / MOC
└── _templates/
```

> ZK 卡片链接到 PARA 项目/领域，PARA 资源引用 ZK 卡片。双向打通。

## 选择建议

```
你主要写作/研究 → Zettelkasten 为核心
你事务繁多 → GTD 为核心
你文件/信息多 → PARA 为核心
你全都想要 → 组合方案（如上）
```
