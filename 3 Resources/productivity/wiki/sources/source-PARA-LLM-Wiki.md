# PARA + LLM-WIKI 架构

## 摘要

论述传统手动知识管理的结构性矛盾（语义污染、多库切换成本），提出从"人管理工具"到"AI 操作系统"的范式转移。详细阐述了 PARA 与 LLM-Wiki 如何互补融合。

## 关键引用

> **语义污染**：专业知识和工作任务被混入同一系统后，AI 输出的上下文质量会急剧下降。

> **范式转移**：旧范式是人管理工具 → 工具存储信息 → 人检索使用。新范式是人输入信息 → AI 治理分流 → 系统自动代谢 → 知识主动服务人。
