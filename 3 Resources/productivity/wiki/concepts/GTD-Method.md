# GTD (Getting Things Done)

## 定义

GTD 是 David Allen 提出的任务管理方法，核心理念是将所有待办事项从大脑中"卸载"到外部系统，通过五步工作流实现无压力高效工作。

## 核心原理

### 五步工作流

```
收集 (Capture)
  → 理清 (Clarify)
    → 整理 (Organize)
      → 回顾 (Review)
        → 执行 (Engage)
```

### 两步原则

1. **大脑是用来思考的，不是用来记事的**
2. **任何 2 分钟内能完成的事，立即做**

## 关键要点

| 步骤 | 目标 | 产出 |
|------|------|------|
| **收集** | 100% 捕获所有承诺 | Inbox |
| **理清** | 逐项判断下一步行动 | 可执行 / 可委派 / 可延期 |
| **整理** | 建立清单系统 | Next Actions / Projects / Waiting For |
| **回顾** | 每周复盘，保持系统可信 | Weekly Review |
| **执行** | 基于情境选择行动 | Context-based 选择 |

### 关键清单

```
Inbox         → 待处理入口
Next Actions  → 按情境组织的下一步行动
Projects      → 需要多步骤完成的结果
Waiting For   → 委派给他人等待回复
Someday/Maybe → 将来可能做的
Calendar      → 硬性时间的约会
```
