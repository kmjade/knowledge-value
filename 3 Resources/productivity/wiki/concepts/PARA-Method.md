# PARA Method

## 定义

PARA 是 Tiago Forte 提出的个人知识管理框架，将信息按**行动性**分为四个层级：Projects（项目）、Areas（领域）、Resources（资源）、Archives（归档）。

## 核心原理

### 四层结构

```
P → Projects   有截止日期的具体行动
A → Areas      持续维护的责任范围
R → Resources  感兴趣的主题材料
A → Archives   已完成或暂停的内容
```

### 核心准则

1. **按行动性组织，而非按主题** — 每个项目最多一个文件夹
2. **文件夹不超过 4 层深度** — 保持扁平
3. **项目必须有明确的完成条件** — 否则是领域
4. **定期归档不活跃项目** — 保持系统清新

### 与传统分类对比

| 传统分类 | PARA |
|----------|------|
| 按主题归类 | 按行动性归类 |
| 深层嵌套 | 4 层扁平 |
| 静态知识库 | 动态行动系统 |
| 强调完美组织 | 强调可操作 |

## 关键要点

1. **P 和 A 的区分**：有 deadline → Project；没 deadline → Area
2. **R 和 A 的区分**：正在用 → Resource；不用了 → Archive
3. **宁可 4 层也别 5 层**：深度越深，维护成本越高
4. **从 P 开始**：新信息先看能否放入现有项目
