# 06-强化学习 (Reinforcement Learning)

> 强化学习是机器学习的一种范式，智能体通过与环境交互、接收奖励或惩罚来学习最优行为策略——就像训练一只狗：做对了给零食，做错了不给。

## RL 的核心框架

### 智能体-环境交互循环

```
        ┌─────────────────────────────┐
        │           环境               │
        │  ┌─────────────────────┐    │
        │  │  状态 sₜ             │    │
        │  └─────────────────────┘    │
        │         ↑         ↓         │
        │   奖励 rₜ    动作 aₜ         │
        │         ↓         ↑         │
        │  ┌─────────────────────┐    │
        │  │  智能体 (Agent)       │    │
        │  │  策略 π(a|s)          │    │
        │  └─────────────────────┘    │
        └─────────────────────────────┘
```

每一步：
1. 智能体观察状态 sₜ
2. 根据策略选择动作 aₜ
3. 环境返回奖励 rₜ₊₁ 和新状态 sₜ₊₁
4. 智能体从经验中更新策略

### 核心概念

| 概念 | 符号 | 含义 |
|------|------|------|
| **状态 (State)** | s | 环境在某一时刻的完整描述 |
| **动作 (Action)** | a | 智能体可以选择的行为 |
| **奖励 (Reward)** | r | 环境对动作的即时反馈（标量） |
| **策略 (Policy)** | π(a\|s) | 给定状态下选择动作的概率分布 |
| **价值函数 (Value Function)** | V(s) / Q(s,a) | 从某个状态（或状态-动作对）开始的期望累积奖励 |
| **折扣因子 (Discount Factor)** | γ | 未来奖励的权重（0~1，越接近 1 越看重未来） |

### RL 的目标

找到最优策略 π*，使**期望累积折扣奖励**最大化：

```
Gₜ = rₜ₊₁ + γ·rₜ₊₂ + γ²·rₜ₊₃ + ...
```

> γ 接近 1：有远见，重视长期。γ 接近 0：短视，只看眼前。

## RL 算法的两大阵营

### 基于模型 (Model-Based) vs 无模型 (Model-Free)

| 类型 | 特点 | 代表 |
|------|------|------|
| **Model-Based** | 先学习环境的转移模型，再在模型中规划 | AlphaZero, MuZero |
| **Model-Free** | 直接从经验中学习策略/价值 | DQN, PPO, SAC |

### 基于价值 (Value-Based) vs 基于策略 (Policy-Based)

| 类型 | 核心 | 代表算法 | 特点 |
|------|------|----------|------|
| **Value-Based** | 学习最优 Q 函数，从 Q 推导策略 | DQN, Double DQN | 离散动作空间 |
| **Policy-Based** | 直接优化策略参数 | REINFORCE, PPO, TRPO | 支持连续动作 |
| **Actor-Critic** | 结合两者：Actor 选动作，Critic 评估 | A3C, SAC, TD3 | 当前主流 |

## 关键算法

### Q-Learning 与 DQN

**Q-Learning**：学习 Q(s, a) = 在状态 s 执行动作 a 后能获得的期望累积奖励。

Q 值更新（贝尔曼方程）：
```
Q(s, a) ← Q(s, a) + α [r + γ·max Q(s', a') - Q(s, a)]
                      └──────┬──────┘   └───┬───┘
                        TD 目标        当前估计
```

**DQN** (Deep Q-Network) 的关键创新：

| 创新 | 解决的问题 |
|------|-----------|
| 经验回放 (Experience Replay) | 打破样本间的相关性，提高数据效率 |
| 目标网络 (Target Network) | 稳定训练，防止 Q 值估计震荡 |

> DQN 的里程碑：2013 年在 Atari 游戏上达到人类水平，从原始像素输入直接学习。

### Policy Gradient 与 PPO

**Policy Gradient**：直接优化策略的参数，使高奖励动作的概率增加。

**PPO** (Proximal Policy Optimization) — 当前最广泛使用的 RL 算法之一：

- 限制每次策略更新的幅度（"不要太激进地改变策略"）
- 简单、稳定、效果好
- OpenAI Five (Dota 2)、ChatGPT 的 RLHF 都用到了 PPO

### RLHF (从人类反馈中强化学习)

这是 ChatGPT 训练流程中的关键一步：

```
预训练语言模型 (GPT base)
        ↓
有监督微调 (SFT) — 用高质量人类回答训练
        ↓
训练奖励模型 (RM) — 让人类标注偏好的回答
        ↓
PPO 强化学习 — 用奖励模型优化语言模型
        ↓
对齐后的模型 (ChatGPT/Claude)
```

## RL 的应用

| 领域 | 里程碑 | 算法 |
|------|--------|------|
| **游戏** | AlphaGo 击败李世石 (2016)、OpenAI Five Dota 2 (2019) | MCTS + DL, PPO |
| **机器人** | 灵巧手操控魔方、四足机器人行走 | PPO, SAC |
| **LLM 对齐** | ChatGPT, Claude | RLHF → PPO / DPO |
| **推荐系统** | YouTube, 抖音的推荐 | Bandit, off-policy RL |
| **自动驾驶** | 决策规划 | 分层 RL |
| **芯片设计** | Google TPU 芯片布局 | RL + 搜索 |

## RL 的挑战

| 挑战 | 说明 |
|------|------|
| **样本效率低** | 需要海量交互经验（AlphaGo 训练了数百万局自我对弈） |
| **奖励设计难** | 奖励函数的设计直接影响学习结果（奖励黑客问题） |
| **探索 vs 利用** | 尝试新动作（探索）还是做已知最好的（利用）？ |
| **Sim-to-Real** | 在仿真中训练的策略迁移到真实世界时常常失效 |
| **安全** | 物理世界中的 RL——一个错误动作可能代价高昂 |
