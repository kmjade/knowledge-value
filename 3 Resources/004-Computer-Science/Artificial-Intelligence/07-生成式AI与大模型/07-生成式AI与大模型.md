# 07-生成式AI与大模型

> 生成式 AI 是能创建新内容（文本、图像、代码、音频、视频）的 AI 系统。2022 年 ChatGPT 发布后，这一领域经历了前所未有的爆发。

## 什么是大语言模型 (LLM)？

LLM 是**大规模自回归语言模型**——在海量文本上训练，学习预测下一个 token。

| 特性 | 说明 |
|------|------|
| 训练目标 | 预测下一个 token: P(tokenₙ | token₁, token₂, ... tokenₙ₋₁) |
| 参数规模 | 从 7B (十亿) 到 1000B+ |
| 训练数据 | 数万亿 token 的互联网文本、书籍、代码 |
| 能力来源 | 规模——模型越大、数据越多、训练越久，能力越强（Scaling Laws） |

## LLM 的核心能力

### 涌现能力 (Emergent Abilities)

当模型规模超过某个临界点（约 10B 参数）时，突然出现训练时未明确教授的能力：

| 能力 | 说明 |
|------|------|
| **上下文学习 (In-context Learning)** | 通过提示中的示例学会新任务，无需微调 |
| **思维链 (Chain-of-Thought)** | 展示逐步推理过程，显著提高复杂推理准确率 |
| **指令跟随** | 理解并执行自然语言指令 |
| **工具使用** | 调用 API、搜索、代码解释器等外部工具 |

### 关键局限

| 局限 | 说明 |
|------|------|
| **幻觉 (Hallucination)** | 生成看似合理但事实错误的文本 |
| **知识截止** | 训练数据有截止日期，不知道后续事件 |
| **上下文窗口有限** | 单次对话能"记住"的信息有上限（虽然从 2K 扩展到 1M+ token） |
| **推理能力不稳定** | 复杂多步推理可能出错，尤其数学和逻辑 |
| **偏见** | 反映训练数据中的社会偏见 |

## 提示工程 (Prompt Engineering)

> 提示工程是用自然语言"编程"LLM 的艺术和科学。

### 基本技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| **零样本 (Zero-shot)** | 直接提问 | "翻译成英文：你好" |
| **少样本 (Few-shot)** | 给几个例子 | "汉→英：你好→Hello / 再见→Goodbye / 谢谢→" |
| **思维链 (CoT)** | 要求逐步推理 | "让我们一步步思考..." |
| **角色扮演** | 指定人格 | "你是一位有 20 年经验的 Python 工程师..." |
| **结构化输出** | 指定输出格式 | "用 JSON 格式回答，包含 name 和 age 字段" |

### 高级技巧

| 技巧 | 说明 |
|------|------|
| **ReAct** (Reasoning + Acting) | 交替进行思考和行动（调用工具） |
| **Self-Consistency** | 多次采样 → 投票选出最一致的答案 |
| **Tree of Thoughts** | 探索多个推理路径，评估后选择最优 |
| **Constrained Decoding** | 强制输出符合某种语法/格式 |

## 检索增强生成 (RAG)

RAG 是将 LLM 与外部知识库结合的技术——解决幻觉和知识过期问题。

```
用户提问
    ↓
检索相关文档（从向量数据库/搜索引擎）
    ↓
将检索到的文档 + 原始问题拼接为 prompt
    ↓
LLM 生成基于检索内容的回答
    ↓
返回（附引用来源）
```

### RAG 的技术栈

| 组件 | 常用工具 |
|------|----------|
| 文档加载/切分 | LangChain, LlamaIndex |
| 嵌入模型 | text-embedding-3, BGE, Jina |
| 向量数据库 | Pinecone, Weaviate, Chroma, Milvus |
| LLM | GPT-4, Claude, DeepSeek, 开源模型 |
| 编排框架 | LangChain, LlamaIndex, DSPy |

## AI Agent

> AI Agent 是能够自主规划、使用工具、执行多步任务的 LLM 系统。

### Agent 的核心组件

| 组件 | 作用 |
|------|------|
| **LLM 大脑** | 推理和决策核心 |
| **规划** | 任务分解、反思、自我修正 |
| **工具** | 搜索、代码执行、API 调用、文件操作 |
| **记忆** | 短期（上下文窗口）、长期（外部存储） |

### Agent 框架对比

| 框架 | 特点 |
|------|------|
| **LangChain** | 最成熟的 LLM 应用框架，生态丰富 |
| **AutoGPT** | 最早的自主 Agent 尝试之一 |
| **CrewAI** | 多 Agent 协作 |
| **Hermes Agent** | 开源 CLI Agent 框架，支持多平台 |

### 闭源

| 模型 | 公司 | 特点 |

## 多模态模型

| 模型类型 | 输入 | 输出 | 代表 |
|----------|------|------|------|
| 文本→图像 | 文字 | 图像 | Stable Diffusion, DALL-E 3, Midjourney |
| 图像理解 | 图像 + 文字 | 文字 | GPT-4V, Claude Vision, Qwen-VL |
| 文本→视频 | 文字 | 视频 | Sora, Runway Gen-3, Kling |
| 文本→音频/音乐 | 文字 | 音频 | Suno, Udio, MusicGen |
| 全模态 | 文字+图+音频+视频 | 文字+图+音频+视频 | GPT-4o, Gemini |
