# 光貓管理基礎知識

> [!info] 概述
> 光貓（光纖貓）是將光信號轉換為電信號的網絡設備，本文檔介紹光貓的基礎管理知識，包括帳戶類型、訪問方式和基本配置。

---

## 一、光貓概述

### 1.1 什麼是光貓

光貓（ONT - Optical Network Terminal）是光纖入戶的核心設備，負責將光纖傳輸的光信號轉換為以太網絡信號。

| 組件 | 功能 |
|------|------|
| **光模塊** | 接收/發送光信號 |
| **處理器** | 數據轉發和協議轉換 |
| **LAN接口** | 連接路由器/電腦 |
| **管理界面** | 配置和管理功能 |

### 1.2 訪問地址

| 類型 | 默認地址 | 說明 |
|------|----------|------|
| **電信光貓** | 192.168.1.1 | 大多數電信設備 |
| **移動光貓** | 192.168.1.1 | 部分型號不同 |
| **聯通光貓** | 192.168.1.1 | 視型號而定 |
| **自定義地址** | 需查詢設備標籤 | 可能已被修改 |

## 二、帳戶類型

### 2.1 雙帳戶體系

光貓通常提供兩個管理帳戶：

| 帳戶名稱 | 權限級別 | 功能範圍 |
|----------|----------|----------|
| **useradmin** | 普通用戶 | 瀏覽配置、查看統計 |
| **telecomadmin** | 超級管理員 | 完全控制、修改所有設置 |

### 2.2 帳戶說明

#### useradmin 帳戶

| 屬性 | 說明 |
|------|------|
| **權限** | 有限管理權限 |
| **可訪問** | 配置頁面、統計信息 |
| **不可訪問** | 超級密碼、運營商配置 |

#### telecomadmin 帳戶

| 屬性 | 說明 |
|------|------|
| **權限** | 完全管理員權限 |
| **可訪問** | 所有功能和配置 |
| **密碼獲取** | 需要通過技術手段獲取 |
| **用途** | 修改 UPNP、DMZ、橋接模式等 |

### 2.3 密碼要求

> [!warning] 密碼限制
> 用戶名和密碼必須滿足以下條件：
> - 長度：不超過 16 個字符
> - 字符：不包含空格
> - 格式：字母數字組合

## 三、訪問步驟

### 3.1 連接光貓

```mermaid
flowchart LR
    A[電腦/筆記本] -->|網線| B[光貓 LAN 口]
    B --> C[確認 IP 地址]
    C --> D[訪問管理界面]
```

### 3.2 獲取光貓地址

如果地址被修改，可通過以下方式獲取：

```bash
# Windows - 查看網關地址
ipconfig

# 查找默認網關
# 默認網關即為光貓地址
```

### 3.3 登錄界面

| 步驟 | 操作 |
|------|------|
| 1 | 打開瀏覽器，輸入 `http://192.168.1.1` |
| 2 | 輸入用戶名和密碼 |
| 3 | 進入管理界面 |

## 四、常見配置

### 4.1 UPNP 配置

| 設置 | 說明 |
|------|------|
| **開啟** | 允許設備自動端口映射 |
| **關閉** | 需手動配置端口轉發 |
| **用途** | P2P 下載、遊戲聯機 |

### 4.2 DMZ 主機

| 設置 | 說明 |
|------|------|
| **功能** | 將指定設備暴露於公網 |
| **風險** | 安全性降低 |
| **用途** | 服務器搭建、遠程訪問 |

### 4.3 橋接模式

| 設置 | 說明 |
|------|------|
| **路由模式** | 光貓負責 PPPoE 撥號 |
| **橋接模式** | 將撥號交給路由器 |
| **用途** | 獲得公網 IP、減少 NAT 層 |

## 五、風險提示

> [!warning] 安全風險
> 獲取和使用超級密碼可能涉及以下風險：
> - 違反運營商服務條款
> - 導致網絡服務中斷
> - 造成設備配置錯誤
> - 喪失保修資格

### 建議操作

| 操作 | 說明 |
|------|------|
| **謹慎操作** | 了解每一步的影響 |
| **備份配置** | 修改前導出配置 |
| **記錄密碼** | 記錄修改過的配置 |
| **恢復默認** | 出現問題時恢復出廠設置 |
