# 04-编程与示教

> Cobot 的编程门槛远低于传统工业机器人——拖动示教让操作员"手把手"教机器人，图形化界面替代了代码。

## 编程方式对比

| 方式 | 难度 | 灵活度 | 适合 |
|------|------|--------|------|
| **拖动示教** | 极低 | 低 | 简单点位——取放 |
| **图形化编程** | 低 | 中 | 逻辑控制+点位 |
| **脚本编程** | 中 | 高 | 复杂逻辑、算法 |
| **离线编程** | 中 | 最高 | 复杂轨迹、仿真 |

## 拖动示教

### 操作流程

```
1. 切换到示教模式
2. 手握住末端→移动到目标位置
3. 按下"记录航点"按钮
4. 重复 2-3 直到路径完成
5. 插入逻辑 (夹爪开/合, 等待信号)
6. 测试运行
```

### 示教技巧

| 技巧 | 说明 |
|------|------|
| 先粗后细 | 快速拖到大致位置→微调用坐标输入 |
| 统一方向 | 所有航点保持同方向→避免不必要的关节翻转 |
| 加过渡半径 | 航点之间加圆角→运动更平滑更快 |
| 命名航点 | 用有意义的名字——"取料位""放料位" |

## 图形化编程 (UR Polyscope)

UR 的编程界面——树状结构和流程图：

```
程序 (Program)
├── 取料
│   ├── 移动到 取料位 (MoveJ)
│   ├── 夹爪 闭合 (Set DO)
│   └── 等待 0.5s (Wait)
├── 搬运
│   ├── 移动到 途经点 (MoveJ, 过渡半径 50mm)
│   └── 移动到 放料位 (MoveL)
├── 放料
│   ├── 夹爪 张开 (Set DO)
│   └── 等待 0.3s (Wait)
└── 返回
    └── 移动到 待机位 (MoveJ)
```

## 脚本编程 (URScript)

```python
# URScript 示例
def pick_and_place():
    movej([0.1, -1.5, 1.2, -0.5, 1.5, 0.0])  # 取料位
    set_digital_out(0, True)  # 夹紧
    sleep(0.5)
    movel([0.3, -0.2, 0.1, 0, 3.14, 0])  # 放料位 (直线)
    set_digital_out(0, False)  # 松开
    sleep(0.3)
    movej([0.1, -1.5, 1.2, -0.5, 1.5, 0.0])  # 返回
```
