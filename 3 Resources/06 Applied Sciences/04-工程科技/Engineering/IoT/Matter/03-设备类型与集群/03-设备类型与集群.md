# 03-设备类型与集群

> Matter 通过标准化的"设备类型"和"集群"来定义设备能力——不再需要为每种设备写自定义协议。

## 设备类型

Matter v1.3 支持的设备类型：

| 类别 | 设备类型 |
|------|----------|
| **照明** | 开关灯、可调光灯、彩色灯 |
| **传感器** | 接触传感器、光照传感器、占空传感器、温度传感器、湿度传感器、压力传感器、流量传感器 |
| **安防** | 门锁 |
| **窗帘** | 卷帘、百叶窗 |
| **媒体** | 电视、流媒体播放器、扬声器 |
| **家电** | 空调、空气净化器、加湿器、风扇、扫地机器人 |
| **厨房** (v1.3) | 微波炉、烤箱 |
| **洗衣** (v1.3) | 洗衣机、烘干机 |
| **能源** (v1.3) | EV 充电桩、能源管理 |
| **桥接** | 将非 Matter 设备接入 |

## 集群

集群是设备能力的标准化描述：

### 通用集群

| 集群 | 功能 |
|------|------|
| **Identify** | 让设备闪烁——"我在这里" |
| **Groups** | 群组控制——"所有楼下灯" |
| **Scenes** | 场景——"电影模式"=灯暗+窗帘关 |
| **OtaSoftwareUpdate** | 固件升级 |
| **Diagnostics** | 设备诊断信息 |

### 功能集群

| 集群 | 功能 | 属性示例 |
|------|------|------|
| **OnOff** | 开关 | OnOff (bool) |
| **LevelControl** | 亮度/速度 | CurrentLevel (0-254) |
| **ColorControl** | 颜色 | Hue, Saturation, ColorTemp |
| **DoorLock** | 门锁 | LockState, 支持密码/PIN |
| **Thermostat** | 恒温器 | TargetTemp, CurrentTemp, Mode |
| **WindowCovering** | 窗帘 | Position, Tilt |
| **AirQuality** | 空气质量 | PM2.5, VOC, CO2 |
| **Energy** | 能源 | Power, Energy, Voltage |

### 例子：一盏可调光彩色灯

```
设备类型: Extended Color Light
  ├── OnOff Cluster          (开关)
  ├── LevelControl Cluster   (亮度 1-100%)
  ├── ColorControl Cluster   (颜色和色温)
  ├── Identify Cluster       (闪烁定位)
  ├── Groups Cluster         (群组)
  ├── Scenes Cluster         (场景)
  └── OtaSoftwareUpdate      (升级)
```

> 这就是 Matter 的互操作原理——只要设备声明了它支持的集群，任何 Controller 都能理解和控制它。
