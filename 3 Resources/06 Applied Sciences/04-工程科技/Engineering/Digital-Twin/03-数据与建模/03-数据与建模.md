# 03-数据与建模

> 数据是数字孪生的血液，模型是骨架。没有高质量的数据流和正确语义的模型，数字孪生只是一个空壳。

## 数据流的挑战

| 挑战 | 说明 |
|------|------|
| **异构性** | 不同设备协议不同、格式不同、采样率不同 |
| **数据质量** | 传感器漂移、丢包、异常值、时间戳不齐 |
| **海量存储** | 一个工厂每天产生 TB 级数据 |
| **时效性** | 仿真需要 ms 级数据，历史分析可以用小时级 |

### 数据预处理流水线

```
原始数据 → 去噪 → 对齐(时间戳统一) → 填补缺失 → 异常检测 → 特征提取
```

| 步骤 | 方法 |
|------|------|
| 去噪 | 低通滤波、中值滤波、卡尔曼滤波 |
| 时间对齐 | 重采样到统一频率 |
| 缺失填补 | 线性插值、模型预测填充 |
| 异常检测 | 统计 (3σ)、ML (Isolation Forest) |

## 建模方法

### 三类模型

| 类型 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **白盒 (物理模型)** | 基于物理定律（热力学、力学方程） | 可解释、可外推 | 需要深入理解物理、慢 |
| **黑盒 (数据驱动)** | 从数据中学习（神经网络、随机森林） | 不需要物理知识、快 | 不可解释、训练数据外不可靠 |
| **灰盒 (混合)** | 物理模型+数据驱动校正 | 结合两者优点 | 设计复杂 |

### 混合模型——多数的现实选择

```
物理模型 (简化) → 输出 → 与实测偏差 → 数据模型(校正偏差) → 最终输出
```

> 纯物理模型太慢，纯 ML 在新工况下不可靠。混合模型是工程中的主流方案。

## 语义模型

为什么需要语义模型？因为"值=85"本身没有意义。

| 没有语义 | 有语义 |
|----------|--------|
| value=85 | 3 号产线灌装站出口温度 = 85°C，范围 80-95°C，传感器 PT100 |
| status=1 | 设备状态=正常运行 |
| val=4.2 | 振动加速度 RMS=4.2 mm/s²，ISO 10816-3 标准：Zone B (可接受) |

### Asset Administration Shell (AAS)

工业 4.0 的核心信息模型——每个资产都有一个标准的数字描述：

```
AAS (水泵 #123)
├── 标识: 厂商/型号/序列号
├── 技术规格: 功率 15kW、扬程 30m
├── 实时数据: 电流、振动、温度、流量
├── 文档: 操作手册、维护记录
└── 关系: 属于"水处理系统"，由 PLC #5 控制
```
