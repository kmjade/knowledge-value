# 09-工业物联网与智能制造

> 工业物联网 (IIoT) 将传感器、PLC 和云端连接——从"自动化"到"智能化"。智能制造是工业 4.0 的落地——数据驱动的生产优化。

## IIoT 架构

```
┌─────────────────────────────┐
│         云端/数据中心         │  AI分析、数字孪生、ERP集成
├─────────────────────────────┤
│         边缘层               │  协议转换、数据预处理、本地决策
├─────────────────────────────┤
│         现场层               │  PLC、传感器、执行器
└─────────────────────────────┘
```

## 从 OT 到 IT 的融合

| 维度 | OT (运营技术) | IT (信息技术) |
|------|-------------|-------------|
| 关注 | 可用性、安全 | 数据、效率 |
| 周期 | 设备运行 10-30 年 | 系统更新 3-5 年 |
| 协议 | Modbus, Profinet | HTTP, MQTT |
| 安全 | 物理隔离 | 防火墙、加密 |

> IT/OT 融合的挑战：IT 想给 PLC 打补丁，OT 说"别碰——它在控制生产线"。两种文化要融合。

## 关键技术

### MQTT——IIoT 的消息协议

| 特性 | 说明 |
|------|------|
| 发布/订阅 | 解耦数据生产者和消费者 |
| 轻量 | 最小报文 2 字节 |
| QoS | 0(最多一次)/1(至少一次)/2(恰好一次) |
| 遗嘱消息 | 设备断连时自动通知 |

```
温度传感器 → 发布 "temp/line1"=85°C
                                     ↓
                         MQTT Broker (Mosquitto/EMQX)
                                     ↓
                     订阅者: SCADA / 云平台 / 告警系统
```

### 数字孪生

物理产线的实时虚拟镜像——不只是 3D 模型，是**实时数据驱动**的动态仿真。

| 应用 | 价值 |
|------|------|
| 虚拟调试 | 在实际 PLC 到货前用数字孪生调试程序 |
| 预测性维护 | 在设备真正坏之前预警 |
| 优化 | 在不干扰生产的前提下测试新参数 |
| 培训 | 新操作员在数字孪生上练习 |

### 预测性维护

```
传感器数据 → 特征提取 → ML 模型 → 剩余使用寿命 (RUL) 预测
```

| 维护策略 | 说明 |
|----------|------|
| 事后维修 | 坏了再修 |
| 定期维护 | 按时间/周期修——可能过早或过晚 |
| 状态监测 | 根据实时状态判断 |
| **预测性维护** | 预测"什么时候会坏"——最优 |

## 智能制造的应用场景

| 场景 | 说明 |
|------|------|
| OEE 监控 | 设备综合效率——可用性×性能×质量 |
| 质量预测 | 用过程参数预测产品质量，而非等检测结果 |
| 能耗优化 | 找出能耗异常，优化运行参数 |
| 柔性制造 | 快速切换产品——小批量多品种 |
| 追溯 | 每一件产品的完整生产过程记录 |
