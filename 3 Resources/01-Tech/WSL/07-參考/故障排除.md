# 故障排除

> [!info] 說明
> WSL 常見問題的診斷與解決方案。

## 診斷工具

### 收集診斷資訊

```powershell
# 產生診斷報告
wsl --diagnostic-info

# 查看 WSL 狀態
wsl --status

# 查看版本資訊
wsl --version
```

### 查看日誌

```bash
# 在 WSL 中查看系統日誌
journalctl -xe

# 查看 dmesg
dmesg | tail -50
```

### Windows 事件日誌

```powershell
# 查看 WSL 相關事件
Get-WinEvent -LogName "Microsoft-Windows-Subsystem-Linux/Operational" |
    Select-Object TimeCreated, Message -First 20
```

## 安裝問題

### WSL 功能啟用失敗

**症狀**: `wsl --install` 失敗或功能無法啟用

**解決方案**:

```powershell
# 方法一：使用 DISM
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# 方法二：使用 PowerShell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux -NoRestart
Enable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform -NoRestart

# 重新啟動電腦
```

### 虛擬化未啟用

**症狀**: 錯誤 0x80370102

**解決方案**:

1. 進入 BIOS/UEFI 設定
2. 啟用虛擬化技術：
   - Intel: VT-x / Intel Virtualization Technology
   - AMD: AMD-V / SVM Mode
3. 儲存並重新啟動

### WSL 2 核心安裝失敗

**症狀**: 錯誤 0x800701bc

**解決方案**:

```powershell
# 下載並手動安裝
Invoke-WebRequest -Uri "https://wslstorestorage.blob.core.windows.net/wslblob/wsl_update_x64.msi" -OutFile "wsl_update.msi"
Start-Process msiexec.exe -ArgumentList "/i wsl_update.msi /quiet" -Wait
```

## 效能問題

### WSL 執行緩慢

**診斷**:

```bash
# 檢查檔案系統位置
pwd
# 如果在 /mnt/c/，效能會較慢

# 檢查 WSL 版本
wsl --list --verbose
```

**解決方案**:

1. 將專案移到 Linux 檔案系統
2. 確保使用 WSL 2
3. 調整 .wslconfig:

```ini
[wsl2]
memory=8GB
processors=4

[experimental]
autoMemoryReclaim=gradual
```

### 記憶體不足

**症狀**: WSL 當機或執行緩慢

**解決方案**:

```ini
# .wslconfig
[wsl2]
memory=4GB
swap=2GB
```

### 磁碟空間問題

**診斷**:

```bash
# 查看磁碟使用
df -h

# 查看大檔案
find / -type f -size +100M 2>/dev/null | head -20
```

**解決方案**:

```bash
# 清理套件快取
sudo apt clean
sudo apt autoremove

# 清理 Docker
docker system prune -a
```

```powershell
# 壓縮虛擬磁碟
wsl --shutdown
diskpart
select vdisk file="path\to\ext4.vhdx"
compact vdisk
exit
```

## 網路問題

### 無法連線網路

**診斷**:

```bash
# 檢查 DNS
cat /etc/resolv.conf

# 測試連線
ping google.com
curl -I https://www.google.com
```

**解決方案**:

```bash
# 修正 DNS
sudo rm /etc/resolv.conf
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
sudo chattr +i /etc/resolv.conf
```

### 本地服務無法從 Windows 存取

**症狀**: WSL 中啟動的服務無法從 Windows 瀏覽器存取

**解決方案**:

```bash
# 確保服務監聽所有介面
python -m http.server 8000 --bind 0.0.0.0

# 或使用 localhost (WSL 2 會自動轉送)
python -m http.server 8000
```

### VPN 問題

**症狀**: 啟用 VPN 後 WSL 無法連線

**解決方案**:

1. 設定 VPN Split Tunneling
2. 或手動設定 DNS:

```bash
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
```

## 檔案系統問題

### 權限問題

**症狀**: Permission denied

**解決方案**:

```bash
# 檢查權限
ls -la /mnt/c/file

# 啟用 metadata 選項
# /etc/wsl.conf
[automount]
options = "metadata,umask=22,fmask=11"

# 重新啟動 WSL
```

### 檔案鎖定

**症狀**: 無法刪除或修改檔案

**解決方案**:

```bash
# 查看誰在使用
lsof /path/to/file
fuser -v /path/to/file

# 終止程序
fuser -k /path/to/file
```

### 區分大小寫問題

**症狀**: Git 或編譯問題

**解決方案**:

```powershell
# 啟用目錄區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\projects\myproject" enable
```

## 發行版問題

### 發行版無法啟動

**診斷**:

```powershell
wsl --list --verbose
```

**解決方案**:

```powershell
# 重新註冊
wsl --unregister Ubuntu
wsl --install -d Ubuntu
```

### 發行版損壞

**解決方案**:

```powershell
# 從備份還原
wsl --import Ubuntu-Restored D:\WSL\ backup.tar
```

## systemd 問題

### systemd 未啟動

**診斷**:

```bash
systemctl status
ps -p 1 -o comm=
```

**解決方案**:

```ini
# /etc/wsl.conf
[boot]
systemd=true
```

```powershell
wsl --shutdown
wsl
```

### 服務無法啟動

**解決方案**:

```bash
# 查看日誌
journalctl -xeu service-name

# 重新啟用服務
sudo systemctl daemon-reload
sudo systemctl enable service-name
sudo systemctl start service-name
```

## 重置 WSL

### 完全重置

```powershell
# 停止 WSL
wsl --shutdown

# 取消註冊所有發行版
wsl --unregister Ubuntu
# ... 其他發行版

# 重新安裝
wsl --install
```

### 重置特定發行版

```powershell
# 匯出設定
wsl --export Ubuntu backup.tar

# 重置
wsl --unregister Ubuntu

# 從備份還原
wsl --import Ubuntu D:\WSL\Ubuntu backup.tar
```
