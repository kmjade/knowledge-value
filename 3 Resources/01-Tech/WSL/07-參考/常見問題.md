# 常見問題

> [!info] 說明
> WSL 常見問題與解答。

## 一般問題

### WSL 是什麼？

WSL (Windows Subsystem for Linux) 是 Windows 的功能，可讓您直接在 Windows 上執行 Linux 環境，不需要虛擬機器或雙重開機。

### WSL 1 和 WSL 2 有什麼差異？

| 特性 | WSL 1 | WSL 2 |
|------|-------|-------|
| 架構 | 翻譯層 | 真實 Linux 核心 |
| 效能 | 跨系統較快 | Linux 內部極快 |
| Docker | 有限支援 | 完整支援 |
| 系統呼叫 | 部分 | 完整 |

### WSL 免費嗎？

是的，WSL 是 Windows 的內建功能，免費使用。

### WSL 可以執行 GUI 應用程式嗎？

可以，WSL 2 支援 Linux GUI 應用程式 (WSLg)。

## 安裝問題

### 安裝 WSL 需要什麼系統需求？

- Windows 10 版本 2004+ (組建 19041)
- Windows 11
- 啟用虛擬化功能的 CPU

### 如何安裝 WSL？

```powershell
wsl --install
```

### 安裝失敗怎麼辦？

```powershell
# 手動啟用功能
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart

# 重新啟動電腦後安裝
```

### 可以離線安裝 WSL 嗎？

可以，需要下載：
1. WSL2 Linux 核心更新套件
2. Linux 發行版 appx 檔案

## 使用問題

### 如何從 Windows 存取 Linux 檔案？

```
\\wsl$\Ubuntu\home\username
```

### 如何從 Linux 存取 Windows 檔案？

```bash
cd /mnt/c/Users/username
```

### 專案應該放在哪裡？

建議放在 Linux 檔案系統中 (`/home/username/projects/`) 以獲得最佳效能。

### 如何設定 Git 憑證？

```bash
# 使用 Windows 憑證管理員
git config --global credential.helper "/mnt/c/Program\ Files/Git/mingw64/bin/git-credential-manager.exe"
```

### 如何設定 SSH 金鑰？

```bash
ssh-keygen -t ed25519 -C "your@email.com"
cat ~/.ssh/id_ed25519.pub | clip.exe
```

## 效能問題

### WSL 執行緩慢怎麼辦？

1. 確保使用 WSL 2
2. 將專案移到 Linux 檔案系統
3. 調整 .wslconfig 設定

```ini
[wsl2]
memory=8GB
processors=4
```

### 如何限制 WSL 記憶體使用？

```ini
# %UserProfile%\.wslconfig
[wsl2]
memory=4GB
```

### 磁碟空間太大怎麼辦？

```powershell
# 壓縮虛擬磁碟
wsl --shutdown
diskpart
select vdisk file="path\to\ext4.vhdx"
compact vdisk
exit
```

## 網路問題

### 無法從 WSL 連線網路怎麼辦？

```bash
# 檢查 DNS
cat /etc/resolv.conf

# 手動設定 DNS
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
```

### 如何從外部網路連線到 WSL 服務？

```powershell
# 設定連接埠轉送
netsh interface portproxy add v4tov4 listenport=80 listenaddress=0.0.0.0 connectport=80 connectaddress=<WSL_IP>
```

### VPN 下 WSL 無法連線？

設定 VPN 的 Split Tunneling，排除 WSL 網路。

## 發行版管理

### 如何安裝多個發行版？

```powershell
wsl --install -d Ubuntu-22.04
wsl --install -d Debian
```

### 如何切換發行版？

```powershell
wsl -d Ubuntu-22.04
```

### 如何設定預設發行版？

```powershell
wsl --set-default Ubuntu-22.04
```

### 如何備份發行版？

```powershell
wsl --export Ubuntu backup.tar
```

### 如何刪除發行版？

```powershell
wsl --unregister Ubuntu
```

## 開發問題

### VS Code 無法連線 WSL？

1. 安裝 WSL 擴充功能
2. 關閉並重新開啟 VS Code
3. 從 WSL 內執行 `code .`

### Docker 無法在 WSL 中使用？

1. 安裝 Docker Desktop for Windows
2. 啟用 WSL 2 後端
3. 在 Docker Desktop 設定中啟用 WSL 整合

### Python 開發環境如何設定？

```bash
# 使用 pyenv
curl https://pyenv.run | bash
pyenv install 3.12.0
pyenv global 3.12.0
```

### Node.js 開發環境如何設定？

```bash
# 使用 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install --lts
```

## 錯誤代碼

### 錯誤 0x8007019e

WSL 功能未啟用。

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
```

### 錯誤 0x800701bc

WSL 2 核心未安裝。

下載並安裝 [WSL2 Linux 核心更新套件](https://aka.ms/wsl2kernel)。

### 錯誤 0x80370102

虛擬化功能未啟用。

在 BIOS 中啟用 Intel VT-x 或 AMD-V。

### 錯誤 0x80004005

一般性錯誤。

```powershell
wsl --shutdown
wsl
```
