# OpenCode TUI 终端界面

> [!info] 概述
> TUI（Terminal User Interface）是 OpenCode 的核心使用方式，所有代码分析、修改、执行都通过这个界面完成。

## 核心概念

OpenCode TUI 本质是一个**可执行命令的 AI 对话终端**，把开发、命令行和 AI 融合在一起。

### 交互语法

| 语法 | 说明 |
|------|------|
| `/` | 控制行为（类似 IDE 命令面板） |
| `@` | 注入上下文（文件级 RAG） |
| `!` | 执行环境操作（Shell 工具调用） |
| `ctrl+x` | 高效快捷操作前缀 |

---

## 启动 TUI

### 当前目录

```bash
opencode
```

### 指定目录

```bash
opencode /path/to/project
```

---

## TUI 命令一览

在输入框输入 `/` 会列出所有命令：

| 命令 | 作用 | 快捷键 |
|------|------|--------|
| `/connect` | 添加 LLM 提供商并配置 API Key | — |
| `/compact` | 压缩当前会话上下文 | ctrl+x → c |
| `/details` | 切换工具执行详情显示 | ctrl+x → d |
| `/editor` | 打开外部编辑器编写消息 | ctrl+x → e |
| `/exit` | 退出 TUI | ctrl+x → q |
| `/export` | 导出当前对话为 Markdown | ctrl+x → x |
| `/help` | 打开帮助面板 | ctrl+x → h |
| `/init` | 创建/更新 AGENTS.md | ctrl+x → i |
| `/models` | 列出可用模型 | ctrl+x → m |
| `/new` | 新建会话 | ctrl+x → n |
| `/redo` | 重做被 `/undo` 撤销的操作 | ctrl+x → r |
| `/sessions` | 会话列表与切换 | ctrl+x → l |
| `/share` | 分享当前会话 | ctrl+x → s |
| `/themes` | 列出/切换主题 | ctrl+x → t |
| `/thinking` | 显示/隐藏模型推理过程 | — |
| `/undo` | 撤销上一条消息（含文件变更） | ctrl+x → u |
| `/unshare` | 取消分享会话 | — |

## 核心交互

### 1. 文件引用 `@`

使用 `@` 引用文件内容：

```
@main.py 解释这个文件
@src/config.json 这个配置是什么意思
@README.md 帮我改进这个文档
```

**特点：**
- 模糊搜索文件名
- 自动注入文件内容到上下文
- 支持多个文件同时引用

### 2. 执行命令 `!`

使用 `!` 执行 shell 命令：

```
!git status
!npm install
!python test.py
!ls -la
```

**常用场景：**
- 查看项目状态
- 安装依赖
- 运行测试
- 构建项目

### 3. 自然语言对话

直接输入问题或指令：

```
帮我分析这个项目的架构
优化这个函数的性能
添加单元测试
```

## 启动参数

| 标志 | 简写 | 说明 |
|------|------|------|
| `--continue` | `-c` | 继续上一次会话 |
| `--session` | `-s` | 指定会话 ID |
| `--fork` | — | 分叉会话 |
| `--prompt` | — | 初始提示词 |
| `--model` | `-m` | 指定模型 |
| `--agent` | — | 指定代理 |
| `--port` | — | 监听端口 |
| `--hostname` | — | 主机地址 |

## 会话管理

### 查看会话列表

```
/sessions
```

### 继续上次会话

```bash
opencode --continue
```

### 新建会话

```
/new
```

## 实用技巧

### 快速项目分析

```
帮我快速总结这个项目的结构
```

### 代码重构

```
@old_code.py 重构这个文件，提取公共函数
```

```
!npm run build

```

```
为 @main.py 编写单元测试
```
