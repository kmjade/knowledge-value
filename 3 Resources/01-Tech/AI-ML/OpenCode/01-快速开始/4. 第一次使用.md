# OpenCode 第一次使用

> [!info] 概述
> 完成 API 配置后，开始你的第一次 OpenCode 体验。

## 启动 OpenCode

### 在当前目录启动

```bash
opencode
```

### 指定项目目录

```bash
opencode /path/to/project
```

---

## 基本界面

启动后会看到 TUI（终端用户界面）：


## 第一次对话

### 示例 1：项目分析

```
帮我快速总结这个项目的结构
```

OpenCode 会分析项目并给出结果。

### 示例 2：代码解释

```
解释一下 main.py 的核心功能
```

### 示例 3：文件引用

```
@main.py 这个函数是做什么的？
```

使用 `@` 符号引用特定文件。

### 示例 4：执行命令

```
!ls -la
```

使用 `!` 执行 shell 命令并获取结果。

---

## 常用斜杠命令

在输入框中输入 `/` 会列出可用命令：

| 命令 | 作用 | 快捷键 |
|------|------|--------|
| `/connect` | 配置 API Key | — |
| `/model` | 切换模型 | ctrl+x → m |
| `/help` | 打开帮助 | ctrl+x → h |
| `/init` | 创建 AGENTS.md | ctrl+x → i |
| `/new` | 新建会话 | ctrl+x → n |
| `/exit` | 退出 TUI | ctrl+x → q |

---

## 创建项目规则

使用 `/init` 命令创建 AGENTS.md：

```
/init
```

这会：
- 扫描项目结构
- 分析代码组织方式
- 自动生成 AGENTS.md

> [!tip] 建议
> 将 AGENTS.md 提交到 Git，作为团队统一规范。

---

## 核心交互语法

### 文件引用 `@`

```
@src/main.py 解释这个文件
@config.json 这个配置是什么意思
```

模糊搜索并注入文件内容。

### 执行命令 `!`

```
!git status
!npm test
!python main.py
```

执行 shell 命令并返回结果。

### 斜杠命令 `/`

```
/model
/help
/init
```

控制 OpenCode 行为。

---

## 会话管理

### 新建会话

```
/new
```

或使用快捷键 `ctrl+x → n`

### 继续上次会话

```bash
opencode --continue
# 或
opencode -c
```

### 查看会话列表

```
/sessions
```

---

## 退出 OpenCode

```
/exit
```

或使用快捷键 `ctrl+x → q`
