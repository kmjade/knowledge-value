# OpenCode 简介

> [!info] 核心定位
> OpenCode 是一款开源 AI 编程协作工具，定位为基于命令行（CLI）的 AI 编程 Agent，**运行在终端中的 AI 程序员助手**。

## 核心能力

OpenCode 脱离网页聊天框的局限，直接在终端（Terminal）中运行：

| 能力 | 说明 |
|------|------|
| 读取整个项目代码 | 全面覆盖项目文件，无需手动粘贴代码片段 |
| 理解文件依赖关系 | 精准识别模块、函数间的关联，把握项目整体结构 |
| 直接修改代码文件 | 无需人工介入，根据指令完成代码编辑 |
| 执行开发命令 | 支持安装依赖、运行测试等操作，打通开发全流程 |
| 迭代优化代码 | 根据命令执行结果，自动调整优化方案 |

## 三大核心特征

### 1. 上下文感知
突破单文件理解局限，深度掌握整个项目的结构与逻辑。

### 2. 执行能力
不止提供代码建议，更能直接修改文件、执行命令，落地开发动作。

### 3. 工程化导向
聚焦代码规范、结构合理性、可维护性及测试覆盖，贴合实际开发需求。

---

## 与其他工具的区别

| 工具类型 | 特点 | 局限性 |
|----------|------|--------|
| 聊天式 AI | 提供代码后给出建议 | 所有修改需手动完成 |
| 代码补全工具 | 辅助补全当前代码片段 | 无项目级处理能力 |
| **OpenCode** | 自主读取项目、修改代码、执行命令 | 全程参与开发流程 |

**比喻说明：**
- 聊天式 AI = 技术顾问
- 代码补全工具 = 智能输入法
- OpenCode = 并肩开发的搭档

## OpenCode 能做什么？

### 1. 代码理解与解释
- 这个函数的核心功能是什么？
- 此处报错的原因及解决方案是什么？
- 这段代码的性能瓶颈在哪里？

### 2. 多文件上下文分析
- 函数的完整调用链路
- 模块之间的依赖关系
- 项目的整体结构

### 3. 工程级代码修改
- 将项目中所有 `var` 声明替换为 `let`
- 为所有接口添加统一的错误处理逻辑
- 将指定函数拆分为多个独立模块

### 4. 执行开发任务
- 安装项目依赖（npm / pip 等）
- 启动项目，实时反馈运行状态
- 执行测试用例，定位测试问题
- 根据执行结果自动优化代码

---

## OpenCode 不能做什么

- 无法替代人类做出最终的技术架构与设计决策
- 不能保证生成或修改的代码完全无 Bug，需人工校验
- 无法理解未明确说明的业务逻辑，需清晰传达需求
- 不适合完全脱离人类干预的全自动开发，需协同配合

> [!tip] 正确使用方式
> 让 OpenCode 提供高质量开发方案，结合人工判断校验结果，实现高效协作。

## 核心价值

### 理念：协作，而不是替代
- **人负责**：明确开发目标、制定设计方案、做出关键判断
- **AI 负责**：执行繁琐开发操作、分析代码问题、处理重复工作

### 对新手
- 用通俗语言解释复杂代码逻辑
- 快速定位代码错误并给出解决方案
- 提供规范写法建议，帮助养成良好编程习惯

### 对独立开发者
- 快速搭建项目基础功能，缩短开发周期
- 自动生成测试用例，提升代码可靠性
- 优化旧代码、重构冗余逻辑

### 对团队
- 统一代码风格与规范，减少 Code Review 成本
- 辅助完成 Code Review，快速识别潜在问题
- 帮助新人快速熟悉项目结构
