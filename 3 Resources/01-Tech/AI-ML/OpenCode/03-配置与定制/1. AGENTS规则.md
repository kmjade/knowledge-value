# OpenCode AGENTS 规则

> [!info] 概述
> **AGENTS.md = 给 AI 制定团队开发规范**
> 工具决定 AI 能做什么，AGENTS.md 决定 AI 应该怎么做。

## 什么是 AGENTS.md

AGENTS.md 是一个规则文件，用于：
- 定义项目结构
- 约束代码风格
- 规范开发流程
- 指导 AI 如何执行任务

这些内容会被加入到 LLM 的上下文中。

**效果：**
- AI 更懂你的项目
- 生成的代码更符合规范
- 减少反复修改

---

## 快速初始化

在 OpenCode 中运行：

```
/init
```

该命令会：
- 扫描项目结构
- 分析代码组织方式
- 自动生成 AGENTS.md

> [!tip] 建议
> 将 AGENTS.md 提交到 Git，作为团队统一规范。

---

## 示例结构

```markdown
# 项目说明

这是一个基于 TypeScript 的 monorepo 项目。

## 项目结构

- packages/ 核心模块
- infra/ 基础设施
- web/ 前端应用

## 代码规范

- 使用 TypeScript 严格模式
- 公共代码放在 packages/core
- 所有函数必须添加注释

## 开发约定

- 使用 workspace 方式引用模块
- API 必须统一错误处理
```

**本质：把团队规范"写给 AI 看"**

---

## 规则的作用范围

### 项目级规则

**路径：** `./AGENTS.md`

**作用：**
- 仅当前项目生效
- 团队共享

### 全局规则

**路径：** `~/.config/opencode/AGENTS.md`

**作用：**
- 对所有项目生效
- 个人使用

**建议用途：**
- 个人编码习惯
- 输出风格控制

---

## Claude Code 兼容性

OpenCode 兼容 Claude Code 的规则体系：

| 类型 | OpenCode | Claude Code |
|------|----------|-------------|
| 项目规则 | `./AGENTS.md` | `./CLAUDE.md` |
| 全局规则 | `~/.config/opencode/AGENTS.md` | `~/.claude/CLAUDE.md` |
| 技能目录 | — | `~/.claude/skills/` |

### 禁用 Claude Code 兼容

```bash
export OPENCODE_DISABLE_CLAUDE_CODE=1
```

## AGENTS.md 最佳实践

### 1. 项目描述

```markdown
# 项目描述

[项目名称] 是一个 [项目类型]，用于 [主要功能]。

技术栈：
- 前端：React + TypeScript
- 后端：Node.js + Express
- 数据库：PostgreSQL
```

### 2. 目录结构说明

```markdown
## 目录结构

src/
├── components/   # UI 组件
├── hooks/        # 自定义 Hooks
├── services/     # API 服务
├── utils/        # 工具函数
└── types/        # 类型定义
```

### 3. 代码规范

```markdown
## 代码规范

### 命名规范
- 组件：PascalCase
- 函数：camelCase
- 常量：UPPER_SNAKE_CASE

### 文件规范
- 一个文件一个组件
- 使用 index.ts 导出

### 注释规范
- 公共函数必须有 JSDoc
- 复杂逻辑添加行内注释
```

### 4. 开发约定

```markdown
## 开发约定

### Git 提交
- feat: 新功能
- fix: 修复 bug
- docs: 文档更新
- refactor: 重构

### 测试要求
- 新功能必须有单元测试
- 覆盖率要求 > 80%
```

## 常用模板

### 前端项目

```markdown
# 前端项目

## 技术栈
- React 18 + TypeScript
- Vite 构建工具
- Tailwind CSS

## 代码规范
- 使用函数组件
- 使用 Hooks
- 组件文件结构：
  - 样式
  - 组件
  - 类型
  - 测试
```

### 后端项目

```markdown
# 后端项目

## 技术栈
- Node.js + Express
- TypeScript
- PostgreSQL

## API 规范
- RESTful 风格
- 统一响应格式
- 错误处理中间件

## 安全要求
- 输入验证
- SQL 注入防护
- XSS 防护
```
