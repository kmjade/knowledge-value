# 通用Prompt模板

> [!info] 说明
> 这是一套通用 Prompt 模板，遵循"角色 + 目标 + 输入 + 约束 + 输出格式"的结构，提高AI响应的准确率与可控性。

---

## 模板概览

| 模板 | 用途 | 场景 |
|------|------|------|
| 结构化信息提取 & 汇总 | 从长文中抽取关键要点 | 会议纪要、报告、网页内容分析 |
| 创意/方案生成（头脑风暴） | 产生可执行的想法 | 产品点子、营销活动、写作构思 |
| 专家式深度解答（教学/辅导） | 系统化知识讲解 | 技术教程、案例分析、职业指导 |

---

## 模板一：结构化信息提取 & 汇总

> 适用于：从长文、报告、会议纪要、网页等非结构化文本中快速抽取关键要点、生成表格或清单。

### 模板内容

```
## 角色
你是一名擅长信息梳理的专业分析师（或编辑/研究员），请以简洁、条理化的方式呈现信息。

## 目标
从下面提供的原始文本中提取指定的信息，并以结构化的形式输出。

## 输入（请粘贴原始文本）
【在此处粘贴待处理的长文本】

## 约束
1. 只关注下面列出的信息字段（如有遗漏请注明"未出现"）
2. 每个字段的内容请保持 **原文核心意义**，不添加主观评价。
3. 如文本中出现时间、金额、地点、角色等数据，请统一使用 **ISO8601 日期、千分位数字、标准英文地名**（如适用）。
4. 输出使用 Markdown 表格或有序列表，保持可复制/粘贴的格式。

## 信息字段（可自行增删）
- 事件/核心议题
- 关键时间点（日期+时间，ISO8601）
- 参与者/角色
- 决策/结论
- 关键数值（金额、数量、百分比等）
- 后续行动/待办

## 输出格式（示例）
| 事件/议题 | 时间 | 参与者 | 决策/结论 | 关键数值 | 待办 |
|----------|------|--------|----------|----------|------|
| …        | …    | …      | …        | …        | …    |
```

### 使用示例
- 会议纪要 → 只要把纪要粘进去，模型会返回一张"一目了然"的行动表。
- 产品需求文档 → 提取"功能点、优先级、负责人、交付时间"。
- 法律法规原文 → 抽取"适用范围、处罚标准、执行日期"。

## 模板二：创意/方案生成（头脑风暴）

> 适用于：产品点子、营销活动、写作构思、业务改进等需要多元思考的场景。

### 模板内容

```
## 角色
你是拥有丰富行业经验的创新顾问（或创意总监、策划师），擅长产生大量可执行的想法并提供简要实现路径。

## 目标
围绕「【主题】」产生不少于 **N** 条创意/方案，每条都包括核心概念、亮点、潜在受众、实现要点及风险点。

## 输入
- 主题/需求描述：【在此处填写】
- 关键限制或已有资源（可选）：【如预算上限、技术栈、时间窗口等】

## 约束
1. 创意必须 **可执行**（不接受纯概念性、技术不可实现的想法）。
2. 每条创意不超过 **3 句核心描述** + **1 行实现要点**。
3. 按 **类别** 或 **层级** 列出（例如：① 市场营销 ② 产品功能 ③ 用户运营）。
4. 若有明显风险，请在每条结尾标注 **[风险]** 并简要说明。

## 输出格式（示例）
### 市场营销
1️⃣ **概念**：社媒挑战赛+UGC抽奖
   **亮点**：利用短视频平台的病毒式传播，提升品牌曝光。
   **受众**：18‑30 岁年轻消费群体。
   **实现要点**：设计挑战主题 → 制作官方模板 → 设置抽奖规则 → 投放广告。
   **[风险]**：奖品成本超预算 → 控制奖品价值。

### 产品功能
2️⃣ **概念**：一键生成个性化报告
   **亮点**：用户只需上传数据，系统自动输出 PDF。
   **受众**：B2B 客户经理。
   **实现要点**：集成模板引擎 + 后端数据处理 + PDF 导出。
   **[风险]**：数据安全合规，需加密存储。
```

### 使用技巧
- N 可以写成 "5 条" 或 "10 条"，模型会自动满足或略超。
- 若想强调 **创新性**，在 "约束" 中加入 "鼓励跨行业借鉴"。
- 对于特定行业（如教育、金融），把 "角色" 换成对应的行业顾问即可获得更专业的想法。

## 模板三：专家式深度解答（教学/辅导）

> 适用于：需要系统化、层次分明的知识讲解、技术教程、案例分析等。

### 模板内容

```
## 角色
你是一位资深【领域】专家（如机器学习、财税、法律、心理学），擅长把复杂概念拆解成易懂的步骤，并配合案例、图示和常见误区。

## 目标
对下面的提问进行 **完整、系统** 的解答，帮助提问者快速掌握核心要点并能实际操作。

## 输入（提问/需求）
【在此处粘贴提问或需要解释的概念】

## 约束
1. 先给出 **简要概述**（1‑2 行），随后展开 **3‑5 层级** 的详细阐述。
2. 每层级使用 Markdown 标题（`##、###、####`），层次分明。
3. 包含以下可选模块（视需求自行保留）
   - **关键概念定义**（列表）
   - **步骤/流程**（有序列表）
   - **实际案例**（简短情境 + 结果）
   - **常见误区**（⚠️ 标记）
   - **延伸阅读/工具**（链接或推荐）
4. 文字尽量 **通俗易懂**，但必要时保留专业术语并给出解释。
5. 总结部分不超过 **3 行**，给出行动建议或复习要点。

## 输出格式（示例）
### 简要概述
（1‑2 行摘要）

## 1. 基础概念
- **概念 A**：定义……
- **概念 B**：定义……

## 2. 核心原理/流程
### 2.1 步骤一
1. ……
2. ……

### 2.2 步骤二
……

## 3. 实际案例
> **案例**：某公司在使用 X 方法时 ……

## 4. 常见误区
- ⚠️ **误区 1**：描述……
- ⚠️ **误区 2**：描述……

## 5. 延伸阅读 / 推荐工具
- 链接1：…
- 工具 A：…

### 行动建议
- ……
- ……
```

### 使用示例
- 技术学习：`领域 = "前端开发（React）"`，提问 "React 中的 Hook 原理"。
- 法律解读：`领域 = "合同法"`，提问 "违约金的计算方式"。
- 职业发展：`领域 = "数据分析"`，提问 "如何从零开始成为数据分析师"。

---

## 使用技巧

| 步骤 | 操作 |
|------|------|
| 1️⃣ 复制对应模板 | 将上面的完整文本（包括 `## 角色`、`## 目标`…）复制到聊天框。 |
| 2️⃣ 替换占位符 | 用自己的实际信息替换 `【主题】`、`【领域】`、`【提问】`、`N` 等。 |
| 3️⃣ 发起请求 | 发送后模型会自动遵循约束，返回结构化、可直接使用的结果。 |
| 4️⃣ 二次迭代（可选） | 若需要补充细节，可在结果上继续追问，例如 "把案例改成金融行业的"。 |
| 5️⃣ 保存模板 | 把这三个模板保存为 Obsidian / Notion / 记事本 中的快捷片段，下次直接调用。 |

> 小技巧：在 Obsidian、Logseq 等支持 **模板** 的笔记工具里，把这三个模板分别保存为 `Prompt-提取信息.md`、`Prompt-创意头脑风暴.md`、`Prompt-专家解答.md`，配合 QuickAdd 插件，一键弹出编辑框，效率瞬间提升！
