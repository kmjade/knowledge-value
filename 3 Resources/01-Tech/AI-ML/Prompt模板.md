# 通用Prompt模板

> [!info] 說明
> 这是一套通用 Prompt 模板，遵循"角色 + 目標 + 輸入 + 约束 + 輸出格式"的结构，提高AI響應的准确率与可控性。

---

## 模板概览

| 模板 | 用途 | 場景 |
|------|------|------|
# 分析
| 创意/方案生成（头脑风暴） | 产生可执行的想法 | 产品点子、营销活动、写作构思 |
# 知識

---

## 模板一：结构化資訊提取 & 彙總

> 适用于：从长文、報告、會議纪要、網頁等非结构化文本中快速抽取關鍵要點、生成表格或清單。

### 模板內容

```
## 角色
# 分析

## 目標
从下面提供的原始文本中提取指定的資訊，并以结构化的形式輸出。

## 輸入（请貼上原始文本）
【在此处貼上待處理的长文本】

## 约束
1. 只關注下面列出的資訊字段（如有遗漏请注明"未出现"）
2. 每个字段的內容请保持 **原文核心意义**，不新增主观評價。
3. 如文本中出现時間、金额、地点、角色等數據，请統一使用 **ISO8601 日期、千分位数字、標準英文地名**（如适用）。
4. 輸出使用 Markdown 表格或有序列表，保持可複製/貼上的格式。

## 資訊字段（可自行增删）
- 事件/核心议题
- 關鍵時間点（日期+時間，ISO8601）
- 参与者/角色
- 決策/结论
- 關鍵数值（金额、数量、百分比等）
- 後續行动/待辦

## 輸出格式（示例）
| 事件/议题 | 時間 | 参与者 | 決策/结论 | 關鍵数值 | 待辦 |
|----------|------|--------|----------|----------|------|
| …        | …    | …      | …        | …        | …    |
```

### 使用示例
- 會議纪要 → 只要把纪要粘进去，模型会返回一张"一目了然"的行动表。
- 产品需求文檔 → 提取"功能点、優先級、负责人、交付時間"。
- 法律法规原文 → 抽取"适用范围、处罚標準、执行日期"。

## 模板二：创意/方案生成（头脑风暴）

> 适用于：产品点子、营销活动、写作构思、业务改进等需要多元思考的場景。

### 模板內容

```
## 角色
你是拥有豐富行业經驗的創新顾问（或创意总监、策划师），擅长产生大量可执行的想法并提供简要實現路徑。

## 目標
围绕「【主題】」产生不少于 **N** 条创意/方案，每条都包括核心概念、亮点、潜在受众、實現要點及风险点。

## 輸入
- 主題/需求描述：【在此处填寫】
- 關鍵限制或已有資源（可选）：【如预算上限、技術棧、時間視窗等】

## 约束
1. 创意必须 **可执行**（不接受纯概念性、技術不可實現的想法）。
2. 每条创意不超过 **3 句核心描述** + **1 行實現要點**。
3. 按 **类别** 或 **层级** 列出（例如：① 市场营销 ② 产品功能 ③ 使用者运营）。
4. 若有明显风险，请在每条结尾标注 **[风险]** 并简要說明。

## 輸出格式（示例）
### 市场营销
1️⃣ **概念**：社媒挑戰赛+UGC抽奖
   **亮点**：利用短視訊平台的病毒式传播，提升品牌曝光。
   **受众**：18‑30 岁年轻消费群体。
   **實現要點**：設計挑戰主題 → 制作官方模板 → 設置抽奖规则 → 投放广告。
   **[风险]**：奖品成本超预算 → 控制奖品價值。

### 产品功能
2️⃣ **概念**：一键生成个性化報告
   **亮点**：使用者只需上傳數據，系統自動輸出 PDF。
   **受众**：B2B 客户经理。
   **實現要點**：整合模板引擎 + 后端數據處理 + PDF 导出。
   **[风险]**：數據安全合规，需加密儲存。
```

### 使用技巧
- N 可以写成 "5 条" 或 "10 条"，模型会自動满足或略超。
- 若想强调 **創新性**，在 "约束" 中加入 "鼓励跨行业借鉴"。
- 对于特定行业（如教育、金融），把 "角色" 换成对应的行业顾问即可獲得更专业的想法。

## 模板三：专家式深度解答（教学/辅导）

# 知識

### 模板內容

```
## 角色
你是一位资深【领域】专家（如机器學習、财税、法律、心理学），擅长把复杂概念拆解成易懂的步骤，并配合案例、圖示和常见误区。

## 目標
对下面的提问進行 **完整、系統** 的解答，帮助提问者快速掌握核心要點并能实际操作。

## 輸入（提问/需求）
【在此处貼上提问或需要解釋的概念】

## 约束
1. 先给出 **简要概述**（1‑2 行），随后展开 **3‑5 层级** 的詳細阐述。
2. 每层级使用 Markdown 标题（`##、###、####`），层次分明。
3. 包含以下可选模块（视需求自行保留）
   - **關鍵概念定义**（列表）
   - **步骤/流程**（有序列表）
   - **实际案例**（简短情境 + 結果）
   - **常见误区**（⚠️ 標記）
   - **延伸阅读/工具**（連結或推荐）
4. 文字尽量 **通俗易懂**，但必要时保留专业术语并给出解釋。
5. 總結部分不超过 **3 行**，给出行动建議或复习要點。

## 輸出格式（示例）
### 简要概述
（1‑2 行摘要）

## 1. 基礎概念
- **概念 A**：定义……
- **概念 B**：定义……

## 2. 核心原理/流程
### 2.1 步骤一
1. ……
2. ……

### 2.2 步骤二
……

## 3. 实际案例
# 方法

## 4. 常见误区
- ⚠️ **误区 1**：描述……
- ⚠️ **误区 2**：描述……

## 5. 延伸阅读 / 推荐工具
- 連結1：…
- 工具 A：…

### 行动建議
- ……
- ……
```

### 使用示例
- 技術學習：`领域 = "前端開發（React）"`，提问 "React 中的 Hook 原理"。
- 法律解读：`领域 = "合同法"`，提问 "违约金的計算方式"。

## 使用技巧

| 步骤 | 操作 |
|------|------|
| 1️⃣ 複製对应模板 | 将上面的完整文本（包括 `## 角色`、`## 目標`…）複製到聊天框。 |
| 2️⃣ 替换占位符 | 用自己的实际資訊替换 `【主題】`、`【领域】`、`【提问】`、`N` 等。 |
| 3️⃣ 发起请求 | 發送后模型会自動遵循约束，返回结构化、可直接使用的結果。 |
| 4️⃣ 二次迭代（可选） | 若需要补充细节，可在結果上繼續追问，例如 "把案例改成金融行业的"。 |
| 5️⃣ 儲存模板 | 把这三个模板儲存为 Obsidian / Notion / 记事本 中的快捷片段，下次直接調用。 |
