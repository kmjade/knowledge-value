# OpenClaw 多智能体路由

> [!info] 概述
> 多智能体路由让不同类型的消息自动分配给最合适的代理处理。

## 路由概述

### 为什么需要路由

- 不同任务需要不同专长的代理
- 优化成本（小任务用小模型）
- 提高质量（专业代理更精准）

### 路由模式

| 模式 | 说明 |
|------|------|
| 手动切换 | 用户显式选择代理 |
| 关键词路由 | 根据关键词自动路由 |
| 意图识别 | AI 分析意图后路由 |
| 分层路由 | 主代理分发子任务 |

## 配置多代理

### 定义代理

```json
{
  "agents": {
    "general": {
      "provider": "anthropic",
      "model": "claude-sonnet-4-6",
      "systemPrompt": "你是一个通用助手。"
    },
    "coder": {
      "provider": "anthropic",

      "systemPrompt": "你是编程专家，精通多种编程语言。"
    },
    "writer": {
      "provider": "openai",
      "model": "gpt-4o",
      "systemPrompt": "你是专业的写作助手。"
    },
    "translator": {
      "provider": "deepseek",
      "model": "deepseek-chat",
      "systemPrompt": "你是专业翻译，精通中英日韩。"
    }
  }
}
```

## 关键词路由

### 配置路由规则

```json
{
  "router": {
    "mode": "keyword",
    "rules": [
      {
        "keywords": ["代码", "编程", "debug", "function", "class"],
        "agent": "coder"
      },
      {
        "keywords": ["翻译", "translate", "译成"],
        "agent": "translator"
      },
      {
        "keywords": ["写作", "文章", "write", "essay"],
        "agent": "writer"
      }
    ],
    "fallback": "general"
  }
}
```

### 路由优先级

规则按顺序匹配，先匹配的生效：

```json
{
  "router": {
    "mode": "keyword",
    "rules": [
      { "keywords": ["python"], "agent": "python-expert" },
      { "keywords": ["代码"], "agent": "coder" }
    ]
  }
}
```

## 意图识别路由

### 启用意图识别

```json
{
  "router": {
    "mode": "intent",
    "classifier": {
      "provider": "anthropic",
      "model": "claude-haiku-4-5-20251001"
    },
    "agents": ["general", "coder", "writer", "translator"]
  }
}
```

### 工作流程

1. 用户发送消息
2. 分类器分析消息意图
3. 选择最合适的代理
4. 代理处理并回复

## 分层路由

### 主从架构

```json
{
  "agents": {
    "orchestrator": {
      "role": "orchestrator",
      "subAgents": ["coder", "researcher", "writer"],
      "systemPrompt": "你是协调员，负责分配任务给专业代理。"
    },
    "coder": {
      "role": "worker",
      "systemPrompt": "你是编程专家。"
    },
    "researcher": {
      "role": "worker",
      "systemPrompt": "你是研究助手。"
    },
    "writer": {
      "role": "worker",
      "systemPrompt": "你是写作助手。"
    }
  }
}
```

### 协作流程

```
用户请求 → Orchestrator → 分析任务
                           ↓
           ┌───────────────┼───────────────┐
           ↓               ↓               ↓
        Coder         Researcher        Writer
           ↓               ↓               ↓
           └───────────────┴───────────────┘
                           ↓
                      汇总回复
```

## 渠道级路由

### 为不同渠道配置代理

```json
{
  "channels": {
    "telegram": {
      "defaultAgent": "general"
    },
    "discord": {
      "defaultAgent": "coder"
    },
    "slack": {
      "defaultAgent": "writer"
    }
  }
}
```

### 群组路由

```json
{
  "channels": {
    "slack": {
      "groups": {
        "dev-team": { "agent": "coder" },
        "marketing": { "agent": "writer" },
        "support": { "agent": "general" }
      }
    }
  }
}
```

## 路由调试

### 查看路由日志

```bash
openclaw gateway logs --router
```

### 测试路由

```bash
openclaw router test "帮我写一段Python代码"
```

输出示例：
```
Input: 帮我写一段Python代码
Matched: keyword rule
Keywords: [python, 代码]
Agent: coder
```

## 性能优化

### 分类器选择

| 场景 | 推荐 |
|------|------|
| 简单路由 | 关键词模式 |
| 复杂意图 | 意图识别模式 |
| 高性能 | 使用小模型分类 |

### 缓存策略

```json
{
  "router": {
    "cache": {
      "enabled": true,
      "ttl": 300
    }
  }
}
```
