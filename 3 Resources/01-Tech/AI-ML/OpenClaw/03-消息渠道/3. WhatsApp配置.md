# OpenClaw WhatsApp 配置

> [!info] 概述
> OpenClaw 通过 WhatsApp Web 协议连接，无需官方 API，个人账号即可使用。

## 配置方式

### 命令行启动

```bash
openclaw channel add whatsapp
```

### 扫码配对

1. 运行命令后，终端会显示二维码
2. 打开手机 WhatsApp
3. 进入 设置 > 已关联的设备 > 关联设备
4. 扫描二维码完成配对

---

## 配置文件

```json
{
  "channels": {
    "whatsapp": {
      "enabled": true,
      "sessionPath": "~/.openclaw/sessions/whatsapp"
    }
  }
}
```

| 配置项 | 说明 |
|--------|------|
| `enabled` | 是否启用 |
| `sessionPath` | 会话数据存储路径 |

---

## 安全配置

### 白名单配置

```json
{
  "channels": {
    "whatsapp": {
      "allowFrom": [
        "+15555550123",
        "+15555550124"
      ]
    }
  }
}
```

只有白名单中的号码可以与 Bot 交互。

### 群组权限

```json
{
  "channels": {
    "whatsapp": {
      "groups": {
        "*": { "requireMention": true },
        "120363025123456789": { "requireMention": false }
      }
    }
  }
}
```

| 配置项 | 说明 |
|--------|------|
| `*` | 所有群组的默认配置 |
| `requireMention` | 是否需要 @ 触发 |

## 会话管理

### 会话持久化

```json
{
  "channels": {
    "whatsapp": {
      "sessionPath": "~/.openclaw/sessions/whatsapp",
      "saveSession": true
    }
  }
}
```

### 重新配对

```bash
# 删除现有会话
openclaw channel remove whatsapp

# 重新添加
openclaw channel add whatsapp
```

## 消息功能

### 支持的消息类型

| 类型 | 发送 | 接收 |
|------|------|------|
| 文本 | ✅ | ✅ |
| 图片 | ✅ | ✅ |
| 文档 | ✅ | ✅ |
| 音频 | ✅ | ✅ |
| 视频 | ✅ | ✅ |
| 位置 | ⚠️ | ✅ |

### Markdown 支持

WhatsApp 对 Markdown 支持有限：
- `*粗体*` → **粗体**
- `_斜体_` → *斜体*
- `~删除线~` → ~~删除线~~
- `` `代码` `` → `代码`

## 群组使用

### 添加到群组

1. 直接将 Bot 手机号添加到群组
2. 或分享群组邀请链接

### 群组 ID 获取

```bash
openclaw channel debug whatsapp
```

发送消息后，日志会显示群组 ID。

---

## 注意事项

### 账号安全

> [!warning] 重要提示
> 使用 WhatsApp Web 协议可能违反 WhatsApp 服务条款，建议：
> - 使用专用号码
> - 不要发送大量消息
> - 避免商业用途

### 限制

- 单个账号最多关联 4 台设备
- 消息发送有频率限制
- 群发消息可能被封号

---

## 故障排除

### 配对失败

1. 确认手机网络正常
2. 更新手机 WhatsApp 版本
3. 清除手机 WhatsApp 缓存
4. 尝试重新配对

### 消息不同步

1. 检查会话文件是否损坏
2. 确认 Gateway 正在运行
3. 查看 Gateway 日志

### 自动登出

WhatsApp 可能自动登出关联设备：
- 手机 WhatsApp 更新后
- 长时间不活动
- 账号异常检测

重新扫码即可恢复。
