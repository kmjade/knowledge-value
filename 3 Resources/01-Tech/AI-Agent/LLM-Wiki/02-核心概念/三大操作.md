# 三大操作

LLM Wiki 的三個核心操作：Ingest（導入）、Query（查詢）、Lint（維護）。

---

## 1. Ingest（導入資料）

### 觸發方式

```
請導入 raw/文章標題.md 到知識庫
```

### 執行流程

一篇 2000-3000 字的文章通常觸及 **5-8 個文件**：

1. **閱讀資料** - 與你討論關鍵要點
2. **創建摘要頁** - `wiki/source-xxx.md`
3. **更新實體頁/概念頁** - 通常 1-3 個新頁面
4. **建立雙向鏈接** - `[[交叉引用]]`
5. **更新索引** - `wiki/index.md`
6. **追加日誌** - `wiki/log.md`
7. **標註關係** - 矛盾或補充

### 復利效應

從第二篇資料開始：

| 操作 | 說明 |
|------|------|
| 更新已有頁面 | 新案例/數據補充到已有頁面 |
| 標註跨資料關聯 | 多篇資料的相似觀點 |
| 標註矛盾 | 不覆蓋，明確標註衝突 |

---

## 2. Query（查詢）

### 查詢示例

```
根據知識庫，對比 X 和 Y 的區別
知識庫裡關於 XX 有什麼要點？
總結一下目前所有資料的核心觀點
```

### 查詢流程

1. 讀取 `wiki/index.md` 定位相關頁面
2. 深入閱讀相關頁面
3. 綜合回答並引用來源

### 回寫機制

> [!tip] 關鍵洞見
> 有價值的回答可以回寫為新 Wiki 頁面，讓探索也能積累。

---

## 3. Lint（維護健康）

### 觸發方式

```
請檢查知識庫的健康狀態
```

### 檢查項目

| 檢查項 | 說明 | 嚴重程度 |
|--------|------|----------|
| 🚨 頁面間矛盾 | 不同頁面的衝突內容 | 高 |
| ⚠️ 過時信息 | 被新資料取代的內容 | 中 |
| 📄 孤立頁面 | 無入站鏈接的頁面 | 低 |
| 🔗 缺失引用 | 可補充的交叉引用 | 中 |
| 📋 缺失頁面 | 被提及但無專屬頁面的概念 | 中 |
| 📊 數據空白 | 可填補的數據空白 | 低 |

---

## 操作頻率建議

| 操作 | 頻率 | 說明 |
|------|------|------|
| Ingest | 按需 | 新資料到達時 |
| Query | 隨時 | 需要信息時 |
| Lint | 每週 | 定期健康檢查 |
