# 工具执行模式

> [!summary] 概述
> Hermes Agent 中工具的执行模式——前台、后台、PTY 以及 Agent 循环中的链式调用。

---

## 执行模式

### 前台执行（默认）

命令立即返回结果，会阻塞 Agent 循环直到完成：

```bash
# CLI 中 Hermes 自动选择模式
# 所有工具调用默认都是同步的
```

Agent 循环：
```
LLM 响应 → tool_call → 执行工具 → 结果返回 → LLM 继续推理
```

### 后台执行

长时间任务可以后台运行，Agent 继续工作：

```
terminal(background=True, notify_on_complete=True)
```

适用于：
- 编译/构建（`npm build`、`cargo build`）
- 测试套件运行
- 长时间数据处理
- 服务器启动

### PTY 模式（交互式 CLI）

用于需要伪终端的交互式工具：

```
terminal(pty=True)
```

适用于：
- Claude Code CLI
- Codex CLI
- Python REPL

---

## Agent 循环中的链式调用

Hermes 的 Agent 循环天然支持多步工具链：

```
用户: 审查 src/ 下的代码并修复问题

→ read_file("src/main.py")     # 第1轮工具调用
→ read_file("src/utils.py")    # 第1轮（并行）
→ LLM 分析代码                  # 推理
→ patch("src/main.py", ...)    # 第2轮工具调用
→ terminal("pytest")           # 第3轮工具调用
→ 测试通过，回复用户            # 最终响应
```

---

## 并行工具调用

LLM 可以在一次响应中调用多个独立工具：

```
用户: 同时读取三个配置文件

LLM 一次返回:
→ read_file("config/a.yaml")
→ read_file("config/b.yaml")
→ read_file("config/c.yaml")

→ 三个结果一起返回给 LLM → 综合分析
```

---

## delegate_task（子智能体委派）

对于需要大量推理的子任务，使用专门工具：

```
delegate_task(goal="审查 PR #42 的代码变更", context="...")
```

特性：
- 独立上下文（不污染主会话）
- 并行执行（最多 3 个并发）
- 仅返回摘要结果
