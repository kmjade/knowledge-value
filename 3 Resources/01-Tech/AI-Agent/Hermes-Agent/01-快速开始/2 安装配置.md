# 安装配置

> [!summary] 概述
> Hermes Agent 的安装和配置指南。Hermes Agent 是 AI 智能体**框架**，不是 LLM 模型——安装极其简单，无需 GPU、PyTorch 或 CUDA。

> [!warning] 框架 vs 模型
> **Hermes Agent** 是 AI 智能体框架（CLI 应用）。
> **Hermes 3** 是 LLM 模型系列，可在框架中使用。
> 安装 Hermes Agent 不等于安装 Hermes 3 模型。

---

## 快速安裝

### Linux / macOS / WSL2

```bash
# 中文社区镜像（推荐，国内直连）
curl -fsSL https://res1.hermesagent.org.cn/install.sh | bash

# 官方安装脚本
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

### Windows PowerShell

```powershell
# 中文社区镜像（推荐）
irm https://res1.hermesagent.org.cn/install.ps1 | iex

# 官方安装脚本
irm https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.ps1 | iex
```

> [!tip] PowerShell 小白步骤
> 1. 按 Windows 键 → 输入 `PowerShell` → 打开
> 2. 粘贴上面的命令 → 回车
> 3. 等待安装完成
> 4. 关闭并重新打开 PowerShell
> 5. 输入 `hermes` 启动

### Android / Termux

```bash
curl -fsSL https://res1.hermesagent.org.cn/install.sh | bash
```

安装程序会自动检测 Termux 并切换到 Android 专用流程。

## 安裝後操作

### 类 Unix / WSL2

```bash
source ~/.bashrc   # 或：source ~/.zshrc
hermes             # 开始聊天
```

### Windows PowerShell

```powershell
# 关闭并重新打开 PowerShell 后再运行
hermes
```

### 后续配置命令

```bash
hermes model          # 选择大语言模型提供商和模型
hermes tools          # 配置启用哪些工具
hermes gateway setup  # 设置消息平台（可选）
hermes config set     # 单独设置某个配置项
hermes setup          # 或再次运行完整设置向导
```

## 環境要求

### 最低要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Linux、macOS、Windows (WSL2/PowerShell)、Android |
| **Python** | ≥ 3.11（安装脚本自动处理） |
| **磁盘** | ~500MB（不含模型） |
| **内存** | 无特殊要求（模型推理在云端） |
| **GPU** | **不需要**（使用云端 API） |

> [!tip] 不需要 GPU
> Hermes Agent 是一个**框架**，不是模型。它通过 API 调用云端的 LLM。你只需要 API 密钥，不需要本地 GPU。

### 可选依赖

| 依赖 | 用途 |
|------|------|
| **ripgrep** | 文件搜索加速 |
| **ffmpeg** | 音频处理（语音功能） |
| **Node.js v22+** | 浏览器自动化、WhatsApp 桥接 |

## 安裝程序執行的操作

安装器会自动处理以下工作：

1. **下载源码** - 优先使用国内镜像，失败时回退 GitHub
2. **Python 环境** - 复用系统 Python（≥3.11）或通过 uv 安装
3. **虚拟环境** - 创建独立虚拟环境
4. **安装命令** - 安装 `hermes` 命令到系统
5. **配置引导** - 引导完成模型配置与首次启动

## API 密鑰配置

### 编辑 .env 文件

```bash
hermes config env-path   # 查看 .env 路径
hermes config edit       # 编辑配置
```

在 `~/.hermes/.env` 中添加：

```bash
# 推荐：OpenRouter（一站式访问 300+ 模型）
OPENROUTER_API_KEY=sk-or-v1-xxxxx

# 或单独提供商
ANTHROPIC_API_KEY=sk-ant-xxxxx
OPENAI_API_KEY=sk-xxxxx
DEEPSEEK_API_KEY=sk-xxxxx
GOOGLE_API_KEY=xxxxx
XAI_API_KEY=xxxxx
```

### 支持的提供商（38 个）

| 提供商 | API Key 环境变量 | 说明 |
|--------|-----------------|------|
| **OpenRouter** | `OPENROUTER_API_KEY` | 推荐，300+ 模型一站式 |
| Anthropic | `ANTHROPIC_API_KEY` | Claude 系列 |
| OpenAI | `OPENAI_API_KEY` | GPT 系列 |
| DeepSeek | `DEEPSEEK_API_KEY` | DeepSeek V3/R1 |
| Google | `GOOGLE_API_KEY` | Gemini 系列 |
| xAI | `XAI_API_KEY` | Grok 系列 |
| Nous Portal | `hermes login` | OAuth 登录 |
| 本地 Ollama | 无需密钥 | 本地推理 |

## 基本配置

### 编辑 config.yaml

```bash
hermes config edit    # 在编辑器中打开
hermes config         # 查看当前配置
```

### 关键配置项

| 配置路径 | 说明 | 默认值 |
|---------|------|--------|
| `model.default` | 默认模型 | 由 `hermes model` 设置 |
| `model.provider` | 默认提供商 | - |
| `agent.max_turns` | 每会话最大轮次 | 90 |
| `terminal.timeout` | 命令超时（秒） | 180 |
| `checkpoints.enabled` | 文件系统快照 | false |
| `approvals.mode` | 命令审批: manual/smart/off | manual |
| `memory.memory_enabled` | 记忆功能 | true |
| `compression.enabled` | 上下文压缩 | true |

### 审批模式

| 值 | 说明 |
|----|------|
| `manual` | 危险命令需要确认（默认） |
| `smart` | LLM 辅助判断风险 |
| `off` | 跳过所有确认（= `--yolo`） |

## Windows 特別說明

### WSL2 vs PowerShell

| 方案 | 优点 | 适合 |
|------|------|------|
| **WSL2** | 最稳定、兼容性最好 | 长期使用、高级功能 |
| **PowerShell** | 快速体验、无需额外安装 | 快速试用 |

### WSL2 安装

```powershell
# 在 PowerShell（管理员）中
wsl --install

# 安装完成后重启电脑，然后运行
curl -fsSL https://res1.hermesagent.org.cn/install.sh | bash
```

### 常见 Windows 问题

| 问题 | 解决方案 |
|------|----------|
| Alt+Enter 无法换行 | 使用 Ctrl+Enter |
| UTF-8 BOM 导致配置错误 | 使用 `hermes config edit` 编辑 |
| 窗口一闪而过 | 确保在 PowerShell 中执行，不是双击文件 |

## 手動安裝

如果你希望完全控制安装过程：

```bash
# 1. 克隆仓库
git clone --recurse-submodules https://github.com/NousResearch/hermes-agent.git
cd hermes-agent

# 2. 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 3. 创建虚拟环境
uv venv venv --python 3.11

# 4. 安装依赖
export VIRTUAL_ENV="$(pwd)/venv"
uv pip install -e ".[all]"
```

### 可选组件

| 组件 | 安装命令 | 功能 |
|------|----------|------|
| 全部 | `uv pip install -e ".[all]"` | 所有组件 |
| messaging | `uv pip install -e ".[messaging]"` | Telegram/Discord 网关 |
| cron | `uv pip install -e ".[cron]"` | 定时任务 |
| voice | `uv pip install -e ".[voice]"` | 语音输入输出 |
| mcp | `uv pip install -e ".[mcp]"` | MCP 协议支持 |
