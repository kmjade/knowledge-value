# 3. RoboMaster SDK 和机器人建立连接

> [!info] 本章概述
> 介绍如何使用 RoboMaster SDK 与 EP 机器人和教育无人机建立连接，包括 WiFi 直连、组网和 USB 三种连接方式，以及 TCP/UDP 通讯协议选择。

---

## 3.1 EP 固件升级

> [!warning] 重要前提
> 请通过 RoboMaster App 查看 EP 固件，确保 EP 固件版本在 **01.01.0500** 及其以上版本。如果不是，请通过 RoboMaster APP 进行升级，否则会影响 RoboMaster SDK 的使用。

| 检查项 | 要求 |
|--------|------|
| **固件版本** | >= 01.01.0500 |
| **升级方式** | RoboMaster App |

---

## 3.2 EP 连接方式

RoboMaster SDK 支持 **3 种** 与 EP 的连接方式：

| 连接方式 | 说明 | 默认 IP |
|----------|------|---------|
| **WiFi 直连模式** | 连接机器人热点 | 192.168.2.1 |
| **WiFi 组网模式** | 同一局域网 | 自动获取 |
| **USB (RNDIS) 模式** | USB 有线连接 | 192.168.42.2 |

---

### 3.2.1 WiFi 直连模式

> [!summary] 概述
> 通过将机器人设置为直连模式，并连接机器人的 WiFi 热点进行接入。

**默认 IP：** `192.168.2.1`

#### 连接步骤

**步骤 1：切换直连模式**

开启机器人电源，切换智能中控的连接模式开关至 **直连模式**：

![](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/direct_connection_change.png)

---

**步骤 2：连接 WiFi**

准备具有 WiFi 连接功能的设备（如 DJI 妙算、Jetson Nano 或 PC），连接 EP 的 WiFi 热点：

![](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/wifi_direct.png)

**DJI 妙算、Jetson Nano 或 PC 通过 WiFi 直连 到 EP**

---

**步骤 3：运行示例代码**

参考例程：`/examples/01_robot/04_ap_conn.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
import robomaster
from robomaster import robot


if __name__ == '__main__':
    # 如果本地IP 自动获取不正确，手动指定本地IP地址
    # robomaster.config.LOCAL_IP_STR = "192.168.2.20"
    ep_robot = robot.Robot()

    # 指定连接方式为AP 直连模式
    ep_robot.initialize(conn_type='ap')

    version = ep_robot.get_version()
    print("Robot version: {0}".format(version))
    ep_robot.close()
```

**运行结果：**

```
Robot Version: xx.xx.xx.xx
```

---

### 3.2.2 USB 连接模式

> [!summary] 概述
> 通过机器人的智能中控上的 USB 端口接入，使用 RNDIS 协议将 USB 设备虚拟为网卡设备。

**默认 IP：** `192.168.42.2`

> [!info] RNDIS 协议
> USB 连接模式实质上是使用 RNDIS 协议，将机器人上的 USB 设备虚拟为一张网卡设备，通过 USB 发起 TCP/IP 连接。详见 [RNDIS Wikipedia](https://www.wikipedia.org/wiki/RNDIS)

#### 连接步骤

**步骤 1：准备设备**

选择具有 TypeA USB 接口，并支持 RNDIS 功能的第三方平台。以树莓派为例（蓝线为 USB 连接，红线为供电线）：

![](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/raspberry.png)

**树莓派连接示意图**

---

**步骤 2：运行示例代码**

参考例程：`/examples/01_robot/06_rndis_conn.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
import robomaster
from robomaster import robot


if __name__ == '__main__':
    ep_robot = robot.Robot()

    # 指定连接方式为USB RNDIS模式
    ep_robot.initialize(conn_type='rndis')

    version = ep_robot.get_version()
    print("Robot version: {0}".format(version))
    ep_robot.close()
```

**运行结果：**

```
Robot Version: xx.xx.xx.xx
```

---

### 3.2.3 WiFi 组网模式

> [!summary] 概述
> 通过将机器人设置为组网模式，并将计算设备与机器人加入到同一个局域网内，实现组网连接。

#### 连接步骤

**步骤 1：切换组网模式**

开启机器人电源，切换智能中控的连接模式开关至 **组网模式**：

![](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/networking_connection_change.png)

---

**步骤 2：设备连接局域网**

DJI 妙算、Jetson Nano 或 PC 和 EP 连接到同一个局域网：

![](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/wifi_sta.png)

**DJI 妙算、Jetson Nano 或 PC 路由连接至 EP**

---

**步骤 3：安装 myqr 库**

按 `Win + R`，输入 `cmd` 打开命令提示符，安装 myqr 库生成二维码：

```bash
pip install myqr
```

---

**步骤 4：生成二维码**

参考例程：`/examples/01_robot/05_sta_conn_helper.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
import time
import robomaster
from robomaster import conn
from MyQR import myqr
from PIL import Image


QRCODE_NAME = "qrcode.png"

if __name__ == '__main__':

    helper = conn.ConnectionHelper()
    info = helper.build_qrcode_string(ssid="RoboMaster_SDK_WIFI", password="12341234")
    myqr.run(words=info)
    time.sleep(1)
    img = Image.open(QRCODE_NAME)
    img.show()
    if helper.wait_for_connection():
        print("Connected!")
    else:
        print("Connect failed!")
```

> [!warning] 注意
> 示例代码中的 `ssid`（路由器名称）和 `password`（路由器密码）需要根据实际的路由器信息进行填写。

---

**步骤 5：扫码连接**

运行示例代码后会出现二维码图片，按下机器人智能中控上的扫码连接按键，扫描二维码进行组网连接：

![](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/networking_connection_key.png)

**运行结果：**

```
Connected!
```

> [!tip] 灯效提示
> 连接成功后，机器人的灯效从白色呼吸变为青绿色常亮。

---

#### 通过 SN 连接指定机器人

> [!tip] 多机场景
> 支持在组网模式下通过 SN 连接指定的机器人。不指定 SN 时，SDK 默认与搜索到的第一台机器人建立连接。

参考例程：`/examples/01_robot/05_sta_conn_sn.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
from robomaster import robot
from robomaster import config


if __name__ == '__main__':
    ep_robot = robot.Robot()
    # 指定机器人的 SN 号
    ep_robot.initialize(conn_type="sta", sn="3JKDH2T001ULTD")

    ep_version = ep_robot.get_version()
    print("Robot Version: {0}".format(ep_version))

    ep_robot.close()
```

## 3.3 教育无人机系列连接方式

教育无人机主要包括 **Tello EDU** 和 **Tello Talent**。

### 3.3.1 WiFi 直连模式

**默认 IP：** `192.168.10.1`

#### Tello EDU 设置

1. 短按电源按键，等待黄灯快闪开机完成
2. 机器人出厂默认即为 WiFi 直连模式
3. 重置 WiFi：在开机状态下，长按电源键 5s，状态指示灯熄灭后闪烁黄灯

![](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/tello_power.png)

#### Tello Talent 设置

开启机器人电源，切换扩展模块的模式开关至 **直连模式**：

![](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/direct_connection_change.png)

#### 示例代码

参考例程：`/examples/12_drone/01_ap_conn.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
from robomaster import robot


if __name__ == '__main__':
    # 如果本地IP 自动获取不正确，手动指定本地IP地址
    # robomaster.config.LOCAL_IP_STR = "192.168.10.22"
    tl_drone = robot.Drone()
    # 初始化
    tl_drone.initialize()

    # 获取飞机SDK版本信息
    version = tl_drone.get_sdk_version()
    print("Drone SDK Version: {0}".format(version))

    tl_drone.close()
```

**运行结果：**

```
Drone SDK Version: xx.xx.xx.xx
```

---

### 3.3.2 组网模式

**步骤：**

1. 首先将飞机设置为直连模式，与运行 SDK 的设备连接
2. 运行示例程序设置路由器信息

参考例程：`/examples/12_drone/23_set_sta.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
import robomaster
from robomaster import robot


if __name__ == '__main__':
    tl_drone = robot.Drone()
    tl_drone.initialize()

    # 切换飞行器WiFi模式为组网模式，指定路由器SSID和密码
    tl_drone.config_sta(ssid="RoboMaster_SDK_WiFi", password="12341234")

    tl_drone.close()
```

3. 切换扩展模块的模式开关至组网模式
4. 机器会自动连接到指定的路由器所在的局域网
5. 将运行 SDK 的设备也连接至该局域网

## 3.4 通讯方式

### 3.4.1 EP 通讯方式

RoboMaster SDK 与 EP 的 3 种连接方式在通讯协议上支持 **TCP** 和 **UDP** 通讯。

| 参数 | TCP | UDP |
|------|-----|-----|
| **可靠性** | 可靠 | 不可靠 |
| **连接性** | 面向连接 | 无连接 |
| **效率** | 低 | 高 |
| **适用场景** | 对数据准确性要求高 | 对数据传输实时性要求高 |

> [!tip] 选择建议
> - 机器人要实时控制运动 → 选用 **UDP**
> - 机器人进行事件型控制 → 选用 **TCP**

---

#### TCP 通讯

参考例程：`/examples/01_robot/07_tcp_protocol.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
import robomaster
from robomaster import robot


if __name__ == '__main__':
    ep_robot = robot.Robot()

    # 指定连接方式为STA 组网模式, 网络通讯方式 tcp
    ep_robot.initialize(conn_type='sta', proto_type='tcp')

    version = ep_robot.get_version()
    print("Robot Version: {0}".format(version))
    ep_robot.close()
```

**conn_type 参数说明：**
- `sta` - 组网连接
- `ap` - WiFi 直连
- `rndis` - USB 连接

---

#### UDP 通讯

参考例程：`/examples/01_robot/08_udp_protocol.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
import robomaster
from robomaster import robot


if __name__ == '__main__':
    ep_robot = robot.Robot()
    # 指定连接方式为AP 直连模式, 网络通讯方式 udp
    ep_robot.initialize(conn_type='ap', proto_type='udp')

    version = ep_robot.get_version()
    print("Robot Version: {0}".format(version))
    ep_robot.close()
```

> [!tip] 参数修改
> 通过修改 `robot.initialize()` 函数的 `proto_type` 参数来变更通讯方式：
> - `tcp` - TCP 通讯
> - `udp` - UDP 通讯

---

### 3.4.2 教育无人机通讯方式

目前 **Tello EDU** 与 **Tello Talent** 只支持 **UDP** 通信方式，因此不需要额外的配置。

## 连接参数速查表

### EP 机器人

| 参数 | 值 | 说明 |
|------|-----|------|
| `conn_type='ap'` | WiFi 直连 | 默认 IP: 192.168.2.1 |
| `conn_type='sta'` | WiFi 组网 | 需要 SN 号指定机器人 |
| `conn_type='rndis'` | USB 连接 | 默认 IP: 192.168.42.2 |
| `proto_type='tcp'` | TCP 协议 | 可靠传输 |
| `proto_type='udp'` | UDP 协议 | 实时传输 |

### 教育无人机

| 参数 | 值 | 说明 |
|------|-----|------|
| `conn_type='ap'` | WiFi 直连 | 默认 IP: 192.168.10.1 |
| 通讯协议 | UDP | 固定，不可更改 |

## 常见问题

### 1. IP 自动获取不正确

**解决方案：** 手动指定本地 IP 地址

```python
import robomaster
robomaster.config.LOCAL_IP_STR = "192.168.2.20"
```

### 2. 组网连接失败

**解决方案：**
- 确认路由器名称和密码正确
- 确认设备和机器人在同一局域网
- 检查机器人灯效状态

### 3. USB 连接识别失败

**解决方案：**
- 确认设备支持 RNDIS 功能
- 安装必要的驱动程序
- 检查 USB 线缆连接
