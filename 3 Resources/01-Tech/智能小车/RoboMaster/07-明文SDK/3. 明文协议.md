# 3. RoboMaster 明文 SDK 协议格式

> [!info] 本章概述
> 详细介绍 RoboMaster 明文 SDK 的协议格式，包括控制命令、消息推送、事件上报等。

---

## 3.1 协议格式概述

明文 SDK 使用文本协议进行通信，所有命令以分号 `;` 作为结束符。

---

## 3.1.1 控制命令

### 协议格式

```
<obj> <command> <params> [seq <seq_value>];
```

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | str | 控制对象字符串 |
| `command` | str | 控制命令字符串 |
| `params` | str | 命令参数，通常为 `<key> <value>` 形式 |
| `seq` | str | 命令序号，可选参数 |
| `;` | - | 结束符 |

### 返回值

- 成功：`ok`
- 失败：`error` 或错误信息

### 示例

```text
# 控制底盘速度
发送: chassis speed 0.5 0 0;
返回: ok

# 带序号的命令
发送: chassis move 1 0 0 seq 1;
返回: ok seq 1
```

---

## 3.1.2 消息推送

### 协议格式

```
<obj> push <attr> <value>;
```

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | str | 推送对象 |
| `attr` | str | 推送数据属性 |
| `value` | str | 推送数据数值 |
| `;` | - | 结束符 |

### 说明

- 消息推送需要先通过控制命令开启
- 推送频率取决于使能时的频率设置
- 会持续推送直到关闭

### 示例

```text
# 开启位置推送
发送: chassis position on;

# 接收推送消息
接收: chassis push position 1.5 0.2 90;

# 关闭位置推送
发送: chassis position off;
```

---

## 3.1.3 事件上报

### 协议格式

```
<obj> event <attr> <value>;
```

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | str | 发生事件的对象 |
| `attr` | str | 事件数据属性 |
| `value` | str | 事件数据数值 |
| `;` | - | 结束符 |

### 说明

- 事件上报需要先开启对应的事件监听
- 事件发生时主动上报
- 不定期触发

### 示例

```text
# 开启装甲板打击事件
发送: armor event hit on;

# 接收事件
接收: armor event hit 1 90;

# 关闭事件
发送: armor event hit off;
```

---

## 3.1.4 IP 广播

### 协议格式

```
robot ip <addr>;
```

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| `addr` | str | 当前连接模式下机器人的 IPv4 地址 |

### 说明

- UDP 广播，端口 40926
- 用于在组网模式下发现机器人
- 定期发送

### 示例

```python
import socket

# 监听 IP 广播
sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
sock.bind(('0.0.0.0', 40926))

while True:
    data, addr = sock.recvfrom(1024)
    message = data.decode()
    # 接收格式: robot ip 192.168.1.100;
    print(f"收到: {message}")
```

## 协议类型汇总

| 类型 | 方向 | 格式 | 用途 |
|------|------|------|------|
| **控制命令** | IN | `<obj> <cmd> <params>;` | 控制机器人 |
| **消息推送** | OUT | `<obj> push <attr> <val>;` | 定期数据推送 |
| **事件上报** | OUT | `<obj> event <attr> <val>;` | 事件通知 |
| **IP 广播** | OUT | `robot ip <addr>;` | 设备发现 |
| **视频流** | OUT | H.264 编码 | 实时画面 |
| **音频流** | OUT | Opus 编码 | 实时音频 |

## 完整通信示例

### Python TCP 通信

```python
import socket
import threading

class RoboMasterTextSDK:
    """RoboMaster 明文 SDK 通信类"""
    
    def __init__(self, ip='192.168.2.1', port=40923):
        self.ip = ip
        self.port = port
        self.socket = None
        self.running = False
        self.recv_thread = None
    
    def connect(self):
        """建立连接"""
        self.socket = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        self.socket.connect((self.ip, self.port))
        
        # 进入 SDK 模式
        self.send('command')
        
        # 启动接收线程
        self.running = True
        self.recv_thread = threading.Thread(target=self._receive_loop)
        self.recv_thread.daemon = True
        self.recv_thread.start()
        
        print(f"已连接到 {self.ip}")
    
    def disconnect(self):
        """断开连接"""
        self.running = False
        if self.socket:
            self.send('quit')
            self.socket.close()
            self.socket = None
        print("已断开连接")
    
    def send(self, command):
        """发送命令"""
        if not command.endswith(';'):
            command += ';'
        self.socket.send(command.encode())
        response = self.socket.recv(1024).decode().strip()
        return response
    
    def _receive_loop(self):
        """接收推送/事件消息"""
        while self.running:
            try:
                data = self.socket.recv(1024)
                if data:
                    message = data.decode().strip()
                    self._handle_message(message)
            except:
                pass
    
    def _handle_message(self, message):
        """处理消息"""
        if 'push' in message:
            print(f"[推送] {message}")
        elif 'event' in message:
            print(f"[事件] {message}")
        else:
            print(f"[响应] {message}")

# 使用示例
if __name__ == "__main__":
    sdk = RoboMasterTextSDK()
    sdk.connect()
    
    # 查询电量
    battery = sdk.send('robot battery?')
    print(f"电量: {battery}")
    
    # 开启位置推送
    sdk.send('chassis position on')
    
    # 控制移动
    sdk.send('chassis move 0.5 0 0')
    
    import time
    time.sleep(3)
    
    # 关闭推送
    sdk.send('chassis position off')
    
    sdk.disconnect()
```

## 常见问题

### 1. 命令无响应

**解决方案：**
- 确认已发送 `command;` 进入 SDK 模式
- 检查命令格式是否正确
- 验证网络连接是否正常

### 2. 推送消息接收不到

**解决方案：**
- 确认已开启对应推送
- 检查推送频率设置
- 验证接收端口是否正确

### 3. 视频流无法解码

**解决方案：**
- 确认使用 H.264 解码器
- 检查视频流端口连接
- 参考 GitHub 示例代码
