# Obsidian 性能優化

## 定義

性能優化是通過合理的配置和管理，提高 Obsidian 的運行速度、響應能力和穩定性，確保知識庫的高效使用。

## 文件數量控制

### 文件夾限制
- **建議**: 單個文件夾 < 500 個筆記
- **原因**: 過多文件影響索引和搜索速度
- **解決方案**: 使用子文件夾分類

### 文件大小限制
- **建議**: 單個文件 < 1MB
- **原因**: 大文件影響打開和保存速度
- **解決方案**: 拆分成多個小文件

### 總文件數量
- **建議**: 整個知識庫 < 10,000 個文件
- **原因**: 過多文件影響整體性能
- **解決方案**: 定期歸檔和清理

## 圖片處理優化

### 圖片壓縮
- 使用圖片壓縮工具（如 TinyPNG）
- 選擇適合的圖片格式（JPEG、WebP）
- 壓縮到合理大小（< 500KB）

### 附件管理
- 使用附件文件夾集中管理圖片
- 避免在筆記文件夾中散落圖片
- 定期清理無用圖片

### 圖片嵌入
詳見：[[Obsidian 筆記組織原則]]

**最佳實踐**:
- 優先使用外部圖片連結
- 使用 wiki-link `![[image.png]]` 嵌入本地圖片
- 避免過多圖片影響性能

## 插件管理

### 插件選擇
- ✅ 只安裝必需的插件
- ✅ 選擇性能優化的插件
- ❌ 避免安裝過多插件
- ❌ 避免安裝性能較差的插件

### 插件配置
- 調整插件設置以減少資源佔用
- 禁用不常用的插件功能
- 定期檢查插件更新

### 插件禁用
- 禁用長期不使用的插件
- 禁用影響性能的插件
- 按需啟用和禁用插件

## 搜索優化

詳見：[[Obsidian 搜索優化]]

### 搜索索引
- 定期重建搜索索引
- 清理無用的搜索緩存
- 優化搜索查詢

### Dataview 優化
詳見：[[Obsidian Dataview 插件]]

**查詢優化**:
- 使用索引字段
- 限制查詢結果數量
- 避免複雜的嵌套查詢

## 系統配置

### 緩存設置
```
設置 → 編輯器 → 顯示行號
設置 → 文件與連結 → 啟用鏈接建議
設置 → 關於 → 清理緩存
```

### 視圖設置
```
設置 → 外觀 → 基礎字體大小
設置 → 外觀 → 禁用滾動條
設置 → 編輯器 → 行寬
```

### 性能設置
```
設置 → 編輯器 → 啟用 Vim 鍵盤綁定（如需要）
設置 → 文件與連結 → 排除特定文件夾
設置 → 關於 → 適配器
```

## 工作流程優化

### 定期維護
- 每週清理無用文件
- 每月優化文件結構
- 每季清理緩存和歷史
- 每年重組知識庫

### 文件組織
詳見：[[Obsidian 筆記組織原則]]

**優化策略**:
- 使用 PARA 方法論組織
- 定期歸檔舊筆記
- 優化文件夾結構

### 連結管理
詳見：[[Obsidian 雙向連結系統]]

**優化策略**:
- 定期檢查和修復斷鏈
- 優化連結結構
- 避免過度連結

## 硬件優化

### 系統要求
- **最低配置**: 4GB RAM，SSD 硬盤
- **推薦配置**: 8GB+ RAM，SSD 硬盤
- **最佳配置**: 16GB+ RAM，NVMe SSD

### 硬盤優化
- 使用 SSD 硬盤
- 定期清理硬盤空間
- 避免硬盤碎片

### 內存優化
- 關閉不必要的應用
- 增加 Virtual Memory
- 使用 64 位版本的 Obsidian

## 監控和診斷

### 性能監控
- 使用任務管理器監控資源佔用
- 觀察啟動時間和響應速度
- 記錄性能瓶頸

### 問題診斷
- 檢查日誌文件（Ctrl/Cmd + I）
- 使用安全模式啟動（Shift 鍵）
- 禁用所有插件測試

### 性能測試
- 測試打開大型文件的時間
- 測試搜索和查詢速度
- 測試同步和備份速度

## 故障排除

### 常見性能問題

| 問題 | 可能原因 | 解決方案 |
|------|----------|----------|
| 啟動緩慢 | 插件過多 | 禁用不常用插件 |
| 搜索緩慢 | 文件過多 | 優化文件結構 |
| 卡頓 | 硬盤問題 | 使用 SSD 硬盤 |
| 崩潰 | 內存不足 | 增加內存或清理緩存 |

### 優化檢查清單
- [ ] 檢查文件數量是否過多
- [ ] 檢查單個文件是否過大
- [ ] 檢查插件數量是否合理
- [ ] 檢查圖片是否過多或過大
- [ ] 檢查搜索和查詢是否優化
- [ ] 檢查系統資源是否足夠
- [ ] 檢查硬盤和內存狀況
