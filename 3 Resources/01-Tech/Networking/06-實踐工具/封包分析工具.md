## 🦅 Wireshark

### 定義

Wireshark是開源的網絡協議分析器，支持幾百種協議，提供圖形化用戶界面。

### Wireshark功能

| 功能 | 說明 |
|------|------|
| 協議解析 | 自動識別和解析各種協議 |
| 過濾器 | 強大的過濾語法 |
| 統計 | 多種統計和圖表 |
| 導出/導入 | 多種格式支持 |
| 實時捕獲 | 支持多個接口同時捕獲 |
| 彩色着碼 | 按協議/字段着色 |

### Wireshark安裝

```bash
# Ubuntu/Debian
sudo apt-get install wireshark

# CentOS/RHEL
sudo yum install wireshark

# macOS
brew install --cask wireshark
```

### Wireshark過濾語法

#### 基本過濾器

```wireshark
# IP地址
ip.addr == 192.168.1.1
ip.addr == 192.168.1.0/24

# TCP端口
tcp.port == 80
tcp.port == 443

# 組合條件
ip.addr == 192.168.1.1 and tcp.port == 22

# 否定條件
!(tcp.port == 80)

# 比較
tcp.len >= 100
```

#### 過濾器表達式

| 運符 | 說明 | 示例 |
|------|------|------|
| == | 等於 | ip.addr == 192.168.1.1 |
| != | 不等於 | ip.addr != 192.168.1.1 |
| and | 邏輯且 | ip.src == 192.168.1.1 and tcp.port == 80 |
| or | 或 | ip.src == 192.168.1.1 or ip.src == 192.168.1.2 |
| not | 非 | not tcp |
| && | 邏輯與 | ip.src == 192.168.1.1 && ip.dst == 192.168.1.2 |
| \|\|\| | 或多 | tcp.port == 80 || tcp.port == 443 |
| in | 包含 | tcp.port in {22, 80, 443} |

#### 過濾器協議字段

```wireshark
# TCP標誌
tcp.flags.syn == 1
tcp.flags.ack == 1
tcp.flags.fin == 1
tcp.flags.reset == 1

# HTTP請求
http.request.method == "GET"
http.request.uri contains "login"
http.response.code == 404

# DNS查詢
dns.qry.name == "example.com"
dns.flags.response == 1

# Ethernet
eth.src == aa:bb:cc:dd:ee:ff
eth.dst == 11:22:33:44:55:66
```

### 高級過濾技巧

```wireshark
# 捕捉特定主機的通信
ip.host == 192.168.1.1

# 捕捉TCP三次握手
tcp.flags.syn == 1 and tcp.flags.ack == 0
tcp.flags.ack == 1 and tcp.seq == 1

# 捕捉重傳包
tcp.analysis.retransmission

# 捕捉TCP窗口更新
tcp.analysis.window_update

# 捕捉TCP零窗口
tcp.analysis.zero_window

# 捕捉HTTP 404/500錯誤
http.response.code == 404 or http.response.code == 500
```

## 💻 tcpdump

### 定義

tcpdump是命令行的網絡包抓取工具，功能強大，輕量級。

### tcpdump安裝

```bash
# Ubuntu/Debian
sudo apt-get install tcpdump

# CentOS/RHEL
sudo yum install tcpdump

# macOS
brew install tcpdump
```

### tcpdump基本用法

```bash
# 捕捉所有網卡流量
sudo tcpdump -i any

# 捕捉指定接口
sudo tcpdump -i eth0

# 捕捉指定數據包
sudo tcpdump -i eth0 host 192.168.1.1

# 捕捉指定端口
sudo tcpdump -i eth0 port 22

# 捕捉TCP SYN包
sudo tcpdump -i eth0 'tcp[tcpflags] == tcp-syn'

# 保存到文件
sudo tcpdump -i eth0 -w capture.pcap

# 讀取文件
tcpdump -r capture.pcap
```

### tcpdump過濾器

```bash
# 複雜過濾器
sudo tcpdump -i eth0 'tcp and port 80'
sudo tcpdump -i eth0 'ip host 192.168.1.1 and not port 22'

# TCP標誌過濾
sudo tcpdump -i eth0 'tcp[tcpflags] & (tcp-syn|tcp-fin)'

# 包長度過濾
sudo tcpdump -i eth0 'less 100'

# 字符串過濾
sudo tcpdump -i eth0 -A 'tcp port 80' | grep "GET"
```

### tcpdump輸出格式

```bash
# 默認輸出
tcpdump -i eth0

# 詳細輸出（- v）
tcpdump -i eth0 -v

# 十六進制輸出（- x）
tcpdump -i eth0 -x

# ASCII輸出（- A）
tcpdump -i eth0 -A

# 顯示時間戳（- t）
tcpdump -i eth0 -tttt

# 不解析域名（- n）
tcpdump -i eth0 -n
```

### tcpdump實用技巧

```bash
# 環路模式捕獲
sudo tcpdump -i eth0 -c 1000

# 旋轉捕獲（保存多個文件）
sudo tcpdump -i eth0 -w capture_%Y%m%d_%H%M%S.pcap -G 3600

# 過濾器捕獲並保存
sudo tcpdump -i eth0 'port 80 or port 443' -w web_traffic.pcap

# 查看捕獲包的統計
tcpdump -r capture.pcap -q
```

## 🖥️ tshark

### 定義

tshark是Wireshark的命令行版本，支持Wireshark的絕大多數功能。

### tshark安裝

```bash
tshark是Wireshark套件的一部分，安裝Wireshark後即可使用
```

### tshark基本用法

```bash
# 列出捕獲的包
tshark -r capture.pcap

# 顯示特定協議
tshark -r capture.pcap -Y http

# 顯示特定字段
tshark -r capture.pcap -T fields -e ip.src -e ip.dst -e tcp.len

# 只顯示摘要
tshark -r capture.pcap -q

# 輸出到文件
tshark -r capture.pcap -T fields -E separator=, > output.csv
```

### tshark常用命令

```bash
# 統計摘要
tshark -r capture.pcap -q -z io,phs

# 協議分層統計
tshark -r capture.pcap -q -z conv,tcp

# HTTP請求統計
tshark -r capture.pcap -Y http -T fields -e http.request.method -e http.request.uri

# 過濾並統計
tshark -r capture.pcap -Y "http.request.method == \"GET\"" -T fields -e http.host
```

## 🎯 抓包場景

### 場景1：捕捉HTTP流量

```bash
# Wireshark
# 過濾器：http.request.method == "GET"
# 分析：查看HTTP請求和響應頭

# tcpdump
tcpdump -i eth0 -A 'tcp port 80 and (((ip[2:2] & 0x4000) = 0) or ((ip[2] & 0x4000) = 0)) and (tcp[13] & 0x7 and tcp[13] & 2) = 2)'
```

### 場景2：分析TCP連接問題

```bash
# Wireshark
# 過濾器：tcp.analysis.retransmission
# 分析：查看重傳包、TCP窗口、RTT

# tcpdump
tcpdump -i eth0 'tcp[tcpflags] & (tcp-syn|tcp-ack)' -vv
```

### 場景3：檢測DNS查詢

```bash
# Wireshark
# 過濾器：dns.qry.name
# 分析：查看DNS查詢和響應

# tcpdump
tcpdump -i eth0 -n 'udp port 53'
```

### 場景4：分析SSL/TLS握手

```bash
# Wireshark
# 過濾器：ssl.handshake.type == 1
# 分析：查看ClientHello、ServerHello、證書交換

# 提示：需要安裝SSL密鑰才能查看解密內容
```

## 🔬 封包分析技巧

### 1. 使用顏色着碼

**目的**：通過顏色區分不同類型的流量。

```
Wireshark → View → Coloring Rules
- TCP流量：藍色
- UDP流量：綠色
- HTTP流量：紅色
- DNS流量：黃色
```

### 2. 使用統計工具

**目的**：快速了解網絡流量分佈。

```
Wireshark → Statistics → Protocol Hierarchy
Wireshark → Statistics → Conversations
Wireshark → Statistics → Endpoints
```

### 3. 使用流追蹤

**目的**：查看完整TCP/UDP對話。

```
右鍵點擊某個數據包 → Follow → TCP Stream
```

### 4. 使用專家信息

**目的**：了解協議的標準字段含義。

```
Wireshark → Analyze → Expert Info
```

### 5. 自定義列

**目的**：只顯示關鍵字段，提高分析效率。

```
Edit → Preferences → Columns
添加：ip.src, ip.dst, tcp.len, http.request.uri
```

### 6. 使用時間格式

**目的**：精確分析時間間隔。

```
View → Time Display Format
- UTC時間
- 秒精度（自定義）
```
