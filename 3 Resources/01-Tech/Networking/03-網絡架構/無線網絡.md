## 📶 Wi-Fi標準

### 802.11標準演進

| 標準 | 發布年份 | 頻寬 | 頻段 | 最大速度 | 特點 |
|-------|----------|--------|-------|----------|------|
| 802.11 | 1997 | 2.4 GHz | 1 | 2 Mbps | 原始標準 |
| 802.11a | 1999 | 5 GHz | 12 | 54 Mbps | 高頻，短距離 |
| 802.11b | 1999 | 2.4 GHz | 1-11 | 11 Mbps | 低速，兼容性好 |
| 802.11g | 2003 | 2.4 GHz | 1-11 | 54 Mbps | 向下兼容b |
| 802.11n | 2009 | 2.4/5 GHz | 1-11 | 600 Mbps | MIMO技術 |
| 802.11ac | 2013 | 5 GHz | 1-11 | 6.9 Gbps | 千兆Wi-Fi |
| 802.11ax (Wi-Fi 6) | 2019 | 2.4/5/6 GHz | 1-11 | 9.6 Gbps | 高密度，OFDMA |
| 802.11be | 2024 | 2.4/5/6 GHz | 1-11 | 46 Gbps | 超高清，多鏈路操作 |

### Wi-Fi信道

#### 2.4 GHz (13信道)

```
信道 1 (2412 MHz)
信道 2 (2417 MHz)
...
信道 13 (2472 MHz)

非重疊：1, 6, 11
```

#### 5 GHz (非DFS)

```
信道 36 (5180 MHz)
信道 40 (5200 MHz)
...
信道 149 (5745 MHz)
...
信道 165 (5825 MHz)

非重疊：36, 40, 44, 48, 149, 153, 157, 161, 165
```

### MIMO技術

**定義**：MIMO (Multiple Input Multiple Output) 使用多個天線同時傳輸和接收數據。

**技術演進**：
- **SU-MIMO**：單用戶多天線
- **MU-MIMO**：多用戶多天線（802.11ac/ax）
- **Beamforming**：定個向天線傳輸

### OFDMA (Orthogonal Frequency Division Multiple Access)

**定義**：允許多個用戶同時使用同一信道（Wi-Fi 6特點）。

**優勢**：
- 提高頻譜效率
- 減少延遲
- 支持更多同時用戶

## 🔐 Wi-Fi安全

### WEP (Wired Equivalent Privacy)

| 特性 | 說明 |
|------|------|
| 加密 | RC4 |
| 安全性 | 低，已被破解 |
| 使用 | 不建議 |

### WPA (Wi-Fi Protected Access)

| 版本 | 說明 |
|------|------|
| WPA | 使用TKIP加密 |
| WPA2 | 使用AES加密，強制 |
| WPA3 | 企業級安全性 |

### WPA3 (Wi-Fi Protected Access 3)

**定義**：最新的Wi-Fi安全標準。

**特點**：
- **SAE (Simultaneous Authentication of Equals)**：更強的握手協議
- **前向保密**：即使密碼洩漏，舊數據仍受保護
- **PMF (Protected Management Frames)**：保護管理幀

### 認證方式

| 認證方式 | 說明 |
|----------|------|
| Open (開放） | 無密碼，不安全 |
| WPA-PSK | 預共享密碼（個人Wi-Fi） |
| WPA-Enterprise | 802.1X認證（企業級） |
| WPA3-SAE | 增強的預共享密碼 |

## 📡 Wi-Fi設計

### 信道規劃

**目標**：減少信道干擾

**策略**：
1. 鄰頻道錯開：使用1, 6, 11信道
2. 5 GHz優先使用：干擾少，頻譜豐富
3. 避免DFS信道：雷達檢測會導致跳頻

### 覆蓋優化

**AP密度**：
```
覆蓋半徑 = 估計覆蓋範圍 / (N × 覆蓋半徑)
N = AP數量
```

**功率設置**：
- 低功率：減少干擾，提高容量
- 高功率：增加覆蓋，但可能干擾遠處

### 頻譜規劃

```
2.4 GHz (高密度）        5 GHz (高吞吐）
[AP1, AP2, AP3] ←────→ [AP4, AP5, AP6]
     │                     │
   客戶端                  客戶端
```

## 🏗️ AC/AP架構

### 獨立AP模式

```
[AP1] [AP2] [AP3]
  │     │     │
  ├─────┼─────┤
  │     │     │
客戶端 客戶端 客戶端
```

**特點**：
- 每個AP獨立配置
- 需要手動配置SSID、密碼、信道
- 漫遊需手動切換
- 適合小型辦公室

### AC + AP架構

```
              [AC]
                │
       ┌────────┴────────┐
       │    │    │    │
     [AP1] [AP2] [AP3] [AP4]
       │    │    │    │
       ├────┼────┤    │
       │    │    │    │
    客戶端 客戶端 客戶端 客戶端
```

**特點**：
- 統一配置和管理
- 自動調整功率和信道
- 無縫漫遊（Fast Roaming）
- 集中認證和授權

### 漫遊設計

**漫游類型**：

| 類型 | 說明 | 漫遊時間 |
|------|------|----------|
| 802.11r/k/v | 基本漫游 | >100ms |
| 802.11r | 快速漫游 | 50-100ms |
| 802.11k | 續密集密鑰 | <50ms |

**漫游條件**：
1. 信號強度門限：RSSI < -70 dBm觸發
2. 漫遊延遲：<50ms完成切換
3. 帶寬保持：漫遊期間不中斷業務

## 🎯 Wi-Fi故障排查

### 常見問題

| 問題 | 可能原因 | 解決方法 |
|------|----------|----------|
| 連接失敗 | 密碼錯誤、信道干擾 | 檢查密碼、切換信道 |
| 速度慢 | 信號弱、干擾大 | 靠近AP、降低干擾 |
| 漫遊斷開 | AP配置不一致 | 統一AC、優化漫游參數 |
| 某些設備無法連接 | Wi-Fi版本不兼容 | 檢查設備支持的模式 |

### 診斷工具

```bash
# 查看Wi-Fi網絡
nmcli dev wifi list

# 掃描Wi-Fi
nmcli device wifi rescan

# 查看連接狀態
nmcli connection show

# Linux工具
iwconfig wlan0
iwlist wlan0 scan
```
