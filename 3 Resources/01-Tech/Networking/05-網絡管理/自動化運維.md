## 🔵 Ansible網絡自動化

### Ansible架構

```
Control Machine (Ansible控制節點)
       ↓ SSH
┌─────────────────────────────────────────────┐
│         Inventory (清單)              │
├─────────────────────────────────────────────┤
│   [routers]                         │
│   [switches]                        │
│   [firewalls]                       │
│   [waps]                           │
└─────────────────────────────────────────────┘
       ↓ Playbooks (劇本)
    執行網絡任務
```

### Ansible Inventory示例

```ini
# hosts.ini
[routers]
router-01 ansible_host=192.168.1.1
router-02 ansible_host=192.168.1.2

[switches]
switch-01 ansible_host=192.168.1.10
switch-02 ansible_host=192.168.1.11

[firewalls]
fw-01 ansible_host=192.168.1.254

[all:vars]
ansible_user=admin
ansible_ssh_private_key_file=~/.ssh/id_rsa
```

### Ansible Playbook示例

```yaml
---
# 配置VLAN
- name: Configure VLANs on switches
  hosts: switches
  gather_facts: no

  tasks:
    - name: Create VLAN 10
      ios_vlan:
        vlan_id: 10
        name: Sales
        state: present

    - name: Assign port to VLAN 10
      ios_l2_interface:
        name: GigabitEthernet0/1
        mode: access
        access_vlan: 10
```

### Python示例

#### Netmiko示例

```python
from netmiko import ConnectHandler

# 連接到設備
device = ConnectHandler(
    device_type='cisco_ios',
    host='192.168.1.1',
    username='admin',
    password='password'
)

# 執行命令
output = device.send_command('show running-config')
print(output)

# 配置命令
config_commands = [
    'vlan 10',
    'name Sales',
    'exit',
    'interface GigabitEthernet0/1',
    'switchport mode access',
    'switchport access vlan 10'
]
output = device.send_config_set(config_commands)
print(output)

# 關閉連接
device.disconnect()
```

#### NAPALM示例

```python
from napalm import get_network_driver

# 獲取驅動
driver = get_network_driver('ios')
device = driver(
    hostname='192.168.1.1',
    username='admin',
    password='password'
)

# 獲取配置
config = device.get_config()
print(config['running'])

# 獲取ARP表
arp_table = device.get_arp_table()
print(arp_table)

# 關閉連接
device.close()
```

## 📡 自動化監控

### 監控即代碼 (IaC)

```python
# 使用NAPALM自動配置監控
from napalm import get_network_driver

driver = get_network_driver('ios')
device = driver(
    hostname='switch-01',
    username='admin',
    password='password'
)

# 啟用SNMP
snmp_commands = [
    'snmp-server enable',
    'snmp-server community public RO',
    'snmp-server location Data Center'
]

device.send_config_set(snmp_commands)
device.close()
```

## 🤖 自動化故障處理

### 故障自動修復腳本

```bash
#!/bin/bash
# 自動檢測和修復網絡故障

# 檢查網關可達性
ping -c 3 192.168.1.254
if [ $? -ne 0 ]; then
    echo "Gateway unreachable, restarting interface..."
    # 重啟網絡接口
    ip link set eth0 down
    sleep 2
    ip link set eth0 up
    sleep 2
    # 檢查DHCP
    dhclient eth0
fi

# 檢查DNS解析
nslookup www.google.com > /dev/null 2>&1
if [ $? -ne 0 ]; then
    echo "DNS resolution failed, updating DNS servers..."
    # 更新DNS服務器
    nmcli connection modify "Wired connection 1" ipv4.dns "8.8.8.8 8.8.4.4"
fi

# 檢查關鍵服務
for port in 80 443 22; do
    nc -zv localhost $port
    if [ $? -ne 0 ]; then
        echo "Service on port $port is down, restarting..."
        # 重啟服務
        systemctl restart nginx
    fi
done
```

### 自動化日誌分析

```python
import re
import logging

def analyze_logs(log_file):
    """分析日誌檢測異常"""
    error_patterns = [
        r'connection timeout',
        r'authentication failed',
        r'interface down'
    ]

    with open(log_file, 'r') as f:
        for line in f:
            for pattern in error_patterns:
                if re.search(pattern, line):
                    logging.error(f"Error found: {line}")
                    # 觸發告警
                    send_alert(line)
```
