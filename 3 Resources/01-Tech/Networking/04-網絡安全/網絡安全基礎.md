> [!summary] 網路安全概述
> 網路安全是指保護網路系統、資料和使用者免受未授權訪問、使用、披露、破壞、修改或破壞的措施和技術。

---

## 🎯 CIA三元组

網路安全的核心原則，也称AIC三元组。

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│            Confidentiality (机密性)                  │
│             防止未经授权的访问                      │
│                                                     │
│        ↖                           ↗               │
│         ↖                     ↗                  │
│  Integrity (完整性)     Availability (可用性)       │
# 修改
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 机密性 (Confidentiality)

**定义**：确保資訊只能被授权实体访问。

**實現方式**：
- 加密技術（对称加密、非对称加密）
- 访问控制
- 隱私保護

**威胁**：
- 窃听
- 社会工程学
- 數據泄露

### 完整性 (Integrity)

**定義**：確保資訊在傳輸和存儲過程中不被未授權篡改。

**實現方式**：
- 哈希算法
- 数字签名
- 校验和
- 审计日志

**威胁**：
- 篡改
- 重放攻击
- 中间人攻击
- 偽造

### 可用性 (Availability)

**定义**：确保授权使用者在需要时可以访问系統和服务。

**實現方式**：
- 冗余設計
- 负载均衡
- 備份和恢復
- DDoS防护

**威胁**：
- DDoS攻击
- 系統故障
- 網路中断

---

## 🔐 威胁类型

### 被动攻击

**定義**：攻擊者不直接干擾網路流量，只是監聽和收集資訊。

| 攻击类型 | 說明 |
|----------|------|
| 窃听 | 监听網路流量 |
| 流量分析 | 通過流量模式推斷資訊 |
| 社会工程学 | 通過欺骗获取資訊 |

### 主动攻击

**定義**：攻擊者主動干擾網路流量或修改數據。

| 攻击类型 | 說明 |
|----------|------|
| 拒絕服务 (DoS/DDoS) | 使服务不可用 |
| 中间人攻击 | 攔截和修改通信 |
| 重放攻击 | 重放捕获的數據包 |
| 伪造攻击 | 伪造數據源 |
| 篡改 | 修改數據內容 |

## 🛡️ 安全機制

### 1. 加密

**对称加密**：
- 特點：加密和解密使用相同密钥
- 算法：AES, DES, 3DES, RC4
- 优点：速度快
- 缺点：密钥分发困难

**非对称加密**：
- 特點：使用公钥和私钥对
- 算法：RSA, ECC, DSA
- 优点：密钥分发方便
- 缺点：速度慢

### 2. 哈希

**定义**：将任意长度數據映射为固定长度摘要。

**特性**：
- 单向性：不可逆
- 抗碰撞性：很难找到两个不同數據产生相同哈希
- 雪崩效应：微小变化导致完全不同的哈希

**算法**：
- MD5（已被破解，不推荐）
- SHA-1（已被破解，不推荐）
- SHA-256, SHA-512

### 3. 访问控制

**身份认证**：
- 你知道什么（密碼）
- 你拥有什么（令牌、智能卡）
- 你是什么（指纹、虹膜、人脸）

**授权**：
- DAC（自主访问控制）
- MAC（强制访问控制）
- RBAC（基于角色的访问控制）

### 4. 防火墙

**包过滤防火墙**：根据规则过滤數據包
**狀態检测防火墙**：跟踪連接狀態
**應用程式层防火墙**：深入檢查應用程式层协议
**Web應用程式防火墙（WAF）**：专门防护Web應用程式

## 🔴 常见網路攻击

### 1. DDoS攻击

**定义**：分布式拒絕服务攻击，利用大量僵尸網路向目標發送大量请求。

**类型**：
- **SYN Flood**：發送大量SYN包，耗尽TCP連接資源
- **UDP Flood**：發送大量UDP包
- **ICMP Flood**：發送大量ICMP包
- **HTTP Flood**：發送大量HTTP请求

**防护**：
- 流量清洗
- 流量限制
- 分布式防御
- CDN

### 2. 中间人攻击

**定義**：攻擊者攔截並可能修改兩方之間的通信。
- 公共Wi-Fi
- ARP欺骗
- DNS劫持

**防护**：
- 使用HTTPS
- 证书固定
- VPN

### 3. ARP欺骗

**原理**：發送伪造的ARP響應，將網關MAC映射到攻擊者MAC。

**防护**：
- 動態ARP检测（DAI）
- 静态ARP表
- 交换机端口安全

### 4. DNS劫持

**定义**：将DNS查詢重定向到恶意DNS伺服器。

**类型**：
- 路由器DNS劫持
- DNS投毒
- 中间人劫持

**防护**：
- 使用可信DNS伺服器
- DNS over HTTPS (DoH)
- DNSSEC

### 5. 端口扫描

**定义**：扫描目標主机開放的端口。

**工具**：
- Nmap
- Masscan

**防护**：
- 關閉不必要的服务
- 防火墙规则
- 入侵检测系統（IDS）

## 📋 安全策略

### 1. 密碼策略

- 最小长度：12位
- 包含：大小写字母、数字、特殊字符
- 定期更换（如90天）
- 歷史密碼不重复

### 2. 系統更新

- 及时安裝安全補丁
- 測試補丁相容性
- 优先修復高危漏洞

### 3. 備份策略

- 定期備份
- 异地備份
- 測試恢復

### 4. 审计和監控

- 日志記錄
- 异常检测
- 定期审计

## 🎯 安全最佳實踐

### 網路层安全

1. **分段隔离**
   - VLAN隔离
   - 子网划分
   - DMZ区域

2. **边界防护**
   - 防火墙
   - IDS/IPS
   - WAF

3. **最小权限**
   - ACL限制
   - 端口限制
   - 服务限制

### 主机层安全

1. **系統加固**
   - 關閉不必要服务
   - 更新系統補丁
   - 安裝殺毒軟體

2. **訪問控制**
   - 强密碼策略
   - 帳戶锁定策略
   - 审计策略

### 應用程式层安全

1. **輸入驗證**
   - SQL注入防护
   - XSS防护
   - CSRF防护

2. **數據保護**
   - 敏感數據加密
   - 傳輸加密（HTTPS）
   - 儲存加密
