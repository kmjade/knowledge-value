## 📚 理論知識問題

### Q1: TCP慢啟動和快速啟動有什麼區別？

**狀態**：✅ 已解決

**答案**：
- **慢啟動**：防止擁塞，指數增長，適用長連接
- **快速啟動**：減少延遲，設置初始窗口為10 MSS，適用短連接

**參考**：[[TCP協議深入]]

---

### Q2: 為什麼需要三次握手而不是兩次？

**狀態**：✅ 已解決

**答案**：
- **防止重復的歷史連接**：兩次握手無法區分新連接和舊連接的延遲包
- **雙向確認**：雙方都確認對方已準備好
- **序列號同步**：同步初始序列號

**參考**：[[TCP協議深入]]

---

### Q3: IPv6和IPv4共存的場景是什麼？

**狀態**：✅ 已解決

**答案**：
- **雙棧**：同時運行IPv4和IPv6協議棧
- **隧道技術**：IPv6 over IPv4（6to4、Teredo）
- **翻譯技術**：NAT64、DNS64
- **優先級**：IPv6通常優先於IPv4

**參考**：[[網路層協議]]

---

### Q4: VLAN和子網有什麼區別？

**狀態**：✅ 已解決

**答案**：
- **VLAN**：二層（數據鏈路層）技術，基於交換機，物理隔離
- **子網**：三層（網絡層）技術，基於IP地址，邏輯隔離
- **關係**：一個VLAN可以包含多個子網
- **應用場景**：VLAN用於安全和管理，子網用於路由和地址管理

**參考**：[[VLAN虛擬局域網]]

### Q5: 如何配置交換機端口為Trunk？

**狀態**：✅ 已解決

**答案**：
```
Cisco IOS配置：
interface GigabitEthernet0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20,30
```

**參考**：[[局域網(LAN)]], [[交換機]]

### Q6: Wi-Fi信道如何選擇以減少干擾？

**狀態**：✅ 已解決

**答案**：
1. 使用Wi-Fi分析工具（如Wireshark、NetSpot）掃描信道干擾
2. 選擇干擾最少的信道
3. 避免鄰頻道：在2.4 GHz使用1、6、11信道
4. 雙頻段使用：2.4 GHz和5 GHz同時使用

**參考**：[[無線網絡]]

### Q7: 如何在Linux上配置靜態路由？

**狀態**：✅ 已解決

**答案**：
```bash
# 添加路由
sudo ip route add 192.168.2.0/24 via 192.168.1.254

# 刪除路由
sudo ip route del 192.168.2.0/24

# 查看路由表
ip route show

# 使路由永久
echo "192.168.2.0/24 via 192.168.1.254" | sudo tee -a /etc/network/interfaces.d/route-192.168.2.conf
```

**參考**：[[網路層協議]], [[故障排查]]

## 🔧 工具使用問題

### Q8: Wireshark過濾器語法中and和or的優先級？

**狀態**：✅ 已解決

**答案**：
- **and優先級高於or**：先計算and，再計算or
- 示例：`A and B or C` = `(A and B) or C`
- 建議：使用括號明確優先級

**參考**：[[封包分析工具]]

---

### Q9: tcpdump如何捕獲只應用數據？

**狀態**：✅ 已解決

**答案**：
```bash
# 只捕獲HTTP數據（不ACK包）
tcpdump -i eth0 'tcp port 80 and tcp[((tcp[13] >> 4) & 0xf) == 0)]'

# 只捕獲TCP SYN包
tcpdump -i eth0 'tcp[tcpflags] & (tcp-syn) = 0 and tcp[tcpflags] & (tcp-syn) != 0'

# 只捕獲特定IP的通信
tcpdump -i eth0 'host 192.168.1.1 and not port 22'
```

**參考**：[[封包分析工具]], [[網絡診斷工具]]
