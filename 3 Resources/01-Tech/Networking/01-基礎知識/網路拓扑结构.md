> [!summary] 網路拓扑概述
> 網路拓扑是指網路中各种设备之间物理連接的布局结构，直接影响網路的效能、可靠性和成本。

---

## 📊 常见拓扑结构类型

```
┌─────────────────────────────────────────────────────────────┐
│  拓扑类型        拓扑图                    特點              │
├─────────────────────────────────────────────────────────────┤
│  总线型        ──────[PC]─────[PC]─────[PC]─────        │
│                  简单、成本低、易冲突                          │
├─────────────────────────────────────────────────────────────┤
│  星型               [PC]                                  │
│                    /  \                                   │
# 管理
│                    \  /                                   │
│                    [Hub]                                   │
├─────────────────────────────────────────────────────────────┤
│  环型               [PC]━━━━━━━━━━[PC]                      │
│                    ║                ║                      │
│                   [PC]━━━━━━━━━━[PC]                      │
│                  令牌传递、無冲突                           │
├─────────────────────────────────────────────────────────────┤
│  网状型         [PC]─────[PC]                              │
│                 ╱  ╲     ╱  ╲                             │
│                [PC] [PC] [PC]                              │
│                高冗余、高成本                               │
└─────────────────────────────────────────────────────────────┘
```

## 🔍 詳細解析

### 1️⃣ 总线型拓扑 (Bus Topology)

#### 结构
所有设备連接到一条公共的通訊线（总线）上。

```
    ┌─────┐       ┌─────┐       ┌─────┐
    │ PC1 │───────│ PC2 │───────│ PC3 │
    └─────┘       └─────┘       └─────┘
       │             │             │
       └─────────────────────────────┘
                    总线
```

#### 特點
| 优点 | 缺点 |
|------|------|
| ✅ 结构简单，成本低 | ❌ 故障隔离困难 |
| ✅ 易于安裝和擴展 | ❌ 冲突严重 |
| ✅ 需要的电缆较少 | ❌ 距离有限 |
| | ❌ 效能随节点增加而下降 |

#### 适用場景
- 小型網路
- 临时網路
- 歷史遗留系統

#### 常见技術
# 版本
- 终端網路

---

### 2️⃣ 星型拓扑 (Star Topology)

#### 结构
所有设备連接到中心设备（交换机或集线器）。

```
             ┌─────┐
             │ PC1 │
             └──┬──┘
                │
         ┌──────┴──────┐
         │  交换机/HUB │
         └──────┬──────┘
                │
      ┌─────┐  ┌─────┐  ┌─────┐
      │ PC2 │  │ PC3 │  │ PC4 │
      └─────┘  └─────┘  └─────┘
```

#### 特點
| 优点 | 缺点 |
|------|------|
| ✅ 故障隔离容易 | ❌ 中心设备单点故障 |
# 管理
| ✅ 易于擴展 | ❌ 成本较高 |
| ✅ 效能稳定 | |
| ✅ 支持多种协议 | |

#### 适用場景
- 现代局域网（最常见）
- 办公室網路
- 數據中心

#### 改进方案
- 使用交换机而非集线器（提供更高的效能和安全性）
- 冗余設計（双核心交换机）

---

### 3️⃣ 环型拓扑 (Ring Topology)

#### 结构
设备首尾相连形成一个环形。

```
         ┌─────┐
    ┌───→│ PC2 │←───┐
    │    └─────┘    │
    ↓               ↑
  ┌─────┐        ┌─────┐
  │ PC1 │←───────│ PC3 │
  └─────┘        └─────┘
```

#### 特點
| 优点 | 缺点 |
|------|------|
| ✅ 無冲突（令牌传递） | ❌ 一个节点故障影响全网 |
| ✅ 效能稳定 | ❌ 擴展困难 |
# 配置

#### 类型
1. **单环**：一个环形，無冗余
2. **双环**：两个环形，提高可靠性（如FDDI）

#### 常见技術
- **Token Ring**（令牌环，IBM）
- **FDDI**（光纤分布式數據接口）
- **SONET/SDH**（同步光網路）

#### 适用場景
- 城域网
- 需要確定性的工业網路
- 高可靠性要求的環境

---

### 4️⃣ 网状拓扑 (Mesh Topology)

#### 结构
每个设备都与其他多個设备直接連接。

```
         [PC1]━━━━━━[PC2]
          ╱╲      ╱╲
        [PC3]━━━[PC4]━━━[PC5]
         ╲╱      ╲╱
         [PC6]━━━━━[PC7]
```

#### 特點
| 优点 | 缺点 |
|------|------|
| ✅ 高冗余、高可靠性 | ❌ 成本极高 |
| ✅ 故障容错能力强 | ❌ 复杂度高 |
# 配置

#### 类型
1. **全网状**：每个节点都与其他所有节点連接
2. **部分网状**：只与部分關鍵节点連接

#### 适用場景
- 網際網路骨干网
- 核心路由器互联
- 军事/政府網路
- 高可用性要求的系統

#### 常见技術
- BGP（边界网关协议）
- OSPF（開放式最短路徑优先）

---

### 5️⃣ 树型拓扑 (Tree Topology)

#### 结构
多個星型拓扑的级联，形成树状结构。

```
                    [核心交换机]
                      /    |    \
                     /     |     \
              [汇聚层]  [汇聚层]  [汇聚层]
               /  \      /  \      /  \
          [接入][接入] [接入][接入] [接入]
```

#### 特點
| 优点 | 缺点 |
|------|------|
| ✅ 擴展性好 | ❌ 中心节点单点故障 |
# 管理
| ✅ 层次清晰 | |
| ✅ 适合大规模網路 | |

#### 适用場景
- 大型企业網路
- 校园网
- 服务提供商網路

#### 設計原則
- **核心层**：高速交换，冗余設計
- **汇聚层**：策略实施，流量聚合
- **接入层**：使用者接入，端口密度

---

### 6️⃣ 混合型拓扑 (Hybrid Topology)

#### 结构
结合多种拓扑结构的优点。

```
                    [核心层]
                      /  \
                    /    \
              [区域A]    [区域B]
                /  \       /  \
             [星型][星型] [环型][星型]
```

#### 特點
- ✅ 灵活性高
- ✅ 可根据需求定制
- ⚠️ 設計复杂

#### 适用場景
- 大型企业網路
- 多样化需求的環境
- 跨地域網路

## 🏗️ 網路設計最佳實踐

### 层次化網路設計

```
┌─────────────────────────────────────────────────────────┐
│                    接入层 (Access Layer)                │
│  - 提供使用者接入                                         │
│  - 端口安全                                             │
│  - VLAN划分                                             │
├─────────────────────────────────────────────────────────┤
│                   汇聚层 (Distribution Layer)           │
│  - 策略实施                                             │
│  - 流量聚合                                             │
│  - 路由功能                                             │
├─────────────────────────────────────────────────────────┤
│                   核心层 (Core Layer)                   │
│  - 高速轉發                                             │
│  - 冗余設計                                             │
│  - 最少策略                                             │
└─────────────────────────────────────────────────────────┘
```

### 冗余設計

#### 双星型設計
```
             ┌─────────────┐
             │ 核心交换机A │◄────┐
             └─────────────┘     │
                   ╱╲            │
                  /  \           │
                /      \         │
        ┌──────┴    ┌─────┴──────┐  冗余鏈路
        │    接入   │    接入    │
        │    交换机A│   交换机B  │
        └───────┬─────┴──────┬────┘
                │            │
             ┌─────────────┐ │
             │ 核心交换机B │◄─┘
             └─────────────┘
```

### 冗余技術
- **鏈路聚合** (LACP)
- **生成树协议** (STP/RSTP/MSTP)
- **虚拟路由器冗余协议** (VRRP)
- **热備份路由协议** (HSRP)

## 💡 選擇拓扑结构的考虑因素

### 1. 预算
- 成本限制 -> 星型/树型
- 预算充足 -> 网状型（關鍵部分）

### 2. 规模
- 小型網路 (< 50节点) -> 星型
- 中型網路 (50-500节点) -> 树型
- 大型網路 (> 500节点) -> 混合型

### 3. 可靠性要求
- 低要求 -> 星型
- 中等要求 -> 树型 + 冗余
- 高要求 -> 网状型

# 管理
# 管理
- 专业團隊 -> 混合型

### 5. 擴展需求
- 稳定规模 -> 星型
- 持續擴展 -> 树型/网状型

## 🛠️ 實踐示例

### 示例1：小型办公室網路設計

```
需求：30台电脑，需要列印机共享

推荐方案：星型拓扑
- 1台千兆交换机（32口）
- 30台电脑連接到交换机
- 列印机也連接到交换机
- 路由器連接到交换机作为网关
```

### 示例2：中型企业網路設計

```
需求：300台电脑，3个部门，需要網路隔离

推荐方案：树型拓扑 + VLAN
- 2台核心交换机（冗余）
- 3台汇聚交换机（每个部门一台）
- 多台接入交换机
- 使用VLAN隔离部门流量
```
