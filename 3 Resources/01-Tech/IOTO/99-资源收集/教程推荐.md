## 學習路徑建議

### 初學者路徑

```
1. 閱讀核心理念
   └→ 理解 PKM 2.0 範式轉變

2. 安裝插件
   └→ 完成系統部署

3. 創建第一個項目
   └→ 實踐項目創建流程

4. 每日使用
   └→ 形成習慣
```

### 進階路徑

```
1. 自定義模板
   └→ 適應個人需求

2. 配置同步
   └→ 多設備協作

3. 優化工作流
   └→ 持續改進
```

## 學習建議

### 學習原則

1. **理念先行**：先理解為什麼
2. **小步快跑**：從最小可用開始
3. **持續實踐**：每天使用
4. **漸進增強**：先掌握核心

### 常見誤區

| 誤區 | 正確做法 |
|------|---------|
| 過度定製 | 先用默認配置 |
| 追求完美 | 先完成再完善 |
| 收藏成癮 | 成果導向收集 |
| 頻繁切換 | 持久堅持一個系統 |
