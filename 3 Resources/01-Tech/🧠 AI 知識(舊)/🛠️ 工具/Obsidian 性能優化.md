## 檔案數量控制

### 檔案夾限制
- **建議**: 單個檔案夾 < 500 個筆記
- **原因**: 過多檔案影響索引和搜尋速度
- **解決方案**: 使用子檔案夾分類

### 檔案大小限制
- **建議**: 單個檔案 < 1MB
- **原因**: 大檔案影響打開和儲存速度
- **解決方案**: 拆分成多個小檔案

### 總檔案數量
- **建議**: 整個知識庫 < 10,000 個檔案
- **原因**: 過多檔案影響整體效能
- **解決方案**: 定期歸檔和清理

## 圖片處理優化

### 圖片壓縮
- 使用圖片壓縮工具（如 TinyPNG）
- 選擇適合的圖片格式（JPEG、WebP）
- 壓縮到合理大小（< 500KB）

# 配置

### 緩存設置
```
設置 → 編輯器 → 顯示行號
設置 → 檔案與連結 → 啟用鏈接建議
設置 → 關於 → 清理緩存
```

### 視圖設置
```
設置 → 外觀 → 基礎字體大小
設置 → 外觀 → 禁用滾動條
設置 → 編輯器 → 行寬
```

### 效能設置

設置 → 編輯器 → 啟用 Vim 鍵盤綁定（如需要）

## 故障排除

### 常見效能問題

| 問題 | 可能原因 | 解決方案 |
|------|----------|----------|
| 啟動緩慢 | 外掛過多 | 禁用不常用外掛 |
| 搜尋緩慢 | 檔案過多 | 優化檔案結構 |
| 卡頓 | 硬盤問題 | 使用 SSD 硬盤 |
| 崩潰 | 內存不足 | 增加內存或清理緩存 |
