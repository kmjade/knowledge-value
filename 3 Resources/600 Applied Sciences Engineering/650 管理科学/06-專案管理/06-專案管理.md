# 06 — 專案管理

> 專案是為創造獨特產品、服務或成果而進行的臨時性工作。

## 專案管理鐵三角

範圍 Scope ←→ 成本 Cost ←→ 時間 Time

三者互相制約——改變一個，必然影響其他。

## PMBOK 五大過程組

| 過程組 | 關鍵活動 |
|--------|---------|
| **啟動** | 專案章程、利害關係人識別 |
| **規劃** | WBS、時程、預算、風險計畫 |
| **執行** | 團隊管理、溝通、採購 |
| **監控** | 進度追蹤、變更控制 |
| **結束** | 交付驗收、經驗總結 |

## WBS (工作分解結構)

```
專案
├── 需求分析
├── 設計
│   ├── 系統架構
│   └── 介面設計
├── 開發
│   ├── 前端
│   └── 後端
└── 測試
```

## 敏捷 vs 傳統

| 維度 | 傳統 (Waterfall) | 敏捷 (Scrum) |
|------|-----------------|-------------|
| 規劃 | 前期完整規劃 | 漸進式規劃 |
| 變更 | 正式變更控制 | 歡迎變更 |
| 交付 | 一次交付 | 迭代增量交付 |
| 適合 | 需求穩定 | 需求不明確 |

## Scrum 框架

| 角色 | 責任 |
|------|------|
| Product Owner | 定義需求、排優先級 |
| Scrum Master | 移除障礙、促進流程 |
| Development Team | 交付增量 |

| 事件 | 頻率 |
|------|------|
| Sprint Planning | 每 Sprint 開始 |
| Daily Scrum | 每天 15 分鐘 |
| Sprint Review | Sprint 結束 |
| Sprint Retrospective | Sprint 結束 |

> 💡 專案管理的本質是「在不確定中交付確定」——好的 PM 管理的是風險，不是任務清單。
