# 04-长距离通信

> 当设备分布在数公里甚至数百公里的范围内——农田传感器、城市水表、物流追踪器——需要远距离、低功耗的通信技术。

## LPWAN 协议对比

| 协议 | 频段 | 速率 | 距离 | 功耗 | 授权频谱 |
|------|------|------|------|------|----------|
| **LoRaWAN** | Sub-GHz | 0.3-50kbps | 2-15km | 极低 | 否 |
| **NB-IoT** | 蜂窝 | 250kbps | 1-10km | 极低 | 是 |
| **LTE-M** | 蜂窝 | 1Mbps | 1-10km | 低 | 是 |
| **Sigfox** | Sub-GHz | 100bps | 10-40km | 极低 | 否 |
| **5G RedCap** | 蜂窝 | 150Mbps | 1-3km | 中 | 是 |

## LoRaWAN

| 特性 | 说明 |
|------|------|
| **调制** | Chirp 扩频——抗干扰极强 |
| **架构** | 终端 → 网关 → 网络服务器 → 应用 |
| **频段** | CN470 (中国), EU868, US915 |
| **自适应速率** | 根据信号质量自动调整速率 |
| **A 类 (最省电)** | 上行后开两个接收窗口→电池 10 年+ |
| **C 类 (最低延迟)** | 持续接收→耗电，适合常供电设备 |

```
终端设备 → 网关 (1-N) → LoRaWAN 网络服务器 → 应用服务器
  (电池)    (常供电)        (云端/本地)
```

> LoRaWAN 的典型应用：农业土壤传感器——电池+太阳能，每天传几次数据，工作数年。

## NB-IoT

| 特性 | 说明 |
|------|------|
| 运营商 | 电信/移动/联通均有覆盖 |
| 部署 | 基于现有 4G 基站软件升级 |
| 优势 | 深度覆盖(地下水表)、海量连接 |
| 局限 | 不支持基站切换(不适合移动场景) |

> NB-IoT vs LoRaWAN：NB-IoT 不需要自建网关（用运营商网络），但按连接收费。LoRa 网络免费但需要自己建网关。

## 5G IoT

| 能力 | 技术 | 场景 |
|------|------|------|
| **eMBB** (高带宽) | 5G NR | 8K 摄像头、AR |
| **uRLLC** (低延迟) | 5G URLLC | 工业控制、自动驾驶 |
| **mMTC** (海量) | NB-IoT/LTE-M (5G 兼容) | 电表、传感器 |
| **RedCap** (轻量5G) | NR Reduced Capability | 视频监控、可穿戴 |

## 卫星 IoT

| 方案 | 特点 |
|------|------|
| **星链 (Starlink)** | LEO 卫星，直接连地面终端 |
| **NTN (非地面网络)** | 3GPP Release 17，普通手机直连卫星 |
| **北斗短报文** | 中国北斗——发送短消息 |

> 卫星 IoT 的场景：海洋浮标、偏远矿山、极地科考——地面网络覆盖不到的地方。

## 协议选型指南

| 场景 | 推荐 |
|------|------|
| 城市固定 (电表、水表) | NB-IoT |
| 农业/环境 (大范围、电池) | LoRaWAN |
| 物流追踪 (移动) | LTE-M |
| 视频监控 (高带宽) | 5G / WiFi 6 |
| 工业控制 (低延迟) | 5G URLLC / WiFi 6E |
| 无网络覆盖 | 卫星 IoT |
