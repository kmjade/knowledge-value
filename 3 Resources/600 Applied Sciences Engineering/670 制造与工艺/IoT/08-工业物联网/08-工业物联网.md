# 08-工业物联网

> 工业物联网 (IIoT) 是 IoT 在制造业、能源、矿业等工业领域的应用——与消费 IoT 不同，IIoT 的首要任务是**可靠性、安全性和互操作性**。

## 消费 IoT vs 工业 IoT

| 维度 | 消费 IoT | 工业 IoT |
|------|----------|----------|
| 故障后果 | 不便 | 停产/安全事故 |
| 环境 | 室内 | -40~85°C/粉尘/振动/防爆 |
| 生命周期 | 2-5 年 | 10-30 年 |
| 网络 | WiFi | 有线+冗余+确定性延迟 |
| 协议 | 通用 | Modbus/Profinet/EtherCAT/OPC UA |
| 安全 | 数据隐私 | 人身安全+生产安全+国家安全 |
| 供应链 | 快速迭代 | 长期供货 (10年+) 要求 |

## IIoT 架构

```
现场层: PLC/传感器/执行器 (Modbus, Profinet, 4-20mA)
   ↓
边缘层: 工业网关/边缘服务器 (协议转换、本地处理)
   ↓
平台层: SCADA/MES/IIoT平台 (Ignition, MindSphere)
   ↓
企业层: ERP/MRP (SAP)
```

## OPC UA——工业互操作的基石

| 特性 | 说明 |
|------|------|
| 安全 | 加密、认证、审计——工业安全标准 |
| 信息建模 | 不只是传数据——传语义 |
| 跨平台 | Windows/Linux/嵌入式 |
| Pub/Sub | OPC UA over MQTT——云原生扩展 |

> 如果所有工业设备都说 OPC UA，就不需要那么多协议转换网关。现实是 Modbus 和 Profinet 还会存在很久。

## 典型 IIoT 应用

### 预测性维护

| 数据 | 分析 | 输出 |
|------|------|------|
| 振动、温度、电流 | ML 异常检测+退化模型 | 剩余寿命 (RUL) 预测 |

### OEE 监控

```
OEE = 可用性 × 性能 × 质量
    = (运行时间/计划时间) × (实际产量/理论产量) × (良品/总产量)
```

### 能源管理

| 监测 | 分析 | 行动 |
|------|------|------|
| 分项电表+流量计 | 能耗基线+异常检测 | 压缩空气泄漏定位、峰谷优化 |
