# 09-未来演进

> Matter v1.0-1.3 已经覆盖了灯、传感器、门锁、窗帘和部分家电。未来版本将扩展到更多设备类型和更丰富的功能。

## Matter 2.0 路线图 (预计 2025-2026)

| 新增 | 说明 |
|------|------|
| **摄像头** | 视频流和控制——智能门铃、安防摄像头 |
| **门铃** | 视频+双向语音 |
| **更多大家电** | 冰箱、热水器、全功能空调 |
| **能源管理增强** | 太阳能、储能、需求响应 |
| **高级自动化** | 跨设备的复杂场景 |
| **语音集成** | Matter 标准化语音命令 |

## 长期演进方向

### 到商业和工业

| 场景 | 说明 |
|------|------|
| 商业照明 | 办公室、商场——Matter 替代 DALI/KNX 的可能性 |
| 酒店 | 客人手机 = 房间控制器 |
| 小型商业 | 零售店、餐厅的标准化控制 |

### 能源管理

Matter 能源管理可能是最大的突破：

```
EV + 太阳能 + 储能 + 家电 → Matter 统一管理
  ↓
根据电价自动调度：电价低→充电，电价高→卖电回电网
```

### AI 集成

| 能力 | 场景 |
|------|------|
| 意图理解 | "我要看电影"→自动调暗灯、关窗帘、开电视 |
| 异常预测 | "你的冰箱门没关好超过 5 分钟" |
| 能效优化 | 根据天气和习惯自动优化 HVAC |

## 挑战

| 挑战 | 说明 |
|------|------|
| **碎片化风险** | 平台可能在 Matter 之上加私有的"增值功能"→重新碎片化 |
| **认证成本** | 小公司可能负担不起认证 |
| **消费者认知** | "Matter 是什么？"——需要教育市场 |
| **中国生态** | 小米/华为/阿里的智能家居生态是否会完全拥抱 Matter？ |
| **协议膨胀** | 每增加一个设备类型→规范变厚→实现变复杂 |

## 对开发者的建议

| 现在 | 未来 |
|------|------|
| 新产品首选 Matter over Thread | 减少私有协议依赖 |
| Zigbee 存量设备→Matter Bridge | 用户不需要换设备 |
| 关注 v2.0 的摄像头/门铃 | 准备抢先机 |
| 参与 CSA 工作组 | 影响标准方向 |
