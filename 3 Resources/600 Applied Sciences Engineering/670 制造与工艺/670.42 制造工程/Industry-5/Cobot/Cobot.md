# 协作机器人 (Cobot)

> **协作机器人 = 专为人机安全协作而设计的机器人。不需要围栏，通过力控、碰撞检测和速度限制，与人共享工作空间。**
>
> 来自: [[Industry-5|工业 5.0]] → 以人为本支柱的核心技术实现

Cobot 不是传统工业机器人的"缩小版"——它是**范式转换**。传统机器人的逻辑是"围栏隔离 + 精确轨迹"，Cobot 的逻辑是"力控感知 + 安全协作"。这让**非程序员**（操作员）可以通过拖动示教直接"教"机器人，绕过了"工程师翻译"这一传统瓶颈。

## 核心技术差异：为什么 Cobot 是范式转换

### 力控——而非位置控制

传统工业机器人的控制目标是**精确跟踪预设轨迹**（位置控制）。Cobot 在此基础上增加了**力控制**——这是"协作"的技术本质：

```
传统机器人: 目标位置 → PID → 电机 → 实际位置
                              ↑
                         位置反馈

Cobot:       目标位置 + 力约束 → 阻抗控制 → 电机 → 实际位置
                                   ↑            ↑
                              力矩反馈 ←── 关节力矩传感器
```

> Cobot 的关节力矩传感器在 **<10ms** 内检测到碰撞，50ms 内停止——这是 PFL（功率和力限制）模式的硬性要求。

### 拖动示教——操作员的默会知识直接编码

Cobot 最标志性的能力：操作员握住 Cobot 末端，"手把手"拖动到目标位置，路径被自动记录为程序。

```
人手施加力 → 关节力矩传感器检测 → 阻抗控制器 → 电机平滑跟随
                                             ↓
                                  记录路径点 → 生成程序
```

**为什么这是革命性的？** 传统机器人编程需要"操作员口述需求 → 工程师翻译为代码 → 调试 → 修改"。Cobot 把"翻译"这一步直接消除了——操作员的**默会知识**（"这个角度刚好""这里要稍微快一点"）直接成为机器行为。

### ISO/TS 15066 —— Cobot 的安全宪法

四种协作模式中，**PFL（功率和力限制）** 是最常用的。Cobot 不是"无力"，而是碰撞力被限制在人体承受范围内：

| 身体部位 | 准静态接触限值 |
|----------|--------------|
| 头部/脸部 | 65 N |
| 手/手指 | 140 N |
| 手臂 | 150 N |

> 安全设计三要素：关节力矩传感器 + <10ms 碰撞检测 + 圆形无锐角外表面。缺一不可。

## 部署经济学

| 成本项 | 传统工业机器人 | Cobot |
|--------|-------------|-------|
| 机器人本体 | ¥200K-1M+ | ¥50K-300K |
| 安全围栏/光幕 | ¥50K-200K | ¥0（PFL 内置） |
| 安装/集成 | ¥100K-500K | ¥20K-80K |
| 编程 | ¥50K-200K（专业程序员） | ¥5K-20K（操作员可做） |
| **3 年 TCO** | ¥500K-2M+ | **¥100K-500K** |
| **典型回收期** | 18-36 个月 | **6-12 个月** |

> Cobot 的回收期短不仅因为便宜——更因为**部署快 + 操作员自己能改程序**，不需要每次换型都叫集成商来。

## 市场格局 (2025)

| 品牌 | 全球份额 | 差异化 |
|------|---------|--------|
| **Universal Robots** | ~40% | 先驱、最大生态、UR+ 配件市场 |
| **FANUC** | ~15% | 工业级可靠性，高负载 CRX-30iA (30kg) |
| **ABB** | ~10% | GoFa/SWIFTI 双线 |
| **Techman (达明)** | ~8% | 内置视觉、图形化编程 |
| **越疆/节卡** | 快速增长 | 中国市场，教育+轻工业 |

趋势：负载上升（突破 30kg）→ 视觉集成成为标配 → AI + LLM 自然语言编程（"抓起红色的那个"）
