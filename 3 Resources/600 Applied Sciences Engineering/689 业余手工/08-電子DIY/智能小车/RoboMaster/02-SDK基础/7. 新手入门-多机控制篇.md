# 7. RoboMaster SDK 新手入门 - 多机控制篇

> [!info] 本章概述
> 介绍如何使用 RoboMaster SDK 实现多机控制，包括多机初始化、编号、分组和任务控制，实现复杂的多机编队任务。

---

## 7.1 多机控制简介

RoboMaster SDK 支持多机控制，用户可以调用相应的多机接口，轻松控制多台机器，实现复杂的多机编队等任务。

**支持的产品：**
- RoboMaster EP 系列
- Tello EDU / Tello Talent 教育无人机

---

## 7.2 多机控制流程

多机控制主要分为以下几个方面：

| 步骤 | 说明 |
|------|------|
| **多机初始化** | 与局域网内的多台机器建立连接，并初始化相关机器人 |
| **多机编号** | 通过飞机的 SN 号对飞机进行编号，便于后面进行多机的选中控制 |
| **多机分组 & 群组控制** | 通过将多台机器进行分组，实现多机选中的效果 |
| **任务控制** | 通过任务控制的方式可以同时控制不同组执行不同的动作 |

---

## 7.3 多机初始化

### 环境准备

安装 `netifaces` 包：

```bash
pip install netifaces
```

> [!tip] Windows 环境
> 如果出现编译问题，请下载安装 `visualcppbuildtools_full.exe`（[GitHub 下载](https://github.com/dji-sdk/robomaster-sdk)）

### 步骤 1：设置组网模式

首先将机器人设置为 **路由器组网模式**，并将所有机器人与运行 RoboMaster SDK 的设备连接至同一个局域网内。

- EP 参考：[[3. 建立连接#EP 连接方式|EP 连接方式]]
- 无人机参考：[[3. 建立连接#教育无人机系列连接方式|教育无人机连接方式]]

### 步骤 2：导入多机控制包

```python
from multi_robomaster import multi_robot
```

### 步骤 3：生成多机对象

```python
# EP 机器人多机对象
multi_robot = multi_robot.MultiEP()

# 教育无人机多机对象
multi_drones = multi_robot.MultiDrone()
```

### 步骤 4：调用多机初始化函数

```python
# EP 多机初始化（不需要参数）
multi_robots.initialize()

# 教育无人机初始化（需要指定飞机数量）
multi_drones.initialize(2)  # 指定扫描 2 架飞机
```

> [!important] 注意
> EP 机器人与教育无人机在多机初始化时有所区别！
> - EP：不需要输入参数
> - 教育无人机：需要指明需要扫描的飞机数量

## 7.4 多机编号

通过将机器人编号，可以方便用户进行多机控制。

### 编号方法

目前支持的编号策略为根据输入的 SN 进行机器人与编号的绑定：

```python
number_id_by_sn([id1, SN1], [id2, SN2], [id3, SN3] ...)
```

**参数说明：**
- 每个列表包含两个元素：`[id, SN]`
- 第一个元素：期望的编号数字
- 第二个元素：机器人 SN 信息的字符串
- 返回值：成功编号的机器数目

### 示例：EP 多机编号

假设有两台 EP 机器人需要编号：

```python
# 将 SN 为 3JKDH2T001ULTD 的机器人编号为 0 号
# 将 SN 为 3JKDH3B001NN0E 的机器人编号为 1 号
multi_robots.number_id_by_sn(
    [0, '3JKDH2T001ULTD'], 
    [1, '3JKDH3B001NN0E']
)
```

## 7.5 多机分组 & 群组控制

通过将机器人分为不同组，可以更加简单地进行多机控制。**组对象**的控制接口调用形式与单机类似。

### 7.5.1 生成组对象

```python
build_group(robot_id_list)
```

**参数：** 包含需要分组的机器人 ID 信息的列表
**返回值：** 创建的组对象

### 示例：EP 分组操作

假设有三台 EP 机器人，编号分别为 0、1、2 号：

```python
# 0 号与 1 号机器人分为一组
robot_group1 = multi_robots.build_group([0, 1])

# 2 号机器人单独一组
robot_group2 = multi_robots.build_group([2])

# 全部三台机器人分为一组
robot_group_all = multi_robots.build_group([0, 1, 2])
```

---

### 7.5.2 组对象的相关操作

#### 7.5.2.1 更新成员

**添加成员：**

```python
append(robots_id_list)
```

**删除成员：**

```python
remove(robots_id_list)
```

**示例：**

```python
# 从群组中移除 1 号和 2 号机器人
robot_group_all.remove([1, 2])

# 将 1 号和 2 号机器人添加回来
robot_group_all.append([1, 2])
```

---

#### 7.5.2.2 群组控制

在大多数情况下，群组控制的动作类接口形式与单机控制的接口形式一致。

**EP 群组控制示例：**

```python
# 组内所有机器人前进 1 米
robot_group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()

# 组内所有机器人云台向左旋转 90 度
robot_group.gimbal.move(0, 90).wait_for_completed()
```

> [!tip] 使用提示
> 可以将组对象当成单机对象使用，接口参数与单机部分相同。

---

#### 7.5.2.3 单机控制

从群组中获取单机对象进行单独控制：

```python
get_robot(robot_id)
```

**参数：** 机器人编号数字
**返回值：** 单机对象

**获取组内所有机器人编号列表：**

```python
robot_group.robots_id_list
```

**教育无人机单机控制示例：**

```python
# 组内的教育飞机依次起飞
for drone_id in drone_group.robots_id_list:
    drone_obj = drone_group.get_robot(drone_id)
    drone_obj.flight.takeoff().wait_for_completed()
```

## 7.6 任务控制

### 问题场景

- 如何同时让不同组同时做不同的动作？
- 如何在实现不同组同时执行任务的时候保证同步？

### run 方法

```python
run([robot_group1, action_task1], [robot_group2, action_task2], [robot_group3, action_task3]...)
```

**功能：**
- 实现不同的组同时执行不同的动作
- 保证语句执行结束时，所有动作任务都执行完毕

**参数格式：**
- 列表包含两个元素：`[组对象, 任务函数]`
- 任务函数必须满足固定的接口形式

### 任务函数格式

用户定义的任务函数必须满足：

```python
def task_function(robot_group):
    # 函数应只有一个参数
    # 参数为执行函数内动作的组对象
    robot_group.xxx.action().wait_for_completed()
```

### 示例：EP 任务控制

假设有三个机器人组：
- `robot_group1`：包含 0 号、1 号机器人
- `robot_group2`：包含 2 号机器人
- `robot_group_all`：包含 0、1、2 号三台机器人

**定义任务函数：**

```python
def move_forward_task(robot_group):
    """组内机器人前进 1 米"""
    robot_group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()

def move_backward_task(robot_group):
    """组内机器人后退 1 米"""
    robot_group.chassis.move(x=-1, y=0, z=0, xy_speed=0.7).wait_for_completed()

def move_left_task(robot_group):
    """组内机器人左移 1 米"""
    robot_group.chassis.move(x=0, y=-1, z=0, xy_speed=0.7).wait_for_completed()
```

**执行任务控制：**

```python
# 0 号与 1 号机器人前进 1m，同时 2 号机器人后退 1m
multi_robots.run([robot_group1, move_forward_task], [robot_group2, move_backward_task])

# 三台机器人同时左移 1m
multi_robots.run([robot_group_all, move_left_task])
```

> [!important] 注意
> 用户自定义的动作任务函数需要满足固定的接口形式！

## 多机控制完整示例

### EP 多机控制完整流程

```python
from multi_robomaster import multi_robot

# 1. 创建多机对象
multi_robots = multi_robot.MultiEP()

# 2. 初始化
multi_robots.initialize()

# 3. 编号
multi_robots.number_id_by_sn(
    [0, '3JKDH2T001ULTD'], 
    [1, '3JKDH3B001NN0E'],
    [2, '3JKDH4B001NN0F']
)

# 4. 分组
group_01 = multi_robots.build_group([0, 1])
group_2 = multi_robots.build_group([2])
group_all = multi_robots.build_group([0, 1, 2])

# 5. 定义任务
def forward_task(group):
    group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()

def backward_task(group):
    group.chassis.move(x=-1, y=0, z=0, xy_speed=0.7).wait_for_completed()

# 6. 执行并行任务
multi_robots.run([group_01, forward_task], [group_2, backward_task])

# 7. 释放资源
multi_robots.close()
```

### 教育无人机多机控制完整流程

```python
from multi_robomaster import multi_robot

# 1. 创建多机对象
multi_drones = multi_robot.MultiDrone()

# 2. 初始化（指定飞机数量）
multi_drones.initialize(2)

# 3. 编号
multi_drones.number_id_by_sn(
    [0, 'TALLE1Q001XXXXX'],
    [1, 'TALLE1Q002XXXXX']
)

# 4. 分组
group_all = multi_drones.build_group([0, 1])

# 5. 定义任务
def takeoff_task(group):
    for drone_id in group.robots_id_list:
        drone = group.get_robot(drone_id)
        drone.flight.takeoff().wait_for_completed()

def land_task(group):
    for drone_id in group.robots_id_list:
        drone = group.get_robot(drone_id)
        drone.flight.land().wait_for_completed()

# 6. 执行任务
multi_drones.run([group_all, takeoff_task])
# ... 执行其他动作 ...
multi_drones.run([group_all, land_task])

# 7. 释放资源
multi_drones.close()
```

## 多机控制对比

| 特性 | EP 机器人 | 教育无人机 |
|------|----------|------------|
| **对象类型** | `MultiEP()` | `MultiDrone()` |
| **初始化参数** | 无需参数 | 需要指定飞机数量 |
| **控制模块** | 底盘、云台、发射器等 | 飞行控制 |
| **分组控制** | 支持 | 支持 |
| **任务控制** | 支持 | 支持 |

## 常见问题

### 1. 扫描不到机器人

**解决方案：**
- 确认所有机器人在同一局域网
- 确认路由器模式设置正确
- 检查网络连接状态

### 2. 编号失败

**解决方案：**
- 确认 SN 号输入正确
- 确认机器人已成功初始化
- 检查机器人是否在线

### 3. 任务执行不同步

**解决方案：**
- 使用 `run()` 方法确保同步
- 检查任务函数格式是否正确
- 确认 `wait_for_completed()` 调用
