# 08 — 系統工程

> 系統工程是跨學科的整合方法——確保複雜系統的所有元件協同工作，達成整體目標。

## 系統工程 V 模型

```
需求分析 ←──────────────→ 驗證測試
    │                        │
    ▼                        ▲
系統設計 ←──────────────→ 整合測試
    │                        │
    ▼                        ▲
細部設計 ←──────────────→ 單元測試
    │                        │
    ▼                        ▲
      實作 ────────────→
```

## 核心流程

| 流程 | 說明 |
|------|------|
| **需求工程** | 擷取、分析、驗證需求 |
| **架構設計** | 系統結構與介面定義 |
| **整合** | 逐步組裝與測試 |
| **驗證與確認 (V&V)** | 做對了東西？東西做對了？ |
| **組態管理** | 控制變更與版本 |

## 可靠度工程

| 指標 | 定義 |
|------|------|
| MTBF | 平均故障間隔時間 |
| MTTR | 平均修復時間 |
| Availability | MTBF/(MTBF+MTTR) |

## 系統思考

- 整體大於部分之和
- 回饋迴路決定系統行為
- 短期最佳化可能損害長期效能

> 💡 系統工程的核心不是技術深度，而是整合廣度——把已經存在的技術正確地組合起來。
