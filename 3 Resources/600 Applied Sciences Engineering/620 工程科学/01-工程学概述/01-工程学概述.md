# 01-工程学概述

> 工程是在约束条件下设计、建造和维护系统的学科。工程师不是"知道最多科学的人"，而是"能在不完全信息下做出安全有效决策的人"。

## 工程思维 vs 科学思维

| 维度 | 科学思维 | 工程思维 |
|------|----------|----------|
| 目标 | 发现真理 | 解决实际问题 |
| 约束 | 尽量控制变量 | 接受所有约束同时存在 |
| 失败 | 实验失败 = 学到东西 | 工程失败 = 可能有人死亡 |
| 最优解 | 没有"最优真理" | 必须在多目标间权衡 |
| 时间 | 没有截止日期 | 预算和工期是硬约束 |

## 工程设计过程

```
需求定义 → 概念设计 → 详细设计 → 原型 → 测试 → 生产 → 运维
   ↑______________________________________________________↓
                        迭代反馈
```

### 设计中的关键概念

| 概念 | 含义 | 例子 |
|------|------|------|
| **安全系数** | 设计强度 / 预期负载 | 电梯钢缆安全系数 ~10-12 |
| **冗余** | 关键系统有备份 | 飞机多个液压系统 |
| **容错** | 部分故障不影响整体 | RAID 磁盘阵列 |
| **防错 (Poka-yoke)** | 设计使人不可能犯错 | USB 只能单向插入（虽然经常失败…） |

## 工程伦理

| 原则 | 含义 |
|------|------|
| 公众安全第一 | 首要责任是保护公众健康和安全 |
| 诚实 | 只在自己能力范围内工作 |
| 可持续发展 | 考虑项目的长期环境影响 |
| 举报义务 | 当公众安全受威胁时，有义务上报 |

### 何时"说不"

> 挑战者号灾难 (1986) 的核心教训：工程师 Morton Thiokol 建议不要低温发射，但管理层推翻了技术判断。工程师要有勇气和能力说"这个设计不安全"。
