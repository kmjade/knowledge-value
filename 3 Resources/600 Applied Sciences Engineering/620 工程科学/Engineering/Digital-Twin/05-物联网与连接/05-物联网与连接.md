# 05-物联网与连接

> 没有数据连接，数字孪生就是死的。物联网 (IoT) 是物理世界和数字孪生之间的桥梁——从传感器到边缘到云端。

## 连接架构

```
传感器 → 边缘网关 → 消息队列 → 流处理 → 时序数据库 → 数字孪生
   ↓         ↓                     ↓
 有线/无线  协议转换              批处理→数据湖
```

## 传感器网络

| 类型 | 协议 | 特点 |
|------|------|------|
| **有线** | 4-20mA, Modbus RTU, IO-Link | 高可靠、有供电 |
| **无线短距** | WiFi, Zigbee, BLE | 灵活、低功耗 |
| **无线广域** | LoRaWAN, NB-IoT, 5G | 远距离、低功耗 |
| **光纤** | FBG (光纤光栅) | 抗电磁干扰、长距离、分布式 |

### 传感器选型考量

| 因素 | 说明 |
|------|------|
| 采样率 | 振动监测需要 kHz 级，温度监测 1Hz 就够了 |
| 精度 | 越精确越贵——确定真正需要的精度 |
| 环境 | 温度、湿度、粉尘、振动、防爆 |
| 供电 | 有线供电 vs 电池（影响采样率） |
| 安装 | 改造现有设备——停机时间和成本 |

## 边缘计算

在靠近数据源的地方处理——不全部传到云端：

| 功能 | 价值 |
|------|------|
| 数据预处理 | 过滤、聚合、压缩——减少 90% 的上传流量 |
| 协议转换 | Modbus→MQTT、OPC UA |
| 实时响应 | 毫秒级本地决策——不依赖云端延迟 |
| 离线缓冲 | 断网时暂存数据，恢复后补传 |
| 运行轻量 ML | 异常检测、简单预测 |

> 边缘计算的关键权衡：什么必须在现场处理（安全停机），什么可以传到云端（复杂分析、历史对比）。

## 消息协议

| 协议 | 特点 | 适用 |
|------|------|------|
| **MQTT** | 轻量、发布/订阅、QoS | IIoT 首选 |
| **OPC UA** | 工业语义、安全 | 工业自动化 |
| **AMQP** | 企业级 | 金融、大型集成 |
| **HTTP/WebSocket** | 通用 | Web 应用 |

## 连接安全

| 层面 | 措施 |
|------|------|
| 设备认证 | X.509 证书、安全启动 |
| 传输加密 | TLS/DTLS |
| 网络隔离 | VLAN、防火墙、DMZ |
| 访问控制 | 最小权限原则 |
