# 02-核心技术栈

> 数字孪生不是单一技术——是物联网、云计算、AI/ML、仿真和可视化的融合。

## 技术栈全景

```
┌───────────────────────────────────────┐
│           可视化与交互                  │  3D引擎、AR/VR、Dashboard
├───────────────────────────────────────┤
│           分析与智能                    │  AI/ML、规则引擎、优化
├───────────────────────────────────────┤
│           仿真与建模                    │  多物理场、降阶模型、实时仿真
├───────────────────────────────────────┤
│           数据管理与集成                │  数据湖、语义模型、数字线程
├───────────────────────────────────────┤
│           连接与采集                    │  IoT、传感器、PLC、边缘计算
└───────────────────────────────────────┘
```

## 各层详解

### 连接与采集

| 技术 | 作用 |
|------|------|
| **传感器** | 温度、振动、压力、电流——数据的源头 |
| **PLC/SCADA** | 工业数据的传统出口 |
| **IoT 网关** | 协议转换（Modbus→MQTT）、边缘预处理 |
| **5G/WiFi 6** | 高带宽低延迟的无线连接 |

### 数据管理与集成

| 技术 | 作用 |
|------|------|
| **数据湖/数据仓库** | 存储海量时序数据 |
| **语义模型/本体** | 给数据赋予含义——不只是"值为 85"，而是"这是 3 号线灌装站的出口温度" |
| **AAS** (Asset Administration Shell) | 工业 4.0 的标准信息模型 |
| **OPC UA** | 不仅传输数据，还传输数据的语义 |

### 仿真与建模

| 技术 | 说明 |
|------|------|
| **多物理场仿真** (FEA/CFD) | 结构、流体、热、电磁——精确但慢 |
| **降阶模型 (ROM)** | 将复杂仿真模型压缩为实时可运行的简化版本 |
| **1D 系统仿真** | 整系统行为——比 3D 快，精度可接受 |
| **数据驱动模型** | 用 ML 从历史数据学习——替代物理模型 |

### 分析与智能

| 技术 | 应用 |
|------|------|
| 异常检测 | 自动发现异常运行状态 |
| 预测性维护 | 预测剩余使用寿命 (RUL) |
| 根因分析 | 故障发生时自动追溯根因 |
| 优化算法 | 在数字孪生上跑优化→最优参数下发物理设备 |

### 可视化与交互

| 技术 | 用途 |
|------|------|
| **3D 引擎** (Unity, Unreal, Omniverse) | 照片级实时渲染 |
| **AR (增强现实)** | 维修时通过手机/眼镜看到叠加的运行数据 |
| **VR (虚拟现实)** | 培训操作员在虚拟厂房中演练 |
| **Web Dashboard** | 轻量级 2D 监控——最实用 |

## 技术组合示例

### 工业设备数字孪生

```
设备传感器 → IoT网关(MQTT) → 时序数据库(InfluxDB/TimescaleDB)
                                    ↓
            3D可视化(Unity)  ←  分析(ML模型)  ←→  仿真(降阶模型)
```

### 建筑数字孪生

```
BIM模型(Revit) + IoT传感器 → Azure Digital Twins
                                    ↓
                    能耗分析 → HVAC优化控制 → 返回BACnet指令
```
