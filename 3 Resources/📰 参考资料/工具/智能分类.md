## 分类规则

### Projects（有明确目标和截止日期）

**特征**：
- 有明确的完成目标
- 有明确的截止日期
- 是短期活跃的项目

**判断标准**：
- 问题："这件事要在什么时候完成？"
- 是 → `1 Projects/`

**示例**：
- "完成季度报告" - 有截止日期
- "准备产品发布会" - 有明确目标
- "学习 Python 一个月" - 有时间限制

---

### Areas（长期责任领域）

**特征**：
- 没有明确的截止日期
- 需要长期维护和改进
- 是一种持续的责任

**判断标准**：
- 问题："这件事需要永远做下去吗？"
- 是 → `2 Areas/`

**示例**：
- "健康管理" - 长期责任
- "财务管理" - 持续领域
- "人际关系维护" - 长期维护

---

### Resources（感兴趣的话题）

**特征**：
- 有趣但不需要立即行动
- 纯粹出于兴趣收集
- 未来可能有用的信息

**判断标准**：
- 问题："这件事我感兴趣，但不需要行动？"
- 是 → `3 Resources/`

**示例**：
- "人工智能最新研究" - 感兴趣的话题
- "投资理财知识" - 学习资源
- "摄影技巧" - 兴趣爱好

---

### Archives（已完成或不活跃）

**特征**：
- 项目已完成
- 兴趣已转移
- 不再需要频繁访问

**判断标准**：
- 问题："这件事已经结束或不相关了吗？"
- 是 → `4 Archives/`

**示例**：
- "旧项目总结" - 已完成
- "过时的技术文档" - 已不相关
- "已完成的培训" - 已结束

## 自动分类建议

### 根据标签分类

| 标签 | 建议分类 |
|------|----------|
| `#project` / `#项目` | `1 Projects/` |
| `#area` / `#领域` | `2 Areas/` |
| `#resource` / `#资源` | `3 Resources/` |
| `#archive` / `#归档` | `4 Archives/` |

### 根据文件名分类

| 模式 | 建议分类 |
|------|----------|
| 包含 "项目" / "Project" | `1 Projects/` |
| 包含 "领域" / "Area" | `2 Areas/` |
| 包含 "资源" / "Resource" | `3 Resources/` |
| 包含 "归档" / "Archive" | `4 Archives/` |

### 根据内容分类

**高优先级指标**：
- 有 `due_date` 字段 → `1 Projects/`
- 有 `status: in-progress` → 可能是 Projects
- 包含 "目标" / "目标" → 可能是 Projects

**中优先级指标**：
- 标签包含 `area` → `2 Areas/`
- 内容涉及长期责任 → `2 Areas/`

**低优先级指标**：
- 内容为学习笔记 → `3 Resources/`
- 内容为参考资料 → `3 Resources/`

## 分类流程

### 1. 分析内容

```dataview
TABLE file.link as "文件", file.mtime as "修改时间"
FROM "0 Personals/📥 00_InBox"
WHERE file.name != "📥 00_InBox.md"
SORT file.mtime DESC
```

### 2. 评估分类

对每个文件评估：

1. **有截止日期吗？**
   - 是 → Projects
   - 否 → 继续

2. **有明确目标吗？**
   - 是 → Projects
   - 否 → 继续

3. **是长期责任吗？**
   - 是 → Areas
   - 否 → 继续

4. **是感兴趣的话题吗？**
   - 是 → Resources
   - 否 → Archives

### 3. 确认并移动

确认分类后移动到对应文件夹：

```
1 Projects/     # 短期项目
2 Areas/        # 长期领域
3 Resources/    # 感兴趣话题
4 Archives/     # 已完成项目
```

## 常见问题

### Q1: 某个内容符合多个分类？

**A**: 选择最近的时间跨度
- 项目结束日期近 → Projects
- 需要持续维护 → Areas
- 只是收藏 → Resources

### Q2: 不知道如何分类怎么办？

**A**:
1. 先放入 `3 Resources/`
2. 设置提醒，一周后重新评估
3. 如果需要行动，移到 Projects
4. 如果不再需要，移到 Archives

### Q3: 分类后可以修改吗？

**A**: 可以！PARA 是动态的
- 项目结束 → 移到 Archives
- 兴趣变成项目 → 移到 Projects
- 资源变成领域 → 移到 Areas
