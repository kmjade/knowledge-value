## 為什麼學習 PARA？

### PARA 能解決的問題

| 問題 | PARA 解決方案 |
|------|--------------|
| 筆記太多找不到 | 按行動價值分類 |
| 不知道該放哪裡 | 四個類別快速決策 |
| 筆記堆積灰塵 | 定期歸檔清理 |
| 難以追蹤進度 | 項目化追蹤 |
| 壓力大不知從何開始 | 優先順序清晰 |

### 預期成果

- **Week 1**: 整理現有筆記，建立基礎結構
- **Week 2**: 養成每日檢視習慣
- **Week 4**: 流暢使用 PARA 工作流
- **Month 3**: 完全自動化 PARA 管理

## 實施計劃

### 實施前準備

#### 1. 清理現有筆記（Day 0）

**步驟：**
1. 導出所有筆記列表
2. 建立收件匣資料夾 `Inbox/`
3. 移動所有筆記到 Inbox
4. 從零開始分類

**為什麼要這樣做？**
- 重新審視每個筆記的價值
- 避免「帶著舊習慣進入新系統」
- 確保進入系統的筆記都是經過思考的

#### 2. 設定工具

**必備設定：**
- [ ] Templater 外掛
- [ ] QuickAdd 外掛（可選）
- [ ] Dataview 外掛
- [ ] 快捷鍵設定

**建議快捷鍵：**
| 快捷鍵 | 功能 |
|--------|------|
| `Ctrl+Shift+N` | 新建 Project |
| `Ctrl+Shift+A` | 新建 Area |
| `Ctrl+Shift+R` | 新建 Resource |

## Resources
```dataview

```dataview

```

## 統計

```

#### Step 3：首次分類

```
時間：2-4 小時（取決於筆記數量）

策略：
1. 每次處理一個資料夾
2. 問自己三個問題：
   - 需要行動嗎？→ Project
   - 是長期責任嗎？→ Area
   - 是感興趣的主題嗎？→ Resource
   - 都不需要？→ Archive 或刪除
3. 移動後立即添加標籤
```

#### Step 4：建立檢視提醒

**每日提醒（早晨）：**
- 檢查 Projects 進度
- 確認今日任務

**每週提醒（週日）：**
- 每週檢視
- 更新狀態
- 規劃下週

**每月提醒（每月初）：**
- 月度檢視
- 歸檔已完成項目
- 評估 Areas

## 常見障礙與解決

### 障礙 1：不知道該放哪裡

**症狀：**
- 花太多時間分類
- 筆記一直在 Inbox

**解決方案：**
1. 設定「兩分鐘規則」：2 分鐘內決定
2. 建立「待處理」清單，定期批量分類
3. 記住：沒有完美分類，放錯可以移動

### 障礙 2：忘記檢視

**症狀：**
- 一週沒有打開 PARA
- 筆記開始堆積

**解決方案：**
1. 設定提醒（手機/電腦）
2. 綁定現有習慣（早晨咖啡後）
3. 從簡單開始（每天 2 分鐘）
4. 追蹤執行率，看到進步

### 障礙 3：過度複雜化

**症狀：**
- 分類太多層次
- 模板太複雜
- 不想使用

**解決方案：**
1. 回到基本：只有四個類別
2. 簡化模板：只保留必要字段
3. 先用起來，再優化

### 障礙 4：維持動力

**症狀：**
- 初期很有熱情
- 幾週後放棄

**解決方案：**
1. 設定小目標，看到進步
2. 記錄成功案例
3. 找到 PARA 帶來的實際好處
4. 社群支持（論壇、Discord）

### 障礙 5：舊筆記太多

**症狀：**
- 有數百個未分類筆記
- 不知從何開始

**解決方案：**
1. 全部移動到 Inbox
2. 設定每天處理數量（如 20 個）
3. 批量移動，不要逐個思考
4. 大膽刪除或歸檔

### 障礙 6：家庭/工作筆記混雜

**症狀：**
- 想要分開管理
- 但只有一個 vault

**解決方案：**
1. 在 Projects/Areas 下建立子資料夾
2. 使用標籤區分：`#work`、`#personal`
3. 建立兩個 INDEX 頁面
4. 使用 DataView 過濾

3. **維護**：定期檢視和歸檔
4. **改進**：持續優化流程
