## 分類规则

### Projects（有明确目標和截止日期）

**特征**：
- 有明确的完成目標
- 有明确的截止日期
- 是短期活跃的專案

**判斷標準**：
- 問題："这件事要在什么时候完成？"
- 是 → `1 Projects/`

**示例**：
- "完成季度報告" - 有截止日期
- "准备产品發佈会" - 有明确目標
- "學習 Python 一个月" - 有時間限制

---

### Areas（长期责任领域）

**特征**：
- 没有明确的截止日期
- 需要长期維護和改进
- 是一种持續的责任

**判斷標準**：
- 問題："这件事需要永远做下去吗？"
- 是 → `2 Areas/`

**示例**：
# 管理
# 管理
- "人际关系維護" - 长期維護

---

### Resources（感兴趣的话题）

**特征**：
- 有趣但不需要立即行动
- 纯粹出于兴趣收集
- 未来可能有用的資訊

**判斷標準**：
- 問題："这件事我感兴趣，但不需要行动？"
- 是 → `3 Resources/`

**示例**：
- "人工智能最新研究" - 感兴趣的话题
# 知識
- "摄影技巧" - 兴趣爱好

---

### Archives（已完成或不活跃）

**特征**：
- 專案已完成
- 兴趣已转移
- 不再需要频繁访问

**判斷標準**：
- 問題："这件事已经結束或不相關了吗？"
- 是 → `4 Archives/`

**示例**：
- "旧專案總結" - 已完成
- "过时的技術文檔" - 已不相關
- "已完成的培训" - 已結束

## 自動分類建議

### 根据標籤分類

| 標籤 | 建議分類 |
|------|----------|
| `#project` / `#專案` | `1 Projects/` |
| `#area` / `#领域` | `2 Areas/` |
| `#resource` / `#資源` | `3 Resources/` |
| `#archive` / `#歸檔` | `4 Archives/` |

### 根据檔案名分類

| 模式 | 建議分類 |
|------|----------|
| 包含 "專案" / "Project" | `1 Projects/` |
| 包含 "领域" / "Area" | `2 Areas/` |
| 包含 "資源" / "Resource" | `3 Resources/` |
| 包含 "歸檔" / "Archive" | `4 Archives/` |

### 根据內容分類

**高優先級指标**：
- 有 `due_date` 字段 → `1 Projects/`
- 有 `status: in-progress` → 可能是 Projects
- 包含 "目標" / "目標" → 可能是 Projects

**中優先級指标**：
- 標籤包含 `area` → `2 Areas/`
- 內容涉及长期责任 → `2 Areas/`

**低優先級指标**：
- 內容为學習筆記 → `3 Resources/`
- 內容为參考资料 → `3 Resources/`

## 分類流程

# 分析

```dataview
# 修改
FROM "0 Personals/📥 00_InBox"
WHERE file.name != "📥 00_InBox.md"
SORT file.mtime DESC
```

### 2. 評估分類

对每个檔案評估：

1. **有截止日期吗？**
   - 是 → Projects
   - 否 → 繼續

2. **有明确目標吗？**
   - 是 → Projects
   - 否 → 繼續

3. **是长期责任吗？**
   - 是 → Areas
   - 否 → 繼續

4. **是感兴趣的话题吗？**
   - 是 → Resources
   - 否 → Archives

### 3. 確認并移动

確認分類后移动到对应資料夾：

```
1 Projects/     # 短期專案
2 Areas/        # 长期领域
3 Resources/    # 感兴趣话题
4 Archives/     # 已完成專案
```

## 常见問題

### Q1: 某个內容符合多個分類？

**A**: 選擇最近的時間跨度
- 專案結束日期近 → Projects
- 需要持續維護 → Areas
# 收藏

### Q2: 不知道如何分類怎么办？

**A**:
1. 先放入 `3 Resources/`
2. 設置提醒，一周后重新評估
3. 如果需要行动，移到 Projects
4. 如果不再需要，移到 Archives

# 修改

**A**: 可以！PARA 是動態的
- 專案結束 → 移到 Archives
- 兴趣变成專案 → 移到 Projects
- 資源变成领域 → 移到 Areas
