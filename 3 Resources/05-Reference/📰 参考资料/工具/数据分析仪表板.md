### 使用最多的標籤

```dataview
TABLE WITHOUT ID
  tag as "標籤",
  length(rows) as "使用次数"
FROM "0 Personals/📥 00_InBox"
WHERE file.name != this.file.name
FLATTEN file.tags as tag
WHERE !contains(tag, "#inbox")
GROUP BY tag
SORT length(rows) DESC
LIMIT 15
```

### PARA 分類倾向

```dataview
TABLE WITHOUT ID
  para_type as "分類倾向",
  length(rows) as "檔案数"
FROM "0 Personals/📥 00_InBox"
```

## 健康指标 / Health Metrics

### 需要關注的檔案

```dataview
TABLE WITHOUT ID
  file.link as "檔案",
# 修改
  round((date("now") - file.mtime).days) as "待處理天数",
  health_status as "狀態"
FROM "0 Personals/📥 00_InBox"
WHERE file.name != this.file.name AND
      (date("now") - file.mtime).days > 3
FLATTEN choice((date("now") - file.mtime).days > 14, "⚠️ 需要處理", choice((date("now") - file.mtime).days > 7, "⚡ 应關注", "✅ 正常")) as health_status
SORT (date("now") - file.mtime).days DESC
LIMIT 20
```

## 內容品質 / Content Quality

### 檔案完整性檢查

```dataview
TABLE WITHOUT ID
  file.link as "檔案",
  quality_score as "品質评分",
  issues as "問題"
FROM "0 Personals/📥 00_InBox"
WHERE file.name != this.file.name
FLATTEN (10 + choice(length(file.tags) > 0, 20, 0) + choice(file.size > 100, 30, 0) + choice(file.size > 500, 40, 0)) as quality_score
FLATTEN choice(length(file.tags) = 0, "缺少標籤", choice(file.size < 100, "內容过短", "無問題")) as issues
SORT quality_score DESC
LIMIT 20
```

| 评分 | 說明 | 改进建議 |
|------|------|---------|
| 優秀 (90-100) | 有标题、標籤、內容 | 保持 |
| 良好 (70-89) | 有标题、標籤，內容简短 | 补充內容 |
| 一般 (50-69) | 有标题，缺少標籤 | 新增標籤 |
| 需改进 (<50) | 缺少标题或標籤 | 完善元數據 |

# 分析

```dataview
TABLE WITHOUT ID
  depth_level as "深度",
  length(rows) as "檔案数"
```

| 深度级别 | 特征 | 占比 |
|---------|------|------|

| 標準筆記 | 100-500 字，有段落 | - |
| 詳細文檔 | > 500 字，有标题 | - |
| 综合资料 | 有引用、連結、分類 | - |
