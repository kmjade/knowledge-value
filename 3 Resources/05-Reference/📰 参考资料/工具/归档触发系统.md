> [!note] 归档触发系统 / Archive Trigger System
> 自动或手动触发 PARA 内容的归档流程

## 触发条件

### 自动触发条件

| 条件 | 说明 | 处理方式 |
|------|------|---------|
| 项目完成 | Projects 中所有任务完成时 | 移动到 Archives |
| 项目取消 | status 设为 cancelled 时 | 移动到 Archives |
| 过期未处理 | InBox 中超过 7 天未处理 | 标记需要处理 |
| 定期归档 | 每周自动归档完成的 InBox 内容 | 按周创建归档记录 |

### 手动触发

```
┌────────────────────────────────────────┐
│  📥 触发归档                      │
│  ↓ 选择归档类型                     │
│  ┌────────────────────────────────┐   │
│  │  [ ] 项目完成归档                 │
│  │  [ ] 手动归档                   │
│  │  [ ] 批量归档                   │
│  │  [ ] 清空 InBox                  │
│  └────────────────────────────────┘   │
│  ↓ 创建归档记录                     │
│  ┌────────────────────────────────┐   │
│  │  ✅ 归档记录已创建            │
│  └────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

## 归档记录格式

### 归档项目

```
---
title: "2024-W1-归档记录"
archived: 2024-01-18
original_category: projects
items:
  - [[1 Projects/测试项目]]
reason: 项目完成
notes: 项目于 2024-01-15 完成
---
```

### 归档资源

```
---
title: "2024-W1-归档记录"
archived: 2024-01-18
original_category: resources
items:
  - [[3 Resources/测试资源]]
reason: 已处理，不再需要保留
notes: 信息已提取到笔记中
---
```

## 归档回顾

创建归档回顾命令 `/para-归档回顾`：
- 查看最近归档记录
- 评估归档策略
- 恢复已归档的内容

---

> [!tip] 归档管理建议
> - 归档前提取有价值内容到 PARA
> - 记录归档原因便于未来查找
> - 定期回顾归档内容，清理无用文件
> - 归档完成后清理 InBox 原内容
