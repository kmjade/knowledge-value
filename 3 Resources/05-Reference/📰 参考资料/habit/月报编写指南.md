# 月报编写指南

> 复盘每一月，成长每一步。

---

## 为什么写月报？

月报是深度复盘的最佳周期：

| 作用 | 说明 |
|------|------|
| **趋势洞察** | 发现跨周的模式和长期趋势 |
| **目标校准** | 检查月度目标完成情况 |
| **习惯检验** | 评估习惯养成进度 |
| **资源配置** | 分析时间和精力分配 |
| **战略调整** | 根据数据调整方向 |
| **成长记录** | 积累长期数据资产 |

---

## 何时写月报？

### 推荐时间

| 时间 | 场景 | 优点 |
|------|------|------|
| **月末最后一天** (20:00-22:00) | 复盘整月，规划下月 | 数据完整，衔接流畅 |
| **月初第一天** (9:00-11:00) | 开启新月 | 充满能量，方向清晰 |

### 核心原则

1. **固定时间** - 每月固定一天进行
2. **数据驱动** - 基于周报数据
3. **深度复盘** - 不只是汇总，要分析
4. **连接长期** - 与年度目标对齐

---

## 月报结构

```
1. 本月概览       → 一页纸了解整月
2. Projects      → 项目进展与评估
3. Areas         → 领域状态与长期目标
4. Resources     → 学习与资源积累
5. 反思与收获    → 深度思考
6. 数据分析      → 量化自我
7. 下月计划      → 明确行动
8. 主题复盘      → 月初目标回顾
```

### 各模块说明

#### 1. 本月概览
- **关键指标** - 与上月对比
- **本月主题** - 用一个词概括本月
- **关键成果** - 最重要的3个成就

#### 2. Projects
- **本月完成** - 项目清单和贡献
- **进行中** - 当前进度
- **新启动** - 新项目计划
- **阻塞问题** - 分析与解决

#### 3. Areas
- **本月关注** - 各领域的进展
- **长期目标** - 年度目标进度
- **待改进** - 发现的问题

#### 4. Resources
- **新增资源** - 收藏和积累
- **学习完成** - 本月学习成果
- **技能提升** - 能力增长

#### 5. 反思与收获
- **本月亮点** - 做对了什么
- **改进空间** - 可以如何优化
- **学习心得** - 认知升级
- **困惑与突破** - 成长卡点
- **认知升级** - 思维转变

#### 6. 数据分析
- **习惯养成** - 月度习惯追踪
- **目标达成** - 目标完成率
- **时间分配** - 时间和精力分配

#### 7. 下月计划
- **优先级高** - 必须完成的任务
- **优先级中** - 计划完成的任务
- **新项目** - 计划启动的项目
- **习惯目标** - 下月习惯目标

#### 8. 主题复盘
- **月初目标回顾** - 对照检查
- **下月主题设定** - 聚焦重点

1. 打开 `_Template/_template-monthly-report.md`
2. 复制全部内容
3. 创建新笔记，粘贴内容
4. 修改占位符

## 月报与周报的关系

```
日报 → 周报 → 月报 → 季报 → 年报
```

### 数据流向

| 周报内容 | 月报提取 |
|----------|----------|
| 每周任务完成 | 月度任务汇总 |
| 每周反思 | 本月亮点/改进 |
| 每周计划 | 下月计划 |
| 每周数据 | 月度趋势分析 |

### 最佳实践

**周报记录时**：
- 完成周报，勿拖延
- 标注关键成果和问题
- 保持数据一致性

**月报生成时**：
- 回顾4篇周报
- 提取关键数据
- 分析趋势和模式
- 提炼洞察

---

## 最佳实践

### ✅ 推荐做法

- **月末最后一天** - 数据最完整
- **基于周报数据** - 准确可靠
- **深度分析** - 不只是汇总
- **诚实复盘** - 不回避问题
- **设定下月主题** - 聚焦重点
- **连接年度目标** - 保持对齐

### ❌ 避免做法

- **拖延到下月初** - 数据遗忘
- **只看最后一周** - 以偏概全
- **流于形式** - 没有深度
- **只报喜不报忧** - 失去意义
- **写完不执行** - 流于表面

## 常见问题

### Q1: 周报没写齐怎么写月报？

**A**:
1. 诚实标注"数据不完整"
2. 基于已有周报分析
3. 反思为什么周报缺失
4. 下月改进周报习惯

### Q2: 月报太长了怎么办？

**A**: 精简原则
- 成果不超过5项
- 问题不超过3个
- 计划不超过5项
- 总字数控制在 1500-2500 字

### Q3: 如何从月报看趋势？

**A**: 对比分析
- **横向对比** - 与上月对比
- **纵向对比** - 与历史同期对比
- **趋势判断** - 是在进步还是退步？
- **归因分析** - 为什么会有这个趋势？

### Q4: 月报要给别人看吗？

**A**: 视情况而定
- **个人复盘** - 重点写给自己看
- **团队同步** - 突出成果和计划
- **向上汇报** - 调整结构和侧重点

## 12月月报挑战

尝试连续12个月写月报：

| 季度 | 月份 | 目标 | 重点 |
|------|------|------|------|
| **Q1** | 1-3月 | 建立习惯 | 每月坚持完成 |
| **Q2** | 4-6月 | 数据积累 | 开始趋势分析 |
| **Q3** | 7-9月 | 深度复盘 | 分析原因模式 |
| **Q4** | 10-12月 | 优化迭代 | 调整和提升 |

完成一年月报后，撰写年度总结。
