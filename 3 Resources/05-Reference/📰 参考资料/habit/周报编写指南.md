# 周报编写指南

> 回顾每一周，成长每一天。

---

## 为什么写周报？

周报是连接日报和月度复盘的重要桥梁：

| 作用 | 说明 |
|------|------|
| **宏观视角** | 从每日细节跳脱出来，看清整体进展 |
| **趋势分析** | 发现连续几周的模式和趋势 |
| **重点聚焦** | 识别本周最重要的3件事 |
| **计划优化** | 根据周数据调整下周计划 |
| **沟通工具** | 向上汇报、团队同步的好帮手 |

---

## 何时写周报？

### 推荐时间

| 时间 | 场景 | 优点 |
|------|------|------|
| **周日晚上** (20:00-21:00) | 回顾本周，规划下周 | 有完整数据，衔接新一周 |
| **周一早晨** (8:00-9:00) | 开启新一周 | 充满能量，方向清晰 |

### 核心原则

1. **固定时间** - 养成稳定的周回顾习惯
2. **数据驱动** - 基于日报数据，而非凭印象
3. **简洁高效** - 15-30分钟完成
4. **重点突出** - 聚焦关键成果和关键问题

---

## 周报结构

```
1. 本周概览       → 一页纸了解整周
2. Projects      → 项目进展与问题
3. Areas         → 领域关注与改进
4. Resources     → 资源与学习动态
5. 反思与收获    → 深度思考
6. 下周计划      → 明确行动
7. 数据追踪      → 量化自我
```

### 各模块说明

#### 1. 本周概览
- **关键指标** - 完成任务数、专注时长等
- **本周主题** - 用一句话概括本周焦点
- **关键成果** - 最重要的3个成就

#### 2. Projects
- **本周完成** - 列清单，勾选状态
- **进行中** - 当前进度百分比
- **阻塞/延期** - 问题分析与解决

#### 3. Areas
- **本周关注** - 各领域的进展
- **待改进** - 发现的问题

#### 4. Resources
- **新增资源** - 收藏的文章、工具等
- **学习完成** - 本周学到的知识
- **待学习** - 学习计划清单

#### 5. 反思与收获
- **本周亮点** - 做对了什么
- **改进空间** - 可以如何优化
- **学习心得** - 认知升级
- **待解决问题** - 困惑与卡点

#### 6. 下周计划
- **高优先级** - 必须完成
- **中优先级** - 尽量完成
- **低优先级** - 有空就做

#### 7. 数据追踪
- **习惯养成** - 周习惯完成率
- **目标进度** - 长期目标进展

## 使用模板

### 方法一：Templater 插件（推荐）

1. 按 `Ctrl/Cmd + P` 打开命令面板

## 周报与日报的关系

```
日报（每日） → 周报（每周） → 月报（每月） → 年报（每年）
```

### 数据流向

| 日报内容 | 周报提取 |
|----------|----------|
| 每日完成任务 | 本周完成清单 |
| 每日反思 | 本周亮点/改进 |
| 明日计划 | 下周计划 |
| 情绪状态 | 本周状态趋势 |

### 最佳实践

**日报记录时**：
- 使用标签分类（#project, #area, #resource）
- 填写 `status` 字段（in-progress, completed）
- 记录 `due_date` 便于追踪

**周报生成时**：
- 从日报提取结构化数据
- 关注趋势和模式
- 提炼关键洞察

---

## 最佳实践

### ✅ 推荐做法

- **周日生成** - 数据完整，时间充裕
- **基于日报** - 减少记忆偏差
- **量化追踪** - 用数据说话
- **连接PARA** - 同步项目状态
- **诚实复盘** - 不回避问题
- **设定下周** - 明确优先级

### ❌ 避免做法

- **拖延到周一** - 数据可能遗忘
- **凭印象写** - 不准确
- **只报喜不报忧** - 失去复盘意义
- **流水账** - 没有重点
- **写完就忘** - 不对照执行

## 进阶技巧

### 周报汇总

从周报生成月度汇总：

```
本月主题：
├── 本周完成任务数：X
├── 项目进展：
│   ├── 项目A：X%
│   └── 项目B：X%
├── 习惯养成：
│   ├── 写日报：X/30天
│   └── 整理InBox：X/4周
├── 关键学习：
│   ├── 学习主题1
│   └── 学习主题2
└── 下月重点：
    ├── 重点1
    └── 重点2
```

### 标签系统

| 标签 | 用途 |
|------|------|
| `#weekly-report` | 所有周报 |
| `#本周完成` | 已完成任务 |
| `#下周计划` | 待办计划 |
| `#项目进展` | Project 更新 |
| `#领域关注` | Area 更新 |

### 周报回顾

每月回顾本月4篇周报：

1. **横向对比** - 找出模式
2. **趋势分析** - 进步还是退步？
3. **目标校准** - 调整月度目标
4. **经验萃取** - 总结成功/失败原因

## 常见问题

### Q1: 周报和日报内容重复怎么办？

**A**: 周报是日报的提炼和升华
- 日报：记录细节
- 周报：萃取精华
- 相同内容用"详见X月X日日报"概括

### Q2: 一周没写几篇日报怎么写周报？

**A**:
1. 诚实标注"数据不完整"
2. 重点写有记录的那几天
3. 反思为什么没写日报
4. 设置下周改进目标

### Q3: 周报太长了怎么办？

**A**: 精简原则
- 成果不超过3项
- 问题不超过3个
- 计划不超过5项
- 总字数控制在 800-1500 字

### Q4: 可以用周报向上级汇报吗？

**A**: 可以，但需要调整
- 增加业务指标
- 减少个人反思
- 突出团队贡献
- 明确下周目标

## 52周周报挑战

尝试连续52周写周报：

| 阶段 | 周次 | 目标 | 重点 |
|------|------|------|------|
| **第一阶段** | 第1-4周 | 建立习惯 | 每周坚持 |
| **第二阶段** | 第5-12周 | 数据追踪 | 开始量化 |
| **第三阶段** | 第13-26周 | 深度反思 | 分析趋势 |
| **第四阶段** | 第27-40周 | 优化流程 | 提高效率 |
| **第五阶段** | 第41-52周 | 年度总结 | 萃取经验 |

完成一年周报后，撰写一篇年度回顾。
