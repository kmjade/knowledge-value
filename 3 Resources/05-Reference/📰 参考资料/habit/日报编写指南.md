# 日报编写指南

> 记录每一天，成长每一周。

---

## 为什么写日报？

日报是个人成长的重要工具，帮助您：

- **保持专注** - 每天明确最重要的任务
- **追踪进展** - 可视化您的进步轨迹
- **深度反思** - 从经验中学习，持续改进
- **建立习惯** - 通过规律记录培养自律

---

## 何时写日报？

### 推荐时间

| 时间 | 场景 |
|------|------|
| **晚间** (21:00-22:00) | 回顾一天完成的事项，规划明日 |
| **次日早晨** (7:00-8:00) | 回顾昨日，总结今日计划 |

### 核心原则

1. **固定时间** - 选择适合自己的时段，坚持执行
2. **简短精炼** - 5-10分钟完成，避免负担过重
3. **真实记录** - 如实反映情况，不夸大不隐瞒
4. **持续行动** - 每天记录，哪怕只有一句话

---

## 日报结构

### 核心模块

```
1. 今日概览        → 快速了解今日整体状态
2. 今日完成        → 记录完成的任务和成果
3. 待办事项        → 追踪未完成事项和明日计划
4. 反思与收获      → 深度思考，萃取经验
5. 关联笔记        → 连接知识库，形成网络
```

### 各模块说明

#### 1. 今日概览
- 基础统计数据（笔记编辑数、完成任务数等）
- 今日情绪/状态评估
- 快速自检今日精力状况

#### 2. 今日完成
- **主要成果** - 最重要的3件事
- **已完成任务** - 勾选清单中的完成项
- 成就感来源，增强动力

#### 3. 待办事项
- **今日未完成** - 延续到明天
- **明日计划** - 优先级排序
- 避免任务遗漏，保持连续性

#### 4. 反思与收获
- **今日亮点** - 做得好的是什么
- **改进空间** - 可以如何优化
- **学习心得** - 新学到的知识
- **困惑与问题** - 待解决的疑问

#### 5. 关联笔记
- 今日编辑/创建的笔记链接
- 新增标签汇总
- 连接知识网络

## 使用模板

### 方法一：Templater 插件（推荐）

1. 按 `Ctrl/Cmd + P` 打开命令面板
2. 搜索 "Templater: Insert Template"
3. 选择 `_template-daily-report.md`
4. 模板会自动填入当前日期

### 方法二：手动复制

1. 打开 `_Template/_template-daily-report.md`
2. 复制全部内容
3. 创建新笔记，粘贴内容
4. 修改占位符（如日期）

### 方法三：快速捕获

对于简短的记录，可以使用：
- 闪念笔记格式 `💡 fleeting/`
- 快速记录灵感，无需完整模板

## 最佳实践

### ✅ 推荐做法

- **每天坚持** - 即使只写一句话也比不写好
- **诚实记录** - 真实反映情况，便于复盘
- **关注感受** - 记录情绪状态，了解自己
- **连接知识** - 与其他笔记建立链接
- **定期回顾** - 每周/每月回顾旧日报

### ❌ 避免做法

- **拖延** - 不要等到周末才补
- **流水账** - 避免无意义的细节
- **过度完美** - 不需要每次都写得很详尽
- **只记待办** - 反思和收获同样重要

## 进阶技巧

### 周报汇总

从日报中提取关键信息，生成周报：

```
本周主题：
├── 完成的任务数：X
├── 遇到的挑战：
│   ├── 挑战1
│   └── 挑战2
├── 学到的新知：
│   ├── 新知1
│   └── 新知2
└── 下周计划：
    ├── 计划1
    └── 计划2
```

### 标签系统

使用标签追踪特定主题：

| 标签 | 用途 |
|------|------|
| `#daily-report` | 所有日报 |
| `#今日完成` | 已完成任务 |
| `#待办` | 待办事项 |
| `#反思` | 反思内容 |
| `#学习` | 学习心得 |

### 数据分析

定期统计日报数据：

- 完成率 = 完成任务 / 计划任务
- 专注领域分布
- 情绪波动趋势
- 知识产出统计

## 常见问题

### Q1: 每天写日报太费时间怎么办？

**A**: 从简单开始。第一周只写3行：
1. 今天做了什么
2. 学到了什么
3. 明天要做什么

逐步增加内容深度。

### Q2: 有时候一天什么都没完成，怎么写？

**A**: 这种情况更有记录价值！
- 诚实记录状态
- 分析原因（身体不适？任务太难？干扰太多？）
- 调整第二天的计划

### Q3: 日报应该放在哪里？

**A**: 建议存放在 `0 Personals/Journal/` 文件夹中，与收件箱、项目笔记分开，便于集中管理。

### Q4: 如何避免日报变成流水账？

**A**: 聚焦于：
- **成果** 而非活动
- **反思** 而非描述
- **问题** 而非现象

每次问自己："这对未来的我有什么价值？"

## 30天日报挑战

试试连续30天写日报，见证改变：

| 周次 | 目标 | 重点 |
|------|------|------|
| 第1周 | 每天记录 | 建立习惯，坚持最重要 |
| 第2周 | 增加反思 | 开始深度思考 |
| 第3周 | 连接知识 | 与其他笔记建立链接 |
| 第4周 | 优化流程 | 根据经验调整模板 |

完成挑战后，回顾这30天的日报，撰写一篇总结。
