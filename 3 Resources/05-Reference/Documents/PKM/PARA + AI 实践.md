### 什么是 PARA？

# 管理

| 分類 | 英文 | 時間跨度 | 可执行性 | 示例 |
|-----|------|---------|---------|------|
| **P**rojects | 專案 | 短期 | 高 | 完成某篇文章、装修房子 |
| **A**reas | 领域 | 长期 | 中 | 健康、财务、职业發展 |
| **R**esources | 資源 | 持續 | 低 | 學習资料、兴趣主題 |
| **A**rchives | 歸檔 | 过去 | 無 | 已完成的專案、过期的资料 |

### PARA 的核心原則

1. **按行动性分類** - 能现在做的事 vs 需要持續關注的
2. **按時間分類** - 短期專案 vs 长期责任
# 整理
4. **快速檢索** - 需要时能快速找到

## AI 如何增强 PARA

### 传统 PARA 的痛点

| 痛点 | 表现 |
|-----|------|
| 分類困难 | 一个筆記不知道放哪 |
# 整理
| 連結缺失 | 筆記间没有关联 |
| 維護成本高 | 需要持續人工投入 |

### AI 赋能 PARA

| PARA 环节 | AI 增强 | 效果 |
|-----------|---------|------|
| **收集** | 智能识别、自動打標籤 | 更快速、更准确 |
# 整理
| **檢索** | 语义搜尋、关联推荐 | 更精准、更智能 |
| **使用** | 自動生成、智能提取 | 更高效、更创造 |

# 工作流

### 1. 收集（Capture）- 無压輸入

**传统方式**
- 手動創建筆記
- 思考放在哪
- 手動新增標籤

**AI 增强方式**

```bash
# 直接記錄，無需分類
"今天开会讨论了新專案..."

# AI 自動處理
- 识别：这是專案相關內容
- 標籤：自動新增 [專案] [會議]
- 建議：可能适合放在「1 Projects」
```

**工具**: 快速 capture + AI 自動標記

# 整理

**传统方式**
```bash
# 手動分類
1. 阅读每个筆記
2. 思考：是專案？领域？資源？
3. 移动到对应資料夾
4. 手動建立連結
```

**AI 增强方式**
```bash
# 整理
# 整理

# AI 自動處理
扫描收集箱 → 识别內容 → 分類 → 移动 → 連結
✓ 15 个檔案已分類
  - 5 个專案 → 1 Projects
  - 3 个领域 → 2 Areas
  - 2 个資源 → 3 Resources
  - 5 个歸檔 → 4 Archives
✓ 自動建立了 8 个連結
```

# 整理

### 3. 檢索（Retrieve）- 语义搜尋

**传统方式**
```bash
# 關鍵词搜尋
搜尋"會議"
→ 找到 100+ 个結果
# 查看
```

**AI 增强方式**
```bash
# 语义搜尋
/search 找到所有与"專案決策"相關的筆記

# AI 理解语义
→ 找到包含"決策"、"選擇"、"结论"的筆記
# 排序
→ 高亮關鍵內容
→ 提供上下文
```

**工具**: `/search` 命令 + AI 理解

### 4. 使用（Use）- 智能提取

**传统方式**
```bash
# 手動回顧
1. 找到專案筆記
2. 逐个阅读
3. 提取資訊
4. 手動總結
```

**AI 增强方式**
```bash
# 让 AI 提取
"帮我總結這個專案的当前狀態"

# AI 自動處理
1. 讀取專案所有相關筆記
2. 提取關鍵資訊
3. 识别进度、风险、決策
4. 生成结构化報告

✓ 專案 A 狀態報告
  - 进度: 前端 80%, 后端 60%
  - 风险: 后端可能成为瓶颈
  - 決策: 决定增加 2 名后端開發
  - 下一步: 2026-01-25 里程碑
```
