## 🎯 什么是快速捕获？

### 定义

**快速捕获**是指在想法、灵感、观察或重要資訊出现的**第一時間**，用**最快的方式**記錄下来的過程。

### 为什么需要快速捕获？

| 原因 | 說明 |
|------|------|
| **避免遗忘** | 灵感和想法往往转瞬即逝 |
| **减轻认知负担** | 把想法写下来，大脑可以專注于其他任務 |
| **培养意识** | 重视自己的想法，养成記錄习惯 |
| **积累素材**为後續的深度思考和组织提供素材 |

### 快速捕获的核心原則

1. **速度优先** - 不追求完美，先記錄下来再说
2. **保持简单** - 使用最简单的方式記錄
3. **抓住时机** - 趁热打铁，不要拖延

## 💡 最佳實踐

### 捕获技巧

#### 1. 速度优先

✅ **推荐做法**：
- 先記錄标题和核心想法
- 其他字段可以後續补充
- 不要追求完美格式

❌ **避免做法**：
- 一開始就想写出完美的筆記
- 在格式上花费太多時間
- 因为追求完美而延遲記錄

#### 2. 使用快捷键

**Obsidian 命令面板**：
- 按 `Ctrl/Cmd + P` 打開命令面板
- 輸入"Template"快速選擇模板
- 設置自定义快捷键提高速度

**外掛推荐**：
- **Templater** - 提供更强大的模板功能
- **QuickAdd** - 快速新增筆記和內容
- **Hotkeys for specific templates** - 为特定模板設置快捷键

#### 3. 保持一致性

**命名規範**：
- 使用简洁、描述性的标题
- 包含關鍵資訊（主題、类型）
- 例如：`[想法] 产品功能優化建議`

**標籤規範**：
- 建立標籤体系
- 使用分层標籤（如 `#领域/子领域`）
- 保持標籤一致性

### 从闪念到 PARA

#### 转化为 Project

**适用情况**：
- 有明确目標和截止日期
- 需要多步骤完成
- 有具体交付物

**转化步骤**：
1. 使用 [[_templates/para/📁 quick/Project]] 模板創建專案
2. 将闪念內容複製到專案描述
3. 新增任務和里程碑
4. 从 InBox 移动到 `1 Projects/`

#### 转化为 Area

**适用情况**：
- 持續的责任和职责
- 需要长期維護的领域
- 無明确截止日期

**转化步骤**：
1. 使用 [[_templates/para/📁 quick/Area]] 模板創建领域
2. 将闪念內容作为领域內容
3. 新增維護目標和計劃
4. 从 InBox 移动到 `2 Areas/`

#### 转化为 Resource

**适用情况**：
- 有價值的參考资料
- 學習材料和筆記
- 最佳實踐和工具

**转化步骤**：
1. 使用 [[_templates/para/📁 quick/Resource]] 模板創建資源
2. 擴展和完善內容
3. 新增標籤和分類
4. 从 InBox 移动到 `3 Resources/`

#### 转化为 Archive

**适用情况**：
- 已完成或过时的內容
- 不再需要的临时筆記
- 歷史资料

**转化步骤**：
1. 使用 [[_templates/para/📁 quick/Archive]] 模板創建歸檔
2. 記錄歸檔原因
3. 从 InBox 移动到 `4 Archives/`
