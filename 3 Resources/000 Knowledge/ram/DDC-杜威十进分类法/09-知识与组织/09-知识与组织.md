# 09-DDC 与知识组织

> DDC 不只是一种排架工具——它是知识组织理论的实践。理解 DDC 就是理解"人类如何划分知识"这个更深层的问题。

## 分类的认知基础

| 概念 | 与 DDC 的关系 |
|------|-------------|
| **范畴化** | 大脑将世界分组——DDC 将此形式化为 10 大类 |
| **层级结构** | 从一般到特殊的树状——DDC 的十进层级 |
| **原型理论** | 某些成员比另一些"更典型"——DDC 的中心学科 |
| **互斥性** | 每本书只能在一个位置——DDC 的核心约束 |

## DDC 的逻辑前提

| 前提 | 问题 |
|------|------|
| 知识可以分成互斥的类 | 跨学科主题怎么办？ |
| 层级顺序反映自然顺序 | 谁决定"农业"在"工程"前面？ |
| 十进制是"自然"的 | 为什么是 10 而不是 7 或 12？ |

## 你的 Obsidian 与 DDC

你的个人知识管理使用标签、链接和 MOC——这其实是在做个人知识组织：

| 你做的 | DDC 等价 |
|--------|----------|
| 文件夹结构 | 层级分类 |
| 标签 | 受控词表 |
| `[[wikilink]]` | 关联索引 |
| MOC | 分类导航 |

> DDC 是给 10 亿本书用的，你的标签系统是给你自己用的——但原理相同。
