# 03 - GTD 理清（Clarify）

## 核心问题

> **"这件事的下一步物理动作是什么？"**

逐条处理收集篮，每条只问两个问题：

## 理清流程图

```
        收集篮中的事项
              │
        ┌─────┴─────┐
        │ 可执行吗？  │
        └─────┬─────┘
         YES ↓    ↓ NO
    ┌────────┐  ┌────────────┐
    │下一步是?│  │ 三种处理    │
    └────────┘  └────────────┘
         │          │
    ┌────┴────┐  ├→ 垃圾（删除）
    │< 2分钟？ │  ├→ 将来也许（备忘）
    └─────────┘  └→ 参考资料（归档）
    YES ↓  ↓ NO
   立即做！  └─→ 委派他人
                └─→ 推迟（放入清单）
```

## 两分钟法则

> 如果一件事能在 **2 分钟内完成**，立刻做！不要放入系统。

```
✅ 回复简短邮件
✅ 确认一个时间
✅ 转达一句话
✅ 丢进垃圾桶
✅ 在日历上标注
```

## 下一步行动

> 必须是具体的、物理的、可执行的动作。

| ❌ 模糊 | ✅ 清晰 |
|---------|---------|
| 处理税务 | 整理去年的发票 |
| 准备面试 | 写 3 个面试问题答案 |
| 联系客户 | 给张总打电话确认时间 |
| 学 Python | 打开 Python 教程第 1 章 |
| 写报告 | 新建文档并写大纲 |

## 项目识别

```
一件事需要多个步骤才能完成 → 这就是一个「项目」
```

**判断标准**：需要 2 个以上的「下一步行动」才能完成。

## 不可执行的事项处理

| 类型 | 处理方式 | 存放位置 |
|------|----------|----------|
| 垃圾 | 删除 | 回收站 |
| 将来也许 | 记下来，定期回顾 | 将来/也许清单 |
| 参考资料 | 分类归档 | 参考资料系统 |

## 理清时的纪律

```
⚠️ 从收集篮顶部开始，逐条处理
⚠️ 不挑拣、不跳过、不放回收集篮
⚠️ 每件事都得出「下一步」或「归宿」
⚠️ 处理完一条再处理下一条
⚠️ 理想状态：收集篮清零
```
