# 02 - PARA Projects（项目）

## 定义

> 项目 = 有明确目标和截止日期的一系列任务

**必须满足**：
- 有明确的结果
- 有截止日期（或时间范围）
- 需要多个步骤才能完成

## 项目示例

```
工作：
├── Q2 销售报告（截止 6/30）
├── 网站改版（截止 7/15）
├── 年度预算（截止 12/1）
└── 招聘前端工程师（截止招到为止）

个人：
├── 驾照更换（截止 8/15）
├── 孩子生日聚会（截止 6/20）
├── 家庭旅行计划（截止 7/1）
└── 年度体检（截止 6/30）
```

## 项目命名

```
[动词][对象][可选截止日]

✅ 写 Q2 报告
✅ 计划暑期旅行
✅ 学习 Obsidian（30天）

❌ 报告
❌ 旅行
❌ Obsidian
```

## 项目状态

```yaml
status: active       ← 正在进行
status: on-hold      ← 暂停
status: completed    ← 已完成 → 准备移入 Archives
status: cancelled    ← 已取消 → 移入 Archives
```

## 项目 vs 任务

| 判断 | 是 |
|------|-----|
| 一个步骤就能完成 | 任务（放 GTD 下一步行动） |
| 需要多步骤 | **项目** |

```
写报告 ← 项目
  ├── 收集数据 ← 任务
  ├── 分析数据 ← 任务
  └── 写结论 ← 任务
```

## 项目与 GTD 整合

```
PARA Projects → GTD 项目清单
每个项目 → 至少一个「下一步行动」

GTD 项目清单 = PARA Projects 的 action 视角
PARA Projects = GTD 项目的信息容器
```

## 项目数量建议

```
活跃项目 < 10 个
等待项目 < 5 个

如果超过 15 个活跃项目 → 需要精简或暂停
```
