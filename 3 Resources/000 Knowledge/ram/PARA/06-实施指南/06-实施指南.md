# 06 - PARA 实施指南

## 第一步：清点现有信息

```
1. 列出你所有的"信息容器"：
   □ 电脑文件夹（桌面、文档、下载）
   □ 笔记软件（Obsidian、Notion、飞书）
   □ 云盘（iCloud、OneDrive、Google Drive）
   □ 书签
   □ 邮箱存档
   □ 纸质笔记本

2. 预估总量 → 接受现实
```

## 第二步：创建顶层结构

```
在你主要使用的工具中创建：

1 Projects/          ← 现在进行的项目
2 Areas/             ← 长期负责的领域
3 Resources/         ← 感兴趣的话题
4 Archives/          ← 已完成/不活跃

+ 0 Inbox/           ← 暂存区（临时）
+ _templates/        ← 模板
```

## 第三步：迁移现有内容

### 从 Projects 开始

```
列出所有活跃项目 → 每个一个文件夹 → 移入 1 Projects/
```

### 识别 Areas

```
用这个清单发现你的领域：

工作相关：
□ 我的职位/角色是什么？
□ 我向谁汇报？管谁？
□ 我需要参加哪些定期会议？
□ 我持续负责什么指标？

个人相关：
□ 健康管理
□ 财务管理
□ 重要关系（家人/伴侣/朋友）
□ 学习与成长
□ 家庭维护
```

### 资源整理

```
把剩下的内容 → 按主题放入 3 Resources/

问：这属于「责任」还是「兴趣」？
责任 → Areas
兴趣 → Resources
```

### 归档清理

```
把以下内容移入 4 Archives/：
□ 去年的项目
□ 不再使用的工具/笔记
□ 过时的参考资料
□ 任何你 6 个月内不会打开的
```

## 第四步：建立日常习惯

| 频率 | 动作 |
|------|------|
| 每日 | 清空 0 Inbox → 分类到 P/A/R |
| 每周 | 检查 Projects，完成/暂停的归档 |
| 每月 | 扫一遍 Areas，确保没有遗漏 |
| 每季 | 扫一遍 Resources，过时的归档 |

## 常见错误

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| 按主题分类（不是可执行性） | 优先看「我现在需要做什么」 |
| 文件夹太深（8 层） | 最多 4 层 |
| 太细的分类（15 个 Areas） | 5-10 个领域就够了 |
| 从不归档 | 每季度归档一次 |
| 一开始就追求完美 | 先搭框架，边用边调 |
| 什么都留着 | 果断归档，相信搜索 |

## 渐进调整

```
Week 1: 搭框架 + 移 20%
Week 2: 移 50%
Week 3: 移剩余 + 归档
Week 4: 调整优化

一个月后系统基本稳定，之后就是维护。
```
