# 05 - PARA Archives（归档）

## 定义

> 归档 = 已完成、不活跃、或暂时不需要的内容

**核心原则**：**冷藏，不是删除**

## 什么该归档？

| 来源 | 何时归档 |
|------|----------|
| **Projects** | 项目完成/取消后 |
| **Areas** | 不再负责的领域 |
| **Resources** | 不再感兴趣的话题 |
| **其他** | 过时的内容、去年的笔记 |

## 归档结构

```
4 Archives/
├── 2024/                    ← 按年份
├── 2025/
│   ├── 01-January/
│   ├── 06-June/
│   └── 12-December/
├── 2026/
└── Completed-Projects/      ← 已完成项目
    ├── 2025-网站改版/
    └── 2026-Q2 报告/
```

## 归档 vs 删除

```
删除 → 彻底消失 → 后悔了找不回
归档 → 移出视野 → 需要时可以翻出来

95% 的情况：归档即可
仅删除：个人敏感信息、完全没价值的草稿
```

## 何时归档

### 触发时机

```
✅ 项目标记为"完成" → 立即归档
✅ 每周回顾时 → 发现不活跃的内容
✅ 季度大扫除 → 批量整理
```

### 判断标准

```
问自己：未来 6 个月，我会需要这个吗？

NO → 归档
不确定 → 归档（可以随时找回来！）
YES → 留在原处
```

## 归档的好处

```
┌──────────────────────────────────┐
│  归档前          │  归档后         │
├──────────────────┼────────────────┤
│  Projects: 15    │  Projects: 5    │
│  Areas: 12       │  Areas: 8      │
│  加载慢，眼花    │  清晰，专注     │
└──────────────────┴────────────────┘
```

> 归档 = 给你的数字空间"减负"。看到的越少，专注的越多。

## 搜索胜于分类

PARA 的理念：

```
花 20% 的时间分类 → 80% 的时间用搜索找到
而不是
花 80% 的时间分类 → 20% 的时间用搜索
```

> 归档的东西可以用搜索随时找到。不需要精细分类归档内容。
