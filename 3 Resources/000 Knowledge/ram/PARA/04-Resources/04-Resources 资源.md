# 04 - PARA Resources（资源）

## 定义

> 资源 = 你感兴趣的话题、可能用到的参考资料

**特点**：
- 没有截止日期
- 不是必须维护的责任
- "将来可能有用"

## 资源示例

```
技术：
├── Python
├── AI & ML
├── Obsidian
├── Markdown
└── Web 开发

兴趣：
├── 摄影
├── 咖啡
├── 园艺
└── 攀岩

参考：
├── 模板
├── 常用命令
├── 书摘
└── 会议记录模板
```

## 什么时候创建资源？

```
你在收集某个主题的信息 →
持续积累 → 内容超过 5 条 →

→ 创建资源文件夹！
```

## 资源 vs 领域

| | 领域 | 资源 |
|---|------|------|
| 我需要维护吗？ | ✅ 是责任 | ❌ 只是兴趣 |
| 没做好有后果？ | ✅ 有 | ❌ 没有 |
| 示例 | 健康、财务 | 咖啡知识、电影清单 |

```
领域：财务 ← 我得管好钱
资源：咖啡 ← 我喜欢，但不管也没事
```

## 资源子分类

```
3 Resources/
├── 01-Tech/           ← 技术
│   ├── Python/
│   ├── AI-ML/
│   └── DevOps/
├── 02-Books/          ← 读书笔记
├── 03-Productivity/   ← 效率方法
│   ├── GTD/
│   └── PARA/
├── 04-Design/         ← 设计
└── 05-Reference/      ← 参考手册
```

## 渐进整理

```
不要一开始就建完美的资源分类！

发现一个主题 → 记下来 → 内容多了 → 再建文件夹
```

> PARA 的核心是**轻量**。别陷入过度分类的陷阱。

## 资源维护

```
每季度扫一遍 Resources：
□ 哪些资源可以升级为「领域」？（你开始真正投入）
□ 哪些资源已经不再感兴趣？ → 归档
□ 哪些资源可以转成项目？（你决定做点什么）
```
