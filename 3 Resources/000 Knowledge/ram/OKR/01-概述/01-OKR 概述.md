# 01 - OKR 概述

## 起源

```
Intel, 1970s
    └── Andy Grove（Intel CEO）发明 OKR
        └── 称其为 "MBO 2.0"（目标管理升级版）

Google, 1999
    └── John Doerr（前 Intel 员工）向 Google 创始团队介绍 OKR
        └── Google 用了 20+ 年，规模从 40 人到 10 万+
```

## 四个超级力量

John Doerr 在《这就是 OKR》中总结：

| 力量 | 说明 |
|------|------|
| **聚焦与承诺** | 只做最重要的事 |
| **对齐与连接** | 团队目标与公司目标一致 |
| **追踪** | 持续检查进展，不是年底才知道 |
| **挑战** | 设定"不可能"的目标，追求 10x 增长 |

## OKR 的核心原则

### 1. 少即是多

```
每季度：3-5 个 O
每个 O：3-5 个 KR

不是"把所有事列出来" → 是"选出最重要的事"
```

### 2. 从下到上 + 从上到下

```
公司 OKR（方向）
    ↓ ↑
团队 OKR（对齐公司，有自主权）
    ↓ ↑
个人 OKR（对齐团队，发挥主动性）

约 60% 由下而上提出，40% 由上而下传递
```

### 3. 不做绩效考核

```
⚠️ 关键原则：OKR ≠ 绩效考核

OKR 追求挑战 → 允许失败 → 60-70% 完成即好
KPI 追求达标 → 期望 100% → 关联奖金

两者分开，才能鼓励"登月"目标
```

### 4. 透明公开

```
所有人的 OKR 对全员可见：

CEO → 全员可见
部门 → 全员可见
个人 → 全员可见

好处：消除重复、促进协作、互相监督
```

## 两类 OKR

| | 承诺型 | 挑战型 |
|---|--------|--------|
| 完成率期望 | 100% | 60-70% |
| 目标来源 | 日常业务 | 突破创新 |
| 风险 | 低 | 高 |
| 示例 | Q2 营收增长 10% | 新市场取得第一笔订单 |

## OKR 周期

```
年度 OKR（大方向）
    └── Q1 OKR
        └── 月度检查
        └── 周度检查
    └── Q2 OKR
    ...
```
