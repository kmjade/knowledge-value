# 07 - OKR 工具实践

## 专用工具 vs 通用工具

| 类型 | 工具 | 适合 |
|------|------|------|
| 专用 | **飞书 OKR** | 飞书用户 |
| 专用 | **Worktile / 飞书 OKR** | 国内企业 |
| 专用 | **Lattice / 15five** | 海外企业 |
| 通用 | **Notion** | 灵活定制 |
| 通用 | **Obsidian** | 个人/小团队 |
| 通用 | **多维表格** | 飞书/Notion 均可 |
| 极简 | **白板 + 便利贴** | 初创团队 |

## 方案 A：Notion OKR

### 数据库结构

```
OKR 数据库
├── 周期：2026 Q2
├── 类型：公司 / 部门 / 个人
├── 负责人
├── O（文本）
├── KR1（文本 + 起点 + 目标 + 当前值 + 进度%）
├── KR2 / KR3 / KR4
├── 总分（公式：average(KR1-4 进度)）
└── 状态：进行中 / 已完成

看板视图：按状态分组
时间线：按周期展示
```

## 方案 B：飞书多维表格 OKR

### 字段设计

```
字段：
├── 周期（单选：2026 Q1/Q2/Q3/Q4）
├── 层级（单选：公司/部门/个人）
├── 部门（单选）
├── 负责人（人员字段）
├── O（文本）
├── KR1 描述 / KR1 起点 / KR1 目标 / KR1 当前
├── KR进度（公式：avg(KR1/KR2/KR3)）
└── 状态（看板：进行中/有风险/已完成）
```

### 自动化

```
每周五 17:00 自动提醒负责人更新 KR 进度
KR 进度 < 预期时自动通知 OKR 教练
```
