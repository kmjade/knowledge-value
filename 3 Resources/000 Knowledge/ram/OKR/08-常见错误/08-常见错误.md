# 08 - OKR 常见错误

## 十大常见错误

### 1. O 写成任务清单

```
❌ O：完成网站改版、发布新功能、优化数据库
✅ O：打造让用户上瘾的产品体验
  （然后把具体任务放到 KR 或项目计划里）
```

### 2. KR 无法衡量

```
❌ KR：提升用户体验
❌ KR：加强团队协作

✅ KR：NPS 从 30 提升到 60
✅ KR：跨部门协作满意度评分从 3.2 到 4.0
```

### 3. OKR 太多

```
❌ 10 个 O，50 个 KR（你什么都想做）
✅ 3 个 O，每个 3 个 KR（你只做最重要的）

删掉那些"也不错"的 → 只留"必须做"的
```

### 4. 设完就忘

```
❌ 季度初写完 → 季度末打分（期间没看过）
✅ 每周检查 KR 进度，每月回顾方向
```

### 5. OKR 与绩效挂钩

```
❌ KR 完成度 = 奖金系数 → 没人敢设挑战目标
✅ OKR 追求突破，绩效看综合贡献 → 两套独立体系
```

### 6. 自上而下强压

```
❌ CEO 写完 → 层层分解 → 个人只是执行流水线
✅ 60% 自下而上提出 + 40% 自上而下统一方向
```

### 7. O 不够鼓舞人心

```
❌ O：完成 Q2 常规工作
✅ O：让客户在 Q2 成为我们的推荐者而非使用者

好 O 让人看完想说："我想做这个！"
```

### 8. 害怕"红色"

```
❌ KR 进度落后就藏着 → 季度末突然发现完不成
✅ 进度落后就亮红 → 绿色表示可以再加码，红色才是管理的起点
```

### 9. KR 是任务而非结果

```
❌ KR：完成 10 次客户访谈
✅ KR：客户访谈发现的 Top 3 需求转化为产品需求文档

任务衡量活动，结果衡量影响
```

### 10. 只设承诺型 OKR

```
❌ 所有 OKR 都追求 100% 完成
✅ 至少一个"挑战型"OKR → 追求 60-70% → 推动创新
```
