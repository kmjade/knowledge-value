# 02 - 目标 Objective（O）

## O 是什么？

> 目标 = 你要去哪里（定性的、有方向感的、鼓舞人心的）

**O 的三要素**：
1. **有方向** — 知道往哪走
2. **鼓舞人心** — 让人想做
3. **有时限** — 要在季度内完成

## 好 O 的特征

### ✅ 鼓舞人心

```
✅ 让中小企业在 3 分钟内完成报税
✅ 打造让开发者上瘾的 API 体验
✅ 让我们的产品成为用户每天打开的第一个 App

❌ 提高营收
❌ 优化产品
❌ 增加用户
```

### ✅ 定性表达

```
O 不是数字，O 是"方向 + 结果状态"

✅ 成为华南区市场份额第一
✅ 打造零 Bug 的产品质量
✅ 建立行业标杆级的客户服务

❌ 市场份额达到 30%（这是 KR！）
❌ Bug 数量降到 0（这是 KR！）
```

### ✅ 可以在一季度内推进

```
✅ Q2 建立可复用的销售流程

❌ 十年内成为行业第一（太长，适合做愿景/年度 O）
```

## O 的写法公式

```
[动词] + [对象] + [期望的结果状态]

动词：打造、成为、建立、实现、突破、推出
对象：产品、服务、流程、团队、市场
结果：第一、标杆、领先、可复制、稳定
```

## 不同层级的 O

### 公司级

```
O：成为跨境电商 SaaS 领域的首选平台
```

### 部门级（产品）

```
O：让用户在 5 分钟内完成第一个有效操作
```

### 个人级

```
O：成为团队中 Go-To 的数据分析专家
```

## 数量建议

```
公司级：3-5 个 O
团队级：3-4 个 O  
个人级：2-3 个 O

太多 → 焦点分散
太少 → 覆盖面不够
```
