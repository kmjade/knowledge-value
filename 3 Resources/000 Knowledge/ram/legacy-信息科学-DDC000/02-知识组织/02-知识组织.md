# 02-知识组织

> 知识组织（Knowledge Organization）是信息科学的核心分支——研究如何系统地描述、分类和关联信息与知识，使之可发现、可理解、可复用。

## 知识组织的层次

```
高 ┌──────────────┐
    │   本体 (Ontology)   │  概念的关系网络 + 逻辑推理
    ├──────────────┤
    │   叙词表 (Thesaurus) │  概念层级 + 相关关系
    ├──────────────┤
    │   分类法 (Classification)│  层级结构，学科分类
    ├──────────────┤
低 ┌──────────────┐
    │  元数据 (Metadata)   │  描述单个资源的属性
    ├──────────────┤
    │  标签/关键词 (Tags)  │  非受控的用户标注
    └──────────────┘
```

## 分类法 (Classification)

### 主要图书分类法

| 分类法 | 使用地区 | 特点 |
|--------|----------|------|
| **DDC** (杜威十进分类法) | 全球（尤其是英美） | 000-999 十进，全球使用最广 |
| **LCC** (美国国会图书馆分类法) | 英美研究型图书馆 | 字母+数字，更灵活 |
| **UDC** (国际十进分类法) | 欧洲 | DDC 的扩展，支持更复杂的组合 |
| **CLC** (中国图书馆分类法) | 中国 | 22 大类，A-Z 字母表示 |
| **NLM** (美国国立医学图书馆) | 医学图书馆 | 医学专业分类 |

### 中图法 22 大类

| 字母 | 大类 |
|------|------|
| A | 马克思主义、列宁主义、毛泽东思想 |
| B | 哲学、宗教 |
| C | 社会科学总论 |
| D | 政治、法律 |
| G | 文化、科学、教育、体育 |
| **G25** | **图书馆学、图书馆事业** |
| **G35** | **情报学、情报工作** |
| T | 工业技术 |
| **TP** | **自动化技术、计算机技术** |
| Z | 综合性图书 |

### 分类法 vs 标签 (Folksonomy)

| 维度 | 分类法 (Taxonomy) | 大众分类 (Folksonomy) |
|------|-------------------|----------------------|
| 创建者 | 领域专家 | 用户 |
| 结构 | 严格层级 | 扁平 |
| 一致性 | 高（受控词表） | 低（同义词、拼写变体） |
| 发现能力 | 浏览 | 搜索 |
| 例子 | DDC, MeSH | Flickr 标签, #hashtag |

> 分类法提供"浏览"能力，Folksonomy 提供"搜索"入口——两者互补。

## 元数据 (Metadata)

元数据 = 关于数据的数据。

### 元数据类型

| 类型 | 作用 | 例子 |
|------|------|------|
| **描述性** | 识别和发现资源 | 标题、作者、摘要 |
| **结构性** | 资源内部或之间的关系 | 章节、页码、是某系列的一部分 |
| **管理性** | 资源的管理和使用 | 创建日期、文件格式、访问权限 |
| **技术性** | 技术属性 | 分辨率、编码、校验和 |
| **保存性** | 长期保存所需信息 | 迁移历史、原始环境 |

### Dublin Core（都柏林核心）

最简洁通用的元数据标准，15 个元素：

| 元素 | 说明 |
|------|------|
| Title | 资源名称 |
| Creator | 创建者 |
| Subject | 主题关键词 |
| Description | 内容描述 |
| Publisher | 发布者 |
| Contributor | 其他贡献者 |
| Date | 日期 |
| Type | 资源类型 |
| Format | 格式 |
| Identifier | 唯一标识符（如 DOI） |
| Source | 来源 |
| Language | 语言 |
| Relation | 相关资源 |
| Coverage | 时空范围 |
| Rights | 权限信息 |

## 叙词表 (Thesaurus)

受控词表——不仅列出术语，还标注术语间的关系：

| 关系类型 | 含义 | 示例 |
|----------|------|------|
| **BT** (上位词) | Broader Term | "猫" BT "哺乳动物" |
| **NT** (下位词) | Narrower Term | "猫" NT "暹罗猫" |
| **RT** (相关词) | Related Term | "猫" RT "宠物" |
| **USE** | 指引到规范词 | "猫咪" USE "猫" |
| **UF** | 规范词的非规范形式 | "猫" UF "猫咪" |

## 本体 (Ontology)

最形式化的知识组织方式——明确定义概念、属性和关系，支持逻辑推理。

```
实体: "Book"
  ├── 属性: title (string), author (Person), publishedYear (integer)
  └── 关系: writtenBy → Person, publishedBy → Publisher
```

> SKOS (Simple Knowledge Organization System) 是表达叙词表和分类法的 W3C 标准，是"轻量级本体"。

## 关联数据 (Linked Data)

Tim Berners-Lee 提出的语义网实践：用 URI 标识实体，用 RDF 表达关系。

四原则：
1. 用 URI 作为事物的名称
2. 使用 HTTP URI 使人们能查找这些名称
3. 当有人查找 URI 时，提供有用的信息（使用标准 RDF）
4. 包含指向其他 URI 的链接，以便发现更多事物
