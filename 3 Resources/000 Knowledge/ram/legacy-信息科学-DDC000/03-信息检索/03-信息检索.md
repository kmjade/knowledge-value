# 03-信息检索

> 信息检索 (Information Retrieval, IR) 研究如何从大规模信息集合中找到满足用户信息需求的材料。Google 只是 IR 的应用之一。

## IR 的核心模型

### 布尔模型

最基础的检索模型——用 AND/OR/NOT 逻辑组合关键词。

| 查询 | 含义 |
|------|------|
| 猫 AND 狗 | 同时包含猫和狗的文档 |
| 猫 OR 狗 | 包含猫或狗的文档 |
| 猫 NOT 狗 | 包含猫但不包含狗的文档 |

> 简单、精确、可解释，但缺少排序——所有匹配的文档同等"相关"。

### 向量空间模型 (VSM)

把文档和查询都表示为向量，用余弦相似度计算相关性。

```
查询: "人工智能 深度学习"
文档A: "人工智能 机器学习 神经网络"
文档B: "文艺复兴 油画 雕塑"

cos(query, docA) ≈ 0.7  ← 相关
cos(query, docB) ≈ 0.02 ← 不相关
```

### TF-IDF

最有影响力的词权重方案之一：

| 指标 | 全称 | 含义 |
|------|------|------|
| **TF** | Term Frequency | 词在文档中出现越多次越重要 |
| **IDF** | Inverse Document Frequency | 词在越少文档中出现越重要 |

```
TF-IDF(t, d) = TF(t, d) × log(N / DF(t))
```

> "的"出现在所有文档中 → IDF 极低 → 权重极低。"深度学习"只出现在少数文档 → IDF 高 → 重要。

### BM25

TF-IDF 的现代改进版，至今仍是文本搜索的基准算法。Elasticsearch 默认使用 BM25。

## 搜索引擎架构

```
用户查询
    ↓
查询理解 (分词、纠错、意图识别)
    ↓
检索 (倒排索引查找)
    ↓
粗排 (简单模型快速筛选 Top K)
    ↓
精排 (复杂模型精细排序)
    ↓
重排 (业务规则、多样性调整)
    ↓
结果展示 (snippet 生成、高亮)
```

## 倒排索引 (Inverted Index)

搜索引擎的核心数据结构：

```
正向索引: Doc1 → ["猫", "吃", "鱼"]
          Doc2 → ["狗", "吃", "骨头"]

倒排索引: "猫" → [Doc1]
          "狗" → [Doc2]
          "吃" → [Doc1, Doc2]
          "鱼" → [Doc1]
          "骨头" → [Doc2]
```

有了倒排索引，查询"猫"只需访问"猫"的倒排列表，不用扫描所有文档。

## 检索评价

### 精确率 vs 召回率

| 指标 | 公式 | 含义 |
|------|------|------|
| **精确率 (Precision)** | TP / (TP + FP) | 返回的结果中，有多少是真正相关的？ |
| **召回率 (Recall)** | TP / (TP + FN) | 所有相关文档中，有多少被找到了？ |

> 精确率和召回率通常是权衡的——提高一个会降低另一个。

### F-measure

```
F₁ = 2 × (P × R) / (P + R)
```

### 其他评价指标

| 指标 | 说明 |
|------|------|
| **MAP** (Mean Average Precision) | 考虑排序的平均精确率 |
| **NDCG** (Normalized DCG) | 考虑排序和相关性分级 |
| **MRR** (Mean Reciprocal Rank) | 第一个正确答案的排名的倒数均值 |

## 信息检索 vs 数据检索

| 维度 | 数据检索 (SQL) | 信息检索 (搜索引擎) |
|------|---------------|-------------------|
| 查询 | 精确、结构化 | 模糊、自然语言 |
| 匹配 | 精确匹配 | 部分匹配、最佳匹配 |
| 结果 | 确定集 | 排序列 |
| 错误容忍 | 无 | 拼写错误、同义词自动处理 |
| 例子 | `SELECT * WHERE age=25` | "25 岁适合做什么工作" |

## 现代 IR 趋势

| 趋势 | 说明 |
|------|------|
| **神经 IR** | 用深度学习做检索和排序（BERT, Dense Retrieval） |
| **语义搜索** | 从关键词匹配到意图理解 |
| **向量检索** | 将文档和查询编码为稠密向量，用 ANN 搜索 |
| **多模态检索** | 用文本搜索图片、视频 |
| **对话式搜索** | 多轮交互，逐步明确需求（ChatGPT 搜索） |
