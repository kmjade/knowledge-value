# 06-档案学与数字保存

> 档案学是研究档案的形成、管理、保存和利用的学科。数字保存是其中增长最快的领域——如何在技术和格式不断迭代的世界里，让今天的数字信息在 100 年后仍可访问？

## 档案 vs 图书 vs 文献

| 维度 | 档案 | 图书 | 期刊论文 |
|------|------|------|----------|
| 性质 | 原始记录（唯一性） | 出版品（多副本） | 出版品（多副本） |
| 产生方式 | 自然积累（工作过程） | 有意创作 | 有意创作 |
| 组织方式 | 全宗原则（按来源） | 分类法（按主题） | ISSN/卷期 |
| 核心价值 | 证据价值 > 信息价值 | 信息价值 | 信息价值 |
| 获取方式 | 受控（隐私、保密） | 开放 | 开放（但可能付费） |

## 档案管理的核心原则

### 来源原则 (Principle of Provenance)

档案必须按**产生它的机构或个人**来组织——不能打散到按主题重组。这是档案学的基石。

```
全宗 (Fonds) ← 一个机构/个人的全部档案
  └── 系列 (Series) ← 按职能/活动分组
        └── 案卷 (File) ← 同一事务的相关文件
              └── 件 (Item) ← 单份文件
```

### 原始顺序 (Original Order)

档案应保持形成时的原始顺序——这个顺序本身承载了信息。

### 档案鉴定

决定哪些档案值得永久保存：

| 价值类型 | 含义 |
|----------|------|
| **证据价值** | 证明机构活动和个人权利 |
| **信息价值** | 对历史研究和社会记忆的意义 |
| **内在价值** | 档案本身的物理特征不可替代（如签名原件） |

> 档案鉴定是最难、最需要判断力的档案工作——保存一切是不可能的。

## OAIS 参考模型

OAIS (Open Archival Information System) 是数字保存的国际标准 (ISO 14721)。

### OAIS 功能模型

```
生产者 → SIP (提交信息包)
              ↓
      ┌──────────────┐
      │  摄取 (Ingest)  │
      ├──────────────┤
      │ 归档存储        │ ← AIP (存档信息包)
      ├──────────────┤
      │  数据管理      │
      ├──────────────┤
      │  访问 (Access)  │
      └──────────────┘
              ↓
消费者 ← DIP (分发信息包)

        行政管理 (贯穿全程)
```

### 信息包概念

| 包类型 | 含义 |
|--------|------|
| **SIP** (Submission Information Package) | 生产者提交的原始数据包 |
| **AIP** (Archival Information Package) | 经过整理、包含完整元数据的保存包 |
| **DIP** (Dissemination Information Package) | 用户请求时分发的数据包 |

## 数字保存的核心挑战

| 挑战 | 说明 |
|------|------|
| **格式过时** | 今天的 .docx 50 年后可能没有软件能打开 |
| **介质退化** | 硬盘、光盘、磁带寿命有限 |
| **技术依赖** | 网站需要特定浏览器版本、数据库需要特定引擎 |
| **规模** | 数字数据量呈指数增长 |
| **真实性** | 如何证明数字文件没有被篡改？ |

### 数字保存策略

| 策略 | 说明 | 适用 |
|------|------|------|
| **迁移 (Migration)** | 将文件转换为新格式 | 大多数数字资料 |
| **仿真 (Emulation)** | 模拟原始软件环境 | 复杂交互式内容（游戏、软件） |
| **技术保存** | 保留原始硬件和软件 | 极少——成本高、不可扩展 |
| **封装 (Encapsulation)** | 将内容和元数据打包，附描述 | 作为辅助策略 |

## 数字保存的可信仓储

| 认证标准 | 说明 |
|----------|------|
| **CoreTrustSeal** | 数据仓储的可信认证（原 DSA） |
| **ISO 16363** | 可信数字仓储审计认证（更严格） |
| ** nestor Seal** | 德国的数字保存标准 |
