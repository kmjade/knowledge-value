# 项目创建流程

> [!summary] 概述
> 項目創建是 IOTO 中的低頻操作，卻是知識生產的起點。

---

## 流程概覽

```
01 喚醒指令 → 02 輸入名稱 → 03 自動基建 → 04 動態擴展
```

---

## 詳細步驟

### Step 1：喚醒指令

**方法一**：命令面板
```
Ctrl+P → 輸入「創建新項目」→ Enter
```

**方法二**：Framework Settings
```
打開 Framework Settings → 創建項目
```

### Step 2：輸入項目名稱

輸入項目名稱，確立「認知防火牆」的邊界。

**命名建議**：
- 簡潔明確
- 體現項目目標
- 便於識別和搜索

**示例**：
- ✅ AI學習
- ✅ IOTO研發
- ✅ 內容營銷
- ❌ 雜項
- ❌ 待辦

### Step 3：自動化基建

IOTO 會自動創建以下結構：

```
3-任務/
└── [項目名稱]/
    └── 任務清單.md

4-成果/
└── [項目名稱]/
    ├── 📁 文章
    ├── 📁 總結
    └── 📁 其他
```

### Step 4：動態擴展

如需添加其他成果類型，可手動在成果文件夾下創建。

---

## 項目准入守則

> [!warning] 重要提示
> 項目是 IOTO 的心臟。但在開啟一個新項目之前，請務必保持克制。

### 拒絕碎片化

如果一個目標可以在短期內速成，請將其作為子項目（Subject）歸併入更大的母項，不要讓瑣碎的文件夾撕裂你的專注力。

### 追求深廣度

只有那些能讓你持續投入、值得反覆咀嚼的領域，才配得上成為 IOTO 的獨立項目。

### 低頻即高效

創建項目是一個低頻操作。頻繁的創建往往意味著對目標的動搖；而持久的堅持，才是產生「意想不到之收穫」的唯一路徑。

---

## 項目評估清單

在創建項目前，問自己：

```
□ 這個項目對我的人生目標有何貢獻？
□ 我是否有足夠的時間和資源投入？
□ 這個項目是否值得長期堅持？
□ 我是否願意為此犧牲其他機會？
□ 這個項目是否有明確的預期成果？
```

---

## 項目管理最佳實踐

### 活躍項目數量

建議同時維持 **3-5 個活躍項目**：
- 2 個核心項目（最重要）
- 1-2 個次要項目
- 1 個探索項目

### 項目生命週期

```
立項 → 執行 → 交付 → 歸檔
```

### 項目歸檔

當項目完成或終止時：
1. 將項目文件夾移動到歸檔區
2. 撰寫項目總結
3. 提取可複用的知識卡片
