# 05 - Zettelkasten 写作流程

## 传统写作 vs Zettelkasten 写作

```
传统写作：
  空白页 → 选题 → 收集素材 → 写大纲 → 填充内容 → 这很难！

Zettelkasten 写作：
  已有 500 张卡片 → 发现一个主题聚集了 30 张 → 写作 = 排序 + 过渡
```

> Luhmann："我的写作不是从零开始的。我只需要在卡片盒里走一圈，然后写下来。"

## 六步写作法

### 1. 发现主题簇

```
在卡片盒中浏览：
- 哪些卡片互相链接最密？
- 哪些话题的卡片数量超过了 10 张？
- 哪些链接形成了有趣的交叉？

→ 选一个作为写作主题
```

### 2. 收集相关卡片

```
用 Obsidian 的方法：
  - 打开局部图谱
  - 用标签筛选
  - 看反向链接面板
  - 提取所有相关的卡片

用 Dataview 辅助：
```dataview
LIST
FROM #某个标签
WHERE file.size > 200
```
```

### 3. 排序成大纲

```
把收集到的卡片按逻辑顺序排列：

卡片A（问题引入）
  → 卡片B（背景）
    → 卡片C（核心论点）
      → 卡片D（论据 1）
      → 卡片E（论据 2）
    → 卡片F（反对意见）
  → 卡片G（结论）
```

### 4. 填充过渡

```
卡片之间加入过渡语句：
"这引出了另一个问题..."
"与上述观点相反的是..."
"值得注意的是..."
```

### 5. 写出初稿

```
大纲的每个节点 = 文章的一段
卡片的原标题 = 该段的主题句
卡片的正文 = 该段的内容基础
```

### 6. 润色成稿

```
加入开头和结尾
统一语气和风格
校对语法和逻辑
```

## 写作中的卡片角色

| 卡片类型 | 在文章中的作用 |
|----------|---------------|
| 定义卡 | 解释概念 |
| 论据卡 | 支持论点 |
| 对比卡 | 展示不同观点 |
| 案例卡 | 举例说明 |
| 引用卡 | 引用权威 |
| 问题卡 | 引出讨论 |

## Zettelkasten 写作的优势

```
✅ 永不从空白开始 → 初稿痛苦大幅降低
✅ 卡片可复用 → 同一张卡片可用于多篇文章
✅ 自然涌现主题 → 写你真正思考过的东西
✅ 逻辑自洽 → 链接过的卡片逻辑上已经连通
✅ 迭代改进 → 同一话题可以反复写、越写越深
```
