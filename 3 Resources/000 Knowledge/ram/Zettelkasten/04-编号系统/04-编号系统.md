# 04 - Zettelkasten 编号系统

## Luhmann 的原始编号

Luhmann 用了一种分支编号法：

```
1        ← 大主题
1.1      ← 子主题
1.1a     ← 展开 1.1 的第一条笔记
1.1b     ← 1.1 的第二条
1.1b1    ← 1.1b 的延伸
1.1b2
1.2      ← 1 的新子主题
1.2a
2        ← 新的大主题
```

**特点**：
- 数字表示层级
- 字母表示顺序
- 交替使用数字和字母
- 可以无限插入新卡片

### 插入示例

```
已有：1.1, 1.2
想在 1.1 和 1.2 之间插入：

→ 1.1a, 1.1b（挂在 1.1 下）
  （然后自然过渡到 1.2）
```

## 现代替代方案

### Obsidian：不需要编号

```
Wikilink + Tag + 搜索 → 编号变得不必要

[[卡片名]] 就能精准定位
Ctrl+O 就能跳转
搜索就能找到
```

### 按日期编号

```
YYYYMMDDHHMM  <卡片标题>

20260525 1530 Zettelkasten 原子性原则
20260525 1601 信息熵与不确定性的关系
```

> 简单，易实现，但缺乏层级关系。

### 混合方案

```
在 Obsidian 中：
  - 不需要物理编号
  - 用 MOC（Map of Content）做索引笔记
  - 用 wikilink 做交叉引用
  - 用 Dataview 做自动汇总
```

## 编号的核心价值（在现代工具中）

| Luhmann 时代的需要 | 现代工具的实现 |
|-------------------|---------------|
| 物理排序（卡片放哪） | 搜索 + wikilink |
| 找到相邻卡片 | 反向链接面板 |
| 知道层级关系 | 图谱视图 |
| 引用卡片 | `[[卡片名]]` |

> 编号只是一种"定位策略"。在数字时代，**链接就是最好的编号**。

## 你要不要编号？

```
✅ 需要编号的情况：
  - 你用纸笔（物理卡片需要排序）
  - 你的卡片数量 > 5000 且有强烈层级需求

❌ 不需要编号的情况：
  - 你用 Obsidian/Logseq/Roam
  - 你相信搜索 + 链接足以导航
```
