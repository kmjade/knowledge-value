# 02 - Zettelkasten 原子笔记

## 原子性原则

> 一条笔记 = 一个想法

就像物理学中的原子——不能再分割的基本单元。

## 好卡片的特征

### ✅ 独立可理解

```
脱离上下文也能读懂这条笔记在说什么。

✅ "香农信息论的核心洞察：信息是不确定性的减少。
    这与 Boltzmann 的熵公式在数学形式上一致。"

❌ "这是关于信息论的内容"（太模糊）
❌ "Shannon 1948, p.379"（缺乏内容）
```

### ✅ 连接而非收藏

```
每条永久笔记都应该：
1. 链接到至少一条已有笔记
2. 解释为什么链接（可选但推荐）

不要说"参考 X" → 说"与 X 的不同/相似/补充之处"
```

### ✅ 用自己的语言

```
原文：Chunking is the process of organizing pieces of information into meaningful groups.

✅ 组块化是把零散信息打包成有意义的单元。
   这解释了为什么专家能记住更多——
   他们把细节"组块"成了更大的模式。

❌ Chunking 是把信息组织成有意义的组。
   （太接近原文，缺乏自己的思考）
```

## 卡片长度

```
理想长度：屏幕一屏能看完
中文：100-300 字
英文：5-10 行

如果超过 → 考虑拆成两张卡
```

## 卡片必须包含

```markdown
---
tags: [tag]
---

# 标题（一个完整的陈述句）

核心观点（1-3 句话）

为什么重要/与什么相关

## 链接
- [[相关笔记1]] — 为什么链接
- [[相关笔记2]] — 不同之处

---
来源：作者, 书名/文章, 年份
```

## 写永久笔记时问自己

```
1. 这个想法本身完整吗？（能独立看懂吗）
2. 它和哪些已有的卡片相关？
3. 它能触发什么新问题？
4. 它和什么观点矛盾/互补？
5. 我未来读这张卡片时，需要什么上下文？
```

## 常见错误

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| 一个卡片写多个想法 | 拆成多张卡片 |
| 照搬原文 | 用自己的话重新表达 |
| 不链接 | 至少链接一条 |
| 太短/太长 | 一屏能看完 |
| 写得很学术/很空洞 | 像微信发消息一样写 |
| 只记录"是什么" | 还要记录"为什么重要" |
