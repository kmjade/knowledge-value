# 03 - Zettelkasten 链接思维

## 链接是灵魂

> "没有链接的卡片盒只是一堆卡片。有链接的卡片盒是思想网络。"

Luhmann 的卡片盒有 9 万张卡片，但有数十万条交叉链接。

## 链接类型

### 1. 直接链接

```
[[卡片A]] ← 卡片B 中直接引用 A

"这个概念与 [[信息熵]] 有关..."
```

### 2. 反向链接

```
不写在卡片里，系统自动显示：

卡片A ← 哪些卡片链接了 A？（Obsidian 右键面板）
```

### 3. 索引笔记（Index Note / Hub Note）

```
一张专门的卡片，列出某个主题下所有相关卡片的入口。

标题：Zettelkasten 索引
内容：
  - [[01-概述]] — 什么是 Zettelkasten
  - [[02-原子笔记]] — 原子性原则
  - [[03-链接思维]] — 链接策略
  ...
```

### 4. 结构笔记（Structure Note）

```
比索引更深一层，不是简单列表，而是有逻辑的"地图"。

标题：笔记方法的对比
内容：
  核心维度：存储 vs 链接 vs 结构化程度
  Zettelkasten → 弱结构化，强链接
  PARA → 强结构化（4 层），中等链接
  GTD → 行动导向，轻链接
```

## 链接策略

### 寻找链接

```
写完一张永久笔记后，问自己：

1. 这个想法之前出现过吗？ → 链接已有卡片
2. 这个想法反驳了什么？ → 链接对立观点
3. 这个想法补充了什么？ → 链接互补观点
4. 这个想法打开了什么新问题？ → 创建新卡片 + 链接
5. 这个想法用在了哪里？ → 链接应用场景
```

### 链接的质量 > 数量

```
差的链接 ❌：
  "参考 [[其他卡片]]"
  "相关：[[卡片A]], [[卡片B]], [[卡片C]]"

好的链接 ✅：
  "这与 [[卡片A]] 形成了有趣的对比：A 认为 X，
   但这里的发现表明 Y..."
```

## Luhmann 的链接方式

Luhmann 在每张卡片上手动写：

```
"参见卡片 21/3a5b7"（引向相关卡片）
"继续于卡片 21/3a5b8"（扩展讨论）
```

他的链接不追求"树状"，而是**网状**—— 一张卡片可以链接到卡片盒的任何位置。

## 涌现结构

```
不要预设知识结构！
      ↓
写卡片 → 链接 → 聚集 → 自然形成「主题群」
      ↓
主题群长大了 → 创建索引笔记
      ↓
索引笔记互相链接 → 更高层的结构笔记
      ↓
结构笔记之间产生关联 → 书的章节自然出现
```

> "结构不是设计出来的，是从链接中涌现出来的。"
