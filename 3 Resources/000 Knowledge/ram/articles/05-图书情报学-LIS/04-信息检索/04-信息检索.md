# Information Retrieval

> 信息检索 (Information Retrieval, IR) 研究如何从大规模信息集合中找到满足用户需求的信息。Google 是 IR 的成功应用——但远非全部。

## IR 的核心问题

给定一个信息需求（通常是查询字符串）和文档集合，返回按相关性排序的文档列表。

```
用户: "气候变化对农业的影响"
        ↓
    查询理解（分词、扩展、意图识别）
        ↓
    检索（从千万文档中初筛）
        ↓
    排序（精确计算相关性）
        ↓
    结果：Top 10 最相关文档
```

## 检索模型

### 布尔模型

用 AND/OR/NOT 精确匹配。简单、可解释，但没有排序。

### 向量空间模型 (VSM)

把文档和查询都表示为向量，余弦相似度排序。

### TF-IDF

最经典的词权重方案：

| 指标 | 含义 |
|------|------|
| **TF** (词频) | 词在文档中出现越多越重要 |
| **IDF** (逆文档频率) | 包含该词的文档越少越重要 |

> "的"出现在所有文档中 → IDF 极低 → 几乎不贡献相关性。"本体论"只出现在 3 篇文档 → IDF 极高 → 是区分的利器。

### BM25

TF-IDF 的现代改进，Elasticsearch 默认算法，至今是全文检索基准。

## 倒排索引

搜索引擎的核心数据结构：

```
正排: Doc1 → ["猫","吃","鱼"]  Doc2 → ["狗","吃","骨头"]
倒排: "猫"→[Doc1]  "狗"→[Doc2]  "吃"→[Doc1,Doc2]
```

查询"猫吃" → 取"猫"的倒排列表 ∩ "吃"的倒排列表 → Doc1。

## 评价指标

| 指标 | 含义 |
|------|------|
| **精确率 (Precision)** | 返回的结果中，有多少是相关的？ |
| **召回率 (Recall)** | 所有相关文档中，找到了多少？ |
| **F₁** | 精确率和召回率的调和平均 |
| **MAP** | 考虑排序位置的平均精确率 |
| **NDCG** | 考虑排序位置 + 相关性分级 |

> 精确率 vs 召回率是永恒权衡——提高一个通常降低另一个。

## 学术检索 vs Web 检索

| 维度 | Web 检索 | 学术检索 |
|------|----------|----------|
| 用户 | 所有人 | 研究者、学生 |
| 信息需求 | 多样化（导航、事实、探索） | 系统性（文献综述、前沿跟踪） |
| 权威性 | PageRank 等链接分析 | 同行评审、影响因子 |
| 召回优先 | 不需要（第一页够好就行） | 需要（系统综述不能漏） |
| 代表 | Google, Bing | Web of Science, Scopus, CNKI |

## 现代趋势

| 趋势 | 说明 |
|------|------|
| **神经 IR** | BERT 等深度模型做检索和重排序 |
| **语义搜索** | 从关键词匹配到意图理解 |
| **向量检索 (ANN)** | 稠密向量 + 近似最近邻搜索 |
| **多模态检索** | 用文本搜图片/视频 |
| **RAG** | 检索增强生成——LLM 时代 IR 的新范式 |
