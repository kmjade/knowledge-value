# 08-信息机构与管理

> 信息机构是收集、组织、保存和提供信息获取的社会组织——图书馆、档案馆、数据中心、博物馆的信息管理职能。理解它们如何运作与被管理。

## 信息机构类型

| 类型 | 代表 | 核心使命 |
|------|------|----------|
| **图书馆** | 国家图书馆、公共图书馆、高校图书馆 | 信息获取和素养教育 |
| **档案馆** | 国家档案馆、企业档案馆 | 记录的长期保存和证据提供 |
| **数据中心** | 科研数据中心、政府数据平台 | 数据的组织和共享 |
| **博物馆** | 历史/艺术/科学博物馆 | 实物+信息的保存和展示 |
| **情报机构** | 竞争情报、政府情报 | 信息分析和决策支持 |

## 图书馆/信息机构的管理维度

### 战略规划

| 要素 | 核心问题 |
|------|----------|
| 使命 | 这个机构为什么存在？ |
| 愿景 | 未来要成为什么？ |
| 环境扫描 | 外部趋势（技术、人口、政策） |
| SWOT 分析 | 优势、劣势、机会、威胁 |

### 人力资源管理

信息机构是知识密集型组织——核心资产是人：

| 角色 | 职责 |
|------|------|
| 学科馆员 | 嵌入院系，做深度学科服务 |
| 系统馆员 | 管理图书馆自动化系统、数字资源平台 |
| 数据馆员 | 科研数据管理、数据素养教育 |
| 数字保存官 | 数字资产的长期保存策略 |

### 资源管理

| 维度 | 内容 |
|------|------|
| **物理资源** | 空间设计（从书库到学习共享空间）、设施管理 |
| **数字资源** | 数据库订阅、电子书采购、机构知识库建设 |
| **财政管理** | 预算规划、成本效益分析 |
| **联盟与共享** | 馆际互借、联合采购（如 DRAA 数据库联盟） |

## 知识管理 (Knowledge Management)

信息机构的核心使命正在从"文献管理"转向"知识管理"：

| 过程 | 含义 | 工具/方法 |
|------|------|-----------|
| **知识创造** | 产生新知识 | 研究、头脑风暴、经验总结 |
| **知识捕获** | 将隐形知识变为显性知识 | 文档化、事后回顾 (AAR) |
| **知识组织** | 分类和结构化 | 知识库、维基、本体 |
| **知识共享** | 在组织内传播 | 实践社区 (CoP)、导师制 |
| **知识应用** | 用知识解决问题 | 决策支持系统、专家系统 |

### SECI 模型（野中郁次郎）

知识转化的四种模式：

```
    隐性知识                显性知识
隐性 社会化(S)     ←→     外化(E)      显性
      ↑                     ↓
隐性  内化(I)      ←→     组合(C)      显性
    隐性知识                显性知识
```

| 模式 | 含义 | 例子 |
|------|------|------|
| **S**ocialization | 隐性 → 隐性（师徒制） | 跟资深馆员学习参考咨询 |
| **E**xternalization | 隐性 → 显性（文档化） | 把经验写成操作手册 |
| **C**ombination | 显性 → 显性（整合） | 把多个数据库的使用说明整合成一本指南 |
| **I**nternalization | 显性 → 隐性（学以致用） | 读了手册后在实践中形成直觉 |

## 信息政策与治理

| 领域 | 核心议题 |
|------|----------|
| 信息自由 | 公众获取政府信息的权利 |
| 隐私保护 | 个人信息在信息机构中的处理 |
| 知识产权 | 版权、合理使用、开放许可 |
| 数字包容 | 缩小数字鸿沟、信息无障碍 |
| 网络安全 | 保护信息基础设施 |
