# 04-自然语言处理 (NLP)

> 自然语言处理是 AI 的一个子领域，研究计算机与人类（自然）语言之间的交互——让机器理解、生成和翻译文本。

## NLP 的演进

| 时代 | 方法 | 代表技术 | 特点 |
|------|------|----------|------|
| 1950s-1980s | 规则系统 | 语法规则、词典 | 精确但脆弱，覆盖有限 |
| 1990s-2012 | 统计 NLP | n-gram、HMM、CRF | 数据驱动，但仍需大量特征工程 |
| 2013-2017 | 神经网络 NLP | Word2Vec、RNN/LSTM、Seq2Seq | 端到端学习，词向量 |
| 2018-现在 | 预训练 + 微调 | BERT、GPT、T5 | 大规模预训练，通用语言理解 |
| 2022-现在 | LLM / Prompt | GPT-4, Claude, DeepSeek | 指令跟随，零样本/少样本学习 |

## 核心 NLP 任务

### 理解任务

| 任务 | 含义 | 输入 → 输出 |
|------|------|------------|
| **文本分类** | 给文本分配标签 | "这个产品很棒" → 正面 |
| **命名实体识别 (NER)** | 识别文本中的实体 | "Steve Jobs founded Apple in Cupertino" → [人: Steve Jobs] [组织: Apple] [位置: Cupertino] |
| **关系抽取** | 识别实体间的关系 | "Steve Jobs founded Apple" → (Steve Jobs, 创始人, Apple) |
| **情感分析** | 判断文本的情感倾向 | "服务太差了" → 负面 |
| **文本蕴含 (NLI)** | 判断两句话的逻辑关系 | "猫在垫子上" + "有动物在垫子上" → 蕴含 |
| **问答 (QA)** | 从给定文本中找出答案 | 段落 + 问题 → 答案片段 |

### 生成任务

| 任务 | 含义 | 输入 → 输出 |
|------|------|------------|
| **机器翻译** | 将文本从一种语言翻译为另一种 | 中文 → 英文 |
| **文本摘要** | 生成简洁的内容概括 | 长文章 → 三句话摘要 |
| **对话生成** | 生成连贯的对话回复 | "今天天气真好" → "是啊，适合出去玩" |
| **文本补全** | 继续生成给定文本后续内容 | "人工智能的发展..." → ... |

## 文本预处理

### Tokenization（分词）

将文本切分为 token（基本处理单元）：

| 方法 | 说明 | 示例 |
|------|------|------|
| **按空格分词** | 简单但不够好 | "I'm studying" → ["I'm", "studying"] |
| **BPE (Byte Pair Encoding)** | 数据驱动的子词切分 | "studying" → ["stud", "ying"] |
| **WordPiece** | 类似 BPE，BERT 使用 | "unaffable" → ["un", "##aff", "##able"] |
| **SentencePiece** | 语言无关，直接处理原始文本 | 中文日语无需预分词 |

> 现代 LLM（GPT、Claude 等）几乎都使用子词级别的 tokenization。

### 其他预处理步骤

| 步骤 | 说明 | 是否总是需要 |
|------|------|-------------|
| 小写化 | 统一为小写 | 英文常用，中文不适用 |
| 去停用词 | 移除 "的", "the", "is" | 传统方法需要，现代 DL 不需要 |
| 词干提取/词形还原 | 将词还原为基本形式 | 传统方法需要 |
| 去标点/特殊字符 | 清理非文本内容 | 视任务而定 |

## 从词向量到上下文表示

### 静态词向量

| 方法 | 特点 | 局限 |
|------|------|------|
| **Word2Vec** (2013) | CBOW/Skip-gram，相似词有相似向量 | "bank" 在 "river bank" 和 "bank account" 中是同一个向量 |
| **GloVe** (2014) | 基于全局词共现矩阵 | 同上——无法处理一词多义 |
| **FastText** | 子词信息，处理 OOV | 仍是静态向量 |

### 上下文词向量

| 方法 | 特点 |
|------|------|
| **ELMo** (2018) | 双向 LSTM，每个上下文有不同的词向量 |
| **BERT** (2018) | Transformer 编码器，掩码语言模型预训练 |
| **GPT 系列** | Transformer 解码器，自回归语言模型预训练 |

> 上下文词向量是 NLP 的分水岭——同一个词在不同句子中有不同的向量表示。这是现代 LLM 的基础。

## BERT vs GPT 架构对比

| 特性 | BERT | GPT |
|------|------|-----|
| 架构 | Transformer 编码器（双向） | Transformer 解码器（单向/因果） |
| 预训练目标 | 掩码语言模型 (MLM) | 自回归语言模型 (CLM) |
| 强项 | 理解任务（分类、NER、QA） | 生成任务（对话、写作、翻译） |
| 典型规模 | BERT-base: 110M | GPT-3: 175B |
| 微调方式 | 加分类头微调 | 提示工程 / 指令微调 |

## NLP 评估

| 任务 | 常用指标 |
|------|----------|
| 分类/NER | 准确率、F1 |
| 机器翻译 | BLEU, COMET |
| 文本摘要 | ROUGE |
| 对话/生成 | 人工评估 (ELO)、困惑度 (PPL)、LLM-as-Judge |
