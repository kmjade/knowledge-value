# OpenClaw 简介

> [!info] 核心定位
> OpenClaw 🦞 是一个**开源的个人 AI 助手平台** —— 本地部署、安装即用，你的 AI 智能体军团，住在你自己的电脑上。

## 什么是 OpenClaw

OpenClaw 是一个**自托管 Gateway 网关**：
- 把常用的聊天应用和渠道界面连接到 AI 智能体
- 在自己的机器（或服务器）上运行单个 Gateway 进程
- 成为消息应用和 AI 助手之间的桥梁

### 适合谁使用？

- 希望拥有可从任何地方发消息调用的个人 AI 助手
- 不想放弃数据控制权或依赖托管服务
- 开发者和高级用户

---

## 核心特点

| 特点 | 说明 |
|------|------|
| **自托管** | 在你的硬件上运行，遵循你的规则 |
| **多渠道** | 一个 Gateway 可同时服务内置渠道及多个插件渠道 |
| **智能体原生** | 为具备工具使用、会话、记忆和多智能体路由能力的编码智能体而构建 |
| **开源** | MIT 许可，社区驱动 |
| **本地优先** | 数据不离开你的设备 |

---

## 核心能力

### 多渠道 Gateway 网关
- 通过单个 Gateway 进程使用 Discord、iMessage、Signal、Slack、Telegram、WhatsApp、WebChat 等

### 插件渠道
- 捆绑插件添加 Matrix、Nostr、Twitch、Zalo 等

### 多智能体路由
- 按智能体、工作区或发送者隔离会话

### 媒体支持
- 发送和接收图像、音频和文档

### Web 控制 UI
- 用于聊天、配置、会话和节点的浏览器仪表板

### 移动节点
- 配对 iOS 和 Android 节点，用于 Canvas、相机和支持语音的工作流

---

## 工作原理

```
Chat apps + plugins → Gateway → Pi agent
                              → CLI
                              → Web Control UI
                              → macOS app
                              → iOS/Android nodes
```

Gateway 是会话、路由和渠道连接的单一事实来源。

## 与其他工具的区别

| 特性 | OpenClaw | 托管 AI 服务 | 传统聊天机器人 |
|------|----------|--------------|----------------|
| 自托管 | ✅ | ❌ | 取决于部署 |
| 数据控制 | 完全控制 | 服务商控制 | 取决于部署 |
| 多渠道 | ✅ | 通常单一 | 通常单一 |
| 工具使用 | ✅ | ✅ | ❌ |
| 开源 | ✅ MIT | ❌ | 取决于项目 |
