# 05-计算机网络

> 计算机网络研究计算机之间如何通信——从物理电缆到网页加载，每一层都有协议和设计权衡。

## 为什么分层？

分层架构是网络设计的核心思想——每层只负责一件事，上下层通过接口交互。

## TCP/IP 四层模型

```
┌──────────────────────────────┐
│      应用层 (Application)      │  HTTP, DNS, SMTP, WebSocket
├──────────────────────────────┤
│      传输层 (Transport)        │  TCP, UDP
├──────────────────────────────┤
│      网络层 (Internet)         │  IP, ICMP, 路由协议
├──────────────────────────────┤
│   网络接口层 (Link)            │  以太网, Wi-Fi, ARP
└──────────────────────────────┘
```

| 层 | 数据单元 | 核心协议 | 解决什么问题 |
|----|----------|----------|-------------|
| 应用层 | 消息 | HTTP, DNS | 应用之间的语义 |
| 传输层 | 段 (Segment) | TCP, UDP | 端到端的可靠传输 |
| 网络层 | 包 (Packet) | IP | 跨网络寻址和路由 |
| 链路层 | 帧 (Frame) | Ethernet | 同一网络内的传输 |

## 从输入 URL 到页面加载

```
浏览器输入 example.com
        ↓
1. DNS 解析：example.com → 93.184.216.34
        ↓
2. TCP 三次握手：建立连接
   SYN → ← SYN-ACK → ACK
        ↓
3. TLS 握手 (HTTPS)：加密连接
        ↓
4. HTTP 请求：GET /index.html
        ↓
5. 服务器处理并返回响应
        ↓
6. 浏览器渲染页面
        ↓
7. TCP 四次挥手：关闭连接
```

> 理解这个过程是网络面试和排障的基础。每一步都可能出问题。

## 传输层

### TCP vs UDP

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接（三次握手） | 无连接 |
| 可靠性 | 保证送达、有序、无重复 | 不保证 |
| 流量控制 | 滑动窗口 | 无 |
| 拥塞控制 | 慢启动、拥塞避免 | 无 |
| 速度 | 较慢 | 快 |
| 应用 | HTTP, 数据库, SSH | DNS, 视频流, 游戏, VoIP |

### TCP 关键机制

| 机制 | 作用 |
|------|------|
| **三次握手** | 建立双向连接，同步序列号 |
| **滑动窗口** | 流量控制——接收方告诉发送方能收多少 |
| **拥塞控制** | 防止网络过载——慢启动、拥塞避免、快速重传 |
| **四次挥手** | 关闭连接，双方各发 FIN |

### TCP 拥塞控制算法

| 算法 | 特点 | 使用场景 |
|------|------|----------|
| **Reno** | 经典，快速重传+快速恢复 | 传统网络 |
| **CUBIC** | Linux 默认，针对高带宽延迟积优化 | 现代 Web |
| **BBR** | Google 提出，基于带宽和 RTT 建模 | YouTube, Google 服务 |

## 网络层

### IP 地址

| 版本 | 格式 | 地址空间 |
|------|------|----------|
| IPv4 | 32 位，如 192.168.1.1 | ~43 亿（已耗尽） |
| IPv6 | 128 位，如 2001:db8::1 | 3.4×10³⁸ |

### 子网与 CIDR

192.168.1.0/24 → 前 24 位是网络地址，后 8 位是主机地址。子网掩码 255.255.255.0。

### NAT (网络地址转换)

私有 IP 地址（如 192.168.x.x）通过 NAT 转换为公网 IP——这是缓解 IPv4 耗尽的关键技术。

## 应用层

### HTTP 版本演进

| 版本 | 特点 |
|------|------|
| HTTP/1.0 | 每次请求新建 TCP 连接 |
| HTTP/1.1 | 持久连接、管线化、Host 头（支持虚拟主机） |
| HTTP/2 | 多路复用、头部压缩、服务器推送（二进制帧） |
| HTTP/3 | 基于 QUIC (UDP)，解决队头阻塞 |

### HTTP 状态码

| 范围 | 含义 | 常见 |
|------|------|------|
| 1xx | 信息 | 101 Switching Protocols |
| 2xx | 成功 | 200 OK, 201 Created, 204 No Content |
| 3xx | 重定向 | 301 永久, 302 临时, 304 Not Modified |
| 4xx | 客户端错误 | 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 429 Too Many Requests |
| 5xx | 服务器错误 | 500 Internal Error, 502 Bad Gateway, 503 Service Unavailable, 504 Gateway Timeout |

### DNS

域名 → IP 的分布式数据库。查询链路：
```
浏览器缓存 → OS 缓存 (hosts) → 本地 DNS 服务器 → 根服务器 → TLD 服务器 → 权威服务器
```

### HTTPS 与 TLS

| 版本 | 握手次数 | 安全性 |
|------|----------|--------|
| TLS 1.2 | 2-RTT | 安全，但握手慢 |
| TLS 1.3 | 1-RTT (或 0-RTT) | 更快，移除不安全算法 |
