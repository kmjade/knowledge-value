# 006.3 LLM-Wiki 方法論 — 知識庫總入口

> LLM-Wiki：以 LLM 為編譯器，將原始資訊轉化為結構化知識的系統方法論。
> 本庫記錄此方法論的全部設計、規範與操作指南，是整個 vault 的 META 層。
>
> —— [[../006 Artificial Intelligence|006 AI]] 子庫 · 方法論基礎設施

---

## 三層架構圖 3-Layer Architecture

```
┌──────────────────────────────────────────────────────────────┐
│  Layer 1: raw/ — 原始資料層 (Human-maintained, AI read-only)  │
│  剪藏 · 對話 · 論文 · 書摘 · API 文檔                          │
├──────────────────────────────────────────────────────────────┤
│  Layer 2: wiki/ — 知識編譯層 (AI-exclusive, compiled output)   │
│  concepts/ · entities/ · sources/ · index.md · log.md         │
├──────────────────────────────────────────────────────────────┤
│  Layer 3: CLAUDE.md — 配置與約定層 (Schema, rules, templates)  │
│  三大操作：Ingest → Query → Lint                               │
└──────────────────────────────────────────────────────────────┘
```

## 設計哲學 Design Philosophy

| 原則 | 說明 |
|------|------|
| **AI 只讀 raw/** | raw/ 是人類的領地，AI 絕不寫入——確保原始資料純淨 |
| **AI 獨佔 wiki/** | wiki/ 是 AI 的領地，人類只讀不寫——確保編譯一致性 |
| **增量編譯** | 只處理 `compiled: false` 的文件，支持持續整合 |
| **溯源完整性** | 每條 wiki 知識均可追溯到 raw/ 中的具體來源 |
| **雙語混排** | 繁體中文正文 + English 關鍵術語，面向全球 AI 生態 |
| **復利增長** | 知識通過 wikilinks 連接，每次查詢成果可回寫為 wiki 頁面 |
