# 06 — 跨庫連接

> 跨庫連接是 LLM-Wiki 從「單庫知識」升級為「知識網絡」的關鍵——通過 DDC 映射、wikilinks 模式和分類碼聯動，實現知識庫間的有機互聯。

---

## 一、DDC 映射體系

### 核心映射表

| DDC | 知識庫 | 與 006 AI 的關係 | 連接類型 |
|:---:|--------|-----------------|----------|
| **000** | Knowledge | LLM-Wiki 方法論來源 | 方法論依賴 |
| **004** | Computer Science | 硬體、OS、網路、演算法基礎 | 基礎設施 |
| **004.3** | Programming | Python 等開發工具 | 工具鏈 |
| **004.43** | Python | AI/ML 主要開發語言 | 核心工具 |
| **004.9** | Document Processing | Markdown, Obsidian | 載體 |
| **005** | Software | 軟體工程、架構設計 | 工程實踐 |
| **510** | Mathematics | 概率、線性代數、優化 | 理論基礎 |
| **332.6** | Investment | AI 交易模型 | 應用場景 |

### 映射格式

```markdown
## 跨庫連接 Cross-KB Links

| DDC | 子庫 | 關係 |
|:---:|------|------|
| 004.43 | Python | 開發工具 |
| 510 | 數學 | 概率·線性代數 |
```

## 二、跨庫連結模式

### 模式 1：直接 Wikilink

```markdown
- [[3 Resources/000 Knowledge/004 CS/004.3-Python/Python|Python 知識庫]]
```

**適用**：需要引用另一知識庫的具體頁面
**注意**：使用相對路徑從 vault 根開始

### 模式 2：DDC 碼引用

```markdown
- 參見 DDC 510（數學）中的概率論章節
```

**適用**：不需要引用具體頁面，只需指明域
**優點**：不依賴具體路徑，重構友好

### 模式 3：跨庫概念映射

```markdown
## 相關概念（跨庫）
- [[../../000 Knowledge/wiki/concepts/KOS|KOS 理論]] — 知識組織系統
```

**適用**：概念跨越多個知識庫
**維護**：需要定期檢查跨庫鏈接有效性

### 模式 4：分類碼聯動

```
DDC 006.3 (LLM-Wiki) ←─→ DDC 000.7 (Digital Knowledge)
     │                          │
     └── 共享概念：知識編譯、AI 知識組織
```

## 三、知識圖譜 Knowledge Graph

### 圖譜層級

```
Level 3: Vault Graph      — 所有知識庫的全局圖譜
Level 2: Domain Graph     — 單一 DDC 域的內部圖譜（如 006 AI）
Level 1: Sub-KB Graph     — 單一子庫的內部圖譜（如 006.3 LLM-Wiki）
Level 0: Page Graph       — 單一頁面的入/出鏈接圖
```

## 四、分類碼聯動機制

### 聯動規則

| 規則 | 說明 | 範例 |
|------|------|------|
| **父子繼承** | 子分類自動繼承父分類的跨庫連接 | 006.3 繼承 006 的所有跨庫映射 |
| **關聯擴展** | 同父分類下的兄弟庫自動提示關聯 | 006.3 ↔ 006.7 共享 AI 上下文 |
| **跨域橋接** | 明確定義不同 DDC 域間的橋接概念 | 000.LLM-Wiki ↔ 006.3.LLM-Wiki |

### 聯動配置

在 CLAUDE.md 中定義：

```markdown
## 跨庫

| DDC | 子庫 | 關係 |
|:---:|------|------|
| 000 | Knowledge | 方法論來源（雙向） |
| 004.3 | Python | 工具依賴（單向） |
| 006.7 | GenAI & LLMs | 兄弟庫（雙向） |
```

## 五、連接健康檢查

| 檢查項 | 頻率 | 說明 |
|--------|------|------|
| **死鏈掃描** | 每次 Lint | 檢查所有跨庫 wikilinks 是否有效 |
| **雙向性驗證** | 每月 | 驗證標記為「雙向」的連接是否對稱 |
| **DDC 一致性** | 每季 | 確保 DDC 映射與實際目錄結構一致 |
| **冗餘清理** | 每半年 | 刪除不再需要的跨庫連接 |
