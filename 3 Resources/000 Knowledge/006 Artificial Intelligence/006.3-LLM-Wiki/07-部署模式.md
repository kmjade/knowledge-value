# 07 — 部署模式

> LLM-Wiki 支持三種部署模式，從輕量的 ram/ 基礎到完整的 KB-chapter 級別，可根據知識庫規模和需求靈活選擇。

---

## 部署模式總覽

| 模式 | 結構 | 適用場景 | 規模 |
|:----:|------|---------|:----:|
| **Pattern A** | ram/ + wiki/ + CLAUDE.md | 小型主題、單一概念域 | S |
| **Pattern B** | KB-chapter 級（獨立目錄 + 子 raw/wiki） | 中型子庫、多個概念域 | M |
| **Pattern C** | Integrated Build（全 vault 統一編譯） | 大型 vault、跨庫強耦合 | L |

---

## 一、Pattern A：ram/ 基礎模式

### 結構

```
<topic>/
├── CLAUDE.md          # Schema 定義
├── <topic>.md         # MOC 入口
├── ram/               # 原始資料（人類維護）
│   ├── articles/
│   ├── papers/
│   └── conversations/
├── wiki/              # AI 編譯產物
│   ├── index.md
│   ├── log.md
│   ├── concepts/
│   ├── entities/
│   └── sources/
└── outputs/           # 基於 wiki 的製成品
```

### 特點

| 特點 | 說明 |
|------|------|
| **輕量** | 最簡結構，5 個目錄即可運行 |
| **獨立** | 每個 topic 完全自治 |
| **快速啟動** | 無需與其他庫協調 |
| **限制** | 不支持跨庫查詢 |

### 適用場景

- 個人筆記主題（如「Hermes-Agent 配置」）
- 單一技術棧學習（如「Rust 非同步編程」）
- 小團隊文檔庫

---

## 二、Pattern B：KB-chapter 模式

### 結構

```
<DDC-domain>/
├── CLAUDE.md                 # 域級 Schema
├── <domain>.md               # 域 MOC
├── raw/                      # 域級共享原始資料
│   ├── articles/
│   └── papers/
├── wiki/                     # 域級共享 wiki
│   ├── concepts/
│   └── entities/
│
├── <chapter-01>/             # 子庫 1
│   ├── CLAUDE.md             # 子庫 Schema
│   ├── <chapter-01>.md       # 子庫 MOC
│   ├── 01-xxx.md             # 章節文件
│   ├── raw/                  # 子庫專屬 raw
│   └── wiki/                 # 子庫專屬 wiki
│
├── <chapter-02>/             # 子庫 2
│   └── ...
│
└── 99-Resources/             # 域級共享資源
```

### 特點

| 特點 | 說明 |
|------|------|
| **分層** | 域級 + 子庫級雙層 wiki |
| **共享** | 域級概念跨子庫共享 |
| **規模** | 適合 5–15 個子庫的域 |
| **複雜度** | 需要域級 CLAUDE.md 協調子庫 |

### 適用場景

- DDC 域級知識庫（如 006 AI）
- 大型技術文檔（如公司內部 Wiki）
- 多團隊共享知識庫

---

## 三、Pattern C：Integrated Build 模式

### 結構

```
vault/
├── CLAUDE.md                 # 全局 Schema
├── raw/                      # 全局 raw（可選）
├── wiki/                     # 全局 wiki（可選）
│
├── Domain-001/               # 域 1
│   ├── wiki/  (本地編譯)
│   └── sub-kb/
│
├── Domain-002/               # 域 2
│   └── ...
│
└── _cross-kb/                # 跨庫索引
    ├── global-index.md       # 全局概念索引
    ├── ddc-map.md            # DDC 完整映射
    └── graph.json            # 知識圖譜數據
```

### 特點

| 特點 | 說明 |
|------|------|
| **統一** | 全 vault 共享編譯規則 |
| **全局查詢** | Query 可跨所有域 |
| **圖譜** | 支持全局知識圖譜 |
| **複雜度** | 需要精心設計的全局 Schema |

### 適用場景

- 完整個人知識管理 vault
- 企業級知識庫
- 研究機構跨領域知識管理

---

## 四、批量部署 Batch Deployment

### 部署流程

```
1. 準備階段
   ├── 確認 CLAUDE.md 配置完備
   ├── 準備 raw/ 文件（frontmatter 含 compiled: false）
   └── 確認目錄結構符合所選模式

2. 編譯階段
   ├── /wiki-compile <domain>     → 域級全量編譯
   ├── /wiki-compile --incr       → 增量編譯
   └── /wiki-compile --target <kb> → 定向編譯

3. 驗證階段
   ├── /lint <domain>             → 健康檢查
   ├── 檢查 index.md 更新
   └── 驗證 wikilinks 有效性

4. 發布階段
   ├── git commit + push
   └── 通知相關方更新
```

### 批量部署策略

| 策略 | 說明 | 適用 |
|------|------|------|
| **Big Bang** | 一次性全量編譯所有域 | 新建 vault |
| **Rolling** | 逐域滾動部署 | 已有 vault 擴展 |
| **Canary** | 先編譯 1 個子庫驗證，再推廣 | 新 Schema 上線 |
