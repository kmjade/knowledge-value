# 03 — 編譯工作流

> LLM-Wiki 的三大核心操作：Ingest（導入）、Query（查詢）、Lint（維護），構成完整的知識生命週期。

---

## 工作流總覽

```
         ┌──────────┐     ┌──────────┐     ┌──────────┐
         │  Ingest  │ ──→ │  Query   │ ──→ │   Lint   │
         │ raw→wiki │     │ 檢索推理  │     │ 健康檢查  │
         └──────────┘     └──────────┘     └──────────┘
              │                │                │
              ▼                ▼                ▼
         wiki/ 頁面       查詢答案         修復報告
         index 更新       可回寫 wiki      過時/矛盾/孤立
```

---

## 一、Ingest（導入）：raw/ → wiki/

### 流程

```
1. 掃描 raw/                           掃描 compiled: false 的文件
2. 讀取 CLAUDE.md                      載入概念域定義 + 編譯規則
3. 調用 LLM 提取                       從 raw 提取概念、實體、關係
4. 生成 wiki 頁面                      寫入 concepts/ entities/ sources/
5. 建立 wikilinks                      自動建立頁面間雙向連結
6. 更新 index.md                       追加新頁面到索引
7. 寫入 log.md                         記錄本次編譯操作
8. 標記 compiled: true                 更新 raw 文件的 frontmatter
```

### Ingest 類型

| 類型 | 說明 | 觸發條件 |
|------|------|----------|
| **全量編譯** | 從零構建整個 wiki | 首次初始化或重建 |
| **增量編譯** | 只處理新增 raw 文件 | 日常使用，預設模式 |
| **定向編譯** | 指定特定 raw 文件編譯 | 單篇重要文章處理 |

### 輸出範例

```
Ingest 完成：
  掃描 raw 文件: 7 篇 (compiled: false)
  提取概念: 6 個
  提取實體: 5 個
  建立來源頁: 1 個
  建立 wikilinks: 34 條
  更新 index.md: ✅
  寫入 log.md: ✅
```

## 二、Query（查詢）：index → deep answer

### 流程

```
1. 理解查詢意圖                      解析用戶問題
2. 讀取 wiki/index.md                 獲取全局知識地圖
3. 關鍵詞匹配                         在 index 中定位相關頁面
4. 深度讀取                           載入匹配的概念頁 + 實體頁
5. 跨頁推理                           通過 wikilinks 遍歷知識網絡
6. 綜合回答                           生成結構化答案
7. [可選] 回寫 wiki                   有價值的查詢結果可寫入 wiki
```

### Query 策略

| 策略 | 適用場景 | 說明 |
|------|---------|------|
| **Index-first** | 一般查詢 | 先讀 index 定位，再深入具體頁面 |
| **Full-scan** | 模糊查詢 | 遍歷所有頁面進行語義匹配 |
| **Graph-traverse** | 關聯查詢 | 沿 wikilinks 圖遍歷相關知識 |
| **Cross-KB** | 跨庫查詢 | 通過 DDC 映射跳轉到其他知識庫 |

---

## 三、Lint（維護）：定期健康檢查

### 檢查項目

| 檢查項 | 說明 | 嚴重級別 |
|--------|------|----------|
| **矛盾檢查** | 不同頁面對同一概念有衝突描述 | ERROR |
| **過時檢查** | 頁面超過 N 天未更新（如 90 天） | WARN |
| **孤立檢查** | 頁面沒有任何 wikilinks 指向或被指向 | WARN |
| **死鏈檢查** | wikilinks 指向不存在的頁面 | ERROR |
| **模板合規** | 頁面 frontmatter 缺失必要欄位 | WARN |
| **來源斷鏈** | 概念頁缺少 source 追溯 | INFO |
| **重複檢查** | 多個頁面描述同一概念 | WARN |

### Lint 輸出

```
Lint Report — 2026-06-01
─────────────────────────
ERROR (2):
  - [[Transformer]] 與 [[Attention-Mechanism]] 維度描述矛盾
  - [[Deprecated-Tool]] wikilink 指向不存在頁面
WARN (3):
  - [[Old-Concept]] 超過 90 天未更新
  - [[Orphan-Page]] 孤立頁面（0 in/out links）
  - [[Duplicate-Concept-A]] 與 [[Duplicate-Concept-B]] 疑似重複
INFO (1):
  - [[New-Concept]] 缺少 source 溯源
```

## 四、增量編譯機制

```
raw/ 文件 frontmatter:
  compiled: false  →  待處理（Ingest 輸入）
  compiled: true   →  已處理（跳過）

編譯流程：
  for file in raw/**/*.md:
    if file.compiled == false:
      ingest(file)
      file.compiled = true
      log(f"Ingested: {file}")
```
