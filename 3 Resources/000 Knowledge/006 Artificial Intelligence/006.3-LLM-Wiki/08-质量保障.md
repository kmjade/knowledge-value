# 08 — 質量保障

> LLM-Wiki 的質量保障體系包括 Lint 檢查清單、去重機制、版本控制策略和編譯日誌標準，確保知識庫長期健康。

---

## 一、Lint 檢查清單

### 完整檢查項

| # | 檢查項 | 類型 | 嚴重級別 | 自動修復 |
|---|--------|------|:--------:|:--------:|
| 1 | 矛盾描述 | 兩個頁面對同一概念定義衝突 | 🔴 ERROR | ❌ |
| 2 | 死鏈接 | wikilinks 指向不存在的頁面 | 🔴 ERROR | ❌ |
| 3 | 循環引用 | A→B→C→A 無出口循環 | 🔴 ERROR | ❌ |
| 4 | 過時頁面 | 超過 90 天未更新 | 🟡 WARN | ❌ |
| 5 | 孤立頁面 | 0 入鏈 + 0 出鏈 | 🟡 WARN | ❌ |
| 6 | 弱連接 | 僅 1 入鏈或 1 出鏈 | 🟡 WARN | ❌ |
| 7 | Frontmatter 缺失 | 缺少 title/tags/type/created | 🟡 WARN | ✅ |
| 8 | 來源缺失 | 概念/實體頁缺少 Sources 節 | 🔵 INFO | ❌ |
| 9 | 模板不符 | 頁面結構不符合模板規範 | 🔵 INFO | ❌ |
| 10 | 重複概念 | 多個頁面覆蓋同一概念 | 🟡 WARN | ❌ |
| 11 | 語言混亂 | 非預期的語言使用 | 🔵 INFO | ❌ |
| 12 | 標記不一致 | compiled 標記與實際不符 | 🟡 WARN | ✅ |

### 檢查優先級

```
每次 Lint 執行順序：
  1. 死鏈接檢查（快速，阻塞性）
  2. 矛盾檢查（需語義分析，阻塞性）
  3. 過時/孤立檢查（啟發式，非阻塞）
  4. 模板/來源檢查（結構性，非阻塞）
  5. 重複檢查（語義相似度，非阻塞）
```

## 二、去重機制 Dedup

### 檢測方法

| 方法 | 說明 | 準確率 |
|------|------|:------:|
| **標題相似度** | Levenshtein 距離或餘弦相似度 | 中 |
| **Wikilinks 重疊** | 兩個頁面共享 ≥80% 的出鏈 | 高 |
| **定義相似度** | 定義段落的語義相似度 | 高 |
| **來源重疊** | 兩個頁面引用相同的 source | 中 |

### 合併策略

```
檢測到重複後：
  1. 保留質量更高（字數多、來源多、wikilinks 多）的頁面
  2. 將被合併頁面的獨有 wikilinks 遷移
  3. 在合併目標頁添加「合併自 [[old-page]]」註記
  4. 將被合併頁面標記為 deprecated
  5. 更新 index.md 和 log.md
```

## 三、版本控制 Version Control

### 頁面生命週期

```
draft → reviewed → evergreen → deprecated
  │        │           │            │
  └─ 新建   └─ 人工審核  └─ 穩定長期   └─ 過時/被替代
```

### 狀態標記

```yaml
---
status: draft|reviewed|evergreen|deprecated
reviewed_by: <reviewer>
reviewed_date: YYYY-MM-DD
deprecated_reason: <reason>
superseded_by: [[replacement-page]]
---
```

### Git 策略

| 規則 | 說明 |
|------|------|
| **Commit 粒度** | 每次 /wiki-compile 完成後一個 commit |
| **Commit 信息** | `wiki-compile: <domain> — <summary>` |
| **分支策略** | `main` 為穩定版，`draft` 為編譯草稿 |
| **PR 流程** | draft → PR → review → merge to main |
| **回滾** | 通過 git revert 回滾錯誤編譯 |

## 四、編譯日誌標準

### 日誌格式

```markdown
## [YYYY-MM-DD] <操作類型> | <標題>

**操作類型**: ingest|query|lint|init|merge|deprecate

**耗時**: <seconds>s

**處理文件**:
- `raw/path/file1.md` → `wiki/concepts/X.md`
- `raw/path/file2.md` → `wiki/entities/Y.md`

**統計**:
| 指標 | 數量 |
|------|------|
| 新建頁面 | N |
| 更新頁面 | N |
| 新增 wikilinks | N |
| 修復問題 | N |

**說明**: 本次操作的詳細描述與上下文。
```

### 日誌審計

| 審計項 | 頻率 | 說明 |
|--------|------|------|
| 編譯頻率 | 每月 | 確保定期編譯 |
| 錯誤率 | 每月 | Lint ERROR 數量趨勢 |
| 覆蓋率 | 每季 | wiki 頁面 / raw 文件比例 |
| 連接密度 | 每季 | 平均每頁 wikilinks 數量 |
