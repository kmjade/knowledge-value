# OpenClaw 远程访问

> [!info] 概述
> 配置远程访问让你可以从任何地方使用你的 AI 助手。

## 远程访问方案

| 方案 | 复杂度 | 安全性 | 适用场景 |
|------|--------|--------|----------|
| Tailscale | ⭐ | ✅ 高 | 个人使用 |
| Cloudflare Tunnel | ⭐⭐ | ✅ 高 | 公网访问 |
| 反向代理 | ⭐⭐⭐ | ⚠️ 中 | 自有域名 |
| 端口转发 | ⭐ | ⚠️ 低 | 内网测试 |

---

## Tailscale（推荐）

### 安装 Tailscale

**Linux**
```bash
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up
```

**macOS**
```bash
brew install tailscale
```

**Windows**
下载安装：https://tailscale.com/download

### 配置 OpenClaw

```json
{
  "gateway": {
    "host": "0.0.0.0",
    "port": 18789
  }
}
```

### 访问方式

通过 Tailscale IP 访问：
```
http://100.x.y.z:18789
```

## Cloudflare Tunnel

### 安装 cloudflared

**Linux**
```bash
# 下载
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64
chmod +x cloudflared-linux-amd64
sudo mv cloudflared-linux-amd64 /usr/local/bin/cloudflared
```

**macOS**
```bash
brew install cloudflared
```

### 配置 Tunnel

```bash
# 登录
cloudflared tunnel login

# 创建 Tunnel
cloudflared tunnel create openclaw

# 配置路由
cloudflared tunnel route dns openclaw openclaw.yourdomain.com
```

### 配置文件

`~/.cloudflared/config.yml`:
```yaml
tunnel: <TUNNEL_ID>
credentials-file: /root/.cloudflared/<TUNNEL_ID>.json

ingress:
  - hostname: openclaw.yourdomain.com
    service: http://localhost:18789
  - service: http_status:404
```

### 启动 Tunnel

```bash
cloudflared tunnel run openclaw
```

## 反向代理

### Nginx 配置

```nginx
server {
    listen 80;
    server_name openclaw.yourdomain.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name openclaw.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/openclaw.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/openclaw.yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:18789;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Caddy 配置

```caddyfile
openclaw.yourdomain.com {
    reverse_proxy localhost:18789
}
```

Caddy 自动处理 HTTPS 证书。

## 端口转发

### 路由器配置

1. 登录路由器管理界面
2. 找到「端口转发」或「NAT」设置
3. 添加规则：
   - 外部端口：18789
   - 内部 IP：服务器 IP
   - 内部端口：18789

### 安全警告

> [!warning] 注意
> 直接暴露端口存在安全风险，建议：
> - 使用强认证
> - 配置 IP 白名单
> - 使用非标准端口

## 安全加固

### 启用 HTTPS

```json
{
  "gateway": {
    "ssl": {
      "enabled": true,
      "cert": "/path/to/cert.pem",
      "key": "/path/to/key.pem"
    }
  }
}
```

### 启用认证

```json
{
  "gateway": {
    "controlUi": {
      "auth": {
        "enabled": true,
        "username": "admin",
        "password": "strong-password"
      }
    }
  }
}
```

### IP 白名单

```json
{
  "gateway": {
    "ipWhitelist": {
      "enabled": true,
      "allowed": ["你的IP地址"]
    }
  }
}
```

## 故障排除

### 无法访问

1. 检查防火墙规则
2. 确认 Gateway 正在运行
3. 验证网络连通性

```bash
# 测试本地访问
curl http://localhost:18789/health

# 测试远程访问
curl http://your-domain.com/health
```

### SSL 证书问题

```bash
# 检查证书
openssl s_client -connect openclaw.yourdomain.com:443

# 更新证书
certbot renew
```

### Webhook 不工作

1. 确认 HTTPS 配置正确
2. 检查回调 URL 可访问
3. 查看 Gateway 日志
