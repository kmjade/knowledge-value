# OpenCode LSP 服务器配置

> [!info] 概述
> LSP（Language Server Protocol）让 OpenCode 能够理解代码语义，提供更精准的代码分析。

## 什么是 LSP

LSP 提供以下能力：
- 代码补全
- 跳转定义
- 查找引用
- 重命名
- 诊断（错误、警告）

---

## 配置 LSP

### opencode.json

```json
{
  "lsp": {
    "typescript": {
      "command": "typescript-language-server",
      "args": ["--stdio"]
    },
    "python": {
      "command": "pylsp",
      "args": []
    },
    "go": {
      "command": "gopls",
      "args": []
    }
  }
}
```

---

## 配置字段

| 字段 | 说明 |
|------|------|
| `command` | LSP 服务器启动命令 |
| `args` | 命令行参数 |
| `env` | 环境变量 |
| `enabled` | 是否启用 |

---

## 常用语言服务器

### TypeScript

```json
{
  "typescript": {
    "command": "typescript-language-server",
    "args": ["--stdio"]
  }
}
```

安装：
```bash
npm install -g typescript-language-server typescript
```

### Python

```json
{
  "python": {
    "command": "pylsp",
    "args": []
  }
}
```

安装：
```bash
pip install python-lsp-server
```

### Go

```json
{
  "go": {
    "command": "gopls",
    "args": []
  }
}
```

安装：
```bash
go install golang.org/x/tools/gopls@latest
```

### Rust

```json
{
  "rust": {
    "command": "rust-analyzer",
    "args": []
  }
}
```

安装：
```bash
rustup component add rust-analyzer
```

## 文件类型映射

OpenCode 会根据文件扩展名自动选择 LSP：

| 扩展名 | LSP |
|--------|-----|
| `.ts`, `.tsx` | typescript |
| `.js`, `.jsx` | javascript |
| `.py` | python |
| `.go` | go |
| `.rs` | rust |

## 自定义映射

```json
{
  "lsp": {
    "custom": {
      "command": "my-lsp-server",
      "filetypes": [".custom", ".cust"]
    }
  }
}
```

## LSP 功能使用

OpenCode 会自动使用 LSP 能力：

### 代码理解
```
这个函数在哪里定义？
```

### 查找引用
```
查找这个函数的所有使用位置
```

### 重命名
```
将这个变量重命名为 newName
```

---

## 调试 LSP

### 查看状态

```
/lsp status
```

### 重启 LSP

```
/lsp restart
```

---

## 常见问题

### LSP 未启动

1. 检查命令是否正确
2. 确保 LSP 服务器已安装
3. 检查 PATH 环境变量

### LSP 响应慢

1. 减少项目文件数量
2. 配置排除目录
3. 使用更快的 LSP 实现
