# OpenCode 自定义命令

> [!info] 概述
> OpenCode 支持创建自定义命令，封装可复用的操作流程。

## 命令文件位置

```
.opencode/commands/
├── review.md
├── commit.md
└── test.md
```

---

## 命令文件格式

```markdown
---
name: review
description: 代码审查命令
---

# 代码审查流程

1. 分析代码变更
2. 检查代码质量
3. 检查安全性
4. 生成审查报告

## 审查维度

### 代码质量
- 可读性
- 复杂度
- 重复代码

### 安全性
- SQL 注入
- XSS 漏洞
- 敏感信息

### 最佳实践
- 设计模式
- 错误处理
- 测试覆盖
```

---

## Frontmatter 字段

| 字段 | 必需 | 说明 |
|------|------|------|
| `name` | 是 | 命令名称 |
| `description` | 是 | 命令描述 |
| `aliases` | 否 | 命令别名 |

---

## 使用命令

```
/review
```

或使用别名：

```
/r
```

---

## 常用自定义命令示例

### Git 提交

```markdown
---
name: commit
description: 生成规范的 Git 提交信息
aliases:
  - c
---

# Git 提交

## 步骤

1. 检查暂存区变更
2. 分析变更内容
3. 生成提交信息

## 格式

<type>(<scope>): <subject>

<body>

<footer>

## Type 类型

- feat: 新功能
- fix: 修复
- docs: 文档
- style: 格式
- refactor: 重构
- test: 测试
- chore: 构建
```

### 日报

```markdown
---
name: daily
description: 生成每日工作日报
---

# 日报生成

## 格式

### 今日完成
- [任务1] ...
- [任务2] ...

### 遇到问题
- ...

### 明日计划
- ...
```

### 测试

```markdown
---
name: test
description: 运行测试并分析结果
---

# 测试执行

## 步骤

1. 运行测试命令
2. 分析测试结果
3. 定位失败用例
4. 生成修复建议
```

---

## 命令最佳实践

### 1. 命名清晰

```
/review      ✅
/commit      ✅
/a           ❌ 不明确
/test123     ❌ 无意义
```

### 2. 单一职责

每个命令只做一件事。

### 3. 清晰的步骤

```markdown
## 步骤

1. 第一步
2. 第二步
3. 第三步
```

### 4. 提供示例

```markdown
## 示例

输入：/review
输出：审查报告
```

---

## 命令 vs Skills

| 特性 | 命令 | Skills |
|------|------|--------|
| 复杂度 | 简单 | 复杂 |
| 文件位置 | `.opencode/commands/` | `.opencode/skills/` |
| 上下文 | 直接注入 | 渐进式加载 |
| 适用场景 | 快速指令 | 可复用能力模块 |

**建议**：简单任务用命令，复杂任务用 Skills。
