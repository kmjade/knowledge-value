# OpenCode 自定义工具

> [!info] 概述
> OpenCode 支持通过 MCP 创建自定义工具，扩展 AI 的执行能力。

## 工具的本质

工具是 LLM 在项目中执行操作的能力接口。

**没有工具**：只能生成代码
**有工具**：可以**直接操作项目**

---

## 内置工具一览

| 工具 | 功能 | 风险 |
|------|------|------|
| `bash` | 执行 shell 命令 | 高 |
| `edit` | 编辑文件 | 中 |
| `write` | 创建文件 | 中 |
| `read` | 读取文件 | 低 |
| `grep` | 搜索内容 | 低 |
| `glob` | 查找文件 | 低 |
| `webfetch` | 网络请求 | 中 |

---

## 创建自定义工具

### 方式一：MCP 服务器

通过 MCP 创建自定义工具：

```json
{
  "mcp": {
    "my-tools": {
      "type": "local",
      "command": "my-mcp-server",
      "enabled": true
    }
  }
}
```

### 方式二：Shell 脚本

创建 shell 脚本作为工具：

```bash
#!/bin/bash
# my-tool.sh
echo "Tool executed with args: $@"
```

---

## MCP 工具示例

### 简单 MCP 服务器

```python
# my_mcp_server.py
from mcp import Server

server = Server("my-tools")

@server.tool()
def my_custom_tool(arg: str) -> str:
    """自定义工具描述"""
    return f"Processed: {arg}"

server.run()
```

### 配置

```json
{
  "mcp": {
    "my-tools": {
      "type": "local",
      "command": "python",
      "args": ["my_mcp_server.py"]
    }
  }
}
```

## 工具权限控制

### 基础权限

```json
{
  "permission": {
    "my_custom_tool": "ask"
  }
}
```

### 通配符权限

```json
{
  "permission": {
    "my-tools_*": "ask"
  }
}
```

## 工具设计原则

### 1. 单一职责

每个工具只做一件事。

### 2. 清晰的输入输出

```python
@server.tool()
def process_file(file_path: str) -> dict:
    """处理文件并返回结果"""
    return {
        "status": "success",
        "file": file_path
    }
```

### 3. 错误处理

```python
@server.tool()
def safe_operation(data: str) -> str:
    """安全的操作"""
    try:
        return process(data)
    except Exception as e:
        return f"Error: {str(e)}"
```
