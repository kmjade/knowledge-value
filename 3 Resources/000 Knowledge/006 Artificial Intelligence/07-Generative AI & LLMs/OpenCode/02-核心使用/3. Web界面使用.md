# OpenCode Web 界面使用

> [!info] 概述
> OpenCode 提供了 Web 界面，适合需要图形化操作的场景。

## 启动 Web 界面

```bash
opencode web
```

默认会在浏览器中打开 Web 界面。

### 指定端口

```bash
opencode web --port 8080
```

### 指定主机

```bash
opencode web --hostname 0.0.0.0
```

### 核心功能

| 功能 | 说明 |
|------|------|
| 项目导航 | 浏览项目文件结构 |

## 使用场景

### 1. 远程开发

在远程服务器上启动 Web 界面，通过浏览器访问：

```bash
# 在远程服务器上
opencode web --hostname 0.0.0.0 --port 8080

# 本地浏览器访问
http://remote-server:8080
```

### 2. 团队协作

分享 Web 界面给团队成员：
- 实时协作
- 共享 AI 对话
- 代码评审

### 3. 移动办公

通过平板或手机浏览器访问：
- 查看代码
- 简单编辑
- AI 辅助

## 配置选项

### opencode.json 配置

```json
{
  "web": {
    "port": 8080,
    "hostname": "0.0.0.0",
    "open": true
  }
}
```

### 环境变量

| 变量 | 说明 |
|------|------|
| `OPENCODE_WEB_PORT` | Web 端口 |
| `OPENCODE_WEB_HOST` | Web 主机 |

## 与 TUI 对比

| 特性 | TUI | Web |
|------|-----|-----|
| 运行环境 | 终端 | 浏览器 |
| 图形化 | 否 | 是 |
| 远程访问 | 需要 SSH | 直接访问 |
| 文件编辑 | 外部编辑器 | 内置编辑器 |
| 资源占用 | 低 | 中等 |

---

## 常见问题

### 无法访问 Web 界面

1. 检查防火墙设置
2. 确认端口未被占用
3. 尝试使用 `0.0.0.0` 作为主机名

### 性能问题

1. 减少同时打开的文件数
2. 关闭不必要的浏览器标签
3. 使用更小的模型
