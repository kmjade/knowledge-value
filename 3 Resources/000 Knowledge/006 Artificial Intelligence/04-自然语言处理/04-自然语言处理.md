# 04 — Natural Language Processing 自然語言處理

> NLP is about teaching machines to understand, interpret, and generate human language. From rule-based systems to Transformer-based LLMs — five eras of progress.
> NLP 教機器理解、解釋和生成人類語言。從規則系統到 Transformer —— 五個時代的演進。

---

## NLP 五個時代 Five Eras of NLP

| 時代 Era | 時間 Period | 方法 Approach | 特點 |
|----------|-----------|--------------|------|
| **1. 規則時代** | 1950s–1980s | 手寫語法規則、詞典 | 覆蓋率低，無法處理語言的模糊性 |
| **2. 統計時代** | 1990s–2000s | n-gram、HMM、CRF | 資料驅動，但依賴特徵工程 |
| **3. 詞向量時代** | 2013–2017 | Word2Vec、GloVe、FastText | 語義以稠密向量表示，king−man+woman≈queen |
| **4. 預訓練時代** | 2018–2020 | BERT、GPT、ELMo | "Pre-train + Fine-tune" 範式 |
| **5. LLM 時代** | 2020–至今 | GPT-3/4、Claude、LLaMA | Few-shot、涌现能力、通用 NLP |

## 核心任務 Core NLP Tasks

### 基礎任務 Foundational

| 任務 Task | 說明 Description | 範例 Example |
|-----------|-----------------|-------------|
| **Tokenization** | 將文本切分為 token | "我愛 NLP" → ["我","愛","NLP"] |
| **POS Tagging** | 標註詞性 | 我/PN 愛/VV NLP/NN |
| **NER** (Named Entity Recognition) | 識別命名實體 | [PER 賈伯斯] 創立了 [ORG Apple] |
| **Parsing** | 句法/依存分析 | 主謂賓結構分析 |

### 理解任務 Understanding

| 任務 | 說明 |
|------|------|
| **Text Classification** | 情感分析、主題分類、垃圾郵件檢測 |
| **NLI** (Natural Language Inference) | 判斷兩個句子的邏輯關係（蘊含/矛盾/中立） |
| **Question Answering** | 從文檔中找到答案（抽取式 / 生成式） |
| **Semantic Search** | 基於語義相似度的搜尋 |

### 生成任務 Generation

| 任務 | 說明 |
|------|------|
| **Machine Translation** | 語言翻譯（神經機器翻譯 NMT） |
| **Summarization** | 自動摘要（抽取式 / 抽象式） |
| **Text Generation** | 給定提示生成連貫文本 |
| **Dialogue Systems** | 聊天機器人、任務導向對話 |

## 關鍵模型 Key Models

### 編碼器模型 Encoder Models

| 模型 | 年份 | 特點 |
|------|------|------|
| **BERT** | 2018 | 雙向上下文 (Masked LM)，NLU 任務王者 |
| **RoBERTa** | 2019 | BERT 的最佳化版，更多資料、更大 batch |
| **DeBERTa** | 2020 | Disentangled attention，超越人類基準 |

### 解碼器模型 Decoder Models

| 模型 | 年份 | 特點 |
|------|------|------|
| **GPT** | 2018 | 自迴歸生成，單向（left-to-right） |
| **GPT-2** | 2019 | 1.5B 參數，zero-shot 能力初現 |
| **GPT-3** | 2020 | 175B 參數，few-shot learning |
| **GPT-4** | 2023 | 多模態，推理能力大幅提升 |

### 編碼器-解碼器 Encoder-Decoder

| 模型 | 用途 |
|------|------|
| **T5** | 統一把所有 NLP 任務轉為 text-to-text 格式 |
| **BART** | 降噪自編碼器，擅長生成任務 |

---

## BERT vs GPT

| 維度 | BERT | GPT |
|------|------|-----|
| **架構** | Encoder-only | Decoder-only |
| **注意力方向** | Bidirectional (雙向) | Unidirectional (單向/因果) |
| **預訓練目標** | Masked LM + NSP | Next Token Prediction |
| **擅長** | NLU：分類、NER、QA | NLG：生成、對話、翻譯 |
| **Fine-tune** | 需要 | Few-shot / In-context |

---

## Tokenization 方法

| 方法 | 說明 | 代表 |
|------|------|------|
| **BPE** (Byte-Pair Encoding) | 從字元開始合併最常見的子詞對 | GPT-2/3/4 |
| **WordPiece** | 類似 BPE，但以 likelihood 而非頻率選擇合併 | BERT |
| **SentencePiece** | 語言無關的 subword tokenizer，直接從 raw text 訓練 | T5, LLaMA |
| **Unigram** | 從大字彙開始逐步刪除 | XLNet |

## 評估指標 Evaluation Metrics

| 任務 | 指標 | 說明 |
|------|------|------|
| 分類 | F1, Accuracy | 標準分類指標 |
| 翻譯/摘要 | **BLEU** | n-gram 重疊度 |
| 翻譯/摘要 | **ROUGE** | Recall-oriented，適合摘要 |
| 對話/生成 | **Perplexity** | 模型對測試資料的「困惑度」，越低越好 |
| 語義相似度 | Cosine Similarity | 嵌入向量的餘弦相似度 |
