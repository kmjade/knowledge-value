# OpenCode 格式化工具

> [!info] 概述
> OpenCode 支持配置代码格式化工具，确保生成的代码符合项目规范。

## 配置格式化工具

### opencode.json

```json
{
  "formatter": {
    "javascript": "prettier",
    "typescript": "prettier",
    "python": "black",
    "go": "gofmt",
    "rust": "rustfmt"
  }
}
```

---

## 支持的格式化工具

### JavaScript/TypeScript

| 工具 | 说明 |
|------|------|
| `prettier` | 最流行的代码格式化工具 |
| `eslint --fix` | ESLint 修复模式 |

```json
{
  "formatter": {
    "javascript": "prettier --write",
    "typescript": "prettier --write"
  }
}
```

### Python

| 工具 | 说明 |
|------|------|
| `black` | 固执的 Python 格式化工具 |
| `autopep8` | PEP 8 兼容 |
| `yapf` | Google 的格式化工具 |

```json
{
  "formatter": {
    "python": "black"
  }
}
```

### Go

```json
{
  "formatter": {
    "go": "gofmt -w"
  }
}
```

### Rust

```json
{
  "formatter": {
    "rust": "rustfmt"
  }
}
```

---

## 格式化配置

### 配置文件

格式化工具会自动读取项目中的配置文件：

| 工具 | 配置文件 |
|------|----------|
| Prettier | `.prettierrc`, `.prettierrc.json` |
| Black | `pyproject.toml` |
| ESLint | `.eslintrc`, `.eslintrc.json` |

### 自定义参数

```json
{
  "formatter": {
    "javascript": "prettier --write --print-width 100 --tab-width 2"
  }
}
```

---

## 格式化时机

OpenCode 会在以下时机自动格式化：

1. 创建新文件后
2. 编辑文件后
3. 生成代码后

---

## 禁用自动格式化

```json
{
  "formatter": {
    "enabled": false
  }
}
```

---

## 按语言配置

```json
{
  "formatter": {
    "enabled": true,
    "javascript": {
      "command": "prettier --write",
      "enabled": true
    },
    "python": {
      "command": "black",
      "enabled": true
    },
    "go": {
      "command": "gofmt -w",
      "enabled": false
    }
  }
}
```

---

## 安装格式化工具

### Prettier

```bash
npm install -g prettier
```

### Black

```bash
pip install black
```

### gofmt

Go 自带，无需安装。

### rustfmt

```bash
rustup component add rustfmt
```

---

## 格式化最佳实践

1. **统一配置**：在项目中使用配置文件
2. **团队共享**：将格式化配置提交到 Git
3. **CI 集成**：在 CI 中检查格式化
