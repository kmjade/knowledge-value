# 错误处理

> [!summary] 概述
> Hermes Agent 中工具调用的错误处理机制。

---

## 常见错误类型

| 错误 | 说明 | 处理方式 |
|------|------|---------|
| 命令执行失败 | `exit_code != 0` | 返回错误输出，LLM 自动调整 |
| 文件不存在 | `FileNotFoundError` | 返回清晰错误，LLM 尝试其他路径 |
| 网络错误 | API 请求超时/失败 | 自动重试（内置指数退避） |
| 权限不足 | `PermissionError` | 提示用户授权 |
| 参数错误 | Schema 验证失败 | 返回参数要求，LLM 重新生成 |

---

## Hermes Agent 的内置容错

### 命令审批

```bash
# 危险命令自动拦截，需要用户确认
hermes config set approvals.mode manual   # 始终询问
hermes config set approvals.mode smart    # LLM 辅助判断（推荐）
hermes config set approvals.mode off      # 跳过（--yolo）
```

### 自动重试

- 网络请求：指数退避重试
- 文件操作：路径规范化后重试
- execute_code：超时和异常自动捕获

### 上下文压缩

接近 token 限制时自动压缩上下文，防止因超长而失败：

```bash
hermes config set compression.enabled true
hermes config set compression.threshold 0.5   # 50% 时触发
```

## LLM 自我纠错

Hermes Agent 的 Agent 循环设计使 LLM 能自动纠错：

```
LLM 调用: read_file("/wrong/path.txt")
   ↓
Tool 返回: "File not found: /wrong/path.txt"
   ↓
LLM 调整: search_files(pattern="*.txt") 
   ↓
Tool 返回: ["/correct/path.txt"]
   ↓
LLM 调用: read_file("/correct/path.txt") ✓
```

## 会话恢复

意外中断后恢复：

```bash
hermes --continue          # 恢复最近会话
hermes --resume <id>       # 恢复指定会话
```
