这是一篇给中文用户准备的 **Hermes Agent 配置向导（Setup Wizard）胎教级别教程** 。

> [!-info] -info
> 适用环境
> 
> 本教程以 Ubuntu 系统为例，同样适用于 Windows WSL2 及 Linux **Unix** 一个操作系统家族，Linux 和 macOS 都属于 Unix 体系。Unix 不是一个具体的系统，而是一类系统的统称。Hermes 运行在 Unix 体系的环境中（macOS、Linux），这意味着你可以用终端、管道、文件权限等 Unix 工具来增强 Agent 的能力。Windows 用户通常需要通过 WSL 来使用。 服务器上的安装场景。

> [!-success] -success
> 开始之前
> 
> - **前置条件** ：你已经通过中文社区的安装指令完成了 Hermes Agent 的安装。如果还没有，请先看 [安装教程](https://hermesagent.org.cn/docs/getting-started/installation)
> - **预计耗时** ：约 10 分钟
> - **你需要准备** ：一个大模型提供商的账号（本教程以 [DeepSeek 开放平台](https://platform.deepseek.com/) 的 deepseek-v4-flash 模型为例，价格低廉、性能优秀）；一个 [飞书](https://www.feishu.cn/) 账号（用于配置消息平台）

整个配置分为三大步： **选择模型提供商** → **选择终端后端** → **配置消息平台（可选）** 。下面一步步来。

---

## 第一步：进入配置向导

![安装完成界面](https://hermesagent.org.cn/assets/images/01-install-complete-c5289f95902533739ecf22ef965f234d.png)

如果你已经看到了类似上图的界面，说明 Hermes Agent 已经安装完成了。

在这个界面输入 `1` 并按回车，进入快速设置模式。

---

## 第二步：选择模型提供商

![模型提供商选择](https://hermesagent.org.cn/assets/images/02-select-provider-c4072939a779b8b49b068be3ffb30318.png)

接下来会出现模型提供商选择界面。 **模型提供商** 就是为 Hermes **Hermes Agent** 开源 AI 智能体框架与产品的正式名称。Hermes Agent 是项目和产品的正式名称。文档、GitHub 仓库、安装命令、社区讨论和搜索结果都统一使用这一写法。 提供 AI 大脑的服务商，你需要选一个来驱动 Hermes。

> [!-success] -success
> 不知道选哪个？
> 
> **国内用户推荐选 16. DeepSeek** ——注册简单、价格便宜、国内直连不需要翻墙。本教程以 DeepSeek 为例。

使用方向键 ↑↓ 移动光标到你想要的提供商上，按回车确认。

> [!-info] -info
> 完整提供商列表及中文对照（点击展开）

### 输入 API 密钥

选择提供商后，会提示你输入 API Key（API 密钥）。

> **什么是 API 密钥？** 可以理解为一把"钥匙"，让 Hermes 能以你的身份调用大模型服务。每个提供商都可以在其后台免费生成。

![输入 API Key](https://hermesagent.org.cn/assets/images/03-enter-api-key-bc847d2ca072a2b654a5e6bd77247ca7.png)

这时需要前往模型提供商的后台生成密钥。以 DeepSeek 为例，打开 [https://platform.deepseek.com/api\_keys](https://platform.deepseek.com/api_keys) ，创建一个 API 密钥并复制：

![DeepSeek 后台创建 API 密钥](https://hermesagent.org.cn/assets/images/04-deepseek-api-key-c47e533505894faa30de7d9a0762cd0b.png)

然后回到终端窗口中粘贴（可右键选择粘贴）并回车。

> [!-warning] -warning
> 粘贴后看不到内容？
> 
> 这是正常的！为了保护密钥安全，终端不会显示你粘贴的内容。直接按回车即可。

![粘贴 API Key](https://hermesagent.org.cn/assets/images/05-paste-api-key-b8fde502a331e9e98ec90149628182bf.png)

### 选择具体模型

稍等片刻，Hermes 会向 DeepSeek 发送请求获取可用的模型列表。

选择你想用的模型。这里我选 **deepseek-v4-flash** （社区比较推荐的模型，智力在线且便宜），输入对应编号并回车：

![选择模型](https://hermesagent.org.cn/assets/images/06-select-model-e9caa8c8318d7fbbebaeb6434f246fc5.png)

---

## 第三步：选择终端后端

![终端后端选择](https://hermesagent.org.cn/assets/images/07-terminal-backend-0f605f47ed719ba0a3179b748c7e7820.png)

接下来是 **Terminal Backend（终端后端）** 选择。简单来说就是：Hermes 在 **哪里** 执行命令和代码？

| 选项 | 说明 |
| --- | --- |
| **Local** （默认） | 直接在你的电脑上执行，最简单，推荐新手选这个 |
| Docker **Docker** 一种应用容器化平台，把软件及其依赖打包成独立的"集装箱"。就像把整个厨房装进一个集装箱，不管运到哪家餐厅，打开就能用。Docker 让 Hermes 可以在任何安装了 Docker 的机器上一键部署，不用担心环境差异导致的"在我电脑上能跑"问题。 | 在 Docker 容器 **Docker** 一种应用容器化平台，把软件及其依赖打包成独立的"集装箱"。就像把整个厨房装进一个集装箱，不管运到哪家餐厅，打开就能用。Docker 让 Hermes 可以在任何安装了 Docker 的机器上一键部署，不用担心环境差异导致的"在我电脑上能跑"问题。 中运行，与系统隔离，需要先安装 Docker |
| Modal | 使用 Modal 云平台运行，按量计费 |
| SSH | 通过 SSH 连接到远程服务器执行 |
| Daytona | 使用 Daytona 持久化云开发环境 |
| Vercel Sandbox **Sandbox** 受限制、隔离开的执行环境。让 Agent 在一个更安全的环境里跑命令、改文件、执行代码，避免直接碰到你的真实系统。 | Vercel 提供的云端微虚拟机 |
| Singularity/Apptainer | 面向超算/HPC 集群的 容器 **Docker** 一种应用容器化平台，把软件及其依赖打包成独立的"集装箱"。就像把整个厨房装进一个集装箱，不管运到哪家餐厅，打开就能用。Docker 让 Hermes 可以在任何安装了 Docker 的机器上一键部署，不用担心环境差异导致的"在我电脑上能跑"问题。 方案，适合学术科研场景 |
| Keep current | 保持当前设置不变 |

**大多数用户直接选 Local（默认）即可** ，按回车跳过。

## 第四步：配置消息平台（可选）

> [!-success] -success
> 可以跳过
> 
> 如果你只想先在终端里体验 Hermes，可以选择 **Skip** 跳过这一步。之后随时可以通过 `hermes setup` 重新配置。

![消息平台选择](https://hermesagent.org.cn/assets/images/08-messaging-platform-897df02cab6570b5861e959ff82a9ae8.png)

消息平台指的是把 Hermes **Hermes Agent** 开源 AI 智能体框架与产品的正式名称。Hermes Agent 是项目和产品的正式名称。文档、GitHub 仓库、安装命令、社区讨论和搜索结果都统一使用这一写法。 接入飞书、微信、Discord 等聊天工具，让你可以在手机上跟 Hermes 对话。目前社区最推荐的是 **飞书** ，下面以飞书为例。

### 4.1 选择飞书

使用方向键选中 **Feishu / Lark** 并回车：

![选择飞书](https://hermesagent.org.cn/assets/images/09-feishu-select-210ddbf5e2a1f4359beb989f0f8b1567.png)

### 4.2 选择飞书应用创建方式

这个界面问你用哪种方式创建飞书应用，选第一个（自动创建）回车即可：

![飞书创建方式](https://hermesagent.org.cn/assets/images/10-feishu-setup-method-b0dd8c84f05a4d133a364dcd3d738749.png)

### 4.3 在浏览器中创建飞书应用

终端会显示一个 `open.feishu.cn` 开头的链接，把它复制到浏览器中打开：

![复制飞书链接](https://hermesagent.org.cn/assets/images/11-feishu-open-link-42c9bfbac30eeed30ee9dbe4ca64a565.png)

在打开的页面中，给你的飞书应用起个名字（随便取，比如"Hermes助手"），然后点击 **立即创建** ：

![设定应用名称](https://hermesagent.org.cn/assets/images/12-feishu-create-app-name-1ffe2bd2bfabbeed4c454cd075fd62a8.png)

![创建完成](https://hermesagent.org.cn/assets/images/13-feishu-create-app-done-fb70bcf76bc428e830b5fbfc421769e8.png)

### 4.4 回到终端完成配置

回到终端，接下来会有几个确认界面。

这一步是确认应用已经创建好了，直接回车：

![确认应用创建](https://hermesagent.org.cn/assets/images/14-feishu-confirm-1-56441f4c937fb737c7bc8530586f0fe6.png)

这一步是确认权限配置，同样回车：

![确认权限配置](https://hermesagent.org.cn/assets/images/15-feishu-confirm-2-f85560953cc6a7dff67bbb2f84c4e95d.png)

Hermes 会自动为飞书应用配置所需的权限：

![权限配置中](https://hermesagent.org.cn/assets/images/16-feishu-permissions-a336582b24cae780a4763e0e0616a933.png)

看到 **DM pairing enabled** （私聊配对已开启）后，说明飞书应用配置成功了。继续回车：

![DM pairing 完成](https://hermesagent.org.cn/assets/images/17-feishu-dm-pairing-afcce6386095d28d0b0bbd2522f95892.png)

---

## 第五步：启动 Hermes

配置完成！现在关掉当前终端窗口，打开一个 **新的终端** ，输入以下命令：

```bash
hermes
```

你会看到 Hermes 的聊天界面，可以直接在这里跟它对话了：

![Hermes 聊天界面](https://hermesagent.org.cn/assets/images/18-hermes-chat-15a82a454226311b475f4768977725dc.png)

---

## 第六步：绑定飞书机器人（如果上一步配置了飞书）

如果你在第四步配置了飞书，还需要一个"配对"步骤，让飞书机器人和你的 Hermes **Hermes Agent** 开源 AI 智能体框架与产品的正式名称。Hermes Agent 是项目和产品的正式名称。文档、GitHub 仓库、安装命令、社区讨论和搜索结果都统一使用这一写法。 绑定在一起。

### 6.1 在飞书中给机器人发一条消息

打开飞书，找到你刚创建的机器人（就是第 4.3 步起的那个名字），随便发一句话给它：

![给飞书机器人发消息](https://hermesagent.org.cn/assets/images/19-feishu-bot-message-22052d2fadb7f6231cd3866a7e988dd3.png)

### 6.2 在终端中执行配对命令

机器人收到消息后，Hermes 终端会显示一条 `hermes pairing approve` 开头的命令。打开一个 **新终端** ，把这条命令粘贴进去并回车：

![执行配对命令](https://hermesagent.org.cn/assets/images/20-pairing-approve-d875af41382e86883ec5ba6542dd43ae.png)

看到类似下图的确认信息，说明配对成功：

![配对成功](https://hermesagent.org.cn/assets/images/21-pairing-done-91071079b598ea0e6c9986415cf24073.png)

### 6.3 验证

现在回到飞书，给机器人发条消息试试，它应该能正常回复了：

![飞书机器人正常工作](https://hermesagent.org.cn/assets/images/22-feishu-working-0dd239d36faa5275ad50f6e0f585ae54.png)

---

## 配置完成

恭喜！Hermes Agent 已经配置完成并可以正常使用了。你可以：

- **在终端中对话** ：打开终端输入 `hermes` 开始聊天
- **在飞书中对话** ：直接给机器人发消息
- **重新配置** ：随时运行 `hermes setup` 修改设置
