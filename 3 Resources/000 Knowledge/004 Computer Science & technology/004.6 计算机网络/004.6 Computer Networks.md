## 為什麼分層？Why Layers?

分層架構是網路設計的核心思想——每層只負責一件事，上下層通過介面互動。Layered architecture is the core idea of network design — each layer handles one responsibility.

---

## TCP/IP 四層模型 Four-Layer Model

```
┌──────────────────────────────┐
│   Application Layer 應用層    │  HTTP, DNS, SMTP, WebSocket
├──────────────────────────────┤
│   Transport Layer 傳輸層      │  TCP, UDP
├──────────────────────────────┤
│   Internet Layer 網路層       │  IP, ICMP, Routing Protocols
├──────────────────────────────┤
│   Link Layer 鏈路層           │  Ethernet, Wi-Fi, ARP
└──────────────────────────────┘
```

| 層 Layer | 資料單元 PDU | 核心協議 | 解決的問題 Problem Solved |
|----------|-------------|---------|-------------------------|
| Application | Message | HTTP, DNS | Application semantics |
| Transport | Segment | TCP, UDP | End-to-end reliable delivery |
| Internet | Packet | IP | Cross-network addressing & routing |
| Link | Frame | Ethernet | Same-network transmission |

## 從輸入 URL 到頁面載入 From URL to Page Load

```
Browser enters example.com
    ↓ DNS Lookup (UDP port 53)
    ↓ TCP 3-Way Handshake (SYN → SYN-ACK → ACK)
    ↓ TLS Handshake (HTTPS)
    ↓ HTTP GET Request
    ↓ Server Response (HTML)
    ↓ Browser renders page
```
