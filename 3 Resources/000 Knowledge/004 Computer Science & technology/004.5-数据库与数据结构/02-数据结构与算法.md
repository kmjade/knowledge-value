# 02-数据结构与算法

> 程序 = 数据结构 + 算法。这是 CS 最核心的基石——面试、系统设计、日常编码的基础。

## 算法复杂度

### 大 O 表示法

描述算法随输入规模增长的趋势，忽略常数和低阶项。

| 复杂度 | 名称 | 典型场景 | n=100 时 |
|--------|------|----------|----------|
| O(1) | 常数 | 哈希表查找 | 1 |
| O(log n) | 对数 | 二分查找 | ~7 |
| O(n) | 线性 | 遍历数组 | 100 |
| O(n log n) | 线性对数 | 归并排序、快速排序 | ~664 |
| O(n²) | 平方 | 冒泡排序、嵌套循环 | 10,000 |
| O(2ⁿ) | 指数 | 递归斐波那契 | ~1.27×10³⁰ |
| O(n!) | 阶乘 | 旅行商问题暴力解 | ~9.33×10¹⁵⁷ |

> 经验法则：O(n log n) 以下是"快"，O(n²) 开始在大数据上变慢，O(2ⁿ) 只在 n 很小时可行。

## 基础数据结构

### 数组 (Array)

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 随机访问 | O(1) | 通过索引直接访问 |
| 末尾插入 | O(1)* | 均摊 |
| 中间插入/删除 | O(n) | 需要移动元素 |
| 搜索（未排序） | O(n) | 线性扫描 |

### 链表 (Linked List)

| 类型 | 特点 |
|------|------|
| 单向链表 | 每个节点指向下一个 |
| 双向链表 | 每个节点指向前后 |
| 循环链表 | 尾节点指向头节点 |

| 操作 | 时间复杂度 |
|------|-----------|
| 头插/头删 | O(1) |
| 尾插（有尾指针） | O(1) |
| 按索引访问 | O(n) |
| 搜索 | O(n) |

> 链表 vs 数组：链表插入删除 O(1)（知道位置时），但随机访问 O(n)。数组相反。

### 栈 (Stack) — LIFO

| 操作 | 复杂度 |
|------|--------|
| push / pop / peek | O(1) |

应用：函数调用栈、括号匹配、撤销操作、DFS 的迭代实现。

### 队列 (Queue) — FIFO

| 操作 | 复杂度 |
|------|--------|
| enqueue / dequeue / front | O(1) |

应用：BFS、任务调度、消息队列、打印队列。

### 哈希表 (Hash Table)

| 操作 | 平均 | 最坏 |
|------|------|------|
| 插入/查找/删除 | O(1) | O(n) |

核心：哈希函数 + 冲突解决（拉链法 / 开放寻址法）。

> 哈希表是工程中使用频率最高的数据结构。Python 的 dict、Java 的 HashMap、Go 的 map 都是哈希表。

### 树 (Tree)

| 类型 | 特点 |
|------|------|
| **二叉树** | 每个节点最多 2 个子节点 |
| **二叉搜索树 (BST)** | 左 < 根 < 右，查找/插入 O(log n) 平均 |
| **AVL 树 / 红黑树** | 自平衡 BST，保证 O(log n) 最坏 |
| **堆 (Heap)** | 根是最大/最小值，插入/删除 O(log n) |
| **Trie (前缀树)** | 按字符路径存储字符串，前缀搜索 O(k) |
| **B 树 / B+ 树** | 数据库索引的基础 |

### 图 (Graph)

| 表示方式 | 空间 | 特点 |
|----------|------|------|
| 邻接矩阵 | O(V²) | 快速判断两顶点是否相连 |
| 邻接表 | O(V+E) | 遍历邻居快，稀疏图首选 |

## 核心算法

### 排序算法

| 算法 | 平均 | 最坏 | 空间 | 稳定 |
|------|------|------|------|------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | ✅ |
| 选择排序 | O(n²) | O(n²) | O(1) | ❌ |
| 插入排序 | O(n²) | O(n²) | O(1) | ✅ |
| **归并排序** | O(n log n) | O(n log n) | O(n) | ✅ |
| **快速排序** | O(n log n) | O(n²) | O(log n) | ❌ |
| **堆排序** | O(n log n) | O(n log n) | O(1) | ❌ |

> 工程中：小数组用插入排序（常数小），大数组用快速排序（缓存友好）或 TimSort（Python/Java 默认）。

### 搜索算法

| 算法 | 复杂度 | 前提 |
|------|--------|------|
| 线性搜索 | O(n) | 无 |
| **二分查找** | O(log n) | 有序数组 |
| BFS（广度优先） | O(V+E) | 图/树 |
| DFS（深度优先） | O(V+E) | 图/树 |

### 动态规划 (DP)

解决有**重叠子问题**和**最优子结构**的问题。

两个关键步骤：
1. 定义状态：dp[i] 表示什么？
2. 找到转移方程：dp[i] = f(dp[i-1], dp[i-2], ...)

经典 DP 问题：
- 斐波那契数列
- 0-1 背包
- 最长公共子序列 (LCS)
- 最长递增子序列 (LIS)
- 编辑距离

### 贪心算法

每步做局部最优的选择——不一定全局最优，但当问题满足**贪心选择性质**时是正确的。

经典贪心问题：找零钱、活动选择、Huffman 编码、Prim/Kruskal（最小生成树）。

## 算法设计模式

| 模式 | 思路 | 适用 |
|------|------|------|
| **双指针** | 两个指针遍历，通常 O(n) | 有序数组、链表问题 |
| **滑动窗口** | 维护一个窗口，O(n) | 子数组/子串问题 |
| **快慢指针** | 一快一慢 | 链表环检测、中点 |
| **前缀和** | pre[i] = sum(arr[0..i]) | 区间和查询 O(1) |
| **分治** | 分解 → 解决 → 合并 | 归并排序、快速排序 |
| **回溯** | 尝试 → 回退 → 再尝试 | 排列、组合、N 皇后 |
