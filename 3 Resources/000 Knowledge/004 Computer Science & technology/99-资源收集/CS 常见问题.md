# CS 常见问题

## 十大常见误区

| ❌ 误解 | ✅ 事实 |
|---------|---------|
| 计算机科学 = 编程 | 编程是 CS 的工具，不是 CS 本身。CS 研究的是计算、信息、算法和系统——编程只是表达计算的方式 |
| 学 CS 必须数学好 | 实用 CS（开发）需要逻辑思维而非高等数学。系统编程和 AI 研究才需要较强的数学基础 |
| 框架比基础重要 | 框架 2-3 年过时，数据结构和算法 30 年不变。基础好的人学新框架只需一周 |
| 最新的技术就是最好的 | 新技术不一定成熟。PostgreSQL (1996) 和 Redis (2009) 比很多"下一代"数据库更可靠 |
| 大学教的东西没用 | CS 核心课程（OS、网络、编译）教你的是"为什么"，这些知识让你不被框架绑定 |
| LeetCode 刷题 = 算法能力 | 刷题培养模式识别，真正的算法能力是设计新算法和证明正确性。二者不同 |
| 学再多语言越好 | 深入一门语言 + 了解不同范式（函数式、面向对象）比浅尝五种语言更有价值 |
| NoSQL 会取代 SQL | 关系数据库仍是大多数应用的最佳选择。NoSQL 解决的是特定场景的问题，不是替代品 |
| 分布式系统 = 高并发 | 分布式系统首先解决的是可用性和分区容错，高并发是副产品。理解 CAP 比搞懂 RPS 更重要 |
| 编译器只能编译程序 | 编译原理的思想广泛用于任何"从一种表示转换到另一种"的场景——SQL 解析、模板引擎、配置文件解析 |

## 自学的常见困惑

### "应该先学什么语言？"

| 目标 | 推荐 |
|------|------|
| 理解计算机底层 | **C**——没有魔法，一切都是手动管理 |
| 快速上手 + 面试 | **Python**——语法简单，面试刷题友好 |
| 深入 OOP + 企业开发 | **Java** 或 **C#** |
| 系统编程 + 高性能 | **Rust** 或 **C++** |
| 函数式思维 | **Haskell** 或 **Scala**（先学别的） |

### "需要学 C 吗？"

**强烈建议**。不是必须用 C 工作，而是用 C 理解指针、内存布局、栈 vs 堆——这些知识会提高你在任何语言中的代码质量。

### "刷多少道题够？"

| 目标 | 建议 |
|------|------|
| 小厂面试 | 50-100 道（覆盖基础类型） |
| 大厂面试 | 150-200 道 + 理解背后的原理 |
| 竞赛 | 无上限 |
| 真正提升能力 | 50 道 + 确保每道都理解"为什么是这个解法" |
