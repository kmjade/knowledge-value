# 01-计算机科学概述

> 计算机科学是关于计算的科学——研究什么是可计算的、如何高效地计算、以及如何构建执行计算的系统。

## CS 不是"学编程"

| 计算机科学 (CS) | 编程/软件开发 |
|-----------------|--------------|
| 研究计算本身的性质和极限 | 用编程语言实现特定功能 |
| "为什么 QuickSort 是 O(n log n)？" | "怎么用 Python 写排序？" |
| 设计更好的算法 | 实现给定的算法 |
| 时间尺度：几十年不变 | 时间尺度：框架每 2-3 年换代 |

> CS 和编程的关系就像物理学和工程学：物理学家研究原理，工程师建造东西。好的工程师懂原理。

## CS 简史

| 时期 | 里程碑 |
|------|--------|
| 1830s | Charles Babbage 设计分析机（机械通用计算机），Ada Lovelace 写第一个"程序" |
| 1936 | Alan Turing 提出图灵机——定义了"计算"的数学模型 |
| 1945 | John von Neumann 提出存储程序概念（冯·诺依曼架构） |
| 1948 | Claude Shannon 发表《通信的数学理论》——信息论诞生 |
| 1950s-60s | 高级语言出现：FORTRAN, COBOL, LISP |
| 1969 | ARPANET——互联网的前身 |
| 1970 | Codd 提出关系数据库模型 |
| 1970s | UNIX, C 语言诞生 |
| 1980s | 个人计算机时代（IBM PC, Macintosh） |
| 1990s | 互联网普及、World Wide Web |
| 2000s | 智能手机、云计算 |
| 2010s-至今 | AI/ML 革命、大模型、量子计算原型 |

## 计算理论三大问题

| 问题 | 领域 | 含义 |
|------|------|------|
| **可计算性**：什么能被计算？ | 可计算性理论 | 有些问题计算机永远解不了（如停机问题） |
| **复杂性**：计算有多难？ | 计算复杂性理论 | P vs NP——千禧年难题之一 |
| **效率**：如何算得更快？ | 算法设计与分析 | 这是日常工作中最相关的部分 |

## 冯·诺依曼架构

至今几乎所有计算机的基础模型：

```
┌─────────────────────────────────────┐
│              CPU                     │
│  ┌─────────┐    ┌──────────────┐    │
│  │ 控制单元  │ ←→ │ 算术逻辑单元  │    │
│  └────┬─────┘    │   (ALU)      │    │
│       │          └──────────────┘    │
│       ↓                              │
│  ┌─────────┐                        │
│  │ 寄存器    │                        │
│  └─────────┘                        │
└──────┬──────────────────────────────┘
       │ 总线 (Bus)
       ↓
┌──────────────────────────────────────┐
│              内存 (Memory)             │
│  ┌────────────────────────────────┐  │
│  │ 指令和数据存储在同一内存中        │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
       ↓
┌──────────────────────────────────────┐
│          输入/输出设备 (I/O)           │
│  键盘 鼠标 显示器 磁盘 网络...         │
└──────────────────────────────────────┘
```

核心思想：**程序和数据存储在同一内存中**，CPU 从内存取指令、执行、再取下一条。

## CS 核心概念速查

| 概念 | 一句话解释 |
|------|-----------|
| **抽象** | 隐藏复杂细节，只暴露必要的接口 |
| **算法** | 解决特定问题的明确定义的步骤序列 |
| **数据结构** | 组织和存储数据的方式 |
| **复杂度** | 算法随输入规模增长所需的资源（时间/空间）|
| **并发** | 多个计算同时进行 |
| **缓存** | 用空间换时间——把常用数据放在更快的地方 |
| **协议** | 通信双方约定的规则 |
