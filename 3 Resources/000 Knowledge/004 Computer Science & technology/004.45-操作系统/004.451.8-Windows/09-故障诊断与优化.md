# 09 — 故障診斷與優化 Troubleshooting & Optimization

> Windows 故障排除工具鏈：事件檢視器、WinDbg/minidump 分析、SFC/DISM 系統修復、系統還原與效能調校。
> Event Viewer, WinDbg analysis, SFC/DISM repair, System Restore, MSConfig, and performance optimization.

---

## 診斷工具鏈 Diagnostic Toolchain

```
問題發生 → 事件檢視器 (Event Viewer) → 可靠性監視器 → minidump 分析 → SFC/DISM
                ↓                            ↓                ↓
           錯誤代碼/事件 ID            時間線關聯           系統檔案修復
```

## 事件檢視器 Event Viewer

### 啟動與導航

| 方法 | 指令 |
|:-----|:-----|
| GUI | `eventvwr.msc` |
| PowerShell | `Get-WinEvent` |
| CMD | `wevtutil` |

### 主要記錄類型 Key Log Categories

| 記錄 | 內容 | 關鍵事件 |
|:-----|:-----|:---------|
| Application | 應用程式事件 | 安裝/錯誤/當機 |
| Security | 安全性稽核 | 登入/登出/權限變更 (4624, 4625, 4672) |
| System | 系統元件 | 服務/驅動/核心事件 |
| Setup | 安裝/更新 | 功能更新歷程 |
| Forwarded Events | 轉送的遠端事件 | 集中記錄管理 |

### 關鍵事件 ID 速查

| 事件 ID | 來源 | 說明 |
|:-------:|:-----|:-----|
| 41 | Kernel-Power | 未預期關機 (當機/斷電) |
| 1001 | BugCheck | 藍屏當機 (BSOD) |
| 6008 | EventLog | 未預期關機 |
| 1074 | User32 | 使用者/程序觸發關機 |
| 4624 | Security | 登入成功 |
| 4625 | Security | 登入失敗 |
| 7045 | Service Control Manager | 服務安裝 |
| 7036 | Service Control Manager | 服務狀態變更 |
| 10016 | DistributedCOM | DCOM 權限錯誤 |
| 20001 | WLAN-AutoConfig | Wi-Fi 連線事件 |

### PowerShell 事件查詢

```powershell
# 過去 24 小時的錯誤事件
Get-WinEvent -FilterHashtable @{LogName='System'; Level=2; StartTime=(Get-Date).AddHours(-24)}

# 特定事件 ID
Get-WinEvent -LogName System | Where-Object Id -eq 41 | Select -First 5

# 匯出事件
Get-WinEvent -LogName System -MaxEvents 1000 | Export-Csv system_events.csv
```

## 藍屏分析 BSOD / BugCheck

### Minidump 位置

| 類型 | 位置 |
|:-----|:-----|
| Minidump | `%SystemRoot%\Minidump\` |
| Kernel/Full dump | `%SystemRoot%\MEMORY.DMP` |

### WinDbg 分析流程

```
1. 開啟 WinDbg (Microsoft Store 安裝)
2. File → Open Crash Dump → 選擇 .dmp 檔案
3. 執行: !analyze -v
4. 查看: BUGCHECK 代碼、FAILURE 模組、堆疊
```

### 常見 BugCheck 代碼

| 代碼 | 名稱 | 常見原因 |
|:-----|:-----|:---------|
| 0x1A | MEMORY_MANAGEMENT | RAM 故障、驅動程式 |
| 0x3B | SYSTEM_SERVICE_EXCEPTION | 驅動程式、防毒軟體 |
| 0x50 | PAGE_FAULT_IN_NONPAGED_AREA | RAM 或驅動記憶體錯誤 |
| 0x7E | SYSTEM_THREAD_EXCEPTION_NOT_HANDLED | 驅動不處理例外 |
| 0x9F | DRIVER_POWER_STATE_FAILURE | 電源管理驅動問題 |
| 0xD1 | DRIVER_IRQL_NOT_LESS_OR_EQUAL | 驅動存取無效記憶體 (最常見) |
| 0xEF | CRITICAL_PROCESS_DIED | 關鍵系統行程終止 |
| 0x116 | VIDEO_TDR_FAILURE | GPU 驅動逾時 |
| 0x133 | DPC_WATCHDOG_VIOLATION | 驅動/韌體 DPC 逾時 |
| 0x139 | KERNEL_SECURITY_CHECK_FAILURE | 核心資料結構損毀 |

## SFC / DISM 系統修復

### 修復流程 Standard Repair Flow

```cmd
# 步驟 1：檢查系統檔案 (SFC)
sfc /scannow

# 步驟 2：檢查元件儲存區 (DISM)
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth

# 步驟 3：離線修復 (使用安裝媒體)
DISM /Image:C:\ /Cleanup-Image /RestoreHealth /Source:D:\sources\install.wim

# 步驟 4：再跑一次 SFC
sfc /scannow
```

| 指令 | 時間 | 用途 |
|:-----|:----:|:-----|
| `sfc /scannow` | 5-20 min | 掃描並修復受保護系統檔案 |
| `sfc /verifyonly` | 5-15 min | 僅檢查不修復 |
| `DISM /CheckHealth` | <1 min | 檢查元件儲存區是否標記為損毀 |
| `DISM /ScanHealth` | 10-30 min | 完整掃描元件儲存區 |
| `DISM /RestoreHealth` | 15-60 min | 從 Windows Update 修復 |

## 系統還原與進階啟動 System Restore & Recovery

### 系統還原 System Restore

| 操作 | 指令 / 路徑 |
|:-----|:------------|
| 建立還原點 | `rstrui` → 或 System Properties → System Protection |
| GUI | `sysdm.cpl` → System Protection |
| 命令列 | `wmic.exe /Namespace:\\root\default Path SystemRestore Call CreateRestorePoint "MyRestore", 100, 7` |
| PowerShell | `Checkpoint-Computer -Description "BeforeUpdate"` |

### 進階啟動選項 Advanced Startup

| 路徑 | 說明 |
|:-----|:-----|
| Settings → Recovery → Advanced Startup | 圖形化方式 |
| Shift + Restart | 快速鍵方式 |
| `shutdown /r /o` | 命令列方式 |

### 進階啟動功能

| 選項 | 用途 |
|:-----|:-----|
| Startup Repair | 自動診斷啟動問題 |
| Safe Mode | 安全模式 (含網路/命令提示字元) |
| Command Prompt | 進階 CLI 修復 |
| System Restore | 還原至還原點 |
| System Image Recovery | 從映像備份還原 |
| UEFI Firmware Settings | 進入 BIOS/UEFI |

## MSConfig 系統設定

| 分頁 | 功能 |
|:----:|:-----|
| General | 正常/診斷/選擇性啟動 |

## 效能調校 Performance Optimization

### 磁碟清理

```cmd
cleanmgr                              # GUI
cleanmgr /sageset:1                   # 設定清理類別
cleanmgr /sagerun:1                   # 執行預設清理
cleanmgr /verylowdisk                 # 所有類別

# 清除 Windows Update 快取
DISM /Online /Cleanup-Image /StartComponentCleanup
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase  # 最徹底
```

### 虛擬記憶體 Virtual Memory

| 設定 | 建議 |
|:-----|:-----|
| 初始大小 | RAM × 1.5 |
| 最大值 | RAM × 3 |
| 位置 | SSD (非 OS 碟更好) |
| GUI | `sysdm.cpl` → Advanced → Performance → Settings → Advanced → Virtual Memory |

### 電源計劃 Power Plans

| 計劃 | 用途 | 關鍵差異 |
|:-----|:-----|:---------|
| Balanced | 日常使用 | 效能與省電平衡 |
| High Performance | 桌機/遊戲 | CPU 最低 100% |
| Power Saver | 筆電省電 | 限制效能 |
| Ultimate Performance | 工作站 | 消除微延遲 (Win 10 Pro+) |

```cmd
powercfg /list                      # 列出計劃
powercfg /setactive SCHEME_MIN      # 切換計劃
powercfg /batteryreport             # 電池報告
powercfg /energy                    # 能源效率診斷 (60秒)
powercfg /sleepstudy                # 睡眠研究
```
