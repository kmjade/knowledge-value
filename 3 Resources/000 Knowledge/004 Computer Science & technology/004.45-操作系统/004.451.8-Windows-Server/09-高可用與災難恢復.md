# 09 — 高可用與災難恢復 High Availability & DR

> 確保關鍵服務不中斷：容錯移轉叢集、叢集共用磁碟區 (CSV)、備份策略與 Azure Site Recovery 災難復原。

---

## HA/DR 策略階層 HA/DR Strategy Tiers

| 層級 Tier | 技術 Technology | RTO (恢復時間) | RPO (資料遺失) | 成本 Cost |
|:---:|------|:---:|:---:|:---:|
| **Local HA** | Failover Clustering | 秒~分鐘 | 0 | 🟡 中 |
| **站點 HA** | Stretched Cluster | 秒~分鐘 | 0 (同步) | 🔴 高 |
| **DR (異地)** | Azure Site Recovery / Hyper-V Replica | 分鐘~小時 | 秒~分鐘 | 🟢 低 (Azure) |
| **備份 Backup** | Windows Server Backup / DPM / Veeam | 小時 | 24 小時 (日備份) | 🟡 中 |
| **歸檔 Archive** | Azure Archive Storage / Tape | 天 | > 24 小時 | 🟢 低 |

## 容錯移轉叢集 Failover Clustering

### 叢集驗證與建立 Cluster Validation & Creation

```powershell
# 安裝角色
Install-WindowsFeature -Name Failover-Clustering -IncludeManagementTools -Restart

# 執行叢集驗證 (必要步驟，不可跳過)
Test-Cluster -Node node01,node02 -Include "Storage","Network","System Configuration"

# 建立叢集
New-Cluster -Name "Cluster01" -Node node01,node02 `
  -StaticAddress 192.168.1.50 -AdministrativeAccessPoint DNS

# 設定叢集見證 Quorum Witness
# 選項 1: 檔案共用見證 File Share Witness
Set-ClusterQuorum -FileShareWitness "\\dc01\Witness"

# 選項 2: 雲端見證 Cloud Witness (Azure Storage Account)
Set-ClusterQuorum -CloudWitness -AccountName "storageaccount" -AccessKey "..."

# 選項 3: 磁碟見證 Disk Witness
Set-ClusterQuorum -DiskWitness "Cluster Disk 1"
```

### 仲裁模式 Quorum Modes

| 節點數 Nodes | 建議仲裁模式 Recommended Quorum | 說明 |
|:---:|------|------|
| 1 | Node Majority | 無需見證 |
| 2 | Node + File Share Witness / Cloud Witness | 需要外部見證來打破平手 |
| 3 | Node Majority | 無需見證 |
| 4+ (偶數) | Node + File Share Witness | 增加一個"突破票" |
| 4+ (奇數) | Node Majority | 無需見證 |

### 叢集角色 Cluster Roles

```powershell
# 建立 Generic Service / Application
Add-ClusterGenericServiceRole -ServiceName "MyService" -Name "MyClusteredService"

# 建立檔案伺服器角色 (Scale-Out File Server)
Add-ClusterScaleOutFileServerRole -Name "SOFS01"

# 建立 Hyper-V VM 叢集角色
Add-ClusterVirtualMachineRole -VMName "SRV01"

# 即時移轉角色
Move-ClusterGroup -Name "SRV01" -Node node02

# 維護模式
Suspend-ClusterNode -Name node01 -Drain
# ... 維護 ...
Resume-ClusterNode -Name node01
```

## CSV — 叢集共用磁碟區 Cluster Shared Volume

| 特性 Feature | CSV | 傳統叢集磁碟 Traditional |
|:------|:---:|:---:|
| **同時存取 Concurrent Access** | ✅ 所有節點 | ❌ 僅擁有者節點 |
| **檔案系統** | NTFS / ReFS | NTFS / ReFS |
| **Live Migration** | ✅ 無磁碟所有權變更 | ❌ 需要磁碟移轉 |
| **Direct I/O** | ✅ (無需經過擁有者協調) | ❌ |
| **CSV Cache** | ✅ (讀取快取) | ❌ |
| **適用場景** | Hyper-V VM 儲存, SOFS | 一般叢集應用 |

### CSV 管理 CSV Management

```powershell
# 新增磁碟到 CSV
Add-ClusterSharedVolume -Name "Cluster Disk 2"

# 檢視 CSV 狀態
Get-ClusterSharedVolumeState

# 設定 CSV Cache (每個 VM 建議 512 MB)
(Get-ClusterSharedVolume -Name "Cluster Disk 2").CacheSizeInMB = 512

# 設定 CSV 網路優先順序 (容錯移轉網路)
Get-ClusterNetwork | Where Metric -eq 0 | Set-ClusterNetwork -Metric 1
```

## Windows Server Backup

| 備份類型 Type | 說明 Description | cmdlet |
|:------|------|------|
| **Full Backup** | 完整備份所有資料 | `wbadmin start backup` |
| **Incremental** | 僅上次備份後的變更 (WSB 不支援原生增量) | — (需第三方) |
| **BMR (Bare Metal Recovery)** | 完整系統復原 | `-allCritical` 參數 |
| **System State** | AD DB, SYSVOL, Registry, IIS metabase | `-systemState` |
| **Hyper-V VM** | 個別 VM 備份 | `-hyperv:<VMname>` |

```powershell
# 安裝 Windows Server Backup
Install-WindowsFeature Windows-Server-Backup

# 一次性備份至網路共用
wbadmin start backup -backupTarget:\\nas01\backups -allCritical -quiet

# 建立每日備份排程
$policy = New-WBPolicy
$volume = Get-WBVolume -AllCriticalVolumes
Add-WBVolume -Policy $policy -Volume $volume
$target = New-WBBackupTarget -NetworkPath "\\nas01\backups"
Add-WBBackupTarget -Policy $policy -Target $target
Set-WBSchedule -Policy $policy -Schedule "21:00"
Set-WBPolicy -Policy $policy
```

## Azure Site Recovery (ASR) 災難恢復 Disaster Recovery

| 複寫類型 Replication Type | 來源 Source | 目標 Target | 場景 |
|:------|------|------|------|
| **Hyper-V → Azure** | 內部 Hyper-V VM | Azure | DR to cloud |
| **VMware → Azure** | 內部 VMware VM | Azure | 跨平台 DR |
| **Physical → Azure** | 實體伺服器 | Azure | Legacy 系統 |
| **Hyper-V → Hyper-V (Secondary)** | 內部站點 A | 內部站點 B | 雙站點 DR |

### ASR 部署架構 ASR Deployment Architecture

```
  On-prem Site (Primary)                  Azure (DR Site)
  ┌──────────────────────┐         ┌──────────────────────┐
  │  Hyper-V Hosts       │         │  Azure Recovery Svcs │
  │  ┌────────────────┐  │  HTTPS  │  ┌────────────────┐  │
  │  │  ASR Provider   │──┼────────>│  │  Vault         │  │
  │  │  + Agent        │  │  443    │  │  - Repl policies│  │
  │  └────────────────┘  │         │  │  - Recovery plans│  │
  │                       │         │  └───────┬────────┘  │
  │  ┌────────────────┐  │         │          │            │
  │  │  Process Server │──┼────────>│  ┌───────┴────────┐  │
  │  │  (Cache +       │  │         │  │  Cache Storage │  │
  │  │   Compression)  │  │         │  │  Account       │  │
  │  └────────────────┘  │         │  └────────────────┘  │
  └──────────────────────┘         └──────────────────────┘
```

## Hyper-V Replica (內建非同步複寫) Built-in Async Replication

| 特性 Feature | Hyper-V Replica | ASR |
|:------|:---:|:---:|
| **授權 Licensing** | ✅ 內建 (無額外成本) | 需 Azure 訂閱 |
| **複寫頻率 Replication Frequency** | 30s / 5min / 15min | 30s (Hyper-V), 15min+ (VMware) |
| **容錯移轉 Failover** | 手動 (可腳本) | 手動/自動 (Recovery Plan) |
| **測試容錯移轉 Test Failover** | ✅ | ✅ |
| **多 VM 一致性 Multi-VM Consistency** | ✅ (VSS group) | ✅ |
| **監控與警示 Monitoring** | Event Log only | ✅ Azure Monitor |

```powershell
# 啟用 Hyper-V Replica (來源)
Enable-VMReplication -VMName "SRV01" `
  -ReplicaServerName "hv-dr01.contoso.com" -ReplicaServerPort 80 `
  -AuthenticationType Kerberos -CompressionEnabled $true

# 排程容錯移轉
Start-VMFailover -VMName "SRV01" -ComputerName "hv-dr01.contoso.com"

# 測試容錯移轉
Start-VMFailover -VMName "SRV01" -ComputerName "hv-dr01.contoso.com" -AsTest
# ... 驗證 ...
Stop-VMFailover -VMName "SRV01"
```

## 恢復計劃 Recovery Plan (RTO/RPO Targets)

| 工作負載 Workload | 建議 RTO | 建議 RPO | 技術方案 Solution |
|:------|:---:|:---:|------|
| AD DS Domain Controller | < 15 min | 0 | Multi-DC (no DR needed for single DC) |
| DNS | < 5 min | 0 | AD-Integrated replication |
| DHCP | < 30 min | 24 hr | DHCP Failover + Backup |
| File Server | < 1 hr | < 15 min | DFS-R + Failover Cluster |
| SQL Server | < 5 min | 0 | Always On AG |
| IIS Web | < 15 min | < 5 min | ARR Farm + Shared Config |
| Hyper-V VMs | < 1 hr | < 1 min | ASR / Hyper-V Replica |
