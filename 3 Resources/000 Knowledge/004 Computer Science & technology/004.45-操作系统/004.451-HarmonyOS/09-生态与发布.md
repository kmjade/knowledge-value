# 09 — HarmonyOS 生態與發佈

> HarmonyOS 生態以 AppGallery Connect 為中心，涵蓋應用開發→測試→發佈→營運全流程。
> HarmonyOS 5 NEXT 起徹底脫離 Android 生態，建構獨立的應用分發體系。

---

## AppGallery Connect (AGC)

| 服務 | 說明 |
|------|------|
| **應用管理** | 應用建立、版本管理、證書管理 |
| **質量測試** | 雲測、相容性測試、性能測試 |
| **分發** | 應用上架、灰度發佈、地區管理 |
| **營運分析** | 下載量、活躍用戶、崩潰率分析 |
| **增長服務** | Push Kit、A/B Testing、Remote Config |
| **HMS Core** | 地圖、定位、登入、支付等基礎服務 |

---

## 應用發佈流程 Publishing Workflow

```
開發 DevEco Studio
  → 簽名 Signing
    → 打包 Build HAP/APP
      → 測試 Cloud Test
        → 提交 Submit to AGC
          → 審核 Review
            → 上架 Published
```

| 步驟 | 工具/平台 | 說明 |
|:----:|-----------|------|
| 1 | DevEco Studio | 開發與本地調試 |
| 2 | AGC (簽名) | 建立發佈證書及 Profile |
| 3 | Build | 構建 Release HAP/APP |
| 4 | AGC (測試) | 雲測：自動化 / 相容性測試 |
| 5 | AGC (提交) | 填寫應用資訊、截圖、隱私聲明 |
| 6 | 審核 | 華為審核團隊檢查（1~7 天） |
| 7 | 上架 | 可選擇全量或灰度發佈 |

---

## 原子化服務 Atomic Service 發佈

| 項目 | 說明 |
|------|------|
| **與應用關係** | 原子化服務可獨立發佈，也可隨應用捆綁 |
| **大小限制** | 主包 ≤10MB |
| **入口配置** | 服務中心卡片、NFC 標籤、URL Scheme |
| **審核** | 與應用審核同流程，但更輕量 |
| **更新** | 服務端即時更新，無需用戶操作 |

---

## HUAWEI ID 與 Push Kit

| 服務 | 說明 |
|------|------|
| **HUAWEI ID (Account Kit)** | 統一帳號體系，支援手機/郵箱/第三方登入 |
| **Push Kit** | 推播服務，支援通知欄/數據消息 |
| **Push 類型** | Notification Message (展示型) / Data Message (數據型) |
| **Push 通道** | 長連接，高到達率 |
| **IAM (In-App Messaging)** | 應用內消息，增強用戶留存 |

## Android 兼容說明 (pre-5.0)

| HarmonyOS 版本 | Android 兼容性 | 說明 |
|:--------------:|:--------------:|------|
| **1.0 ~ 4.x** | ✅ 兼容 AOSP | 可運行 Android APK (透過 AOSP 兼容層) |
| **5 NEXT** | ❌ 不再兼容 | 僅支援 HAP 原生格式，徹底脫離 Android |
| **遷移** | ArkUI-X 跨平台 | 提供 Android/iOS 跨平台方案，而非反向兼容 |
