# 05 — HarmonyOS 分散式能力

> HarmonyOS 的核心差異化能力：透過分散式軟總線實現多設備無縫協同。
> 包含分散式數據、任務調度、跨設備遷移、超級設備與原子化服務。

---

## 分散式能力全景

| 能力 | 說明 | 典型場景 |
|------|------|----------|
| **分散式數據** | 多設備數據即時同步 | 筆記在手機/平板/PC 即時同步 |
| **分散式任務調度** | 跨設備遠端調用 Ability | 手機調用智慧屏的攝像頭 |
| **跨設備遷移** | 應用運行狀態跨設備無縫遷移 | 手機視頻一鍵流轉到智慧屏 |
| **超級設備** | 多設備虛擬化為一個超級終端 | 手機+平板+PC 組成超級工作站 |
| **原子化服務** | 免安裝、即用即走的輕量服務 | 掃碼點餐、NFC 乘車 |

---

## 分散式數據 Distributed Data

| 特性 | 說明 |
|------|------|
| **KV 數據庫** | 分散式 KVStore，多設備自動同步 |
| **RDB** | 分散式關聯式數據庫，支援 SQL |
| **DataShare** | 跨應用數據共享，類似 Android ContentProvider |
| **一致性** | 支援最終一致性、強一致性配置 |
| **衝突解決** | 支援 LWW (Last Write Wins) 與自定義策略 |
| **離線支援** | 設備離線後本地操作，聯網後自動合併 |

---

## 跨設備遷移 Continuation

| 遷移模式 | 說明 |
|----------|------|
| **應用遷移** | 整個應用狀態遷移到另一設備繼續運行 |
| **頁面遷移** | 單個頁面棧遷移 |
| **數據遷移** | 僅遷移特定數據對象 |
| **單向遷移** | A→B 單方向 |
| **雙向遷移** | A↔B 可來回切換 |
| **觸發方式** | NFC 碰一碰 / 超級設備拖拽 / API 調用 |

---

## 超級設備 Super Device

| 概念 | 說明 |
|------|------|
| **定義** | 多個物理設備虛擬化為一個邏輯「超級終端」 |
| **資源池化** | 所有設備的攝像頭、螢幕、喇叭、麥克風等硬體資源統一池化 |
| **發現** | 透過分散式軟總線自動發現附近設備 |
| **組建** | 用戶拖拽設備圖標即可組建超級設備 |
| **場景** | 手機通話→智慧屏接聽、平板畫布→PC 鍵鼠輸入 |

---

## 原子化服務 Atomic Service

| 特性 | 說明 |
|------|------|
| **免安裝** | 無需下載安裝，即點即用 |
| **入口** | 服務中心、負一屏、NFC 碰一碰、掃碼、搜索 |
| **格式** | HAP (輕量級) |
| **大小限制** | ≤10MB (推薦) |
| **生命週期** | 使用完即銷毀，不留殘留 |
| **更新** | 服務端即時更新，用戶無感知 |
| **典型場景** | 點餐、購票、導航、快遞查詢 |
