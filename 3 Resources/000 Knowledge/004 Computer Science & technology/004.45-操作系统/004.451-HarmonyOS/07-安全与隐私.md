# 07 — HarmonyOS 安全與隱私

> HarmonyOS 採用分級安全模型，從內核到應用層層防護。
> 微內核最小 TCB、TEE 可信執行環境、應用簽名與權限管理構成完整安全體系。

---

## 安全架構總覽

```
┌────────────────────────────────────┐
│  應用安全 (App Signing / Perm.)     │
├────────────────────────────────────┤
│  數據安全 (Encryption / DP / ACL)  │
├────────────────────────────────────┤
│  系統安全 (Verified Boot / TEE)     │
├────────────────────────────────────┤
│  內核安全 (Microkernel TCB <10K)   │
└────────────────────────────────────┘
```

---

## 權限模型 Permission Model

| 權限類型 | 授權方式 | 說明 | 範例 |
|----------|:--------:|------|------|
| **system_grant** | 安裝時自動授權 | 低風險，無需用戶確認 | 網絡、震動 |
| **user_grant** | 運行時彈窗授權 | 敏感權限，需用戶顯式同意 | 相機、位置、通訊錄 |
| **權限組** | 按組管理 | 同類權限歸為一組 | CAMERA 組含拍照+錄像 |

### 安全等級分層 Data Security Levels

| 等級 | 標記 | 說明 | 範例數據 |
|:----:|------|------|----------|
| **S0** | 無保護 | 無安全要求 | 公開配置 |
| **S1** | 基礎保護 | 設備內本地保護 | 應用偏好設置 |
| **S2** | 增強保護 | 設備認證後訪問 | 個人照片、文件 |
| **S3** | 高級保護 | 生物識別+安全晶片 | 支付憑證、密碼 |
| **S4** | 最高保護 | 專用安全硬體隔離 | 銀行密鑰、數字身份 |

## 核心安全技術

| 技術 | 說明 |
|------|------|
| **Verified Boot** | 引導時逐級驗證 bootloader→kernel→system 完整性 |
| **TEE (Trusted Execution Environment)** | 基於 ARM TrustZone 的可信執行環境，隔離敏感運算 |
| **HUKS (Huawei Universal KeyStore)** | 統一密鑰管理服務，金鑰不出 TEE |
| **應用簽名** | 所有 HAP 必須簽名，調試簽名 vs 發佈簽名 |
| **沙箱隔離** | 每個應用運行在獨立沙箱，APL (Ability Privilege Level) 控制 |
| **生物識別** | 指紋、人臉識別，模板存儲在 TEE 安全區域 |
| **加密算法** | AES-256-GCM / RSA-2048 / ECC / SM2/3/4 (國密) |

## 應用簽名機制

| 簽名類型 | 用途 | 有效期 |
|----------|------|:------:|
| **調試簽名** (Debug) | 開發階段真機調試 | 1 年 |
| **發佈簽名** (Release) | 應用市場上架 | 25 年 |
| **簽名工具** | hap-sign-tool (CLI) / DevEco Studio 自動簽名 | — |
| **證書類型** | 調試證書 / 發佈證書 | — |
| **Profile** | 描述應用權限與設備範圍 | 與證書綁定 |

## 隱私 Privacy

| 特性 | 說明 |
|------|------|
| **隱私聲明** | 應用必須在 AppGallery Connect 填寫隱私聲明 |
| **最小化權限** | 僅申請必要權限，過度申請會被審核拒絕 |
| **剪貼簿保護** | 應用讀取剪貼簿需用戶授權 |
| **模糊定位** | 支援提供大致位置而非精確位置 |
| **廣告 ID** | 可重置的廣告標識，保護用戶隱私 |
| **權限使用記錄** | 系統記錄權限調用，用戶可查看 |
