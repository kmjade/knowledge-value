# 02 — HarmonyOS 系統架構

> HarmonyOS 採用分層架構：Kernel → System Service → Framework → Application。
> 核心創新在於分散式軟總線、確定時延引擎與 ArkCompiler 方舟編譯器。

---

## 四層架構總覽

```
┌──────────────────────────────────────────────┐
│  Application Layer (應用層)                    │
│  System Apps / Third-party Apps / Atomic Svc  │
├──────────────────────────────────────────────┤
│  Framework Layer (框架層)                      │
│  ArkUI Framework / Ability Kit / SDK          │
├──────────────────────────────────────────────┤
│  System Service Layer (系統服務層)              │
│  Distributed Soft Bus / Package / Power / ... │
├──────────────────────────────────────────────┤
│  Kernel Layer (內核層)                         │
│  HarmonyOS Microkernel / Linux / LiteOS       │
└──────────────────────────────────────────────┘
```

---

## 各層詳細

| 層級 | 組件 | 功能描述 |
|------|------|----------|
| **Kernel** | 鴻蒙微內核 / LiteOS-M / LiteOS-A | 行程調度、記憶體管理、IPC、驅動框架 |
| **Kernel** | HDF (Hardware Driver Foundation) | 統一硬體驅動框架，跨設備復用 |
| **System Service** | Distributed Soft Bus | 分散式軟總線：設備發現、連接、數據傳輸 |
| **System Service** | Distributed Data Management | 分散式數據同步與管理 |
| **System Service** | Distributed Scheduler | 跨設備任務調度與遷移 |
| **System Service** | Determ. Latency Engine | 確定時延引擎：資源調度的即時精確控制 |
| **Framework** | ArkUI | 聲明式 UI 框架（跨設備自我調整） |
| **Framework** | ArkTS | 擴展 TypeScript 的開發語言 |
| **Framework** | Ability Kit | 應用模型：FA/Stage Model |
| **Application** | HAP / HSP | 應用封裝與分發格式 |

---

## 分散式軟總線 Distributed Soft Bus

| 能力 | 說明 |
|------|------|
| **設備發現** | 基於藍牙/WiFi/NFC 的自動發現，無需用戶手動配對 |
| **設備連接** | 建立安全加密通道，支援 P2P 直連 |
| **數據傳輸** | 高速低延遲，支援流媒體、文件、控制指令 |
| **拓撲管理** | 支援星型、網狀、鏈路聚合 |
| **協議統一** | 抽象底層硬體差異，上層 API 統一 |

---

## ArkCompiler (方舟編譯器)

| 特性 | 說明 |
|------|------|
| **AOT + JIT** | 混合編譯：安裝時 AOT 預編譯 + 運行時 JIT 熱點優化 |
| **ArkTS 支援** | 將 ArkTS 編譯為高效字節碼或機器碼 |
| **類型推導** | 靜態類型分析，消除運行時類型檢查開銷 |
| **並發 GC** | 並發垃圾回收，減少 UI 卡頓 |
| **IR 優化** | 多層中間表示優化流水線 |

---

## 確定時延引擎 Determ. Latency Engine

| 指標 | 說明 |
|------|------|
| **目標** | 任務響應延遲可預測、可保證 |
| **機制** | 優先級繼承、頻寬預留、時間感知調度 |
| **應用場景** | 車機 V2X、工業控制、AR/VR 即時渲染 |
| **對比 Linux** | CFS 公平調度無法提供確定性延遲保證 |
