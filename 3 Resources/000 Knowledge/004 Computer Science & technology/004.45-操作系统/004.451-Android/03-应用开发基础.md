# 03 應用開發基礎 App Development Basics

> Kotlin 為 Android 官方首選語言，搭配 Android Studio + Gradle + Jetpack 生態，涵蓋四大元件與現代宣告式 UI。
> Kotlin is the official preferred language, with Android Studio + Gradle + Jetpack ecosystem, covering four components and modern declarative UI.

## 開發環境 Development Environment

| 工具 | 用途 | 關鍵配置 |
|------|------|------|
| **Android Studio** | IDE | Hedgehog+ 版本，內建 Emulator |
| **Gradle (Kotlin DSL)** | 構建系統 | `build.gradle.kts` — dependencies, plugins |
| **Kotlin** | 主要語言 | 1.9+ / 2.0 K2 compiler |
| **AndroidX / Jetpack** | 支援庫 | 取代 legacy Support Library |
| **ADB** | 調試橋接 | `adb shell`, `adb logcat`, `adb install` |

## Gradle 核心配置

```kotlin
// build.gradle.kts (app module)
android {
    namespace = "com.example.myapp"
    compileSdk = 35
    defaultConfig {
        applicationId = "com.example.myapp"
        minSdk = 26
        targetSdk = 35
        versionCode = 1
        versionName = "1.0"
    }
    buildFeatures { compose = true }
    composeOptions { kotlinCompilerExtensionVersion = "1.5.0" }
}

dependencies {
    implementation("androidx.core:core-ktx:1.13.0")
    implementation("androidx.compose.material3:material3:1.2.0")
}
```

## 四大元件 Four Components

| 元件 Component | 角色 Role | 生命週期管理 | 註冊方式 |
|------|------|------|------|
| **Activity** | 單一 UI 畫面，使用者互動入口 | `onCreate` → `onStart` → `onResume` → `onPause` → `onStop` → `onDestroy` | Manifest + Intent |
| **Service** | 背景長時間執行任務 | `onCreate` → `onStartCommand` → `onDestroy` | Manifest + `startService()` / `bindService()` |
| **BroadcastReceiver** | 接收系統或 App 廣播事件 | `onReceive()` (短暫) | Manifest (靜態) 或 `registerReceiver()` (動態) |
| **ContentProvider** | 跨 App 結構化資料共用 | 自動管理 | Manifest + URI authority |

## Manifest 結構

```xml
<manifest xmlns:android="..."
    package="com.example.myapp">
    <uses-permission android:name="android.permission.INTERNET"/>
    <application
        android:allowBackup="true"
        android:label="@string/app_name">
        <activity android:name=".MainActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
    </application>
</manifest>
```

## Jetpack Compose vs View 系統

| 面向 | View (XML) | Jetpack Compose |
|------|------|------|
| **UI 定義** | XML layout + Java/Kotlin binding | Kotlin DSL，宣告式 |
| **狀態管理** | 手動 `findViewById` / ViewBinding | `remember` / `mutableStateOf` |
| **預覽** | Layout Editor + Design 模式 | `@Preview` composable 即時預覽 |
| **效能** | View 層級深時重繪昂貴 | 智能重組 (recomposition) |
| **學習曲線** | 傳統 Android 開發者熟悉 | 新專案首選，Google 官方主推 |
