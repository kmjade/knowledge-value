# 09 發布與分發 Publishing & Distribution

> Google Play Console 為主要分發渠道，App Bundle (AAB) 取代 APK 以實現動態交付，內部測試軌道與 Firebase 整合確保品質。
> Google Play Console is the primary distribution channel, App Bundle (AAB) replaces APK for dynamic delivery, internal testing tracks and Firebase integration ensure quality.

## APK vs AAB

| 面向 | APK | Android App Bundle (AAB) |
|------|------|------|
| **格式** | 單一完整安裝包 | 壓縮包 (base + config splits) |
| **上傳** | 直接上傳 Play Store | 上傳 AAB，Google Play 生成 APK |
| **大小** | 包含所有資源密度/語言/架構 | 按裝置動態拆分，減少下載大小 |
| **動態功能** | ❌ | ✅ Dynamic Feature Modules |
| **Play Feature Delivery** | ❌ | ✅ Install-time / On-demand / Conditional |
| **Google Play Signing** | 可選 | **強制** (Play 管理簽名金鑰) |

## Google Play Console 發布流程

| 步驟 | 內容 |
|:---:|------|
| 1 | **建立應用** — 名稱、語言、App 分類 |
| 2 | **商店設定** — 圖標 (512×512)、螢幕截圖、特色圖、影片 |
| 3 | **內容分級** — IARC 問卷 (年齡分級) |
| 4 | **定價與分發** — 國家/地區、價格 (免費/付費) |
| 5 | **上傳 AAB** — 第一個正式版本 |
| 6 | **隱私權政策** — 必填 URL |
| 7 | **審核提交** — Google 審核 (數小時～數天) |

## 測試軌道 Testing Tracks

| 軌道 | 使用者 | 審核 | 用途 |
|------|:---:|:---:|------|
| **Internal Testing** | 最多 100 人 | ❌ 無需審核 | 快速開發迭代 (分鐘級上線) |
| **Closed Testing (Alpha)** | 指定名單/社群 | ⚠️ 新 App 需 20 人 14 天 | 小範圍驗證 |
| **Open Testing (Beta)** | 公開試用 | ✅ 需審核 | 大規模測試 |
| **Production** | 所有使用者 | ✅ 完整審核 | 正式上線 |

## Firebase 整合

| 服務 | 用途 |
|------|------|
| **Crashlytics** | 即時崩潰報告、Non-fatal 異常 |
| **Analytics** | 用戶行為、留存、轉換漏斗 |
| **Remote Config** | 無需發版即可變更 App 行為 |
| **App Distribution** | 內部測試分發 (取代 email APK) |
| **Performance Monitoring** | App 啟動時間、HTTP 請求延遲 |
| **A/B Testing** | 與 Remote Config 整合的實驗平台 |

```kotlin
// Crashlytics 自訂 log
Firebase.crashlytics.log("User action: checkout")
Firebase.crashlytics.setCustomKey("cart_items", cart.size)

// Remote Config
val config = Firebase.remoteConfig
config.setDefaultsAsync(R.xml.remote_config_defaults)
config.fetchAndActivate().addOnCompleteListener { /* apply changes */ }
```

## Play 政策關鍵 Policy Key Points

| 政策 | 重點 |
|------|------|
| **隱私權政策** | 必須有有效的隱私權政策 URL |
| **資料安全性** | 必須在 Play Console 填寫 Data Safety 表單 |
| **權限宣告** | 敏感權限需提交使用說明影片 |
| **目標 API** | 新 App 必須 target 最新主要版本 -1 以內 |
| **惡意軟體** | 禁止任何形式的惡意行為 |
| **廣告** | 必須遵循廣告政策，宣告廣告 SDK |
