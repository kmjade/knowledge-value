# Network Automation & Operations 自動化運維

## Why Automate?

- Eliminate manual CLI errors
- Consistent configuration across devices
- Faster deployments and recovery
- Infrastructure as Code (IaC)

## Tools

| Tool | Use |
|------|-----|
| **Ansible** | Configuration management |
| **Terraform** | Infrastructure provisioning |
| **Python + Netmiko** | Scripted CLI automation |
| **Nornir** | Python automation framework |
| **Git** | Version control for configs |

## NetDevOps Principles

| Principle | Implementation |
|-----------|---------------|
| **Version Control** | Network configs in Git |
| **CI/CD** | Test configs before deploy |
| **Monitoring** | Telemetry, not just SNMP polling |
| **Declarative** | Define desired state, not steps |

> 💡 Network automation is DevOps applied to networking — treat network configs like code.
