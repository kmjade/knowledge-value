# 06 — 內容平台與社交 Content Platforms & Social Media

> 內容是 Web 的本質——從靜態頁面到 UGC 生態，CMS、SEO 與社交 API 構成「內容生產→分發→消費」的閉環。

---

## CMS 對比 CMS Comparison

| CMS | 類型 | 適合 | API |
|-----|------|------|-----|
| **WordPress** | 傳統 CMS（PHP/MySQL） | 部落格、企業站 | REST（需擴展） |
| **Strapi** | Headless CMS（Node.js） | 前後分離專案 | REST + GraphQL 原生 |
| **Ghost** | 內容優先 CMS | 出版、Newsletter | Content API 原生 |
| **Sanity** | Real-time Headless CMS | 協作編輯 | GROQ + GraphQL |
| **Contentful** | Headless CMS（SaaS） | 企業多通路 | REST + GraphQL |

---

## SEO 核心要素 SEO Essentials

| 維度 | 內容 | 關鍵指標 |
|------|------|---------|
| **技術 SEO** | Core Web Vitals, HTTPS, 行動優先 | LCP < 2.5s, CLS < 0.1 |
| **內容 SEO** | 標題、Meta、Schema.org（JSON-LD） | CTR, 跳出率 |
| **連結** | 內部連結結構 + Backlinks | Domain Authority |
| **在地 SEO** | Google My Business, NAP 一致 | 在地搜尋排名 |

---

## 社交媒體 API Social Media APIs

| 平台 | API | 用途 | 限制 |
|------|-----|------|------|
| **Meta (FB/IG)** | Graph API | 發布、洞察、廣告 | 嚴格審查 |
| **X (Twitter)** | API v2 | 推文、分析 | 分級收費 |
| **LinkedIn** | Marketing/Community API | B2B 內容 | 需審核 |
| **YouTube** | Data API v3 | 影片上傳/分析 | 配額 (quota) |

---

## RSS / Atom 聚合 Syndication

| 格式 | 特徵 | 使用 |
|------|------|------|
| **RSS 2.0** | 簡潔，最普及 | 多數部落格 |
| **Atom** | 更嚴格規範，支援完整內容 | 技術社群 |
| **JSON Feed** | 現代化 JSON 格式 | 開發者友好 |

> 關鍵字：RSS Reader（Inoreader, Feedly）、RSS-to-email、webhook 觸發

---

## 郵件服務 Email Services

| 服務 | 類型 | 特色 |
|------|------|------|
| **SendGrid** (Twilio) | Transactional + Marketing | 高送達率，API 豐富 |
| **Mailchimp** | Marketing | 自動化流程, 模板編輯器 |
| **Resend** | Transactional（開發者優先） | React Email, 簡潔 API |
| **Amazon SES** | Transactional（低成本） | 大量發送 $0.10/1000 封 |
