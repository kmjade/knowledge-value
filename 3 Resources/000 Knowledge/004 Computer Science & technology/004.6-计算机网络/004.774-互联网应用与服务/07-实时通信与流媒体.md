# 07 — 即時通訊與流媒體 Real-time Communication & Streaming

> 從 WebRTC 點對點視訊到 Kafka 事件流——即時數據傳輸是現代互聯網應用的核心脈搏。

---

## WebRTC 即時通訊

| 元件 | 作用 | 協議 |
|------|------|------|
| **getUserMedia** | 捕獲鏡頭/麥克風 | 瀏覽器 API |
| **RTCPeerConnection** | P2P 媒體/數據傳輸 | SRTP/SCTP |
| **RTCDataChannel** | 任意數據 P2P | SCTP (低延遲) |
| **STUN / TURN** | NAT 穿透 / 中繼 | UDP |
| **Signaling** | 交換 SDP/ICE 候選 | WebSocket（自訂） |

---

## 流媒體協議 Streaming Protocols

| 協議 | 延遲 | 基於 | 適用場景 |
|------|------|------|---------|
| **HLS** | 6–30s | HTTP (TS/fMP4 分段) | VOD, 直播（主流） |
| **RTMP** | 2–5s | TCP 持久連接 | 推流端（OBS → 伺服器） |
| **WebRTC** | <1s | UDP (SRTP) | 視訊會議、超低延遲直播 |
| **MPEG-DASH** | 6–30s | HTTP (自適應) | VOD（HLS 替代） |
| **SRT** | <1s | UDP (可靠) | 專業廣播傳輸 |

## Web Push / FCM 推送通知

| 技術 | 範圍 | 說明 |
|------|------|------|
| **Web Push API** | 瀏覽器 (PWA) | Service Worker 接收，無需打開網頁 |
| **FCM** (Firebase) | Android + Web | Google 推送服務，免費 |
| **APNs** | iOS/macOS | Apple 推送，強制使用 |
| **VAPID** | 自託管 Web Push | 自定義推送伺服器 |

---

## 訊息佇列 Message Queues

| 工具 | 模型 | 吞吐量 | 適用 |
|------|------|--------|------|
| **Kafka** | 分散式 Log（持久化） | 百萬 msg/s | 事件溯源、數據管道 |
| **RabbitMQ** | AMQP Broker（多協議） | 萬級 msg/s | 任務佇列、RPC |
| **NATS** | 輕量 Pub/Sub | 千萬 msg/s | 微服務通訊 |
| **Redis Streams** | 輕量 Log（記憶體內） | 十萬 msg/s | 簡單佇列 / 即時 |

## FFmpeg 核心用法 FFmpeg Essentials

| 任務 | 命令範例 |
|------|---------|
| 轉碼 H.264 | `ffmpeg -i in.mp4 -c:v libx264 -preset fast out.mp4` |
| 擷取片段 | `ffmpeg -ss 00:01:00 -t 30 -i in.mp4 out.mp4` |
| RTMP 推流 | `ffmpeg -re -i in.mp4 -c copy -f flv rtmp://...` |
| HLS 轉換 | `ffmpeg -i in.mp4 -hls_time 6 -hls_list_size 10 out.m3u8` |
| 縮圖生成 | `ffmpeg -i in.mp4 -vf thumbnail -frames:v 1 thumb.jpg` |
