# 01 — 概述與架構 Overview & Architecture

> Web 從靜態文檔到雲原生應用的架構演進，每一代都重新定義了「用戶端－伺服器」邊界。

---

## Web 演進 Web Evolution

| 維度 | Web 1.0 (1990–2004) | Web 2.0 (2004–2014) | Web 3.0 (2014–) |
|------|---------------------|---------------------|-----------------|
| **內容** | 只讀，靜態 HTML | 讀寫，UGC 社交 | 語義，去中心化 |
| **互動** | 表單提交 | AJAX SPA | WASM / AI Agent |
| **部署** | FTP → 伺服器 | LAMP / 託管 | Serverless / Edge |
| **協議** | HTTP/1.0 | HTTP/1.1 + REST | HTTP/3 + gRPC |
| **典型** | GeoCities, Yahoo | Facebook, YouTube | Ethereum, ChatGPT |

---

## 架構範式 Architecture Paradigms

| 範式 | 核心思想 | 適用場景 | 代表 |
|------|---------|---------|------|
| **Client-Server** | 請求→響應，伺服器渲染 | 傳統網站 | LAMP stack |
| **RESTful** | 資源導向，無狀態 | API 服務 | JSON over HTTP |
| **Microservices** | 服務拆分，獨立部署 | 大型分散系統 | K8s + gRPC |
| **Jamstack** | 預渲染 + CDN + API | 靜態內容站 | Next.js + Vercel |
| **Serverless** | 事件驅動，按需擴縮 | 低成本 API | AWS Lambda |

---

## PWA vs SPA 比較 Comparison

| 維度 | PWA (Progressive Web App) | SPA (Single Page App) |
|------|--------------------------|-----------------------|
| **載入** | Service Worker 緩存，離線可用 | 首次載入 JS bundle，需聯網 |
| **體驗** | 接近原生 App（全螢幕、推送） | 無頁面跳轉，路由在前端 |
| **SEO** | 可 SSR/預渲染，SEO 友好 | 需額外 SSR 方案 |
| **安裝** | 可添加到主螢幕 | 僅瀏覽器內 |
| **技術** | Manifest + Service Worker | React/Vue 路由 + API |
