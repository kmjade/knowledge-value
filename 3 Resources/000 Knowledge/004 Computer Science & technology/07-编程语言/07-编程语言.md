# 07-编程语言

> 编程语言是人与计算机之间的接口——不同的语言代表了不同的思维范式、抽象层次和设计权衡。

## 编译 vs 解释

| 方式 | 过程 | 代表 | 特点 |
|------|------|------|------|
| **编译型** | 源码 → 编译器 → 机器码 → 执行 | C, C++, Rust, Go | 执行快、编译慢、提前发现错误 |
| **解释型** | 源码 → 解释器逐行执行 | Python, Ruby, JavaScript (传统) | 快速开发、执行慢 |
| **JIT 编译** | 解释 + 热点编译为机器码 | JavaScript (V8), Java (JVM), C# (.NET) | 兼顾开发速度和执行效率 |

### 编译过程

```
源代码
  ↓ 词法分析 (Lexer)
Token 流
  ↓ 语法分析 (Parser)
抽象语法树 (AST)
  ↓ 语义分析
带类型标注的 AST
  ↓ 中间代码生成 (IR)
中间表示 (LLVM IR / Java Bytecode)
  ↓ 优化
优化后的 IR
  ↓ 代码生成
目标机器码
```

## 类型系统

### 静态类型 vs 动态类型

| | 静态类型 | 动态类型 |
|--|---------|---------|
| 类型检查时机 | 编译时 | 运行时 |
| 需要类型注解 | 通常需要 | 不需要 |
| 错误发现 | 早（编译期） | 晚（运行时） |
| 重构安全性 | 高 | 低（需要测试覆盖） |
| 代表 | Java, C++, Rust, Go, TypeScript | Python, Ruby, JavaScript |

### 强类型 vs 弱类型

| | 强类型 | 弱类型 |
|--|-------|-------|
| 隐式类型转换 | 少/无 | 多 |
| 代表 | Python, Rust, Java | JavaScript, PHP, C |

> Python 是"强动态类型"——运行时类型严格但不需要声明。JavaScript 是"弱动态类型"——`1 + "1" = "11"`。

### 类型多态

| 方式 | 说明 | 示例 |
|------|------|------|
| **参数多态 (泛型)** | 同一代码适用于多种类型 | `List<T>` |
| **子类型多态** | 子类替代父类 | `Animal a = new Dog()` |
| **特设多态 (重载)** | 同名函数，不同参数 | `add(int, int)` / `add(float, float)` |

## 编程范式

| 范式 | 核心思想 | 语言 | 适合场景 |
|------|----------|------|----------|
| **命令式** | 告诉计算机"怎么做" | C, Python, Go | 大多数日常编程 |
| **面向对象 (OOP)** | 数据 + 行为封装为对象 | Java, C++, Python | 大型系统、GUI |
| **函数式 (FP)** | 计算 = 数学函数求值，无副作用 | Haskell, Scala, Clojure | 数据处理、并发 |
| **声明式** | 描述"要什么"而非"怎么做" | SQL, HTML, Terraform | 查询、配置 |
| **逻辑式** | 基于规则和事实推理 | Prolog | 专家系统、定理证明 |

### OOP 四大特性

| 特性 | 含义 |
|------|------|
| **封装** | 隐藏内部细节，只暴露接口 |
| **继承** | 子类复用父类的属性和方法 |
| **多态** | 同一接口，不同实现 |
| **抽象** | 将共性提取为抽象类或接口 |

> 现代趋势：**组合优于继承**。深继承链难以维护——能用组合就用组合。

### 函数式编程核心

| 概念 | 含义 |
|------|------|
| **纯函数** | 同样输入永远同样输出，无副作用 |
| **不可变数据** | 数据一旦创建就不能修改 |
| **高阶函数** | 函数作为参数或返回值 (map, filter, reduce) |
| **柯里化** | 把多参函数转为单参函数链 |
| **模式匹配** | 根据数据结构的不同形态执行不同逻辑 |

## 内存管理

### 手动 vs 自动

| 方式 | 代表 | 特点 |
|------|------|------|
| 手动管理 | C (malloc/free), C++ (new/delete) | 性能极致，容易出错（泄漏、悬垂指针） |
| 引用计数 | Python (CPython), Swift (部分) | 简单，但循环引用需要额外处理 |
| 追踪 GC | Java, Go, C# | 自动但偶尔 STW (Stop The World) |
| 所有权系统 | Rust | 编译期保证内存安全，无 GC 开销 |

### Rust 的所有权

Rust 的内存管理是一大创新——在编译时通过所有权规则保证内存安全，无需 GC：

| 规则 | 说明 |
|------|------|
| 每个值有且只有一个所有者 |
| 值离开作用域时自动释放 |
| 同一时刻只能有一个可变引用 **或** 多个不可变引用 |

## 语言选择指南

| 场景 | 推荐 | 原因 |
|------|------|------|
| 系统编程 | Rust, C, Zig | 无 GC、精细控制 |
| 后端服务 | Go, Java, C# | 并发模型好、生态成熟 |
| 数据/AI | Python | 生态无可替代 |
| 前端 | TypeScript | 类型安全 + 全栈生态 |
| 高性能计算 | C++, Rust | 接近硬件的性能 |
| 快速原型 | Python, JavaScript | 开发效率高 |
| 移动端 | Swift (iOS), Kotlin (Android) | 官方首选 |
