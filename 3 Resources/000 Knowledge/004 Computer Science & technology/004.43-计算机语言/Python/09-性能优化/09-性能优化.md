# Python 性能優化 — Performance Optimization

## 性能分析 Profiling

| 工具 | 用途 | 命令 |
|------|------|------|
| **cProfile** | CPU 時間分析 | `python -m cProfile -s cumtime script.py` |
| **py-spy** | 生產環境採樣（無侵入） | `py-spy top --pid <PID>` |
| **line_profiler** | 逐行耗時 | `@profile` 裝飾器 |
| **memory_profiler** | 記憶體逐行分析 | `@profile` + `mprof run` |
| **tracemalloc** | 記憶體分配追踪（內建） | `tracemalloc.start()` |
| **objgraph** | 物件引用關係圖 | `objgraph.show_backrefs()` |
| **Scalene** | CPU+GPU+記憶體一體 | `scalene script.py` |

## 常見瓶頸與解法

| 瓶頸 | 表現 | 解法 |
|------|------|------|
| **循環內屬性訪問** | `.` 操作頻繁 | 本地變量緩存 `method = obj.method` |
| **字串拼接** | `+` 在循環中 | `''.join(list)` |
| **列表推導濫用** | 多層嵌套 | 生成器表達式、`itertools` |
| **GIL 競爭** | CPU 密集多線程無加速 | `multiprocessing` / C 擴展 |
| **I/O 阻塞** | 網路/文件串行 | `asyncio` / 線程池 |
| **遞迴過深** | 遞迴解 Fibonacci | 迭代 / `@lru_cache` |

## 高效數據結構

```python
# 使用正確的容器
from collections import deque     # O(1) 頭尾操作
from array import array           # 節省內存的數值陣列
import bisect                     # O(log n) 有序插入

# 避免不必要的複製
items = (x*2 for x in data)       # 生成器 vs list comprehension
chunk = memoryview(large_bytes)   # 零拷貝切片
```

## C 擴展加速

| 方案 | 適合 | 語法 | 加速比 |
|------|------|------|:---:|
| **Cython** | 數值計算、循環 | `.pyx` → `.so` | 10–100× |
| **pybind11** | 大型 C++ 庫綁定 | C++ header-only | 100×+ |
| **Numba** | NumPy 數值操作 | `@jit` 裝飾器 | 10–200× |
| **ctypes/cffi** | 調用現有 C 庫 | 純 Python | 視 C 端 |

## PyPy JIT

```bash
pypy3 script.py  # 替代 CPython，長腳本加速 3–7×
```

不適用：C 擴展（NumPy 除外）、`asyncio` 重度使用。

## 算法複雜度速查

| 操作 | Python 實現 | 複雜度 |
|------|------------|:---:|
| list append/pop | `.append()` `.pop()` | O(1) |
| list insert/remove | `.insert(0)` `.remove()` | O(n) |
| dict lookup | `d[key]` | O(1) |
| set membership | `x in s` | O(1) |
| sort | `.sort()` | O(n log n) |
| deque popleft | `d.popleft()` | O(1) |
| bisect | `bisect.insort()` | O(log n) insert + O(n) shift |

## 實戰檢查清單

- [ ] cProfile 定位最慢 3 個函數
- [ ] 檢查循環內是否有不必要的屬性訪問
- [ ] I/O 操作使用 `asyncio` 或線程池
- [ ] 大數據使用 `numpy` / `array` 替代 list
- [ ] 考慮 Cython 優化熱點循環
- [ ] 記憶體使用 `tracemalloc` 檢查洩漏
