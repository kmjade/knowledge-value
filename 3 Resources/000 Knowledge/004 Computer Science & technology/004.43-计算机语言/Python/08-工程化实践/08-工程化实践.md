## 🐍 虛擬環境與依賴管理

### venv (內建)

```bash
# 建立虛擬環境
python -m venv .venv

# 啟動
source .venv/bin/activate   # Linux/macOS
.venv\Scripts\activate      # Windows

# 退出
deactivate

# 匯出依賴
pip freeze > requirements.txt

# 安裝依賴
pip install -r requirements.txt
```

### poetry — 現代依賴管理

```bash
# 安裝: pip install poetry

# 初始化專案
poetry new myproject
# cd existing_project && poetry init

# 新增依賴
poetry add requests
poetry add --group dev pytest black ruff

# 安裝所有依賴
poetry install

# 啟動 shell
poetry shell

# 鎖定版本
poetry lock --no-update

# 建置與發布
poetry build
poetry publish
```

### uv — 極速 Python 套件管理器 (Rust)

```bash
# 安裝: pip install uv  # 或 curl -LsSf https://astral.sh/uv/install.sh | sh

# 建立虛擬環境 (比 venv 快 10-100x)
uv venv

# 安裝套件 (比 pip 快 10-100x)
uv pip install ruff pytest
uv pip install -r requirements.txt

# 建立 lock 檔案
uv pip compile pyproject.toml -o requirements.txt

# 工具執行 (無需安裝)
uvx ruff check .
uvx black --check .
```

### 依賴管理工具對比

| | pip + venv | poetry | uv | pipenv |
|:---|:---|:---|:---|:---|
| 速度 | 慢 | 中 | 🚀 極快 | 慢 |
| Lock 檔案 | ❌ (需 pip-tools) | ✅ poetry.lock | ✅ (uv pip compile) | ✅ Pipfile.lock |
| 依賴解析 | 基礎 | SAT solver | SAT solver | 基礎 |
| 建置發布 | ❌ | ✅ | ✅ (開發中) | ❌ |
| 學習曲線 | 低 | 中 | 低 | 中 |
| 社群採用 | 傳統 | ⭐ 主流 | 🔥 新星 | 式微 |
| 推薦度 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐ |

## 🎨 程式碼格式化與 Linting

### Ruff — 全能型 Linter + Formatter (Rust)

```bash
# 安裝: pip install ruff  # 或 uv pip install ruff

ruff check .              # Lint 檢查
ruff check --fix .        # 自動修復
ruff format .             # 格式化
ruff format --check .     # 檢查格式 (CI 用)

# VS Code 整合: 安裝 ruff extension
# 存檔時自動格式化:
# "editor.formatOnSave": true
# "[python]": { "editor.defaultFormatter": "charliermarsh.ruff" }
```

### Black — 固執的格式化器

```bash
# pip install black
black .                   # 格式化
black --check --diff .    # 僅檢查差異 (CI 用)
black --line-length 100 . # 自訂行長
```

### isort — Import 排序

```bash
# pip install isort
isort .                   # 排序 imports
isort --check-only .      # 僅檢查 (CI 用)
```

### pre-commit — Git Hook 自動化

```yaml
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.5.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v1.10.0
    hooks:
      - id: mypy
        additional_dependencies: [types-requests]

  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.6.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer
      - id: check-yaml
      - id: check-added-large-files
```

```bash
# 安裝 pre-commit hooks
pre-commit install            # 每次 commit 自動執行
pre-commit run --all-files    # 手動執行所有檔案
```

| 工具 | 功能 | 語言 | 速度 | 推薦 |

## 🔒 Typing 進階 (Protocol / Generic / Literal)

```python
from typing import (
    Protocol, Generic, TypeVar, Literal, TypedDict,
    overload, final, Self, TypeGuard
)

# Protocol — 結構化型別 (structural subtyping)
class SupportsClose(Protocol):
    def close(self) -> None: ...

def cleanup(resource: SupportsClose) -> None:
    resource.close()

# 任何有 close() 方法的物件都滿足 SupportsClose

# overload — 函數重載
@overload
def parse(value: str) -> str: ...
@overload
def parse(value: bytes) -> bytes: ...
@overload
def parse(value: None) -> None: ...

def parse(value: str | bytes | None) -> str | bytes | None:
    if value is None:
        return None
    if isinstance(value, bytes):
        return value.decode()
    return value

# Generic — 泛型
T = TypeVar('T')
K = TypeVar('K')
V = TypeVar('V')

class Repository(Generic[T]):
    def __init__(self) -> None:
        self._items: dict[int, T] = {}

    def get(self, id: int) -> T | None:
        return self._items.get(id)

    def save(self, id: int, item: T) -> None:
        self._items[id] = item

user_repo: Repository[dict] = Repository()
user_repo.save(1, {"name": "Alice"})

# Self — 返回自身型別 (Python 3.11+)
from typing import Self

class Builder:
    def set_name(self, name: str) -> Self:
        self.name = name
        return self

    def set_age(self, age: int) -> Self:
        self.age = age
        return self

# TypedDict + Literal — 精確字典型別
class APIConfig(TypedDict, total=False):
    host: str
    port: int
    protocol: Literal['http', 'https']
```

## 🐳 Docker — Python 容器化

### 多階段構建 Dockerfile

```dockerfile
# ── Build Stage ──
FROM python:3.12-slim AS builder

WORKDIR /app

# 安裝依賴 (利用 Docker cache)
COPY pyproject.toml ./
RUN pip install --no-cache-dir .

# 複製原始碼
COPY src/ ./src/

# ── Runtime Stage ──
FROM python:3.12-slim AS runtime

WORKDIR /app

# 建立非 root 使用者
RUN groupadd -r appuser && useradd -r -g appuser appuser

# 複製已安裝的套件
COPY --from=builder /usr/local/lib/python3.12/site-packages /usr/local/lib/python3.12/site-packages
COPY --from=builder /app/src ./src

# 切換使用者
USER appuser

EXPOSE 8000

# 健康檢查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
    CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"

CMD ["python", "-m", "src.main"]
```

### Docker Compose

```yaml
# docker-compose.yml
version: '3.8'

services:
  app:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/mydb
    depends_on:
      db:
        condition: service_healthy
    volumes:
      - ./src:/app/src  # 開發時即時重載

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: mydb
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U user -d mydb"]
      interval: 5s
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:
```

## 🔄 CI/CD (GitHub Actions)

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v2
        with:
          python-version: "3.12"
      - run: uv pip install ruff mypy
      - run: ruff check .
      - run: ruff format --check .
      - run: mypy src/

  test:
    needs: lint
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.11", "3.12"]
    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: test
          POSTGRES_PASSWORD: test
          POSTGRES_DB: testdb
        ports:
          - 5432:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v2
        with:
          python-version: ${{ matrix.python-version }}
      - run: uv pip install -e ".[dev]"
      - run: pytest --cov=src --cov-report=xml
      - uses: codecov/codecov-action@v4
        with:
          file: ./coverage.xml

  build:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build Docker image
        run: docker build -t myapp:${{ github.sha }} .
```

## 📊 專案初始化速查

```bash
# 一分鐘初始化新專案
mkdir myproject && cd myproject

# 1. 虛擬環境
uv venv
source .venv/bin/activate

# 2. 初始化 pyproject.toml (手寫或 poetry init)
# 3. 安裝核心依賴
uv pip install ruff pytest pytest-cov mypy pre-commit

# 4. 建立目錄結構
mkdir -p src/myproject tests docs .github/workflows

# 5. 設定 pre-commit
cat > .pre-commit-config.yaml << 'EOF'
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.5.0
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format
EOF
pre-commit install

# 6. Git init
git init
echo ".venv/\n__pycache__/\n*.pyc\n.env" > .gitignore
git add -A && git commit -m "Initial commit"
```

## 🎯 重點回顧

1. **虛擬環境**：`python -m venv .venv` 或 `uv venv`
2. **依賴管理**：`pyproject.toml` (PEP 621) + `poetry` 或 `uv`
3. **格式化**：`ruff format` 取代 black + isort (一體化，Rust 極速)
4. **Linting**：`ruff check` + `mypy --strict`
5. **pre-commit**：自動化品質檢查，每次 commit 前執行
6. **Type Hints**：Protocol / Generic / Literal / TypedDict 進階型別
7. **Docker**：多階段構建縮小 image、非 root 使用者、healthcheck
8. **CI/CD**：GitHub Actions — lint → test (matrix) → build → deploy

---

> 💡 **最佳實踐**：一個 `ruff` 取代 flake8/black/isort/pylint；`mypy --strict` 從專案第一天就啟用；Docker multi-stage build 把 image 壓到最小。
