# Python 併發與非同步 — Concurrency & Asyncio

> 🎯 **學習目標**：理解 GIL 限制、掌握 threading / multiprocessing / asyncio 三種併發模型及其選擇策略。

---

## 📑 三種併發模型對比

| 維度 | threading | multiprocessing | asyncio |
|:---|:---|:---|:---|
| 並行模型 | 多執行緒 (concurrent) | 多進程 (parallel) | 協程 (concurrent) |
| GIL 影響 | ⚠️ 受限 (CPU bound 無加速) | ✅ 不受限 | ✅ 不受限 |
| 適用場景 | I/O bound | CPU bound | I/O bound（高併發） |
| 記憶體 | 共享 | 獨立 (IPC 通訊) | 共享（單執行緒） |
| 通訊方式 | Queue / Lock / shared var | Pipe / Queue / Manager | 不需鎖（單執行緒） |
| 開銷 | 低 | 高 (fork/spawn) | 極低 |
| 推薦度 | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

## 🔒 GIL 深入理解

### GIL (Global Interpreter Lock)

```
┌──────────────────────────────────────────────┐
│              CPython Interpreter               │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐       │
│  │ Thread 1│  │ Thread 2│  │ Thread 3│       │
│  │  acquires│  │ waiting │  │ waiting │       │
│  │   GIL ✓ │  │   ...   │  │   ...   │       │
│  └────┬────┘  └─────────┘  └─────────┘       │
│       │                                        │
│  ┌────▼────────────────────────────────────┐  │
│  │   GIL (Global Interpreter Lock)         │  │
│  │   Only ONE thread executes Python       │  │
│  │   bytecode at a time                    │  │
│  └─────────────────────────────────────────┘  │
└──────────────────────────────────────────────┘
```

| GIL 關鍵事實 | 說明 |
|:---|:---|
| **僅 CPython 有** | Jython / IronPython 無 GIL；PyPy 有 STM |
| **每 5ms 切換** | `sys.getswitchinterval()` (Python 3.2+，預設 5ms) |
| **I/O 自動釋放** | I/O 操作 (read/write/network) 會釋放 GIL |
| **C extension 可釋放** | 在 C 程式碼中手動釋放 GIL |
| **Python 3.13 free-threading** | 實驗性 `--disable-gil` 模式 |

### 繞過 GIL 的策略

```python
# 策略 1: multiprocessing — CPU bound
from multiprocessing import Pool

def cpu_heavy(n):
    return sum(i * i for i in range(n))

with Pool() as pool:  # 每個進程有獨立 GIL
    results = pool.map(cpu_heavy, [10_000_000] * 4)

# 策略 2: C extension 釋放 GIL (概念性)
# 使用 numpy / Cython / pybind11 等，在 C 程式碼中
# Py_BEGIN_ALLOW_THREADS / Py_END_ALLOW_THREADS 區塊

# 策略 3: 使用 asyncio (單執行緒)
# 策略 4: subprocess 啟動獨立 Python 進程
```

## 🧵 threading — 多執行緒

### 基礎用法

```python
import threading
import time

def worker(name: str, delay: float):
    """Worker thread"""
    print(f"[{name}] Started")
    time.sleep(delay)   # I/O 會釋放 GIL
    print(f"[{name}] Finished")

threads = []
for i in range(3):
    t = threading.Thread(target=worker, args=(f"T-{i}", 0.5))
    t.start()
    threads.append(t)

# 等待所有執行緒完成
for t in threads:
    t.join()

print("All threads done")
```

### Lock / RLock

```python
import threading

class ThreadSafeCounter:
    """執行緒安全計數器"""

    def __init__(self):
        self._value = 0
        self._lock = threading.Lock()

    def increment(self):
        with self._lock:         # acquire + release
            self._value += 1

    def get(self) -> int:
        with self._lock:
            return self._value

# Lock vs RLock
lock = threading.Lock()
# lock.acquire()
# lock.acquire()  # Deadlock! Lock 不可重入

rlock = threading.RLock()  # Re-entrant Lock
rlock.acquire()
rlock.acquire()  # OK，同執行緒可重入
rlock.release()
rlock.release()
```

### Queue — 執行緒安全佇列

```python
import threading
import queue
import time

def producer(q: queue.Queue, items):
    for item in items:
        time.sleep(0.1)
        q.put(item)
        print(f"Produced: {item}")
    q.put(None)  # Sentinel（哨兵）信號結束

def consumer(q: queue.Queue, name: str):
    while True:
        item = q.get()
        if item is None:      # 哨兵信號
            q.task_done()
            break
        print(f"[{name}] Consumed: {item}")
        time.sleep(0.2)
        q.task_done()

q = queue.Queue(maxsize=5)

prod = threading.Thread(target=producer, args=(q, range(10)))
cons = threading.Thread(target=consumer, args=(q, "C1"))

prod.start()
cons.start()
prod.join()
cons.join()
```

### ThreadPoolExecutor

```python
from concurrent.futures import ThreadPoolExecutor, as_completed
import requests  # 假設已安裝

def fetch_url(url: str) -> tuple[str, int]:
    try:
        resp = requests.get(url, timeout=5)
        return url, resp.status_code
    except Exception as e:
        return url, str(e)

urls = [
    "https://httpbin.org/delay/1",
    "https://httpbin.org/delay/2",
    "https://httpbin.org/status/200",
]

with ThreadPoolExecutor(max_workers=4) as executor:
    # map — 保持順序
    results = executor.map(fetch_url, urls)
    for r in results:
        print(r)

    # submit + as_completed — 按完成順序
    futures = {executor.submit(fetch_url, url): url for url in urls}
    for future in as_completed(futures):
        url = futures[future]
        print(f"{url} → {future.result()}")
```

## 🔀 multiprocessing — 多進程

### Pool 與 map

```python
from multiprocessing import Pool, cpu_count
import math

def is_prime(n: int) -> bool:
    """CPU-bound 質數判斷"""
    if n < 2:
        return False
    for i in range(2, int(math.sqrt(n)) + 1):
        if n % i == 0:
            return False
    return True

numbers = [10_000_000 + i for i in range(100)]

# 使用所有 CPU 核心
with Pool(processes=cpu_count()) as pool:
    results = pool.map(is_prime, numbers)
    primes = [n for n, p in zip(numbers, results) if p]
    print(f"Found {len(primes)} primes")

# imap — 惰性 (回傳迭代器)
# pool.imap(is_prime, numbers)

# starmap — 多參數
# pool.starmap(func, [(arg1, arg2), ...])
```

### Pipe, Queue, Manager

```python
from multiprocessing import Process, Pipe, Queue, Manager

# Pipe — 雙向通訊 (僅兩個端點)
def pipe_worker(conn):
    conn.send("Hello from child")
    msg = conn.recv()
    print(f"Child received: {msg}")
    conn.close()

parent_conn, child_conn = Pipe()
p = Process(target=pipe_worker, args=(child_conn,))
p.start()
print(parent_conn.recv())  # Hello from child
parent_conn.send("Hi from parent")
p.join()

# Manager — 跨進程共享物件
def manager_worker(shared_list, shared_dict):
    shared_list.append("child_added")
    shared_dict["child_key"] = "child_value"

with Manager() as manager:
    shared_list = manager.list(["initial"])
    shared_dict = manager.dict({"parent_key": "parent_value"})

    p = Process(target=manager_worker, args=(shared_list, shared_dict))
    p.start()
    p.join()

    print(shared_list)  # ['initial', 'child_added']
    print(shared_dict)  # {'parent_key': 'parent_value', 'child_key': 'child_value'}
```

### ProcessPoolExecutor

```python
from concurrent.futures import ProcessPoolExecutor

with ProcessPoolExecutor(max_workers=4) as executor:

```

## 🌊 asyncio — 非同步協程

### 核心概念

```python
import asyncio

# 協程函數 (coroutine function) — async def
async def fetch_data(delay: float, name: str):
    """模擬 async I/O 操作"""
    print(f"[{name}] Start fetching...")
    await asyncio.sleep(delay)  # 模擬 async I/O (不會阻塞事件循環)
    print(f"[{name}] Done!")
    return f"result-{name}"

# Task — 將協程包裝為可並發執行的任務
async def main():
    # gather — 並發執行多個協程
    results = await asyncio.gather(
        fetch_data(1, "A"),
        fetch_data(0.5, "B"),
        fetch_data(0.3, "C"),
    )
    print(f"All results: {results}")

    # create_task — 顯式創建任務
    task = asyncio.create_task(fetch_data(0.2, "D"))
    result = await task

asyncio.run(main())
# Output order: B → C → D → A (依 delay 排序)
```

### 同步原語 (asyncio.Lock / Semaphore / Event)

```python
import asyncio

class AsyncLimiter:
    """基於 Semaphore 的並發限制器"""

    def __init__(self, concurrency: int = 5):
        self.semaphore = asyncio.Semaphore(concurrency)

    async def fetch(self, url: str):
        async with self.semaphore:
            # 同時最多 concurrency 個協程進入
            await asyncio.sleep(0.1)  # 模擬 I/O
            return f"fetched {url}"

async def worker_with_lock(name: str, lock: asyncio.Lock):
    """展示 asyncio.Lock"""
    async with lock:
        print(f"[{name}] Critical section start")
        await asyncio.sleep(0.5)
        print(f"[{name}] Critical section end")
```

### asyncio vs threading 選擇指南

```python
# ── I/O bound (網路請求、檔案讀寫) ──
# ✅ 首選 asyncio (單執行緒，協程調度，無鎖)
# ⚠️ 次選 threading (需注意執行緒安全)

# ── CPU bound (計算密集) ──
# ✅ 首選 multiprocessing (繞過 GIL)
# ⚠️ 次選 ProcessPoolExecutor

# ── 混合場景 ──
# 使用 run_in_executor 將 CPU 任務卸載到執行緒/進程池
async def mixed_workload():
    loop = asyncio.get_running_loop()
    # 在執行緒池中執行 CPU bound 任務
    result = await loop.run_in_executor(None, cpu_heavy, 1_000_000)
    # 繼續 asyncio I/O 任務
    await asyncio.sleep(0.1)
    return result
```

### asyncio 生態模組

| 模組 | 用途 |
|:---|:---|
| `asyncio.Queue` | 非同步佇列 (生產者-消費者) |
| `asyncio.Semaphore` | 並發限制 |
| `asyncio.Lock` | 非同步鎖 |
| `asyncio.Event` | 跨協程事件通知 |
| `asyncio.Condition` | 條件變數 |
| `asyncio.subprocess` | 非同步子進程 |
| `asyncio.to_thread()` | (3.9+) 輕鬆卸載到執行緒 |

## 📊 三種模型決策樹

```
需要併發？
├── CPU bound？
│   ├── 是 → multiprocessing / ProcessPoolExecutor
│   │         └── 需要大量共享狀態？ → Manager / shared_memory
│   └── 否 → I/O bound？
│            ├── 大量並行網路連線 (10k+) → asyncio ✅
│            ├── 簡單多執行緒 I/O → threading / ThreadPoolExecutor
│            └── 混合 (I/O + CPU) → asyncio + run_in_executor
├── 需要每個任務獨立隔離？ → multiprocessing
└── 無需隔離，只需非同步 → asyncio (推薦)
```

## 🎯 重點回顧

1. **GIL** 讓 CPython 單一時刻只有一個執行緒執行 Python bytecode
2. **CPU bound** 用 `multiprocessing` 繞過 GIL
3. **I/O bound** 用 `asyncio`（協程，單執行緒，無鎖）
4. **threading** 適合遺留程式碼或 I/O bound 簡單場景
5. **`concurrent.futures`** 提供統一的 ThreadPoolExecutor / ProcessPoolExecutor 介面
6. **`asyncio.gather`** 並發執行多個協程；`create_task` 顯式創建任務
7. **`run_in_executor`** 將 CPU 任務卸載到執行緒池而不阻塞事件循環

---

> 💡 **最佳實踐**：新專案 I/O 密集優先 `asyncio`；舊程式碼相容用 `ThreadPoolExecutor`；CPU 密集別無選擇用 `multiprocessing`；避免線上程間共享可變狀態。
