# Python 測試與除錯 — Testing & Debugging

> 🎯 **學習目標**：掌握 unittest / pytest 測試框架、mock 與 parametrize、coverage 覆蓋率、pdb 除錯、logging 最佳實踐、mypy 靜態型別檢查。

---

## 📑 測試金字塔

```
          ┌──────────┐
          │  E2E     │  少量：端到端 (Playwright / Selenium)
         ┌┴──────────┴┐
         │ Integration│  中量：整合測試 (API / DB)
        ┌┴────────────┴┐
        │   Unit Tests  │  大量：單元測試 (pytest / unittest)
        └───────────────┘
```

| 層級 | 範圍 | 速度 | 維護成本 | 工具 |
|:---|:---|:---|:---|:---|
| Unit | 函數 / 方法 | 毫秒 | 低 | pytest, unittest |
| Integration | 模組間互動 | 秒 | 中 | pytest + fixtures |
| E2E | 整個系統 | 分鐘 | 高 | Playwright, Selenium |

## 🧪 pytest — 現代測試框架

### 基礎測試

```python
# test_math.py
import pytest

def add(a, b):
    return a + b

# 測試函數 (以 test_ 開頭)
def test_add_positive():
    assert add(2, 3) == 5

def test_add_negative():
    assert add(-1, -1) == -2

def test_add_zero():
    assert add(0, 5) == 5
    assert add(0, 0) == 0

# 執行: pytest test_math.py -v
```

### Fixture — 測試前置/清理

```python
import pytest
import tempfile
import os

@pytest.fixture
def temp_file():
    """Fixture: 建立暫存檔，測試結束後自動清理"""
    fd, path = tempfile.mkstemp(suffix='.txt')
    os.close(fd)
    yield path           # 提供給測試使用
    os.unlink(path)      # teardown: 清理

@pytest.fixture(scope='module')
def database_connection():
    """scope='module': 整個模組共用一次"""
    conn = create_test_db()
    yield conn
    conn.close()

def test_write_read(temp_file):
    with open(temp_file, 'w') as f:
        f.write('hello')
    with open(temp_file) as f:
        assert f.read() == 'hello'

# Fixture scope: function (預設), class, module, package, session
```

### Parametrize — 資料驅動測試

```python
import pytest

def is_palindrome(s: str) -> bool:
    s = ''.join(c.lower() for c in s if c.isalnum())
    return s == s[::-1]

@pytest.mark.parametrize('text,expected', [
    ('racecar', True),
    ('hello',   False),
    ('A man, a plan, a canal: Panama', True),
    ('',        True),
    ('  ',      True),
])
def test_palindrome(text, expected):
    assert is_palindrome(text) == expected

# 堆疊 parametrize — 笛卡爾積
@pytest.mark.parametrize('x', [1, 2, 3])
@pytest.mark.parametrize('y', [10, 20])
def test_combinations(x, y):
    assert (x + y) > 0  # 共 6 次測試
```

### Mock / Monkeypatch

```python
import pytest
from unittest.mock import Mock, patch, MagicMock

# monkeypatch — 簡單替換 (pytest 內建)
def test_config_override(monkeypatch):
    monkeypatch.setenv('DB_HOST', 'test-db')
    monkeypatch.setattr('os.path.exists', lambda p: True)
    assert os.environ['DB_HOST'] == 'test-db'

# unittest.mock — 進階 mock
class ExternalAPI:
    def fetch(self, url: str) -> dict:
        # 實際會呼叫外部 API
        pass

def process_data(api: ExternalAPI) -> str:
    data = api.fetch('/users')
    return f"Found {len(data)} users"

def test_process_data():
    mock_api = Mock(spec=ExternalAPI)
    mock_api.fetch.return_value = [{'id': 1}, {'id': 2}]

    result = process_data(mock_api)
    assert result == 'Found 2 users'
    mock_api.fetch.assert_called_once_with('/users')

# @patch 裝飾器
@patch('mymodule.ExternalAPI.fetch')
def test_with_patch(mock_fetch):
    mock_fetch.return_value = [{'id': 1}]
    # 測試代碼...
```

### 例外測試

```python
import pytest

def divide(a, b):
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b

def test_divide_by_zero():
    with pytest.raises(ValueError, match="divide by zero"):
        divide(10, 0)

# 檢查例外訊息
def test_divide_by_zero_info():
    with pytest.raises(ValueError) as exc_info:
        divide(10, 0)
    assert "zero" in str(exc_info.value)
```

## 🔬 unittest — 標準庫測試框架

```python
import unittest

class TestStringMethods(unittest.TestCase):

    def setUp(self):
        """每個 test 前執行"""
        self.text = "hello world"

    def tearDown(self):
        """每個 test 後執行"""
        pass

    @classmethod
    def setUpClass(cls):
        """整個 class 執行一次"""
        pass

    def test_upper(self):
        self.assertEqual(self.text.upper(), "HELLO WORLD")

    def test_split(self):
        self.assertEqual(self.text.split(), ["hello", "world"])
        with self.assertRaises(TypeError):
            "hello".split(2)

    def test_in(self):
        self.assertIn("hello", self.text)
        self.assertNotIn("python", self.text)

if __name__ == '__main__':
    unittest.main()
```

### pytest vs unittest

| | pytest | unittest |
|:---|:---|:---|
| 語法 | `assert` 原生語法 | `self.assertEqual` 等 |
| 參數化 | `@pytest.mark.parametrize` | `subTest()` |
| Fixture | `@pytest.fixture` | `setUp/tearDown` |
| Mock | `monkeypatch` + `unittest.mock` | `unittest.mock` |
| Plugin 生態 | 豐富 (coverage, xdist, asyncio) | 有限 |
| 輸出 | 彩色、詳細 | 簡潔 |
| 推薦 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

## 🐛 pdb / breakpoint() — 除錯器

```python
# Python 3.7+ 內建 breakpoint()
def buggy_function(data: list):
    result = []
    for i, item in enumerate(data):
        breakpoint()  # 程式在此暫停，進入 pdb
        processed = item * 2
        result.append(processed)
    return result

# pdb 常用命令:
# n (next)      — 執行下一行
# s (step)      — 進入函數內部
# c (continue)  — 繼續執行
# p <var>       — 印出變數值
# pp <var>      — 美化印出
# l (list)      — 顯示當前程式碼
# w (where)     — 顯示 call stack
# u / d         — 向上/下移動 stack frame
# b <line>      — 設置斷點
# q (quit)      — 退出
# h (help)      — 幫助

# 環境變數控制 breakpoint()
# PYTHONBREAKPOINT=0          → 停用所有 breakpoint()
# PYTHONBREAKPOINT=ipdb.set_trace → 使用 ipdb (需安裝)
```

### post-mortem 除錯

```python
import pdb
import traceback
import sys

def main():
    try:
        # 主要程式碼
        result = 1 / 0
    except Exception:
        exc_type, exc_value, exc_tb = sys.exc_info()
        traceback.print_exc()
        pdb.post_mortem(exc_tb)  # 進入事後除錯模式

# 或從命令列:
# python -m pdb script.py
```

## 📝 Logging 最佳實踐

```python
import logging
import json
import sys

# 結構化日誌 (JSON)
class JsonFormatter(logging.Formatter):
    def format(self, record):
        log_entry = {
            'timestamp': self.formatTime(record),
            'level': record.levelname,
            'logger': record.name,
            'message': record.getMessage(),
            'module': record.module,
            'function': record.funcName,
            'line': record.lineno,
        }
        if record.exc_info:
            log_entry['exception'] = self.formatException(record.exc_info)
        return json.dumps(log_entry, ensure_ascii=False)

# 配置
def setup_logging(level=logging.INFO):
    logger = logging.getLogger()  # root
    logger.setLevel(level)

    # Console handler (structured JSON)
    handler = logging.StreamHandler(sys.stdout)
    handler.setFormatter(JsonFormatter())
    logger.addHandler(handler)

    return logger

# 使用
logger = setup_logging()
logger.info("Server started", extra={"port": 8080})

# 最佳實踐
# 1. 使用 __name__ 作為 logger 名稱
#    logger = logging.getLogger(__name__)
# 2. 敏感資訊 (密碼/token) 不應記錄在日誌
# 3. 使用 logging 而非 print() — 可控層級、輸出目標
# 4. 生產環境用 JSON 結構化日誌，方便 ELK/Splunk 蒐集
```

### typing 進階型別

```python
from typing import Protocol, Generic, TypeVar, Literal, TypedDict

# Protocol — 結構化子型別 (structural subtyping)
class Drawable(Protocol):
    def draw(self) -> None: ...

def render(obj: Drawable):
    obj.draw()

# Literal — 枚舉型別 (Python 3.8+)
def set_mode(mode: Literal['r', 'w', 'a']) -> None:
    ...

set_mode('r')   # OK
# set_mode('x') # mypy error

# TypedDict — 字典結構
class UserDict(TypedDict):
    name: str
    age: int
    email: str

def create_user(data: UserDict) -> str:
    return data['name']

# TypeVar / Generic
T = TypeVar('T')

class Stack(Generic[T]):
    def __init__(self) -> None:
        self._items: List[T] = []

    def push(self, item: T) -> None:
        self._items.append(item)

    def pop(self) -> T:
        return self._items.pop()

int_stack: Stack[int] = Stack()
int_stack.push(1)
# int_stack.push("str")  # mypy error
```

## 📊 除錯流程速查

```
遇到 Bug →
├── 1. 重現 (Reproduce)     — 最小可重現案例
├── 2. 定位 (Locate)        — breakpoint() / pdb / print
├── 3. 理解 (Understand)    — 查看變數、call stack
├── 4. 修復 (Fix)           — 撰寫修正程式碼
├── 5. 測試 (Test)          — 撰寫回歸測試 (regression test)
└── 6. 反思 (Reflect)       — 為什麼會發生？如何預防？
```

## 🎯 重點回顧

1. **pytest** 是現代 Python 測試首選，`assert` 原生語法 + fixture + parametrize
2. **Fixture** 有三種 scope：function (預設)、class、module、session
3. **Mock** 用 `unittest.mock` 隔離外部依賴；`monkeypatch` 適合輕量替換
4. **`pytest.raises`** 測試例外情境
5. **`breakpoint()`** (3.7+) 進入 pdb 除錯器，`PYTHONBREAKPOINT=0` 關閉
6. **coverage** 用 `pytest-cov` 產生覆蓋率報告，目標 80%+
7. **mypy** 靜態型別檢查，`strict=true` 最嚴格模式
8. **logging** 結構化 JSON 格式，避免 `print()` 除錯

---

> 💡 **最佳實踐**：先寫測試再寫程式碼 (TDD)；每個 bug 修復前先寫回歸測試；CI 中必須執行 `pytest` + `mypy` + `coverage`。
