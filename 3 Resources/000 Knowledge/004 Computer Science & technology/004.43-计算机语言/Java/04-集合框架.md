# 04 — 集合框架 Collections Framework

> Java Collection Framework (JCF) — 統一的資料結構體系，包含介面、實作、演算法。

---

## 4.1 集合框架總覽

```
                    Iterable
                       │
                  Collection                        Map
                 /     │      \                      │
              List    Set    Queue            SortedMap
              │       │        │                    │
         ArrayList  HashSet  LinkedList        TreeMap
         LinkedList TreeSet PriorityQueue     HashMap
         Vector     LinkedHashSet  Deque     LinkedHashMap
                                         ConcurrentHashMap
```

### 核心介面

| 介面 | 特性 | 順序 | 重複 | null | 實作 |
|------|------|:---:|:---:|:---:|------|
| **List** | 有序集合 | ✅ 插入序 | ✅ | ✅ | ArrayList, LinkedList, Vector |
| **Set** | 不重複集合 | ❌ (Hash), ✅ (Tree/Linked) | ❌ | ✅ (多數) | HashSet, TreeSet, LinkedHashSet |
| **Queue** | FIFO 隊列 | ✅ | ✅ | ❌ (多數) | LinkedList, PriorityQueue, ArrayDeque |
| **Deque** | 雙端隊列 | ✅ | ✅ | ❌ | ArrayDeque, LinkedList |
| **Map** | 鍵值對 | ❌ (Hash), ✅ (Tree/Linked) | K ❌, V ✅ | ✅ | HashMap, TreeMap, LinkedHashMap |

## 4.2 List 家族

### ArrayList vs LinkedList vs Vector

| 特性 | ArrayList | LinkedList | Vector |
|------|:--------:|:----------:|:------:|
| 底層結構 | 動態陣列 | 雙向鏈結串列 | 動態陣列 |
| 隨機訪問 | O(1) ✅ | O(n) ❌ | O(1) |
| 頭部插入/刪除 | O(n) ❌ | O(1) ✅ | O(n) |
| 記憶體 | 連續，浪費少 | 節點開銷大 (prev/next refs) | 同 ArrayList |
| 執行緒安全 | ❌ | ❌ | ✅ (synchronized) |
| 擴容 | 1.5x | 無 | 2x |
| 預設容量 | 10 | 無 | 10 |
| 適用場景 | 讀多寫少 | 頻繁頭部操作 | 遺留代碼 |

```java
// ArrayList 常用操作
List<String> list = new ArrayList<>();
list.add("A");                            // 尾部添加, O(1) amortized
list.add(0, "B");                         // 指定位置插入, O(n)
list.get(3);                              // 隨機訪問, O(1)
list.set(1, "C");                         // 修改, O(1)
list.remove(2);                           // 依索引刪除, O(n)
list.remove("A");                         // 依值刪除, O(n)
list.contains("B");                       // 查找, O(n)
list.indexOf("C");                        // 找索引, O(n)

// 迭代
// 1. for-each
for (String s : list) { /* ... */ }

// 2. Iterator (可安全刪除)
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().isEmpty()) it.remove();
}

// 3. ListIterator (雙向)
ListIterator<String> lit = list.listIterator(list.size());
while (lit.hasPrevious()) {
    System.out.println(lit.previous());
}

// 4. forEach (Java 8+)
list.forEach(System.out::println);
```

### Stack (Deque 替代)

```java
// ❌ 舊 Stack 類別 (extends Vector，效能差)
Stack<String> stack = new Stack<>();

// ✅ Deque 作為 Stack
Deque<String> stack2 = new ArrayDeque<>();
stack2.push("A");           // 壓棧, addFirst
stack2.push("B");
String top = stack2.pop();  // 出棧, removeFirst  → "B"
String peek = stack2.peek();// 查看棧頂, getFirst → "A"
```

## 4.3 Set 家族

### HashSet vs TreeSet vs LinkedHashSet

| 特性 | HashSet | LinkedHashSet | TreeSet |
|------|:-------:|:------------:|:-------:|
| 底層 | HashMap | LinkedHashMap | TreeMap (紅黑樹) |
| 順序 | 無序 | 插入序 | 自然排序/Comparator |
| 複雜度 | O(1) | O(1) | O(log n) |
| null | ✅ | ✅ | ❌ (需排序) |
| 適用場景 | 一般去重 | 去重+保留插入序 | 排序去重 |

```java
Set<String> hashSet = new HashSet<>();
Set<String> linkedSet = new LinkedHashSet<>();
Set<String> treeSet = new TreeSet<>();

// TreeSet 自訂排序
Set<User> users = new TreeSet<>(Comparator.comparing(User::getAge));

// 不可變 Set (Java 9+)
Set<String> immutable = Set.of("A", "B", "C");

// 集合運算
Set<String> set1 = new HashSet<>(Set.of("A", "B", "C"));
Set<String> set2 = new HashSet<>(Set.of("B", "C", "D"));

Set<String> union = new HashSet<>(set1);
union.addAll(set2);                          // {"A","B","C","D"}

Set<String> intersection = new HashSet<>(set1);
intersection.retainAll(set2);                // {"B","C"}

Set<String> difference = new HashSet<>(set1);
difference.removeAll(set2);                  // {"A"}
```

## 4.4 Queue / Deque 家族

| 實作 | 底層 | 特性 |
|------|------|------|
| **LinkedList** | 雙向鏈結串列 | 同時實作 List 和 Deque |
| **ArrayDeque** | 循環陣列 | 比 LinkedList 快，推薦用作 Stack/Queue |
| **PriorityQueue** | 二叉堆 (Binary Heap) | 自然排序或 Comparator |

```java
// Queue (FIFO)
Queue<String> queue = new ArrayDeque<>();
queue.offer("A");              // 入隊 (失敗返回 false)
queue.add("B");                // 入隊 (失敗拋異常)
String head = queue.poll();    // 出隊 (空則 null)
String peek = queue.peek();    // 查看頭部 (空則 null)

// Deque (雙端)
Deque<Integer> deque = new ArrayDeque<>();
deque.addFirst(1);
deque.addLast(2);
deque.removeFirst();           // 1
deque.removeLast();            // 2

// PriorityQueue (最小堆)
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.offer(3); pq.offer(1); pq.offer(2);
pq.poll();  // 1 (最小)
pq.poll();  // 2
```

## 4.5 Map 家族

### HashMap vs TreeMap vs LinkedHashMap

| 特性 | HashMap | LinkedHashMap | TreeMap |
|------|:-------:|:------------:|:-------:|
| 底層 | 陣列+鏈結/紅黑樹 | HashMap + 雙向鏈結 | 紅黑樹 |
| 順序 | 無序 | 插入序 or 訪問序 | 鍵的自然序/Comparator |
| get/put | O(1) | O(1) | O(log n) |
| null key | ✅ (1個) | ✅ (1個) | ❌ |
| 記憶體 | 中 | 略大 (鏈結) | 中 |
| 執行緒安全 | ❌ | ❌ | ❌ |

```java
Map<String, Integer> map = new HashMap<>();

// 基本操作
map.put("A", 1);                      // 新增/更新
map.get("A");                         // 取值 (null if absent)
map.getOrDefault("B", 0);             // 取不到返預設
map.containsKey("A");                 // 是否包含鍵
map.remove("A");                      // 刪除
map.size();                           // 大小

// 便捷方法 (Java 8+)
map.putIfAbsent("A", 1);             // 僅不存在時寫入
map.computeIfAbsent("A", k -> load(k));  // 不存在時計算
map.computeIfPresent("A", (k, v) -> v + 1); // 存在時計算
map.merge("A", 1, Integer::sum);         // 合併

// 迭代
for (Map.Entry<String, Integer> e : map.entrySet()) {
    System.out.println(e.getKey() + "=" + e.getValue());
}
map.forEach((k, v) -> System.out.println(k + "=" + v));

// 不可變 Map (Java 9+)
Map<String, Integer> immutable = Map.of("A", 1, "B", 2);
Map<String, Integer> immutable2 = Map.ofEntries(
    Map.entry("A", 1),
    Map.entry("B", 2)
);
```

## 4.6 HashMap 內部原理

### 資料結構

```
HashMap = 陣列 (Node<K,V>[] table) + 鏈結串列 + 紅黑樹 (Java 8+)

table[0]  → Node("A",1) → Node("B",2) → null
table[1]  → null
table[2]  → Node("C",3) → null
table[3]  → TreeNode → TreeNode → ... (紅黑樹, 鏈長≥8 時)
...
table[15] → null
```

### put 流程

```
1. 計算 hash = (key == null) ? 0 : h = key.hashCode() ^ (h >>> 16)
2. 計算 index = (n - 1) & hash         // n = table.length
3. table[index] 為 null?
   ├─ 是 → 新建 Node，放入
   └─ 否 → 遍歷鏈結/紅黑樹
             ├─ key 已存在 → 更新 value
             └─ key 不存在 → 尾插新 Node
                              → 若鏈長 ≥ 8 → 轉紅黑樹 (需 table.length ≥ 64)
4. size++，若 size > threshold → resize()
```

### 雜湊碰撞與紅黑樹轉換

```java
// 雜湊計算
static final int hash(Object key) {
    int h;
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
    // 高位參與低位的 XOR，降低碰撞
}

// 桶索引計算
index = (table.length - 1) & hash;
// 等價於 hash % table.length (table.length 總是 2 的冪)

// 紅黑樹條件
static final int TREEIFY_THRESHOLD = 8;     // 鏈長 ≥ 8 時轉樹
static final int UNTREEIFY_THRESHOLD = 6;   // 樹節點 ≤ 6 時退為鏈
static final int MIN_TREEIFY_CAPACITY = 64; // table.length ≥ 64 才轉樹
```

### resize (擴容) 機制

```java
// 預設值
static final int DEFAULT_INITIAL_CAPACITY = 16;  // 初始容量
static final float DEFAULT_LOAD_FACTOR = 0.75f;  // 負載因子
// threshold = capacity * loadFactor = 16 * 0.75 = 12

// 擴容條件: size > threshold
// 擴容策略: newCapacity = oldCapacity << 1 (2 倍)
// 重新雜湊: 每個節點重新計算 index = hash & (newCap - 1)
```

### 為什麼 HashMap 容量是 2 的冪？

```
index = hash & (length - 1)

當 length = 16 (10000):
  length - 1 = 15 (01111)
  hash & 01111 → 均勻分布在 0-15

當 length = 10 (非 2 的冪):
  length - 1 = 9 (01001)
  某些 bit 位置永遠為 0 → 分布不均，碰撞增加
```

### 執行緒不安全問題

```java
// HashMap 在並發環境下的問題：
// 1. 死循環 (Java 7 頭插法擴容時的鏈結反轉)
// 2. 資料遺失 (put 時覆蓋)
// 3. size 計數不準確

// ✅ 並發場景用 ConcurrentHashMap
Map<String, Integer> concurrentMap = new ConcurrentHashMap<>();
```

## 4.7 ConcurrentHashMap

### 原理

| Java 版本 | 實現 | 分段鎖 |
|-----------|------|:-----:|
| **Java 7** | Segment[] + HashEntry[] | ✅ 16 段 (預設) |
| **Java 8+** | Node[] + synchronized + CAS | ❌ 鎖粒度為單個桶 |

```java
// Java 8+ ConcurrentHashMap 原理
// - 陣列 + 鏈結 + 紅黑樹 (與 HashMap 結構相同)
// - put: 空桶用 CAS 寫入; 非空桶用 synchronized 鎖住桶頭節點
// - get: 無鎖 (volatile 保證可見性)
// - size: 分段計數 + 樂觀重試
// - 擴容: 多執行緒協同遷移 (transfer)

ConcurrentHashMap<String, Integer> chm = new ConcurrentHashMap<>();

// 原子複合操作
chm.putIfAbsent("key", 1);
chm.compute("key", (k, v) -> v == null ? 1 : v + 1);
chm.merge("key", 1, Integer::sum);

// 並行遍歷 (弱一致性)
chm.forEach(1, (k, v) -> System.out.println(k + "=" + v));

// search / reduce (Java 8+)
String found = chm.search(4, (k, v) -> v > 10 ? k : null);
```

### Map 執行緒安全選項

| 方案 | 機制 | 效能 | 適用場景 |
|------|------|:---:|---------|
| `Hashtable` | 全表 synchronized | 低 | 遺留代碼，不推薦 |
| `Collections.synchronizedMap()` | 包裝器，全表 synchronized | 低 | 低並發 |
| `ConcurrentHashMap` | CAS + synchronized (桶級) | 高 | 高並發讀寫 |

## 4.8 Collections 工具類

```java
// 排序
Collections.sort(list);
Collections.sort(list, Comparator.reverseOrder());

// 搜尋
int idx = Collections.binarySearch(list, "C");  // 需先排序

// 反轉/洗牌
Collections.reverse(list);
Collections.shuffle(list);

// 極值
String min = Collections.min(list);
String max = Collections.max(list, String.CASE_INSENSITIVE_ORDER);

// 填充/複製
Collections.fill(list, "X");
Collections.copy(dest, src);

// 不可變包裝 (返回唯讀視圖)
List<String> unmodifiable = Collections.unmodifiableList(list);
Set<String> singleton = Collections.singleton("A");
List<String> empty = Collections.emptyList();

// 同步包裝
List<String> syncList = Collections.synchronizedList(new ArrayList<>());

// 頻率
int freq = Collections.frequency(list, "A");

// 旋轉
Collections.rotate(list, 2);  // 右移 2 位
```

### Sequenced Collections (Java 21+)

```java
// 新介面階層
interface SequencedCollection<E> extends Collection<E> {
    void addFirst(E);          E removeFirst();        E getFirst();
    void addLast(E);           E removeLast();         E getLast();
    SequencedCollection<E> reversed();
}

interface SequencedSet<E> extends Set<E>, SequencedCollection<E> {}
interface SequencedMap<K,V> extends Map<K,V> {
    Entry<K,V> firstEntry();   Entry<K,V> lastEntry();
    Entry<K,V> pollFirstEntry(); Entry<K,V> pollLastEntry();
    SequencedMap<K,V> reversed();
}

// ArrayList, LinkedHashSet, TreeSet, LinkedHashMap, TreeMap 都實作了新介面
```

## 4.9 選擇指南 Selection Guide

| 場景 | 推薦 | 原因 |
|------|------|------|
| 一般列表，讀多寫少 | `ArrayList` | O(1) 隨機訪問 |
| 頻繁頭部增刪 | `LinkedList` or `ArrayDeque` | O(1) 頭部操作 |
| 去重，無序即可 | `HashSet` | O(1) |
| 去重，需有序 | `TreeSet` | O(log n)，自動排序 |
| 去重，需保留插入序 | `LinkedHashSet` | O(1) + 插入序 |
| KV 存儲，一般場景 | `HashMap` | O(1) |
| KV 存儲，需排序 | `TreeMap` | O(log n) |
| KV 存儲，需 LRU | `LinkedHashMap` (access-order) | 內建訪問序 |
| 高並發場景 | `ConcurrentHashMap` | 分段鎖/CAS |
| 棧 (LIFO) | `ArrayDeque` (不用 Stack) | 高效 |
| FIFO 隊列 | `ArrayDeque` or `LinkedList` | — |
| 優先級隊列 | `PriorityQueue` | 最小/最大堆 |

## 要點回顧 Key Takeaways

| # | 要點 |
|---|------|
| 1 | Collection 是頂級介面，分 List / Set / Queue 三大分支；Map 獨立體系 |
| 2 | ArrayList O(1) 讀, O(n) 插刪; LinkedList O(n) 讀, O(1) 頭尾操作 |
| 3 | HashMap: 陣列+鏈結+紅黑樹; 容量 2 的冪, 負載因子 0.75; Java 8+ 鏈長 ≥8 轉樹 |
| 4 | ConcurrentHashMap Java 8+ 用 CAS + synchronized (桶級), get 無鎖 |
| 5 | 永遠用 ArrayDeque 代替 Stack; 用 ConcurrentHashMap 代替 Hashtable |
| 6 | Java 21 引入 SequencedCollection / SequencedMap 統一有序操作介面 |
