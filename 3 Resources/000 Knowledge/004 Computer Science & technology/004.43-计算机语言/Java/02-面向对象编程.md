# 02 — 物件導向程式設計 Object-Oriented Programming

> OOP 四大支柱：封裝、繼承、多型、抽象。Java 是純物件導向語言（除了基本型別）。

---

## 2.1 類別與物件 Class & Object

### 類別定義

```java
public class Person {
    // 欄位 (Fields) — 實例變量
    private String name;        // private: 僅類別內部可訪問
    private int age;

    // 靜態欄位 — 屬於類別，所有實例共享
    private static int count = 0;

    // 建構子 (Constructor) — 可重載
    public Person() {
        this("Unknown", 0);     // 調用另一個建構子
    }

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
        count++;
    }

    // Getter / Setter (封裝)
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }

    // 方法
    public void introduce() {
        System.out.println("Hi, I'm " + name + ", " + age + " years old.");
    }

    // toString
    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }
}

// 使用
Person p = new Person("Alice", 25);
p.introduce();
```

### 訪問修飾子 Access Modifiers

| 修飾子 | 同類 | 同套件 | 子類 | 全域 |
|--------|:---:|:-----:|:---:|:---:|
| `private` | ✅ | ❌ | ❌ | ❌ |
| (default) | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ |

### static 關鍵字

| 對象 | 說明 |
|------|------|
| `static` 變量 | 類別變量，所有實例共享，可透過 `Person.count` 訪問 |
| `static` 方法 | 類別方法，不可訪問 `this`，不可調用非靜態方法 |
| `static` 內部類 | 靜態巢狀類，不依賴外部類實例 |
| `static` 初始化塊 | 類別載入時執行一次 |

```java
public class Config {
    public static final String APP_NAME = "MyApp";  // 常數
    private static String env;

    static {
        // 靜態初始化塊 — 類別載入時執行一次
        env = System.getenv("APP_ENV");
        System.out.println("Config loaded");
    }
}
```

### this 與 super

```java
public class Child extends Parent {
    private String name;

    public Child(String name, String parentName) {
        super(parentName);     // 調用父類建構子，必須在第一行
        this.name = name;      // 區分欄位與參數
        this.init();           // 調用本類其他方法
    }
}
```

## 2.2 繼承 Inheritance

```java
// 父類（基底類）
public class Animal {
    protected String species;

    public Animal(String species) {
        this.species = species;
    }

    public void eat() {
        System.out.println(species + " is eating.");
    }

    // final 方法不可被覆寫
    public final void breathe() {
        System.out.println(species + " is breathing.");
    }
}

// 子類（衍生類） — Java 單繼承
public class Dog extends Animal {
    private String breed;

    public Dog(String breed) {
        super("Dog");          // 必須顯式調用父類建構子
        this.breed = breed;
    }

    @Override                      // 覆寫標記（編譯器檢查）
    public void eat() {
        System.out.println(breed + " dog is eating bones.");
    }
}

// final 類別不可被繼承
public final class Cat extends Animal {
    public Cat() { super("Cat"); }
}
```

### 繼承規則

| 規則 | 說明 |
|------|------|
| 單繼承 | 一個類別只能有一個直接父類（但可實現多個介面） |
| `final` 類 | 不能被繼承，如 `String`, `Integer` |
| `final` 方法 | 不能被子類覆寫 |
| 建構子 | 不可繼承，子類必須透過 `super()` 調用父類建構子 |
| `Object` 類 | 所有類別的終極父類，提供 `toString()`, `equals()`, `hashCode()`, `clone()` 等 |

### Object 類別關鍵方法

```java
public class EqualsDemo {
    private String name;
    private int id;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof EqualsDemo that)) return false;
        return id == that.id && Objects.equals(name, that.name);
    }

    @Override
    public int hashCode() {
        return Objects.hash(name, id);
    }

    @Override
    public String toString() {
        return "EqualsDemo{name='" + name + "', id=" + id + "}";
    }

    // clone() 需實現 Cloneable 介面
}
```

| 方法 | 用途 |
|------|------|
| `equals(Object)` | 比較兩個物件是否「相等」（預設比較參考） |
| `hashCode()` | 返回雜湊碼，與 `equals` 一致 |
| `toString()` | 字串表示 |
| `clone()` | 淺複製 (需 `Cloneable`) |
| `getClass()` | 返回執行時類別資訊 |
| `finalize()` | GC 前回調（已棄用，Java 9+） |

## 2.3 多型 Polymorphism

### 編譯時多型 vs 執行時多型

| 類型 | 機制 | 範例 |
|------|------|------|
| **編譯時多型** (靜態) | 方法重載 (Overload) | 同方法名，不同參數列表 |
| **執行時多型** (動態) | 方法覆寫 (Override) | 父類參考 → 子類物件，動態綁定 |

```java
// 動態多型示範
Animal a1 = new Dog("Golden Retriever");
Animal a2 = new Cat();

a1.eat();   // 輸出: "Golden Retriever dog is eating bones." (Dog.eat)
a2.eat();   // 輸出: "Cat is eating." (Cat.eat)

// 向上轉型 (Upcasting) — 自動
Dog dog = new Dog("Husky");
Animal animal = dog;          // Dog → Animal, 安全

// 向下轉型 (Downcasting) — 需顯式
if (animal instanceof Dog d) {  // Java 16+ pattern matching
    System.out.println(d.getBreed());
}
```

### 方法呼叫的綁定規則

```
1. 編譯器根據參考型別決定可用的方法簽名
2. JVM 在執行時根據實際物件型別查找方法 (Dynamic Dispatch)
3. 查找順序: 實際類別 → 父類別 → ... → Object
4. static / final / private 方法為靜態綁定
```

### 抽象類別 Abstract Class

```java
public abstract class Shape {
    protected String color;

    // 抽象方法 — 無實作，子類必須覆寫
    public abstract double area();
    public abstract double perimeter();

    // 具體方法 — 可被繼承或覆寫
    public String getColor() {
        return color;
    }
}

public class Circle extends Shape {
    private double radius;

    public Circle(double radius) {
        this.radius = radius;
    }

    @Override
    public double area() {
        return Math.PI * radius * radius;
    }

    @Override
    public double perimeter() {
        return 2 * Math.PI * radius;
    }
}
```

## 2.4 介面 Interface

### 介面定義與實作

```java
// 介面定義
public interface Drawable {
    // 抽象方法 (public abstract)
    void draw();

    // 預設方法 (Java 8+) — 提供預設實作
    default void print() {
        System.out.println("Printing...");
    }

    // 靜態方法 (Java 8+)
    static Drawable createDefault() {
        return () -> System.out.println("Default drawable");
    }

    // 私有方法 (Java 9+) — 輔助預設方法
    private void helper() {
        System.out.println("helper");
    }
}

// 實作多個介面
public class Rectangle implements Drawable, Comparable<Rectangle> {
    @Override
    public void draw() {
        System.out.println("Drawing rectangle");
    }

    @Override
    public int compareTo(Rectangle other) {
        return 0;
    }
}
```

### 抽象類別 vs 介面

| 特性 | Abstract Class | Interface |
|------|:-------------:|:---------:|
| 繼承 | 單繼承 | 多實作 |
| 建構子 | ✅ 有 | ❌ 無 |
| 欄位 | 任意 | 僅 `public static final` 常數 |
| 方法 | abstract + concrete | abstract + default + static (Java 8+) |
| 存取修飾子 | 任意 | 方法自動 `public` |
| 何時用 | is-a 關係，有共用狀態 | can-do 能力，定義契約 |

### 標記介面 Marker Interface

```java
// 標記介面 — 無方法，僅作型別標記
public interface Serializable {}    // 表示可序列化
public interface Cloneable {}       // 表示可用 clone()
public interface RandomAccess {}    // 表示支援快速隨機訪問 (ArrayList)
```

### 函數式介面 Functional Interface (Java 8+)

```java
@FunctionalInterface  // 確保只有一個抽象方法
public interface Calculator {
    int calculate(int a, int b);
}

// 使用 Lambda
Calculator add = (a, b) -> a + b;
Calculator multiply = (a, b) -> a * b;
System.out.println(add.calculate(3, 5));  // 8
```

| 內建函數式介面 | 方法 | 說明 |
|---------------|------|------|
| `Function<T,R>` | `R apply(T t)` | 轉換函數 |
| `Consumer<T>` | `void accept(T t)` | 消費型 |
| `Supplier<T>` | `T get()` | 提供型 |
| `Predicate<T>` | `boolean test(T t)` | 斷言型 |
| `BiFunction<T,U,R>` | `R apply(T t, U u)` | 雙參數函數 |
| `UnaryOperator<T>` | `T apply(T t)` | 一元運算元（繼承 Function） |

## 2.5 內部類別 Inner Classes

| 類型 | 定義 | 特點 |
|------|------|------|
| **成員內部類** | 在類別內部定義的非靜態類 | 可訪問外部類所有成員；需外部實例 |
| **靜態巢狀類** | `static class` | 不可訪問外部實例成員；獨立存在 |
| **局部內部類** | 方法內部定義 | 可訪問 final/effectively final 局部變量 |
| **匿名內部類** | `new Interface() {...}` | 無類名，用於一次性實作 |

```java
public class Outer {
    private int x = 10;

    // 1. 成員內部類
    class Inner {
        void print() { System.out.println(x); }
    }

    // 2. 靜態巢狀類
    static class StaticNested {
        void hello() { System.out.println("Static Nested"); }
    }

    // 3 & 4: 局部 + 匿名
    public Runnable getRunnable() {
        // 匿名內部類
        return new Runnable() {
            @Override
            public void run() {
                System.out.println("Anonymous inner class");
            }
        };
    }

    // Lambda 是匿名內部類的語法糖（限函數式介面）
    public Runnable getLambda() {
        return () -> System.out.println("Lambda version");
    }
}

// 創建
Outer.Inner inner = new Outer().new Inner();
Outer.StaticNested nested = new Outer.StaticNested();
```

## 2.6 枚舉 Enum

```java
// 基本枚舉
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

// 增強枚舉 — 帶欄位、建構子、方法
public enum Planet {
    MERCURY(3.303e+23, 2.4397e6),
    VENUS  (4.869e+24, 6.0518e6),
    EARTH  (5.976e+24, 6.37814e6);

    private final double mass;    // kg
    private final double radius;  // m

    Planet(double mass, double radius) {
        this.mass = mass;
        this.radius = radius;
    }

    public double surfaceGravity() {
        return G * mass / (radius * radius);
    }

    private static final double G = 6.67300E-11;
}

// 使用
Planet earth = Planet.EARTH;
System.out.println(earth.surfaceGravity());
```

### Enum 常用方法

| 方法 | 說明 |
|------|------|
| `values()` | 返回所有枚舉值陣列 |
| `valueOf(String)` | 依名返回枚舉值（大小寫敏感） |
| `ordinal()` | 返回宣告順序（從 0 開始） |
| `name()` | 返回枚舉名 |
| `compareTo()` | 依宣告順序比較 |

### Enum + Switch

```java
// Java 14+ Switch 表達式
String type = switch (day) {
    case SATURDAY, SUNDAY -> "Weekend";
    case MONDAY -> "Start of week";
    default -> "Weekday";
};
```

## 2.7 Record (Java 14+ 預覽, Java 16+ 正式)

> Record 是不可變資料載體，自動生成建構子、accessor、equals/hashCode/toString。

```java
// 基本 Record
public record Point(int x, int y) {}

// 使用
Point p = new Point(3, 4);
System.out.println(p.x());    // accessor, 不是 getX()
System.out.println(p.y());
System.out.println(p);        // Point[x=3, y=4]

// Record 可自訂建構子（緊湊形式）
public record Range(int lo, int hi) {
    public Range {  // 緊湊建構子 — 無參數列表
        if (lo > hi) throw new IllegalArgumentException(
            "lo(" + lo + ") > hi(" + hi + ")");
    }
}

// Record 可實作介面，可定義靜態方法
public record Person(String name, int age) implements Comparable<Person> {
    // 靜態欄位 OK
    private static final String UNKNOWN = "Unknown";

    // 靜態方法 OK
    public static Person unnamed(int age) {
        return new Person(UNKNOWN, age);
    }

    // 實例方法 OK
    public String nameToUpperCase() {
        return name.toUpperCase();
    }

    @Override
    public int compareTo(Person other) {
        return Integer.compare(this.age, other.age);
    }
}

// Record Patterns (Java 19+ 預覽, Java 21+ 正式)
if (p instanceof Point(int x, int y)) {
    System.out.println("x=" + x + ", y=" + y);
}

// 巢狀 Record Pattern
record Rectangle(Point topLeft, Point bottomRight) {}
if (r instanceof Rectangle(Point(int x1, int y1), Point(int x2, int y2))) {
    // ...
}
```

### Record vs Class vs Lombok @Data

| 特性 | Record | Class | Lombok @Data |
|------|:------:|:-----:|:------------:|
| 不可變 | ✅ 強制 | 可選 | 可選 |
| 繼承 | ❌ 不可 extends | ✅ | 可選 |
| 可被繼承 | ❌ (隱式 final) | ✅ | 可選 |
| 欄位 | 僅 component | 任意 | 任意 |
| getter | `x()` (accessor) | `getX()` | `getX()` |
| 序列化 | 特殊支援 | 一般 | 一般 |

## 2.8 封裝與不可變性 Encapsulation & Immutability

### 封裝原則

```java
// 好的封裝
public class BankAccount {
    private final String accountId;      // final: 不可重新賦值
    private double balance;              // private: 封裝

    public BankAccount(String accountId) {
        this.accountId = accountId;
    }

    public synchronized void deposit(double amount) {
        if (amount <= 0) throw new IllegalArgumentException("Amount must be positive");
        balance += amount;
    }

    public synchronized boolean withdraw(double amount) {
        if (amount > balance) return false;
        balance -= amount;
        return true;
    }

    public double getBalance() { return balance; }
    // 無 setBalance — 透過業務方法控制修改
}
```

### 不可變類別 Immutable Class

```java
// 不可變類別設計清單
public final class ImmutablePerson {          // 1. final class
    private final String name;                 // 2. private final fields
    private final List<String> hobbies;

    public ImmutablePerson(String name, List<String> hobbies) {
        this.name = name;
        this.hobbies = List.copyOf(hobbies);   // 3. 防禦性複製
    }

    public String getName() { return name; }   // 4. 無 setter

    public List<String> getHobbies() {
        return hobbies;  // OK: List.copyOf 已返回不可變 List
    }

    // 可提供 withXxx 方法返回新實例
    public ImmutablePerson withName(String newName) {
        return new ImmutablePerson(newName, this.hobbies);
    }
}
```

## 2.9 Sealed Classes (Java 17+)

```java
// 密封類別 — 限制哪些類可以繼承/實作
public sealed class Shape
    permits Circle, Rectangle, Triangle {
}

public final class Circle extends Shape {
    private final double radius;
    public Circle(double radius) { this.radius = radius; }
}

public non-sealed class Rectangle extends Shape {
    // non-sealed: 開放給任意子類
}

public final class Triangle extends Shape {}

// sealed interface
sealed interface Expression
    permits Constant, Add, Multiply {}

record Constant(int value) implements Expression {}
record Add(Expression left, Expression right) implements Expression {}
record Multiply(Expression left, Expression right) implements Expression {}
```

### sealed 子類修飾子

| 修飾子 | 含義 |
|--------|------|
| `final` | 不能被進一步繼承 |
| `sealed` | 進一步限制子類 |
| `non-sealed` | 開放給任意子類繼承 |
