## 7.1 IO 模型演進 BIO → NIO → AIO

| 模型 | 說明 | Blocking | 執行緒模型 | 適用場景 |
|------|------|:--------:|-----------|---------|
| **BIO** | 每連接一執行緒 | 阻塞 | Thread-per-Connection | 連接數少、固定架構 |
| **NIO** | Selector 多工復用 | 非阻塞 | Reactor (少數執行緒) | 高併發連接 |
| **AIO** | 非同步回調 | 非阻塞 | Proactor (系統通知) | 大檔案傳輸 |

```java
// BIO 模型 (最簡單, 每連接一個執行緒)
ServerSocket server = new ServerSocket(8080);
while (true) {
    Socket client = server.accept();  // 阻塞等待
    new Thread(() -> handle(client)).start();
}

// NIO 模型 (Selector 多工復用)
Selector selector = Selector.open();
ServerSocketChannel serverChannel = ServerSocketChannel.open();
serverChannel.configureBlocking(false);
serverChannel.register(selector, SelectionKey.OP_ACCEPT);

while (true) {
    selector.select();  // 阻塞直到有事件
    Set<SelectionKey> keys = selector.selectedKeys();
    for (SelectionKey key : keys) {
        if (key.isAcceptable()) { /* 接受連接 */ }
        if (key.isReadable())   { /* 讀取資料 */ }
    }
    keys.clear();
}
```

## 7.3 NIO 核心元件 (java.nio)

### Buffer 緩衝區

```java
// Buffer 三屬性: position / limit / capacity
ByteBuffer buffer = ByteBuffer.allocate(1024);   // Heap Buffer
ByteBuffer direct = ByteBuffer.allocateDirect(1024); // Direct Buffer (堆外)

buffer.put("hello".getBytes());
buffer.flip();             // 切換到讀模式: limit=position, position=0
while (buffer.hasRemaining()) {
    System.out.print((char) buffer.get());
}
buffer.clear();            // 重置為寫模式 (不清除資料)
// buffer.compact();      // 保留未讀資料，壓縮到頭部

// Direct vs Heap
// Heap Buffer: 分配在 JVM Heap，GC 管理，資料拷貝到堆外
// Direct Buffer: 分配在原生記憶體，零拷貝潛力，分配/釋放成本高
```

### Channel 通道

```java
// Channel: 雙向，可讀可寫
FileChannel fc = FileChannel.open(Paths.get("file.dat"));
MappedByteBuffer mmap = fc.map(
    FileChannel.MapMode.READ_ONLY, 0, fc.size());
fc.close();

// SocketChannel (NIO 網路)
SocketChannel sc = SocketChannel.open(new InetSocketAddress("localhost", 8080));
sc.configureBlocking(false);
sc.write(ByteBuffer.wrap("hello".getBytes()));

// transferTo / transferFrom — 零拷貝
try (FileChannel src = FileChannel.open(Paths.get("src"));
     FileChannel dest = FileChannel.open(Paths.get("dest"),
         StandardOpenOption.CREATE, StandardOpenOption.WRITE)) {
    src.transferTo(0, src.size(), dest);  // DMA 零拷貝 (Linux sendfile)
}
```

### Selector 選擇器

```java
Selector selector = Selector.open();
ServerSocketChannel ssChannel = ServerSocketChannel.open();
ssChannel.bind(new InetSocketAddress(8080));
ssChannel.configureBlocking(false);
ssChannel.register(selector, SelectionKey.OP_ACCEPT);

while (true) {
    int ready = selector.select();  // 可加超時 select(1000)
    if (ready == 0) continue;

    Iterator<SelectionKey> iter = selector.selectedKeys().iterator();
    while (iter.hasNext()) {
        SelectionKey key = iter.next();
        if (key.isAcceptable()) {
            SocketChannel client = ssChannel.accept();
            client.configureBlocking(false);
            client.register(selector, SelectionKey.OP_READ);
        } else if (key.isReadable()) {
            SocketChannel client = (SocketChannel) key.channel();
            ByteBuffer buf = ByteBuffer.allocate(256);
            int read = client.read(buf);
            if (read == -1) {
                client.close();   // 連接關閉
            } else {
                buf.flip();
                // 處理資料...
            }
        }
        iter.remove();  // 必須移除已處理的 key
    }
}
```

## 7.4 Netty 網路框架

### 為什麼選擇 Netty？

| 優勢 | 說明 |
|------|------|
| **高效** | 基於 NIO + epoll，零拷貝，記憶體池化 |
| **易用** | Channel Pipeline 鏈式處理，遮罩底層細節 |
| **健壯** | 心跳/重連/流量整形/優雅關閉 |
| **生態** | gRPC/Dubbo/RocketMQ 底層依賴 |

### Netty 入門範例

```java
// 服務端
EventLoopGroup bossGroup = new NioEventLoopGroup(1);
EventLoopGroup workerGroup = new NioEventLoopGroup();
try {
    ServerBootstrap b = new ServerBootstrap();
    b.group(bossGroup, workerGroup)
     .channel(NioServerSocketChannel.class)
     .childHandler(new ChannelInitializer<SocketChannel>() {
         @Override
         protected void initChannel(SocketChannel ch) {
             ch.pipeline().addLast(
                 new StringDecoder(),
                 new StringEncoder(),
                 new SimpleChannelInboundHandler<String>() {
                     @Override
                     protected void channelRead0(ChannelHandlerContext ctx, String msg) {
                         System.out.println("收到: " + msg);
                         ctx.writeAndFlush("Echo: " + msg);
                     }
                 }
             );
         }
     });
    ChannelFuture f = b.bind(8080).sync();
    f.channel().closeFuture().sync();
} finally {
    bossGroup.shutdownGracefully();
    workerGroup.shutdownGracefully();
}
```

### Channel Pipeline 與 Handler

```
Inbound  ──────────────────────────────────────────►
         FrameDecoder → StringDecoder → BusinessHandler
Outbound ◄──────────────────────────────────────────
         FrameEncoder ← StringEncoder ← BusinessHandler
```

```java
// ChannelHandler 生命週期
public class LifecycleHandler extends ChannelInboundHandlerAdapter {
    @Override public void channelRegistered(ChannelHandlerContext ctx)   { /* 註冊 */ }
    @Override public void channelActive(ChannelHandlerContext ctx)       { /* 活躍 */ }
    @Override public void channelRead(ChannelHandlerContext ctx, Object msg) { /* 讀取 */ }
    @Override public void channelReadComplete(ChannelHandlerContext ctx) { /* 讀取完成 */ }
    @Override public void channelInactive(ChannelHandlerContext ctx)     { /* 非活躍 */ }
    @Override public void channelUnregistered(ChannelHandlerContext ctx) { /* 移除 */ }
    @Override public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) {
        cause.printStackTrace();
        ctx.close();
    }
}
```

### ByteBuf — Netty 的緩衝區

```java
// ByteBuf 優化: 讀寫分離、引用計數、記憶體池化
ByteBuf buf = Unpooled.buffer(256);       // 非池化
// ByteBuf buf = PooledByteBufAllocator.DEFAULT.buffer(256); // 池化

buf.writeInt(42);
buf.writeBytes("hello".getBytes());

int value = buf.readInt();
byte[] bytes = new byte[buf.readableBytes()];
buf.readBytes(bytes);

buf.release();  // 引用計數 -1，歸還池

// 零拷貝操作
ByteBuf slice = buf.slice(0, 5);    // 共享記憶體，不拷貝
ByteBuf copy = buf.copy();           // 深拷貝
CompositeByteBuf composite = Unpooled.compositeBuffer();  // 聚合多個 ByteBuf
```

## 7.5 序列化 Serialization

### Java 原生序列化

```java
// 實現 Serializable 介面
public class User implements Serializable {
    private static final long serialVersionUID = 1L;  // 版本控制
    private String name;
    private transient String password;  // 不序列化
}

// 序列化/反序列化
try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("user.dat"))) {
    oos.writeObject(new User("alice", "secret"));
}

try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream("user.dat"))) {
    User user = (User) ois.readObject();
}
```

### 第三方序列化框架

| 框架 | 特點 | 跨語言 | 生態 |
|------|------|:------:|------|
| **Protobuf** | 高效二進制，Schema 先行 | ✅ | gRPC |
| **JSON (Jackson/Gson)** | 人類可讀，通用 | ✅ | REST API |
| **MessagePack** | 類 JSON 的二進制 | ✅ | 高效 RPC |
| **Thrift** | Apache，RPC+序列化 | ✅ | 大數據 |
| **Hessian** | 二進制，輕量 | ✅ | Dubbo |
| **Kryo** | 高效 Java 專用 | ❌ | Spark/Flink |

## 7.6 實戰：簡單 HTTP Server (NIO / Netty)

```java
// JDK 內建 HttpServer (Java 6+, com.sun.net.httpserver)
HttpServer server = HttpServer.create(new InetSocketAddress(8080), 0);
server.createContext("/hello", exchange -> {
    String response = "Hello, World!";
    exchange.sendResponseHeaders(200, response.length());
    try (OutputStream os = exchange.getResponseBody()) {
        os.write(response.getBytes());
    }
});
server.setExecutor(Executors.newVirtualThreadPerTaskExecutor()); // Java 21
server.start();
```

## 總結 Summary

| 層次 | 元件 | 核心場景 |
|------|------|---------|
| **BIO** | java.io + 執行緒池 | 簡單、低並發 |
| **NIO** | Buffer + Channel + Selector | 高並發連接 (自建框架) |
| **Netty** | Channel Pipeline + ByteBuf | 生產級網路應用 |
| **序列化** | JSON/Protobuf/Kryo | API / RPC / 大數據 |
| **檔案** | Files + Path | 替代 File API |
