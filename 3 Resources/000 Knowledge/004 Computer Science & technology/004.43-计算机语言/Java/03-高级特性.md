## 3.1 泛型 Generics

### 為什麼需要泛型？

```java
// 沒有泛型之前 (Java 1.4)
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0);  // 需要強制轉型，可能 ClassCastException

// 泛型 (Java 5+)
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0);           // 型別安全，無需轉型
```

### 泛型類別、介面、方法

```java
// 泛型類別
public class Box<T> {
    private T value;

    public void set(T value) { this.value = value; }
    public T get() { return value; }

    // 泛型方法 — 獨立於類別泛型參數
    public static <U> Box<U> of(U value) {
        Box<U> box = new Box<>();
        box.set(value);
        return box;
    }
}

// 泛型介面
public interface Pair<K, V> {
    K getKey();
    V getValue();
}

// 多邊界
public class SortedBox<T extends Comparable<T> & Serializable> {
    // T 必須實作 Comparable 和 Serializable
}
```

### 型別擦除 Type Erasure

> Java 泛型在編譯後擦除型別資訊，位元組碼中無泛型。

```java
// 編譯前
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0);

// 編譯後（等價於）
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0);  // 編譯器自動插入轉型
```

| 影響 | 說明 |
|------|------|
| 無法 `instanceof` 泛型 | `obj instanceof List<String>` ❌ — 型別資訊已擦除 |
| 無法 `new T()` | 執行時不知道 T 是什麼 |
| 無法建立泛型陣列 | `new T[10]` ❌ |
| 方法簽名衝突 | `void print(List<String>)` 與 `void print(List<Integer>)` 擦除後相同 ❌ |

### 萬用字元 Wildcards

```java
// ? — 無界萬用字元
void printList(List<?> list) {
    for (Object o : list) System.out.println(o);
    // list.add(anything);  ❌ 無法寫入（除了 null）
}

// ? extends T — 上界萬用字元 (協變, Producer)
double sum(List<? extends Number> numbers) {
    double s = 0;
    for (Number n : numbers) s += n.doubleValue();
    return s;
    // numbers.add(42);  ❌ 無法寫入
}
// 可傳入 List<Integer>, List<Double>, List<Number>

// ? super T — 下界萬用字元 (逆變, Consumer)
void addNumbers(List<? super Integer> list) {
    list.add(42);  // ✅ 可寫入 Integer
    list.add(99);
}
// 可傳入 List<Integer>, List<Number>, List<Object>
```

### PECS 原則 (Producer Extends, Consumer Super)

```
"Producer Extends, Consumer Super"

Producer (從集合讀取): ? extends T
Consumer (往集合寫入): ? super T
既要讀又要寫: 不用萬用字元，直接用 T
```

```java
// 經典案例: Collections.copy
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    // dest 是 consumer → ? super T
    // src 是 producer → ? extends T
}
```

## 3.2 註解 Annotations

### 內建註解 Built-in Annotations

| 註解 | 適用於 | 說明 |
|------|--------|------|
| `@Override` | 方法 | 標記方法覆寫，編譯器檢查 |
| `@Deprecated` | 類/方法/欄位 | 標記已過時 |
| `@SuppressWarnings` | 各處 | 抑制編譯器警告 |
| `@FunctionalInterface` | 介面 | 確保函數式介面只有一個抽象方法 |
| `@SafeVarargs` | 方法/建構子 | 抑制 varargs 泛型警告 |

```java
public class Legacy {
    @Deprecated(since = "2.0", forRemoval = true)
    public void oldMethod() { /* ... */ }

    @SuppressWarnings("unchecked")
    public static <T> List<T> unsafeCast(Object o) {
        return (List<T>) o;
    }

    @SafeVarargs
    public static <T> List<T> asList(T... items) {
        return Arrays.asList(items);
    }
}
```

### 元註解 Meta-Annotations

| 元註解 | 說明 |
|--------|------|
| `@Retention` | 註解保留範圍：SOURCE / CLASS / RUNTIME |
| `@Target` | 註解可用於何處：TYPE / METHOD / FIELD / PARAMETER... |
| `@Documented` | 包含在 Javadoc 中 |
| `@Inherited` | 子類可繼承父類註解 |
| `@Repeatable` | 可重複標記 (Java 8+) |

### 自訂註解 Custom Annotation

```java
// 定義
@Retention(RetentionPolicy.RUNTIME)
@Target({ElementType.METHOD, ElementType.TYPE})
@Documented
public @interface LogExecution {
    String value() default "";         // 元素，可設預設值
    Level level() default Level.INFO;

    enum Level { DEBUG, INFO, WARN, ERROR }
}

// 使用
@LogExecution(value = "UserService", level = LogExecution.Level.WARN)
public class UserService {

    @LogExecution("createUser")
    public void createUser(String name) {
        // ...
    }
}
```

### 註解處理器 Annotation Processor (編譯時)

```java
// 註解處理器 — 編譯時生成代碼
@SupportedAnnotationTypes("com.example.Immutable")
@SupportedSourceVersion(SourceVersion.RELEASE_21)
public class ImmutableProcessor extends AbstractProcessor {
    @Override
    public boolean process(Set<? extends TypeElement> annotations,
                           RoundEnvironment roundEnv) {
        for (Element element : roundEnv.getElementsAnnotatedWith(Immutable.class)) {
            // 生成構建器類別...
        }
        return true;
    }
}
```

### 常用第三方註解框架

| 框架 | 用途 |
|------|------|
| **Lombok** | `@Data`, `@Builder`, `@Slf4j` — 減少重複代碼 |
| **Spring** | `@Autowired`, `@Component`, `@Transactional` |
| **Jackson** | `@JsonProperty`, `@JsonIgnore` |
| **JUnit 5** | `@Test`, `@BeforeEach`, `@ParameterizedTest` |
| **Jakarta Validation** | `@NotNull`, `@Size`, `@Email` |

## 3.3 反射 Reflection

### 基本用法

```java
// 獲取 Class 物件的三種方式
Class<?> c1 = String.class;
Class<?> c2 = "hello".getClass();
Class<?> c3 = Class.forName("java.lang.String");

// 獲取類別資訊
System.out.println(c1.getName());          // java.lang.String
System.out.println(c1.getSimpleName());    // String
System.out.println(c1.getPackageName());   // java.lang
System.out.println(Modifier.isFinal(c1.getModifiers()));  // true

// 獲取建構子
Constructor<?>[] constructors = c1.getDeclaredConstructors();

// 獲取方法
Method[] methods = c1.getDeclaredMethods();

// 獲取欄位
Field[] fields = c1.getDeclaredFields();
```

### 動態操作

```java
public class ReflectDemo {
    public static void main(String[] args) throws Exception {
        // 1. 動態建立物件
        Class<?> personClass = Class.forName("com.example.Person");
        Constructor<?> constructor = personClass.getConstructor(String.class, int.class);
        Object person = constructor.newInstance("Alice", 25);

        // 2. 調用方法
        Method getName = personClass.getMethod("getName");
        String name = (String) getName.invoke(person);

        // 3. 存取私有欄位
        Field ageField = personClass.getDeclaredField("age");
        ageField.setAccessible(true);          // 繞過訪問控制
        int age = ageField.getInt(person);
        ageField.set(person, 30);

        // 4. 獲取註解
        if (personClass.isAnnotationPresent(Entity.class)) {
            Entity entity = personClass.getAnnotation(Entity.class);
            System.out.println(entity.value());
        }
    }
}
```

### 反射 API 總覽

| 類別 | 用途 |
|------|------|
| `Class<?>` | 類別的執行時描述 |
| `Field` | 欄位資訊與操作 |
| `Method` | 方法資訊與調用 |
| `Constructor` | 建構子資訊與實例化 |
| `Modifier` | 解析修飾子 |
| `Array` | 動態建立/操作陣列 |
| `Proxy` | 動態代理 (JDK Dynamic Proxy) |

### 反射的應用場景與代價

| 應用場景 | 範例 |
|----------|------|
| ORM 框架 | Hibernate, MyBatis |
| DI 容器 | Spring IoC |
| 序列化 | Jackson, Gson |
| 單元測試 | JUnit, Mockito |
| 動態代理 | Spring AOP |

> ⚠️ 反射代價：效能開銷大、喪失編譯時型別安全、繞過封裝。

## 3.4 Lambda 表達式與方法引用

### Lambda 語法

```java
// 完整形式
(Type1 p1, Type2 p2) -> { statements; return result; }

// 簡化形式
p -> p.getName()                // 單參數，單表達式
(p1, p2) -> p1 + p2             // 多參數，單表達式
() -> System.out.println()      // 無參數
(String s) -> s.length()        // 可帶型別
```

### 方法引用 Method Reference

| 類型 | 語法 | Lambda 等價 |
|------|------|-----------|
| 靜態方法引用 | `ClassName::method` | `x -> ClassName.method(x)` |
| 實例方法引用 | `instance::method` | `x -> instance.method(x)` |
| 任意物件實例方法 | `ClassName::method` | `(x, y) -> x.method(y)` |
| 建構子引用 | `ClassName::new` | `() -> new ClassName()` |

```java
// 範例
List<String> names = Arrays.asList("Alice", "Bob", "Charlie");

// Lambda
names.forEach(name -> System.out.println(name));

// 方法引用
names.forEach(System.out::println);

// 排序
names.sort(String::compareToIgnoreCase);

// Stream 中
names.stream()
     .map(String::toUpperCase)      // 實例方法引用
     .filter(s -> s.length() > 3)
     .collect(Collectors.toList());

// 建構子引用
Supplier<List<String>> supplier = ArrayList::new;
List<String> list = supplier.get();
```

### Lambda vs 匿名內部類

| 特性 | Lambda | 匿名內部類 |
|------|:------:|:--------:|
| this 含義 | 外圍類別實例 | 匿名類本身 |
| 可定義新欄位/方法 | ❌ | ✅ |
| 字節碼 | invokedynamic | 獨立 .class 檔案 |
| 僅限函數式介面 | ✅ | ❌ (任意介面/類別) |

## 3.5 Stream API

### Stream 生命週期

```
來源 (Source) → 中間操作 (Intermediate) → 終端操作 (Terminal)
   集合/陣列        filter/map/sorted...    collect/forEach/reduce...
                  (惰性求值，返回 Stream)   (觸發計算，返回結果)
```

### 創建 Stream

```java
// 從集合
Stream<String> s1 = list.stream();
Stream<String> s2 = list.parallelStream();

// 從陣列
Stream<Integer> s3 = Arrays.stream(new Integer[]{1, 2, 3});
Stream<Integer> s4 = Stream.of(1, 2, 3);

// 生成
Stream<Double> randoms = Stream.generate(Math::random).limit(10);
Stream<Integer> iterate = Stream.iterate(0, n -> n + 2).limit(10);

// 其他
IntStream.range(1, 100);                    // [1, 100)
IntStream.rangeClosed(1, 100);              // [1, 100]
"hello\nworld".lines();                     // Java 11+
```

### 中間操作 Intermediate Operations

| 操作 | 說明 | 範例 |
|------|------|------|
| `filter(Predicate)` | 過濾 | `.filter(s -> s.length() > 3)` |
| `map(Function)` | 轉換 | `.map(String::length)` |
| `flatMap(Function)` | 扁平化 | `.flatMap(line -> line.chars())` |
| `distinct()` | 去重 | `.distinct()` |
| `sorted()` | 排序 | `.sorted()` / `.sorted(Comparator)` |
| `peek(Consumer)` | 偵錯 peek | `.peek(System.out::println)` |
| `limit(n)` | 截取前 n | `.limit(10)` |
| `skip(n)` | 跳過前 n | `.skip(5)` |
| `takeWhile()` | 條件為真時保留 (Java 9+) | `.takeWhile(x -> x < 10)` |
| `dropWhile()` | 條件為真時丟棄 (Java 9+) | `.dropWhile(x -> x < 10)` |

### 終端操作 Terminal Operations

| 操作 | 返回 | 說明 |
|------|------|------|
| `collect(Collector)` | 集合/Map/String | 最通用的收集操作 |
| `toList()` | List (不可變) | Java 16+ 簡寫 |
| `forEach(Consumer)` | void | 遍歷 |
| `reduce(identity, BinaryOperator)` | Optional / T | 歸約 |
| `count()` | long | 計數 |
| `min(Comparator)` / `max()` | Optional | 最小/最大 |
| `findFirst()` / `findAny()` | Optional | 找第一個/任意一個 |
| `anyMatch()` / `allMatch()` / `noneMatch()` | boolean | 匹配 |
| `toArray()` | Object[] | 轉陣列 |

### Collectors 常用方法

```java
List<Person> people = getPeople();

// toList (Java 16+)
List<String> names = people.stream()
    .map(Person::getName)
    .toList();

// toMap
Map<Integer, Person> byId = people.stream()
    .collect(Collectors.toMap(Person::getId, Function.identity()));

// groupingBy
Map<String, List<Person>> byCity = people.stream()
    .collect(Collectors.groupingBy(Person::getCity));

// partitioningBy
Map<Boolean, List<Person>> adults = people.stream()
    .collect(Collectors.partitioningBy(p -> p.getAge() >= 18));

// joining
String csv = people.stream()
    .map(Person::getName)
    .collect(Collectors.joining(", "));

// summarizing
IntSummaryStatistics stats = people.stream()
    .collect(Collectors.summarizingInt(Person::getAge));
System.out.println(stats.getAverage());
System.out.println(stats.getMax());
```

### Parallel Stream 並行串流

```java
// 注意：不適合有狀態操作、共享可變狀態、或資料量小的場景
long sum = LongStream.rangeClosed(1, 1_000_000_000L)
    .parallel()                    // 啟用並行
    .sum();

// ForkJoinPool 共同執行緒池 (工作竊取演算法)
System.setProperty("java.util.concurrent.ForkJoinPool.common.parallelism", "8");
```

## 3.6 Optional

### 建立 Optional

```java
Optional<String> opt1 = Optional.of("hello");     // 非 null
Optional<String> opt2 = Optional.ofNullable(x);   // 可為 null
Optional<String> opt3 = Optional.empty();         // 空
```

### 常用方法

```java
Optional<String> opt = Optional.ofNullable(getName());

// 獲取值
String s1 = opt.get();                     // ⚠️ NoSuchElementException 如果空
String s2 = opt.orElse("default");         // 有值返回值，無值返回預設
String s3 = opt.orElseGet(() -> compute());// 延遲計算預設值
String s4 = opt.orElseThrow();             // Java 10+, 等同 get()
String s5 = opt.orElseThrow(NotFound::new);// 自訂異常

// 條件操作
opt.ifPresent(System.out::println);             // 有值時執行
opt.ifPresentOrElse(                            // Java 9+
    System.out::println,
    () -> System.out.println("empty"));

// 轉換
Optional<Integer> len = opt.map(String::length);
Optional<Integer> len2 = opt.flatMap(s -> Optional.of(s.length()));

// 過濾
Optional<String> filtered = opt.filter(s -> s.length() > 3);

// Stream (Java 9+)
opt.stream().forEach(System.out::println);
```

### Optional 最佳實踐

| ✅ DO | ❌ DON'T |
|------|---------|
| 用於方法返回值 | 用於欄位或方法參數 |
| 結合 `orElseGet` 延遲計算 | 在效能敏感路徑使用 |
| 用 `map`/`flatMap` 鏈式轉換 | `opt.get()` 不先 `isPresent()` 檢查 |
| `orElseThrow` 明確表達語義 | 用 Optional 取代 null 檢查過度 |

```java
// ✅ 好的寫法
public Optional<User> findUser(String email) {
    return Optional.ofNullable(userRepo.findByEmail(email));
}

// ✅ 鏈式處理
String city = findUser(email)
    .map(User::getAddress)
    .map(Address::getCity)
    .orElse("Unknown");
```

## 要點回顧 Key Takeaways

| # | 要點 |
|---|------|
| 1 | 泛型型別擦除：執行時無泛型資訊，使用 PECS 原則處理萬用字元 |
| 2 | 註解分 SOURCE/CLASS/RUNTIME 三級保留；`@Retention(RUNTIME)` 才能反射讀取 |
| 3 | 反射功能強大但效能較差且有安全顧慮，框架中用得最多 |
| 4 | Lambda 底層用 invokedynamic，非匿名內部類 |
| 5 | Stream 是惰性求值；終端操作才觸發計算；謹慎使用 parallelStream |
| 6 | Optional 是方法返回值的容器，不是欄位也不是參數 |
