# 05 — 並發程式設計 Concurrency Programming

> Java 並發從 Thread 到 Virtual Thread，從 synchronized 到 AQS，涵蓋完整的並發工具生態。

## 5.1 執行緒基礎 Thread Fundamentals

### 建立執行緒的三種方式

```java
// 方式 1: 繼承 Thread
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println(Thread.currentThread().getName() + " running");
    }
}
new MyThread().start();

// 方式 2: 實作 Runnable
new Thread(() -> {
    System.out.println("Runnable with Lambda");
}).start();

// 方式 3: 實作 Callable (有返回值，可拋異常)
ExecutorService executor = Executors.newFixedThreadPool(2);
Future<String> future = executor.submit(() -> {
    Thread.sleep(1000);
    return "Result";
});
String result = future.get(2, TimeUnit.SECONDS);  // 阻塞等待結果
```

### 執行緒生命週期 Thread States

```
        NEW
         │  start()
         ▼
    ┌─ RUNNABLE ──────────────┐
    │    │  │  │  (CPU 調度)    │
    │    │  │  └───────────────┘
    │    │  │  synchronized/Lock
    │    │  ▼
    │    │ BLOCKED ── (獲得鎖) ──→ RUNNABLE
    │    │
    │    │  wait() / park() / sleep() / join()
    │    ▼
    │  WAITING / TIMED_WAITING ── (notify / timeout) ──→ RUNNABLE
    │
    └─ (run 完成) ──→ TERMINATED
```

| 狀態 | 說明 | 進入方式 |
|------|------|---------|
| NEW | 已建立未啟動 | `new Thread()` |
| RUNNABLE | 可執行 (含運行+就緒) | `start()` |
| BLOCKED | 等待鎖 | 競爭 synchronized |
| WAITING | 無限等待 | `wait()`, `join()`, `park()` |
| TIMED_WAITING | 限時等待 | `sleep()`, `wait(timeout)` |
| TERMINATED | 執行完畢 | run() 完成或異常退出 |

### 執行緒基本方法

| 方法 | 說明 |
|------|------|
| `start()` | 啟動執行緒，JVM 調用 `run()` |
| `run()` | 執行緒執行體（不要直接調用） |
| `sleep(long)` | 當前執行緒休眠，不釋放鎖 |
| `join()` | 等待該執行緒結束 |
| `yield()` | 提示排程器讓出 CPU |
| `interrupt()` | 中斷執行緒（設定中斷標記） |
| `setDaemon(true)` | 設為守護執行緒 |
| `setPriority(int)` | 設定優先級 (1-10) |
| `getState()` | 獲取當前狀態 |

## 5.2 synchronized 與 volatile

### synchronized

```java
public class SynchronizedDemo {
    private int count = 0;
    private static int staticCount = 0;

    // 1. 同步實例方法 — 鎖當前物件 this
    public synchronized void increment() {
        count++;
    }

    // 2. 同步靜態方法 — 鎖 Class 物件
    public static synchronized void staticIncrement() {
        staticCount++;
    }

    // 3. 同步代碼塊 — 鎖指定物件
    private final Object lock = new Object();
    public void blockSync() {
        synchronized (lock) {
            count++;
        }
    }

    // 等價於
    public void explicitSync() {
        synchronized (this) {
            count++;
        }
    }
}
```

### synchronized 底層實現

```
Java 代碼: synchronized(obj) { ... }

字節碼:
  monitorenter   ← 獲取鎖
  ... body ...
  monitorexit    ← 釋放鎖 (正常路徑)
  monitorexit    ← 釋放鎖 (異常路徑)

HotSpot 實現:
  - 偏向鎖 (Biased Locking): 同一執行緒重複獲取，無競爭開銷
  - 輕量級鎖 (Thin Lock): CAS 自旋獲取
  - 重量級鎖 (Fat Lock): 作業系統 mutex (執行緒阻塞/喚醒)

  Java 15 預設禁用偏向鎖; Java 21 Loom 後進一步簡化
```

### volatile

```java
public class VolatileDemo {
    private volatile boolean flag = false;
    // volatile 保證:
    // 1. 可見性: 一個執行緒修改，其他執行緒立即可見
    // 2. 有序性: 禁止指令重排序

    public void writer() {
        flag = true;  // volatile 寫入
    }

    public void reader() {
        if (flag) {   // volatile 讀取
            // 一定能看到 writer 的修改
        }
    }
}

// 雙重檢查鎖定 (DCL) — 經典 volatile 應用
public class Singleton {
    private static volatile Singleton instance;

    public static Singleton getInstance() {
        if (instance == null) {                    // 第一次檢查
            synchronized (Singleton.class) {
                if (instance == null) {            // 第二次檢查
                    instance = new Singleton();    // 非原子操作
                }
            }
        }
        return instance;
    }
    // volatile 防止 instance = new Singleton() 的指令重排序
    // (分配記憶體 → 初始化物件 → 賦值給 instance)
    // 若無 volatile，可能先賦值再初始化，導致其他執行緒拿到未初始化物件
}
```

### synchronized vs volatile vs Lock

| 特性 | synchronized | volatile | Lock |
|------|:-----------:|:--------:|:----:|
| 原子性 | ✅ | ❌ | ✅ |
| 可見性 | ✅ | ✅ | ✅ |
| 有序性 | ✅ (部分) | ✅ | ✅ |
| 可中斷 | ❌ | — | ✅ |
| 嘗試獲取 | ❌ | — | ✅ `tryLock()` |
| 公平鎖 | ❌ | — | ✅ |
| 條件變量 | wait/notify | — | Condition |
| 效能 | 較輕 (偏向/輕量鎖) | 輕 | 較重 |

## 5.3 JUC Lock 框架 (AQS)

### AQS (AbstractQueuedSynchronizer) 原理

```
AQS = volatile int state + FIFO 等待隊列 (CLH 變體)

  state 表示同步狀態:
    ReentrantLock:  state=0 未鎖, state>0 重入次數
    Semaphore:      state=N 剩餘許可數
    CountDownLatch: state=N 倒數計數

  Node 節點封裝執行緒，雙向鏈結隊列:
    head → Node(T1) → Node(T2) → Node(T3) → tail
            等待獲取鎖的執行緒排隊 (CLH 自旋)
```

### ReentrantLock

```java
public class LockDemo {
    private final ReentrantLock lock = new ReentrantLock(true);  // 公平鎖

    public void safeMethod() {
        lock.lock();
        try {
            // 臨界區
        } finally {
            lock.unlock();  // 必須在 finally 中釋放
        }
    }

    // 嘗試獲取鎖
    public void tryLockDemo() throws InterruptedException {
        if (lock.tryLock(1, TimeUnit.SECONDS)) {
            try {
                // 獲取成功
            } finally {
                lock.unlock();
            }
        } else {
            // 獲取失敗，做其他事
        }
    }

    // 可中斷獲取
    public void interruptibleLock() throws InterruptedException {
        lock.lockInterruptibly();  // 可被 Thread.interrupt() 中斷
        try {
            // ...
        } finally {
            lock.unlock();
        }
    }
}
```

### Condition (條件變量)

```java
// 生產者-消費者 (ArrayBlockingQueue 的簡化原理)
public class BoundedBuffer<T> {
    private final ReentrantLock lock = new ReentrantLock();
    private final Condition notFull = lock.newCondition();
    private final Condition notEmpty = lock.newCondition();
    private final T[] items;
    private int putptr, takeptr, count;

    public void put(T x) throws InterruptedException {
        lock.lock();
        try {
            while (count == items.length)
                notFull.await();          // 等待 notFull
            items[putptr] = x;
            if (++putptr == items.length) putptr = 0;
            count++;
            notEmpty.signal();            // 通知 notEmpty
        } finally {
            lock.unlock();
        }
    }

    public T take() throws InterruptedException {
        lock.lock();
        try {
            while (count == 0)
                notEmpty.await();         // 等待 notEmpty
            T x = items[takeptr];
            if (++takeptr == items.length) takeptr = 0;
            count--;
            notFull.signal();             // 通知 notFull
            return x;
        } finally {
            lock.unlock();
        }
    }
}
```

### ReadWriteLock / StampedLock

```java
public class CacheDemo {
    // ReadWriteLock: 讀讀共享，讀寫互斥，寫寫互斥
    private final ReadWriteLock rwLock = new ReentrantReadWriteLock();
    private final Map<String, Object> cache = new HashMap<>();
    private volatile boolean cacheValid;

    public Object get(String key) {
        rwLock.readLock().lock();
        try {
            return cache.get(key);
        } finally {
            rwLock.readLock().unlock();
        }
    }

    // StampedLock (Java 8+): 樂觀讀，效能更高
    private final StampedLock stampedLock = new StampedLock();

    public Object optimisticRead(String key) {
        long stamp = stampedLock.tryOptimisticRead();
        Object value = cache.get(key);
        if (!stampedLock.validate(stamp)) {
            // 樂觀讀失敗，降級為悲觀讀
            stamp = stampedLock.readLock();
            try {
                value = cache.get(key);
            } finally {
                stampedLock.unlockRead(stamp);
            }
        }
        return value;
    }
}
```

## 5.4 原子類與 CAS

### CAS (Compare-And-Swap)

```
CAS(V, expected, newValue):
  if (V == expected) {
      V = newValue;
      return true;
  } else {
      return false;
  }
// 由 CPU 指令 (cmpxchg) 保證原子性
// 問題: ABA 問題 → 用 AtomicStampedReference 解決
```

### 原子類 Atomic Classes

| 類別 | 用途 |
|------|------|
| `AtomicInteger` / `AtomicLong` / `AtomicBoolean` | 基本型別原子操作 |
| `AtomicReference<V>` | 參考型別原子操作 |
| `AtomicIntegerArray` / `AtomicLongArray` | 陣列原子操作 |
| `AtomicIntegerFieldUpdater` | 欄位原子更新 |
| `LongAdder` / `DoubleAdder` | 高並發計數器 (比 AtomicLong 效能好) |
| `AtomicStampedReference` | 帶版本號 (解決 ABA) |

```java
// AtomicInteger 常用操作
AtomicInteger ai = new AtomicInteger(0);
ai.incrementAndGet();        // ++i
ai.getAndIncrement();        // i++
ai.compareAndSet(0, 1);     // CAS
ai.updateAndGet(x -> x * 2); // 原子函數式更新

// LongAdder: 高並發場景下比 AtomicLong 快
LongAdder adder = new LongAdder();
adder.increment();
adder.add(10);
long sum = adder.sum();     // 獲取總和 (非原子快照)
```

## 5.5 執行緒池 ThreadPoolExecutor

### 核心參數

```
ThreadPoolExecutor(
    int corePoolSize,         核心執行緒數
    int maximumPoolSize,      最大執行緒數
    long keepAliveTime,       空閒執行緒存活時間
    TimeUnit unit,
    BlockingQueue<Runnable> workQueue,  工作隊列
    ThreadFactory threadFactory,
    RejectedExecutionHandler handler    拒絕策略
)

執行流程:
  任務提交 → 核心執行緒處理 → 隊列等待 → 擴容至最大 → 拒絕
```

### 執行緒池類型

| 方法 | corePoolSize | maxPoolSize | 隊列 | 適用場景 |
|------|:-----------:|:-----------:|------|---------|
| `newFixedThreadPool(n)` | n | n | LinkedBlockingQueue (無限) | 穩定並發 |
| `newCachedThreadPool()` | 0 | Integer.MAX | SynchronousQueue | 短任務大量 |
| `newSingleThreadExecutor()` | 1 | 1 | LinkedBlockingQueue | 順序執行 |
| `newScheduledThreadPool(n)` | n | Integer.MAX | DelayedWorkQueue | 定時調度 |
| `newWorkStealingPool()` | CPU 核數 | CPU 核數 | — | ForkJoinPool |

```java
// 推薦：手動建立執行緒池
ThreadPoolExecutor executor = new ThreadPoolExecutor(
    4,                              // corePoolSize
    8,                              // maximumPoolSize
    60L, TimeUnit.SECONDS,          // keepAliveTime
    new LinkedBlockingQueue<>(100), // 有界隊列！
    new ThreadFactory() {
        private final AtomicInteger count = new AtomicInteger(1);
        @Override
        public Thread newThread(Runnable r) {
            Thread t = new Thread(r, "pool-worker-" + count.getAndIncrement());
            t.setDaemon(false);
            return t;
        }
    },
    new ThreadPoolExecutor.CallerRunsPolicy()  // 拒絕策略
);

// 提交任務
executor.execute(() -> System.out.println("Runnable"));              // Runnable, 無返回
Future<String> f = executor.submit(() -> "Callable result");         // Callable, 有返回

// 關閉
executor.shutdown();                          // 平穩關閉
executor.shutdownNow();                       // 立即關閉
executor.awaitTermination(30, TimeUnit.SECONDS);
```

### 四種拒絕策略

| 策略 | 行為 |
|------|------|
| `AbortPolicy` (預設) | 拋 RejectedExecutionException |
| `CallerRunsPolicy` | 由提交任務的執行緒直接執行 |
| `DiscardPolicy` | 直接丟棄 |
| `DiscardOldestPolicy` | 丟棄隊列中最舊的任務 |

### ExecutorService 生命週期

```
RUNNING → shutdown() → SHUTDOWN → 隊列清空 → TIDYING → TERMINATED
RUNNING → shutdownNow() → STOP → TIDYING → TERMINATED
```

## 5.6 JUC 工具類

### CountDownLatch / CyclicBarrier / Semaphore

```java
// CountDownLatch: 一次性等待多個任務完成
CountDownLatch latch = new CountDownLatch(3);
for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        // do work
        latch.countDown();   // 倒數
    }).start();
}
latch.await();               // 等待歸零

// CyclicBarrier: 多執行緒互相等待，可重複使用
CyclicBarrier barrier = new CyclicBarrier(3, () -> {
    System.out.println("所有執行緒到達柵欄，執行回調");
});
for (int i = 0; i < 3; i++) {
    new Thread(() -> {
        // phase 1
        barrier.await();     // 等待其他執行緒
        // phase 2
        barrier.await();     // 可重複使用
    }).start();
}

// Semaphore: 控制同時訪問的執行緒數
Semaphore semaphore = new Semaphore(3);  // 最多 3 個執行緒
semaphore.acquire();        // 獲取許可 (阻塞)
// ... 臨界區
semaphore.release();        // 釋放許可
```

### Exchanger / Phaser

```java
// Exchanger: 兩個執行緒交換資料
Exchanger<String> exchanger = new Exchanger<>();
new Thread(() -> {
    String data = "Thread-A data";
    String received = exchanger.exchange(data);
}).start();
new Thread(() -> {
    String data = "Thread-B data";
    String received = exchanger.exchange(data);
}).start();

// Phaser: 多階段並行柵欄 (CyclicBarrier 的增強版)
Phaser phaser = new Phaser(1);  // 註冊自己
for (int i = 0; i < 3; i++) {
    phaser.register();
    new Thread(() -> {
        phaser.arriveAndAwaitAdvance();  // phase 1
        phaser.arriveAndAwaitAdvance();  // phase 2
        phaser.arriveAndDeregister();    // 退出
    }).start();
}
phaser.arriveAndDeregister();  // 主執行緒退出
```

## 5.7 CompletableFuture (異步程式設計)

```java
// 建立
CompletableFuture<String> cf1 = CompletableFuture.supplyAsync(() -> {
    return "Hello";
});
CompletableFuture<Void> cf2 = CompletableFuture.runAsync(() -> {
    System.out.println("Async task");
});

// 轉換 (thenApply / thenCompose)
CompletableFuture<Integer> length = cf1.thenApply(String::length);
CompletableFuture<Integer> composed = cf1.thenCompose(s ->
    CompletableFuture.supplyAsync(() -> s.length()));

// 消費 (thenAccept / thenRun)
cf1.thenAccept(s -> System.out.println("Result: " + s));
cf1.thenRun(() -> System.out.println("Done"));

// 組合 (thenCombine)
CompletableFuture<String> cf3 = CompletableFuture.supplyAsync(() -> "World");
CompletableFuture<String> combined = cf1.thenCombine(cf3, (a, b) -> a + " " + b);

// 任一完成 (applyToEither)
CompletableFuture<String> cf4 = CompletableFuture.supplyAsync(() -> {
    sleep(1000); return "Slow";
});
CompletableFuture<String> cf5 = CompletableFuture.supplyAsync(() -> {
    sleep(100); return "Fast";
});
cf4.applyToEither(cf5, s -> "Winner: " + s)
   .thenAccept(System.out::println);  // "Winner: Fast"

// 全部完成 (allOf / anyOf)
CompletableFuture<Void> all = CompletableFuture.allOf(cf1, cf3, cf4);
all.join();  // 等待全部完成

// 異常處理
cf1.exceptionally(ex -> "Recovered")
   .handle((result, ex) -> ex != null ? "fallback" : result);
```

### CompletableFuture 組合速查表

| 方法 | 輸入 | 輸出 | 說明 |
|------|------|------|------|
| `thenApply(Function)` | T | U | 同步轉換 |
| `thenApplyAsync(Function)` | T | U | 異步轉換 |
| `thenCompose(Function)` | T | Future<U> | 扁平化 (類似 flatMap) |
| `thenCombine(CF, BiFunction)` | T, U | V | 組合兩個結果 |
| `thenAccept(Consumer)` | T | void | 消費結果 |
| `applyToEither(CF, Function)` | T or U | V | 任一完成即轉換 |
| `exceptionally(Function)` | Throwable | T | 異常恢復 |
| `handle(BiFunction)` | T, Throwable | U | 正常+異常處理 |

## 5.8 ForkJoinPool

```java
// ForkJoinPool: 工作竊取 (Work-Stealing) 演算法
// 適合大任務分割為小任務的場景 (Divide & Conquer)

public class SumTask extends RecursiveTask<Long> {
    private final long[] numbers;
    private final int start, end;
    private static final int THRESHOLD = 1000;

    public SumTask(long[] numbers, int start, int end) {
        this.numbers = numbers; this.start = start; this.end = end;
    }

    @Override
    protected Long compute() {
        if (end - start <= THRESHOLD) {
            long sum = 0;
            for (int i = start; i < end; i++) sum += numbers[i];
            return sum;
        }
        int mid = (start + end) / 2;
        SumTask left = new SumTask(numbers, start, mid);
        SumTask right = new SumTask(numbers, mid, end);
        left.fork();                  // 非同步執行 left
        Long rightResult = right.compute();  // 同步執行 right
        Long leftResult = left.join();       // 等待 left
        return leftResult + rightResult;
    }
}

ForkJoinPool pool = new ForkJoinPool();
long[] nums = new long[100_000];
SumTask task = new SumTask(nums, 0, nums.length);
long result = pool.invoke(task);
```

## 5.9 Virtual Threads (Java 21, Project Loom)

> 虛擬執行緒是 JVM 管理的輕量級執行緒，由平台執行緒承載，解決傳統執行緒的資源消耗問題。

### 平台執行緒 vs 虛擬執行緒

| 特性 | 平台執行緒 (Platform) | 虛擬執行緒 (Virtual) |
|------|:-------------------:|:------------------:|
| 調度者 | OS Kernel | JVM (ForkJoinPool) |
| 記憶體 | ~1MB stack | ~KB (heap 分配, 動態伸縮) |
| 上下文切換 | 昂貴 (kernel mode) | 廉價 (user mode) |
| 最大數量 | 數千 | 數百萬 |
| 阻塞後 | OS 執行緒阻塞 | Mount → Unmount (釋放 Carrier) |

### Virtual Thread 使用

```java
// 方式 1: Thread.ofVirtual()
Thread vThread = Thread.ofVirtual()
    .name("virtual-1")
    .start(() -> {
        System.out.println("Running in virtual thread");
    });

// 方式 2: Executors.newVirtualThreadPerTaskExecutor()
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    // 每個任務一個虛擬執行緒，語意上等價於無界執行緒池
    for (int i = 0; i < 10_000; i++) {
        executor.submit(() -> {
            Thread.sleep(Duration.ofSeconds(1));  // 阻塞時 unmount carrier
            return "Done";
        });
    }
}  // try-with-resources 自動關閉

// 方式 3: Structured Concurrency (預覽, Java 21)
try (var scope = new StructuredTaskScope.ShutdownOnFailure()) {
    Future<String> user = scope.fork(() -> fetchUser());
    Future<Integer> order = scope.fork(() -> fetchOrder());

    scope.join();            // 等待全部完成
    scope.throwIfFailed();   // 任一失敗則拋異常

    return new Response(user.resultNow(), order.resultNow());
}
```

### Virtual Thread 排程原理

```
┌──────────────────────────────────────┐
│            Virtual Threads            │
│  VT1  VT2  VT3  ...  VT_N            │
└────┬───────┬───────┬─────────────────┘
     │       │       │
┌────▼───────▼───────▼─────────────────┐
│         Carrier Threads               │
│  Platform Thread 1, 2, 3 ...         │
│  (ForkJoinPool, parallelism = cores) │
└──────────────────────────────────────┘
     │
     ▼
┌────────────┐
│  OS Kernel  │
└────────────┘

當 VT 遇到阻塞 (I/O, sleep, lock):
  1. VT 狀態設為 PARKED
  2. 從 Carrier unmount (釋放 Carrier)
  3. Carrier 可承載其他 VT
  4. 阻塞結束後，VT mount 到可用 Carrier
```

### Virtual Thread 注意事項

| ✅ 適合 | ❌ 不適合 |
|--------|---------|
| 大量短暫阻塞任務 | CPU 密集型長時間計算 |
| I/O 密集型 (HTTP, DB) | 使用 synchronized 阻塞的遺留代碼 |
| 每請求一執行緒模式 | Native 方法呼叫 (pinned) |
| 微服務後端 | 非常短的生命週期 (< 1μs) |

```java
// synchronized 會 pin Carrier 執行緒，虛擬執行緒效益打折
// ✅ 用 ReentrantLock 代替
synchronized (this) { ... }   // ❌ pins carrier
lock.lock(); try { ... } finally { lock.unlock(); }  // ✅
```

## 5.10 並發集合 Concurrent Collections

| 集合 | 原理 | 特性 |
|------|------|------|
| `ConcurrentHashMap` | CAS + synchronized (桶) | get 無鎖 |
| `CopyOnWriteArrayList` | 寫時複製陣列 | 讀無鎖，寫 O(n)，適合讀多寫少 |
| `CopyOnWriteArraySet` | 基於 CopyOnWriteArrayList | 同左 |
| `ConcurrentLinkedQueue` | CAS 的 Michael-Scott 無鎖隊列 | 無界無鎖 FIFO |
| `ConcurrentLinkedDeque` | CAS 無鎖雙端隊列 | 無界無鎖 Deque |
| `ArrayBlockingQueue` | 陣列 + ReentrantLock | 有界 FIFO |
| `LinkedBlockingQueue` | 鏈結 + 兩個 Lock | 可選有界 |
| `PriorityBlockingQueue` | 堆 + Lock | 無界優先級 |
| `DelayQueue` | PriorityQueue + Lock | 延遲隊列 |
| `SynchronousQueue` | 無容量，直接交接 | 生產者必須等待消費者 |
| `LinkedTransferQueue` | CAS + 鏈結 | 高效傳遞 |

## 要點回顧 Key Takeaways

| # | 要點 |
|---|------|
| 1 | synchronized 經歷偏向鎖 → 輕量級鎖 → 重量級鎖的升級過程 (Java 15+ 禁用偏向鎖) |
| 2 | AQS (AbstractQueuedSynchronizer) 是 JUC 的核心，基於 state + CLH 隊列 |
| 3 | 禁止用 Executors 快捷方法 (無界隊列 OOM)，手動建立 ThreadPoolExecutor |
| 4 | CompletableFuture 是異步程式設計首選，搭配 thenCombine / thenCompose |
| 5 | Virtual Threads (Java 21) 每個僅佔 ~KB，可建立數百萬個；避免 synchronized pinning |
| 6 | 並發集合選 ConcurrentHashMap (高並發) 和 CopyOnWriteArrayList (讀多寫少) |
