## 2.1 Class 基礎 (Class Fundamentals)

### 存取控制 (Access Control)

| 關鍵字 | 同類別 | 衍生類別 | 外部 |
|:---:|:---:|:---:|:---:|
| `public` | ✅ | ✅ | ✅ |
| `protected` | ✅ | ✅ | ❌ |
| `private` | ✅ | ❌ | ❌ |

```cpp
class Widget {
public:     // 外部可見介面
    Widget() = default;
    int getValue() const { return value_; }
    
protected:  // 衍生類別可見
    int value_ = 0;
    
private:    // 僅類別內部
    void internalHelper() { /* ... */ }
};

// struct vs class 唯一差異：struct 預設 public, class 預設 private
struct Point {
    double x, y;    // public by default
};
```

### 特殊成員函數 (Special Member Functions)

```cpp
class Example {
public:
    // 1. Default constructor (預設建構子)
    Example() = default;                     // 編譯器生成
    
    // 2. Destructor (解構子)
    ~Example() { /* 清理資源 */ }
    
    // 3. Copy constructor (複製建構子)
    Example(const Example& other) : data_(other.data_) {}
    
    // 4. Copy assignment operator (複製賦值)
    Example& operator=(const Example& other) {
        if (this != &other) { data_ = other.data_; }
        return *this;
    }
    
    // 5. Move constructor (移動建構子) C++11
    Example(Example&& other) noexcept 
        : data_(std::exchange(other.data_, 0)) {}
    
    // 6. Move assignment operator (移動賦值) C++11
    Example& operator=(Example&& other) noexcept {
        if (this != &other) { data_ = std::exchange(other.data_, 0); }
        return *this;
    }
    
private:
    int data_ = 0;
};
```

## 2.2 Rule of Five (五法則)

> 若你自定義了解構子、複製建構子或複製賦值中任一個，你幾乎一定需要全部五個（含移動語意）。

### 完整規則矩陣

| 你自定義了... | 必須也定義... | 原因 |
|------|------|------|
| Destructor | Copy ctor, Copy assign, Move ctor, Move assign | 資源管理 (RAII 類別) |
| Copy ctor | Copy assign, Destructor, Move 系列 | 深複製語意 |
| Move ctor | Move assign | 移動語意一致性 |
| 以上全部 | — | 完整資源生命週期管理 |

### Rule of Zero (現代 C++ 最佳實踐)

> 若你的類別**不管理**原始資源（記憶體、檔案、socket），使用 `=default`，讓編譯器生成一切。

```cpp
// Rule of Zero: 讓編譯器管理
struct Good {
    std::string name;
    std::vector<int> data;
    std::unique_ptr<Impl> pImpl;  // unique_ptr 自動管理生命週期
    // ✅ 不需自定義解構子/複製/移動 — 編譯器正確生成
};

// Rule of Five: 管理原始資源
class Buffer {
    char* ptr_;
    size_t size_;
public:
    explicit Buffer(size_t n) : ptr_(new char[n]), size_(n) {}
    ~Buffer() { delete[] ptr_; }
    
    // Copy (深複製)
    Buffer(const Buffer& other) : ptr_(new char[other.size_]), size_(other.size_) {
        std::copy(other.ptr_, other.ptr_ + size_, ptr_);
    }
    Buffer& operator=(const Buffer& other) {
        if (this != &other) {
            delete[] ptr_;  // 釋放舊資源
            size_ = other.size_;
            ptr_ = new char[size_];
            std::copy(other.ptr_, other.ptr_ + size_, ptr_);
        }
        return *this;
    }
    
    // Move (所有權轉移)
    Buffer(Buffer&& other) noexcept 
        : ptr_(std::exchange(other.ptr_, nullptr)),
          size_(std::exchange(other.size_, 0)) {}
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] ptr_;
            ptr_ = std::exchange(other.ptr_, nullptr);
            size_ = std::exchange(other.size_, 0);
        }
        return *this;
    }
};
```

### =default / =delete (C++11)

```cpp
class NonCopyable {
public:
    NonCopyable() = default;
    NonCopyable(const NonCopyable&) = delete;            // 禁止複製
    NonCopyable& operator=(const NonCopyable&) = delete;
    NonCopyable(NonCopyable&&) = default;                // 允許移動
    NonCopyable& operator=(NonCopyable&&) = default;
};

class Singleton : private NonCopyable { /* ... */ };
```

## 2.3 繼承 (Inheritance)

### 繼承型別對照表

| 繼承方式 | public 繼承 | protected 繼承 | private 繼承 |
|------|:---:|:---:|:---:|
| 基底 public 成員 → | public | protected | private |
| 基底 protected 成員 → | protected | protected | private |
| 基底 private 成員 → | 不可存取 | 不可存取 | 不可存取 |
| 語意 | IS-A (子型別) | 受限實作繼承 | 實作繼承 (HAS-A 替代) |

```cpp
class Animal {
public:
    virtual ~Animal() = default;          // ⚠️ 多型基底必須 virtual 解構子
    virtual void speak() const { std::cout << "?\n"; }
protected:
    std::string name_;
};

class Dog final : public Animal {         // final: 禁止進一步繼承
public:
    Dog(std::string name) { name_ = std::move(name); }
    void speak() const override {         // override: 顯式標記覆寫
        std::cout << name_ << ": Woof!\n";
    }
};

// 多重繼承 (Multiple Inheritance)
class Drawable {
public:
    virtual void draw() const = 0;        // Pure virtual (純虛擬函數)
};

class Clickable {
public:
    virtual void onClick() = 0;
};

class Button : public Drawable, public Clickable {
public:
    void draw() const override { /* ... */ }
    void onClick() override { /* ... */ }
};
```

### 菱形繼承問題 (Diamond Problem)

```cpp
struct A {
    int a = 0;
};
struct B : public A {};      // B 有 A::a
struct C : public A {};      // C 有 A::a
// struct D : public B, public C {};  // ❌ D 有兩個 A::a！

// ✅ 解決方案：虛擬繼承 (Virtual Inheritance)
struct B_v : virtual public A {};
struct C_v : virtual public A {};
struct D_v : public B_v, public C_v {
    void f() { a = 42; }     // ✅ 只有一個 A::a
};
```

## 2.4 虛擬函數與 vtable (Virtual Functions & vtable)

### vtable 機制

```
Object Layout (Dog 物件記憶體布局):
┌──────────────────┐
│  vtable pointer   │ ───▶ ┌─────────────────────────────┐
├──────────────────┤       │ type_info (RTTI)             │
│  name_ (string)   │       │ ~Dog() override              │
├──────────────────┤       │ speak()  override            │
│  ...              │       │ (若有更多虛擬函數...)          │
└──────────────────┘       └─────────────────────────────┘
```

### 虛擬函數行為矩陣

| 情境 | 呼叫機制 | 開銷 |
|------|:---:|:---:|
| 非虛擬函數 | Static dispatch (編譯期綁定) | 零開銷 |
| 虛擬函數 (透過基底指標/參考) | Dynamic dispatch (vtable 查詢) | ~1 次指標解參考 |
| 虛擬函數 (直接物件呼叫) | Static dispatch (編譯器優化) | 零開銷 |
| `final` 標記的 virtual | 編譯器可 devirtualize | 可能零開銷 |

```cpp
class Base {
public:
    virtual void f() const { std::cout << "Base::f()\n"; }
    void g() const { std::cout << "Base::g() (non-virtual)\n"; }
    virtual ~Base() = default;
};

class Derived : public Base {
public:
    void f() const override { std::cout << "Derived::f()\n"; }
    void g() const { std::cout << "Derived::g()\n"; }  // hides Base::g()!
};

void demo() {
    Derived d;
    Base& b = d;
    
    d.f();    // Derived::f()   — static dispatch (編譯器已知型別)
    b.f();    // Derived::f()   — dynamic dispatch (vtable)
    d.g();    // Derived::g()   — non-virtual
    b.g();    // Base::g()      — non-virtual! (隱藏 ≠ 覆寫)
}
```

### Abstract Base Class (抽象基底類別) / Interface

```cpp
// C++ 中的 Interface = 純虛擬函數 + virtual 解構子
class ILogger {
public:
    virtual ~ILogger() = default;
    virtual void log(const std::string& msg) = 0;  // Pure virtual
    virtual void flush() = 0;
};

class FileLogger : public ILogger {
    std::ofstream file_;
public:
    explicit FileLogger(const std::string& path) : file_(path) {}
    void log(const std::string& msg) override { file_ << msg << "\n"; }
    void flush() override { file_.flush(); }
};

// ✅ 依賴反轉：高層模組依賴 ILogger 而非具體實現
void process(ILogger& logger) {
    logger.log("Processing...");
}
```

## 2.5 RAII (Resource Acquisition Is Initialization)

> C++ 最重要的資源管理典範：建構子獲取資源，解構子釋放資源。保證異常安全。

### RAII 範例矩陣

| 資源型別 | RAII 包裝器 | 取得操作 | 釋放操作 |
|------|------|------|------|
| Heap memory | `std::unique_ptr<T>` / `std::shared_ptr<T>` | `std::make_unique<T>()` | 解構子自動 `delete` |
| File | `std::fstream` / `std::ofstream` | 建構子 `open()` | 解構子自動 `close()` |
| Mutex lock | `std::lock_guard` / `std::scoped_lock` | 建構子 `lock()` | 解構子自動 `unlock()` |
| Socket | Custom RAII class | 建構子 `socket()` | 解構子 `close()` |
| GDI handle (Win) | Custom RAII class | `CreatePen()` | `DeleteObject()` |

```cpp
// RAII 經典模式
template<typename T>
class ScopedPointer {       // 類似 std::unique_ptr 的簡化版
    T* ptr_;
public:
    explicit ScopedPointer(T* p = nullptr) : ptr_(p) {}
    ~ScopedPointer() { delete ptr_; }
    
    ScopedPointer(ScopedPointer&& other) noexcept 
        : ptr_(std::exchange(other.ptr_, nullptr)) {}
    ScopedPointer& operator=(ScopedPointer&& other) noexcept {
        if (this != &other) {
            delete ptr_;
            ptr_ = std::exchange(other.ptr_, nullptr);
        }
        return *this;
    }
    
    // Non-copyable
    ScopedPointer(const ScopedPointer&) = delete;
    ScopedPointer& operator=(const ScopedPointer&) = delete;
    
    T* operator->() const { return ptr_; }
    T& operator*() const { return *ptr_; }
    explicit operator bool() const { return ptr_ != nullptr; }
};

// RAII 實戰：異常安全
void transferMoney(Account& from, Account& to, double amount) {
    // 使用 lock_guard 保證互斥鎖在函數返回時自動釋放
    std::scoped_lock lock(from.mutex_, to.mutex_);  // C++17: 多重鎖定防死鎖
    from.balance_ -= amount;
    to.balance_ += amount;
    // lock 自動釋放 — 即使中間拋出異常
}
```

## 2.6 運算子重載 (Operator Overloading)

### 常用運算子重載對照表

| 運算子                    | 建議  | 簽名                                             | 備註                     |
| ---------------------- | :-: | ---------------------------------------------- | ---------------------- |
| `=`                    | 成員  | `T& operator=(const T&)`                       | 複製賦值                   |
| `+=`, `-=`, `*=`       | 成員  | `T& operator+=(const T&)`                      | 複合賦值                   |
| `[]`                   | 成員  | `T& operator[](size_t)`                        | 下標存取                   |
| `()`                   | 成員  | `Ret operator()(Args...)`                      | 函數呼叫 (functor)         |
| `==`, `!=`, `<`, `<=>` | 非成員 | `bool operator==(const T&, const T&)`          | C++20: `<=>` 可自動生成全部比較 |
| `+`, `-`, `*`, `/`     | 非成員 | `T operator+(const T&, const T&)`              | 基於 `+=` 實作             |
| `<<`, `>>`             | 非成員 | `std::ostream& operator<<(ostream&, const T&)` | 串流 I/O                 |
| `->`, `*`              | 成員  | `T* operator->()`                              | 智慧指標/迭代器               |
| `++`, `--` (prefix)    | 成員  | `T& operator++()`                              | 前置遞增                   |
| `++`, `--` (postfix)   | 成員  | `T operator++(int)`                            | 後置遞增 (int 為 dummy)     |

```cpp
// 完整範例：有理數類別
class Rational {
    int num_, den_;  // numerator / denominator
public:
    Rational(int n = 0, int d = 1) : num_(n), den_(d) {
        if (d == 0) throw std::invalid_argument("denominator zero");
        reduce();
    }
    
    // 複合賦值 (成員)
    Rational& operator+=(const Rational& rhs) {
        num_ = num_ * rhs.den_ + rhs.num_ * den_;
        den_ *= rhs.den_;
        reduce();
        return *this;
    }
    
    // 比較 (C++20 spaceship)
    auto operator<=>(const Rational& rhs) const {
        return num_ * rhs.den_ <=> rhs.num_ * den_;
    }
    bool operator==(const Rational& rhs) const = default;  // C++20
    
    // 輸出 (非成員 friend)
    friend std::ostream& operator<<(std::ostream& os, const Rational& r) {
        return os << r.num_ << '/' << r.den_;
    }
    
private:
    void reduce() {
        int g = std::gcd(num_, den_);
        num_ /= g; den_ /= g;
        if (den_ < 0) { num_ = -num_; den_ = -den_; }
    }
};

// 非成員二元運算子 (基於 += 實作)
inline Rational operator+(Rational lhs, const Rational& rhs) {
    return lhs += rhs;  // 利用 RVO + copy elision
}
```

### C++20 三向比較 (Spaceship Operator)

```cpp
#include <compare>

struct Value {
    int id;
    std::string name;
    
    // auto 生成全部六個比較運算子!
    auto operator<=>(const Value&) const = default;
};

// 手動三向比較
struct Version {
    int major, minor, patch;
    std::strong_ordering operator<=>(const Version& rhs) const {
        if (auto cmp = major <=> rhs.major; cmp != 0) return cmp;
        if (auto cmp = minor <=> rhs.minor; cmp != 0) return cmp;
        return patch <=> rhs.patch;
    }
};
```

## 2.7 物件生命週期 (Object Lifetime)

```
Allocation           Construction       Destruction         Deallocation
    │                      │                  │                   │
    ▼                      ▼                  ▼                   ▼
 operator new  ──▶    Constructor    ──▶   Destructor    ──▶   operator delete
 (記憶體分配)         (初始化物件)        (清理資源)           (歸還記憶體)
```

```cpp
void lifetime_demo() {
    // 1. Static storage — 程式啟動建構、結束解構
    static Widget global_w;
    
    // 2. Automatic storage (stack) — 離開 scope 解構
    {
        Widget local_w;     // 建構
        // ... 
    }                       // local_w 解構 (RAII)
    
    // 3. Dynamic storage (heap) — 手動釋放或智慧指標
    auto p = std::make_unique<Widget>();  // new + constructor
    // ... 
}   // unique_ptr 解構 → delete Widget
```
