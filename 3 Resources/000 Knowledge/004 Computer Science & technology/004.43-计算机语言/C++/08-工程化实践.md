## 8.1 CMake 建置系統

### 基礎 CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.20)
project(MyProject VERSION 1.0.0 LANGUAGES CXX)

# C++20 標準
set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

# 建立執行檔
add_executable(my_app
    src/main.cpp
    src/utils.cpp
)

# 建立靜態/動態庫
add_library(my_lib STATIC
    src/lib/core.cpp
    src/lib/helper.cpp
)
target_include_directories(my_lib PUBLIC include/)
target_link_libraries(my_app PRIVATE my_lib)
```

### Modern CMake 模式

```cmake
# FetchContent — 嵌入第三方依賴 (CMake 3.11+)
include(FetchContent)
FetchContent_Declare(
    fmt
    GIT_REPOSITORY https://github.com/fmtlib/fmt.git
    GIT_TAG 10.2.1
)
FetchContent_MakeAvailable(fmt)

# CTest 整合
enable_testing()
add_subdirectory(tests)

# 編譯選項
target_compile_options(my_lib PRIVATE -Wall -Wextra -Wpedantic)
target_compile_features(my_lib PUBLIC cxx_std_20)

# install 規則
install(TARGETS my_lib DESTINATION lib)
install(DIRECTORY include/ DESTINATION include)
```

## 8.2 套件管理 (vcpkg / Conan)

### vcpkg 快速上手

```bash
# 安裝 vcpkg
git clone https://github.com/Microsoft/vcpkg.git
cd vcpkg && ./bootstrap-vcpkg.sh && ./vcpkg integrate install

# 安裝常用套件
./vcpkg install fmt boost-asio nlohmann-json gtest catch2

# CMake 整合
# cmake -B build -S . -DCMAKE_TOOLCHAIN_FILE=/path/to/vcpkg/scripts/buildsystems/vcpkg.cmake
```

```cmake
# CMakeLists.txt 中使用 vcpkg 安裝的套件
find_package(fmt CONFIG REQUIRED)
find_package(GTest CONFIG REQUIRED)
target_link_libraries(my_app PRIVATE fmt::fmt GTest::gtest_main)
```

### Conan 簡介

```ini
# conanfile.txt
[requires]
fmt/10.2.1
gtest/1.14.0

[generators]
CMakeDeps
CMakeToolchain
```

## 8.3 單元測試 (GoogleTest / Catch2)

### GoogleTest 範例

```cpp
#include <gtest/gtest.h>

// 被測函數
int add(int a, int b) { return a + b; }

TEST(AddTest, PositiveNumbers) {
    EXPECT_EQ(add(2, 3), 5);
    EXPECT_EQ(add(0, 0), 0);
}

TEST(AddTest, NegativeNumbers) {
    EXPECT_EQ(add(-2, -3), -5);
}

// Test Fixture — 共享設定
class CalculatorTest : public ::testing::Test {
protected:
    void SetUp() override {
        // 每個測試前執行
    }
    void TearDown() override {
        // 每個測試後執行
    }
};

TEST_F(CalculatorTest, BasicOperation) {
    EXPECT_EQ(add(1, 1), 2);
}
```

### 常用 GoogleTest 巨集

| 巨集 | 用途 |
|------|------|
| `EXPECT_EQ(a, b)` | 相等 (非致命) |
| `ASSERT_EQ(a, b)` | 相等 (致命，失敗後跳過) |
| `EXPECT_TRUE(cond)` | 布林為真 |
| `EXPECT_THROW(stmt, exc)` | 預期拋出例外 |
| `EXPECT_NEAR(a, b, tol)` | 浮點近似 |
| `TEST_F(Fixture, Name)` | 使用 Fixture 的測試 |

## 8.4 靜態分析與消毒器 (Sanitizers)

### clang-tidy 常用檢查

```bash
# 執行 clang-tidy
clang-tidy src/*.cpp -- -std=c++20 -Iinclude

# .clang-tidy 配置
# Checks: 'modernize-*,performance-*,readability-*,cppcoreguidelines-*'
# modernize-use-auto, modernize-use-nullptr, performance-unnecessary-value-param
```

### Sanitizers 使用

```cmake
# CMake 啟用消毒器
if(CMAKE_CXX_COMPILER_ID MATCHES "Clang|GNU")
    option(ENABLE_ASAN "Enable AddressSanitizer" OFF)
    option(ENABLE_TSAN "Enable ThreadSanitizer" OFF)
    option(ENABLE_UBSAN "Enable UndefinedBehaviorSanitizer" OFF)

    if(ENABLE_ASAN)
        add_compile_options(-fsanitize=address -fno-omit-frame-pointer)
        add_link_options(-fsanitize=address)
    endif()
endif()
```

| Sanitizer | 檢測內容 | 開銷 |
|:---:|------|:---:|
| **ASan** | 堆疊緩衝區溢出、use-after-free、記憶體洩漏 | ~2x |
| **TSan** | 資料競爭 (data race) | ~5-10x |
| **UBSan** | 未定義行為 (整數溢出、nullptr 解引用) | ~1.2x |
| **LSan** | 記憶體洩漏 (LeakSanitizer，常與 ASan 整合) | 低 |

### Valgrind 基礎

```bash
# 記憶體檢查
valgrind --leak-check=full --show-leak-kinds=all ./my_app

# 快取分析
valgrind --tool=cachegrind ./my_app
```
