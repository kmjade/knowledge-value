# 09 — 效能最佳化 (Performance Optimization)

> 🎯 **學習目標**：掌握編譯器最佳化標誌 (-O2/-O3/LTO/PGO)、快取友好設計、內聯展開、Google Benchmark 基準測試、perf/VTune 效能剖析。

## 9.1 編譯器最佳化標誌

### GCC/Clang 最佳化等級

| 標誌 | 說明 | 適用場景 |
|:---:|------|------|
| `-O0` | 無最佳化 (預設) | 除錯 |
| `-O1` | 基本最佳化，不增加編譯時間 | 受限環境 |
| `-O2` | 標準最佳化 (非空間-速度取捨) | **生產環境首選** |
| `-O3` | 激進最佳化 (含迴圈向量化、內聯) | 數值計算密集 |
| `-Os` | 最佳化程式碼大小 | 嵌入式/快取敏感 |
| `-Ofast` | `-O3` + 非標準兼容最佳化 | 科學計算 (謹慎) |

### LTO & PGO (Link-Time Optimization & Profile-Guided Optimization)

```cmake
# CMake 啟用 LTO
set(CMAKE_INTERPROCEDURAL_OPTIMIZATION ON)  # -flto
# 或手動: target_compile_options(my_app PRIVATE -flto)

# PGO 三步驟
# Step 1: 插入 profiling 代碼
# g++ -fprofile-generate -o my_app main.cpp
# Step 2: 用代表性工作負載執行
# ./my_app < typical_input.txt
# Step 3: 重新編譯 (使用收集到的 profile)
# g++ -fprofile-use -o my_app main.cpp
```

```bash
# 常用編譯標誌組合
g++ -std=c++20 -O3 -march=native -flto -Wall -Wextra \
    -DNDEBUG \           # 關閉 assert
    -ffast-math \         # 放寬浮點精度要求
    -funroll-loops \      # 展開小迴圈
    main.cpp -o my_app
```

## 9.2 快取友好設計 (Cache-Friendly Design)

### 資料佈局：AoS vs SoA

```cpp
#include <vector>
#include <cstddef>

// ❌ AoS (Array of Structures) — 快取不友好
struct Particle_AoS {
    float x, y, z;    // 12 bytes
    float vx, vy, vz; // 12 bytes
    float mass;        // 4 bytes → 28 bytes，padding → 32 bytes
};
// 遍歷 mass 時載入整條 cache line (64B) 卻只用 4B

// ✅ SoA (Structure of Arrays) — 快取友好
struct Particles_SoA {
    std::vector<float> x, y, z;
    std::vector<float> vx, vy, vz;
    std::vector<float> mass;
};
// 遍歷 mass: cache line 全是有效資料，SIMD 友好
```

### 快取行對齊與 False Sharing

```cpp
#include <new>  // std::hardware_destructive_interference_size

// ❌ False Sharing — 兩個執行緒修改同一 cache line
struct BadCounter {
    alignas(64) std::atomic<int64_t> a;  // 強制對齊到快取行
    std::atomic<int64_t> b;              // 與 a 共享 cache line！
};

// ✅ 使用 C++17 對齊避免 False Sharing
struct alignas(64) GoodCounter {
    std::atomic<int64_t> a;
    // padding to next cache line (C++17)
    char _pad[std::hardware_destructive_interference_size - sizeof(std::atomic<int64_t>)];
    std::atomic<int64_t> b;
};
```

### 常見快取最佳化模式

| 模式 | 說明 |
|------|------|
| **迴圈交換** | 讓最內層迴圈遍歷連續記憶體 |
| **迴圈合併** | 合併相同範圍的獨立迴圈，減少 cache miss |
| **區塊化 (Tiling)** | 矩陣乘法分塊，使資料留在 cache |
| **預取** | `__builtin_prefetch()` 手動暗示硬體預取 |
| **False Sharing 消除** | 多執行緒寫入變數分散到不同 cache line |

## 9.3 內聯展開 (Inlining)

```cpp
// 隱式內聯：定義在 class 內部的成員函數
struct Widget {
    int get() const { return value_; }  // 隱式 inline
    int value_;
};

// 顯式內聯
inline int square(int n) { return n * n; }

// 編譯器啟發式判斷要點：
// - -O2 自動內聯小函數 (約 < 10 條指令)
// - -O3 更激進內聯
// - 虛擬函數通常不內聯 (除非 devirtualization 成功)
// - 遞迴函數不內聯
// - __attribute__((noinline)) 阻止內聯
// - __attribute__((always_inline)) 強制內聯 (謹慎)
```

## 9.4 Google Benchmark 基準測試

```cpp
#include <benchmark/benchmark.h>
#include <vector>
#include <algorithm>

// 基礎基準測試
static void BM_VectorPushBack(benchmark::State& state) {
    for (auto _ : state) {
        std::vector<int> v;
        v.reserve(state.range(0));  // 預分配避免干擾
        for (int i = 0; i < state.range(0); ++i) {
            v.push_back(i);
        }
        benchmark::DoNotOptimize(v);  // 防止編譯器消除
    }
}
BENCHMARK(BM_VectorPushBack)->Range(8, 8 << 10);  // 8 → 8192

// 使用 fixture
class SortFixture : public benchmark::Fixture {
public:
    void SetUp(const benchmark::State&) override {
        data.resize(10000);
        std::generate(data.begin(), data.end(), std::rand);
    }
    std::vector<int> data;
};

BENCHMARK_F(SortFixture, StdSort)(benchmark::State& st) {
    auto copy = data;
    for (auto _ : st) {
        std::sort(copy.begin(), copy.end());
        benchmark::DoNotOptimize(copy);
    }
}

BENCHMARK_MAIN();
```

### perf 效能剖析速查

```bash
# 取樣 CPU 效能計數器
perf record -g ./my_app          # 記錄 call graph
perf report                       # 互動式查看
perf stat -e cache-misses,cache-references,branches,branch-misses ./my_app

# 熱點分析
perf top                          # 即時查看
perf annotate                     # 原始碼級熱點
```
