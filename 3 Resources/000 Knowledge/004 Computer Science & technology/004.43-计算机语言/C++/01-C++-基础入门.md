## 1.1 C++ 標準演進 (C++ Standard Evolution)

### 關鍵版本特性速覽

| 標準 | 年 | 旗艦特性 |
|:---:|:---:|------|
| **C++11** | 2011 | auto, nullptr, lambda, move semantics, smart pointers, `std::thread`, constexpr, range-for, enum class, `=default/delete`, initializer_list |
| **C++14** | 2014 | generic lambda (`auto` param), `decltype(auto)`, relaxed constexpr, `std::make_unique`, binary literals (`0b1010`), digit separators (`1'000'000`) |
| **C++17** | 2017 | `if constexpr`, structured bindings (`auto [a,b]`), CTAD, fold expressions, `std::optional/variant/any`, `std::string_view`, inline variables, nested namespaces |
| **C++20** | 2020 | Concepts, Ranges, Coroutines, Modules, `<=>` spaceship operator, `std::span`, `<format>`, `constexpr` new/vector, `std::jthread` |
| **C++23** | 2023 | `std::expected`, `std::mdspan`, deducing `this`, `if consteval`, `import std`, `std::flat_map/set`, monadic `std::optional` operations |

### 標準選擇對照

```cpp
// 不同標準下的寫法差異

// C++98: 冗長
std::vector<int>::iterator it = vec.begin();
for (; it != vec.end(); ++it) { /* ... */ }

// C++11: range-for + auto
for (auto& x : vec) { /* ... */ }

// C++17: structured bindings
for (auto& [key, val] : map) { /* ... */ }

// C++20: ranges
for (auto& x : vec | std::views::filter([](int i){ return i > 0; })) { /* ... */ }
```

## 1.2 編譯四階段 (Compilation Pipeline)

### 完整流程

```
source.cpp ──[Preprocessor]──▶ source.i ──[Compiler]──▶ source.s ──[Assembler]──▶ source.o ──[Linker]──▶ a.out
    巨集展開                       IR/語法樹                      組合語言                        目標碼                         可執行檔
```

### 各階段詳解

| 階段 | 指令 | 輸入 | 輸出 | 工作內容 |
|------|------|:---:|:---:|------|
| **Preprocessor (預處理)** | `g++ -E` | `.cpp/.hpp` | `.i` | `#include` 展開、`#define` 巨集替換、`#ifdef` 條件編譯、刪除註解 |
| **Compiler (編譯)** | `g++ -S` | `.i` | `.s` | 語法分析 (Parsing)、語意分析、中間碼 (IR) 生成、優化、組合語言生成 |
| **Assembler (組譯)** | `g++ -c` | `.s` | `.o` | 組合語言 → 機器碼、符號表生成 |
| **Linker (連結)** | `g++ *.o` | `.o/.a/.so` | `a.out` | 符號解析 (symbol resolution)、重定位 (relocation)、函式庫合併 |

### 實戰演示

```cpp
// demo.cpp
#include <iostream>
#define PI 3.14159

int main() {
    std::cout << "PI = " << PI << std::endl;
    return 0;
}
```

```bash
# 1. 預處理：觀察巨集展開
g++ -E demo.cpp -o demo.i
# demo.i 中 PI 已被替換為 3.14159，#include <iostream> 展開為數萬行

# 2. 編譯到組合語言
g++ -S demo.cpp -o demo.s
# demo.s 包含 x86-64 組合語言指令

# 3. 組譯為目標檔
g++ -c demo.cpp -o demo.o

# 4. 連結為可執行檔
g++ demo.o -o demo

# 5. 檢視符號表
nm demo.o          # 未解析符號 (U)
objdump -t demo.o  # 完整符號表
```

### One Definition Rule (ODR)

```cpp
// header.hpp
#ifndef HEADER_HPP_
#define HEADER_HPP_
inline int global_counter = 0;   // C++17 inline variable — 解決 ODR
// 非 inline 的全域變數只能定義於一個 TU
```

## 1.3 基本型別系統 (Basic Type System)

### 基礎型別完整表

| 型別 | 大小 (典型) | 範圍 | 字面值後綴 | 備註 |
|------|:---:|------|:---:|------|
| `bool` | 1 byte | `true` / `false` | — | |
| `char` | 1 byte | -128 ~ 127 (signed) or 0~255 | `'a'` | 實作定義 signedness |
| `signed char` | 1 byte | -128 ~ 127 | — | |
| `unsigned char` | 1 byte | 0 ~ 255 | — | |
| `short` | 2 bytes | -32,768 ~ 32,767 | — | |
| `unsigned short` | 2 bytes | 0 ~ 65,535 | — | |
| `int` | 4 bytes | -2^31 ~ 2^31-1 | — | |
| `unsigned int` | 4 bytes | 0 ~ 2^32-1 | `42u` / `42U` | |
| `long` | 4/8 bytes | 平台相關 | `42L` | Windows:4, Linux64:8 |
| `long long` | 8 bytes | -2^63 ~ 2^63-1 | `42LL` | C++11 保證 |
| `unsigned long long` | 8 bytes | 0 ~ 2^64-1 | `42ULL` | |
| `float` | 4 bytes | ±3.4e±38 (~7 digits) | `3.14f` / `3.14F` | IEEE 754 |
| `double` | 8 bytes | ±1.7e±308 (~15 digits) | `3.14` (default) | IEEE 754 |
| `long double` | 8/10/12/16 bytes | 平台相關 | `3.14L` | x87:10, SSE:8 |
| `wchar_t` | 2/4 bytes | 平台相關 | `L'文'` | |
| `char8_t` (C++20) | 1 byte | UTF-8 | `u8'字'` | |
| `char16_t` | 2 bytes | UTF-16 | `u'字'` | |
| `char32_t` | 4 bytes | UTF-32 | `U'字'` | |

### 易錯陷阱

```cpp
// 陷阱 1: 整數除法 vs 浮點除法
double x = 5 / 2;       // x = 2.0  ← 整數除法！
double y = 5.0 / 2;     // y = 2.5  ← 正確

// 陷阱 2: unsigned 陷阱
std::vector<int> v{1,2,3};
for (unsigned i = 0; i < v.size() - 4; ++i)  // v.size()-4 = 18446744073709551615!
    std::cout << i;                            // 超大正數 → 死循環/越界
// 正確做法：使用 int 或 ssize_t (C++20)

// 陷阱 3: 字面值型別
auto a = 3.14;          // double (預設)
auto b = 3.14f;         // float
auto c = 42;            // int
auto d = 42L;           // long
```

## 1.4 auto 與 decltype (Type Deduction)

### auto 推導規則完整表

| 宣告形式 | 推導規則 | 範例 |
|------|------|------|
| `auto x = expr` | 剝離參考性、頂層 const (by value) | `const int& r = 42; auto x = r;` → `int` |
| `auto& x = expr` | 保留參考性、底層 const | `auto& x = r;` → `const int&` |
| `const auto& x = expr` | 強制 const 參考 | `const auto& x = 42;` → `const int&` |
| `auto&& x = expr` | 轉發參考 (universal ref) | 可繫結左值/右值 |
| `decltype(auto) x = expr` | 完美轉發型別 (C++14) | 保留所有 CV/ref qualifiers |

```cpp
// auto 推導示範
const int ci = 42;
const int& cri = ci;

auto a = ci;            // int          — 頂層 const 剝離
auto& b = ci;           // const int&   — 保留底層 const
auto&& c = ci;          // const int&   — 左值 → 左值參考
auto&& d = 42;          // int&&        — 右值 → 右值參考
decltype(auto) e = ci;  // const int    — 完整保留
decltype(auto) f = cri; // const int&   — 完整保留
decltype(auto) g = 42;  // int          — 完整保留
```

### decltype 用法

```cpp
// decltype(expr) — 不計算 expr，只取得型別
int x = 0;
decltype(x)   y = 1;    // int
decltype((x)) z = x;    // int&  ← 注意！括號會產生左值參考

// decltype 用於泛型返回型別推導 (C++11 trailing return)
template<typename T, typename U>
auto add(T&& t, U&& u) -> decltype(t + u) {
    return t + u;
}
// C++14 後可簡化為 auto
```

### Structured Bindings (C++17)

```cpp
// 結構化繫結 — 一次解構多個值
std::pair<int, std::string> p{42, "hello"};
auto [num, str] = p;           // num: int, str: std::string

std::map<int, std::string> m{{1, "a"}, {2, "b"}};
for (auto&& [key, val] : m) {  // key: const int, val: std::string&
    std::cout << key << ": " << val << "\n";
}

struct Point { double x, y, z; };
Point pt{1.0, 2.0, 3.0};
auto [px, py, pz] = pt;        // px,py,pz: double
```

## 1.5 Namespace (命名空間)

### 基本用法

```cpp
namespace math {
    constexpr double PI = 3.14159265358979323846;
    
    double sin(double x) { /* ... */ return 0.0; }
    double cos(double x) { /* ... */ return 0.0; }
    
    namespace detail {          // 巢狀 namespace
        void helper() { /* ... */ }
    }
}  // namespace math (不需分號)

// C++17 巢狀 namespace 簡寫
namespace app::core::engine {   // 等同於 namespace app { namespace core { namespace engine {
    void init() { /* ... */ }
}  // namespace app::core::engine

// 使用方式
using math::sin;                // using declaration — 匯入單一符號
using namespace math;           // using directive — 匯入整個 namespace (慎用！)
```

### 匿名 namespace 與 inline namespace

```cpp
// 匿名 namespace — file-local linkage (取代 static)
namespace {
    int internal_value = 0;     // 僅限此 TU 可見
    void helper() { /* ... */ }
}

// inline namespace — 版本管理 (C++11)
namespace lib {
    inline namespace v2 {
        void foo();             // 預設版本
    }
    namespace v1 {
        void foo();             // 舊版 API
    }
}
lib::foo();    // 呼叫 v2::foo() — inline 自動穿透
lib::v1::foo(); // 顯式呼叫舊版
```

## 1.6 引用 vs 指標 (Reference vs Pointer)

### 完整對照表

| 特性 | Reference (`T&`) | Pointer (`T*`) |
|------|:---:|:---:|
| 是否可為空 | ❌ 必須初始化 | ✅ 可為 `nullptr` |
| 重新綁定 | ❌ 不可變更參照對象 | ✅ 可指向不同對象 |
| 語法 | `ref.member` | `ptr->member`, `(*ptr).member` |
| sizeof | 等於被參考型別大小 | 固定 8 bytes (64-bit) |
| 底層實現 | 通常等同指標（語法糖） | 記憶體位址 |
| 支援 ++/-- | 操作被參考對象 | 指標算術 |
| 使用場景 | 函數參數、回傳值、for-each | 動態記憶體、可選值、陣列走訪 |

```cpp
void demo_ref_vs_ptr() {
    int x = 42;
    
    // Reference (參考) — 別名
    int& ref = x;               // 必須初始化
    ref = 100;                  // 修改 x
    // ref = y;                 // ❌ 不是重新綁定，是賦值！
    
    // Pointer (指標) — 儲存位址
    int* ptr = &x;              // 取得位址
    *ptr = 200;                 // 解參考 (dereference)
    int y = 0;
    ptr = &y;                   // ✅ 可重新指向
}

// 何時使用哪個？
void func(const BigObject& obj);  // ✅ 避免複製 → 用 const&
void func(BigObject* ptr);        // ✅ 可選參數 → 用指標 (可為 nullptr)
void func(BigObject& obj);        // ⚠️  輸出參數 → 用 & (需明確賦值語意)
```

### 右值參考 (Rvalue Reference) (C++11)

```cpp
int&& rref = 42;              // 右值參考 — 繫結到 temporary
// int&& rref2 = x;          // ❌ 不能綁定左值
int&& rref3 = std::move(x);  // ✅ std::move 轉換為右值

// 用途: Move semantics, Perfect forwarding (見 07 章)
```

## 1.7 const 與 constexpr

### const 的三種用途

| 用途 | 語法 | 語意 |
|------|------|------|
| 常數變數 | `const int x = 42;` | 不可修改的物件 |
| const 參考 | `const T& ref = obj;` | 不可透過此參考修改 |
| const 成員函數 | `int get() const;` | 保證不修改成員變數 (`this` 為 `const T*`) |

```cpp
class Widget {
    int value_;
public:
    int get() const {          // const 成員函數
        // value_ = 0;         // ❌ 編譯錯誤！
        return value_;
    }
    
    // mutable 允許在 const 函數中修改
    mutable int access_count = 0;
    int cached_get() const {
        ++access_count;        // ✅ mutable 許可
        return value_;
    }
};
```

### constexpr — 編譯期常數 (C++11/14/17/20)

| 標準 | constexpr 能力 | 說明 |
|:---:|------|------|
| C++11 | 函數只能含一條 return 語句 | `constexpr int f() { return 42; }` |
| C++14 | 放寬為多語句、區域變數、循環 | `constexpr int f(int n) { int s=0; for(int i=1;i<=n;++i)s+=i; return s; }` |
| C++17 | lambda 可 constexpr | `auto f = []() constexpr { return 42; };` |
| C++20 | `constexpr new`、`constexpr vector`、`constexpr string` | 編譯期動態記憶體！ |
| C++20 | `consteval` — 強制編譯期 (即時函數) | `consteval int f(int n) { return n*n; }` |
| C++23 | `if consteval` — 分支判斷 | `if consteval { /* 編譯期路徑 */ } else { /* 執行期路徑 */ }` |

```cpp
// constexpr vs const
const     int runtime_val = rand();   // ✅ 執行期常數
constexpr int compile_val = 42;       // ✅ 編譯期常數 (必須是 literal type)

// constexpr 函數 — 可編譯期也可執行期
constexpr int factorial(unsigned n) {
    return n <= 1 ? 1 : n * factorial(n - 1);
}
constexpr int f5 = factorial(5);  // 120 — 編譯期計算
int f10 = factorial(10);          // 3628800 — 可選執行期計算

// consteval (C++20) — 強制編譯期
consteval int square(int n) { return n * n; }
constexpr int s = square(10);     // ✅ 編譯期
// int s2 = square(runtime_val); // ❌ 編譯錯誤！runtime_val 非 constexpr

// if consteval (C++23)
constexpr int maybe_const(int n) {
    if consteval {
        return n * 2;             // 編譯期路徑 (constexpr 友善)
    } else {
        return n * 3;             // 執行期路徑
    }
}
```

### const 指標與頂層/底層 const

```cpp
// 從右向左讀：
const int* p1;         // pointer to const int — 指向的內容不可改
int const* p2;         // 同上 (等價寫法)
int* const p3 = &x;    // const pointer to int — 指標本身不可改
const int* const p4 = &x; // const pointer to const int — 兩者皆不可改

// 頂層 const (top-level): 物件本身 const → int* const
// 底層 const (low-level):  指向的物件 const → const int*
```
