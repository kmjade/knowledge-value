# 06 幾何處理 — Geometry Processing

> 3D 網格的表示、操作、簡化與細分，是 CG 資產管線的基石。

---

## 6.1 網格表示 Mesh Representation

| 資料結構 | 說明 | 優缺點 |
|------|------|------|
| **Triangle Mesh** | 頂點 + 索引陣列 (Face List) | 最通用，GPU 原生格式 |
| **Half-Edge** | 每邊儲存兩條有向半邊 | 高效鄰接查詢（細分、簡化） |
| **Winged-Edge** | 每邊儲存 4 個鄰接邊指標 | 歷史經典，記憶體消耗大 |
| **Directed Edge** | Half-Edge 變體 | Blender BMesh 使用 |
| **B-Rep** (Boundary Representation) | 面、邊、頂點層級拓撲 | CAD 標準 (STEP/IGES) |

---

## 6.2 細分曲面 Subdivision Surfaces

| 方法 | 類型 | 連續性 | 適用 |
|------|:---:|:---:|------|
| **Catmull-Clark** | 四邊形面 | C²（規則區）、C¹（奇異點） | 動畫角色（Pixar 標準） |
| **Loop** | 三角形 | C²（規則）、C¹（奇異點） | 三角網格細分 |
| **Doo-Sabin** | 任意 | C¹ | 較少使用 |
| **$\sqrt{3}$-Subdivision** | 三角形 | C² | 每步三角形數×3 |

> **Pixar OpenSubdiv**: 工業級 GPU 加速細分曲面庫，支援特徵邊界/摺痕。

---

## 6.3 LOD (Level of Detail) 細節層次

| 策略 | 說明 |
|------|------|
| **Discrete LOD** | 預先產出多級網格，按距離切換 |
| **Continuous LOD (CLOD)** | 即時調整三角形數 |
| **HLOD (Hierarchical LOD)** | 場景層級合併（遠距離整個建築→單一網格） |
| **Nanite** (UE5) | 虛擬化幾何：Cluster-based，GPU-driven 自動 LOD（百萬→數億三角形） |

---

## 6.4 網格簡化 Mesh Simplification

| 演算法 | 原理 |
|------|------|
| **Quadric Error Metrics (QEM)** | Garland-Heckbert：每次邊折疊 (Edge Collapse) 最小化二次誤差 |
| **Vertex Clustering** | 空間網格合併頂點，快速但品質低 |
| **Progressive Meshes** | Hoppe：儲存邊折疊歷史，可雙向導航精度 |
| **Mesh Decimation** (Blender) | 組合 Collapse / Un-Subdivide / Planar Dissolve |

---

## 6.5 網格優化 Mesh Optimization

| 操作 | 目的 |
|------|------|
| **Remeshing** | 改善三角形品質（等邊、均勻大小） |
| **Smoothing** (Laplacian/Taubin) | 去除噪音，保持體積 |
| **Hole Filling** | 修補掃描網格中的破洞 |
| **Parameterization** | 網格 → 2D 映射（UV 展開的數學基礎） |
| **Boolean Operations** | CSG 布林（聯集/交集/差集），需要穩健的網格布林引擎 |
