# 03 光柵化與渲染管線 — Rasterization & Rendering Pipeline

> 即時圖形的核心：GPU 如何將 3D 三角形轉換為螢幕上的像素。

---

## 3.1 GPU 渲染管線階段 Pipeline Stages

| 階段 | 可程式化 | 功能 |
|------|:---:|------|
| **1. Input Assembler (IA)** | ❌ | 讀取頂點緩衝 (Vertex Buffer)、索引緩衝 (Index Buffer)，組裝三角形 |
| **2. Vertex Shader (VS)** | ✅ | 逐頂點變換：Model→View→Projection，輸出 Clip-space 座標 |
| **3. Tessellation** (可選) | ✅ | Hull Shader → Tessellator → Domain Shader，動態細分曲面 |
| **4. Geometry Shader (GS)** (可選) | ✅ | 逐圖元操作，可產生/刪除幾何體 |
| **5. Rasterizer** | ❌ | 三角形 → 片段 (Fragments)，執行 clipping、背面剔除、透視校正 |
| **6. Fragment Shader (FS/Pixel Shader)** | ✅ | 逐片段著色：紋理採樣、光照計算、輸出顏色 |
| **7. Output Merger (OM)** | ❌ | 深度測試 (Depth Test)、模板測試 (Stencil)、混合 (Blending) |

> **Vulkan/DX12 管線** 更細粒度：可自訂 Rasterization State、Depth-Stencil State、Blend State。

---

## 3.2 光柵化原理 Rasterization

| 步驟 | 說明 |
|------|------|
| **掃描線轉換 Scanline Conversion** | 判斷哪些像素被三角形覆蓋 |
| **重心座標插值 Barycentric Interpolation** | 計算像素在三角形內的參數（顏色、UV、法線） |
| **透視校正 Perspective Correction** | 螢幕空間線性插值 ≠ 世界空間線性插值；需除以 $w$ |

---

## 3.3 深度與模板測試 Depth & Stencil

| 技術 | 說明 |
|------|------|
| **Z-Buffer (Depth Buffer)** | 儲存每個像素的最近深度值，實現遮蔽 |
| **Early-Z / Hi-Z** | 在 Fragment Shader 之前拒絕被遮擋片段，節省計算 |
| **Reverse-Z** | 反轉深度範圍 (1→0 near→far)，提高遠距離精度 |
| **Stencil Buffer** | 8-bit 遮罩，用於陰影體積 (Shadow Volume)、貼花 (Decal) |

---

## 3.4 剔除 Culling

| 類型 | 說明 |
|------|------|
| **Frustum Culling** 視錐剔除 | CPU 端丟棄視錐外的物體 |
| **Back-Face Culling** 背面剔除 | 丟棄背向相機的三角形（預設 CW 為背面） |
| **Occlusion Culling** 遮擋剔除 | 使用 Hi-Z 或硬體查詢跳過被完全遮擋的物體 |
| **Portal / PVS Culling** | 預計算可見性集合（室內場景經典方法） |

---

## 3.5 混合 Blending

| 模式 | 公式 | 效果 |
|------|------|------|
| **Alpha Blend** | $C_{out} = \alpha \cdot C_{src} + (1-\alpha) \cdot C_{dst}$ | 半透明 |
| **Additive** | $C_{out} = C_{src} + C_{dst}$ | 光暈、爆炸 |
| **Multiplicative** | $C_{out} = C_{src} \cdot C_{dst}$ | 陰影、染色 |
| **Premultiplied Alpha** | 預乘 Alpha 避免邊緣黑邊 | 業界推薦 |
