# 01 文檔處理導論

## 定義
文檔處理 (Document Processing) 是使用計算機技術**創建、編輯、格式化、管理、轉換與發佈**文檔的完整流程。涵蓋從純文本編輯到專業印刷品的全光譜。

## 歷史演進 History
| 年代 | 里程碑 | 關鍵技術 |
|:-----|:-----|------|
| 1868 | 打字機發明 | Sholes & Glidden 機械打字機 |
| 1960s | 早期文字處理 | RUNOFF (MIT CTSS)、IBM MT/ST |
| 1979 | WYSIWYG 誕生 | WordStar → WordPerfect → MS Word |
| 1985 | 桌面出版革命 | Aldus PageMaker + Apple LaserWriter |
| 1990s | 標記語言興起 | HTML (1991) → XML (1998) → DocBook |
| 2000s | 開放標準 | ODF (2005) / OOXML (2008) / EPUB |
| 2010s | 雲端協作 | Google Docs / Notion / Obsidian |
| 2020s | AI 輔助寫作 | GPT 系列 / Copilot / 智能校對 |

## 文檔生命週期 Document Lifecycle
```
創作 (Author) → 編輯 (Edit) → 審閱 (Review) → 格式化 (Format) → 轉換 (Convert) → 發佈 (Publish) → 歸檔 (Archive)
```

## 應用領域 Application Domains
| 領域 | 典型工具 | 產出格式 |
|------|------|------|
| 辦公行政 | MS Word / Google Docs | DOCX / PDF |
| 學術出版 | LaTeX / Overleaf | PDF / DVI |
| 技術寫作 | Markdown / AsciiDoc / reST | HTML / PDF / man |
| 新聞出版 | InDesign / QuarkXPress | PDF / print-ready |
| 法律 | WordPerfect / 模板引擎 | DOCX / PDF |
| 軟體工程 | 純文本編輯器 + Git | .md / .rst / .adoc |

## 關鍵二分法 Key Dichotomies
- **WYSIWYG (所見即所得) vs WYSIWYM (所見即所思)**: Word vs LaTeX
- **二進制格式 vs 純文本標記**: DOCX (ZIP+XML) vs Markdown
- **離線 vs 雲端**: Word Desktop vs Google Docs
- **桌面出版 vs 文字處理**: InDesign vs Word — 前者精確控制版面，後者專注內容流
