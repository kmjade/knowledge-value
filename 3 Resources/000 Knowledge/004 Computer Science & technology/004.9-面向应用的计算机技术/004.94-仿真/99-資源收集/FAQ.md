# 仿真 FAQ

### 仿真 vs 解析解 (Analytical Solution) 何時用哪個？
解析解: 簡單系統 (M/M/1 排隊公式) — 精確且即時。仿真: 複雜系統 (非指數分佈、多資源互動) — 當解析解不存在或不現實時。

### DES vs ABM vs SD 如何選擇？
- **DES**: 流程導向、實體流動、資源約束 → 製造/物流/服務業
- **ABM**: 個體行為、異質互動、湧現 → 流行病/市場/人群
- **SD**: 全域反饋、存量流量、政策分析 → 宏觀經濟/人口/環境

### 仿真結果如何確保可信？
V&V (Verification & Validation): Verification = 建對模型 (代碼正確)，Validation = 建對的模型 (符合真實)。方法包括: 極端條件測試、敏感性分析、歷史數據比對、專家審查。

### 仿真需要運行多少次？
隨機仿真需多次重複 (Replication) 以獲得統計顯著性。樣本量: n ≥ (z·σ/ε)²。實務上通常 30–100 次 (取決於變異性)。使用 Welch 移動平均或 batch means 進行穩態分析。

### 數位孿生就是 3D 可視化嗎？
否。3D 可視化是表面，核心是**即時數據雙向同步 + 分析與控制閉環**。孿生 = 數據 + 模型 + 連接 + 服務，非僅展示層。

### 學習仿真的最佳路徑？
理論 → 系統動力學/Vensim (入門) → SimPy/DES (編程) → AnyLogic/Simulink (工業級) → 依領域深入 (機器人: Gazebo，物理: MuJoCo，數位孿生: Azure DT)
