# 03 離散事件仿真 Discrete-Event Simulation (DES)

## DES 核心要素
仿真系統由以下元素構成，狀態僅在**事件 (Event)** 發生時刻改變：

| 元素 | 英文 | 描述 |
|------|------|------|
| 事件 | Event | 系統狀態變化的瞬間 (到達、離開、故障) |
| 實體 | Entity | 在系統中流動的對象 (工件、客戶、封包) |
| 隊列 | Queue | 實體等待服務的緩衝區 |
| 資源 | Resource | 提供服務的設施 (機器、服務員、CPU) |
| 活動 | Activity | 資源服務實體的時間段 |
| 時鐘 | Simulation Clock | 跟踪仿真時間的變量 |

## 兩大世界觀 Simulation World Views

| 世界觀 | 推進方式 | 優點 | 缺點 |
|--------|----------|------|------|
| **事件排程 (Event Scheduling)** | 從未來事件列表 (FEL) 取出最近事件執行 | 靈活、效率高 | 模型邏輯分散 |
| **流程互動 (Process Interaction)** | 實體流經一系列流程/區塊 | 直觀、模組化 | 需要流程管理 |
| **活動掃描 (Activity Scanning)** | 每個時間步掃描所有活動條件 | 簡單 | 效率低，少用 |

## 排隊論基礎 Queueing Theory Basics

| 概念 | 符號 | 描述 |
|------|------|------|
| 到達率 | λ (lambda) | 單位時間實體到達數 |
| 服務率 | μ (mu) | 單位時間服務完成數 |
| 利用率 | ρ = λ/μ | 資源繁忙比例 (<1 穩定) |
| 平均隊列長度 | Lq | Little's Law: L = λW |
| Kendall 記號 | A/S/c | Arrival/Service/Channels (M/M/1, M/G/c) |

## 典型案例

| 場景 | 建模重點 | 輸出指標 |
|------|----------|----------|
| **製造產線** | 機器故障 MTBF/MTTR、設置時間 | 吞吐量、WIP、利用率 |
| **物流中心** | AGV 調度、揀貨策略 | 訂單週期時間、延遲率 |
| **醫院急診** | 分流級別、資源分配 | 等待時間、資源利用率 |
| **呼叫中心** | 來電分佈、技能路由 | 服務水平 SL、平均等待 |
