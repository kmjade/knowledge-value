# 04 — Obsidian 插件體系

## 插件架構總覽

| 類別 | 數量 | 說明 |
|------|:---:|------|
| **核心插件 Core** | 53+ | 內建官方插件，可自由開關 |
| **社群插件 Community** | 2000+ | 第三方開發，需關閉安全模式安裝 |
| **自定義插件** | 無限 | 自行開發（Obsidian API） |

## 核心插件精選 Core Plugins (53+)

| 類別 | 核心插件 | 預設 | 說明 |
|------|----------|:---:|------|
| **導航** | 檔案總管、搜尋、快速切換、書籤 | ✅ | 核心 UI |
| **編輯** | 頁面預覽、筆記重構、模板 | ⚠️ | 部分需手動開啟 |
| **連結** | 反向鏈接、出向鏈接、圖譜視圖 | ✅ | 雙向連結核心 |
| **組織** | 標籤面板、大綱、星標 | ✅ | 內容組織 |
| **發布** | Publish、Sync | ❌ | 付費服務 |
| **格式** | Markdown 格式轉換、筆記導入 | ⚠️ | 遷移工具 |
| **其他** | 錄音機、命令面板、幻燈片 | ⚠️ | 輔助功能 |

> 設定 → 核心插件 → 按需開關。不需要的可以關閉以簡化界面。

## 插件安全考量

| 風險 | 防範措施 |
|------|----------|
| **惡意代碼** | 優先選擇 GitHub Stars > 500 的插件 |
| **過時插件** | 定期檢查「需要更新」標記 |
| **性能影響** | 只裝真正需要的插件，不要超過 50 個 |
| **資料外洩** | 閱讀插件權限，避免裝可疑插件 |
| **相容衝突** | 新增插件後測試核心流程 |
| **Vault 崩潰** | 裝新插件前先備份 `.obsidian` 目錄 |

> 💡 使用 **Sandbox Vault**（設定 → 關於 → 沙箱倉庫）安全測試新插件。
