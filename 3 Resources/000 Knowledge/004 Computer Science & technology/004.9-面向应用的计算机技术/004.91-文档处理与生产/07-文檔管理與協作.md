# 07 文檔管理與協作

## 定義
文檔管理 (Document Management) 涵蓋文檔的**存儲、版本控制、權限、檢索與生命週期管理**。協作 (Collaboration) 則聚焦多人同步編輯與審閱。DMS (Document Management System) 與 ECM (Enterprise Content Management) 是企業級實施。

## DMS vs ECM
| 維度 | DMS | ECM |
|------|------|------|
| 範圍 | 文檔為中心 | 企業全內容 (文檔、網頁、郵件、影像) |
| 功能 | 版本/權限/簽入簽出 | 內容生命週期 + 業務流程 |
| 代表 | SharePoint / Alfresco | OpenText / IBM FileNet |
| 適用 | 中小團隊 | 大型企業 |

## Git 用於文檔管理 Git for Docs
| 優點 | 挑戰 |
|------|------|
| 精確版本追蹤 (commit/diff/blame) | 非技術人員學習曲線 |
| 分支/合併 → 平行編輯 | 二進制文檔 (DOCX/PDF) diff 困難 |
| PR/MR 審閱流程 | 合併衝突解決 |
| CI/CD 自動化發佈 (GitHub Pages) | .git 目錄膨脹 |

## 即時協作技術 Real-time Collaboration
| 技術 | 原理 | 代表產品 |
|------|------|------|
| OT (Operational Transformation) | 操作轉換：將並行編輯轉換為一致狀態 | Google Docs (Google Wave 遺產) |
| CRDT (Conflict-free Replicated Data Types) | 無衝突複製數據類型：數學保證最終一致性 | Notion / Obsidian Sync / Yjs |
| Lock-based | 段落鎖定、編輯後解鎖 | Microsoft 365 (部分) |
| 差異同步 (Diff Sync) | 定期 diff + 合併 | Etherpad → Hedgedoc |

## 協作平台生態 Collaboration Platforms
| 平台 | 核心技術 | 最佳場景 |
|------|------|------|
| Google Docs | OT (即時) | 團隊寫作/試算/簡報 |
| Notion | CRDT + 區塊 | 知識庫/專案管理 |
| Obsidian | 本地優先 + CRDT Sync | 個人知識管理 |
| Microsoft 365 | 混合 (部分即時+鎖定) | 企業辦公 |
| Overleaf | OT (基於 ShareLaTeX) | 學術 LaTeX 協作 |
| Confluence | 頁面鎖定+版本 | 企業 Wiki |

## DMS 核心功能 Core DMS Functions
| 功能 | 說明 |
|------|------|
| 版本控制 (Versioning) | 主版本 1.0, 2.0；次版本 1.1, 1.2 |
| 簽入/簽出 (Check-in/out) | 防止並行編輯衝突 |
| 元數據 (Metadata) | 作者/日期/標籤/自訂屬性 |
| 全文檢索 (Full-text Search) | OCR + 索引 |
| 權限管控 (ACL) | 角色基礎: 讀取/編輯/刪除/管理 |
| 審計追蹤 (Audit Trail) | 誰在何時做了何事 |
| 保留策略 (Retention Policy) | 法規合規 — GDPR/SOX |
