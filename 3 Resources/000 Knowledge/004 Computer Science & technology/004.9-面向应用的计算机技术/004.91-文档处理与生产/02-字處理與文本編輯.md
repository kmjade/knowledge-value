# 02 字處理與文本編輯

## 定義
字處理 (Word Processing) 專注於**內容創作、編輯與基本格式化**，以 WYSIWYG 為主導範式。文本編輯 (Text Editing) 操作純文本，強調精確控制與可程式化。

## 主要字處理軟體比較 Word Processors
| 特性 | MS Word | LibreOffice Writer | Apple Pages | Google Docs |
|------|:------:|:------:|:------:|:------:|
| 格式 | DOCX (OOXML) | ODF (.odt) | .pages (ZIP) | 雲端原生 |
| 授權 | 商業授權 | LGPL/MPLv2 | 免費 (Apple) | 免費增值 |
| 平台 | Win/Mac/Web | 全平台 | Mac/iOS/Web | Web/全平台 |
| 協作 | SharePoint/OneDrive | 有限 | iCloud | 即時 (OT) |
| 巨集 (Macro) | VBA | LibreOffice Basic | AppleScript | Google Apps Script |
| 最佳用途 | 企業辦公 | 開源/跨平台 | Apple 生態 | 團隊協作 |

## 純文本編輯器 Pure Text Editors
| 編輯器 | 特點 | 學習曲線 |
|------|------|:--:|
| Vim / Neovim | 模態編輯、無限插件、終端原生 | 陡峭 |
| Emacs | 可程式化編輯器 (Elisp)、Org-mode | 陡峭 |
| VS Code | IDE 級編輯、LSP/擴展生態、Git 集成 | 平緩 |
| Sublime Text | 極速、多游標、Python API | 平緩 |
| Notepad++ | 輕量、Windows 原生 | 極低 |

## WYSIWYG vs Markup 之爭
| 維度 | WYSIWYG (Word) | 標記語言 (Markdown/LaTeX) |
|------|------|------|
| 學習曲線 | 平緩 (所見即所得) | 中等 (需學語法) |
| 版本控制 | 困難 (二進制 diff) | 優秀 (純文本 git diff) |
| 自動化 | 有限 (VBA/Open XML) | 優秀 (Pandoc/腳本) |
| 精確控制 | 有限 (自動排版) | 精確 (LaTeX) |
| 協作 | 雲端即時 (Google Docs) | Git 工作流 |
| 格式一致性 | 良好 | 極佳 (模板分離) |

## 核心概念
- **Style (樣式)**: 段落/字元樣式實現內容與格式分離 — 標題 Heading 1-9
- **Section (節)**: 文檔分區，不同頁面設置
- **Template (模板)**: .dotx (Word) / .ott (LibreOffice) — 可重複使用的文檔骨架
- **Track Changes (追蹤修訂)**: 協作審閱核心 — 接受/拒絕變更
