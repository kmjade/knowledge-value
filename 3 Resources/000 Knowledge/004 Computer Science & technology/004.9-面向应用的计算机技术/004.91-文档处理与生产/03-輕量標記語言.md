# 03 輕量標記語言

## 定義
輕量標記語言 (Lightweight Markup Languages) 以**人類可讀的純文本語法**表達文檔結構，可經處理器轉換為 HTML/PDF/EPUB 等多種格式。核心理念: Write in plain text, publish anywhere.

## 主要標記語言比較
| 特性 | Markdown | reStructuredText | AsciiDoc | Org-mode |
|------|------|------|------|------|
| 誕生 | 2004 (Gruber) | 2002 (Python 社群) | 2002 (Stuart Rackham) | 2003 (Emacs) |
| 標準 | CommonMark(2014) | docutils 規範 | Asciidoctor 規範 | Emacs 內建 |
| 表格 | 簡單/擴展 | 網格/簡單表 | 完整表格 | 表格編輯器 |
| 交叉引用 | 有限 (GFM 連結) | 完整 (:ref:) | 完整 (<< >>) | 強大 (internal links) |
| 數學公式 | 擴展 (MathJax) | LaTeX math | MathJax 內建 | LaTeX 支援 |
| 擴展性 | 有限 (插件) | 指令系統 | 巨集/Include | Elisp 無限 |
| 最佳用途 | 部落格/README | Python 文檔 | 技術書籍 | 筆記/GTD/學術 |

## Markdown 生態系 Markdown Ecosystem
| 方言 | 特性 | 用途 |
|------|------|------|
| CommonMark | 嚴格規範、無歧義語法 | 標準基礎 |
| GFM (GitHub Flavored) | 表格、任務清單、刪除線 | GitHub 生態 |
| MDX | JSX 嵌入 Markdown | React 文檔/網站 |
| R Markdown | 嵌入 R 程式碼塊、Knitr | 數據科學報告 |
| MyST (Jupyter Book) | 科學寫作、交叉引用 | 學術/Jupyter |

## 語法對照 Syntax Comparison
| 功能 | Markdown | reST | AsciiDoc |
|------|------|------|------|
| 標題 | `# H1 ## H2` | `===` / `---` 底線 | `= H1 == H2` |
| 粗體 | `**bold**` | `**bold**` | `*bold*` |
| 代碼 | `` `code` `` | ` `` ``code`` `` `` ` | `` `code` `` |
| 列表 | `- item` | `- item` | `* item` |
| 連結 | `[text](url)` | `` `text <url>`_ `` | `link:url[text]` |
| 引用 | `> quote` | 縮排段落 | `> quote` 或 blockquote |
| 註腳 | `[^1]` (擴展) | `[1]_` | `footnote:[text]` |

## 為什麼選擇輕量標記語言
1. **版本控制友好**: 純文本 → git diff 清晰
2. **格式分離**: 內容 vs 呈現 (CSS/模板)
3. **工具鏈豐富**: Pandoc 支援 40+ 輸入輸出格式
4. **未來證明**: 純文本永遠可讀，不受廠商鎖定
5. **自動化**: CI/CD 中即時渲染、校對、部署
