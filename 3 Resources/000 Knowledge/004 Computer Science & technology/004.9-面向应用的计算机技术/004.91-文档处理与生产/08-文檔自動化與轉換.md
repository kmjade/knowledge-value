# 08 文檔自動化與轉換

## 定義
文檔自動化 (Document Automation) 使用軟體**程式化生成、轉換與處理文檔**，消除手動重複操作。文檔轉換 (Document Conversion) 則是格式間的橋樑 — 核心工具: Pandoc。

## Pandoc — 萬能文檔轉換器 Universal Document Converter
Pandoc (John MacFarlane, 2006) — 用 Haskell 寫的命令列工具，支援 40+ 格式互轉。

| 輸入格式 | 輸出格式 |
|------|------|
| Markdown、reST、AsciiDoc、Org-mode | HTML、LaTeX/PDF、EPUB、DOCX |
| DOCX、ODT、HTML | Markdown、reST、AsciiDoc |
| LaTeX | EPUB、HTML、DOCX |
| EPUB | Markdown、DOCX |

### Pandoc 工作流 Pandoc Pipeline
```
輸入文件 → Pandoc Reader → AST (Abstract Syntax Tree) → Pandoc Writer → 輸出文件
```
關鍵: 所有轉換經由中間 AST (JSON) 表示 — 可程式化操控。
```bash
# 基本轉換
pandoc input.md -o output.pdf --pdf-engine=xelatex

# 使用模板
pandoc input.md -o output.docx --reference-doc=template.docx

# 交叉格式過濾
pandoc input.md -t json | jq '.meta' | pandoc -f json -o meta.html
```

## 模板引擎 Template Engines
| 引擎 | 語言 | 用途 |
|------|------|------|
| Jinja2 | Python | 報表/DOCX 模板 (docxtpl) |
| Mail Merge (Word) | VBA/OpenXML | 批量信件/標籤 |
| ERB | Ruby | 動態 HTML/文件 |
| Liquid | Ruby/Python | Jekyll/靜態站 |
| Go Templates | Go | Hugo 靜態站 |

## 文件自動化管道 Document Generation Pipeline
```
數據源 (CSV/DB/API)
    │
    ▼
模板引擎 (Jinja2/DocxTemplate)
    │
    ▼
中間格式 (DOCX/MD/HTML)
    │
    ▼
Pandoc 轉換
    │
    ▼
最終格式 (PDF/EPUB/HTML)
    │
    ▼
CI/CD 發佈
```

## 批量文檔處理 Batch Processing
| 工具 | 功能 |
|------|------|
| docxtpl (Python) | 基於 Jinja2 的 DOCX 模板填充 |
| python-docx | 程式化創建/修改 DOCX |
| mailmerge (Python) | Python 版 Mail Merge |
| Apache POI | Java 操作 MS Office 格式 |
| LibreOffice (headless) | `soffice --headless --convert-to pdf *.docx` |

## 文檔轉換矩陣 Conversion Matrix
| 需求 | 最佳方案 |
|------|------|
| Markdown → PDF | `pandoc --pdf-engine=xelatex` |
| DOCX → Markdown | `pandoc input.docx -t markdown -o output.md` |
| Markdown → EPUB | `pandoc input.md -o book.epub` |
| Word Mail Merge | docxtpl (Python) |
| LaTeX → Word | `pandoc` 或 `latex2rtf` |
| HTML → PDF | wkhtmltopdf / Prince XML |
