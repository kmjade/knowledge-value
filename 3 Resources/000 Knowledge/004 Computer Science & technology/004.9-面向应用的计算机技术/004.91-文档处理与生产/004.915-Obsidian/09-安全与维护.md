# 09 — 安全與維護

## Vault 健康檢查 Vault Health

| 檢查項 | 工具/方法 | 頻率 |
|--------|----------|:---:|
| **斷鏈 Broken Links** | 核心插件「反向鏈接」面板 | 每週 |
| **孤立筆記 Orphaned Notes** | Graph View 觀察無連結節點 | 每月 |
| **重複筆記** | Find orphaned files 插件 | 每月 |
| **空白筆記** | Dataview `WHERE length(file.content) < 50` | 每月 |
| **過時標籤** | Tag Wrangler 批量清理 | 每季 |
| **附件引用** | 搜尋 `![[` 檢查嵌入有效性 | 每月 |

### 斷鏈檢測與修復

```dataview
LIST
FROM ""
WHERE length(file.outlinks) = 0 AND length(file.inlinks) = 0
AND !contains(file.path, "00-MOCs")
AND !contains(file.path, "99-")
SORT file.name ASC
```

| 斷鏈類型 | 原因 | 修復方式 |
|----------|------|----------|
| **目標不存在** | 筆記已重新命名/刪除 | 更新鏈接或重建筆記 |
| **路徑變更** | 筆記移到其他資料夾 | 設定 → 自動更新鏈接 |
| **別名失效** | Frontier 的 aliases 被刪除 | 檢查 YAML frontmatter |

## 性能優化 Performance

### 影響性能的因素

| 因素 | 閾值 | 影響 | 解決方案 |
|------|:---:|------|----------|
| **檔案數量** | > 10,000 | 啟動變慢、搜尋變慢 | 歸檔舊筆記、拆分 Vault |
| **社區插件數** | > 50 | 啟動慢、UI 卡頓 | 停用不常用插件 |
| **Dataview 查詢** | > 20 個活躍查詢 | 每次開啟筆記延遲 | 減少即時查詢、使用 `LIMIT` |
| **附件大小** | > 1GB 單檔 | 渲染卡頓 | 壓縮圖片、外鏈大檔案 |
| **Graph View** | > 5000 節點 | 圖譜卡死 | 使用局部圖譜、限制深度 |
| **CSS Snippets** | > 20 個 | 渲染延遲 | 合併 snippet |

### 性能基準

| 檔案數 | 預期啟動時間 | 預期搜尋時間 |
|:------:|:-----------:|:-----------:|
| < 500 | < 1s | < 100ms |
| 500-2,000 | 1-2s | < 300ms |
| 2,000-5,000 | 2-5s | < 1s |
| 5,000-10,000 | 5-10s | 1-3s |
| > 10,000 | > 10s ⚠️ | > 3s ⚠️ |

> ⚠️ 超過 10,000 檔案建議拆分為多個 Vault，或清理歸檔。

## 社群插件安全審計 Community Plugin Security Audit

| 審查維度 | 檢查方法 | 紅旗 🚩 |
|----------|----------|---------|
| **GitHub Stars** | 查看專案頁面 | < 50 stars |
| **最近更新** | 查看 commit 歷史 | > 1 年未更新 |
| **維護者** | 查看 contributor 數量 | 單人維護且無回應 issue |
| **網路請求** | 檢查插件是否發送數據 | 向未知伺服器發送請求 |
| **檔案存取** | 檢查 manifest.json 權限 | 讀取 `.obsidian` 以外敏感路徑 |
| **依賴數量** | npm dependencies | 大量過時依賴 |
| **社群評價** | Obsidian Forum / Discord | 多個負面回報 |

### 安全審計檢查清單

```
□ GitHub Stars > 500
□ 近 3 個月有更新
□ 有 2+ 活躍維護者
□ 無可疑網路請求（僅 API 調用）
□ Obsidian 官方社群有正面討論
□ 無已知安全漏洞（CVE）
□ 通過 Obsidian 官方審核（BRAT 安裝除外）
```

## Sandbox Vault 沙箱測試

> 設定 → 關於 → 沙箱倉庫 → 打開

| 用途 | 說明 |
|------|------|
| **測試新插件** | 在沙箱中安裝測試，不影響主 Vault |
| **重現 Bug** | 最小化環境隔離問題 |
| **實驗配置** | 測試激進的設定變更 |
| **主題測試** | 快速對比不同主題效果 |
| **插件開發** | 開發者除錯環境 |

```
主 Vault              沙箱 Vault
   │                      │
   ├── 真實數據           ├── 無真實數據
   ├── 生產環境           ├── 測試環境
   ├── 謹慎操作           └── 大膽實驗
   └── 穩定版本
```

## 備份與災難恢復

| 災難場景 | 恢復方案 | 恢復時間 |
|----------|----------|:------:|
| **誤刪筆記** | `.trash/` 恢復 或 Git `checkout` | < 1 分鐘 |
| **插件衝突崩潰** | 安全模式啟動 → 逐一排查 | 5-30 分鐘 |
| **Vault 損壞** | Git `reset --hard` 或恢復備份 | 5-15 分鐘 |
| **硬碟故障** | 異地備份恢復 | 1-24 小時 |
| **勒索病毒** | 離線備份恢復 + 安全掃描 | 2-48 小時 |

### 災難恢復步驟

```
1. 保持冷靜，不要恐慌操作
2. 確認損害範圍（哪個資料夾/哪些筆記）
3. 從 .trash/ 或 Git 歷史恢復
4. 若大範圍損壞，從最新備份完整恢復
5. 啟動沙箱 Vault 驗證恢復結果
6. 檢討並改進備份策略
```

## 維護排程 Maintenance Schedule

| 頻率 | 任務 | 時間 |
|:---:|------|:---:|
| **每日** | QuickAdd 清理 Inbox | 5 分鐘 |
| **每週** | 斷鏈檢查、Git commit | 15 分鐘 |
| **每月** | 孤立筆記清理、插件更新審查、附件瘦身 | 30 分鐘 |
| **每季** | 標籤清理、Vault 結構審視、備份完整性驗證 | 1 小時 |
| **每年** | 全面歸檔、性能基線測試、備份策略回顧 | 2 小時 |

## 隱私與數據主權

| 層級 | 措施 |
|:---:|------|
| **存儲** | 所有數據本地存儲，無雲端強制上傳 |
| **傳輸** | 僅 Obsidian Sync 時 E2E 加密傳輸 |
| **插件** | 審查插件網路行為，敏感內容只用本地插件 |
| **AI** | 敏感筆記使用 Ollama 本地模型，不發送 API |
| **共享** | Publish 前雙重確認無敏感內容 |
| **合規** | GDPR-friendly：數據在你手上，隨時可刪除 |
