# 08 — AI 與自動化

## Obsidian AI 插件生態

| 插件 | 類型 | 模型 | 離線 | 價格 | 一句話 |
|------|:---:|------|:---:|:---:|--------|
| **Copilot** | 對話 | GPT/Claude/本地 | ❌ | API | ChatGPT 介面內嵌 Obsidian |
| **Smart Connections** | 語義 | OpenAI/本地 | ⚠️ | API | AI 驅動的筆記相似度推薦 |
| **Text Generator** | 生成 | GPT/Claude/本地 | ⚠️ | API | AI 寫作輔助 + 模板 |
| **Claude Code** | 終端 | Claude | ❌ | API | 終端式 AI 編碼助手 |
| **LLM-Wiki** | 編譯 | GPT/Claude | ❌ | API | 將 ram/ → wiki/ 自動編譯 |
| **Local GPT** | 本地 | Ollama/LM Studio | ✅ | 免費 | 完全本地 AI |
| **BMO Chatbot** | 對話 | 多模型 | ⚠️ | API | 側邊欄 AI 聊天 |
| **Companion** | 對話 | 多模型 | ⚠️ | API | AI 助手側邊欄 |

## Copilot 插件詳解

> 在 Obsidian 內使用 ChatGPT/Claude 對話介面。

| 功能 | 說明 |
|------|------|
| **對話模式** | 側邊欄 AI 聊天，上下文包含當前筆記 |
| **內聯模式** | 選取文字 → 快捷鍵 → AI 補全/改寫 |
| **Vault QA** | 以整個 Vault 作為上下文進行 QA |
| **自定義 Prompt** | 設定固定 System Prompt / 常用 Prompt 模板 |
| **模型選擇** | GPT-4o、Claude 3.5 Sonnet、Gemini 等 |

```markdown
# Copilot Prompt 範例

System Prompt:
你是我的 PKM 助手，擅長將零散筆記整理成結構化知識。
回答時使用繁體中文，提供清晰的層級結構。

Quick Prompt:
- /summarize → 總結當前筆記
- /expand → 擴展當前段落
- /outline → 為當前主題生成大綱
- /link → 建議當前筆記的潛在鏈接
```

## Claude Code 整合工作流

> Obsidian Vault + Claude Code CLI = AI 驅動的知識庫維護。

| 步驟 | Claude Code 操作 |
|:----:|-----------------|
| 1 | `claude` 在 Vault 根目錄啟動 |
| 2 | `/init` 建立 CLAUDE.md 定義工作規範 |
| 3 | `/compile` 將 ram/ 原始素材編譯為 wiki/ |
| 4 | `/lint` 檢查斷鏈、格式一致性 |
| 5 | `/review` 生成定期知識庫回顧報告 |

```markdown
## CLAUDE.md 範例

# Obsidian Vault — AI 助手規範
- 語言: 繁體中文（技術術語保留 English）
- 不得修改 ram/ 內的原始素材
- 所有編譯產出放在 wiki/
- 遵循 Zettelkasten 原子筆記原則
- 每筆 commit 附帶清晰的變更說明
```

## LLM-Wiki 方法論

> 「人類維護 ram，AI 編譯 wiki」的雙層架構。

```
Vault/
├── ram/                   ← 人類領域（AI 只讀）
│   ├── articles/          剪藏的文章
│   ├── conversations/     AI 對話記錄
│   └── notes/             原始筆記
│
├── wiki/                  ← AI 領域（AI 獨佔寫入）
│   ├── concepts/          概念頁（23+）
│   ├── entities/          實體頁（26+）
│   ├── sources/           來源溯源
│   ├── index.md           知識索引
│   └── log.md             編譯日誌
│
└── CLAUDE.md              ← 編譯規範
```

| 操作 | 觸發 | 動作 |
|------|------|------|
| **編譯** | `/compile` | 掃描 ram/ → 萃取概念 → 生成 wiki/concepts/ |
| **關聯** | `/lint` | 檢查斷鏈 → 建議新鏈接 → 更新索引 |
| **回顧** | `/review` | 分析變更歷史 → 生成知識增長報告 |

## Text Generator 模板

```markdown
---
prompt: "將以下筆記總結為 3 個關鍵要點：{{notes}}"
model: gpt-4o
output: replace
---

# AI 總結

{{output}}
```

| 模板變數 | 說明 |
|----------|------|
| `{{title}}` | 當前筆記標題 |
| `{{content}}` | 當前筆記全文 |
| `{{selection}}` | 選取的文字 |
| `{{tags}}` | 當前筆記標籤 |
| `{{folder}}` | 所在資料夾 |

## Smart Connections 語義關聯

| 功能 | 說明 |
|------|------|
| **語義相似度** | Embedding 向量相似 → 推薦相關筆記 |
| **自動標籤** | AI 建議標籤 |
| **智能搜索** | 語義搜尋（非僅關鍵詞匹配） |
| **本地 Embedding** | 支援 Ollama 本地模型，隱私保護 |

## AI 寫作工作流

```
[人類草稿] → Copilot /expand → [AI 擴展]
           → Copilot /summarize → [AI 摘要]
           → Text Generator → [AI 重寫]
           → Smart Connections → [關聯推薦]
           → 人類審閱 → [最終定稿]
```

## AI 使用原則與倫理

| 原則 | 說明 |
|------|------|
| **Human-in-the-loop** | AI 輔助，人類決策；永遠保留審閱權 |
| **出處標記** | AI 生成內容標註為 `🤖 AI-assisted` |
| **隱私優先** | 敏感內容使用本地模型（Ollama） |
| **版本控制** | AI 變更透過 Git 追蹤，可復原 |
| **增量改善** | 不一次性大量生成，逐步迭代 |
