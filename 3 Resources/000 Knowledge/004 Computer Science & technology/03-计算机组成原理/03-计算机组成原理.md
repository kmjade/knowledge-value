# 03-计算机组成原理

> 计算机组成原理研究计算机硬件如何执行程序——从晶体管到指令集，从寄存器到缓存层次。

## 从代码到执行

```
高级语言 (Python/Java/C++)
        ↓ 编译器/解释器
    汇编语言
        ↓ 汇编器
    机器码 (二进制指令)
        ↓ CPU 执行
    硬件操作 (晶体管开关)
```

## CPU 核心组件

### 指令周期

```
取指 (Fetch) → 译码 (Decode) → 执行 (Execute) → 访存 (Memory) → 写回 (Write Back)
      ↑_______________________________________________________________↓
                              循环
```

### 流水线 (Pipelining)

将指令周期各阶段重叠执行——像工厂流水线：

```
时间 →
指令1: F | D | E | M | W |
指令2:     F | D | E | M | W |
指令3:         F | D | E | M | W |
```

> 理想流水线：每时钟周期完成一条指令。实际受限于流水线冒险（数据冒险、控制冒险）。

### 流水线冒险

| 冒险类型 | 问题 | 解决 |
|----------|------|------|
| **数据冒险** | 后一条指令需要前一条的结果 | 前推 (Forwarding)、流水线暂停 |
| **控制冒险** | 分支指令不知道跳不跳 | 分支预测 (Branch Prediction) |
| **结构冒险** | 硬件资源冲突 | 增加硬件资源 |

## 内存层次结构

```
速度 ↑      容量 ↓      成本 ↑
┌────────────────────────────┐
│ 寄存器 (Registers)     ~1ns │ ← CPU 内部
├────────────────────────────┤
│ L1 Cache   (32-64KB)  ~1ns │
├────────────────────────────┤
│ L2 Cache  (256-512KB) ~4ns │ ← SRAM
├────────────────────────────┤
│ L3 Cache   (4-32MB)  ~12ns │
├────────────────────────────┤
│ 主存 RAM   (8-64GB)  ~100ns│ ← DRAM
├────────────────────────────┤
│ SSD         (256GB-4TB)    │
├────────────────────────────┤
│ HDD         (1-20TB)       │
└────────────────────────────┘
```

### 缓存的核心原理

| 原理 | 含义 |
|------|------|
| **时间局部性** | 刚访问过的数据很可能再次访问 |
| **空间局部性** | 刚访问过的地址附近的数据很可能被访问 |

> 这就是为什么数组遍历比链表遍历快得多——数组内存连续，充分利用空间局部性和缓存行。

### 缓存行 (Cache Line)

缓存以固定大小的块（通常 64 字节）为单位读取和存储。即使你只读一个 int (4 字节)，整个 64 字节会被加载到缓存。

**伪共享 (False Sharing)**：两个线程修改同一缓存行的不同变量，导致缓存一致性问题——性能杀手。

## 指令集架构 (ISA)

ISA 是软件和硬件之间的"合同"。

| ISA | 类型 | 特点 | 应用 |
|-----|------|------|------|
| **x86-64** | CISC (复杂指令集) | 指令变长、功能丰富 | PC、服务器 |
| **ARM** | RISC (精简指令集) | 低功耗、定长指令 | 手机、Mac (Apple Silicon) |
| **RISC-V** | RISC (开源) | 免费、可定制 | 嵌入式、学术、新兴服务器 |

### CISC vs RISC

| 特性 | CISC (x86) | RISC (ARM, RISC-V) |
|------|-----------|-------------------|
| 指令长度 | 变长 (1-15 字节) | 定长 (4 字节) |
| 指令种类 | 多，一条指令做很多事 | 少，一条指令做一件事 |
| 内存访问 | 指令可直接访问内存 | 只有 Load/Store 指令访问内存 |
| 典型 | "我做得多，你别管细节" | "我做得简单，编译器来优化" |

> 现代 x86 CPU 内部将 CISC 指令翻译为类似 RISC 的微操作执行——结合了两者的优势。

## 数字表示

| 类型 | 说明 |
|------|------|
| **整数** | 补码表示 (two's complement)，范围：-2ⁿ⁻¹ ~ 2ⁿ⁻¹-1 |
| **浮点数** | IEEE 754：符号位 + 指数 + 尾数。0.1 + 0.2 ≠ 0.3（精度问题） |
| **字符** | ASCII → UTF-8 (变长编码，兼容 ASCII) |
