# 08 使用者與權限 Users & Permissions

> Linux 是多使用者系統。UID/GID 為使用者及群組的身份標識，sudo 提供特權委派，PAM 為認證框架，SELinux/AppArmor 提供強制存取控制 (MAC)。

## UID / GID 體系

| UID 範圍 | 用途 |
|------|------|
| 0 | root (超級使用者) |
| 1-999 | 系統帳號 (daemon, www-data, nobody) |
| 1000+ | 一般使用者 (Ubuntu 預設從 1000 開始) |

```bash
id                              # 查看當前使用者 UID/GID
id username                     # 查看特定使用者
whoami                          # 當前使用者名稱
cat /etc/passwd                 # 使用者帳號資料庫
cat /etc/group                  # 群組資料庫
```

## sudo / su 特權委派

| 指令 | 作用 |
|------|------|
| `sudo command` | 以 root 身份執行單一指令 |
| `sudo -i` | 開啟 root shell (模擬登入) |
| `sudo -u username command` | 以指定使用者身份執行 |
| `su -` | 切換到 root (需要 root 密碼) |
| `visudo` | 安全編輯 `/etc/sudoers` |

```bash
# /etc/sudoers 範例
username ALL=(ALL:ALL) ALL              # 完全 sudo 權限
%admin ALL=(ALL) ALL                    # admin 群組
username ALL=(ALL) NOPASSWD: /usr/bin/systemctl restart nginx  # 免密碼特定指令
```

## PAM 認證框架 Pluggable Authentication Modules

| PAM 模組 | 用途 |
|------|------|
| `pam_unix.so` | 傳統密碼認證 |
| `pam_tally2.so` | 登入失敗鎖定 |
| `pam_faillock.so` | 現代失敗鎖定 (取代 tally2) |
| `pam_limits.so` | 資源限制 (ulimit) |
| `pam_sss.so` | SSSD 整合 (LDAP/Kerberos) |

```bash
# 設定檔位置
/etc/pam.d/                     # 各服務的 PAM 設定
/etc/pam.d/sshd                 # SSH 認證設定
/etc/security/limits.conf       # 資源限制
```

## SELinux vs AppArmor

| 特性 | SELinux | AppArmor |
|------|------|------|
| 預設 Distro | RHEL, Fedora | Ubuntu, openSUSE |
| 控制粒度 | inode 級 (極細) | 路徑級 (較簡單) |
| 策略管理 | 複雜 (type enforcement) | 直觀 (profile) |
| 運行模式 | Enforcing / Permissive | Enforce / Complain |

```bash
# SELinux
getenforce                              # 查看模式
setenforce 0                            # 切換 Permissive (暫時)
semanage fcontext -l | grep httpd       # 查看檔案上下文
restorecon -Rv /var/www                 # 恢復預設上下文

# AppArmor
aa-status                               # 查看狀態
aa-complain /path/to/bin                # 切換 complain 模式
aa-enforce /path/to/bin                 # 強制模式
```

## ACL 存取控制清單

```bash
getfacl file.txt                        # 查看 ACL
setfacl -m u:alice:rwx file.txt         # 授予使用者
setfacl -m g:dev:r-- file.txt           # 授予群組
setfacl -x u:alice file.txt             # 移除條目
setfacl -b file.txt                     # 清除所有 ACL
```

## SSH 金鑰認證

```bash
ssh-keygen -t ed25519                   # 最安全 (比 RSA 更快更小)
ssh-copy-id user@host                   # 複製公鑰
chmod 700 ~/.ssh                        # 目錄權限
chmod 600 ~/.ssh/authorized_keys        # 檔案權限

# sshd_config 安全加強
# PermitRootLogin prohibit-password     # 僅金鑰登入 (或 no)
# PasswordAuthentication no             # 完全禁用密碼
# MaxAuthTries 3
```
