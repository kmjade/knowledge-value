# 09 效能調優與排錯 Performance & Troubleshooting

> 系統效能分析從觀察開始 (top/htop)，深入指標 (vmstat/iostat)，再到行程層級 (strace/lsof) 與核心參數 (sysctl)。排錯的核心是建立假設並驗證。

## 效能監控工具棧 Monitoring Stack

| 層級 | 工具 | 用途 |
|------|------|------|
| **系統概覽** | `top`, `htop`, `btop` | CPU/記憶體/行程即時概覽 |
| **CPU** | `mpstat`, `perf` | CPU 使用率、效能計數器 |
| **記憶體** | `free`, `vmstat` | 記憶體與 swap 活動 |
| **I/O** | `iostat`, `iotop` | 磁碟讀寫統計 |
| **網路** | `ss`, `nethogs` | Socket 與頻寬 |
| **行程** | `pidstat`, `strace`, `lsof` | 行程層級分析 |

```bash
htop -p $(pgrep -d, nginx)              # 僅監控 nginx
vmstat 1 10                              # 每秒輸出，共 10 次
iostat -x 1                              # 擴展 I/O 統計
pidstat 1                                # 行程級 CPU 統計
```

## strace — 系統呼叫追蹤

```bash
strace -p <PID>                          # 附加到運行中行程
strace -c -p <PID>                       # 統計摘要 (Ctrl-C 後輸出)
strace -e trace=open,read,write ls       # 僅追蹤特定 syscall
strace -f -o trace.log ./app             # 追蹤子行程 + 輸出到檔案
strace -T -e trace=network curl google.com  # 顯示每次呼叫耗時
```

## lsof — 列出開啟檔案

```bash
lsof -p <PID>                            # 某行程開啟的檔案
lsof -i :80                              # 使用 port 80 的行程
lsof -u username                         # 某使用者開啟的檔案
lsof /var/log/syslog                     # 哪些行程正在寫此檔案
lsof +D /var/log                         # 目錄下所有開啟檔案
```

## sysctl — 核心參數調優

| 參數 | 預設 | 建議 | 用途 |
|------|:---:|:---:|------|
| `vm.swappiness` | 60 | 10 (伺服器) | 降低 swap 傾向 |
| `vm.dirty_ratio` | 20 | 10 | 髒頁比例上限 |
| `net.core.somaxconn` | 128 | 65535 | TCP 監聽隊列 |
| `net.ipv4.tcp_tw_reuse` | 0 | 1 | TIME_WAIT 重用 |
| `fs.file-max` | — | 2097152 | 全系統檔案描述符上限 |
| `kernel.pid_max` | 32768 | 4194304 | 最大 PID |

```bash
sysctl -a                                # 列出所有參數
sysctl vm.swappiness                     # 查看特定參數
sysctl -w vm.swappiness=10               # 暫時修改
echo "vm.swappiness=10" >> /etc/sysctl.d/99-custom.conf  # 永久
sysctl --system                          # 載入所有設定
```

## 日誌分析 Log Analysis

| 日誌路徑 | 內容 |
|------|------|
| `/var/log/syslog` | 系統日誌 (Debian/Ubuntu) |
| `/var/log/messages` | 系統日誌 (RHEL) |
| `/var/log/auth.log` | 認證日誌 (登入記錄) |
| `/var/log/kern.log` | 核心日誌 |
| `/var/log/nginx/` | Nginx 存取/錯誤日誌 |
| `dmesg` | 核心環形緩衝區 (開機訊息) |

```bash
tail -f /var/log/syslog                   # 即時追蹤
journalctl -f                             # systemd 日誌追蹤
dmesg -T                                  # 帶時間戳的核心訊息
dmesg | grep -i error                     # 過濾錯誤
```

## OOM Killer 排錯

```bash
dmesg | grep -i "out of memory"           # 確認 OOM 事件
dmesg | grep -i "killed process"          # 查看被殺行程
cat /proc/<PID>/oom_score                 # OOM 分數 (越高越可能被殺)
echo -1000 > /proc/<PID>/oom_score_adj    # 保護行程不被殺
```

## 效能分析黃金法則

1. **先觀察後行動**：用 `top`/`htop` 確認瓶頸 (CPU/IO/Mem)
2. **隔離變數**：一次只改一個參數
3. **建立基準**：記錄正常時的指標
4. **USE 方法論**：檢查 Utilization、Saturation、Errors
5. **從外到內**：應用層 → OS → 硬體
