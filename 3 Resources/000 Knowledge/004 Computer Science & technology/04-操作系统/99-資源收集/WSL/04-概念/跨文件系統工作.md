# 跨文件系統工作

> [!info] 說明
> 了解 WSL 中 Windows 和 Linux 檔案系統的互操作方式。

## 檔案系統架構


## 從 Linux 存取 Windows 檔案

### Windows 磁碟掛載位置

Windows 磁碟機掛載在 `/mnt/` 下：

```bash
# C 槽
cd /mnt/c/Users/username

# D 槽
cd /mnt/d/Projects

# 列出所有掛載的磁碟
ls /mnt/
# c d e
```

### 檔案路徑轉換

```bash
# Linux 路徑 → Windows 路徑
wslpath -w /mnt/c/Users
# 輸出: C:\Users

# Windows 路徑 → Linux 路徑
wslpath -u 'C:\Users\username'
# 輸出: /mnt/c/Users/username
```

### 從 Linux 開啟 Windows 檔案

```bash
# 使用預設程式開啟
explorer.exe .

# 開啟特定檔案
explorer.exe file.txt

# 使用 Windows 程式開啟
notepad.exe /mnt/c/Users/file.txt
code.exe /mnt/c/Projects
```

## 從 Windows 存取 Linux 檔案

### 網路共用路徑

Linux 檔案系統可透過 `\\wsl$\` 存取：

```
\\wsl$\Ubuntu\home\username\projects
\\wsl$\Ubuntu-22.04\var\log
```

### 在檔案總管中開啟

```bash
# 在 Linux 終端機中
explorer.exe .
# 會開啟對應的 Windows 檔案總管視窗
```

### PowerShell 存取

```powershell
# 列出 WSL 檔案
dir \\wsl$\Ubuntu\home\username

# 複製檔案
copy \\wsl$\Ubuntu\home\user\file.txt C:\temp\

# 複製檔案到 WSL
copy C:\temp\file.txt \\wsl$\Ubuntu\home\user\
```

## 效能考量

### 效能差異

| 操作 | 位置 | 效能 |
|------|------|------|
| Linux 工具存取 Linux 檔案 | `/home/user/` | ⭐⭐⭐⭐⭐ |
| Windows 工具存取 Windows 檔案 | `C:\` | ⭐⭐⭐⭐⭐ |
| Linux 工具存取 Windows 檔案 | `/mnt/c/` | ⭐⭐ |
| Windows 工具存取 Linux 檔案 | `\\wsl$\` | ⭐⭐ |

### 最佳實務

> [!tip] 效能建議
> 將專案檔案放在 Linux 檔案系統中以獲得最佳效能。

```bash
# ✅ 推薦 - 專案在 Linux 檔案系統
cd ~/projects
git clone repo.git

# ❌ 避免 - 專案在 Windows 檔案系統
cd /mnt/c/Users/username/projects
git clone repo.git
```

## 檔案權限

### DrvFs 檔案系統

Windows 磁碟在 WSL 中使用 DrvFs 掛載，權限由掛載選項決定。

```bash
# 查看掛載選項
mount | grep drvfs
# C: on /mnt/c type drvfs (rw,noatime,uid=1000,gid=1000,case=off)
```

### 掛載選項

在 `/etc/wsl.conf` 中設定：

```ini
[automount]
options = "metadata,umask=22,fmask=11"
```

| 選項 | 說明 |
|------|------|
| `metadata` | 啟用 Linux 權限儲存 |
| `umask` | 預設權限遮罩 |
| `fmask` | 檔案權限遮罩 |
| `case=off` | 不區分大小寫 |

### 設定檔案權限

```bash
# 如果啟用了 metadata
chmod +x /mnt/c/scripts/myscript.sh

# 查看權限
ls -la /mnt/c/scripts/
```

## 區分大小寫

### 檔案名稱大小寫

```bash
# Linux 預設區分大小寫
touch test.txt
touch TEST.txt  # 兩個不同的檔案

# Windows 不區分大小寫
# test.txt 和 TEST.txt 是同一個檔案
```

### 調整大小寫設定

```bash
# 在 Linux 中啟用目錄不區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\projects" disable

# 啟用區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\projects" enable
```

### WSL 設定

```ini
# /etc/wsl.conf
[automount]
options = "case=dir"
# case=off: 不區分大小寫
# case=dir: 目錄區分大小寫
# case=force: 強制區分大小寫
```

## 行尾符號

### 換行符差異

| 系統 | 換行符 |
|------|--------|
| Windows | CRLF (`\r\n`) |
| Linux | LF (`\n`) |

### Git 設定

```bash
# 設定自動轉換
git config --global core.autocrlf input

# 或停用自動轉換
git config --global core.autocrlf false
```

### 轉換工具

```bash
# dos2unix - Windows 換行轉 Linux
sudo apt install dos2unix
dos2unix file.txt

# unix2dos - Linux 換行轉 Windows
unix2dos file.txt
```

## 符號連結

### 從 Linux 建立連結

```bash
# 軟連結
ln -s /mnt/c/Projects ~/projects

# 硬連結 (僅限同一檔案系統)
ln file.txt hardlink.txt
```

### Windows 連結

```powershell
# 符號連結
cmd /c mklink /D "C:\link" "\\wsl$\Ubuntu\home\user\projects"

# 連接點
cmd /c mklink /J "C:\junction" "\\wsl$\Ubuntu\home\user"
```

## 磁碟空間管理

### 查看使用量

```bash
# Linux 檔案系統大小
df -h

# Windows 磁碟使用
df -h /mnt/c
```

### 壓縮虛擬磁碟

```powershell
# 在 Windows PowerShell 中
wsl --shutdown
diskpart
select vdisk file="C:\Users\username\AppData\Local\Packages\CanonicalGroupLimited...\LocalState\ext4.vhdx"
compact vdisk
exit
```
