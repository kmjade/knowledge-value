# 07 — 選型指南 OS Selection Guide

> 基於場景、成本與技術需求的作業系統選型決策矩陣。
> Scenario-based OS selection decision matrix with TCO analysis and technology-fit evaluation.

---

## 場景化推薦矩陣 Scenario Recommendation Matrix

| 場景 Scenario | 首選 Primary | 次選 Secondary | 不推薦 Avoid | 理由 |
|------|:---:|:---:|:---:|------|
| **企業 AD 域控** | Windows Server | — | Linux, macOS | AD DS 為 Windows 原生，無替代品 |
| **Web 伺服器 (LAMP/LEMP)** | Linux (Ubuntu/RHEL) | FreeBSD | Windows Server | 生態、效能、成本皆優 |
| **容器/K8s 集群** | Linux (Alpine/Ubuntu) | Windows Server (node only) | macOS, Android | Linux 是 K8s 原生平台 |
| **資料庫伺服器** | Linux (RHEL/Ubuntu) | Windows (SQL Server) | macOS | IO 效能與穩定性 |
| **檔案伺服器 (SMB)** | Windows Server | Linux (Samba) | macOS | SMB 原生、AD 整合 |
| **檔案伺服器 (NFS)** | Linux/Unix | Windows Server (NFS role) | — | Unix 原生協議 |
| **AI/ML 訓練** | Linux (Ubuntu) | Windows (WSL 2) | macOS | CUDA 生態、驅動支援 |
| **桌面辦公** | Windows | macOS | Linux | Office 365 相容性、周邊生態 |
| **創意設計 (影音/3D)** | macOS | Windows | Linux | Adobe/Motion/FCP、色彩管理 |
| **軟體開發 (後端)** | Linux / macOS | Windows (WSL 2) | — | Unix 工具鏈原生效能 |
| **軟體開發 (.NET)** | Windows | Linux / macOS | — | Visual Studio 原生 |
| **iOS/macOS 開發** | macOS | — | Windows, Linux | Xcode 為必需 |
| **Android 開發** | macOS / Linux | Windows | — | Android Studio 跨平台 |
| **HarmonyOS 開發** | Windows / macOS | Linux | — | DevEco Studio 平台限制 |
| **行動裝置 (手機)** | Android / iOS | HarmonyOS (China) | — | 生態成熟度 |
| **IoT 嵌入式** | Linux (Yocto/Buildroot) | Android Things / FreeRTOS | Windows IoT | 資源佔用、社群 |
| **車載系統** | Android Automotive | Linux (AGL) / QNX | — | 生態整合、認證 |
| **工業控制** | Linux (RTLinux) | QNX / VxWorks | Windows | 即時性需求 |
| **電信網路 (NFV)** | Linux (RHEL/CentOS) | — | Windows | 開源生態主導 |

## TCO 總體擁有成本分析 Total Cost of Ownership

### 授權成本 Licensing Cost (Annual Estimate for Server)

| OS | 授權類型 | 入門級 (1 server) | 中型 (10 servers) | 大型 (100+ servers) |
|------|------|:---:|:---:|:---:|
| **Windows Server Datacenter** | Core-based | $6,155+ (16-core) | ~$60,000+ | ~$600,000+ |
| **Windows Server Standard** | Core-based | $1,069+ (16-core) | ~$10,000+ | ~$100,000+ |
| **RHEL** | Subscription | $349 (self-support) / $799 (standard) | ~$8,000 | ~$80,000 |
| **Ubuntu Pro** | Subscription | $25 (personal) / $500 (server) | ~$5,000 | ~$50,000 |
| **SUSE Linux Enterprise** | Subscription | ~$400 | ~$4,000 | ~$40,000 |
| **Debian / AlmaLinux / Rocky** | Free | $0 | $0 | $0 |
| **macOS (server)** | Device-bound (Mac mini) | $599 (HW) | N/A | N/A |
| **Android** | Free (AOSP) | $0 | $0 | $0 |
| **HarmonyOS** | Free (OpenHarmony) | $0 | $0 | $0 |

> Note: TCO includes licensing + support + training + operational overhead.

### 營運成本 Operational Cost

| 成本類型 | Windows Server | Linux | Unix (commercial) | macOS Server |
|------|:---:|:---:|:---:|:---:|
| **管理員薪資 (年均)** | $75K-$130K | $85K-$150K | $100K-$160K | N/A |
| **培訓成本** | 中等 (MS Learn + cert) | 中等-高 (self-docs + RHCE/etc) | 高 (vendor training) | 低 (Apple ecosystem) |
| **自動化成熟度** | 高 (PowerShell DSC / Ansible) | 高 (Ansible / Terraform / Bash) | 中等 | 低 |
| **修補頻率** | Patch Tuesday (monthly) | 取決於發行版 (weekly/daily) | Vendor-dependent | macOS updates |
| **平均修復時間 (MTTR)** | 2-6 hours (well-known) | 1-4 hours (high skill) | 4-24 hours (vendor) | 1-4 hours |

## 遷移策略建議 Migration Strategy

| 從 From | 到 To | 難度 | 關鍵挑戰 |
|------|:---:|:---:|------|
| **Windows → Linux** | Linux | 🟡 中 | AD 替代 (FreeIPA/Samba)、PowerShell → Bash、.NET Framework → .NET Core |
| **Linux → Windows** | Windows | 🔴 高 | 授權成本、Shell 習慣、套件管理 |
| **Unix → Linux** | Linux | 🟢 低 | POSIX 相容、shell 相似、RHEL 商業支援 |
| **macOS → Linux** | Linux | 🟡 中 | Homebrew → apt/dnf、Metal → Mesa、Automator → script |
| **Android → HarmonyOS** | HarmonyOS | 🟡 中 | Java/Kotlin → ArkTS、Gradle → hvigor、Google Play → AppGallery |
| **iOS → HarmonyOS** | HarmonyOS | 🟡 中 | Swift → ArkTS、App Store → AppGallery |
