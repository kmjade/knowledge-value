# 06 — 安全模型對比 Security Model Comparison

> 六大作業系統的權限模型、沙箱機制、強制存取控制、安全啟動與應用簽名的全方位對比。
> Comprehensive comparison of permission models, sandboxing, MAC, secure boot, and app signing across six operating systems.

---

## 權限模型總覽 Permission Model Overview

| 維度 | Windows Server | Linux | Unix | macOS | Android | HarmonyOS |
|------|:---:|:---:|:---:|:---:|:---:|:---:|
| **基本模型** | ACL + Privileges | POSIX DAC + capabilities | POSIX DAC + ACL | POSIX + Sandbox entitlements | Linux DAC + Runtime permissions | 分級權限 (Graded) |
| **強制存取控制 (MAC)** | Mandatory Integrity Control (MIC) | SELinux / AppArmor | Trusted Extensions (Solaris) | SIP (System Integrity Protection) | SELinux (strict enforcing) | 分級安全 (Graded Security) |
| **最小權限** | UAC + LowBox / AppContainer | seccomp + capabilities + namespaces | RBAC (Solaris) | Sandbox profiles (seatbelt) | App sandbox (UID isolation) | 能力式最小權限 |
| **權限提升** | UAC elevation / RunAs | sudo / polkit / doas | su / sudo / pfexec | sudo / Authorization Services | ❌ (no root by default) | ❌ |
| **檔案權限** | NTFS ACL (SIDs) | UGO + ACL + xattrs | UGO + ACL | POSIX + ACL + SIP paths | UGO + SELinux labels | UGO + 安全標籤 |
| **預設 Deny** | ✅ (via AppContainer) | ✅ (via SELinux enforcing) | ❌ (permissive default) | ✅ (SIP + Sandbox) | ✅ (SELinux + Permissions) | ✅ (分級隔離) |

## 沙箱與隔離 Sandboxing & Isolation

| 維度 | Windows Server | Linux | Unix | macOS | Android | HarmonyOS |
|------|:---:|:---:|:---:|:---:|:---:|:---:|
| **應用沙箱** | AppContainer / LowBox | seccomp + namespaces | Zones / chroot | Seatbelt sandbox (App Sandbox) | UID sandbox per app | 應用沙箱 (App Sandbox) |
| **行程隔離** | Job Objects + Silos | cgroups + namespaces | Zones / Projects (Solaris) | Sandbox profiles | cgroups + namespaces + UID | 行程級隔離 |
| **網路隔離** | Windows Filtering Platform (WFP) | netfilter / nftables + network namespaces | IP Filter (Solaris) | Application Firewall (socket filter) | iptables owner match | 網路安全策略 |
| **檔案系統隔離** | NTFS ACL + Integrity levels | Mount namespaces + chroot | ZFS delegated datasets | Sandbox fs restrictions | Mount namespaces + isolated storage | 檔案隔離引擎 |
| **IPC 隔離** | ALPC security descriptors | IPC namespaces | Varies | Mach port rights | Binder security contexts | DSoftBus 安全通道 |
| **容器隔離** | Hyper-V isolation / Process isolation | Namespace + cgroup + seccomp + SELinux | Zones (native) | ❌ (VM-backed) | ❌ | ❌ |

## 強制存取控制 (MAC) 對比 Mandatory Access Control

| 系統 | MAC 框架 | 策略語言 | 預設策略 | 學習曲線 |
|------|------|------|------|:---:|
| **Windows Server** | Mandatory Integrity Control (MIC) | SDDL (Security Descriptor) | 內建 (Low/Medium/High/System) | 🟡 中等 |
| **Linux (RHEL/Fedora/Android)** | SELinux | SELinux policy (.te) | targeted / strict enforcing | 🔴 高 |
| **Linux (Ubuntu/SUSE)** | AppArmor | AppArmor profile | complain/enforce per app | 🟢 低 |
| **Solaris** | Trusted Extensions | Label-based | permissive | 🟡 中等 |
| **macOS** | SIP | Configuration profiles | 預設啟用核心保護 | 🟢 低 |
| **HarmonyOS** | 分級安全 | Graded security policies | 預設啟用 | 🟡 中等 |

## SELinux vs AppArmor vs SIP 對比

| 維度 | SELinux (Linux/Android) | AppArmor (Ubuntu/SUSE) | SIP (macOS) |
|------|------|------|------|
| **粒度** | Inode-level labeling (fine) | Path-based (coarse but practical) | File path-based + code signing |
| **策略管理** | semodule + semanage | aa-genprof + aa-enforce | csrutil (on/off + exceptions) |
| **預設狀態** | Enforcing (RHEL/Android) | Complain (Ubuntu) / Enforce (SUSE) | Enabled (rootless restrictions) |
| **日誌** | /var/log/audit/audit.log | /var/log/syslog / audit.log | Unified Log (log show) |
| **排除機制** | permissive domains | aa-complain per profile | csrutil enable --without fs/kext/debug |
| **複雜度** | 🔴 高 | 🟢 低 | 🟢 低 |
| **典型用途** | 伺服器、Android 安全模型 | 桌面/輕量伺服器 | macOS 系統完整性 |

## 安全啟動對比 Secure Boot Comparison

| 維度 | Windows Server | Linux | macOS | Android | HarmonyOS |
|------|:---:|:---:|:---:|:---:|:---:|
| **安全啟動** | Secure Boot (UEFI) + TPM 2.0 | UEFI Secure Boot + shim | Secure Boot (T2/M-series) | Verified Boot (AVB) | Verified Boot |
| **信任根** | TPM 2.0 + UEFI PK | TPM 2.0 (optional) + UEFI | Apple Secure Enclave (T2) / built-in (Mx) | Hardware-backed Keystore | TEE (Trusted Execution Environment) |
| **啟動驗證** | Measured Boot + Boot attestation | dm-verity / IMA | Boot attestation (Sealed Key) | dm-verity + AVB | 啟動完整性驗證 |
| **韌體驗證** | ✅ (UEFI Secure Boot) | ✅ (UEFI Secure Boot) | ✅ (iBoot + SecureROM) | ✅ (Bootloader lock) | ✅ |
| **遠端證明** | ✅ (Windows Defender System Guard) | ✅ (Keylime) | ❌ | ✅ (Key Attestation) | ✅ |
| **實體攻擊防護** | BitLocker + TPM PIN | LUKS + TPM | FileVault + Secure Enclave | FBE + Keystore | FBE + TEE |

## 應用簽名與分發 App Signing & Distribution

| 維度 | Windows Server | Linux | macOS | Android | HarmonyOS |
|------|:---:|:---:|:---:|:---:|:---:|
| **簽名格式** | Authenticode (PE) | GPG signature / RPM signing | Apple code signing (Mach-O) | APK Signature v4 | 鴻蒙應用簽名 |
| **憑證來源** | Code Signing Certificate (CA) | GPG key / distribution key | Apple Developer Certificate | Play App Signing / self-signed | AppGallery Connect 憑證 |
| **強制簽名** | ❌ (驅動程式: ✅) | ❌ (optional, Secure Boot for kernel) | ✅ (Gatekeeper enforcement) | ✅ (Google Play) / ❌ (sideload) | ✅ (AppGallery) |
| **簽名驗證** | WinVerifyTrust API | rpm --checksig / dpkg-sig | codesign -v / spctl | apksigner verify | 應用簽名校驗 |
| **撤銷機制** | Certificate Revocation List (CRL) | GPG key revocation | Notarization + revocation | Play Protect + remote uninstall | 遠端撤銷 |
| **時間戳** | Authenticode timestamping | ❌ (reproducible builds) | codesign --timestamp | APK Signature timestamp | ✅ |

## 威脅模型與防護 Threat Model & Protection

| 威脅類型 | Windows Server | Linux | macOS | Android | HarmonyOS |
|------|:---:|:---:|:---:|:---:|:---:|
| **惡意軟體** | Defender Antivirus + ASR | ClamAV / rkhunter | XProtect + MRT + Notarization | Play Protect + Verify Apps | 安全掃描引擎 |
| **勒索軟體** | Controlled Folder Access | 備份策略為主 | TCC + 沙箱限制 | ❌ (app sandbox 有限) | ❌ |
| **0-day 漏洞** | Defender Exploit Guard / HVCI | grsecurity / Kernel Self Protection | KASLR + SIP | Linux kernel mitigations + monthly patches | 安全熱修復 |
| **供應鏈攻擊** | Authenticode + SmartScreen | 套件簽名 + reproducible builds | Notarization + staple | Play Integrity + app signing key | 供應鏈驗證 |
| **記憶體攻擊** | CFG (Control Flow Guard) / CET | CFI (Clang) / Shadow Stack (HW) | Pointer Authentication (ARM64e) | CFI + PAC (ARM) | PAC (ARM) |
| **側通道** | Kernel VA Shadow (Meltdown) / IBPB/STIBP (Spectre) | Same kernel mitigations | Double Map (Meltdown) | Linux mitigations | 側通道防護 |
