# Unix 進程與信號 — Unix Processes & Signals

> 進程是 Unix 的執行單元，fork/exec 是其獨特的創建模式，而信號（signals）則是進程間的非同步通訊機制。

## 進程創建：fork() + exec()

Unix 創建新進程的方式獨樹一幟——先用 `fork()` 複製當前進程，再用 `exec()` 替換執行映像：

```c
pid_t pid = fork();
if (pid == 0) {
    // 子進程 Child Process
    execl("/bin/ls", "ls", "-l", NULL);  // 替換為 ls
} else if (pid > 0) {
    // 父進程 Parent Process
    wait(NULL);  // 等待子進程結束
}
```

### fork() 的特性
- **Copy-on-Write (COW)**：現代實作中，fork 不立即複製記憶體，而是共享頁面並在寫入時才複製
- **返回兩次**：父進程返回子進程 PID，子進程返回 0
- **繼承**：子進程繼承父進程的檔案描述符、環境變數、信號處理設定

### exec() 家族
| 函數 | 參數格式 | 搜尋 PATH |
|------|---------|:--------:|
| execl() | list (可變參數) | ❌ |
| execv() | vector (陣列) | ❌ |
| execlp() | list | ✅ |
| execvp() | vector | ✅ |

## 進程狀態 Process States

```
         ┌─────────┐
         │ Running │ ←→ Ready (可執行)
         └────┬────┘
              │ I/O 等待
         ┌────▼────┐
         │ Blocked  │ (睡眠中)
         └────┬────┘
              │ I/O 完成
         ┌────▼────┐
         │  Ready   │ → 排程 → Running
         └─────────┘

   進程終止：Running → Zombie → (reaped by parent)
```

### Zombie Process (殭屍進程)
子進程結束但父進程尚未呼叫 `wait()` 回收其退出狀態。殭屍進程不佔用記憶體，但佔用 PID 表槽位。若大量殭屍堆積可能耗盡 PID。

### Orphan Process (孤兒進程)
父進程先於子進程終止。孤兒進程被 init (PID 1) 收養，由 init 負責回收。

## 信號 Signals — 非同步通知

信號是 Unix 的軟體中斷機制，用於通知進程發生特定事件。

### 關鍵信號

| 信號 | 編號 | 含義 | 可否攔截 |
|------|:----:|------|:--------:|
| **SIGHUP** | 1 | 終端掛斷 (Hangup) | ✅ |
| **SIGINT** | 2 | 鍵盤中斷 (Ctrl+C) | ✅ |
| **SIGQUIT** | 3 | 鍵盤退出 (Ctrl+\) | ✅ |
| **SIGKILL** | 9 | 強制殺死 | ❌ (不可攔截) |
| **SIGTERM** | 15 | 終止請求（預設 kill） | ✅ |
| **SIGSTOP** | 19 | 暫停進程 | ❌ |
| **SIGCONT** | 18 | 繼續暫停的進程 | ✅ |
| **SIGCHLD** | 17 | 子進程狀態變更 | ✅ |
| **SIGUSR1** | 10 | 使用者定義 1 | ✅ |
| **SIGUSR2** | 12 | 使用者定義 2 | ✅ |

### 發送信號

```bash
kill -9 1234         # 強制殺死 PID 1234
kill -HUP 1234       # 發送 SIGHUP（常用於重新載入設定）
kill -TERM 1234      # 優雅終止
pkill -f "pattern"   # 按名稱模式殺死
```

### 信號處理 Signal Handling

```c
#include <signal.h>
signal(SIGINT, handler);  // 註冊處理函數 (簡單，但不可靠)
sigaction(SIGINT, &act, NULL);  // POSIX 推薦 (可靠)
```

### SIGHUP 的特殊用途
傳統終端機連接中斷時，核心向終端的所有進程發送 SIGHUP。現代守護進程（daemon）習慣以 SIGHUP 觸發設定檔重新載入（reload），因為 daemon 沒有控制終端。

## 排程優先級：nice / renice

Unix 使用優先級（priority）排程。nice 值範圍通常是 -20（最高優先級）到 +19（最低優先級），預設為 0：

```bash
nice -n 10 make          # 以低優先級執行 make
renice -5 -p 1234        # 提高 PID 1234 的優先級 (需 root)
```

## 守護進程 Daemon

Daemon 是在背景執行的系統服務進程，特徵：
1. 父進程是 init (PID 1)
2. 沒有控制終端（detached from TTY）
3. 工作目錄為 `/`
4. 關閉標準 I/O，重導向到日誌

經典 daemon：cron (排程), syslogd (日誌), inetd (網路服務), sshd (SSH 伺服器)。
