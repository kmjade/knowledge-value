# Unix 安全與權限 — Unix Security & Permissions

> Unix 從一開始就是多使用者系統，其權限模型——UID/GID、rwx、setuid——簡潔而強大，影響了後世所有作業系統的安全設計。

## UID 與 GID — 使用者與群組

每個 Unix 進程擁有一個使用者 ID (UID) 和一個或多個群組 ID (GID)，決定其對檔案和資源的存取權限：

| UID | 使用者 | 角色 |
|:---:|------|------|
| 0 | root | 超級使用者，無限制存取 |
| 1-99 | 系統帳號 | daemon, bin, sys, nobody |
| 100+ | 一般使用者 | 普通登入帳號 |

### 實體 vs 有效 UID
```c
uid_t ruid = getuid();   // 實體 UID (Real UID) — 誰啟動了進程
uid_t euid = geteuid();  // 有效 UID (Effective UID) — 權限檢查用
```

有效 UID 通常等於實體 UID，但在 setuid 程式中不同——程式以檔案擁有者的權限執行。

## rwx 權限模型

Unix 的經典三位元組權限系統（Owner/Group/Other × Read/Write/Execute）：

```
-rwxr-xr--  1 alice  staff  4096 May 30 10:00 script.sh
│├┘├┘├┘
││ │ │ └── Other: r-- (唯讀)
││ │ └──── Group: r-x (讀+執行)
││ └────── Owner: rwx (讀+寫+執行)
│
├── 檔案類型：- (普通), d (目錄), l (符號連結), c (字元裝置), b (區塊裝置)
```

### 權限的數值表示

| 八進位 | 二進位 | 權限 |
|:------:|--------|------|
| 7 | 111 | rwx |
| 6 | 110 | rw- |
| 5 | 101 | r-x |
| 4 | 100 | r-- |
| 0 | 000 | --- |

```bash
chmod 755 file   # rwxr-xr-x (目錄/程式預設)
chmod 644 file   # rw-r--r-- (檔案預設)
chmod u+x file   # 為 owner 加入執行權限
chmod g-w file   # 移除 group 的寫入權限
```

### 目錄權限的特殊含義
- **r**：可以列出目錄內容 (ls)
- **w**：可以在目錄中建立/刪除檔案
- **x**：可以進入目錄 (cd)，存取目錄中的檔案

## setuid / setgid — 特權提升

setuid bit 是 Unix 安全模型的關鍵機制——當可執行檔設定了 setuid bit，執行該程式的進程以檔案擁有者（通常是 root）的權限執行：

```bash
-rwsr-xr-x  1 root  wheel  /usr/bin/passwd   # s 表示 setuid
-rwxr-sr-x  1 root  mail   /usr/bin/write    # s 表示 setgid
```

經典 setuid 程式：
| 程式 | 擁有者 | 用途 |
|------|--------|------|
| `/usr/bin/passwd` | root | 修改 /etc/shadow 密碼檔案 |
| `/usr/bin/su` | root | 切換使用者 |
| `/usr/bin/sudo` | root | 以 root 權限執行命令 |
| `/usr/bin/ping` | root | 建立 raw socket |

### 安全風險
setuid root 程式是 Unix 安全最脆弱的環節——任何緩衝區溢位（buffer overflow）漏洞都可能導致提權。現代系統通過 capabilities (Linux) 和 sandboxing 減輕風險。

## chroot — 根目錄隔離

chroot (change root) 於 1979 年在 Version 7 Unix 引入，將進程的根目錄限制在指定路徑：

```bash
chroot /new/root /bin/sh   # Shell 的根目錄變為 /new/root
```

chroot 是現代容器技術（Docker, LXC, Solaris Zones）的鼻祖。但 chroot 並非完整的安全邊界——root 使用者可以逃離 chroot jail。

## syslog — 系統日誌

Eric Allman 在 4.2BSD 中開發的 syslog 成為 Unix 標準日誌系統：

```c
#include <syslog.h>
openlog("myapp", LOG_PID, LOG_DAEMON);         // 開啟日誌
syslog(LOG_ERR, "Error: %s", strerror(errno));  // 寫入日誌
closelog();                                     // 關閉
```

### syslog 設施與等級

**設施 (Facility)**：LOG_AUTH, LOG_DAEMON, LOG_KERN, LOG_MAIL, LOG_USER, LOG_LOCAL0-7
**等級 (Level)**：LOG_EMERG (0) → LOG_ALERT → LOG_CRIT → LOG_ERR → LOG_WARNING → LOG_NOTICE → LOG_INFO → LOG_DEBUG (7)

## 其他安全機制

| 機制 | 描述 | 來源 |
|------|------|------|
| **umask** | 設定新檔案的預設權限遮罩 | Unix v6 |
| **sticky bit** | `/tmp` 目錄上防止非擁有者刪除檔案 | Unix v7 |
| **shadow passwords** | 將密碼 hash 從 `/etc/passwd` 移到 `/etc/shadow` | SunOS 4 |
| **PAM** | 可插拔認證模組 Pluggable Authentication Modules | Sun Solaris |
