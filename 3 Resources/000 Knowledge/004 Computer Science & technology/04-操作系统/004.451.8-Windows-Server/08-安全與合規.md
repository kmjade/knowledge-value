## Microsoft Defender (防毒 + 防火牆) Antivirus & Firewall

### Defender Antivirus 設定

```powershell
# 檢查狀態
Get-MpComputerStatus | Select AntivirusEnabled, AMServiceEnabled, AntispywareEnabled

# 強制更新簽章
Update-MpSignature

# 執行全掃描
Start-MpScan -ScanType FullScan

# 設定排除
Add-MpPreference -ExclusionPath "F:\HyperV\VMs"
Add-MpPreference -ExclusionExtension ".vhd",".vhdx",".avhdx"
Add-MpPreference -ExclusionProcess "sqlservr.exe"
```

### Defender Firewall 設定 Defender Firewall

```powershell
# 啟用所有設定檔
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True

# 阻擋預設入站 (允許出站)
Set-NetFirewallProfile -DefaultInboundAction Block
Set-NetFirewallProfile -DefaultOutboundAction Allow

# 建立規則
New-NetFirewallRule -DisplayName "Allow HTTP 80" `
  -Direction Inbound -Protocol TCP -LocalPort 80 -Action Allow -Profile Domain

# 匯出/匯入規則
netsh advfirewall export "C:\backup\firewall.wfw"
```

## Credential Guard — 認證保護 Credential Protection

| 元件 Component | 保護對象 Protected | 機制 Mechanism |
|:------|------|------|
| **Credential Guard** | NTLM hash, Kerberos TGT, Credential Manager | VBS isolated LSA (LSAIso) |
| **Remote Credential Guard** | RDP 連線中的認證 | Kerberos over RDP (不傳送 hash) |
| **Protected Users Group** | 強制 Kerberos only (停用 NTLM/WDigest) | AD 群組策略 |
| **LSA Protection** | 防止未簽章 DLL 注入 LSASS | RunAsPPL 登錄旗標 |

### 啟用 Credential Guard Enable Credential Guard

```powershell
# 檢查支援 (需 TPM 2.0 / Secure Boot)
Get-ComputerInfo | Select *VirtualizationFirmwareEnabled, *SecureBoot*

# 啟用 Credential Guard (需重啟)
# 透過 GPO: Computer Config → Admin Templates → System → Device Guard
#   → Turn On Virtualization Based Security → Enabled
#   → Credential Guard Configuration → Enabled with UEFI lock

# 或透過 PowerShell + 登錄 (重啟後生效)
Enable-WindowsOptionalFeature -Online -FeatureName IsolatedUserMode -NoRestart
# 需設定 HypervisorEnforcedCodeIntegrity & LsaCfgFlags 登錄鍵

# 驗證 Credential Guard 運作中
Get-CimInstance -ClassName Win32_DeviceGuard -Namespace root\Microsoft\Windows\DeviceGuard
# 查看 SecurityServicesRunning: 應包含 1 (Credential Guard)
```

## LAPS — 本機管理員密碼解決方案 Local Admin Password Solution

| 功能 Feature | Legacy LAPS | Windows LAPS (Server 2022+) |
|:------|:---:|:---:|
| **密碼儲存 Password Storage** | AD attribute (機密) | AD + Azure AD |
| **自動輪替 Auto-rotation** | ✅ | ✅ |
| **密碼複雜度 Password Complexity** | 可設定 | 可設定 + 單字清單 |
| **密碼歷史 Password History** | ❌ | ✅ (可還原上一個密碼) |
| **DCS 權限模式 DCS Permission Mode** | ❌ | ✅ (不需委派寫入權限) |
| **加密模式 Encryption** | ❌ | ✅ (密碼加密儲存) |
| **Azure AD 支援** | ❌ | ✅ (Cloud LAPS) |

### LAPS 部署 LAPS Deployment

```powershell
# 延伸 AD Schema (僅一次)
Update-LapsADSchema

# 設定 LAPS 權限 (DCS 模式)
Set-LapsADComputerSelfPermission -Identity "OU=Servers,DC=contoso,DC=com"

# 設定 LAPS 原則 GPO
Set-LapsADReadPasswordPermission -Identity "OU=Servers,DC=contoso,DC=com" `
  -AllowedPrincipals "CONTOSO\ServerAdmins"

# 設定密碼原則 GPO
Set-LapsPolicy -PasswordLength 14 -PasswordAgeDays 30 -PostAuthenticationResetDelay 24

# 查詢密碼 (需要權限)
Get-LapsADPassword -Identity "SRV01" -AsPlainText
```

## JEA — Just Enough Administration

| 元件 Component | 說明 Description |
|:------|------|------|
| **Role Capability File (.psrc)** | 定義允許的 cmdlet/function/script |
| **Session Configuration File (.pssc)** | 定義 JEA 端點 (誰、用什麼身分) |
| **Virtual Account / gMSA** | 執行委派命令的執行身分 RunAs |

### JEA 設定範例 JEA Configuration

```powershell
# 建立 Role Capability (允許的 cmdlet)
New-PSRoleCapabilityFile -Path "C:\JEA\DnsAdmin.psrc" `
  -VisibleCmdlets "Get-DnsServerResourceRecord","Add-DnsServerResourceRecordA" `
  -VisibleFunctions "Test-Connection"

# 建立 Session Configuration
New-PSSessionConfigurationFile -Path "C:\JEA\DnsJE.pssc" `
  -SessionType RestrictedRemoteServer `
  -RunAsVirtualAccount `
  -RoleDefinitions @{ 'CONTOSO\DNSAdmins' = @{ RoleCapabilities = 'DnsAdmin' } }

# 註冊 JEA 端點
Register-PSSessionConfiguration -Name "DNS_JEA" -Path "C:\JEA\DnsJE.pssc" -Force

# 使用 JEA 端點連線
Enter-PSSession -ComputerName dns01 -ConfigurationName DNS_JEA
```

## 稽核策略 auditpol Audit Policies

| 類別 Category | 子類別 Subcategory | 事件 Event IDs |
|:------|------|------|
| **Account Logon** | Kerberos Service Ticket Operations | 4768, 4769, 4770 |
| **Account Logon** | Credential Validation | 4776 (NTLM) |
| **Account Management** | User Account Management | 4720-4740 |
| **DS Access** | Directory Service Changes | 5136-5139, 5141 |
| **Logon/Logoff** | Logon / Logoff | 4624, 4625, 4634 |
| **Privilege Use** | Sensitive Privilege Use | 4672, 4673 |
| **Policy Change** | Audit Policy Change | 4719 |

### 進階稽核設定 Advanced Audit Configuration

```powershell
# 備份稽核原則
auditpol /backup /file:C:\backup\auditpol.csv

# 啟用詳細稽核
auditpol /set /subcategory:"Logon" /success:enable /failure:enable
auditpol /set /subcategory:"Directory Service Changes" /success:enable

# PowerShell 方式
Set-GPRegistryValue -Name "Default Domain Controllers Policy" `
  -Key "HKLM\Software\Microsoft\Windows\CurrentVersion\Policies\System\Audit" `
  -ValueName "ProcessCreationIncludeCmdLine_Enabled" -Type DWord -Value 1

# 查詢目前設定
auditpol /get /category:*
```

### SCM / Security Baselines 工具鏈

```powershell
# Microsoft Security Compliance Toolkit 下載
# https://www.microsoft.com/en-us/download/details.aspx?id=55319

# LGPO.exe (Local Group Policy Object utility)
lgpo.exe /g "C:\Baselines\MSFT-WinSvr2025-MemberServer"   # 匯入 GPO 備份

# Policy Analyzer — GUI 工具，比對 GPO 差異

# PowerShell DSC 實現基線
# 使用 SecurityPolicyDsc / AuditPolicyDsc 模組
Configuration ServerBaseline {
    Import-DscResource -ModuleName SecurityPolicyDsc
    SecurityOption AccountLockoutDuration { Name = 'AccountLockoutDuration'; AccountLockoutDuration = 15 }
    SecurityOption LockoutBadCount { Name = 'LockoutBadCount'; LockoutBadCount = 5 }
}
```
