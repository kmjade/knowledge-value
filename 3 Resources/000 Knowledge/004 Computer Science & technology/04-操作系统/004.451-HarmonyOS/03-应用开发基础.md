# 03 — HarmonyOS 應用開發基礎

> HarmonyOS 應用使用 ArkTS（TypeScript 超集）開發，基於 Ability 組件模型。
> 開發工具為 DevEco Studio，應用封裝為 HAP/HSP 格式。

---

## 開發環境 DevEco Studio

| 項目 | 說明 |
|------|------|
| **基於** | IntelliJ IDEA Community Edition |
| **支援 OS** | Windows / macOS |
| **語言** | ArkTS (primary), C/C++, JavaScript |
| **模擬器** | 內建 HarmonyOS 模擬器 (Local + Remote) |
| **調試** | 支援 USB/WiFi 真機調試、分散式調試 |
| **最低版本** | 5.0+ (for NEXT 開發) |
| **SDK 管理** | 內建 SDK Manager，支援多 API Level |

---

## ArkTS 語言特性

| 特性 | 說明 |
|------|------|
| **基礎** | TypeScript 超集，完全相容 TS 語法 |
| **靜態類型** | 強類型 + 類型推導 |
| **裝飾器** | @Entry, @Component, @State, @Link, @Prop |
| **聲明式 UI** | 類 SwiftUI/Flutter 的聲明式語法 |
| **並發** | TaskPool + Worker 支援 |
| **記憶體** | 自動 GC + ArkCompiler 優化 |
| **範例** | `@Component struct MyPage { build() { ... } }` |

## Ability 應用模型

### FA Model (Feature Ability) vs Stage Model

| 維度 | FA Model (舊) | Stage Model (新) |
|------|:-----------:|:------------:|
| **引入版本** | API 8 及之前 | API 9+ |
| **Ability 類型** | PageAbility, ServiceAbility, DataAbility | UIAbility, ExtensionAbility |
| **生命週期** | 簡單回調 | 完整生命週期管理 |
| **推薦** | 逐步淘汰 | ✅ 推薦使用 |

### Stage Model 核心類型

| 類型 | 用途 | 範例 |
|------|------|------|
| **UIAbility** | 帶 UI 的用戶交互單元 | 頁面、設置頁 |
| **ExtensionAbility** | 無 UI 後台服務 | 數據共享、卡片、輸入法 |
| **WindowStage** | 窗口管理 | 多窗口、分屏 |
| **AbilityStage** | 應用級初始化 | 全局配置 |

---

## HAP / HSP 封裝格式

| 格式 | 說明 |
|------|------|
| **HAP** (HarmonyOS Ability Package) | 應用安裝包，含代碼 + 資源 + module.json5 |
| **HSP** (HarmonyOS Shared Package) | 動態共享包，運行時按需加載 |
| **HAR** (HarmonyOS Archive) | 靜態共享包，編譯時合併 |
| **APP** (Application Package) | 多 HAP 聚合包，用於應用市場分發 |

---

## module.json5 配置

| 關鍵欄位 | 說明 |
|----------|------|
| `module.name` | 模組名稱 |
| `module.type` | entry / feature / shared |
| `module.abilities[]` | 註冊 UIAbility 及 ExtensionAbility |
| `module.requestPermissions[]` | 聲明所需權限 |
| `module.deviceTypes` | 目標設備類型 (phone / tablet / wearable ...) |
| `app.bundleName` | 應用唯一標識（反向域名） |
