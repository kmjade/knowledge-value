# 05 資料與儲存 Data & Storage

> Android 提供多層次儲存方案：Room (SQLite 抽象層)、DataStore (輕量 KV)、SharedPreferences、ContentProvider 與雲端備份。
> Android provides multi-tier storage: Room (SQLite abstraction), DataStore (lightweight KV), SharedPreferences, ContentProvider, and cloud backup.

## 儲存方案對比 Storage Options

| 方案 | 資料量 | 結構 | 適合場景 | 並行安全 |
|------|:---:|:---:|------|:---:|
| **Room** | 大 | 關聯式 (SQLite) | 複雜查詢、大量結構化資料 | ✅ DAO + Flow 自動 |
| **DataStore (Proto)** | 中 | Protobuf schema | 型別安全、設定檔 | ✅ 基於 Coroutines |
| **DataStore (Preferences)** | 小 | Key-Value | 簡單設定 (取代 SharedPrefs) | ✅ |
| **SharedPreferences** | 小 | Key-Value XML | 舊專案兼容 | ⚠️ 主執行緒阻塞 |
| **File Storage** | 任意 | Raw bytes | 圖片、緩存 | ⚠️ 手動管理 |
| **ContentProvider** | 任意 | URI-based CRUD | 跨 App 共用 | ✅ |

## Room 資料庫

```kotlin
@Entity(tableName = "users")
data class User(
    @PrimaryKey(autoGenerate = true) val id: Int = 0,
    @ColumnInfo(name = "name") val name: String,
    @ColumnInfo(name = "age") val age: Int
)

@Dao
interface UserDao {
    @Query("SELECT * FROM users ORDER BY name ASC")
    fun getAll(): Flow<List<User>>

    @Insert(onConflict = OnConflictStrategy.REPLACE)
    suspend fun insert(user: User)

    @Delete
    suspend fun delete(user: User)
}

@Database(entities = [User::class], version = 1)
abstract class AppDatabase : RoomDatabase() {
    abstract fun userDao(): UserDao
}
```

## DataStore 遷移 (from SharedPreferences)

```kotlin
// DataStore Preferences 建立
val Context.dataStore by preferencesDataStore(name = "settings")

// 讀取
val themeFlow: Flow<String> = context.dataStore.data.map { prefs ->
    prefs[THEME_KEY] ?: "system"
}

// 寫入
suspend fun setTheme(theme: String) {
    context.dataStore.edit { prefs ->
        prefs[THEME_KEY] = theme
    }
}
```

## WorkManager 背景任務

| 特性 | 說明 |
|------|------|
| **保證執行** | 即使 App 退出或裝置重啟，WorkManager 保證任務最終完成 |
| **約束條件** | Network, Battery, Idle, Storage 等多種約束 |
| **鏈式任務** | `beginWith(A).then(B).then(C)` |
| **定期任務** | `PeriodicWorkRequestBuilder` (min 15 min) |
| **觀察狀態** | `workInfoLiveData` / `getWorkInfoByIdFlow()` |

```kotlin
val uploadWork = OneTimeWorkRequestBuilder<UploadWorker>()
    .setConstraints(Constraints.Builder()
        .setRequiredNetworkType(NetworkType.CONNECTED)
        .build())
    .build()
WorkManager.getInstance(context).enqueue(uploadWork)
```
