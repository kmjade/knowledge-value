# 08 安全與權限 Security & Permissions

> Android 安全模型涵蓋沙箱隔離、執行時權限、Keystore 硬體加密、生物識別驗證與 Play Integrity 完整性校驗。
> Android security model covers sandbox isolation, runtime permissions, Keystore hardware encryption, biometric authentication, and Play Integrity verification.

## 權限模型 Permission Model

| 權限類型 | 說明 | 範例 |
|------|------|------|
| **Install-time (Normal)** | 安裝時自動授予 | `INTERNET`, `ACCESS_NETWORK_STATE` |
| **Runtime (Dangerous)** | 執行時彈窗請求 | `CAMERA`, `LOCATION`, `READ_CONTACTS` |
| **Signature** | 僅同簽名 App 可取得 | 系統級 API、自訂權限 |
| **Special (AppOps)** | 須在 Settings 中手動開啟 | `SYSTEM_ALERT_WINDOW`, `WRITE_SETTINGS` |

## Runtime 權限請求流程

```kotlin
// Compose 中請求權限
val permissionState = rememberPermissionState(
    android.Manifest.permission.CAMERA
)

Button(onClick = {
    when {
        permissionState.status.isGranted -> { /* 已授權 */ }
        permissionState.status.shouldShowRationale -> { /* 解釋為何需要 */ }
        else -> { permissionState.launchPermissionRequest() }
    }
}) { Text("Request Camera") }

// Manifest 宣告
// <uses-permission android:name="android.permission.CAMERA"/>
// <uses-feature android:name="android.hardware.camera" android:required="false"/>
```

## 安全架構 Security Architecture

| 層級 | 機制 | 說明 |
|------|------|------|
| **Linux UID 隔離** | 每個 App 獨立 UID | App 無法讀取其他 App 的私有檔案 |
| **SELinux** | MAC 強制存取控制 | Enforcing 模式，拒絕所有未允許操作 |
| **App 簽名** | APK Signature Scheme v3/v4 | 防止 APK 篡改，金鑰輪替支援 |
| **Keystore** | 硬體支援金鑰庫 | TEE/StrongBox 內存放金鑰，不可匯出 |
| **File-based Encryption** | FBE (Direct Boot) | 開機前後分層加密 |
| **Verified Boot** | dm-verity / AVB | 確保系統分割區未修改 |

## R8 / ProGuard 混淆

```properties
# proguard-rules.pro
-keep class com.example.myapp.model.** { *; }  # 保留 data class
-keepattributes Signature                       # 保留泛型
-keep class * extends androidx.room.RoomDatabase
-dontwarn okhttp3.**
```

| 工具 | 說明 |
|------|------|
| **R8** | Android 預設，整合 shrink + obfuscation + optimization |
| **ProGuard** | 舊版混淆工具，R8 已取代 |
| **DexGuard** | 商業版，增強混淆與 Runtime 防護 |

## Play Integrity / SafetyNet

| API | 說明 | 狀態 |
|------|------|:---:|
| **SafetyNet Attestation** | 檢查裝置完整性、CTS 通過 | ⚠️ 已棄用 (2024-2025 逐步淘汰) |
| **Play Integrity API** | 取代 SafetyNet，更細緻的完整性判斷 | ✅ 活躍 |
| **Hardware Attestation** | TEE 硬體證明，無法偽造 | ✅ 強制 (Android 8+) |

## 生物識別 Biometric

```kotlin
val biometricManager = BiometricManager.from(context)
when (biometricManager.canAuthenticate(BIOMETRIC_STRONG)) {
    BiometricManager.BIOMETRIC_SUCCESS -> {
        val promptInfo = BiometricPrompt.PromptInfo.Builder()
            .setTitle("Biometric login")
            .setNegativeButtonText("Cancel")
            .build()
        biometricPrompt.authenticate(promptInfo)
    }
    BiometricManager.BIOMETRIC_ERROR_NONE_ENROLLED -> { /* 尚未註冊 */ }
}
```

| 等級 | 說明 | 硬體需求 |
|:---:|------|------|
| **Class 1** | 軟體實作 | 無 |
| **Class 2** | TEE 內執行 | Trusted Execution Environment |
| **Class 3** | StrongBox 專用硬體 | 獨立安全晶片 (e.g., Titan M) |
