# 調整區分大小寫

> [!info] 說明
> 設定 WSL 和 Windows 檔案系統的大小寫區分行為。

## 大小寫區分行為

### Linux vs Windows

| 系統 | 行為 |
|------|------|
| Linux | 區分大小寫 (`file.txt` ≠ `FILE.TXT`) |
| Windows | 不區分大小寫 (`file.txt` = `FILE.TXT`) |

### WSL 預設行為

- **Linux 檔案系統**: 區分大小寫
- **Windows 磁碟 (/mnt/c/)**: 預設不區分大小寫

## 設定目錄大小寫區分

### 使用 fsutil (Windows)

```powershell
# 在 Windows PowerShell (系統管理員) 中

# 啟用區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\projects\linux-project" enable

# 停用區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\projects\windows-project" disable
```

### 檢查目錄設定

```powershell
# 查看目錄的大小寫設定
fsutil.exe file queryCaseSensitiveInfo "C:\projects\my-project"
```

## WSL 設定

### /etc/wsl.conf 設定

```ini
# /etc/wsl.conf
[automount]
# 設定 Windows 磁碟的預設大小寫行為
options = "case=off"

# case 選項:
# off - 不區分大小寫 (預設)
# dir - 目錄支援個別設定
# force - 強制區分大小寫
```

### case=dir 模式

```ini
[automount]
options = "case=dir"
```

啟用後，可以對個別目錄設定不同的大小寫行為。

## Git 專案設定

### 問題場景

```bash
# 在區分大小寫的專案中
# Git 可能無法正確追蹤重新命名的檔案

git mv MyFile.js myFile.js
# 如果檔案系統不區分大小寫，可能出現問題
```

### 解決方案

#### 設定目錄區分大小寫

```powershell
# 為 Git 專案啟用區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\projects\my-repo" enable
```

#### Git 設定

```bash
# 設定 Git 忽略大小寫
git config core.ignorecase true

# 或在專案中設定
cd /mnt/c/projects/my-repo
git config core.ignorecase true
```

## 目錄遞迴設定

### 設定目錄及其子目錄

```powershell
# PowerShell 腳本 - 遞迴設定
function Set-CaseSensitive {
    param([string]$Path, [bool]$Enable)

    $directories = Get-ChildItem -Path $Path -Directory -Recurse
    foreach ($dir in $directories) {
        $action = if ($Enable) { "enable" } else { "disable" }
        fsutil.exe file setCaseSensitiveInfo $dir.FullName $action
    }
}

# 使用
Set-CaseSensitive "C:\projects\linux-projects" $true
```

## 實際應用場景

### 場景一：跨平台專案

```powershell
# 將 Linux 專案目錄設定為區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\dev\linux-kernel" enable
```

### 場景二：Web 開發

```powershell
# 網頁專案可能需要不區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\www\website" disable
```

### 場景三：混合專案

```powershell
# 主目錄不區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\projects" disable

# 特定子目錄區分大小寫
fsutil.exe file setCaseSensitiveInfo "C:\projects\backend" enable
```

## 注意事項

### 限制

- 只有 NTFS 檔案系統支援此功能
- 需要 Windows 10 1903+ 或更新版本
- 設定僅對目錄生效，不影響現有檔案

### 潛在問題

```bash
# 如果停用區分大小寫，以下檔案會被視為相同:
file.txt
File.txt
FILE.TXT

# 可能導致:
# - Git 衝突
# - 檔案覆蓋
# - 編譯錯誤
```

## 驗證設定

### 在 WSL 中測試

```bash
# 建立測試檔案
cd /mnt/c/projects/test
touch file.txt
touch FILE.TXT

# 如果區分大小寫，會建立兩個檔案
ls -la
# -rw-r--r-- 1 user user 0 ... file.txt
# -rw-r--r-- 1 user user 0 ... FILE.TXT

# 如果不區分大小寫，只會有一個檔案
ls -la
# -rw-r--r-- 1 user user 0 ... file.txt (或 FILE.TXT)
```

## 最佳實務

### 開發建議

| 專案類型 | 建議設定 |
|----------|----------|
| Linux 核心專案 | 區分大小寫 |
| Web 前端專案 | 不區分大小寫 |
| Node.js 專案 | 依需求設定 |
| Python 專案 | 不區分大小寫 |
| Go 專案 | 不區分大小寫 |

### Git 建議

```bash
# 始終設定 Git core.ignorecase
git config --global core.ignorecase false
```
