# 04-操作系统

> 操作系统是管理计算机硬件和软件资源的系统软件——它是应用程序和硬件之间的中间层，提供抽象和资源管理。

## OS 的三大核心抽象

| 抽象 | 管理资源 | 体现为 |
|------|----------|--------|
| **进程/线程** | CPU 时间 | 程序运行的实例 |
| **虚拟内存** | 物理内存 | 每个进程认为自己独占全部内存 |
| **文件系统** | 持久存储 | 文件和目录的层次结构 |

## 进程与线程

### 进程 (Process)

进程 = 运行中的程序。拥有独立的地址空间。

| 状态 | 含义 |
|------|------|
| 新建 (New) | 刚创建 |
| 就绪 (Ready) | 等待 CPU 调度 |
| 运行 (Running) | CPU 正在执行 |
| 等待 (Waiting) | 等待 I/O 或事件 |
| 终止 (Terminated) | 执行完毕 |

### 线程 (Thread)

线程是 CPU 调度的最小单位。同一进程的线程共享地址空间。

```
进程 (独立地址空间)
├── 线程 1 (自己的栈 + PC)
├── 线程 2 (自己的栈 + PC)
└── 线程 3 (自己的栈 + PC)
      └── 共享：堆、全局变量、打开的文件
```

| 维度 | 进程 | 线程 |
|------|------|------|
| 创建开销 | 大 | 小 |
| 通信 | IPC（管道、消息队列） | 共享内存（需要同步） |
| 独立性 | 一个崩溃不影响其他 | 一个崩溃整个进程挂 |

## 并发与同步

### 竞态条件 (Race Condition)

多个线程同时访问共享数据，结果取决于执行顺序。

### 同步原语

| 原语 | 用途 |
|------|------|
| **互斥锁 (Mutex)** | 一次只有一个线程进入临界区 |
| **信号量 (Semaphore)** | 控制同时访问资源的线程数量 |
| **条件变量** | 线程等待某个条件成立 |
| **读写锁** | 多个读可以并发，写要独占 |
| **原子操作** | 不可分割的操作（CAS, FAA） |

### 死锁 (Deadlock)

四个必要条件（缺一不可）：
1. **互斥**：资源不能共享
2. **持有并等待**：持有资源的同时等待其他资源
3. **不可抢占**：资源不能被强制释放
4. **循环等待**：形成资源等待环 P₁→P₂→...→P₁

> 破坏任何一个条件即可防止死锁。

## 内存管理

### 虚拟内存

每个进程看到的是**连续的、私有的地址空间**——这是 OS 和 MMU（内存管理单元）制造的"幻觉"。

```
虚拟地址 (进程看到的)
        ↓ MMU 转换 (页表)
   物理地址 (实际的 RAM)
```

### 分页 (Paging)

| 概念 | 说明 |
|------|------|
| **页 (Page)** | 固定大小的虚拟内存块（通常 4KB） |
| **页框 (Frame)** | 固定大小的物理内存块 |
| **页表** | 虚拟页 → 物理页框的映射 |
| **TLB** | 页表的 CPU 缓存，加速地址转换 |
| **缺页 (Page Fault)** | 访问的页不在物理内存中，需要从磁盘加载 |

### 页面置换算法

当物理内存满时，换出哪一页？

| 算法 | 思路 | 特点 |
|------|------|------|
| **FIFO** | 换出最早的 | 简单但不考虑访问模式 |
| **LRU** | 换出最久未使用的 | 效果好但开销大 |
| **Clock (近似 LRU)** | 使用访问位 + 指针扫描 | 工程折中方案 |

## 文件系统

| 概念 | 说明 |
|------|------|
| **inode** | 文件的元数据（大小、权限、数据块位置）——Unix/Linux |
| **目录** | 文件名 → inode 的映射 |
| **硬链接** | 多个文件名指向同一 inode |
| **软链接 (符号链接)** | 指向文件路径的"快捷方式" |
| **日志 (Journaling)** | 先写日志再写数据，崩溃恢复（ext4, NTFS） |

## I/O 模型

| 模型 | 特点 |
|------|------|
| **阻塞 I/O** | 线程卡住，等 I/O 完成 |
| **非阻塞 I/O** | 立即返回，轮询检查 |
| **I/O 多路复用** | select/poll/epoll——一个线程管理多个连接 |
| **异步 I/O** | 发起请求后继续执行，I/O 完成时通知 |

> epoll (Linux) 是实现高并发网络服务的基础——Nginx、Redis、Node.js 都是这个模式。
