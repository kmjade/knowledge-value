# 09 效能與故障診斷 Performance & Troubleshooting

> macOS 效能監控工具鏈：Activity Monitor (GUI)、sysdiagnose (全面診斷)、Console (系統日誌)、NVRAM/SMC 重置、Safe Mode 隔離測試、DTrace 深度追蹤。

## 監控工具對比 Monitoring Tools

| 工具 | 類型 | 功能 |
|------|:---:|------|
| **Activity Monitor** | GUI | CPU/記憶體/能源/磁碟/網路即時監控 |
| **top / htop** | CLI | 行程級 CPU/記憶體使用率 |
| **vm_stat** | CLI | 虛擬記憶體統計 (macOS 版 vmstat) |
| **sysdiagnose** | CLI | 一鍵收集全面診斷資料 (tar.gz) |
| **Console.app** | GUI | 系統日誌與活動檢視 |
| **log** | CLI | 統一日誌查詢 (Unified Logging) |
| **DTrace** | CLI | 動態追蹤 (需 SIP 部分關閉) |

## CLI 效能指令 Performance CLI Commands

```bash
# CPU / 記憶體
top -l 1 -n 10 -o cpu                    # 一次快照，CPU 排序前 10
vm_stat 1 5                               # 每 1 秒輸出 VM 統計，共 5 次
sysctl hw.memsize                         # 總記憶體 (bytes)
sysctl -n machdep.cpu.brand_string        # CPU 型號

# 磁碟 I/O
iostat -d 1                               # 磁碟 I/O 每秒更新
fs_usage -f filesys                       # 即時檔案系統活動

# 網路
nettop -m tcp                             # 即時網路流量監控
networksetup -listallhardwareports        # 列出所有網路介面

# 電源
pmset -g                                   # 電源管理設定
pmset -g batt                              # 電池狀態
pmset -g therm                             # 溫度/風扇
```

## sysdiagnose 全面診斷 sysdiagnose

```bash
# 一鍵收集系統診斷資料
sudo sysdiagnose                          # 存到 /var/tmp/sysdiagnose_*.tar.gz
# 或按快捷鍵：Ctrl+Opt+Cmd+Shift+Period (同時按住)

# 診斷內容包含：
# - 系統日誌 (system.log, kernel.log)
# - 行程列表與記憶體使用
# - 網路設定與統計
# - 磁碟資訊
# - 電源/電池狀態
# - 核心轉儲 (kernel panic log)
# - spin dump (如果系統無回應)
```

## Console.app 與 Unified Logging

```bash
# 統一日誌查詢 (Unified Logging)
log show --last 1h                         # 過去 1 小時日誌
log show --predicate 'process == "kernel"' --last 30m
log stream --predicate 'subsystem == "com.apple.network"'
log show --predicate 'eventMessage contains "error"' --last 1h

# 查看特定行程
log stream --process "mDNSResponder"

# 收集日誌存檔
log collect --last 1d --output ~/Desktop/logs.logarchive
```

## 疑難排解流程 Troubleshooting Flow

| 問題 | 優先嘗試 | 進階措施 |
|------|------|------|
| **系統緩慢** | Activity Monitor → 關閉高 CPU 行程 | 檢查 swap 使用量、重設 SMC |
| **應用崩潰** | Console.app → Crash Reports | `~Library/Logs/DiagnosticReports/` |
| **無法開機** | Safe Mode (按住 Shift 開機) | 磁碟工具修復、重裝 macOS |
| **WiFi 異常** | 關閉/開啟 WiFi、重設 NVRAM | 刪除網路設定檔 |
| **電池耗電快** | Activity Monitor → Energy tab | 重設 SMC、檢查電池健康度 |
| **Kernel Panic** | 查看 `/Library/Logs/DiagnosticReports/` | sysdiagnose 送 Apple 分析 |

## NVRAM / SMC 重置 Reset Procedures

| 重置目標 | Intel Mac | Apple Silicon |
|------|------|------|
| **NVRAM** | 開機時按住 `Opt+Cmd+P+R` 20 秒 | 關機 30 秒後重新開機 (自動) |
| **SMC** | T2 Mac：關機→按住電源 10 秒→放開等數秒→開機 | 無獨立 SMC (整合在 M1/M2/M3/M4 中) |

## DTrace 進階追蹤 Advanced DTrace

```bash
# DTrace 需要 SIP 部分放寬 (csrutil enable --without dtrace)
sudo dtrace -n 'syscall:::entry { @[execname] = count(); }'
sudo dtrace -n 'proc:::exec-success { trace(copyinstr(arg0)); }'
# 追蹤檔案開啟
sudo dtrace -n 'syscall::open*:entry { printf("%s %s", execname, copyinstr(arg0)); }'
```
