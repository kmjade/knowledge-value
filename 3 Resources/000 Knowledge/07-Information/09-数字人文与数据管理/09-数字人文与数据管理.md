# 09-数字人文与数据管理

> 数字人文 (Digital Humanities) 是计算技术与人文学科的交叉——用文本挖掘分析《红楼梦》，用 GIS 重建历史地图。科研数据管理 (RDM) 则关注如何让研究数据可发现、可获取、可复用。

## 数字人文 (Digital Humanities, DH)

### DH 不是"用电脑研究人文学科"

DH 是方法论的根本转变：

| 传统人文 | 数字人文 |
|----------|----------|
| 精读 (Close Reading) | 远读 (Distant Reading) — 分析数千文本的模式 |
| 单个文本 | 语料库 |
| 定性论证 | 定量+定性混合方法 |
| 个人研究 | 团队协作（人文学者+程序员+设计师） |

### DH 的主要方法

| 方法 | 说明 | 例子 |
|------|------|------|
| **文本挖掘** | 从大量文本中提取模式和趋势 | 分析 100 年报纸中的性别词汇变迁 |
| **网络分析** | 研究实体间的关系网络 | 宋代文人社交网络 |
| **空间分析 / GIS** | 将人文数据映射到地理空间 | 丝绸之路的贸易路线重建 |
| **3D 建模** | 数字化重建历史遗迹和物品 | 圆明园数字复原 |
| **数据可视化** | 用视觉呈现人文数据 | 哲学家的思想脉络图 |
| **语料库语言学** | 大规模语言数据的定量分析 | COCA, BCC 语料库 |

### DH 的关键项目

| 项目 | 说明 |
|------|------|
| **中国历代人物传记数据库 (CBDB)** | 哈佛开发，中国历史人物关系网络 |
| **中国哲学书电子化计划 (CTEXT)** | 先秦两汉典籍的数字化 |
| **唐宋文学编年地图** | 作家行迹的时空可视化 |
| **Voyant Tools** | 在线文本分析工具 |
| **Gephi** | 开源网络分析可视化 |

## 科研数据管理 (RDM)

### 数据生命周期

```
计划 → 收集 → 处理 → 分析 → 保存 → 共享 → 复用
  ↑________________________________________↓
```

### FAIR 原则

| 原则 | 含义 |
|------|------|
| **F**indable (可发现) | 数据有持久标识符（DOI）、丰富的元数据 |
| **A**ccessible (可获取) | 通过标准协议可访问，必要时有认证 |
| **I**nteroperable (可互操作) | 使用标准格式和词汇表 |
| **O**perable (可复用) | 清晰的许可和使用条件，详细的溯源信息 |

> FAIR 不等于 Open——数据可以 FAIR 但有限制访问（如医疗数据保护隐私）。

### 数据管理计划 (DMP)

现在很多基金申请强制要求 DMP。包含：

| 要素 | 问题 |
|------|------|
| 数据类型 | 产生什么数据？格式？体量？ |
| 元数据标准 | 用什么标准描述数据？ |
| 存储与备份 | 数据存在哪？怎么备份？ |
| 共享与获取 | 什么时候、如何共享？有什么限制？ |
| 长期保存 | 项目结束后数据如何保存？ |
| 伦理与法律 | 涉及敏感数据吗？需要伦理审批吗？ |

## 数字人文与信息科学的交叉

| 交叉点 | 信息科学的贡献 |
|--------|---------------|
| 元数据和标准化 | 设计 DH 项目的描述方案 |
| 数字保存 | 确保 DH 成果长期可用 |
| 信息检索 | 让人文数据可被搜索和发现 |
| 知识组织 | 分类法和本体用于人文知识结构 |
