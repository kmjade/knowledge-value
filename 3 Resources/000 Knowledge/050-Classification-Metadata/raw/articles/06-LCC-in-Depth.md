# LCC 美国国会图书馆分类法详解

> 1901年——21大类(A-Z)——Cutter号精确到个体——世界最大图书馆的分类法

---

## 一、概况

| 属性 | 值 |
|------|------|
| 全称 | Library of Congress Classification |
| 首次出版 | 1901年 |
| 维护方 | 美国国会图书馆 (LC) |
| 使用范围 | 美国研究型/大学图书馆为主 |
| 类型 | 枚举式·分卷独立修订 |
| 特色 | Cutter号——字母+数字精确到每本书 |

## 三、Cutter 号——LCC的灵魂

```
DS778.M3    = 毛泽东传记
GV1469.25.W67 = 魔兽世界
Z253.532.A34 = Adobe InDesign
```

| 部分 | 含义 |
|------|------|
| 字母前缀 | 大类+学科 (QA=数学·CS, TK=电气) |
| 数字 | 子类细化 |
| .Cutter | 字母+数字——精确到具体个体 |

> Cutter号让LCC无需复分表即可精确到每本著作——这是与DDC复分机制的根本差异。

---

## 四、LCC vs DDC

| | DDC | LCC |
|------|------|------|
| 大类 | 10 | 21 |
| 编码 | 纯数字 | 字母+数字+Cutter |
| 设计 | 知识树——排架号即位置 | 学科分卷——各科独立 |
| 更新 | 统一版本 (DDC 23) | 分卷独立修订 |
| 精度 | 复分表组合 | Cutter号直达个体 |
| 美国史 | 973-979 | 独占E+F两个大类 |
| 全球使用 | 200,000+馆 | 主要为美国大学 |

---

## 五、独特分类逻辑

| 主题 | LCC归属 | 分类逻辑 |
|------|:---:|------|
| InDesign | Z253（出版） | 以业务流程分类，非软件技术 |
| CS | QA76 | 归入科学(数学)，非技术(T) |
| 电子游戏 | GV1469 | 归入休闲，非CS |
| CMS | TK5105 | 归入电信工程 |

> LCC将工具按**业务流程**分类——InDesign是出版工具所以归入Z类，而非软件类。
