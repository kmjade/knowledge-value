# DDC In-Depth — 杜威十进制分类法详解

| Melvil Dewey (1876) —— 10大类 · 纯数字 · 小数制 · 层级结构 · 全球最广泛使用的图书分类法 |
|:---|
| 基本口号：**一个书号对应一本书的位置** —— 简单，所以有效 |

---

## 一、历史背景

| 人物 | 事件 |
|------|------|
| **Melvil Dewey** (1851–1931) | 1876 年匿名出版《图书馆图书小册子排架及编目分类法》——第一版仅 44 页 |
| 时代背景 | 19 世纪末美国图书馆爆炸式增长——迫切需要统一的排架标准 |
| 迭代 | 已出 **23 版** ——最新印刷版 2011——日常维护转由 OCLC 在线更新 |
| 产权 | DDC 商标归 OCLC 所有——WebDewey 为官方在线版本 |

| 有趣的事实：Dewey 还是个拼写改革者——把自己的姓从 Melville 简化为 Melvil ——还主张简化英语拼写 |
|:---|

## 二、十进制的核心思想

### 2.1 所有知识分 10 大类——每个大类分 10 个分部——每个分部再分 10 个小类

| 级别 | 命名 | 标记位数 | 示例 |
|:----:|------|:--------:|------|
| **Class** 大类 | `000–900` | 1 位 | 600 = 技术 |
| **Division** 分部 | `xxx–xxx` | 2 位 | 620 = 工程 |
| **Section** 小类 | `xxx–xxx` | 3 位 | 629.1 = 交通工程 |

> 原则上可无限细分：629.13 → 629.133 → 629.1333（飞行器→飞机→航空器详细类型）

### 2.2 十大主类一览

| 主类 | 名称 | 本 vault 对应 |
|:----:|------|:-------------:|
| **000** | 总类·知识·计算机 | `000 Knowledge` |
| **100** | 哲学·心理学 | `100 Philosophy` |
| **200** | 宗教 | —— |
| **300** | 社会科学 | `300 Social Sciences` |
| **400** | 语言 | `400 Language` |
| **500** | 自然科学·数学 | `500 Natural Sciences` |
| **600** | 技术·应用科学 | `600 Technology` |
| **700** | 艺术·休闲 | `700 Arts` |
| **800** | 文学 | `800 Literature` |
| **900** | 历史·地理·人物 | `900 History` |

### 2.3 百位分部（前三位关键细分）

| 000 总类 | 300 社科 | 500 自然科学 | 600 技术 |
|:---------:|:--------:|:------------:|:--------:|
| 000 知识总论 | 300 社科总论 | 500 自然科学 | 600 技术总论 |
| 010 书目 | 310 统计 | 510 数学 | 610 医学 |

| 060 组织 | 360 社会问题 | 560 古生物 | 660 化工 |

## 三、标记系统（Notation）

### 3.1 数字即层级

| 层级 | 数字 | 含义 |
|:----:|:----:|------|
| 大类 | **6**00 | 技术 |
| 分部 | **62**0 | 工程 |
| 小类 | **629** | 交通工程 |
| 细分 | **629.1** | 交通工程总论 |
| 更细 | **629.13** | 飞行器 |
| 极细 | **629.133** | 飞机及飞行器详细类型 |

### 3.2 小数点规则

| 规则 | 说明 |
|------|------|
| **三位后加小数点** | 629.13 —— 第三位之后加 `.` —— 增强可读性 |
| **非必须** | 检索时小数点可省略——62913 同样有效 |
| **小数制** | 629.13 < 629.2 —— 按十进制而非字符串排序 |

### 3.3 助记特征（Mnemonic）

| 原则 | 示例 |
|------|------|
| **相同含义往往共享相同数字** | 世界各地理区域的 1–9 编号在多个类目中保持一致 |
| **例** | 05 = 期刊/连续出版物 → 在所有大类中 05 几乎都表示期刊 |

## 四、复分表（Tables）—— DDC 的「乐高」机制

| 不是每个类目都列举——通过 6 张复分表组合——无限扩展性 |
|:---|

### 4.1 六大标准复分表

| 表号 | 名称 | 示例 |
|:----:|------|:----:|
| **T1** | 标准复分表 | -01 哲学理论·-03 辞典·-09 历史 |
| **T2** | 地理/区域表 | -51 中国·-52 日本·-73 美国 |
| **T3** | 文学复分表 | -1 诗歌·-2 戏剧·-3 小说 |
| **T4** | 语言复分表 | — |
| **T5** | 种族/民族表 | — |
| **T6** | 语言族表 | — |

### 4.2 组合示例

| 需求 | DDC 编号 | 如何构建 |
|------|:--------:|----------|
| 日本经济史 | **330.952** | 330(经济) + T2-52(日本) + 09(历史) |
| 中国哲学辞典 | **181.1103** | 181.11(中国哲学) + T1-03(辞典) |
| 教育学百科全书 | **370.3** | 370(教育) + T1-03(百科全书) |
| 量子物理历史 | **530.1209** | 530.12(量子物理) + T1-09(历史) |

> 组合顺序有时会反转——通常将地理/时代放在最后

## 五、相对索引（Relative Index）

| 分类是树·索引是网——相对索引把所有相关术语按字母顺序排列——每个词指向它在 DDC 树中的位置 |
|:---|

| 词 | DDC 位置 |
|:---|:--------:|
| Tea（茶） | 641.3372（饮品）· 394.12（饮食风俗）· 663.94（工业加工） |
| Railroad（铁路） | 385（交通）· 625.1（工程）· 385.09（历史） |

> 同一概念可能出现在多个大类中——索引将它们全部列出——这是 DDC vs 单层级分类的关键优势

## 六、DDC 的优势与局限

### 6.1 优势

| ✅ 简单——十进制人尽皆知 | ✅ 层级明确——分类即排架位置 |
| ✅ 全球采用——2万+图书馆 | ✅ 持续更新——WebDewey 季度更新 |
| ✅ 相对索引——跨类连接 | ✅ 教学资源丰富——编目员供应充足 |

### 6.2 局限

| ❌ 欧洲中心/西方中心 | 600–699 不够容纳现代科技——大量细分用小数点 |
| ❌ 数字分配不均 | 200 宗教 vs 600 技术——后者的压力远大于前者 |
| ❌ 单一排架限制 | 一本书只能放一个位置——而知识是多维的 |
| ❌ 灵活度不如分面分类 | UDC 的辅助符号允许更精确地表达复合主题 |

## 八、DDC 与知识组织发展

| 年代 | 发展 |
|:----:|------|
| 1876 | 第一版——44 页——仅 12 页是分类表 |
| 1885 | 第二版——引入相对索引——大改 |
| 1951 | 第 15 版——正式改名为「杜威十进制分类法」 |
| 1965 | 第 17 版——引入表 1 标准复分 |
| 2003 | 第 22 版——更新 004–006（计算机）——反映互联网时代 |

| 现在 | **WebDewey** 为权威版本——不再等印刷版 |

## 九、DDC 在本 vault 中的应用

本 vault 以 DDC 作为顶层分类骨架——但做了适应性调整：

| 调整 | 说明 |
|:----:|------|
| **DDC 体系** | 000·100·300·400·500·600·700·800·900 ——按 DDC 大类组织 |
| **004 独立** | 计算机科学从 000 中抽出——单独构建——反映现代重要性 |
| **006 合并** | AI 从 006.3 抽取——独立为 `Artificial Intelligence` |
| **200 未建** | 宗教大类暂缺——将来可按需建立 |
| **分类粒度** | 文件级别用 DDC 标签（ddc-000 · ddc-060）——不一定要纯数字排架 |
| **组合使用** | DDC 作为宏观骨架 + 标签/Tag 作为微观维度——避免「一个位置」限制 |

> 在数字笔记系统中——DDC 提供的是**认知地图**而非排架号——分类的价值从「书在哪里」变为「这个知识属于哪个领域」
