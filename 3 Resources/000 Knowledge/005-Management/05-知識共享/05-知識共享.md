# 05 知識共享 (Knowledge Sharing)

> 知識就是力量 — 但共享知識才是真正的力量。

---

## 共享的心理學：為什麼不共享？

| 障礙 | 典型內心獨白 | 對策 |
|:-----|:------------|:-----|
| **權力感** | "這是我唯一的優勢，分享了我就沒價值了" | 獎勵共享·重新定義價值 |
| **時間** | "我忙著做事，沒時間寫文檔" | 降低摩擦·捕獲 > 記錄 |
| **信任** | "他們會把我的想法佔為己有" | 建立心理安全·公開歸屬 |
| **不重要感** | "我的知識不值錢，別人早就知道了" | 創造分享的安全空間 |
| **完美主義** | "等我整理好了再分享…" (永遠不會) | 接受半成品·迭代 |
| **不知道誰需要** | "我有很多經驗但不知道誰用得上" | 建立需求/供應匹配 |

## 知識市場理論 (Davenport & Prusak)

知識在組織內的流動類似於**市場**：

| 市場要素 | KM 對應 |
|:---------|:-------|
| **買方** | 尋求知識的人 |
| **賣方** | 擁有知識的人 |
| **仲介** | 連接供需的人（CoP 協調員·知識 curator） |
| **價格** | 互惠·聲譽·利他主義 |

> 關鍵洞察：知識市場的「貨幣」不是金錢，而是**互惠**（你幫我，我幫你）、**聲譽**（被認為是專家）和**利他主義**。

## 共享激勵設計

| 激勵類型 | 機制 | 效果 | 風險 |
|:---------|:-----|:----|:-----|
| **外在激勵** | 獎金·升遷·積分 | 快速見效 | 可能產生垃圾貢獻 |
| **內在激勵** | 歸屬感·成就感·利他 | 持久·高品質 | 啟動慢 |
| **社會激勵** | 公開表揚·專家地位·peer recognition | 中等持久 | 需要文化支撐 |
| **嵌入式激勵** | 共享本身就是工作流程的一部分 | 最持久 | 設計成本高 |

> 最佳策略：**嵌入式 > 內在 > 社會 > 外在**。獎金可以讓數量暴漲，但無法讓品質提升。

## 共享機制設計

| 機制 | 適合 | 例子 |
|:-----|:-----|:-----|
| **事後回顧** AAR | 專案結束後 | "什麼做得好？什麼可以改善？" |
| **午餐學習** Lunch & Learn | 非正式·跨部門 | 每週五一人分享 20 分鐘 |
| **知識咖啡** Knowledge Café | 深度對話 | 圍繞一個問題的開放式討論 |
| **內部 Wiki** | 持續積累 | 任何人的經驗都可以變成頁面 |
| **導師制** Mentoring | 隱性知識傳遞 | 配對·影子學習 |
| **離職訪談** Exit Interview | 知識保留 | "如果你要教你的接班人一件事…?" |

## 故事化 (Storytelling)

> 人類不是靠 bullet points 記住東西的，是靠**故事**。

| 故事類型 | 功能 | 例子 |
|:---------|:-----|:-----|
| **成功故事** | 傳播最佳實踐 | "我們如何把客戶投訴變成了我們最暢銷的功能" |
| **失敗故事** | 防止重複犯錯 | "那個 200 萬的 bug — 事後發現只是少了一個分號" |
| **起源故事** | 建立身份·價值觀 | "公司創立的第一天我們只有 3 個人和一個信念" |
| **未來故事** | 激發行動·願景 | "如果我們成功了，這個行業會變成什麼樣" |

## 衡量共享成效

| 指標 | 測量 |
|:-----|:-----|
| 文檔貢獻量 | 每人每月新增/修改的 Wiki 頁面 |
| 重用率 | 多少問題通過搜尋已有知識解決了 |
| 回答速度 | 提問到獲得答案的平均時間 |
| 新人上手 | onboarding 到獨立工作的時間 |
| 重複錯誤 | 同樣的錯誤是否再次發生 |
