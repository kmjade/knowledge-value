# 08 — 知識管理工具 (KM Tools)

> Notion · Confluence · SharePoint——組織層面 / Obsidian · Roam——個人層面。工具決定 KM 的天花板。

---

## 1. 工具在 KM 中的角色

工具不是 KM 的**目的**——但它是 KM 的**基礎設施**。好的工具降低知識流動的摩擦，壞的工具讓 KM 變成負擔。

### 工具選擇的黃金法則

> **工具應該適應用戶的工作流，而不是用戶適應工具的工作流。**

### 2.2 按功能分類

| 功能 | 組織級 | 個人級 |
|------|--------|--------|

## 3. 主要工具對比

### 3.1 Obsidian

| 維度 | 評價 |
|------|------|
| **核心範式** | 本地 Markdown 文件 + 雙向鏈接 |
| **優勢** | 離線、快速、純文本、插件生態豐富、隱私 |
| **劣勢** | 協作弱、學習曲線陡 |
| **最適合** | 個人知識庫、研究者、長期知識積累 |
| **KM 價值** | PKM 的黃金標準；支持 Zettelkasten、PARA、LLM-Wiki 等多種方法論 |

### 3.2 Notion

| 維度 | 評價 |
|------|------|
| **核心範式** | 區塊編輯器 + 數據庫 |
| **優勢** | 靈活、美觀、團隊協作強、模板豐富 |
| **劣勢** | 非本地存儲、大文件慢、離線弱 |
| **最適合** | 團隊 Wiki、項目管理、輕量 CRM |
| **KM 價值** | 組織級 KM 的低門檻方案 |

### 3.3 Roam Research

| 維度 | 評價 |
|------|------|
| **核心範式** | 大綱式筆記 + 自動反向鏈接 |
| **優勢** | 極低摩擦的鏈接創建、每日筆記為核心 |
| **劣勢** | 僅在線、價格高、數據導出弱 |
| **最適合** | 深度思考者、研究型寫作 |
| **KM 價值** | 讓知識網絡自然湧現——無需手動建立結構 |

### 3.4 Confluence / SharePoint

| 維度 | 評價 |
|------|------|
| **核心範式** | 企業級文檔管理與協作 |
| **優勢** | 與 Office 365 / Atlassian 生態整合 |
| **劣勢** | 重、慢、用戶體驗差、搜索弱 |
| **最適合** | 大型企業的合規文檔管理 |
| **KM 價值** | 「必要之惡」——大部分企業 KM 的現實基礎 |

## 4. 工具選型框架

### 4.1 選擇維度

| 維度 | 問題 |
|------|------|
| **用戶規模** | 個人？團隊（<50）？組織（>500）？ |
| **內容類型** | 主要是文字？混合媒體？結構化數據？ |
| **協作需求** | 單人？非同步協作？即時協作？ |
| **安全要求** | 可上雲？必須本地？合規要求（GDPR/HIPAA）？ |
| **維護成本** | 誰來管理？需要 IT 支持嗎？ |
| **生態整合** | 需要與現有工具鏈集成嗎？ |

### 4.2 推薦組合

| 場景 | 推薦組合 |
|------|----------|
| **個人研究者** | Obsidian + Zotero + Readwise |
| **小型團隊** | Notion + Slack + Miro |
| **大型組織** | Confluence + Teams + Glean（搜索） |
| **AI 增強 PKM** | Obsidian + Claude Code（LLM-Wiki） |
