# 02 SECI 模型

> 知識不是「傳遞」的，是「轉換」的。 — Ikujiro Nonaka

---

## SECI 是什麼？

SECI 是 Nonaka 和 Takeuchi 在 *The Knowledge-Creating Company* (1995) 中提出的**知識創造螺旋**模型，描述隱性知識和顯性知識之間的四種轉換。

```
         Tacit ────────────→ Explicit
           │                    │
   Socialization          Externalization
    (社會化)                (外化)
   共鳴→共感               對話→隱喻
           │                    │
           ▼                    ▼
   Internalization        Combination
    (內化)                  (組合)
   做中學                  連結→系統
           │                    │
         Tacit ←──────────── Explicit
```

---

## 四種轉換模式

### S: Socialization（社會化）— Tacit → Tacit

> 不靠語言，靠**共處**·**觀察**·**模仿**。

| 機制 | 例子 |
|:-----|:-----|
| 師徒制 | 學徒觀察師傅的手法 |
| 在職訓練 OJT | 新人跟著老手做 |
| 腦力激盪 | 非正式的咖啡閒聊激發靈感 |
| 客戶訪問 | 到現場感受客戶的真實痛點 |

**關鍵概念**: **Ba（場）** — 知識創造需要**共享的時空**。不是會議室，而是「我們在一起思考的空間」。

### E: Externalization（外化）— Tacit → Explicit

> 把「說不出來的」變成「說得出來的」。

這是 SECI 中最**關鍵**也最**困難**的一步。

| 技巧 | 說明 | 例子 |
|:-----|:-----|:-----|
| 隱喻 Metaphor | 用熟悉的事物理解陌生的 | 「組織是大腦」 |
| 類比 Analogy | 結構性比較 | 「KM 像園藝 — 你創造條件，但生長靠自然」 |
| 建模 Model | 抽象化成可操作的框架 | SECI 本身就是一個模型 |
| 對話 Dialogue | 集體反思 | 豐田的「五次為什麼」 |

### C: Combination（組合）— Explicit → Explicit

> 把零散的顯性知識**拼成系統**。

| 機制 | 例子 |
|:-----|:-----|
| 文件整合 | 把各部門 SOP 彙編成企業手冊 |
| 數據分析 | 從銷售數據中發現模式 |
| 教育訓練 | 把專家知識做成課程 |
| 知識庫 | 結構化的 Wiki·FAQ |

### I: Internalization（內化）— Explicit → Tacit

> 看文件學不會 — 必須**做**。

| 機制 | 例子 |
|:-----|:-----|
| 實作練習 | 讀完 SOP 後實際操作 |
| 模擬訓練 | 飛行模擬器·急救演練 |
| 講授他人 | 教別人是最好的學習 |
| 反思日誌 | 行動後回顧 (After Action Review) |

---

## Ba（場）的四種類型

| Ba | 轉換 | 說明 | 例子 |
|:---|:----|:-----|:-----|
| **Originating Ba** 創始場 | S 社會化 | 面對面·共感 | 咖啡角·師徒工坊 |
| **Dialoguing Ba** 對話場 | E 外化 | 集體反思·對話 | Retrospective·腦力激盪會 |
| **Systemizing Ba** 系統場 | C 組合 | 虛擬·異步 | Wiki·Slack·內網 |
| **Exercising Ba** 練習場 | I 內化 | 做中學 | 模擬·實習·OJT |

---

## 知識螺旋的驅動力

| 要素 | 角色 |
|:-----|:-----|
| **意圖** Intention | 組織的戰略方向 — 「我們要創造什麼知識？」 |
| **自主性** Autonomy | 個人和團隊有探索的自由 |
| **波動與創造性混沌** Fluctuation | 外部危機或刻意製造的挑戰 |
| **冗餘** Redundancy | 資訊的重疊和共享 — 讓每個人都知道超過他職位需要的 |
| **必備多樣性** Requisite Variety | 內部多樣性匹配外部複雜性 |

## SECI 的批評與發展

| 批評 | 回應 |
|:-----|:-----|
| 太日本中心 | Nonaka 本人後來增加了跨文化比較 |
| 線性/理想化 | 實際上是螺旋而非線性，允許跳躍 |
| 忽略權力 | 後續研究加入了政治維度 |
| SECI 2.0 | Nonaka (2000s) 加入 Phronesis（實踐智慧）和分散式領導 |
