# 知識審計 (Knowledge Audit)

> 組織有什麼知識？缺什麼？——像財務審計，但更難——因為知識看不見。

---

## 1. 什麼是知識審計

**知識審計 (Knowledge Audit)** 是系統性地**盤點、評估和診斷**組織知識資產的過程。它的目的是回答三個問題：

| 問題 | 說明 |
|------|------|
| **我們有什麼知識？** | 盤點現有知識資產——人、文檔、流程、專利 |
| **知識在哪裡？誰擁有它？** | 繪製知識地圖——識別知識的持有者和位置 |
| **缺什麼？哪裡有風險？** | 識別知識缺口和流失風險——誰快退休了？哪些關鍵知識沒有備份？ |

---

## 2. 知識審計 vs 財務審計

| 維度 | 財務審計 | 知識審計 |
|------|----------|----------|
| **對象** | 有形資產（現金、庫存） | 無形資產（隱性知識、關係、專長） |
| **測量** | 精確定量 | 定性為主，半定量輔助 |
| **標準** | 會計準則 (GAAP/IFRS) | 無統一標準 |
| **頻率** | 年度強制 | 按需進行 |
| **輸出** | 財務報表 | 知識地圖 + 缺口分析報告 |

---

## 3. 知識審計流程

### 3.1 四步方法論

```
Step 1: 定義範圍      Step 2: 數據收集      Step 3: 分析診斷      Step 4: 行動建議
    │                     │                     │                    │
    ├─ 審計目標           ├─ 問卷調查           ├─ 知識地圖          ├─ 優先級排序
    ├─ 範圍邊界           ├─ 深度訪談           ├─ 缺口分析          ├─ 行動方案
    └─ 利害關係人         ├─ 文檔審查           ├─ 流量分析          └─ KPI 設定
                          └─ 社交網絡分析       └─ 風險評估
```

### 3.2 常用工具

| 工具 | 用途 | 說明 |
|------|------|------|
| **知識地圖** | 可視化知識的位置和流向 | 「誰知道什麼」——用圖譜呈現 |
| **社交網絡分析 (SNA)** | 識別知識傳播的關鍵節點 | 找出組織中真正的「信息樞紐」 |
| **知識存量與流量分析** | 區分「已有的」vs「在流動的」 | 庫存很多但沒人用 = 無效知識 |
| **關鍵知識風險評估** | 識別單點故障 | 「如果老張明天離職...」 |
| **能力矩陣** | 團隊技能盤點 | 誰會什麼？到什麼程度？ |

---

## 4. 知識審計的關鍵發現

### 4.1 常見發現模式

| 發現 | 表現 | 影響 |
|------|------|------|
| **知識孤島** | 關鍵知識只存在於少數人的頭腦中 | 單點故障風險 |
| **知識過載** | 太多文檔，沒人知道哪個是對的 | 查找成本 > 知識價值 |
| **幽靈知識** | 過時但仍在流傳的最佳實踐 | 基於錯誤前提的決策 |
| **知識瓶頸** | 某人成為所有問題的必經之路 | 組織速度受限於個人頻寬 |
| **結構性缺口** | 戰略需要的知識 vs 組織實際有的 | 創新能力受限 |

### 4.2 審計之後

審計本身不是目的——其價值在於引出**行動**：

1. **知識保留計劃** — 關鍵人員離職前的知識轉移
2. **知識獲取策略** — 招聘、培訓、外部引進
3. **知識共享機制** — 打破孤島的結構性設計（[[05-Knowledge-Sharing\|見第 5 章]]）
