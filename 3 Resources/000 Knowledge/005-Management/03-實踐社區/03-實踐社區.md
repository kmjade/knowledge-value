# 03 實踐社區 (Communities of Practice)

> 你知道的東西，取決於你跟誰在一起。 — Etienne Wenger

---

## CoP 是什麼？

**實踐社區**是一群人因為**共同的關懷或熱情**而聚在一起，通過**持續互動**來深化知識和專業。

Wenger 定義的三個核心要素：

| 要素 | 問題 | 說明 |
|:-----|:-----|:-----|
| **領域** Domain | 我們關心什麼？ | 共同的知識領域，定義社區的身份和邊界 |
| **社區** Community | 我們如何互動？ | 成員之間的關係、信任、互動節奏 |
| **實踐** Practice | 我們做什麼？ | 共享的工具、方法、故事、詞彙 |

> 不是「一個團隊」— 團隊有任務交付。不是「一個社群」— 社群只有社交。CoP 的核心是**共同學習**。

---

## CoP vs 其他組織形式

| | 實踐社區 CoP | 正式團隊 Team | 專案小組 Project | 興趣社群 Network |
|:--|:------------|:-------------|:----------------|:----------------|
| 目的 | 學習·知識 | 交付結果 | 完成任務 | 社交·資訊 |
| 成員 | 自願·熱情 | 指派 | 指派 | 自願 |
| 持續 | 長期 | 長期 | 有時限 | 不定 |
| 邊界 | 模糊 | 清晰 | 清晰 | 非常模糊 |
| 產出 | 知識·方法 | 產品·服務 | 交付物 | 連結 |

## CoP 生命週期

```
    潛在期  →  凝聚期  →  活躍期  →  分散期  →  紀念期
   Potential  Coalescing  Active   Dispersed  Memorable
      │           │          │          │          │
   發現共同    建立關係   深度實踐   節奏放緩   成為記憶
   興趣和需求  信任和節奏  知識產出   保持連結   和資源庫
```

| 階段 | 關鍵活動 | KM 價值 |
|:-----|:---------|:-------|
| **潛在期** | 發現共同痛點·找到關鍵人物 | 識別知識缺口 |
| **凝聚期** | 建立信任·定義領域·設計互動 | 建立共享詞彙 |
| **活躍期** | 解決問題·創造知識·文檔化 | **最高價值輸出** |
| **分散期** | 知識保留·交接·歸檔 | 防止知識流失 |
| **紀念期** | 故事講述·遺產傳承 | 組織記憶 |

## 設計 CoP 的七原則 (Wenger)

1. **為演化而設計** — CoP 是有機的，不是機械的
2. **開放內部與外部的對話** — 引入新視角
3. **邀請不同程度的參與** — 核心組·活躍組·外圍組
4. **發展公共和私人的空間** — 正式會議 + 非正式交流
5. **聚焦價值** — 持續問「這對誰有價值？」
6. **結合熟悉與興奮** — 常規節奏 + 新鮮刺激
7. **創造社區的節奏** — 太快＝疲勞，太慢＝淡出

## 參與層次

```
        ┌──────────┐
        │  核心組   │  ~3-5%  策劃·領導
        │  ┌────┐  │
        │  │活躍│  │  ~10-15% 常規貢獻
        │  └────┘  │
        │┌────────┐│
        ││ 外圍組  ││  ~80%   觀察·偶爾參與
        │└────────┘│
        └──────────┘
            ↑
      訪客/陌生人 — 偶然接觸
```

> 健康的 CoP 允許不同強度的參與 — 核心組太小會 burnout，外圍組沒有會被孤立。

---

## CoP 常見失敗模式

| 失敗 | 原因 | 解法 |
|:-----|:-----|:-----|
| 變成另一個會議 | 沒有區別於日常工作 | 明確「這是學習時間」 |
| 核心組 burnout | 只有少數人撐著 | 輪流主持·培養新核心 |
| 知識不流動 | 只有輸出沒有整理 | 設置知識 curator 角色 |
| 被管理層扼殺 | 過度正規化 | 保護 CoP 的自發性 |
