# 07 個人知識管理 (Personal KM)

> 你的第二大腦，不應該只有「儲存」功能。

---

## PKM 的核心理念

PKM 是把 KM 的原則應用到**個人層面**：你如何捕獲、組織、連接和應用你所學的一切。

| 問題 | 組織 KM | 個人 KM |
|:-----|:-------|:-------|
| 知識在哪？ | 數據庫·Wiki·專家 | 筆記·大腦·書籤·聊天記錄 |
| 為誰服務？ | 組織績效 | 個人成長·創造力 |
| 誰維護？ | KM 團隊 | 你自己 |

## Zettelkasten（卡片盒筆記法）

Niklas Luhmann 用這個方法寫了 70 本書和 400+ 篇論文。

### 核心原則

| 原則 | 說明 |
|:-----|:-----|
| **原子性** Atomicity | 一張卡片 = 一個想法 |
| **連結** Connections | 卡片之間必須有連結 |
| **自己的話** Your Words | 不是摘抄，是理解後的**重述** |
| **序列** Sequences | 相關卡片形成序列，逐漸發展論點 |

### 三種筆記

| 類型 | 用途 | 處理 |
|:-----|:-----|:-----|
| ** fleeting note** 靈感筆記 | 快速捕獲想法 | 24 小時內整理或丟棄 |
| ** literature note** 文獻筆記 | 閱讀時記錄·用自己的話 | 與永久筆記連結 |
| ** permanent note** 永久筆記 | 一個完整的想法 | 永遠保留·不斷連結 |

## PARA 方法

Tiago Forte 提出，基於「行動性」而非「主題」的組織方式：

| 層級 | 說明 | 判斷 |
|:-----|:-----|:-----|
| **P**rojects 專案 | 有 deadline 的系列任務 | 「我現在正在做的」 |
| **A**reas 領域 | 長期責任範圍 | 「我需要維持的」 |
| **R**esources 資源 | 感興趣的主題 | 「我想參考的」 |
| **A**rchives 歸檔 | 已完成或不活躍 | 「暫時不需要的」 |

> 從 R 到 P 的移動 = 從「學」到「做」。

---

## CODE 框架（第二大腦）

| 階段 | 說明 |
|:-----|:-----|
| **C**apture 捕獲 | 把所有想法·靈感·文章存到一個 inbox |
| **O**rganize 組織 | 用 PARA 結構化 |
| **D**istill 提煉 | 漸進式總結：粗體→高亮→摘要 |
| **E**xpress 表達 | 創造輸出：文章·專案·產品 |

---

## GTD（Getting Things Done）

David Allen 的五步驟：

```
Capture → Clarify → Organize → Reflect → Engage
 (捕獲)    (釐清)     (組織)     (回顧)    (執行)
```

核心洞察：**大腦是用來產出想法的，不是用來儲存想法的**。把所有待辦事項「倒出」大腦放入系統。

## 選擇你的 PKM 系統

| 你是… | 推薦 |
|:-------|:-----|
| 研究者·寫作者 | Zettelkasten (Obsidian/Logseq) |
| 專案多·行動導向 | PARA + GTD (Notion/Obsidian) |
| 創意工作者 | Second Brain (任何工具) |
| 什麼都想要 | 混合：PARA 文件夾 + Zettelkasten 筆記 |
