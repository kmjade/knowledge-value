# 06 組織學習 (Organizational Learning)

> 組織不是因為聰明而學習，是因為**願意承認錯誤**。 — Chris Argyris

---

## 什麼是組織學習？

組織學習是組織**檢測和糾正錯誤**的過程—不僅是個人學習的總和，而是**嵌入流程、文化和系統**的集體能力。

---

## Argyris：單環、雙環、三環學習

```
單環學習 Single-Loop:
  目標 → 行動 → 結果 ──→ 調整行動 ──→ 回到目標
         (不質疑目標本身)

雙環學習 Double-Loop:
  目標 → 行動 → 結果 ──→ 質疑目標和假設 ──→ 修改目標
         (問「我們是不是在解決正確的問題？」)

三環學習 Triple-Loop (Deutero-learning):
  學習如何學習 — 質疑學習過程和組織結構本身
  「我們為什麼是這樣學習的？」
```

| | 單環 | 雙環 | 三環 |
|:--|:-----|:-----|:-----|
| 問題 | 做錯了 | 在做錯的事 | 為什麼總是錯？ |
| 行動 | 調整行為 | 改變目標/策略 | 重構學習系統 |
| 例子 | 恆溫器調溫度 | 問「為什麼設這個溫度？」 | 問「誰設的？為什麼是他設？」 |
| 難度 | 低 | 高 — 需要心理安全 | 最高 — 需要制度變革 |

## Senge 的五項修練

Peter Senge 在 *The Fifth Discipline* (1990) 提出學習型組織的五個核心：

| 修練 | 說明 | 核心問題 |
|:-----|:-----|:---------|
| **系統思考** Systems Thinking | 看到整體而非局部·反饋迴路 | "這個問題的根源在哪？" |
| **個人精進** Personal Mastery | 持續學習·自我超越 | "我在成長嗎？" |
| **心智模式** Mental Models | 檢視深層假設·開放反思 | "我確信的是真的嗎？" |
| **共同願景** Shared Vision | 集體認同的未來圖像 | "我們一起去哪裡？" |
| **團隊學習** Team Learning | 深度對話·集體智慧 | "我們在一起時更聰明嗎？" |

> 第五項修練 = 系統思考 — 它整合其他四項。

## 常見學習障礙

| 障礙 | 表現 | Senge 的解法 |
|:-----|:-----|:------------|
| 「我就是我的職位」 | 只看自己的 KPI | 系統思考 — 看到全局 |
| 「敵人在外部」 | 永遠是別人的錯 | 心智模式 — 檢視自己的假設 |
| 「掌控的幻覺」 | 過度反應·頭痛醫頭 | 系統思考 — 找槓桿點 |
| 「專注事件」 | 只看新聞不看歷史 | 系統思考 — 看模式·看結構 |
| 「煮蛙寓言」 | 對漸變威脅無感 | 系統思考 — 注意緩慢變化 |
| 「從經驗中學習的錯覺」 | 最關鍵決策的後果從未體驗 | 心智模式 — 模擬·情境規劃 |
| 「管理團隊的神話」 | 表面和諧·內部鬥爭 | 團隊學習 — 深度對話 |

## 學習型組織 vs 傳統組織

| 維度 | 傳統組織 | 學習型組織 |
|:-----|:---------|:----------|
| 錯誤 | 隱藏·懲罰 | 公開·學習 |
| 資訊 | 權力 = 掌握資訊 | 資訊共享 = 賦能 |
| 決策 | 由上而下 | 分散式 |
| 衝突 | 迴避 | 視為學習機會 |
| 變革 | 危機驅動 | 持續演化 |
| 領導者角色 | 指揮官 | 設計師·教師·服務者 |

## 行動後回顧 AAR (After Action Review)

美軍開發的最簡潔組織學習工具，4 個問題：

> 1. 我們預期發生什麼？
> 2. 實際發生了什麼？
> 3. 為什麼有差異？
> 4. 下次我們會做什麼不同？

關鍵原則：**不責備·不排名·不記錄績效** — 只記錄學習。
