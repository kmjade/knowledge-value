# 個人知識管理 (Personal Knowledge Management)

> 組織 KM 的前提是每個人管理好自己的知識——PKM 是 KM 的細胞級基礎。

---

## 1. 什麼是 PKM

**個人知識管理 (Personal Knowledge Management, PKM)** 是個體系統性地**捕獲、組織、反思和應用**信息與知識的實踐。PKM 的目標不是「記住一切」，而是**構建一個可持續增長的知識系統**，讓過去的學習支持未來的決策。

### PKM 的四個核心環節

```
  捕獲 (Capture)  →  組織 (Organize)  →  反思 (Reflect)  →  應用 (Apply)
       │                  │                   │                 │
  快速記錄            分類與鏈接          提煉洞察          創作與行動
  降低摩擦            按用途不按來源      把信息變成知識    知識的終極目的是行動
```

## 2. 主流 PKM 方法論

### 2.1 Zettelkasten（卡片盒筆記法）

| 維度 | 說明 |
|------|------|
| **創始人** | Niklas Luhmann（德國社會學家） |
| **核心** | 每張卡片 = 一個原子概念；通過鏈接形成知識網絡 |
| **原則** | 原子化、鏈接優先、用自己的話寫 |
| **工具** | Obsidian, Roam Research, Logseq |

**適用**: 深度研究者、寫作者、需要長期積累知識的人。

### 2.2 PARA 方法

| 維度 | 說明 |
|------|------|
| **創始人** | Tiago Forte（*Building a Second Brain*） |
| **核心** | 按**行動性**（而非主題）分類：Projects / Areas / Resources / Archives |
| **原則** | 文件只存在於四個頂層文件夾中；項目驅動組織 |
| **工具** | Notion, Obsidian, 任何文件系統 |

**適用**: 項目驅動的知識工作者、需要將知識與行動掛鉤的人。

### 2.3 Second Brain（第二大腦）

| 維度 | 說明 |
|------|------|
| **創始人** | Tiago Forte |
| **核心** | CODE 框架：Capture → Organize → Distill → Express |
| **原則** | 漸進式總結 (Progressive Summarization)；只保存有長期價值的 |
| **工具** | Notion, Obsidian, Evernote |

**適用**: 信息過載的現代知識工作者。

### 2.4 GTD（Getting Things Done）

| 維度 | 說明 |
|------|------|
| **創始人** | David Allen |
| **核心** | 將所有「待辦事項」從大腦卸載到外部系統；通過 Context 分類 |
| **原則** | 兩分鐘規則；每週回顧；Inbox Zero |

**適用**: 以任務和項目為工作核心的人。

### 2.5 LLM-Wiki 方法論

| 維度 | 說明 |
|------|------|
| **核心** | AI 負責知識編譯——人類捕獲原始資料，AI 將其結構化為 Wiki 頁面 |
| **原則** | raw/（人維護）與 wiki/（AI 維護）分離；從組織者進化為決策者 |
| **工具** | Obsidian + Claude Code |

**適用**: 追求極低維護成本、願意信任 AI 進行知識結構化的進階用戶。

## 3. PKM 方法論比較

| 維度 | Zettelkasten | PARA | GTD | LLM-Wiki |
|------|:-----------:|:----:|:---:|:--------:|
| **組織方式** | 概念網絡 | 行動分類 | 情境列表 | raw/ + wiki/ |
| **粒度** | 原子卡片 | 文件級 | 任務級 | 文件級 |
| **維護成本** | 中高 | 中 | 中 | **低**（AI 承擔） |
| **知識發現** | 湧現式 | 檢索式 | 不強調 | 編譯式 |
| **最適合** | 研究者/寫作者 | 項目驅動者 | 任務管理者 | AI 輔助的深度學習者 |

## 4. PKM 的常見陷阱

| 陷阱 | 表現 | 解法 |
|------|------|------|
| **工具收集癖** | 不停切換筆記軟體，從不生產 | 選一個，用一年 |
| **完美分類主義** | 花 80% 時間設計文件夾結構 | 先寫再分類；讓結構自然湧現 |
| **收藏 = 學會** | 存了 1000 篇文章從不重讀 | 強制自己對每篇寫 3 句話總結 |
| **過度捕獲** | 什麼都存，知識庫變成垃圾場 | 問自己：「一年後還會需要嗎？」 |
| **無輸出循環** | 只輸入不輸出——知識庫成了黑洞 | PKM 的終點是創作，不是存儲 |

## 5. 構建有效的 PKM 系統

### 5.1 設計原則

| 原則 | 說明 |
|------|------|
| **低摩擦捕獲** | 2 秒內能開始記錄；手機 + 桌面全覆蓋 |
| **快速檢索** | 全文搜索 + 雙向鏈接 + 標籤體系 |
| **漸進式處理** | 不要求一次性完美——先粗糙，再迭代 |
| **以輸出為導向** | 每個知識點最終指向一個行動或一個作品 |
| **定期回顧** | 每週回顧——知識不用就會腐爛 |

### 5.2 每日 PKM 工作流

```
Morning:  檢視今日意圖 + Inbox 狀態            (2 min)
During:   隨時捕獲想法 → 0 Inbox/              (隨時)
Evening:  回顧今日筆記 → 必要時整理 → 歸檔      (5 min)
Weekly:   週回顧 → 清理 Inbox → 知識編譯        (15-30 min)
```
