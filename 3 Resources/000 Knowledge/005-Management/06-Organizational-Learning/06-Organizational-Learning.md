# 組織學習 (Organizational Learning)

> 不只是「我們做得對不？」——是「我們做對了事嗎？」——雙環學習的靈魂之問

---

## 1. 什麼是組織學習

**組織學習 (Organizational Learning)** 是組織通過**經驗、反思和適應**來改進其行動和決策能力的過程。關鍵區別：個體學習 ≠ 組織學習——一個人學會了，不等於組織學會了。

> 「組織學習發生在：當組織中的個體發現問題並推動改變，即使關鍵人物離開後，改變仍然保留在組織的記憶中。」

---

## 2. 單環、雙環、三環學習

Argyris & Schön 提出的三層次學習模型是組織學習的核心框架：

```
         單環學習                          雙環學習                       三環學習
    ┌──────────────┐               ┌──────────────┐              ┌──────────────┐
    │ 做錯了 → 修正  │               │ 為什麼這樣做？  │              │ 我們如何學習   │
    │ 結果 ← 行動    │               │ 目標對嗎？     │              │ 學習本身？     │
    └──────────────┘               └──────────────┘              └──────────────┘
      「修復問題」                     「質疑前提」                    「學習如何學習」
```

### 2.1 單環學習 (Single-Loop Learning)

| 維度 | 說明 |
|------|------|
| **本質** | 在不改變基本假設的前提下，修正行為 |
| **問題** | 「我們做對了嗎？」|
| **例子** | 銷量下降 → 增加廣告預算（不問產品本身） |
| **局限** | 治標不治本——反覆出現的問題需要雙環學習 |

### 2.2 雙環學習 (Double-Loop Learning)

| 維度 | 說明 |
|------|------|
| **本質** | 質疑基本假設和目標——不只是調整行動，而是改變思維框架 |
| **問題** | 「我們做對了事嗎？」|
| **例子** | 銷量下降 → 為什麼用戶不買了？→ 重新定義產品策略 |
| **挑戰** | 需要心理安全——承認「我們的基本假設可能是錯的」 |

### 2.3 三環學習 (Triple-Loop Learning / Deutero-Learning)

| 維度 | 說明 |
|------|------|
| **本質** | 學習如何學習——改進組織的學習機制本身 |
| **問題** | 「我們如何知道我們學到了？我們的學習過程有效嗎？」|
| **例子** | 復盤制度本身需要被復盤——「我們的復盤真的在產生改變嗎？」 |

## 3. 學習型組織的五項修煉

Peter Senge 在 *The Fifth Discipline* 中提出學習型組織需要五項核心能力：

| 修煉 | 說明 | 關鍵實踐 |
|------|------|----------|
| **1. 系統思考** | 看見整體而非局部，識別反饋循環 | 因果迴路圖、系統基模 |
| **2. 個人精進** | 個體持續學習和成長的動力 | 反思實踐、終身學習 |
| **3. 心智模式** | 識別和挑戰深層假設 | 「左手欄」練習、對話 |
| **4. 共同願景** | 不是「老闆的願景」而是真正共享的未來圖景 | 參與式規劃、對話 |
| **5. 團隊學習** | 團隊智商 > 個體智商之和 | 深度匯談、衝突轉化 |

**系統思考是第五項修煉**——它將前四項整合為一個整體。

## 4. 組織學習的障礙

### 4.1 Senge 的七項學習障礙

| # | 障礙 | 表現 |
|:--:|------|------|
| 1 | **我就是我的職位** | 把自己等同於工作，看不到系統全貌 |
| 2 | **敵人在外部** | 永遠是別人的錯——市場、競爭對手、其他部門 |
| 3 | **掌控的幻覺** | 必須「主動出擊」，而非理解系統的動態 |
| 4 | **執著於事件** | 只對緊急事件反應，忽視漸進的系統變化 |
| 5 | **煮蛙寓言** | 對緩慢變化的威脅缺乏感知 |
| 6 | **從經驗中學習的錯覺** | 我們從未經歷過某些決策的長期後果 |
| 7 | **管理團隊的神話** | 表面和諧、實際迴避真正困難的問題 |

### 4.2 知識管理視角的障礙

| 障礙 | KM 含義 |
|------|---------|
| **知識就是權力** | 隱瞞錯誤 = 組織無法從失敗中學習 |
| **成功歸因偏誤** | 成功時歸因於自己，失敗時歸因於環境——無法準確學習 |
| **制度遺忘** | 項目結束後不復盤，同樣的錯反覆出現 |

## 5. 組織學習與知識管理的關係

```
知識管理 (KM)                  組織學習 (OL)
─────────────                 ─────────────
「知識」的管理和流動           「學習」的過程和轉化

KM 提供基礎設施                OL 提供動力和方向
     │                              │
     └──────── 交匯點 ──────────────┘
              │
     ┌────────┴────────┐
     │ 事後復盤 (AAR)   │  ← 兩者共同的核心實踐
     │ 知識保留         │
     │ 最佳實踐轉移     │
     └─────────────────┘
```
