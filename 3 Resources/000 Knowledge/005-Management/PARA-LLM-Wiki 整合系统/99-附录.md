## G. 性能優化與調整

> **目標**: 確保 Vault 在規模增長時保持可用性，AI Agent 保持高效運作。
> **檢討頻率**: 每季度檢討一次，或於 Vault 規模翻倍時觸發。

### G.1 性能維度總覽

| 維度 | 關鍵指標 | 🟢 健康 | 🟡 注意 | 🔴 危險 |
|------|----------|:------:|:------:|:------:|
| Vault 文件數 | `.md` 總數 | < 5,000 | 5,000-20,000 | > 20,000 |
| Wiki 頁面數 | `wiki/` 下頁面 | < 500 | 500-2,000 | > 2,000 |
| 單次 Context | `/context --quick` 耗時 | < 10s | 10-30s | > 30s |
| AI Token 消耗 | 單次會話平均 | < 50K | 50K-200K | > 200K |
| Wiki-Compile 耗時 | 單 topic 編譯 | < 30s | 30s-2min | > 2min |
| Git 倉庫大小 | `.git/` 目錄 | < 100MB | 100MB-500MB | > 500MB |
| Obsidian 啟動 | 冷啟動時間 | < 3s | 3-10s | > 10s |
| Dataview 查詢 | 單查詢耗時 | < 1s | 1-5s | > 5s |

### G.2 Vault 規模管理

#### G.2.1 文件增長控制

```
問題: 文件數線性增長 → Obsidian 啟動變慢、搜索索引膨脹

策略:
  1. 歸檔優先 — 優先移動到 4 Archives/，而非永久保留在活躍目錄
  2. 合併碎片 — 少於 200 字的獨立筆記，評估合併到上層 MOC
  3. 定期清理 — 季度檢查並清理無價值內容
  4. 限制 Inbox — Inbox > 100 時強制分批處理
```

#### G.2.2 目錄深度控制

```
✅ 推薦: 3 Resources/[topic]/[chapter]/note.md  (深度 4)
⚠️ 注意: 3 Resources/[topic]/[chapter]/[sub]/[sub]/note.md  (深度 6)
🔴 禁止: 超過 6 層嵌套目錄

原因: Obsidian 的文件瀏覽器在深層嵌套時性能下降，
      且 Wikilink 解析路徑過長容易出錯。
```

#### G.2.3 附件管理

| 文件類型 | 存放位置 | 策略 |
|----------|----------|------|
| 圖片 (.png/.jpg) | `attachments/` 或 `[topic]/media/` | 壓縮至 < 500KB |
| PDF | `[topic]/raw/papers/` | 僅保留摘要 .md，PDF 存外部 |
| 大型文件 (>10MB) | **不入庫** | 存雲端，Vault 中僅保留鏈接 |
| 音頻/視頻 | **不入庫** | 轉文字後存 .md |

### G.3 AI Agent 效率優化

#### G.3.1 Context Window 管理

```
問題: AI 會話上下文隨操作累積，token 消耗指數增長

優化策略:
  1. 會話分段 ──── 長任務拆分為多個獨立會話
  2. Context 懶加載 ─ /context --quick（<30s）而非完整加載
  3. 精確範圍 ──── /triage --scope clippings 而非全量掃描
  4. 增量編譯 ──── /wiki-compile --incremental 而非 --force
  5. Shell 代替 Read ─ 統計用 bash 命令（ls, wc -l, grep -c）而非讀取文件內容
```

#### G.3.2 Wiki-Compile 批次策略

```
單次 /wiki-compile 的最佳文件數:

  ≤5 個源文件     →  一次性處理（高效）
  6-15 個源文件    →  分 2 批處理
  16-30 個源文件   →  分 3-5 批處理
  >30 個源文件     →  按 topic 拆分，每批 ≤10

信號: 單次編譯 > 2 分鐘 → 立即拆分
```

#### G.3.3 Token 預算管理

| 操作 | 預估 Token | 備註 |
|------|:---------:|------|
| `/context --quick` | ~2K | 6 指標表，不讀文件 |
| `/context` (完整) | ~8K | 含會話歷史 |
| `/triage` (10 文件) | ~15K | 含讀取+分類+寫入 |
| `/wiki-compile` (5 源文件) | ~20K | 含提取+生成+索引 |
| `/lint --quick` | ~3K | 僅關鍵檢查 |
| `/lint --report` | ~8K | 詳細報告 |

**每日 Token 預算建議**:
- 輕度使用（僅開關機）: ~10K/day
- 中度使用（含 1 次 triage）: ~30K/day
- 重度使用（含多次 compile）: ~80K/day

#### G.4.2 大文件處理

```bash
# 查找 Git 倉庫中的大型文件
git rev-list --objects --all | git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | awk '/^blob/ {print substr($0,6)}' | sort -k2 -n -r | head -20

# 從 Git 歷史中移除大文件（謹慎使用）
# git filter-branch --force --index-filter 'git rm --cached --ignore-unmatch "path/to/large-file.pdf"' --prune-empty --tag-name-filter cat -- --all
```

### G.5 Obsidian 效能

#### G.5.1 插件管理

```
插件效能影響排名（高→低）:

  🔴 高影響: Dataview (大量查詢), Excalidraw (大畫布), Kanban (多面板)
  🟡 中影響: Calendar, Tasks, Templater, QuickAdd
  🟢 低影響: 主題, 語法高亮, 快捷鍵, 命令面板

策略:
  - 僅啟用活躍使用的插件
  - Dataview 查詢加 LIMIT 限制
  - Excalidraw 場景拆分為多個小文件
  - 定期審查：禁用 30 天未使用的插件
```

#### G.5.2 啟動加速

```
1. 限制啟動時加載的文件數:
   .obsidian/app.json → "showUnsupportedFiles": false

2. 延遲加載側邊欄:
   啟動時關閉文件瀏覽器 → 常用文件通過 Quick Switcher (Ctrl+O) 打開

3. 減少開機即渲染的視圖:
   關閉自動打開的圖譜視圖、Kanban 面板
```

#### G.5.3 Dataview 查詢優化

**❌ 避免: 全表掃描無限制**

```dataview
TABLE file.ctime, file.mtime
FROM "3 Resources"
```

**✅ 推薦: 加範圍 + 限制**

```dataview
TABLE file.ctime, file.mtime
FROM "3 Resources/000 Knowledge"
LIMIT 50
```

**✅ 推薦: 使用 FROM 精確路徑，避免遍歷整個 Vault**

```dataview
TABLE tags, created
FROM "1 Projects"
WHERE status = "active"
LIMIT 20
```

### G.7 性能監控清單

每月執行一次：

```bash
# 1. 文件數統計
find "3 Resources" -name "*.md" | wc -l
find "0 Inbox" -name "*.md" | wc -l

# 2. Git 倉庫大小
du -sh .git/

# 3. 附件大小
find . -type f \( -name "*.png" -o -name "*.jpg" -o -name "*.gif" \) -exec du -ch {} + | tail -1

# 4. 大文件掃描 (>5MB)
find . -type f -size +5M -not -path "./.git/*" -exec ls -lh {} \;

# 5. 斷鏈統計
/lint --quick | grep "broken.*wikilink"
```

| 監控項 | 命令 | 記錄到 |
|--------|------|--------|
| 文件數趨勢 | `find . -name "*.md" \| wc -l` | `AI-Log/performance-log.md` |
| Git 大小 | `du -sh .git` | 同上 |
| 附件體積 | `find . -type f ...` | 同上 |
| Dataview 查詢耗時 | Obsidian 控制台 (Ctrl+Shift+I) | 手動記錄 |
| AI Token 消耗 | Claude Code 會話報告 | 會話結束時自動 |
