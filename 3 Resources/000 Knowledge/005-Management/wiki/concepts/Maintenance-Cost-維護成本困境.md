# 維護成本困境 (Maintenance Cost Problem)

## 定義

**維護成本困境**指知識管理系統中，維護成本隨筆記量線性增長，最終超過用戶可承受的閾值，導致系統被放棄。這是傳統 KM/PKM 方法論（包括 PARA 和 Zettelkasten）共同面臨的根本性缺陷。

## 核心原理

```
筆記數量      維護成本        用戶體驗
────────     ────────       ────────
   50   →      輕鬆      →   愉快
  500   →      有壓力     →   開始拖延
 5000   →      失控      →   打開軟體就焦慮
50000   →      不可能     →   徹底放棄
```

### 為什麼傳統方法無法解決

| 方法 | 如何組織 | 為什麼最終會失控 |
|------|----------|-----------------|
| **PARA** | 按行動分類 | 邊界模糊——一個文件可能同時屬於多個類別 |
| **Zettelkasten** | 原子卡片 + 鏈接 | 卡片越多，找到正確的插入位置越難 |
| **文件夾分類** | 層級樹 | 分類法永遠趕不上知識的演化速度 |

**根本原因**: 這些方法假設**人類會持續投入時間進行組織**。當筆記量超過閾值，這個假設就崩塌了。

## 解法

**AI 知識編譯**（[[Knowledge-Compilation-知識編譯\|知識編譯]]）將維護工作轉移給 AI：
- 人類只負責捕獲（raw/）和消費（wiki/）
- AI 承擔分類、提取、鏈接、索引的維護工作
