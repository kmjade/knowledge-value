# 04 知識審計 (Knowledge Audit)

> 你不知道你不知道什麼 — 直到你做一次知識審計。

---

## 什麼是知識審計？

**知識審計**是系統性地盤點組織的知識資產：**我們知道什麼？誰知道？知識在哪裡？缺了什麼？**

不是「盤點文件」— 文件只是顯性知識的冰山一角。

---

## 審計的三個維度

| 維度 | 問題 | 代表 |
|:-----|:-----|:-----|
| **知識庫存** Knowledge Stock | 我們擁有什麼知識？ | 專利·SOP·專家·關係網絡 |
| **知識流量** Knowledge Flow | 知識如何在組織內流動？ | 誰問誰？哪個會議最有價值？ |
| **知識差距** Knowledge Gap | 我們缺什麼？誰快離職了？ | 即將退休的專家·新業務所需技能 |

---

## 審計流程

```
1. 定義範圍 → 2. 選擇方法 → 3. 收集數據 → 4. 分析 → 5. 報告 → 6. 行動
```

### 常用方法

| 方法 | 適合 | 產出 |
|:-----|:-----|:-----|
| **問卷調查** | 大規模·量化 | 知識地圖·熱力圖 |
| **深度訪談** | 關鍵專家 | 隱性知識清單 |
| **焦點小組** | 特定領域 | 共識·差距·優先級 |
| **社交網絡分析 SNA** | 知識流動 | 誰是 hub？誰是 bottleneck？ |
| **文檔分析** | 顯性知識 | 文件品質·重複·缺口 |
| **觀察/shadowing** | 日常工作 | 實際知識使用模式 |

---

## 知識地圖 (Knowledge Map)

| 類型 | 描述 | 範例 |
|:-----|:-----|:-----|
| **Know-who** 誰知道 | 專家·關鍵人 | 專家黃頁 |
| **Know-what** 什麼 | 文件·數據·專利 | 知識資產目錄 |
| **Know-how** 怎麼做 | 流程·方法·技巧 | 流程圖·SOP 庫 |
| **Know-why** 為什麼 | 原理·解釋·因果 | 決策記錄·AAR |
| **Know-where** 在哪裡 | 物理/數字位置 | 文件索引·導航 |

---

## 關鍵指標

| 指標 | 公式/說明 | 代表什麼 |
|:-----|:---------|:---------|
| **知識覆蓋率** | 有文檔的關鍵流程 / 總關鍵流程 | 顯性化程度 |
| **關鍵人依賴度** | 只有一個人懂的領域數 | 單點故障風險 |
| **知識重複率** | 重複文檔數 / 總文檔數 | 混亂程度 |
| **知識新鮮度** | 12 個月內更新過的文檔比例 | 知識過時風險 |
| **離職風險暴露** | 3 年內退休的專家持有的獨有知識 | 知識流失風險 |

---

## 審計後的典型發現

| 發現 | 頻率 |
|:-----|:----:|
| 「我們以為有 SOP，其實根本沒人照著做」 | 很高 |
| 「那個只有老王知道」 | 很高 |
| 「同一個主題有 5 份互相矛盾的文件」 | 中高 |
| 「新人的 onboarding 靠口耳相傳」 | 中 |
| 「我們不知道自己已經解決過這個問題」 | 很高 |
