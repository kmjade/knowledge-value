# 02 — SECI 模型 (SECI Model)

> 「我們知道的多於我們能說的」——Michael Polanyi, *The Tacit Dimension* (1966)

---

## 1. 理論概述

**SECI 模型**由 Ikujiro Nonaka 與 Hirotaka Takeuchi 於 1995 年在 *The Knowledge-Creating Company* 中提出，描述了組織中**隱性知識與顯性知識之間的四種轉化模式**，構成一個持續上升的「知識螺旋」。

### 核心命題

知識創造不是「管理已有知識」，而是**在隱性與顯性之間不斷轉化的動態過程**。組織的競爭優勢來自於它創造新知識的能力，而非僅僅利用現有知識。

---

## 2. 四種轉化模式

```
        隱性 ───────────────→ 顯性
         │                      │
  隱性   │   S (社會化)         │   E (外化)
         │   Tacit → Tacit     │   Tacit → Explicit
         │   師徒·觀察·模仿     │   隱喻·模型·手冊
         │                      │
  ───────┼──────────────────────┼─────────
         │                      │
  顯性   │   I (內化)           │   C (組合)
         │   Explicit → Tacit  │   Explicit → Explicit
         │   學習·實踐·內化     │   整合·分類·數據庫
         │                      │
```

### 2.1 社會化 (Socialization) — 隱性 → 隱性

**定義**: 通過**共享體驗**直接傳遞隱性知識，不經過語言或文字。

| 機制 | 示例 |
|------|------|
| 師徒制 | 學徒觀察師傅的手藝，逐步內化 |
| 在職訓練 (OJT) | 新人跟著老員工做，邊看邊學 |
| 頭腦風暴 | 團隊在共同情境中激發新想法 |
| 非正式交流 | 「茶水間談話」傳遞的經驗與直覺 |

**關鍵**: 共享的**物理空間和時間**——沒有共同體驗，隱性知識無法傳遞。

### 2.2 外化 (Externalization) — 隱性 → 顯性

**定義**: 將隱性知識**表達**為顯性形式——這是 SECI 中**最關鍵也最困難**的一步。

| 機制 | 示例 |
|------|------|
| 隱喻與類比 | 「組織像一個有機體」——用已知理解未知 |
| 概念建模 | 將經驗提煉為可溝通的模型 |
| 文件化 | 老員工把經驗寫成操作手冊 |
| 對話與反思 | 通過問答將直覺轉化為明確表述 |

**挑戰**: 人類的知識大部分是隱性的，外化本質上是「翻譯」——永遠會有損耗。

### 2.3 組合 (Combination) — 顯性 → 顯性

**定義**: 將**分散的顯性知識整合**為更系統化的新知識。

| 機制 | 示例 |
|------|------|
| 數據庫整合 | 合併不同部門的報告形成公司白皮書 |
| 文獻綜述 | 整理多篇研究，提煉新見解 |
| 教育培訓 | 將分散的知識點編成課程體系 |
| 知識圖譜 | 將文檔間的關聯結構化 |

**關鍵**: 組合不是簡單的「拼裝」——通過重新配置和分類，可以**湧現出新知識**。

### 2.4 內化 (Internalization) — 顯性 → 隱性

**定義**: 將顯性知識通過**實踐和學習**轉化為個人的隱性知識。

| 機制 | 示例 |
|------|------|
| 做中學 (Learning by Doing) | 看手冊的同時實際操作 |
| 模擬訓練 | 飛行模擬器——顯性規則 → 直覺反應 |
| 案例學習 | 分析案例，內化為自己的判斷力 |
| 反思實踐 | 做完後回顧，將經驗內化 |

**關鍵**: 內化是 SECI 循環的**終點也是起點**——內化的隱性知識為新一輪社會化提供了素材。

---

## 3. Ba（場）— 知識創造的共享空間

Nonaka 引入了日本哲學概念 **Ba（場）**，指知識創造所需的**共享情境**：

| 類型 | 對應 SECI | 說明 |
|------|:--------:|------|
| **發起 Ba** | S (社會化) | 面對面的物理空間——共享感受與經驗 |
| **對話 Ba** | E (外化) | 對話與反思的空間——將隱性知識語言化 |
| **系統 Ba** | C (組合) | 虛擬空間——協作平台、知識庫、內部 Wiki |
| **實踐 Ba** | I (內化) | 行動中的學習空間——模擬、在職訓練 |

---

## 4. 知識螺旋 (Knowledge Spiral)

SECI 不是線性的一次性過程，而是**螺旋上升**的：

```
個體層面:  S → E → C → I → (產生新的隱性知識)
                ↓
團隊層面:  S → E → C → I → (擴散到更多人)
                ↓
組織層面:  S → E → C → I → (制度化、文化化)
                ↓
跨組織層面: S → E → C → I → (行業標準、生態系統)
```

每一次循環，知識的**廣度**（更多人共享）和**深度**（更豐富的內涵）都在增加。

---

## 5. 對知識管理的啟示

| 啟示 | 說明 |
|------|------|
| **別只建數據庫** | 顯性化（組合）只是 SECI 的 1/4——最關鍵的隱性知識轉化需要人與人的互動 |
| **創造「場」比創造「內容」更重要** | 給人們交流的空間和理由 |
| **外化是最難的一步** | 把專家的直覺變成新手能懂的指南，需要專門的方法 |
| **KM 不是一次性項目** | SECI 是持續的螺旋，不是「做完了」就結束的流程 |
