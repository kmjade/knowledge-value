# 02 — 核心工作流與信息生命週期

## 1. 整體流程

```
外部輸入
    │
    ▼
┌───────────────────────────────────────────────────────────────────┐
│                  捕獲 (Capture)                                    │
│                                                                   │
│  Web Clipper 截取                                                  │
│  手動粘貼/快速筆記     ──────────────────────? 0 Inbox/              │
│  AI 對話/聊天記錄                                                 │
│  文件導入                                                         │
└───────────────────────────────────────────────────────────────────┘
                     │
                     ▼ /triage
┌───────────────────────────────────────────────────────────────────┐
│                  分診 (Triage)                                    │
│                                                                   │
│  1. 掃描 Inbox                                                    │
│  2. 分析內容（時效性/主題/意圖）                                  │
│  3. 路由決策                                                     │
│  4. 文件移動                                                     │
│  5. Frontmatter 標記                                              │
│  6. 日誌記錄                                                     │
└───────────────────────────────────────────────────────────────────┘
                     │
          ┌──────────┼──────────┐
          │          │          │
    ephemeral  operational   reference
          │          │          │
          │          │          │
  1 Projects/  1 Projects/  3 Resources/
  tasks.md     筆記目錄     [topic]/raw/
          │          │          │
          │          │          │ /wiki-compile
          │          │  ┌───────────────────────────────────────────┐
          │          │  │          編譯 (Compile)                   │
          │          │  │                                           │
          │          │  │  1. 讀取 raw/ 源文件                      │
          │          │  │  2. 提取概念/實體/關係                    │
          │          │  │  3. 創建/更新 wiki/ 頁面                   │
          │          │  │  4. 更新 index.md                         │
          │          │  │  5. 記錄 compile-log.md                   │
          │          │  └───────────────────────────────────────────┘
          │          │                  │
          │          │                  ▼
          │          │         3 Resources/[topic]/wiki/
          │          │
          └──────────┴─────────────────────────────? 用戶查詢
```

## 2. Triage 信息分診流程

```
輸入: 0 Inbox/ 新的 .md 文件
    │
    ▼
┌────────────────────────┐
│ 1. 過濾                │  排除 _processed/ 或已標記 triaged:true 的文件
└────────────────────────┘
         │
┌────────────────────────┐
│ 2. 分類                │
│                        │
│ 時效性分析:             │  關鍵詞匹配 + 內容語境 + Frontmatter
│  ├─ ephemeral          │  "todo", "task", "提醒", "記得"...
│  ├─ operational        │  "meeting", "會議", "note", 筆記...
│  ├─ reference          │  URL, 論文, 書籍, 含 source
│  └─ evergreen          │  長期定義, 原則說明, 定理公式
│                        │
│ 主題識別:              │  關鍵詞權重匹配
│  ├─ ai-ml              │  LLM, GPT, ML, AI, Agent...
│  ├─ people             │  人名, 聯繫方式, 職位...
│  ├─ finance            │  投資, 股票, 期權...
│  └─ productivity       │  GTD, 筆記, 方法...
└────────────────────────┘
         │
┌────────────────────────┐
│ 3. 路由                │
│                        │
│ ephemeral    → 1 Projects/[項目]/tasks.md           │
│ operational → 1 Projects/[項目]/                    │
│ reference   → 3 Resources/[主題]/raw/               │
│ evergreen   → 3 Resources/[主題]/wiki/              │
│ 人物信息    → 3 Resources/people/raw/               │
└────────────────────────┘
         │
┌────────────────────────┐
│ 4. 執行                │
│                        │
│ - 複製/移動文件         │
│ - 更新 Frontmatter     │
│ - 記錄 triage-log      │
└────────────────────────┘
```

## 3. Wiki-Compile 知識編譯流程

```
輸入: 3 Resources/[topic]/raw/ 目錄
    │
    ▼
┌───────────────────────────────────────────────────────────────┐
│ 1. 掃描                           │
│                                   │
│ raw/articles/    → 未編譯的文件    │
│ raw/papers/      → 未編譯的文件    │
│ raw/books/       → 未編譯的文件    │
│ raw/conversations/ → 未編譯的文件  │
│                                   │
│ 判斷標準: compiled:true frontmatter │
└───────────────────────────────────────────────────────────────┘
                 │
┌───────────────────────────────────────────────────────────────┐
│ 2. 提取                           │
│                                   │
│ 概念提取:                          │
│ - 識別關鍵詞和術語                 │
│ - 提取定義和解釋                   │
│ - 判斷概念層級（主題/子主題）       │
│                                   │
│ 實體提取:                          │
│ - 識別命名實體                     │
│ - 提取屬性                         │
│ - 分類（人物/組織/產品/工具）       │
│                                   │
│ 關係提取:                          │
│ - 概念之間: is-a / part-of / related │
│ - 概念-實體: implements / uses / created │
│ - 實體之間: collaborates / competes  │
└───────────────────────────────────────────────────────────────┘
                 │
┌───────────────────────────────────────────────────────────────┐
│ 3. 生成                           │
│                                   │
│ wiki/concepts/[slug].md           │
│ wiki/entities/[slug].md           │
│ wiki/sources/source-[slug].md     │
│                                   │
│ 衝突處理: 優先保留現有頁面          │
│ 新增內容: 合併信息, 標註新來源      │
│ 交叉鏈接: [[ ]] 自動關聯           │
└───────────────────────────────────────────────────────────────┘
                 │
┌───────────────────────────────────────────────────────────────┐
│ 4. 更新索引與日誌                  │
│                                   │
│ wiki/index.md  → 最新統計和目錄    │
│ wiki/log.md    → 記錄本次編譯操作  │
│ AI-Log/compile-log.md → 全局日誌   │
└───────────────────────────────────────────────────────────────┘
```

## 4. 信息生命週期模型

### 4.1 四階段模型

```
                    ┌───────────────────────────────────────────────┐
                    │                                               │
    ephemeral ────────?  1-7 天                                       │
    (臨時)          │  "明天發郵件", "下午開會"                  │
                    │                                               │
         │          │                                               │
         │          │                                               │
    operational ───?  1 天 - 3 個月                                  │
    (操作型)        │  項目筆記, 項目事件, 項目草稿               │
                    │                                               │
         │          │                                               │
         │          │                                               │
    reference ─────?  3 個月 - 永久                                  │
    (參考)          │  文章, 書摘, 網頁存檔                      │
                    │                                               │
         │          │                                               │
         │          │                                               │
    evergreen ─────?  永久                                           │
    (常青)          │  核心概念知識, Wiki 頁面                    │
                    │                                               │
                    └───────────────────────────────────────────────┘
```

### 4.2 各類型判斷維度

| 判斷維度 | ephemeral | operational | reference | evergreen |
|----------|-----------|-------------|-----------|-----------|
| **時效性** | 高（幾天） | 中（數天-數月） | 低（數年） | 很低（永久） |
| **可複用性** | 不可複用 | 項目內複用 | 跨項目複用 | 通用知識 |
| **結構化程度** | 低 | 中 | 中 | 高 |
| **是否需要編譯** | 否 | 否 | 是 | 已經是 |
| **存儲位置** | tasks.md | 項目目錄 | raw/ | wiki/ |
| **歸檔時機** | 完成後 | 項目完成後 | 長期保留 | 永久保留 |

### 4.3 自動清理機制

```
ephemeral
  →  7 天未修改
  →  [_processed/ 歸檔]
  →  30 天後
  →  自動刪除或手動確認刪除

operational
  →  項目完成後
  →  4 Archives/

reference
  →  定期編譯
  →  wiki/（保留 raw/ 作為源文件）
```
