## 2. Wiki 子庫規範

### 2.1 通用結構

```
3 Resources/[topic]/
├── CLAUDE.md          # 子庫 Schema 定義
├── raw/               # 原始資料（用戶維護）
│   ├── articles/      #   文章摘錄
│   ├── papers/        #   論文筆記
│   ├── books/         #   書籍筆記
│   └── conversations/ #   對話記錄
├── wiki/              # LLM 結構化知識（AI 維護）
│   ├── index.md       #   知識地圖
│   ├── log.md         #   編譯日誌
│   ├── concepts/      #   概念頁面
│   ├── entities/      #   實體頁面
│   └── sources/       #   來源溯源頁面
└── outputs/           # 基於 Wiki 生成的產物
```

### 2.2 Wiki 頁面類型規範

#### 概念頁面 (concepts/)

```yaml
---
aliases:           # 同義詞列表
created: YYYY-MM-DD
type: concept
topic: [topic_name]
status: draft | reviewed | evergreen
---
```

**必備章節**:
- `## 定義` — 一句話定義
- `## 核心原理` — 詳細說明
- `## 相關概念` — `[[]]` 鏈接列表
- `## Sources` — 來源追溯

#### 實體頁面 (entities/)

```yaml
---
aliases:
created: YYYY-MM-DD
type: entity
entity_type: person | company | tool | paper | product
topic: [topic_name]
---
```

**必備章節**:
- `## 基本信息` — 位置/時間/狀態
- `## 詳情` — 詳細描述
- `## 關聯` — 相關列表
- `## Sources` — 來源追溯

#### 來源頁面 (sources/)

```yaml
---
source_url: ""
source_author: ""
source_date: YYYY-MM-DD
created: YYYY-MM-DD
type: source
---
```

**必備章節**:
- `## 摘要` — 內容摘要
- `## 關鍵論點` — 主要觀點
- `## 衍生概念` — 由此產生的概念列表
- `## 引用頁面` — 引用了該來源的頁面

### 2.3 四個子庫示例

| 子庫 | Schema | 狀態 | 核心概念 | 實體類型 |
|------|--------|------|----------|----------|
| **ai-ml** | CLAUDE.md | 活躍 | LLM, Transformer, Agent, RAG | 模型, 公司, 工具, 論文 |

| **productivity** | CLAUDE.md | 活躍 | GTD, PARA, 時間管理 | 工具, 方法, 框架 |

## 3. Skills 接口規範

### 3.1 通用結構

```
Skill 目錄結構:
.claude/skills/[name]/
├── [name].md        ← 指令文件（Markdown）
│   ├── 使用方式
│   ├── 執行流程
│   ├── 輸出格式
│   └── 示例
└── skill.json       ← 註冊文件
    ├── name
    ├── description
    ├── instructions (指向 .md 文件)
    ├── commands (註冊命令列表)
    └── enabled
```

### 3.2 Triage Skill

```
命令: /triage [options]
範圍: 0 Inbox/ 目錄
輸出: AI-Log/triage-log.md + 文件移動

選項:
  --dry-run           預覽執行不修改
  --file "name.md"    針對單個文件
  --scope clippings   限定掃描範圍
  --scope fleeting    限定掃描範圍

返回格式:
{
  "timestamp": "ISO8601",
  "summary": { "total": N, "processed": N, "remaining": N },
  "classification": { "ephemeral": N, "operational": N, ... },
  "routing": [ { "file": "...", "target": "...", "status": "..." } ],
  "errors": []
}
```

### 3.3 Wiki-Compile Skill

```
命令: /wiki-compile [topic] [options]
範圍: 3 Resources/[topic]/raw/
輸出: 3 Resources/[topic]/wiki/ + 日誌

選項:
  --dry-run           僅預覽提取結果
  --incremental       只處理新增/修改文件
  --force             重新編譯所有文件

返回格式:
{
  "timestamp": "ISO8601",
  "topic": "ai-ml",
  "summary": {
    "source_files": N,
    "new_concepts": N,
    "new_entities": N,
    "updated_pages": N
  },
  "created": [ "file1.md", "file2.md" ],
  "updated": [ "file3.md" ],
  "errors": []
}
```

### 3.4 Context Skill

```
命令: /context [options]
輸出: Markdown 格式的狀態摘要

選項:
  --quick         快速狀態總覽
  --projects      看項目狀態
  --inbox         看 Inbox 狀態
  --summary       整體摘要

返回格式:
  Markdown 文本（Vault 狀態、活躍項目等）
```

### 3.5 Lint Skill

```
命令: /lint [options]
輸出: Markdown 格式檢查報告

選項:
  --quick         快速檢查（僅關鍵項）
  --fix           自動修復可修復問題
  --report       詳細報告

返回格式:
{
  "timestamp": "ISO8601",
  "health_score": 0-100,
  "checks": { "passed": N, "warnings": N, "errors": N },
  "details": [
    { "category": "...", "status": "pass|warn|error", "message": "..." }
  ],
  "fixable": [ { "issue": "...", "auto_fix": true|false } ]
}
```

## 4. Frontmatter 標準

### 4.1 Wiki 頁面標準 Frontmatter

```yaml
---
# 必須字段
aliases:              # 同義詞組，方便模糊匹配
  - 中文概念名
  - English Concept Name
created: 2026-05-27   # YYYY-MM-DD 創建日期
type: concept          # concept | entity | source | wiki-index | compile-log
topic: ai-ml           # ai-ml | people | finance | productivity

# 可選字段
modified: 2026-05-27  # 最後修改日期
status: reviewed       # draft | reviewed | evergreen
tags:                  # 標籤組
  - ai
  - llm
---
```

### 4.2 原始資料標準 Frontmatter

```yaml
---
# 必須字段
created: 2026-05-27
source: "https://url"  # 來源 URL
tags:
  - clippings

# 可選字段
title: "文章標題"
author: "作者"
published: 2026-05-20   # 原始發布日期
description: "簡要摘要"
---
```

### 4.3 分診標記

分類完成後的文件標記：
```yaml
---
triaged: true
triaged_at: 2026-05-27T14:30:00
triaged_to: 3 Resources/ai-ml/raw/articles/
original_path: 0 Inbox/Clippings/original-name.md
---
```

### 4.4 編譯標記

編譯完成的源文件標記：
```yaml
---
compiled: true
compiled_at: 2026-05-27T15:00:00
compiled_to:
  - concepts/LLM-Pricing-Model
  - entities/OpenAI
---
```

### 4.5 類型枚舉

| type 值 | 說明 | 位置 |
|---------|------|------|
| `concept` | 概念頁面 | wiki/concepts/ |
| `entity` | 實體頁面 | wiki/entities/ |
| `source` | 來源頁面 | wiki/sources/ |
| `wiki-index` | 索引頁面 | wiki/index.md |
| `compile-log` | 編譯日誌 | wiki/log.md |
| `wiki-schema` | 子庫 Schema | CLAUDE.md |
| `triage-log` | 分診日誌 | AI-Log/ |
| `compile-log` | 編譯日誌（全局） | AI-Log/ |
| `session` | 會話記錄 | AI-Log/sessions/ |
| `implementation-report` | 實施報告 | AI-Log/ |
