# 低摩擦知識捕獲 (Low-Friction Capture)

## 定義

**低摩擦捕獲**是 PKM 系統設計的核心原則：知識捕獲的門檻必須極低（理想情況 ≤ 2 秒），否則用戶會因為摩擦而放棄記錄，導致知識流失。

## 核心原理

### 摩擦成本的構成

| 摩擦來源 | 表現 | 解法 |
|----------|------|------|
| **操作步驟** | 打開 App → 選文件夾 → 選模板 → 寫標題 → 寫內容 | 一鍵捕獲，自動歸檔 |
| **決策負擔** | 「這條放哪個文件夾？」「用什麼標籤？」 | 先捕獲再分類（Inbox 模式） |
| **格式要求** | 必須按模板、必須有標題、必須分類 | 接受原始格式，後續處理 |
| **平台限制** | 手機上不方便、離線不能記錄 | 全平台覆蓋 + 離線支持 |

### 捕獲→處理分離

```
捕獲階段（< 2 秒）              處理階段（有空時）
─────────────────             ─────────────────
- 一句話                      - 分類
- 一張截圖                    - 標籤
- 一條語音                    - 鏈接
- 一個鏈接                    - 提煉
      │                            │
      └──── 全部進入 Inbox ────────┘
```

## 應用場景

- **閱讀時**: 看到有價值的段落 → 一鍵分享到 Obsidian Inbox
- **會議中**: 快速記錄關鍵詞，不做完整筆記
- **散步時**: 語音轉文字記錄靈感
