# Software 學習路徑 Learning Path

> 從零到資深軟體工程師的系統學習路線。
> A structured learning path from beginner to senior software engineer.

---

## 路線總覽 Path Overview

```
Phase 1          Phase 2           Phase 3            Phase 4
入門基礎          中階實務            進階架構             專家視野
(2–4 weeks)      (4–8 weeks)        (8–12 weeks)       (ongoing)
────────────────────────────────────────────────────────────────
  Foundations  →  Engineering   →   Architecture    →   Leadership
  軟體工程基礎       工程實務             系統架構              技術領導
```

---

## Phase 1：入門基礎 Foundations (2–4 weeks)

**目標 Goal**: 理解軟體工程的核心概念與思維方式

| 順序 | 模組 | 學習重點 | 預計時間 |
|------|------|---------|---------|
| 1.1 | [[01-Software Engineering Overview]] | 什麼是軟體工程？Software crisis 與 Brooks 法則 | 3–5 hrs |
| 1.2 | [[02-Programming Paradigms]] | OOP / FP / Procedural / Reactive 四大範式對比 | 5–8 hrs |
| 1.3 | [[09-Open Source & Licensing]] | 開源運動歷史、MIT vs GPL vs Apache | 2–3 hrs |

**檢驗 Checkpoint**: 能用自己的話解釋軟體工程與純程式設計的區別，能說出至少 3 種程式設計範式的核心差異。

---

## Phase 2：中階實務 Engineering Practice (4–8 weeks)

**目標 Goal**: 掌握現代軟體開發的工程實務與協作流程

| 順序 | 模組 | 學習重點 | 預計時間 |
|------|------|---------|---------|
| 2.1 | [[04-Development Methodologies]] | Agile / Scrum / Kanban，從 Waterfall 到 DevOps | 5–8 hrs |
| 2.2 | [[05-Software Testing]] | 測試金字塔、TDD/BDD、automation frameworks | 8–12 hrs |
| 2.3 | [[06-Version Control & Collaboration]] | Git 工作流、CI/CD Pipeline、Code Review | 8–10 hrs |
| 2.4 | [[07-Software Design Patterns]] | GoF 23、SOLID、DRY/KISS/YAGNI | 10–15 hrs |

**檢驗 Checkpoint**: 能在團隊中實踐 Scrum，編寫單元測試與整合測試，熟練使用 Git 分支策略與 CI/CD Pipeline。

---

## Phase 3：進階架構 Architecture (8–12 weeks)

**目標 Goal**: 具備系統架構設計與技術決策能力

| 順序 | 模組 | 學習重點 | 預計時間 |
|------|------|---------|---------|
| 3.1 | [[03-Software Architecture]] | Monolith → Microservices、Clean/Hexagonal、Event-Driven | 10–15 hrs |
| 3.2 | [[08-Programming Language Ecosystems]] | Python/JS/Go/Rust 生態系統深度對比 | 8–12 hrs |
| 3.3 | [[09-Open Source & Licensing]] (revisit) | Contributor workflow、社群治理、Inner Source | 3–5 hrs |

**檢驗 Checkpoint**: 能設計一個中型系統的架構並說明 trade-off，能根據專案需求選擇合適的技術棧。

---

## Phase 4：專家視野 Leadership (ongoing)

**目標 Goal**: 技術領導力、架構治理、開源貢獻

| 順序 | 主題 | 說明 |
|------|------|------|
| 4.1 | 技術寫作 Technical Writing | ADR (Architecture Decision Records)、RFC、技術部落格 |
| 4.2 | 開源貢獻 Open Source Contribution | 選擇專案、提交 PR、維護經驗 |
| 4.3 | 技術指導 Mentorship | Code Review 文化、Pair Programming、Knowledge Sharing |
| 4.4 | 持續學習 Continuous Learning | 追蹤技術趨勢、參加 Conference、閱讀論文 |
