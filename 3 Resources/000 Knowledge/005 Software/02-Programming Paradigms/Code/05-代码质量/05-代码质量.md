# 05-代码质量

## Clean Code 原则

| 原则 | 说明 |
|------|------|
| 好命名 | 变量/函数名应自解释——不需要注释 |
| 小函数 | 一个函数只做一件事——<20行 |
| 少参数 | 参数≤3个——多了用对象 |
| DRY | 不重复——每个知识点只表达一次 |
| KISS | 简单——复杂的方案通常是错的 |

## 重构清单

| 味道 | 重构 |
|------|------|
| 长函数 | Extract Method |
| 大类 | Extract Class |
| 重复代码 | Extract Method/Class |
| 过多参数 | Introduce Parameter Object |
| Switch/if-else 泛滥 | Replace with Polymorphism |

## 代码审查

| 看什么 | 问什么 |
|--------|--------|
| 逻辑 | 这样写对吗？ |
| 设计 | 这样设计合理吗？ |
| 命名 | 名字说清楚了吗？ |
| 测试 | 有测试吗？覆盖到关键路径了吗？ |
