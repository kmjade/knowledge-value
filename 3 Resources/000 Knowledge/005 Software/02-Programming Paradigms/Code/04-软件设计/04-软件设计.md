# 04-软件设计

## SOLID

| 原则 | 含义 |
|------|------|
| **S** | 单一职责——一个类只做一件事 |
| **O** | 对扩展开放、对修改关闭 |
| **L** | 子类可替代父类 |
| **I** | 接口小而专注 |
| **D** | 依赖抽象而非具体 |

## 常用设计模式

| 模式 | 用途 | 例子 |
|------|------|------|
| **Singleton** | 全局唯一实例 | 日志、配置 |
| **Factory** | 创建对象不暴露逻辑 | DI 容器 |
| **Observer** | 一对多通知 | 事件系统 |
| **Strategy** | 运行时切换算法 | 支付方式 |
| **Decorator** | 动态添加职责 | Python `@` 装饰器 |

## 架构模式

| 模式 | 适合 |
|------|------|
| MVC/MVVM | 前端/App |
| 分层 | 企业后端 |
| 微服务 | 大型分布式 |
| 事件驱动 | 异步/消息 |
