# 07-版本控制与协作

## Git 核心

| 概念 | 说明 |
|------|------|
| commit | 保存快照——附信息 |
| branch | 并行开发线 |
| merge/rebase | 合并分支 |
| PR (Pull Request) | 代码审查入口 |

## Git 工作流

| 工作流 | 适合 |
|--------|------|
| **GitHub Flow** | 小团队——main + feature 分支 |
| **Git Flow** | 有版本发布——develop/release/hotfix |
| **Trunk-Based** | 持续部署——直接提交 main |

## CI/CD

```
Push → Build → Test → (Merge) → Deploy
```

| 工具 | 特点 |
|------|------|
| GitHub Actions | 集成 GitHub |
| GitLab CI | 一体化 |
| Jenkins | 老牌 |

## 提交信息规范

```
type(scope): 简短描述

feat(auth): add login
fix(api): fix timeout
docs(readme): update
```
