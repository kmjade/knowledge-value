# DDC：杜威十进分类法

## 基本信息

- **全称**: Dewey Decimal Classification（杜威十进分类法）
- **创始人**: [[Karl-Popper|Melvil Dewey]]（1851–1931）
- **首次出版**: 1876 年（当时仅 42 页）
- **当前版本**: DDC 23（2011 年出版）
- **维护方**: OCLC（Online Computer Library Center）
- **使用范围**: 全球 135+ 个国家、200,000+ 图书馆
- **类型**: 枚举式层级分类法

## 核心特征

### 优势

| 优势 | 说明 |
|------|------|
| **普遍采用** | 事实上的全球标准——跨语言、跨文化广泛使用 |
| **层级直观** | 十进制+层级——读者可以自然地浏览相关主题 |
| **排架效率** | 一本书一个位置——图书馆物理空间管理的最优解 |
| **持续更新** | OCLC 持续维护——DDC 23 反映了数字时代的主题 |

### 局限

| 局限 | 说明 |
|------|------|
| **文化偏见** | Dewey 本人的基督教/西方/男性视角深植于分类中。200 类（宗教）中基督教的篇幅远超其他宗教总和 |
| **更新滞后** | 分类更新远慢于知识前沿——新兴交叉领域直到十几版后才被加入 |
| **单一位置** | 每个主题只有一个"最佳"位置——交叉学科领域被迫"选择"一个家 |
| **非中立** | 分类不是中立的——200 类的结构反映了 Dewey 对宗教的观点 |

### 著名争议

| 争议 | 详情 |
|------|------|
| **LGBTQ 分类历史** | 从"异常性心理"到"性取向"（306.76）——数十年的路线修正 |
| **宗教偏向** | 200 类：220-280 是基督教，290 是"其他宗教"（包括伊斯兰教、佛教、印度教） |
| **数字鸿沟** | DDC 23 在 000 类大量扩展了计算机科学——但新兴领域（如 AI 伦理）仍然难以安置 |

## 历史

| 年份 | 事件 |
|------|------|
| 1873 | Dewey 在 Amherst College 图书馆工作时构思了十进制分类系统 |
| 1876 | 第一版 DDC 出版——仅 42 页，1,000 个分类 |
| 1885 | 第二版——扩展至 314 页 |
| 1988 | OCLC 收购 DDC 的版权 |
| 2003 | DDC 22 出版——引入 Web Dewey |
| 2011 | DDC 23 出版——当前版本 |

## DDC vs UDC

| | DDC | [[UDC]] |
|----|-----|---------|
| 创始人 | Melvil Dewey（美国） | Paul Otlet 和 La Fontaine（比利时，基于 DDC 扩展） |
| 分类策略 | 枚举+层级 | 分面+层级——通过连接符组合类别 |
| 灵活性 | 低——单一位置 | 高——多维度组合 |
| 使用范围 | 公共图书馆和学校图书馆 | 专业/研究图书馆 |
| 地理主导 | 英语世界 | 欧洲、拉丁美洲 |
| 符号 | 纯数字 (336.73) | 数字+符号 (336.73(73)"19") |

## Obsidian 知识管理启示

| DDC 的教训 | Obsidian 的回应 |
|-----------|----------------|
| 树状分类不适合网络状知识 | 使用链接表达真实的知识关联，标签辅助检索 |
| 单一位置原则不适用于数字环境 | 同一概念可属于多个 MOC |
| 分类更新太慢 | 用户维护的标签系统可以即时更新 |
| 中心化分类有文化偏见 | 个人知识管理允许个别化的分类方案 |

> DDC 是物理世界的杰作——在图书馆排架上它是不可替代的。但在数字世界中，知识的组织不应该复制 DDC 的树状逻辑。
