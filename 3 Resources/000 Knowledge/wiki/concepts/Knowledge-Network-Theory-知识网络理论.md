# 知识网络理论

> 知识的价值不在节点本身，而在节点之间的连接。一个孤立的事实是噪音，一张连接的网络是智能。

---

## 核心定义

**知识网络理论**：将个人知识视为一个由**概念节点**和**关系边**构成的网络图。网络的智慧不来自节点数量，来自连接的密度、多样性和结构。

---

## 网络 vs 层级

```
层级结构（文件夹）:              网络结构（wikilink）:

    📁 根                           A ── B
    ├── 📁 A                        │  ╲  │
    │   └── B                       │   ╲ │
    └── 📁 C                        C ── D ── E
        └── D
                                    → B 同时连接 A、C、D
→ B 只能在一个文件夹里              → 多重归类的网络
→ 路径唯一                          → 路径多样
→ 易死（找不到就永远丢失）          → 多条路径可达
```

### 层级的问题

| 问题 | 后果 |
|------|------|
| 单一归类 | `[[机器学习]]` 放进 AI 还是 统计？只能选一个 |
| 路径依赖 | 忘记了文件夹位置 = 永远找不到 |
| 孤岛效应 | 没有跨文件夹连接，知识碎片化 |

### 网络的解法

```
[[机器学习]]  ← 不放在任何地方，而是被多条路径连接
  ├── [[AI-ML/]] 指向它 (从 AI 视角)
  ├── [[Statistics/]] 指向它 (从统计视角)
  ├── [[Projects/富福小车/]] 指向它 (从应用视角)
  └── [[读书笔记/PRML/]] 指向它 (从学习视角)
```

## 三个关键指标

| 指标 | 含义 | 健康值 |
|------|------|:---:|
| **节点数 N** | 概念/笔记总量 | — |
| **连接度 D** | 平均每个节点的出入链数 | ≥ 3 |
| **聚类系数** | 邻居之间也互相连接的程度 | 适中 |

```
低质量网络（高节点、低连接）:
  ●  ●  ●  ●  ●  ●  ●     ← 笔记多但孤立
                            → 能找到内容，但找不到关系

高质量网络（高节点、高连接）:
  ●──●──●──●                ← 笔记多且互相连接
  │╲ │ ╱│ ╱│                → 任意入口都能遍历整个网络
  ●──●──●──●
```

## 网络效应的三个阶段

### 阶段 1：收集期（N < 100）

```
每个节点都值得手动加粗、高亮、写摘要
→ 渐进式总结的黄金期
→ 目标是积累足够的节点
```

### 阶段 2：连接期（100 < N < 500）

```
节点多了，手动连接成本上升
→ wikilink 自动提示已有节点
→ 需要 `/wiki-compile` 辅助建立连接
→ 知识复利开始显现
```

### 阶段 3：涌现期（N > 500）

```
网络自己开始"说话"：
  ├── 写新笔记时自动提示相关旧笔记
  ├── 两个从未直接连接的节点通过共同邻居关联
  └── AI 编译时发现跨域模式 → 生成新概念

→ 不再是你管理知识，而是知识网络在帮你思考
```
