# Classification Systems (分类法)

## 定义

分类法是最古老也是最基础的知识组织系统——将知识划分为层级结构，让相似主题的信息聚集在一起。每本书（或每个概念）在分类体系中有且仅有一个位置。

> 来源: [[3 Resources/000 Knowledge/ram/Knowledge-Systems/02-Classification/02-Classification.md|02-Classification体系]]

## 核心原理

### 四大分类法

| 分类法 | 创建 | 使用范围 | 特征 |
|--------|------|----------|------|
| **DDC** (杜威十进分类法) | 1876 | 135+ 国家 | 纯数字、10 大类 |
| **UDC** (国际十进分类法) | 1905 | 30+ 国家 | 数字+符号、可组合 |
| **CLC** (中图法) | 1975 | 中国 | 字母+数字、22 大类 |
| **LCC** (国会图书馆分类法) | 1897 | 美国研究型图书馆 | 字母+数字、灵活 |

### 三个功能

| 功能 | 说明 |
|------|------|
| **排架** | 让同一主题的书在一起——支持浏览发现 |
| **导航** | 图书馆目录的骨架，用户按学科查找 |
| **统计** | 馆藏学科分布分析 |

### 认知基础

| 概念 | 分类法体现 |
|------|-----------|
| 层级 | 从一般到特殊——600 (技术) → 610 (医学) → 616 (疾病) |
| 互斥 | 每本书只能在一个位置 |
| 穷举 | 所有知识都应有位置 |
| 共识 | 学科划分反映时代共识 |

## 关键要点

1. 分类法的核心是**树状层级**——每个节点只有一个父节点
2. 互斥性原则意味着每个实体只能归入一个类别
3. 学科分类是历史和文化产物，会随时代演变
4. PARA 文件夹本质上是一种"个人分类法"
5. 分类法适合浏览，但不适合多维度检索

## 实践应用

- **本 Vault**: PARA 四层文件夹 = 个人分类法；DDC 编号用于 Resources 子库
- **图书馆**: 物理排架的基石
- **Obsidian**: 文件夹结构是最直观的分类法实现
