# 卡片笔记法

> 一张卡片 = 一个想法。把知识拆成原子，再用链接重组为网络。

---

## 一句话

**不在文件夹里写长篇笔记，而是在独立卡片上记录最小可理解单元，然后用链接把卡片织成网络。**

---

## 三类卡片

| 类型 | 来源 | 处理时限 | 示例 |
|:---:|------|:---:|------|
| 💡 **闪念** | 脑中一闪 | 1-2 天 | "LLM 是编译器不是搜索引擎" |
| 📝 **文献** | 阅读/听课 | 当天 | 用自己的话转述书中的核心论点 |
| 📌 **永久** | 闪念+文献融合 | 持续迭代 | 一条独立的、无论上下文都能理解的原子笔记 |

---

## 卡片格式

```
┌─────────────────────────────┐
│  # 标题（一句话概括）         │
│                              │
│  ## 核心定义                  │
│  1-2 句精确定义               │
│                              │
│  ## 关键要素                  │
│  - 要素 1                    │
│  - 要素 2                    │
│                              │
│  ## 来源                     │
│  [[source-note]]             │
│                              │
│  ## 连接                     │
│  [[related-concept-1]]       │
│  [[related-concept-2]]       │
└─────────────────────────────┘
```

### 在 Obsidian 中的实现

```markdown
# 知识复利效应

一句话：每次投入不只是增加一条笔记，而是放大整张网络。

## 相关
- [[知识网络理论]]
- [[渐进式总结]]
```

> 每张卡片 ≈ 一个 Obsidian 笔记文件。

---

## 五条原则

| # | 原则 | 一句话 |
|:---:|------|------|
| 1 | **原子化** | 一张卡片只讲一件事 |
| 2 | **自主可理解** | 脱离上下文也能读懂 |
| 3 | **始终链接** | 每张卡片至少链接一张已有卡片 |
| 4 | **用自己的话** | 不复制粘贴，转述即理解 |
| 5 | **信任系统** | 写进去就可以忘掉 |

---

## 为什么有效

```
传统笔记：                        卡片笔记：
                                   
📄 一页千言                          📌 知识复利
全在一个主题下                        │
├── 定义                             ├── 📌 知识网络理论
├── 历史                             │   ╱
├── 方法                             ├── 📌 渐进式总结
├── 案例                             │
└── 总结                             └── 📌 第二大脑

→ 查找靠搜索                         → 发现靠遍历
→ 线性结构                           → 网络结构
→ 写完即封闭                         → 持续生长
```

## 与 Vault 系统的对应

| 卡片类型 | Vault 位置 |
|:---:|------|
| 💡 闪念 | `0 Inbox/` — 快速捕获后 `/triage` |
| 📝 文献 | `3 Resources/[topic]/raw/` — 人类维护 |
| 📌 永久 | `3 Resources/[topic]/wiki/concepts/` — AI 编译 |

> LLM-Wiki 的 `/wiki-compile` 将闪念和文献卡片**自动编译**为永久卡片——这是 Luhmann 手工无法做到的规模化。
