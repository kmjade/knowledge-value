# SECI Model

## 定义

SECI 模型是由野中郁次郎 (Ikujiro Nonaka) 提出的组织知识创造理论，描述隐性知识和显性知识之间的四种转化模式：Socialization (社会化)、Externalization (外化)、Combination (组合)、Internalization (内化)。

> 来源: [[3 Resources/000 Knowledge/ram/Knowledge-Systems/07-知识管理/07-知识管理.md|07-知识管理]]

## 核心原理

### 四种转化模式

| 模式 | 转化方向 | 含义 | 例子 |
|------|----------|------|------|
| **S**ocialization | 隐性→隐性 | 通过共享经验传递 | 师徒制、观摩学习 |
| **E**xternalization | 隐性→显性 | 把经验写成文档 | 编写操作手册 |
| **C**ombination | 显性→显性 | 整合多来源显性知识 | 汇总多份报告 |
| **I**nternalization | 显性→隐性 | 学习文档后掌握技能 | 读手册后学会操作 |

### 知识管理工具

| 工具 | 功能 | 对应 SECI 阶段 |
|------|------|---------------|
| Wiki/Confluence | 组织的集体显性知识 | Combination |
| 实践社区 CoP | 同行的非正式交流 | Socialization |
| 事后回顾 AAR | 项目结束的经验沉淀 | Externalization |
| 专家黄页 | "谁知道什么" | Socialization |

## 关键要点

1. 组织知识的创造是一个**螺旋上升**的过程
2. 最困难的转化是**隐性→显性** (Externalization)——把经验写下来
3. 最容易被忽视的是**显性→隐性** (Internalization)——看完不等于学会
4. 本 Vault 的 `/triage` → `/wiki-compile` = Externalization + Combination
5. 阅读 wiki 页面并应用到实践 = Internalization

## 实践应用

- **本 Vault**: raw/ → wiki/ 的编译过程 = SECI 中的 Externalization
- **企业**: Confluence Wiki + 定期 AAR + CoP 活动
- **个人**: 写博客/笔记 (Externalization) → 汇总主题 (Combination) → 实践应用 (Internalization)
