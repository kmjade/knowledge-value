# 渐进式总结

> 不是一次性把笔记写完美，而是分 5 层逐步提炼——只在需要的时候深化到下一层。

---

## 核心定义

**渐进式总结**（Progressive Summarization）：Tiago Forte 提出的笔记处理技术——对同一份笔记分 5 层逐步加粗、高亮、摘要，每层只在"下一次用到这条笔记"时添加。

---

## 五层结构

```
Layer 1 — 原始笔记   ████████████████████████████████   完整原文（捕获时）
Layer 2 — 加粗       ██░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░   首次阅读时加粗关键句
Layer 3 — 高亮       ██▓▓░░░░░░░░░░░░░░░░░░░░░░░░░░░░   再次用到时高亮核心句
Layer 4 — 摘要       ██▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░░░░░░░   用自己话写 2-3 句总结
Layer 5 — 再混合     ██▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░░░░░░░░░░░   与其他笔记融合为新概念
```

| 层 | 动作 | 时机 | 成本 |
|:---:|------|------|:---:|
| L1 | 原文捕获 | 第一时间 | 0 |
| L2 | 加粗关键句 | 首次回读 | 低 |
| L3 | 高亮核心句 | 再次用到 | 低 |
| L4 | 用自己的话总结 | 深度需要 | 中 |
| L5 | 与其他知识融合 | 创建新内容时 | 高 |

---

## 核心原则

### 1. 只在需要时深化

```
❌ 刚读完一篇文章就写 500 字摘要（可能永远用不到）
✅ 第一次回读只加粗 → 第二次用到才高亮 → 真正需要才写摘要
```

### 2. 每一层都为未来自己服务

- L2（加粗）：下次扫读时眼睛自动聚焦
- L3（高亮）：在加粗句中再挑最重要的
- L4（摘要）：脱离原文，形成自己的理解

### 3. 时机驱动，不是计划驱动

不会在日历上标"明天做渐进式总结"。而是每次**自然回读**这条笔记时顺手加一层。

---

## 在 Obsidian 中的实现

|   层   | Obsidian 语法           |
| :---: | --------------------- |
| L2 加粗 | `**关键句**`             |
| L3 高亮 | ` ==核心句== `           |
| L4 摘要 | `> [!summary] 自己的总结`  |
| L5 融合 | `[[概念页]]` — 编译进 wiki/ |

### 示例

```markdown
# 某篇 AI 文章

Layer 1 — 原文捕获的全文

**Layer 2 — AI 正在从检索转向编译** ← 加粗

==Layer 3 — 知识管理的未来是编译而非检索== ← 高亮

> [!summary] Layer 4
> 这篇文章的核心观点：AI 在知识管理中的角色
> 将从"帮我找到"变成"帮我理解并连接"。
> 关键证据：LLM-Wiki 的 wiki-compile 引擎。

→ Layer 5: 融入 [[LLM-Wiki]] 概念页
```

## 与知识复利的关系

```
渐进式总结                     知识复利
─────────────────          ─────────────────
每次回读加一层标记      →   每次投入增加网络价值
摘要脱离原文，形成理解  →   碎片提炼为概念（编译）
Layer 5 融合             →   连接已有知识 = 复利再投资
```

> 渐进式总结是知识复利的**操作手段**——它规定了"怎么投入"才能产生复利。
