# DDC：杜威十进分类法

## 基本信息

- **全称**: Dewey Decimal Classification（杜威十进分类法）
- **创始人**: [[杜威|Melvil Dewey]]（1851–1931）
- **首次出版**: 1876 年（当时仅 42 页）
- **当前版本**: DDC 23（2011 年出版）
- **维护方**: OCLC（Online Computer Library Center）
- **使用范围**: 全球 135+ 个国家、200,000+ 图书馆
- **类型**: 枚举式层级分类法
- **语言**: 英语（有 30+ 种语言的翻译版）

## 在三层模型中的定位

DDC 是学科层**分类维度**的标志性实例：

- **科学层关联**: DDC 500 类（科学）反映了 19 世纪末的科学分工——它反映了那个时代的科学共识，而非今天的
- **知识层关联**: DDC 将网络状的知识强制映射为树状——每个主题只有一个"正确"位置。这在图书馆排架上是必要的（一本书只能在一个书架上），但体现了知识与分类之间的张力
- **学科层关联**: DDC 是全球最广泛使用的学科分类制度——它大规模地塑造了公众对"知识结构"的认知

## 核心特征

### 优势

| 优势 | 说明 |
|------|------|
| **普遍采用** | 事实上的全球标准——跨语言、跨文化广泛使用 |
| **层级直观** | 十进制+层级——读者可以自然地"浏览"相关主题 |
| **排架效率** | 一本书一个位置——图书馆物理空间管理的最优解 |
| **持续更新** | OCLC 持续维护——DDC 23 反映了数字时代的主题 |

### 局限

| 局限 | 说明 | 三层分析 |
|------|------|----------|
| **文化偏见** | Dewey 本人的基督教/西方/男性视角深植于分类中。200 类（宗教）中基督教的篇幅远超其他宗教总和 | 学科层的权力维度 |
| **更新滞后** | 分类更新远慢于知识前沿——新兴交叉领域直到十几版后才被加入 | 学科分类 vs 科学变革（④箭头） |
| **单一位置** | 每个主题只有一个"最佳"位置——交叉学科领域被迫"选择"一个家 | 知识的网络结构与分类的树状结构之间的根本冲突 |
| **非中立** | 分类不是中立的——200 类的结构反映了 Dewey 对宗教的观点 | 分类体系的隐含认识论 |

### 著名争议

| 争议 | 详情 |
|------|------|
| **LGBTQ 分类历史** | 从"异常性心理"（132/136）到"性取向"（306.76）——数十年的路线修正。分类的"滞后"在此期间造成了真实的伤害 |
| **宗教偏向** | 200 类：220 是圣经，230 是基督教神学，240 是基督教实践……290 是"其他宗教"（包括伊斯兰教、佛教、印度教——人类大多数） |
| **数字鸿沟** | DDC 23 在 000 类大量扩展了计算机科学——但新兴领域（如 AI 伦理）仍然难以安置 |

## 历史

| 年份 | 事件 |
|------|------|
| 1873 | Dewey 在 Amherst College 图书馆工作时构思了十进制分类系统 |
| 1876 | 第一版 DDC 出版——仅 42 页，1,000 个分类 |
| 1885 | 第二版——扩展至 314 页 |
| 1927 | 开始采用"凤凰表"（部分分类的完全重写） |
| 1988 | OCLC 收购 DDC 的版权 |
| 2003 | DDC 22 出版——引入 Web Dewey |
| 2011 | DDC 23 出版——当前版本 |

## DDC vs UDC

| | DDC | [[3 Resources/000 Knowledge/epistemology/wiki/entities/UDC]] |
|----|-----|---------|
| 创始人 | [[杜威]]（美国） | [[奥特勒]] 和 La Fontaine（比利时，基于 DDC 扩展） |
| 分类策略 | 枚举+层级 | 分面+层级——通过连接符组合类别 |
| 灵活性 | 低——单一位置 | 高——多维度组合 |
| 使用范围 | 公共图书馆和学校图书馆的王者 | 专业/研究图书馆 |
| 地理主导 | 英语世界 | 欧洲、拉丁美洲 |
| 符号 | 纯数字（336.73） | 数字+符号（336.73(73)"19"） |

## 在 Obsidian 知识管理中的启示

| DDC 的教训 | Obsidian 的回应 |
|-----------|----------------|
| 树状分类不适合网络状知识 | 使用链接表达真实的知识关联，标签辅助检索 |
| 单一位置原则不适用于数字环境 | 同一概念可属于多个 MOC |
| 分类更新太慢 | 用户维护的标签系统可以即时更新 |
| 中心化分类有文化偏见 | 个人知识管理允许个别化的分类方案 |

> DDC 是物理世界的杰作——在图书馆排架上它是不可替代的。但在数字世界中，知识的组织不应该复制 DDC 的树状逻辑。Obsidian 的网络+标签模式是对 DDC 的**回应**，不是 DDC 的数字化。
