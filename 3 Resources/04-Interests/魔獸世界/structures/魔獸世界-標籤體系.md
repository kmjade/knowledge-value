# 魔獸世界-標籤體系

## 🏷️ 標籤設計原則

### 分層結構
```
wow/一級分類/二級分類/粒度標記/版本標記
```

### 標籤格式
- **繁體命名**：採用繁體中文標籤
- **層級明確**：最多4級分類
- **功能性明確**：每個標籤都有明確的查詢用途
- **擴展性好**：預留未來擴展空間

## 🕐 版本標籤

### 主要版本標籤
#### wow/經典懷舊服
- **時間範圍**：1.12.1版本內容
- **特點**：40人NAXX、PVP榮譽系統
- **使用場景**：懷舊服專用內容

#### wow/TBC（燃燒的遠征）
- **時間範圍**：2.x版本內容
- **特點**：25人RAID、飛行系統
- **使用場景**：外域相關內容

#### wow/WLK（巫妖王之怒）
- **時間範圍**：3.x版本內容
- **特點**：10/25人模式、成就系統
- **使用場景**：北裂境相關內容

#### wow/現代
- **時間範圍**：4.0-10.0版本內容
- **特點**：各種現代系統機制
- **使用場景**：現代版本內容

#### wow/地心之战 (The War Within)
- **時間範圍**：12.0版本內容
- **特點**：英雄天賦系統、職業專精重構
- **使用場景**：最新遊戲內容分析

#### wow/12.0版本
- **時間範圍**：地心之战12.0版本
- **特點**：英雄天賦、機制簡化、平衡調整
- **使用場景**：現代服當前版本

### 特殊版本標籤
#### wow/版本演變
- **用途**：比較不同版本差異
- **示例**：wow/版本演變/職業平衡性

## 🏷️ 標籤應用規範

### 文件標籤規則
1. **每個文件至少包含**：一個一級標籤 + 一個二級標籤
2. **推薦組合**：一級 + 二級 + 粒度 + 版本（如需要）
3. **標籤順序**：從概括到具體，順序無關緊要
4. **避免標籤濫用**：每個文件控制在5個以內

### 標籤示例
```
# 暗夜精靈德魯伊職業分析
tags:
  - wow/遊戲機制/職業體系
  - wow/德魯伊
  - wow/中觀
  - wow/WLK

# 宏觀職業體系概覽  
tags:
  - wow/遊戲機制/職業體系
  - wow/宏觀
  - wow/經典懷舊服

# 戰鬥系統機制詳解
tags:
  - wow/遊戲機制/戰鬥系統
```

## 🔄 標籤維護

### 更新頻率
- **基礎分類**：穩定，無需頻繁更新
- **版本標籤**：根據新版本發布調整
- **專門標籤**：根據研究內容擴展

### 質量控制
- **一致性檢查**：定期檢查標籤使用一致性
- **冗餘清理**：移除不必要的標籤
- **擴展規劃**：根據內容發展預留擴展空間
