# 魔獸世界 - 地心之戰 12.0 目录结构设计

> [!info] 目录重构目标
> 根据地心之战 12.0 知识库建设方案，重构魔兽世界知识库目录结构，实现版本独立、深度优先、模块化设计。

---

## 📋 重构目标

### 1. 版本独立性
- 地心之战 12.0 独立于经典版本
- 不干扰现有经典版本知识库
- 清晰的版本标识和导航

### 2. 深度优先
- 按照学习深度组织内容
- 基础 → 核心 → 深度 → 高阶
- 循序渐进的学习路径

### 3. 模块化设计
- 职业模块独立
- 系统模块独立
- 功能模块独立
- 便于维护和扩展

### 4. 分类标准
- 使用 wow/tww 命名体系
- 统一的标签和别名系统
- 标准化的 frontmatter 模板

- **优先级**：P0

- **功能**：地心之战版本更新时间线

## 💡 重构建议

### 1. 保持一致性
- 所有笔记使用统一 frontmatter 模板
- 所有笔记使用统一的标签体系
- 所有笔记使用统一的标题格式

### 2. 建立完整链接
- 每个 MOC 必须链接到相关笔记
- 每个笔记必须双向链接
- 确保链接有效且准确

### 3. 按照深度组织
- 基础知识 → 核心系统 → 技能详解 → 装备指南
- 副本攻略 → PVP 技巧 → 实战技巧 → 高阶进阶
- 循序渐进的学习路径

### 4. 模块独立性
- 每个职业模块完全独立
- 每个系统模块完全独立
- 每个功能模块完全独立
- 便于单独维护和更新
