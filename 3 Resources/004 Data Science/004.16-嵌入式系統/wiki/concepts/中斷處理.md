# 中斷處理 Interrupt Handling

## 定義
中斷使 MCU 能**即時響應外部事件**——硬件信號觸發 CPU 暫停當前任務，執行中斷服務程序 (ISR)，然後返回。

## 中斷流程
硬件事件 → NVIC 仲裁 → 保存上下文 → 執行 ISR → 恢復上下文 → 返回

## ARM Cortex-M NVIC
- **優先級**: 0 (最高) – 255 (最低)，支持巢狀中斷
- **中斷延遲**: 12 時鐘週期 (Cortex-M3/M4)
- **尾鏈 Tail-Chaining**: 連續中斷無需重複保存/恢復

## ISR 設計原則: 越短越好！ISR 中僅做關鍵操作 (設標誌、讀寄存器)，繁重工作推遲到任務/底半部
