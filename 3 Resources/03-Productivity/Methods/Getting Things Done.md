# Getting Things Done (GTD)

> David Allen 的經典時間管理方法論：清空大腦、專注當下、高效執行

---

## 📖 概述

### GTD 是什麼？

GTD (Getting Things Done) 是 David Allen 開發的個人生產力系統，核心思想是：

> **將大腦從記憶和追蹤的負擔中解放出來，讓它專注於思考和創造**

### 核心原則

| 原則 | 說明 |
|------|------|
| 捕獲一切 | 不要依賴記憶，把所有事務記錄下來 |
| 定期處理 | 系統性處理收集的項目 |
| 明確下一步 | 每個項目都有明確的下一步行動 |
| 組織清晰 | 所有項目都有明確歸屬 |
| 定期回顧 | 保持系統新鮮和可靠 |
| 根據情境行動 | 在適合的時間做適合的事 |

---

## 🔄 GTD 工作流程

### 五步法核心流程

```
捕獲 → 阐明 → 組織 → 反思 → 執行
  ↑                                     ↓
  └─────────────────────────────────────┘
```

### 流程圖解

```
外部輸入
   ↓
┌──────────┐
│  捕獲    │ ─→ 收集到 Inbox
└──────────┘
   ↓
┌──────────┐
│  阐明    │ ─→ 每個項目是什麼？
└──────────┘    │
   ↓            ├─ 可執行 → 是項目還是任務？
   │            └─ 不可執行 → 參考資料/垃圾/待辦
┌──────────┐    │
│  組織    │ ─→ 放入正確的容器
└──────────┘    │
   │            ├─ 等待清單
   │            ├─ 將來/可能
   │            ├─ 項目清單
   │            ├─ 日曆
   │            └─ 下一步清單
┌──────────┐    │
│  反思    │ ─→ 定期查看和更新系統
└──────────┘
   ↓
┌──────────┐
│  執行    │ ─→ 根據情境、時間、精力行動
└──────────┘
```

## 📝 第一步：捕獲

### 捕獲原則

**原則 1：清空大腦**
- 寫下所有想到的事
- 不要判斷或過濾
- 放下大腦的負擔

**原則 2：統一入口**
- 所有收集點匯總到一個地方
- 減少遺漏的可能性
- 簡化處理流程

**原則 3：定期清空**
- 每天清空 Inbox
- 不讓項目積壓
- 保持系統流暢

### 收集工具

| 工具類型 | 推薦工具 | 用途 |
|----------|----------|------|
| 紙筆 | 筆記本、便利貼 | 快速手寫、隨時記錄 |
| 數字筆記 | Obsidian、Evernote | 跨設備、可搜索 |
| 行動應用 | 手機筆記、語音輸入 | 移動場景 |
| Email | Inbox | 電子郵件事務 |
| 便條 | 桌面便條 | 電腦旁快速記錄 |

### 收集檢查清單

```
📦 收集點檢查

物理位置：
☐ 書桌 - 文件、紙張
☐ 口袋 - 收據、便利貼
☐ 車內 - 駕駛時想到的點子
☐ 書包 - 隨身攜帶的物品
☐ 家裡各處 - 可能散落的位置

數字位置：
☐ 電郵收件箱 - 未處理郵件
☐ 手機筆記 - 手機記錄的內容
☐ 瀏覽器標籤 - 待閱讀/處理的網頁
☐ 下載文件夾 - 待處理的文件
☐ 社交媒體 - 待回復的訊息

心理位置：
☐ 擔憂 - 焦慮的事情
☐ 想法 - 突發的靈感
☐ 承諾 - 對他人的承諾
☐ 待辦 - 記得要做的但不確定什麼
```

## 🔍 第二步：阐明

### 阐明流程

對每個捕獲的項目，依次問自己：

#### 問題 1：它是可執行的嗎？

```
❓ 是可執行的嗎？
├─ 否
│  ├─ 是垃圾嗎？ → 刪除
│  ├─ 是參考資料嗎？ → 歸檔到參考系統
│  ├─ 是以後可能做的嗎？ → 將來/可能清單
│  └─ 現在還不是時候嗎？ → 待辦/日曆（有特定日期時）
└─ 是 → 繼續
```

#### 問題 2：下一步行動是什麼？

**定義下一步行動**：
- 必須是**具體的、可執行的**行動
- 以動詞開頭
- 明確的對象和時間

**示例**：

| 不好的下一步行動 | 好的下一步行動 |
|----------------|----------------|
| 寫報告 | 起草報告大綱 |
| 聯絡客戶 | 給客戶打電話討論項目進度 |
| 整理房間 | 清理客廳的茶几 |
| 學習 Python | 閱讀 Python 官方教程第一章 |
| 健身 | 今天晚上 7 點去健身房 30 分鐘 |

#### 問題 3：能 2 分鐘內完成嗎？

```
❓ 能在 2 分鐘內完成嗎？
├─ 是 → 立即執行（2 分鐘規則）
└─ 否 → 繼續
```

**2 分鐘規則**：
- 如果某事能在 2 分鐘內完成，立即做
- 比花時間記錄、組織更高效
- 減少小任務的積壓

#### 問題 4：這件事是我要做的嗎？

```
❓ 這件事是我要做的嗎？
├─ 是
│  ├─ 是項目嗎？ → 加入項目清單
│  └─ 是單次任務嗎？ → 繼續
└─ 否 → 委派給他人（等待清單）
```

#### 問題 5：什麼時候做？

```
❓ 什麼時候做？
├─ 特定日期時間 → 加入日曆
├─ 盡快做 → 加入下一步清單（按情境分類）
└─ 將來再做 → 將來/可能清單
```

## 📁 第三步：組織

### GTD 組織系統

```
┌─────────────────────────────────────┐
│         GTD 組織系統               │
├─────────────────────────────────────┤
│  📅 日曆                         │
│    - 特定日期/時間的任務            │
│    - 會議、約定                   │
│    - 最後期限                      │
├─────────────────────────────────────┤
│  📋 下一步清單（按情境）          │
│    - @電腦                       │
│    - @電話                        │
│    - @辦公室                      │
│    - @家                          │
│    - @外出                        │
│    - @等待                       │
├─────────────────────────────────────┤
│  📂 項目清單                     │
│    - 進行中的項目                 │
│    - 每個項目至少一個下一步行動     │
├─────────────────────────────────────┤
│  💭 將來/可能                     │
│    - 將來想做的事                 │
│    - 還沒確定的點子               │
│    - 供應商、工具、書籍等資源      │
├─────────────────────────────────────┤
│  📚 參考資料                     │
│    - 非行動性的信息               │
│    - 文件、文章、筆記             │
│    - 按主題分類存儲              │
└─────────────────────────────────────┘
```

### 情境分類

| 情境 | 適合的行動 | 示例 |
|------|------------|------|
| @電腦 | 需要電腦的工作 | 寫報告、編寫代碼、發送郵件 |
| @電話 | 可以用電話完成 | 聯絡客戶、預約、確認信息 |
| @辦公室 | 需要在辦公室做 | 與同事討論、處理文件、開會 |
| @家 | 在家裡做的事 | 清理房間、家庭會議、個人時間 |
| @外出 | 出去時順便做 | 買東西、辦事、見面 |
| @商店 | 在商店時買 | 採購物品 |
| @等待 | 等待他人回應的項目 | 等待客戶回復、等待部門確認 |

### 項目定義

**什麼是項目？**
- 任何需要**多個步驟**才能完成的事情
- 有**明確目標**和**完成標準**
- 可以在 1 年內完成

**示例**：

| 不是項目 | 是項目 |
|----------|--------|
| 洗衣服 | 組織整個衣櫃 |
| 打電話給朋友 | 策劃朋友聚會 |
| 買一本書 | 建立個人圖書館 |
| 寫一篇博客 | 啟動並運營博客 3 個月 |

## 🤔 第四步：反思

### 反思頻率

| 類型 | 頻率 | 目的 |
|------|------|------|
| 每日復盤 | 每天 | 處理當日 Inbox，更新待辦 |
| 每週復盤 | 每週日 | 全面回顧和更新系統 |
| 每月復盤 | 每月底 | 回顧長期目標和項目 |

### 每日復盤（10-15 分鐘）

```
🌅 早晨例行（5 分鐘）
☐ 查看日曆：今天有什麼固定安排？
☐ 查看待辦清單：選擇今天要做的 3 件事
☐ 查看等待清單：有什麼需要跟進？

🌙 晚間例行（10 分鐘）
☐ 清空 Inbox：處理今天收集的所有項目
☐ 更新待辦：完成的標記，新加入的歸類
☐ 準備明天：列出明天的優先事項
☐ 收拾物理環境：清空書桌，準備明天的工具
```

### 每週復盤（30-60 分鐘）

```
📅 每週復盤清單

收納和清空
☐ 清空所有物理 Inbox
☐ 清空所有數字 Inbox
☐ 確保沒有遺漏的收集點

回顧日曆
☐ 查看過去一週完成的事
☐ 查看即將到來的安排
☐ 捕獲任何漏掉的行動

回顧項目清單
☐ 每個活躍項目都有下一步行動？
☐ 進度是否符合預期？
☐ 需要調整計劃嗎？
☐ 有需要開始/結束的項目嗎？

回顧下一步清單
☐ 所有行動仍然是有效的嗎？
☐ 情境分類是否還準確？
☐ 有沒有過期的任務？

回顧等待清單
☐ 有什麼需要跟進？
☐ 是否需要發送提醒？

回顧將來/可能
☐ 有什麼項目可以啟動？
☐ 有什麼可以刪除？
☐ 捕獲任何新點子

創新創造
☐ 空出 15 分鐘自由思考
☐ 腦力激盪新項目
☐ 記錄新的想法和目標
```

### 每月復盤（60-90 分鐘）

```
📅 每月復盤清單

長期目標
☐ 這個月的進展如何？
☐ 與年度目標的關聯？
☐ 下個月的重點是什麼？

項目審查
☐ 哪些項目完成了？
☐ 哪些項目延後了？
☐ 新開始了哪些項目？

系統健康
☐ GTD 系統運作流暢嗎？
☐ 有什麼可以優化？
☐ 有什麼新的工具或方法可以嘗試？

個人成長
☐ 這個月學到了什麼？
☐ 有什麼需要改進？
☐ 下個月的挑戰是什麼？
```

## ⚡ 第五步：執行

### 行動選擇標準

在做任何事情之前，檢查：

1. **情境**
   - 我現在在哪裡？
   - 有什麼工具可用的？
   - 哪些任務可以在當前情境下執行？

2. **可用時間**
   - 我有多少時間？
   - 能完成什麼樣的任務？

3. **精力水平**
   - 我的精力如何？
   - 是高強度思考時還是簡單處理時？

4. **優先級**
   - 哪些是真正重要的？
   - 哪些事拖延會有後果？

### 三種行動模式

#### 1. 預定工作
```
按日曆安排執行
- 會議和約定
- 有明確時間點的任務
- 最後期限的項目
```

#### 2. 預定情境下的工作
```
根據當前情境選擇任務
- 查看 @電腦 清單
- 查看 @電話 清單
- 查看 @辦公室 清單
```

#### 3. 定義工作
```
當前面的都做完了
- 回顧項目清單
- 制定新的下一步行動
- 處理 Inbox
- 創新和思考
```

## 🎯 GTD 與 PARA 的結合

### 映射關係

| GTD 元素 | PARA 對應 | 說明 |
|----------|-----------|------|
| 項目清單 | 1 Projects | 有目標和截止日期的短期任務 |
| 區域 | 2 Areas | 持續維護的責任領域 |
| 參考資料 | 3 Resources | 有價值的信息和學習資料 |
| 將來/可能 | 4 Archives | 還未啟動或暫時歸檔的內容 |

### 集成流程

```
GTD 捕獲 → PARA 歸位

捕獲階段：
- 所有新想法 → 0 Inbox

阐明階段：
- 需要行動且有截止日期 → 1 Projects
- 持續的責任 → 2 Areas
- 有價值的參考 → 3 Resources
- 將來可能做 → 4 Archives 或保留在 GTD 將來/可能

組織階段：
- 使用 PARA 的分類結構
- GTD 的情境清單作為過濾器
- 項目清單映射到 1 Projects 中的項目
```

### Obsidian GTD 設置

```
0 Inbox/          # GTD 收集點
├── 📋 下一步清單.md
├── ⏰ 日曆.md
├── ⏳ 等待清單.md
└── 💭 將來可能.md

1 Projects/       # 項目清單
└── (各種項目)

2 Areas/         # 區域清單
└── (各種領域)

3 Resources/     # 參考資料
└── (各種資源)
```

## 📊 監控與優化

### GTD 系統健康度評分

| 指標 | 評分標準 |
|------|----------|
| Inbox 清空頻率 | 每天清空 = 10分，每週 = 5分，更久 = 0分 |
| 項目追蹤 | 所有項目都有下一步 = 10分 |
| 每週復盤 | 每週完成 = 10分 |
| 系統信任度 | 完全信任 = 10分，偶爾查詢 = 5分，經常查詢 = 0分 |
| 行動完成率 | >80% = 10分，50-80% = 5分，<50% = 0分 |

**總分 > 40**：系統運作良好
**總分 20-40**：需要改進
**總分 < 20**：需要重新建立系統

### 常見問題排查

#### 問題 1：系統太複雜

**症狀**：
- 不想處理 Inbox
- 系統維護花太多時間
- 經常遺漏項目

**解決方案**：
- 簡化分類
- 減少收集點
- 使用更簡單的工具

#### 問題 2：從不回顧

**症狀**：
- Inbox 積壓
- 任務過期
- 遺忘承諾

**解決方案**：
- 設置固定回顧時間
- 與日曆約定綁定
- 與其他人一起做復盤

#### 問題 3：過度收集

**症狀**：
- 大量無用項目
- 過度標籤和分類
- 分析癱瘓

**解決方案**：
- 在阐明階段更嚴格篩選
- 限制項目數量
- 定期清理
