# 前端數據可視化

> 使用前端技術創建交互式數據可視化與圖表

---

## 📖 概述

數據可視化是將數據轉換為圖形表示的過程，幫助用戶更直觀地理解數據模式和趨勢。

### 為什麼選擇前端可視化？

| 優勢 | 說明 |
|------|------|

# 更新

## 🎯 主流圖表庫

### D3.js - 靈活的數據驅動文檔

**特點**: 最靈活、功能最強大

**適用場景**: 自定義圖表、複雜數據可視化

```javascript
import * as d3 from 'd3';

// 創建 SVG 元素
const svg = d3.select('#chart')
  .append('svg')
  .attr('width', width)
  .attr('height', height);

// 繪製柱狀圖
svg.selectAll('rect')
  .data(data)
  .enter()
  .append('rect')
  .attr('x', d => xScale(d.value))
  .attr('y', d => yScale(d.value))
  .attr('width', xScale.bandwidth())
  .attr('height', d => height - yScale(d.value))
  .attr('fill', '#4a90e2');
```

**優點**:
- 完全控制每個元素
- 強大的數據綁定機制
- 豐富的轉換和動畫支持

**缺點**:
- 學習曲線陡峭
# 效率

### Chart.js - 簡單易用

**特點**: 語法簡潔、開箱即用

**適用場景**: 標準圖表、快速原型

```javascript
import Chart from 'chart.js/auto';

const ctx = document.getElementById('myChart').getContext('2d');
new Chart(ctx, {
  type: 'bar',
  data: {
    labels: ['一月', '二月', '三月', '四月'],
    datasets: [{
      label: '銷售額',
      data: [12, 19, 3, 5],
      backgroundColor: 'rgba(74, 144, 226, 0.5)',
      borderColor: 'rgba(74, 144, 226, 1)',
      borderWidth: 1
    }]
  },
  options: {
    responsive: true,
    scales: {
      y: { beginAtZero: true }
    }
  }
});
```

**優點**:
- 易於上手
- 良好的文檔和社區支持
- 響應式設計

**缺點**:
- 自定義能力有限
- 效能對大數據集不理想

---

### ECharts - 功能強大的企業級方案

**特點**: 功能豐富、適用於複雜場景

**適用場景**: 企業級應用、複雜交互

```javascript
import * as echarts from 'echarts';

const chart = echarts.init(document.getElementById('main'));
chart.setOption({
  title: { text: 'ECharts 示例' },
  tooltip: {},
  xAxis: { data: ['一月', '二月', '三月', '四月'] },
  yAxis: {},
  series: [{
    name: '銷售額',
    type: 'bar',
    data: [12, 19, 3, 5],
    itemStyle: { color: '#4a90e2' }
  }]
});

// 響應式調整
window.addEventListener('resize', () => chart.resize());
```

**優點**:
- 豐富的圖表類型
- 優秀的效能
- 內置主題和交互

**缺點**:
- 打包體積較大
- API 相對複雜

---

### Recharts - React 生態的首選

**特點**: React 友好、聲明式 API

**適用場景**: React 應用、需要組件化的項目

```jsx
import {
  BarChart,
  Bar,
  XAxis,
  YAxis,
  CartesianGrid,
  Tooltip,
  Legend
} from 'recharts';

const data = [
  { name: '一月', value: 12 },
  { name: '二月', value: 19 },
  { name: '三月', value: 3 },
  { name: '四月', value: 5 }
];

function MyChart() {
  return (
    <BarChart width={600} height={300} data={data}>
      <CartesianGrid strokeDasharray="3 3" />
      <XAxis dataKey="name" />
      <YAxis />
      <Tooltip />
      <Legend />
      <Bar dataKey="value" fill="#4a90e2" />
    </BarChart>
  );
}
```

**優點**:
- 聲明式 API
- 與 React 完美整合
- 響應式設計

**缺點**:
- 僅適用於 React
- 功能不如 ECharts 豐富

### Vega / Vega-Lite - 聲明式可視化語法

**特點**: JSON 聲明式語法

# 分析

```javascript
import vegaEmbed from 'vega-embed';

const spec = {
  "$schema": "https://vega.github.io/schema/vega-lite/v5.json",
  "data": {
    "values": [
      {"category": "A", "value": 12},
      {"category": "B", "value": 19},
      {"category": "C", "value": 3}
    ]
  },
  "mark": "bar",
  "encoding": {
    "x": {"field": "category", "type": "nominal"},
    "y": {"field": "value", "type": "quantitative"}
  }
};

vegaEmbed('#vis', spec);
```

**優點**:
- 聲明式語言易於學習
- 可移植性強（JSON 格式）
- 與數據科學工具整合好

**缺點**:
- 靈活性受限
- 不適合高度自定義場景

## 🎨 設計原則與最佳實踐

### 1. 選擇正確的圖表類型

**原則**: 圖表應該讓數據最容易被理解

```
錯誤示例：
- 用餅圖顯示超過 5 個類別
- 用折線圖顯示離散類別數據

正確示例：
- 用柱狀圖比較類別數據
- 用折線圖顯示時間趨勢
```

### 2. 保持簡潔

**原則**: 移除所有不必要的元素

```javascript
// ❌ 不必要的網格線和邊框
options: {
  grid: {
    drawBorder: true,
    drawOnChartArea: true,
    drawTicks: true
  }
}

// ✅ 最小化干擾
options: {
  grid: {
    drawBorder: false,
    drawOnChartArea: false,
    drawTicks: false
  }
}
```

### 3. 使用一致的顏色

**原則**: 使用品牌色或顏色板

```javascript
const colors = {
  primary: '#4a90e2',
  success: '#52c41a',
  warning: '#faad14',
  danger: '#f5222d',
  info: '#1890ff'
};
```

### 4. 新增交互提示

**原則**: 懸停時顯示詳細資訊

```javascript
options: {
  plugins: {
    tooltip: {
      callbacks: {
        label: function(context) {
          return `${context.dataset.label}: ${context.parsed.y}`;
        }
      }
    }
  }
}
```

### 5. 響應式設計

**原則**: 圖表應適應容器大小

```javascript
// React 組件示例
function ResponsiveChart({ data }) {
  const containerRef = useRef(null);
  const chartRef = useRef(null);

  useEffect(() => {
    if (containerRef.current) {
      const chart = echarts.init(containerRef.current);
      chart.setOption(data);
      chartRef.current = chart;

      return () => chart.dispose();
    }
  }, [data]);

  return <div ref={containerRef} style={{ width: '100%', height: '400px' }} />;
}
```

## 🚀 高級技巧

### 1. 動畫與過渡

```javascript
# 配置
options: {
  animation: {
    duration: 1000,
    easing: 'easeOutQuart'
  }
}

// D3.js 過渡
svg.selectAll('rect')
  .transition()
  .duration(750)
  .ease(d3.easeCubicInOut)
  .attr('height', d => yScale(d.value));
```

### 2. 大數據優化

```javascript
// 使用 Canvas 而非 SVG
// ECharts 示例
option = {
  renderer: 'canvas',  // 對大數據集效能更好
  series: [{
    type: 'bar',
    data: largeDataArray,
    progressive: 1000,  // 漸進式渲染
    progressiveThreshold: 5000
  }]
};
```

# 更新

```javascript
# 更新
setInterval(() => {
  const newData = generateRandomData();
  chart.setOption({
    series: [{
      data: newData
    }]
  });
}, 1000);
```

### 4. 自定義工具提示

```javascript
// Chart.js 自定義 Tooltip
options: {
  plugins: {
    tooltip: {
      backgroundColor: 'rgba(0, 0, 0, 0.8)',
      titleFont: { size: 14 },
      bodyFont: { size: 12 },
      padding: 12,
      cornerRadius: 4,
      callbacks: {
        footer: (tooltipItems) => {
          return '總計: ' + tooltipItems.reduce((sum, item) => sum + item.parsed.y, 0);
        }
      }
    }
  }
}
```

```
需要自定義圖表？
├─ 是 → 使用 D3.js
└─ 否
   ├─ React 項目？
```
