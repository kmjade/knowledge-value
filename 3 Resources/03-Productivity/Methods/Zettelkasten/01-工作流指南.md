# Zettelkasten 工作流

> Zettelkasten（卡片盒）是一种个人知识管理系统，通过原子化笔记、双向链接和有机成长来构建知识网络。

---

## 📖 基础概念

### Zettelkasten 是什么？

- **原子化笔记**：每个笔记只包含一个想法
- **双向链接**：在相关笔记之间建立连接
- **有机成长**：知识网络随时间自然演化
- **意外发现**：通过连接产生新见解

### 核心原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **自主性** | 笔记可独立理解 | 不依赖外部上下文 |
| **连接性** | 通过链接建立关系 | [[笔记A]] → [[笔记B]] |
| **成长性** | 系统持续演化 | 定期新增新笔记和连接 |

---

## 🏗️ 笔记类型

### 1. 永久笔记 (Permanent Notes)

**特点**：
- 包含完整的、独立的想法
- 使用持久标识符（日期或编号）
- 是系统的核心构建块

**示例标识符**：
- `202401261234` (YYYYMMDDHHMM)
- `20240126-01` (日期-序号)

### 2. 文献笔记 (Literature Notes)

**特点**：
- 从来源提取的原始资讯
- 包含来源引用
- 未经深度处理

### 3. 项目笔记 (Project Notes)

**特点**：
- 关于特定项目的资讯
- 临时性质，项目结束后可归档
- 不进入永久系统

### 4. 结构笔记 (Structure Notes / MOC)

**特点**：
- 提供某个主题的概览
- 作为多个笔记的入口点
- 帮助导航知识网络

### 5. 概念笔记 (Concept Notes)

**特点**：
- 定义和解释特定概念
- 提供清晰的理解框架
- 作为其他笔记的基础

## 🔄 工作流

### 每日流程

```
收集 → 处理 → 创建永久笔记 → 建立连接 → 审视和精炼 → 生成见解
```

### 步骤详解

#### 1. 收集 (Collect)

**目标**：快速捕获想法，不中断思考

- 使用 Inbox 文件夹
- 移动端快速笔记
- 语音输入
- 浏览器书签和剪藏

**原则**：
- 不要过滤，先记录
- 保持简洁，关键词即可
- 标注来源

#### 2. 处理 (Process)

**目标**：将原始想法转化为永久笔记

**检查清单**：
- [ ] 想法是否清晰明确？
- [ ] 是否与已有笔记重复？
- [ ] 是否可以拆分为多个笔记？
- [ ] 是否需要更多资讯来支持？
- [ ] 是否有来源可以引用？

**处理动作**：
- 转化为永久笔记
- 创建文献笔记
- 新增到项目笔记
- 丢弃

#### 3. 创建永久笔记 (Create Permanent Notes)

**格式模板**：
```markdown
---
title: {简明标题}
type: permanent
created: {YYYY-MM-DD}
tags: #[[主题标签]]
---

# {标题}

## 核心想法
{一个清晰、完整的想法}

## 解释
{为什么这很重要？如何运作？}

## 示例
{具体例子，说明概念}

## 应用
{如何使用？在什么情况下？}

## 相关笔记
- [[{相关笔记1}]]
- [[{相关笔记2}]]

## 参考资料
- {来源引用}
```

**创建原则**：
1. 一个想法一个笔记
2. 完整表达
3. 使用自己的语言
4. 新增上下文

#### 4. 建立连接 (Link)

**目标**：构建知识网络

**连接类型**：
- 直接连接
- 双向连接
- 结构连接
- 标签连接

**连接策略**：
1. 创建时连接
2. 定期审视
3. 使用索引
4. 意外连接

#### 5. 审视和精炼 (Review and Refine)

**目标**：保持系统的活性和质量

**审视频率**：
- 每日：处理 Inbox（5-10 分钟）
- 每周：新笔记和连接（30 分钟）
- 每月：深度审视某个主题（2-3 小时）
- 每季：结构性审视（半天）

#### 6. 生成见解 (Generate Insights)

**目标**：通过连接产生新想法

1. 随机漫步
2. 结构探索
3. 定期创作

## 🛠️ 工具和技术

### 必备插件

| 插件 | 用途 |
|------|------|
| **Dataview** | 数据查询 |
| **QuickAdd** | 快速输入 |
| **Graph Analysis** | 知识图谱 |
| **Backlinks** | 反向链接 |

### 符号系统

| 符号 | 含义 |
|------|------|
| `📌` | 重要 |
| `🔄` | 进行中 |
| `✅` | 已完成 |
| `❓` | 待研究 |
| `🔗` | 连接点 |

### 命名约定

**选项 1：日期标识符**
```
202401261234 永久笔记标题.md
```

**选项 2：标题命名**
```
永久笔记标题.md
```

**推荐组合**：
- 使用标题命名（主要）
- 在 frontmatter 中新增 created 日期

## 📊 与其他系统对比

### Zettelkasten vs PARA

| 维度 | Zettelkasten | PARA |
|------|---------------|------|
| **组织原则** | 主题连接 | 行动导向 |
| **笔记粒度** | 原子化 | 项目/资源级 |
| **连接方式** | 双向链接 | 层级结构 |

### 结合使用建议

```
3 Resources/
└── 永久笔记/   ← Zettelkasten - 存储知识
```

1. 项目笔记（PARA）：跟踪项目进度
2. 永久笔记（Zettelkasten）：提取可重用的知识

## 🎯 最佳实践

### 1. 开始时保持简单
- ❌ 不要：一开始就建立复杂的分类系统
- ✅ 应该：先从三个基本笔记类型开始

### 2. 专注于连接而非完美
- ❌ 不要：过度关注笔记的语法和格式
- ✅ 应该：优先建立笔记之间的连接

### 3. 定期审视而非即时完善
- ❌ 不要：每次创建都追求完美
- ✅ 应该：定期批量审视和改进

### 4. 使用自己的语言
- ❌ 不要：直接复制粘贴原文
- ✅ 应该：用自己的话重新表述

### 5. 连接多个来源
- ❌ 不要：只连接同来源的笔记
- ✅ 应该：跨领域建立连接
