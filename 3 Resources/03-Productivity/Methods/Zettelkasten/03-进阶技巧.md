## 📊 连接健康分析

### 查找孤立笔记

```dataview
LIST
FROM "5 Zettels"
WHERE file.outlinks.length = 0
  AND file.inlinks.length = 0
```

### 连接数统计

```dataview
TABLE without ID
  file.link AS "笔记",
  file.inlinks.length AS "入链",
  file.outlinks.length AS "出链",
  (file.inlinks.length + file.outlinks.length) AS "总连接"
FROM "5 Zettels"
WHERE (file.inlinks.length + file.outlinks.length) < 2
SORT (file.inlinks.length + file.outlinks.length) ASC
```

### 连接质量评估

| 连接数 | 评估 |
|--------|------|
| 0 | 孤立笔记，考虑连接或归档 |
| 1-2 | 较少，可尝试建立更多连接 |
| 3-5 | 正常 |
| 6+ | 高度连接，可能过于复杂 |

### 每周审视流程

1. **清理 Inbox**（5-10 分钟）
   - 处理所有临时笔记
   - 转化为永久笔记或丢弃

2. **检查连接**（15 分钟）
   - 查看孤立笔记
   - 尝试建立新连接

3. **随机漫步**（15 分钟）
   - 随机选择笔记阅读
   - 寻找意外连接

## 🐛 常见问题

### Q1: 如何避免创建过多笔记？

遵循「一个想法一个笔记」原则，但不是一句话一个笔记。思考：
- 这个想法是否独立存在？
- 是否可以从其他笔记中分离？
- 将来是否可能独立引用？

### Q2: 笔记找不到相关连接怎么办？

1. 使用标签来帮助发现
2. 定期使用搜索功能
3. 创建结构笔记作为枢纽
4. 接受一些笔记暂时孤立

### Q3: 需要多少笔记才能看到效果？

- **50-100 笔记**：开始看到一些连接
- **300-500 笔记**：网络开始显示价值
- **1000+ 笔记**：显著的知识综合能力

重点是持续性，不是数量。

---

> **进阶提示**：不要追求完美，专注于建立连接。知识网络的价值随着连接的增长而指数级增加。
