# Zettelkasten 原子化标准

> 原子化是 Zettelkasten 方法的核心原则，确保每张卡片只包含一个想法，便于连接和复用。

---

## ⚛️ 核心原则（4大原则）

| 原则 | 说明 |
|------|------|
| **原子性 (Atomicity)** | 每张卡片只包含一个想法，不可拆分 |
| **自主性 (Autonomy)** | 每张卡片可独立理解，不依赖其他卡片 |
| **链接性 (Linkage)** | 通过 wikilinks 建立知识网络 |
| **编号系统** | 使用唯一ID（如 `202601250001`）便于追溯 |

---

## 🃏 卡片类型

| 类型 | 前缀 | 说明 |
|------|------|------|
| **💡 闪念笔记** | `💡` | 临时、快速记录想法和灵感 |
| **📚 文献笔记** | `📚` | 阅读时的笔记，包含来源资讯 |
| **📌 永久笔记** | `📌` | 标准化的原子化知识卡片 |
| **📁 结构笔记** | `📁` | 主题索引，组织多个卡片 |

---

## 📝 原子化实践

### 错误示范
```
"深度工作的重要性和如何进行深度工作"
```

### 正确做法（拆分为两张）
```
卡片1：深度工作的定义
卡片2：深度工作的实践方法
```

### 判断标准

**问自己**：
1. 这个想法是否可以独立存在？
2. 将来是否可能独立引用？
3. 是否可以从其他笔记中分离？

如果都是「是」→ 拆分为独立卡片

## ✅ 质量检查清单

### 创建笔记时
- [ ] 是否只包含一个核心想法？
- [ ] 是否用我自己的语言写作？
- [ ] 是否清晰可独立理解？
- [ ] 是否新增了相关连接？
- [ ] 是否标记了适当的标签？

### 定期审视
- [ ] 笔记是否有孤立？
- [ ] 连接是否有效？
- [ ] 是否有重复内容？
- [ ] 是否有过时内容？

## 🔗 连接规范

### 连接类型

1. **直接引用**：`[[笔记标题]]`
2. **反向链接**：通过 Backlinks 面板查看
3. **双向连接**：A→B 且 B→A
4. **结构连接**：通过 MOC 索引

### 连接时机

- 创建新笔记时
- 发现相关笔记时
- 定期审视时
- 写作引用时
