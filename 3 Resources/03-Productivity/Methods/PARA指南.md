# 方法

## 什麼是 PARA？

# 管理

| 字母 | 意義 | 描述 |
|------|------|------|
| **P** | Projects | 有明確目標和期限的項目 |
| **A** | Areas | 持續維護的生活領域 |
| **R** | Resources | 有興趣的主題資源 |
| **A** | Archives | 已完成或過時的內容 |

---

## Projects（項目）

### 定義
- 有明確的**完成狀態**
- 有**目標**和**期限**
- 需要**行動**才能推進

### 特點
- 可以被標記為「完成」或「取消」
- 具有時效性
- 通常是暫時性的

### 範例
- 撰寫季度報告
- 規劃家庭旅行
- 完成網站改版
- 準備考試

### 項目筆記模板
```markdown
---
title: 項目名稱
status: active/on-hold/completed/cancelled
priority: high/medium/low
due: YYYY-MM-DD
tags: #project
---

# 項目名稱

## 🎯 目標
<!-- 這個項目要達成什麼？ -->

## 📋 任務清單
- [ ] 任務 1
- [ ] 任務 2
- [ ] 任務 3

## 📝 筆記
<!-- 相關想法和筆記 -->

## ✅ 進度追蹤
- [x] 開始日期：YYYY-MM-DD
- [ ] 完成日期：
```

## Areas（領域）

### 定義
- 需要**持續維護**的責任
- 沒有明確的「完成」狀態
- 有**標準**需要維持

### 特點
- 永遠不會「完成」
- 需要定期關注
- 維持在某個水準以上

### 範例
# 管理
- 財務規劃
- 人際關係
- 職業發展
- 居住環境
- 個人成長

### 領域筆記模板
```markdown
---
title: 領網網域名稱稱稱
tags: #area
---

# 領網網域名稱稱稱

- 標準 2

- [ ] 項目 1
- [ ] 項目 2

## 📝 筆記
# 更新

## 🔗 相關連結
- [[]]
```

## Resources（資源）

### 定義
- 有興趣的**主題**
- 收集的**資訊**
- 未來可能用到的**參考資料**

### 特點
- 沒有時效壓力
- 可以自由探索
- 需要時再取出使用

### 範例
- 學習主題（Python、畫畫）
- 興趣愛好（登山、食譜）
- 研究課題（AI、區塊鏈）
- 靈感收集

### 資源筆記模板
```markdown
---
title: 主題名稱
tags: #resource
---

# 主題名稱

## 📚 內容摘要
<!-- 收集的資訊 -->

## 🔗 有用連結
- [標題](URL)
- [標題](URL)

## 💡 想法和靈感
- 想法 1
- 想法 2

## 📝 個人筆記
<!-- 你的理解和反思 -->

## 🔗 相關主題
- [[]]
```

## Archives（歸檔）

### 定義
- 從 Projects、Arts、Resources 中移出的不活躍內容

### 歸檔時機
- 項目已完成或取消
- 領域已不再相關
- 資源已過時
- 資訊不再需要

### 歸檔原則
- 不要刪除，只做歸檔
- 保留摘要方便日後查詢
- 每年清理一次歸檔

### 歸檔筆記模板
```markdown
---
title: 歸檔主題
archived: YYYY-MM-DD
tags: #archive
---

# 歸檔：原標題

## 📅 歸檔日期：YYYY-MM-DD

## 📋 狀態
- [x] 已完成
- [ ] 已擱置
- [ ] 已取消

## 📝 摘要
<!-- 這是什麼？為什麼重要？ -->

## 🔗 相關
- [[]]
```

## 如何使用 PARA

### 1. 收集（Capture）

任何時候收到新資訊，先問：
- 這需要行動嗎？ → **Project**
- 這影響哪個生活領域？ → **Area**
- 這是我感興趣的主題嗎？ → **Resource**
- 都不需要？ → 刪除

### 2. 組織（Organize）

# 排序
# 排序
- 高優先權在上

# 排序
# 排序
- 定期檢視

# 排序
# 排序
- 可靈活調整

### 3. 檢視（Review）

**每週檢視**
# 更新
- [ ] 檢查 Areas 狀態
- [ ] 新增新 Resources

**每月檢視**
- [ ] 完成本月的 Projects
- [ ] 審視 Areas 標準
# 整理

**每季檢視**
- [ ] 歸檔完成的 Projects
- [ ] 評估 Areas 優先順序
- [ ] 清理過時 Resources

### 4. 執行（Do）

根據情境選擇：
- 有時間且需要專注 → 做 Projects
- 有零碎時間 → 處理 Areas
- 輕鬆探索 → 瀏覽 Resources

## 常見問題

### Q1: 東西該放在 Projects 還是 Resources？
如果需要「行動」才能推進 → Projects
如果只是「收集」參考 → Resources

### Q2: Areas 數量多少合適？
建議 5-10 個核心領域。太多會無法專注。

### Q3: 多久歸檔一次？
至少每季一次。完成的項目立即歸檔。

### Q4: 可以用標籤代替資料夾嗎？
可以。PARA 是概念框架，可根據工具調整。

### Q5: 跨類別的內容怎麼辦？
放在最相關的類別，或建立連結參考。

---

## 最佳實踐

1. **立即分類**：收到資訊當下就決定類別
2. **保持簡單**：不要過度複雜化結構
3. **定期檢視**：系統需要維護才能運作
4. **勇於歸檔**：不要舍不得移動內容
5. **建立連結**：善用 wiki links 連接相關內容
6. **使用模板**：維持一致性，節省時間
