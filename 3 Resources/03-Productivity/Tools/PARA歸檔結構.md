# 指南

## 為什麼需要歸檔？

歸檔不是刪除，而是：
- 保持主資料夾整潔
- 保留歷史記錄
- 日後仍可查詢
- 減少認知負擔

---

## 歸檔時機

| 類別 | 歸檔條件 |
|------|----------|
| **Projects** | 已完成 / 已取消 / 已擱置超過 3 個月 |
| **Areas** | 已不再相關（如換工作、搬家） |
| **Resources** | 已過時（如舊技術、已完成的興趣） |

## 歸檔命名規範

### 按年份結構（推薦）

```
Archive/
├── 2024/
│   ├── 2024-01-01_Website-Redesign_Completed
│   ├── 2024-02-15_Japan-Trip_Cancelled
│   └── 2024-03-01_Old-Project_Purged
│
├── 2023/
└── 2022/
```

**命名格式：**
```
YYYY-MM-DD_名稱_狀態
```

**狀態標籤：**
- `_Completed` - 已完成
- `_Cancelled` - 已取消
- `_On-Hold` - 已擱置
- `_Merged` - 已合併到其他項目
- `_Purged` - 已清理（不再需要）

## 歸檔流程

### 1. 評估是否歸檔
```markdown
- 這個項目是否已完成？
- 這個領域是否還相關？
- 這個資源是否還有用？
```

### 2. 新增歸檔資訊
```markdown
---
archived: 2024-01-26
original-status: completed
reason: Project milestone achieved
---
```

### 3. 建立摘要
在歸檔檔案頂部新增：

```markdown
# 歸檔：原標題

## 📅 歸檔日期：2024-01-26

## 📋 原狀態
- [x] 已完成

## 📝 摘要
這個項目是關於...

## 🎯 成果
- 成果 1
- 成果 2

## 🔗 相關
- [[]]
```

### 4. 移動到歸檔
- 從原始位置移動到 `PARA/Archive/年份/`
- 或使用軟連結（Obsidian 不支援）
- 在原位置保留一個指向歸檔的連結

## 快速歸檔指令

使用 `/archive-note` 指令：

```
/archive-note 完成原因
```

會自動：
1. 建立歸檔副本
2. 新增元數據
3. 移動到歸檔資料夾
4. 在原位置創建重定向

---

## 注意事項

1. **不要刪除** - 歸檔而非刪除
2. **保留摘要** - 方便日後理解內容
3. **按年份組織** - 容易追蹤時間線
4. **狀態標籤** - 快速了解歸檔原因
5. **定期清理** - 避免歸檔過於龐大

---

## 歸檔 vs 刪除

| 歸檔 | 刪除 |
|------|------|
| 可能未來有用 | 確定不需要 |
| 需要保留記錄 | 純粹的垃圾 |
| 敏感資訊 | 過時且無價值 |
| 個人回憶 | 錯誤或測試檔案 |

**原則：寧願歸檔也不要刪除。**
