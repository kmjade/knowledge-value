## 工作方法論

### PARA 方法

**核心概念**:
- **Projects**: 有明確目標和期限的短期任務
- **Areas**: 需要持續維護的長期責任
- **Resources**: 有興趣的主題和參考資料
- **Archives**: 不再活躍但需要保留的內容

**特點**:
- 簡單：只有四個分類
- 實用：按行動價值組織
- 可擴展：從個人到團隊都適用

### Getting Things Done (GTD)

**路徑**: `3 Resources/03-Productivity/Methods/Getting Things Done.md`

**核心原則**:
1. **捕獲一切** - 不要依賴記憶，把所有事務記錄下來
2. **定期處理** - 系統性處理收集的項目
3. **明確下一步** - 每個項目都有明確的下一步行動
4. **歸檔清晰** - 所有項目都有明確歸屬
5. **定期回顧** - 保持系統新鮮和可靠
6. **根據情境行動** - 在適合的時間做適合的事

**五步法流程**:
```
捕獲 → 阐明 → 组织 → 反思 → 执行
```

**收集系統**:
- 物理位置：筆記本、便利貼、書桌
- 數字位置：筆記、郵件、瀏覽器標籤
- 心理位置：擔憂、想法、承諾、待辦

**情境分類**:
- @電腦：需要電腦的工作
- @電話：可以用電話完成
- @辦公室：需要在辦公室做
- @家：在家裡做的事
- @外出：出去時順便做
- @等待：等待他人回應的項目

**GTD 與 PARA 整合**:
- GTD 項目清單 → 1 Projects
- GTD 區域 → 2 Areas
- GTD 參考資料 → 3 Resources
- GTD 將來/可能 → 4 Archives
