# Journal vs Daily

在 Obsidian / PARA 知識管理系統中，**Journal** 和 **Daily** 是兩個不同層級的概念。

## 一句話區別

```
Daily：What happened?        → 事件記錄（外部）
Journal：What did I learn?   → 意義萃取（內部）
```

## 對比

| | Daily（日誌） | Journal（反思筆記） |
|------|------|------|
| **驅動** | 日曆自動觸發 | 意圖觸發 |
| **頻率** | 每天（含空白日） | 不定期（有感才寫） |
| **格式** | 模板化、結構一致 | 自由形式 |
| **內容** | 流水帳：任務完成、會議摘要、想法碎片 | 深度思考：模式識別、決策理由、情緒記錄 |
| **生命週期** | ephemeral — 任務完成後可清理 | reference → evergreen — 長期參考價值 |
| **讀者** | 未來的自己（查「那天做了什麼」） | 未來的自己（查「那時我在想什麼」） |
| **範例** | 「今天開了 3 個會」「修了一個 bug」 | 「這週三次中斷來自同一個干擾源 → 需要設定界線」 |

## 工作流

```
Daily（每日自動記錄）
    │
    ▼ 每週回顧
萃取有價值的洞察
    │
    ▼
Journal（手動反思筆記）
    │
    ▼
從 "what" 到 "so what" 的昇華
```

## Obsidian 實踐

- **Daily**：由 Daily Notes 外掛自動建立，模板化流水帳
- **Journal**：手動建立，自由形式反思

```
1 Projects/Personal/
├── Daily/          ← Obsidian Daily Notes 自動輸出（流水帳）
└── Journal/        ← 手動寫的反思筆記（深度思考）
```
