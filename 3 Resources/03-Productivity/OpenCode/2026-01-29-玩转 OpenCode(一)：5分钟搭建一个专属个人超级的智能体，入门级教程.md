## 📌 原始摘录

### 核心内容

#### AGENTS.md：给AI的工作手册

还记得我们刚才创建的 `AGENTS.md` 文件吗？

这个文件非常非常重要，决定你的智能体是否跟你合得来。

你需要赋予它：

> - 你叫什么名字
> - 你是什么性格
> - 文件该放哪里
> - 有哪些规矩必须遵守

包含智能体本身的工具、能力、以及如何自更新，不定期摘取你的习惯，将它打造成你。

这是你给你的智能体赋予的最强脑力。这里建议设计的时候加上"最强大脑"几个字，别问为什么，问就是大佬们都在用。效果就是好。

#### 完整模板

如果你想让 AI 更懂你，用这个完整版：

```
# AI 管家工作规范
```

#### 个性化定制

AGENTS.md这东西就是定义你跟谁对话，无限定制，你可以随便改。

想让AI说话带点幽默感？加一句：

```
## 性格
```

想让AI模仿某个风格？直接写：

```
## 交互风格
```

想让AI专注某个领域？指定：

```
## 专业领域
```

AI 会读取这个文件，然后按你说的来。

以下是我个人左脑模板，情绪价值拉满！(其实有的内容是不用写的，但是装了不少的skill插件后，会对当前的skill目录进行权限封定，导致自然读取失败，因此锚定到记忆中，指令遵循更好。)

## 💡 我的理解

### 核心观点

1. **AGENTS.md 是智能体的灵魂** - 这个文件定义了智能体的身份、性格、工作方式，是让 AI 更懂用户的关键
2. **左右脑协作模式** - 通过"涛哥"（右脑，决策）和"阿星"（左脑，执行）的协作，实现分工明确、互相成就
3. **规范化的工作流** - 从新闻整理到技术沉淀，每个环节都有明确规范，确保工作质量
4. **持续更新机制** - 通过"铁律"和"自更新"机制，保持智能体的与时俱进
5. **长期记忆管理** - 使用 `.memory/` 隐藏目录存储长期记忆，不定期提取习惯

### 个人思考

#### 核心价值是什么？

这篇文章的核心价值在于**系统性地定义了智能体的完整工作体系**。它不仅仅是简单的提示词配置，而是建立了一个包含：
- 身份认同（名字、性格、角色）
- 工作环境（空间结构、文件规范）
- 执行流程（每日任务、日报规范）
- 能力体系（技能清单、提示词库）
- 知识沉淀（技术沉淀、质量保证）
- 记忆管理（长期记忆、更新机制）

这种系统化的设计让智能体能够：
- 更好地理解用户的工作习惯
- 更精准地执行任务
- 更自然地进行交互
- 更持续地学习和成长

#### 与其他知识的关联？

- **PARA 系统**：AGENTS.md 的工作空间结构（`dailynews/`, `myk/`, `.opencode/`）可以映射到 PARA 系统的 Project/Area/Resource
- **Zettelkasten**：技术沉淀到 `myk/技术沉淀/` 可以转化为永久笔记（Permanent Notes）
- **快速捕获**：提示词库的收集流程可以结合快速捕获模板（`_template-提示词收集.md`）
- **知识管理**：这个完整的智能体配置体系可以作为个人知识管理系统的参考模板
