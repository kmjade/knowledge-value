> [!summary] 网络安全概述
> 网络安全是指保护网络系统免受未经授权的访问、使用、披露、破坏、修改或销毁。

---

## 🎯 CIA三元组

网络安全的核心原则，也称AIC三元组。

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│            Confidentiality (机密性)                  │
│             防止未经授权的访问                      │
│                                                     │
│        ↖                           ↗               │
│         ↖                     ↗                  │
│  Integrity (完整性)     Availability (可用性)       │
│  防止未授权修改           确保授权访问               │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 机密性 (Confidentiality)

**定义**：确保信息只能被授权实体访问。

**实现方式**：
- 加密技术（对称加密、非对称加密）
- 访问控制
- 隐私保护

**威胁**：
- 窃听
- 社会工程学
- 数据泄露

### 完整性 (Integrity)

**定义**：确保信息在传输或存储过程中未被未授权修改。

**实现方式**：
- 哈希算法
- 数字签名
- 校验和
- 审计日志

**威胁**：
- 修改数据
- 重放攻击
- 中间人攻击

### 可用性 (Availability)

**定义**：确保授权用户在需要时可以访问系统和服务。

**实现方式**：
- 冗余设计
- 负载均衡
- 备份和恢复
- DDoS防护

**威胁**：
- DDoS攻击
- 系统故障
- 网络中断

---

## 🔐 威胁类型

### 被动攻击

攻击者仅监听和收集信息，不修改数据。

| 攻击类型 | 说明 |
|----------|------|
| 窃听 | 监听网络流量 |
| 流量分析 | 分析流量模式 |
| 社会工程学 | 通过欺骗获取信息 |

### 主动攻击

攻击者主动修改、删除或注入数据。

| 攻击类型 | 说明 |
|----------|------|
| 拒绝服务 (DoS/DDoS) | 使服务不可用 |
| 中间人攻击 | 拦截并可能修改通信 |
| 重放攻击 | 重放捕获的数据包 |
| 伪造攻击 | 伪造数据源 |
| 修改攻击 | 修改数据内容 |

## 🛡️ 安全机制

### 1. 加密

**对称加密**：
- 特点：加密和解密使用相同密钥
- 算法：AES, DES, 3DES, RC4
- 优点：速度快
- 缺点：密钥分发困难

**非对称加密**：
- 特点：使用公钥和私钥对
- 算法：RSA, ECC, DSA
- 优点：密钥分发方便
- 缺点：速度慢

### 2. 哈希

**定义**：将任意长度数据映射为固定长度摘要。

**特性**：
- 单向性：不可逆
- 抗碰撞性：很难找到两个不同数据产生相同哈希
- 雪崩效应：微小变化导致完全不同的哈希

**算法**：
- MD5（已被破解，不推荐）
- SHA-1（已被破解，不推荐）
- SHA-256, SHA-512

### 3. 访问控制

**身份认证**：
- 你知道什么（密码）
- 你拥有什么（令牌、智能卡）
- 你是什么（指纹、虹膜、人脸）

**授权**：
- DAC（自主访问控制）
- MAC（强制访问控制）
- RBAC（基于角色的访问控制）

### 4. 防火墙

**包过滤防火墙**：根据规则过滤数据包
**状态检测防火墙**：跟踪连接状态
**应用层防火墙**：深入检查应用层协议
**Web应用防火墙（WAF）**：专门防护Web应用

## 🔴 常见网络攻击

### 1. DDoS攻击

**定义**：分布式拒绝服务攻击，利用大量僵尸网络向目标发送大量请求。

**类型**：
- **SYN Flood**：发送大量SYN包，耗尽TCP连接资源
- **UDP Flood**：发送大量UDP包
- **ICMP Flood**：发送大量ICMP包
- **HTTP Flood**：发送大量HTTP请求

**防护**：
- 流量清洗
- 流量限制
- 分布式防御
- CDN

### 2. 中间人攻击

**定义**：攻击者拦截并可能修改通信双方的通信。

**场景**：
- 公共Wi-Fi
- ARP欺骗
- DNS劫持

**防护**：
- 使用HTTPS
- 证书固定
- VPN

### 3. ARP欺骗

**原理**：发送伪造的ARP响应，将网关MAC映射到攻击者MAC。

**影响**：流量重定向到攻击者，可能被窃听或修改。

**防护**：
- 动态ARP检测（DAI）
- 静态ARP表
- 交换机端口安全

### 4. DNS劫持

**定义**：将DNS查询重定向到恶意DNS服务器。

**类型**：
- 本地DNS劫持（修改hosts文件）
- 路由器DNS劫持
- DNS投毒

**防护**：
- 使用可信DNS服务器
- DNS over HTTPS (DoH)
- DNSSEC

### 5. 端口扫描

**定义**：扫描目标主机开放的端口。

**工具**：
- Nmap
- Masscan

**防护**：
- 关闭不必要的服务
- 防火墙规则
- 入侵检测系统（IDS）

## 📋 安全策略

### 1. 密码策略

- 最小长度：12位
- 包含：大小写字母、数字、特殊字符
- 定期更换（如90天）
- 历史密码不重复

### 2. 补丁管理

- 及时安装安全补丁
- 测试补丁兼容性
- 优先修复高危漏洞

### 3. 备份策略

- 定期备份
- 异地备份
- 测试恢复

### 4. 审计和监控

- 日志记录
- 异常检测
- 定期审计

## 🎯 安全最佳实践

### 网络层安全

1. **分段隔离**
   - VLAN隔离
   - 子网划分
   - DMZ区域

2. **边界防护**
   - 防火墙
   - IDS/IPS
   - WAF

3. **最小权限**
   - ACL限制
   - 端口限制
   - 服务限制

### 主机层安全

1. **系统加固**
   - 关闭不必要服务
   - 更新系统和应用
   - 安装杀毒软件

2. **配置安全**
   - 强密码策略
   - 账户锁定策略
   - 审计策略

### 应用层安全

1. **输入验证**
   - SQL注入防护
   - XSS防护
   - CSRF防护

2. **数据保护**
   - 敏感数据加密
   - 传输加密（HTTPS）
   - 存储加密
