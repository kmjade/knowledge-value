# 04-以人为本的设计

> 工业 5.0 的核心宣言：技术服务于人，而非人适应技术。这意味着从系统设计之初就把操作员的需求、能力和成长纳入考量。

## 从"用户"到"工匠"

| 视角 | 工业 4.0 | 工业 5.0 |
|------|----------|----------|
| 操作员的角色 | 监控者 | 决策者和创造者 |
| 培训 | 操作培训 | 终身学习、技能升级 |
| 界面设计 | 数据可视化 | 情境感知+自适应 |
| 工作设计 | 任务分配 | 有意义的工作 |
| 评价 | 效率 | 效率+满意度+成长 |

## 人因工程 (Ergonomics)

| 维度 | 工业 5.0 实践 |
|------|-------------|
| **物理** | 外骨骼、Cobot 辅助搬运、可调高度工作站 |
| **认知** | 减少信息过载、AI 辅助决策、情境化信息推送 |
| **组织** | 自组织团队、灵活排班、参与式改进 |
| **心理** | 有意义的工作、减少单调重复、自主权 |

## 技能升级路径

```
传统操作员 → 数字操作员 → 数字工匠
   │              │              │
 手动操作     理解数据     编程+优化+创新
```

| 阶段 | 需要的新技能 |
|------|------------|
| 数字操作员 | 读 Dashboard、理解 SPC、操作 HMI |
| 数字工匠 | PLC 基础、数据分析、协作机器人编程 |

### 培训策略

| 方法 | 说明 |
|------|------|
| **微学习** | 5-10 分钟短视频——不需要进教室 |
| **AR 辅助** | 在工作中学习——AR 叠加操作指导 |
| **数字孪生培训** | 在虚拟产线上练习——零风险 |
| **同伴学习** | 熟练操作员带新手——最有效的知识传递 |
| **游戏化** | 将学习变成挑战和竞赛 |

## 包容性设计

| 原则 | 实践 |
|------|------|
| **年龄包容** | 字体可调、物理负载可减（外骨骼） |
| **性别包容** | 工作站可调高度、力量辅助 |
| **能力包容** | 为不同身体条件的人设计可达的工位 |
| **文化包容** | 多语言界面、文化中立的图标 |

## 参与式设计

不是"为工人设计"，而是"和工人一起设计"：

| 阶段 | 操作员参与 |
|------|-----------|
| 需求 | 操作员最知道痛点在哪 |
| 设计 | 操作员评审设计方案 |
| 测试 | 操作员是最终验收者 |
| 改进 | Kaizen——操作员提改进建议 |

> Toyota 的 Kaizen 文化：每个操作员都有权停下产线、提出改进。这是以人为本制造的经典案例。
