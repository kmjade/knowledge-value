# 07-计算机控制系统

> 当单台 PLC 不够时——DCS 管理全厂过程控制，工业 PC 提供更高的计算能力，软 PLC 模糊了 PLC 和 PC 的边界。

## PLC vs DCS

| 维度 | PLC | DCS |
|------|-----|-----|
| 擅长 | 离散控制（开关、顺序） | 连续过程控制（温度、压力、流量） |
| 规模 | 单机/产线 | 全厂几百到几万 I/O 点 |
| 冗余 | 可选 | 默认——控制器/网络/电源全冗余 |
| 编程 | 梯形图/ST | 功能块图为主（FBD） |
| 典型行业 | 汽车、包装、机床 | 石化、电力、冶金 |
| 代表 | Siemens S7-1500 | Siemens PCS7, Honeywell Experion |

### DCS 架构

```
┌────────────────────────────────┐
│         工程师站/操作站          │
├────────────────────────────────┤
│       冗余控制网络 (以太网)       │
├──────────┬──────────┬─────────┤
│ 控制器A   │ 控制器B   │ 控制器C   │  冗余对
│ ┌──┐┌──┐ │ ┌──┐┌──┐ │ ┌──┐┌──┐ │
│ │主││备│ │ │主││备│ │ │主││备│ │
│ └──┘└──┘ │ └──┘└──┘ │ └──┘└──┘ │
├──────────┴──────────┴─────────┤
│        冗余 I/O 总线            │
├────────────────────────────────┤
│    分布式 I/O 站 (远程机架)      │
└────────────────────────────────┘
```

## 软 PLC

软件 PLC = 在标准工业 PC 上运行 PLC 运行时系统。

| 方案 | 说明 |
|------|------|
| **TwinCAT** (Beckhoff) | 在 Windows 上运行实时核——最成熟的软 PLC |
| **CODESYS Control** | 独立运行时，可跑在 Linux/Windows/ARM |
| **Simatic S7-1500S** | Siemens 的软 PLC 方案 |

### 软 PLC 的优势与劣势

| 优势 | 劣势 |
|------|------|
| 计算能力远超传统 PLC | 实时性依赖 PC 硬件 |
| 开放生态（C++, Python 扩展） | 可靠性不如专用硬件 |
| 直接运行数据库、Web 服务器 | IT 部门担心 Windows 更新/病毒 |

## 工业 PC (IPC)

| 类型 | 特点 |
|------|------|
| 机架式 | 19" 标准机柜，替代传统 PLC 架 |
| 嵌入式 (无风扇) | 恶劣环境，无移动部件 |
| 面板式 | 一体化 HMI+PC |

## 边缘计算

在靠近数据源头的地方处理数据——PLC 和云端之间：

```
PLC → 边缘网关 → 云端
      (本地处理)
```

| 功能 | 说明 |
|------|------|
| 协议转换 | Modbus → MQTT |
| 数据预处理 | 过滤、聚合、压缩 |
| 本地决策 | 不依赖云端的快速响应 |
| 缓存 | 断网时暂存数据 |
