# 06-实施方法论

> ERP 实施失败率高达 50%+（部分报告）。成功的实施不是 IT 项目，是管理变革。

## 实施阶段

| 阶段 | 内容 | 周期 |
|------|------|------|
| **1. 选型** | 需求分析→招标→演示→选型 | 2-4 月 |
| **2. 蓝图** | 梳理现有流程→设计未来流程 | 2-6 月 |
| **3. 实现** | 系统配置→开发→测试 | 3-8 月 |
| **4. 上线** | 数据迁移→培训→切换 | 1-3 月 |
| **5. 运维** | 支持→优化→升级 | 持续 |

## 常见失败原因

| 原因 | 说明 |
|------|------|
| 高层不重视 | 一把手工程——没有老板推，推不动 |
| 流程没梳理 | 把混乱的流程搬进系统→更混乱 |
| 数据不准 | 垃圾进垃圾出——BOM 不准、库存不准 |
| 培训不够 | 操作员不会用→抵触→回到 Excel |
| 范围蔓延 | 什么都想做→什么都做不完 |

## 上线策略

| 策略 | 说明 | 风险 |
|------|------|------|
| **大爆炸** | 所有模块同时上线 | 高——出问题影响全公司 |
| **分步** | 先财务→再采购→再生产 | 低——逐步消化 |
| **并行** | 新老系统同时跑 1-3 个月 | 中——工作量双倍 |
