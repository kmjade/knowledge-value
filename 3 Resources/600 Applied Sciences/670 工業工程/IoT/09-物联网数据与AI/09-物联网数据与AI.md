# 09-物联网数据与AI

> IoT 的价值不在于"连接"，而在于"数据驱动的洞察"。时序数据库、流处理和边缘 ML 是将 IoT 数据转化为价值的三大技术支柱。

## 时序数据库 (TSDB)

IoT 数据本质是时间序列——每条记录都有时间戳：

| 数据库 | 特点 |
|--------|------|
| **InfluxDB** | 最流行的开源 TSDB |
| **TimescaleDB** | PostgreSQL 扩展——SQL 兼容 |
| **TDengine** | 国产高性能——10× 压缩 |
| **ClickHouse** | 列式存储——分析查询极快 |

### 时序数据的特征

| 特征 | 对数据库的要求 |
|------|---------------|
| 写多读少 | 写入吞吐优先 |
| 时间有序 | 按时间分区 |
| 高压缩比 | 差值编码+压缩算法→95%+ 压缩率 |
| 降采样 | 原始数据→1分钟均值→1小时均值→... |
| 数据生命周期 | 自动删除过期数据 (TTL) |

## 流处理

批处理 = 先存后算。流处理 = 边来边算。

| 引擎 | 特点 |
|------|------|
| **Apache Flink** | 事件时间+精确一次语义——工业级 |
| **Apache Kafka Streams** | Kafka 原生流处理 |
| **Spark Streaming** | 微批处理——近实时 |
| **EMQX Rule Engine** | MQTT Broker 内置流处理 |
| **Node-RED** | 低代码IoT流处理——原型友好 |

### 流处理典型场景

```
传感器数据流 → 过滤(异常值) → 聚合(1分钟均值) → 阈值检测(>90→告警) → 存储
```

## 数据生命周期

```
采集 (ms) → 流处理 (ms-s) → 热存储 (天) → 温存储 (月) → 冷存储 (年) → 删除
   ↑                              ↓
  边缘过滤                      降采样(1s→1min→1h)
```

## IoT 中的 AI

| 层级 | 模型 | 例子 |
|------|------|------|
| **边缘** (TinyML) | 轻量分类/回归 | 振动异常检测 (Cortex-M4) |
| **边缘** (嵌入式 Linux) | CNN/RNN | 声学故障检测、视觉 |
| **云端** | 大模型、LLM | 跨设备模式挖掘、自然语言查询 |

### 典型的 IoT ML 流水线

```
历史数据 → 标注(正常/故障) → 特征工程 → 训练模型 → 量化/压缩 
→ 部署到边缘 → 推理 → 云端持续收集新数据 → 重训练
```
