# 06-开发与实践

> 从芯片选型到通过 Matter 认证——开发一个 Matter 产品的端到端流程。

## 开发流程

```
1. 芯片选型 → 2. SDK 集成 → 3. 定义设备类型 → 4. 编写固件 →
5. 认证测试 → 6. 量产写入 DAC → 7. 上市
```

## 芯片选型

| 场景 | 推荐 | 理由 |
|------|------|------|
| WiFi 设备 (常供电) | ESP32-C6 | WiFi6 + Matter, 性价比高 |
| Thread 设备 (电池) | nRF54L15 | 超低功耗 |
| Thread 设备 (多协议) | SiLabs MG24 | Thread+Zigbee+BLE 三模 |
| 网关/Controller | Linux (树莓派) | 成熟生态 |

## Matter SDK

```bash
# 克隆官方 SDK
git clone https://github.com/project-chip/connectedhomeip

# 构建
source scripts/activate.sh
gn gen out/debug
ninja -C out/debug
```

### SDK 结构

| 目录 | 内容 |
|------|------|
| `examples/` | 参考设备实现 (灯、锁、窗帘...) |
| `src/app/` | 应用层和数据模型 |
| `src/transport/` | 消息层 |
| `src/credentials/` | 安全和认证 |
| `src/platform/` | 各芯片平台适配 |

## 开发 step-by-step

### 1. 定义设备

```cpp
// 在 .zap 文件中定义设备类型和集群
// 例如：一盏可调光灯
Endpoint 1:
  - Device Type: Dimmable Light (0x0101)
  - Server Clusters: OnOff, LevelControl, Identify, Groups
```

### 2. 实现集群回调

```cpp
// 当收到"开灯"命令时
void OnOffCluster::OnOffChanged(bool newValue) {
    if (newValue) {
        gpio_set_level(LED_PIN, 1);  // 开灯
    } else {
        gpio_set_level(LED_PIN, 0);  // 关灯
    }
}
```

### 3. 烧录和配网

```bash
# 用 chip-tool 手动配网调试
chip-tool pairing code 1 MT:Y.K90SO52784A0648G00

# 或者用手机扫码配网 (需要 Matter Controller App)
```

## 调试工具

| 工具 | 用途 |
|------|------|
| **chip-tool** | CLI——配网、控制、读取属性 |
| **chip-repl** | Python REPL 交互调试 |
| **Wireshark** | 抓包分析 Matter 消息 |
| **Matter Sniffer** | Thread 网络抓包 |

## 认证流程

| 步骤 | 说明 |
|------|------|
| 1. 成为 CSA 会员 | 年费 ($7K-$20K) |
| 2. 自测试 | 用 Matter Test Harness 跑测试 |
| 3. 送测 | 到授权测试实验室 (ATL) |
| 4. 提交声明 | 提交符合性声明 |
| 5. 获得认证 | CSA 批准→使用 Matter 标识 |
