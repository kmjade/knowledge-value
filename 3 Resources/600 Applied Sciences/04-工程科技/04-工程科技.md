# 04-工程科技

> 工程是将科学原理应用于设计、建造和维护结构、机器、系统与过程的学科。工程的核心不是"懂科学"，而是**在约束下找到可行的最优解**。

## 工程的核心思维

| 思维 | 含义 | 与科学思维的区别 |
|------|------|-----------------|
| **约束优化** | 在多个限制条件下找最优解 | 科学追求"真"，工程追求"够好" |
| **安全边际** | 设计能力远超预期负载 | 科学家可以失败，工程师不能 |
| **故障模式分析** | 提前预想所有可能的失败 | 科学假设一个变量，工程师假设所有 |
| **权衡** | 快 vs 好 vs 便宜——不可能三角 | 科学可以"在理想条件下" |
| **模块化** | 拆分为可独立测试的子系统 | 科学的还原论，工程的应用 |

> 工程的第一原则：**任何可能出错的都会出错**（墨菲定律不是为了悲观，是为了在设计时就防住）。

## 工程方法论

### 系统工程 V 模型

```
需求分析           ←──────────→          系统验证
    ↓                                       ↑
架构设计           ←──────────→        集成测试
    ↓                                       ↑
详细设计           ←──────────→        单元测试
    ↓                                       ↑
      实现 ────────────────────→ 测试
```

左侧是分解（从上到下），右侧是集成验证（从下到上）。

### 可靠性工程

| 概念 | 含义 | 例子 |
|------|------|------|
| **MTBF** (Mean Time Between Failures) | 平均故障间隔 | 硬盘的 MTBF ~1M 小时 |
| **冗余 (Redundancy)** | 关键组件有备份 | 飞机多个发动机、RAID 磁盘 |
| **FMEA (故障模式与影响分析)** | 系统性地分析每种故障的后果 | 汽车安全关键系统必须做 |
| **容错 (Fault Tolerance)** | 部分故障系统仍能运行 | 互联网路由 |

## 主要工程学科

| 学科 | 做什么 | 核心原理 |
|------|--------|----------|
| **土木工程** | 建筑、桥梁、道路、水坝 | 结构力学、材料强度 |
| **机械工程** | 机器、发动机、制造设备 | 热力学、动力学 |
| **电气工程** | 电力、电子、通信 | 电磁学、信号处理 |
| **化学工程** | 化工厂、制药工艺、材料合成 | 化学动力学、传热传质 |
| **航空航天** | 飞机、火箭、卫星 | 空气动力学、推进、控制 |
| **生物医学工程** | 人工器官、医学影像、植入物 | 生物力学、生物材料 |
| **环境工程** | 水处理、空气污染控制、固废 | 环境化学、微生物学 |

## 工程伦理

| 原则 | 说明 |
|------|------|
| **公众安全第一** | 工程师的首要责任是保护公众的健康和安全 |
| **诚实** | 只在能力范围内工作，不误导 |
| **可持续发展** | 考虑项目和产品对环境的长期影响 |
| **知情同意** | 受工程影响的人有权了解风险 |
