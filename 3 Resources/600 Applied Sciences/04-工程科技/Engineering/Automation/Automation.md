# 自动化控制

> **UDC 62-5**: Control devices. Regulators. Automatic control engineering
> 从继电器到工业 4.0——自动化是工业的神经系统

## 自动化金字塔

这是理解自动化系统的**核心框架**——从毫秒级的物理信号到月度的企业决策，五层架构覆盖了自动化工程师的全部工作场景：

```
        ┌──────────────────┐
        │  ERP (企业层)      │  天-月    SAP, Oracle — 订单、排产、财务
        ├──────────────────┤
        │  MES (执行层)      │  小时-天  生产调度、质量管理、物料跟踪
        ├──────────────────┤
        │ SCADA (监控层)     │  秒-分    WinCC, IFix — 数据采集、可视化、报警
        ├──────────────────┤
        │  PLC/DCS (控制层)  │  ms-秒   S7-1500, PCS7 — 逻辑、PID、顺序
        ├──────────────────┤
        │  现场设备层         │  μs-ms   传感器、执行器、电机、阀门
        └──────────────────┘
```

> **关键洞察**: 层级越高，数据量越小但决策价值越大；层级越低，数据量越大但单条数据的意义越小。自动化的核心问题是**如何把底层数据变成上层决策**。

## 核心概念速览

### PID——90% 的控制回路

$$u(t) = K_p \cdot e(t) + K_i \int e(t)\,dt + K_d \cdot \frac{de(t)}{dt}$$

| 项 | 作用 | 调大时 | 调小时 |
|----|------|--------|--------|
| **P** | 响应误差 | 更快但可能震荡 | 更稳但可能迟钝 |
| **I** | 消除余差 | 消除稳态误差 | 响应变慢 |
| **D** | 预测趋势 | 减小超调 | 对噪声更敏感 |

### IEC 61131-3 五种语言

| 语言 | 形态 | 适用 |
|------|------|------|
| **LD** (梯形图) | 电路图式 | 逻辑控制、电工迁移 |
| **FBD** (功能块) | 逻辑门式 | 过程控制、信号链 |
| **ST** (结构化文本) | Pascal/类 C | 复杂算法、数据处理 |
| **SFC** (顺序功能图) | 流程图式 | 步骤化顺序控制 |
| **IL** (指令表) | 汇编式 | 已淘汰 |

### 通信协议选择

| 场景 | 推荐 |
|------|------|
| 简单、低成本、非实时 | Modbus TCP |
| Siemens 生态 | Profinet |
| 极致同步（运动控制） | EtherCAT |
| 跨品牌互联 | OPC UA |
| 上云 | MQTT |

### PLC 扫描周期

```
┌──────────────────────────────────────┐
│ 1. 读输入 → 2. 执行程序 → 3. 写输出  │
│       ↑__________________________↓   │
│            循环 (1-10ms)              │
└──────────────────────────────────────┘
```

> PLC 不是持续运行——是一个周期一个周期地扫描。这意味着你的程序逻辑**必须在一个扫描周期内完成**。

## 在三层模型中的位置

自动化控制是**应用科学的范例**——它与农学一样展示了三层模型的复合性：

| 层 | 自动化中的体现 |
|----|---------------|
| **科学层** | 控制论（Wiener）、系统辨识、信号处理——控制理论的数学基础 |
| **知识层** | 命题知识（PID 公式）+ 程序知识（梯形图编程）+ 默会知识（"听声音判断电机故障"） |
| **学科层** | 自动化专业是电气工程的子学科——IEC 61131-3 标准化、TÜV 认证体系 |

**认识论特征**: 自动化的知识同时存在于三个**不兼容的媒介**中——

| 媒介 | 形式 | 特点 |
|------|------|------|
| **人脑** | 默会经验 | 老师傅的直觉，"这条线有问题" |
| **代码** | PLC 程序 | 精确、可复制、可版本管理 |
| **硬件** | 接线、机械 | 物理世界不可消除的刚性 |

**三层间的翻译损耗**是自动化工程的核心挑战——工程师的意图→代码→物理行为，每一步都有信息损失。
