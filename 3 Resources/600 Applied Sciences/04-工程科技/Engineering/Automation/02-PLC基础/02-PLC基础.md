# 02-PLC 基础

> PLC (Programmable Logic Controller) —— 可编程逻辑控制器。为工业环境设计的专用计算机，替代继电器控制柜。1968 年由通用汽车需求催生，Dick Morley 发明。

## PLC 硬件架构

```
┌─────────────────────────────────────────┐
│                   PLC                    │
│  ┌──────┐  ┌──────┐  ┌──────────────┐  │
│  │ CPU  │  │ 内存  │  │  通信接口      │  │
│  │      │  │ 程序  │  │  Ethernet/IP  │  │
│  │      │  │ 数据  │  │  Profinet     │  │
│  └──────┘  └──────┘  │  Modbus       │  │
│                      └──────────────┘  │
│  ┌──────────┐  ┌──────────────────┐    │
│  │ 数字输入  │  │   数字输出         │    │
│  │ DI (24V)  │  │   DO (继电器/晶体管) │    │
│  └──────────┘  └──────────────────┘    │
│  ┌──────────┐  ┌──────────────────┐    │
│  │ 模拟输入  │  │   模拟输出         │    │
│  │ AI (4-20mA)│  │   AO (4-20mA/0-10V)│    │
│  └──────────┘  └──────────────────┘    │
└─────────────────────────────────────────┘
```

## 扫描周期

PLC 循环执行——不是持续的，是一个周期一个周期地扫描：

```
┌──────────────────────────────────────┐
│ 1. 读输入 → 2. 执行程序 → 3. 写输出  │
│       ↑__________________________↓   │
│            循环 (扫描周期)             │
└──────────────────────────────────────┘
```

| PLC 型号 | 典型扫描周期 |
|----------|-------------|
| 小型 PLC (S7-1200) | 1-10 ms |
| 中型 PLC (S7-1500) | 0.5-5 ms |
| 高性能 PLC | <0.1 ms |

> 这就是为什么编程时变量的值在扫描周期内不会"实时"更新——它在周期开始时读取，结束时写入。

## I/O 接线

### 数字输入 (DI)

| 类型 | 说明 |
|------|------|
| 源型 (PNP) | 传感器输出 +24V → PLC 检测到高电平 |
| 漏型 (NPN) | 传感器导通到 0V → PLC 拉低 |

### 数字输出 (DO)

| 类型 | 特点 | 适用 |
|------|------|------|
| **继电器输出** | 交直流通用、电流大 (2A+)、机械寿命有限 | 电磁阀、接触器 |
| **晶体管输出** | 仅直流、电流小 (0.5A)、高速、长寿 | 指示灯、小负载、高频 |

### 模拟量

| 标准 | 优点 |
|------|------|
| **4-20mA** | 断线检测（0mA = 断路）、抗干扰 |
| **0-10V** | 简单，短距离可用 |
| **0-20mA** | 无断线检测——已少用 |

> 4-20mA 是工业模拟信号的事实标准。为什么从 4mA 开始？因为 0mA 可能意味着"传感器断线"——可以区分"测到零"和"传感器坏了"。

## 供电与接地

| 要点 | 说明 |
|------|------|
| 控制电源 | 通常是 24V DC——单独的开关电源 |
| 传感器电源 | 从 PLC 或外部 24V 供 |
| 接地 | **必须可靠接地**——干扰是 PLC 最常见的故障原因 |
| 隔离 | 输入和输出与 CPU 之间有光耦隔离 |
