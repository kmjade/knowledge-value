# 02-架构与协议栈

> Matter 是基于 IP 的智能家居应用层协议——在 TCP/UDP 之上定义了设备如何被发现、配网、控制和互操作。

## 协议栈

```
┌──────────────────────────────────────┐
│         Matter 应用层                  │  设备类型、集群、交互模型
├──────────────────────────────────────┤
│         Matter 数据模型               │  属性/命令/事件
├──────────────────────────────────────┤
│         Matter 交互模型               │  读/写/订阅/命令
├──────────────────────────────────────┤
│         Matter 安全层                 │  加密、认证、消息完整性
├──────────────────────────────────────┤
│         Matter 消息层                 │  消息格式和路由
├──────────────────────────────────────┤
│         TCP / UDP                    │  (IPv6)
├──────────────────────────────────────┤
│         Thread / WiFi / Ethernet     │  传输层 (任选其一)
└──────────────────────────────────────┘
```

## 数据模型

### 三层结构

| 层级 | 含义 | 例子 |
|------|------|------|
| **Node (节点)** | 一个物理设备 | 智能灯泡 |
| **Endpoint (端点)** | 设备上的一个功能 | 灯泡的照明功能 |
| **Cluster (集群)** | 一组相关的属性和命令 | OnOff 集群 |

### 交互模型

| 操作 | 说明 | 例子 |
|------|------|------|
| **Read** | 读取属性 | "灯开还是关？" |
| **Write** | 修改属性 | "把灯打开" |
| **Invoke** | 执行命令 | "调光到 50%" |
| **Subscribe** | 订阅属性变化 | "亮度变了告诉我" |

### 例子：控制一盏灯

```
Controller: Write OnOff Cluster, OnOff = True → Light 1
Light 1: 开灯 + 报告 Status = OK
Light 1: Subscribe → 报告 OnOff = True (通知所有订阅者)
```

## Fabric（布料）

Matter 的核心多管理员概念——一台设备同时属于多个生态：

```
灯泡 (Node)
├── Fabric 1: Apple Home (管理员: iPhone)
├── Fabric 2: Google Home (管理员: Nest Hub)
└── Fabric 3: Amazon Alexa (管理员: Echo)
```

> 这就是为什么"一个 Matter 灯泡可以同时被 Apple Home 和 Google Home 控制"——设备上有多个 Fabric。
