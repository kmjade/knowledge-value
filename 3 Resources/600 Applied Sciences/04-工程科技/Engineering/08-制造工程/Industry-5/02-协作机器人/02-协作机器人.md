# 02-协作机器人

> 协作机器人 (Cobot) 是为与人类安全工作而设计的机器人——不像传统工业机器人需要围栏隔离。力控、碰撞检测和安全标准是 Cobot 的三大基石。

> **来自**: [[Industry-5|工业5.0]] → 人机协作支柱的核心技术 | 宏观视角：[[../../Technology/04-自动化与机器人/04-自动化与机器人|科技总论·机器人学]] | 技术底层：[[08-运动控制|自动化·运动控制]]

## 传统机器人 vs 协作机器人

| 维度 | 传统工业机器人 | 协作机器人 |
|------|-------------|-----------|
| 安全 | 围栏隔离 | 力/速度限制→直接接触安全 |
| 编程 | 专业编程 | 拖动示教、图形化 |
| 部署 | 固定工位，数周 | 快速部署，数小时 |
| 负载 | 10-1000+ kg | 3-20 kg |
| 速度 | 极快 | 较慢（安全限制） |
| 成本 | ¥200K-1M+ | ¥50K-300K |
| 适合 | 大批量、高速 | 多品种、小批量、人机协作 |

## 安全标准 ISO 10218 / ISO/TS 15066

协作机器人四种安全模式：

| 模式 | 原理 | 何时用 |
|------|------|--------|
| **安全级监控停止** | 人靠近→机器人停 | 人与机器人偶尔交互 |
| **手动引导** | 人手握住→跟随人手移动 | 拖动示教、辅助搬运 |
| **速度和分离监控** | 距离越近速度越慢→接触前停止 | 人机共享工作区 |
| **功率和力限制** | 碰撞力 < 人体承受极限 | 直接接触也不会受伤 |

### 功率和力限制 (PFL)——最常用模式

| 身体部位 | 准静态接触 | 瞬态接触 |
|----------|-----------|----------|
| 头部 | 65N | — |
| 脸部 | 65N | — |
| 手/手指 | 140N | 280N |

> Cobot 不是"无力"的——是被动受限的。一个 10kg 负载的 Cobot 如果全力撞到人仍可能受伤。安全设计包括：关节力矩传感器、碰撞检测 (<10ms)、圆形无锐角外表面。

## 典型应用

| 应用 | 说明 |
|------|------|
| **上下料** | Cobot 给 CNC 机床上料——操作员做质检和换刀 |
| **装配** | 人做柔性操作，Cobot 做精确定位和高扭矩紧固 |
| **质检** | Cobot 带视觉系统自动检测，人做复杂判断 |
| **包装码垛** | 重物搬运交给 Cobot |
| **实验室** | 移液、样本处理——精度 + 不疲劳 |

## 主要品牌

| 品牌 | 特点 |
|------|------|
| **Universal Robots** | 2008 年推出首款 Cobot——行业先驱 |
| **ABB GoFa** | 负载 5kg，力矩传感器内置各关节 |
| **FANUC CRX** | 工业级可靠性，10-25kg |
| **KUKA LBR iisy** | 灵敏度高 |
| **越疆 (Dobot)** | 中国 Cobot——教育+轻工业 |

## 力控：Cobot 的核心技术差异

传统工业机器人的控制目标是**精确跟踪预设轨迹**（位置控制）。Cobot 在此基础上增加了**力控制**——这才是"协作"的技术本质。

### 两种力控范式

| 范式 | 原理 | 行为比喻 | 典型场景 |
|------|------|----------|----------|
| **阻抗控制** | 机器人表现为"弹簧-阻尼"系统——外力→位移 | 你推它，它让开 | 拖动示教、装配柔顺插入 |
| **导纳控制** | 机器人表现为"质量-阻尼"系统——测力→算位移→执行 | 你推它，它计算该去哪然后过去 | 打磨、抛光——恒力接触 |

```
阻抗控制:  F_ext → 产生位移 Δx (质量-弹簧-阻尼响应)
          "感受到力就让步"

导纳控制:  F_ext → 测量 → 计算目标位移 → 电机执行
          "感受到力后计算出目标位置再过去"
```

> 大多数 Cobot（UR, ABB, FANUC）使用**阻抗控制**——更安全、响应更快。导纳控制适用于需要精确力跟踪的应用（打磨、去毛刺）。

### 零力控制与拖动示教

拖动示教是 Cobot 最标志性的能力——操作员握住 Cobot 末端，直接"手把手"拖动到目标位置：

```
人手施加力 → 关节力矩传感器检测 → 阻抗控制器 → 电机平滑跟随
                                              ↓
                                   记录路径点 → 生成程序
```

**为什么传统机器人做不到？** 传统机器人的减速器（谐波/摆线）有高减速比→反向驱动需要极大的力。Cobot 使用低减速比 + 力矩传感器来克服这一点。

### 碰撞检测时间线

| 步骤 | 时间 | 动作 |
|------|------|------|
| 碰撞发生 | t₀ | 关节力矩瞬间上升 |
| 力矩传感器检测 | t₀ + 1-2 ms | 力矩 > 阈值 → 触发 |
| 控制响应 | t₀ + 5-10 ms | 电机反转/停止 |
| 机器人停止 | t₀ + 10-50 ms | 取决于速度和负载 |

> 50ms 内从碰撞到停止——这是 PFL 模式的硬性要求。如果检测延迟 >50ms，一个 10kg 负载的 Cobot 在 0.5m/s 的碰撞可以造成严重伤害。

## 编程方式对比

| 方式 | 学习门槛 | 速度 | 灵活度 | 适合 |
|------|----------|------|--------|------|
| **拖动示教** | 极低——操作员无需编程 | 分钟级 | 中 | 简单重复任务、频繁换型 |
| **图形化编程** (Blockly/Flow) | 低——拖拽逻辑块 | 小时级 | 中-高 | 中等复杂度、有逻辑判断 |
| **脚本编程** (Python/URScript) | 中——需要编程基础 | 天级 | 高 | 复杂逻辑、外设集成、视觉引导 |
| **离线编程** (仿真) | 高——工程师 | 天-周 | 最高 | 复杂路径、多机器人协调、虚拟调试 |

> **Cobot 的真正革命在编程层**：拖动示教将"编程"从抽象代码变成了物理直观——操作员的默会知识第一次可以直接输入机器，无需程序员翻译。

## 末端执行器

Cobot 本身只是手臂——**手**（末端执行器）决定了它能做什么：

| 类型 | 原理 | 负载 | 适合 |
|------|------|------|------|
| **气动平行爪** | 气缸驱动两指 | 1-20 kg | 规则形状工件 |
| **电动自适应爪** | 电机驱动多关节手指 | 1-10 kg | 不规则形状 |
| **真空吸盘** | 真空吸附 | 轻量（取决于面积） | 平整表面——包装、码垛 |
| **磁力抓手** | 电磁铁 | 取决于磁力 | 金属板材 |
| **软体抓手** | 柔性材料被动适应 | 轻量 | 食品、易碎品 |

> 末端执行器的选择比 Cobot 本体的选择**更影响最终效果**。一个 20 万的 UR5 配上 2000 块的劣质气爪 = 整体精度和可靠性被末端拖垮。

## 部署经济学

### 传统 6 轴机器人 vs Cobot 的 TCO

| 成本项 | 传统工业机器人 | Cobot |
|--------|-------------|-------|
| 机器人本体 | ¥200K-1M+ | ¥50K-300K |
| 安全围栏/光幕 | ¥50K-200K | ¥0（PFL 内置） |
| 安装/集成 | ¥100K-500K | ¥20K-80K |
| 编程 | ¥50K-200K（专业程序员） | ¥5K-20K（操作员可做） |
| 年度维护 | ¥20K-50K | ¥5K-15K |
| **3 年 TCO** | ¥500K-2M+ | ¥100K-500K |
| **典型回收期** | 18-36 个月 | **6-12 个月** |

### 什么时候选 Cobot vs 传统机器人？

| 选 Cobot | 选传统工业机器人 |
|----------|----------------|
| 多品种、小批量 | 单一产品、大批量 |
| 需要频繁换型 | 几年不改产线 |
| 人机共享工作区 | 独立自动化岛 |
| 负载 <10kg | 负载 >20kg |
| 部署周期以天计算 | 部署周期以周-月计算 |
| 操作员需要交互（上下料+质检） | 全自动、无人化 |

|------|-----------------|--------|

## 在三层模型中的位置

Cobot 是工业 5.0 的认识论缩影——它**不是新技术**（硬件仍是电机+编码器+减速器），而是**新范式**（安全设计、力控、人机协作）。它修改了"机器人"这个概念在三个层的含义：

| 层 | 传统机器人 | Cobot |
|----|-----------|-------|
| **科学层** | 刚体动力学、轨迹规划 | 力控算法、碰撞检测、人机交互模型 |
| **知识层** | 专业编程知识（梯形图/G代码） | 拖动示教——操作员的默会知识可以直接输入 |
| **学科层** | 工业机器人在制造学科中 | Cobot 推动 ISO/TS 15066 新标准——学科边界扩展 |

> **关键洞察**: Cobot 最大的创新不在技术层——在于它让**非程序员**（操作员）可以直接"教"机器人。这就是"增强而非替代"的技术实现：操作员的默会知识通过拖动示教直接编码为机器行为，绕过了"工程师翻译"这一传统瓶颈。
