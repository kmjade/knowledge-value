# 04-SCADA 与 HMI

> SCADA (Supervisory Control and Data Acquisition) 是监控层——采集数据、显示状态、管理报警和历史趋势。HMI (Human-Machine Interface) 是操作员触摸的屏幕。

## SCADA vs HMI

| 维度 | HMI | SCADA |
|------|-----|-------|
| 范围 | 单台设备/单条产线 | 全厂/多站点 |
| 硬件 | 触摸屏面板 | PC 服务器 |
| 数据存储 | 有限（报警记录） | 大量（历史数据库） |
| 联网 | 连一台 PLC | 连接多个 PLC/站点 |
| 例子 | Siemens TP1200 Comfort | WinCC, IFix, Ignition |

## HMI 设计原则

| 原则 | 说明 |
|------|------|
| **一目了然** | 3 秒内判断设备状态——正常/警告/故障 |
| **颜色编码** | 绿色=运行、黄色=待机/警告、红色=故障、灰色=停止 |
| **最少点击** | 关键操作不超过 3 次点击 |
| **防误操作** | 关键按钮需确认对话框 |
| **趋势和报警** | 不只是当前值——要能看到变化趋势 |

## 报警管理

| 概念 | 说明 |
|------|------|
| 报警等级 | 紧急(红色)/重要(黄色)/提示(蓝色) |
| 报警泛滥 | 一个故障触发连锁报警 → 淹没真正原因 |
| 首次报警 | 连锁故障中最早出现的报警→通常是根因 |
| ISA 18.2 | 报警管理的国际标准 |

### 报警泛滥——工业自动化的常见病

> "操作员在报警泛滥时只能忽略所有报警——这正是最需要报警的时候。" 解决：报警优先级、死区、延时、分组抑制。

## 趋势与历史

| 功能 | 说明 |
|------|------|
| 实时趋势 | 最近几分钟到几小时的曲线 |
| 历史趋势 | 数据库中存储的数月数据 |
| 对比 | 同一画面叠加不同时间/不同变量的趋势 |
| 导出 | 导出 CSV 用于分析和报告 |
