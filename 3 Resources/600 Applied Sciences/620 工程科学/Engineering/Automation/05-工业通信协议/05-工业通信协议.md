# 05-工业通信协议

> 工业通信是自动化系统的"神经系统"——PLC 与 IO、驱动、HMI 之间的数据交换依赖这些协议。选错协议可能导致毫秒级延迟→产品报废。

## 协议层次

```
┌────────────────────────────────┐
│     信息层 (IT/OT 融合)          │  OPC UA, MQTT
├────────────────────────────────┤
│     控制层 (PLC↔PLC/SCADA)      │  Profinet, Ethernet/IP
├────────────────────────────────┤
│     现场层 (PLC↔IO/驱动)        │  EtherCAT, Profibus, Modbus
├────────────────────────────────┤
│     传感器层 (点对点)            │  4-20mA, IO-Link
└────────────────────────────────┘
```

## 主流协议对比

| 协议 | 物理层 | 速度 | 拓扑 | 主导厂商 |
|------|--------|------|------|----------|
| **Modbus RTU** | RS-485 | ~115 kbps | 总线 | 施耐德（开放） |
| **Modbus TCP** | Ethernet | 100 Mbps | 星型 | 施耐德（开放） |
| **Profibus** | RS-485 | ~12 Mbps | 总线 | Siemens |
| **Profinet** | Ethernet | 100 Mbps-1 Gbps | 星型/环型 | Siemens |
| **EtherCAT** | Ethernet | 100 Mbps | 环型/线型 | Beckhoff |
| **Ethernet/IP** | Ethernet | 100 Mbps-1 Gbps | 星型 | Rockwell |
| **OPC UA** | Ethernet | 任意 | 任意 | 独立标准 |
| **CANopen** | CAN | 1 Mbps | 总线 | 运动控制 |

## Modbus——最通用的协议

### Modbus RTU (串行)

```
主站 → 请求: [地址] [功能码] [数据] [CRC校验]
从站 → 应答: [地址] [功能码] [数据] [CRC校验]
```

| 功能码 | 操作 |
|--------|------|
| 01 | 读线圈 (DO) |
| 02 | 读离散输入 (DI) |
| 03 | 读保持寄存器 (如模拟量) |
| 04 | 读输入寄存器 |
| 05 | 写单个线圈 |
| 06 | 写单个寄存器 |
| 15 | 写多个线圈 |
| 16 | 写多个寄存器 |

> Modbus 的优点：简单、开放、几乎任何设备都支持。缺点：安全性为零（无认证、无加密）、速度慢。适合非关键、低速的数据交换。

## Profinet vs EtherCAT——"实时"之争

| 维度 | Profinet | EtherCAT |
|------|----------|----------|
| 实时原理 | 标准以太网 + RT/IRT 扩展 | "飞速数据报"——帧在通过每个节点时"在线"处理 |
| 最快周期 | ~250 μs (IRT) | ~50 μs |
| 同步精度 | <1 μs (IRT) | <100 ns |
| 硬件要求 | 专用 ASIC (IRT) 或标准网卡 (RT) | 专用 ASIC (ESC) |
| 优势 | 兼容标准 TCP/IP | 极致的速度和同步 |
| 主导 | Siemens (欧洲) | Beckhoff (运动控制) |

## OPC UA——"工业自动化界的 HTTP"

OPC UA 不只是协议——是信息建模框架：

| 特性 | 说明 |
|------|------|
| 平台无关 | Windows/Linux/嵌入式 |
| 安全 | 加密、认证、审计 |
| 信息建模 | 不只是传数据——数据结构完整描述 |
| 面向服务 | 读/写/订阅/方法调用 |
| Pub/Sub | OPC UA over MQTT (云友好) |

> 如果说 Modbus 是"传一个数字"，OPC UA 是"传这个数字是什么、单位是什么、范围是什么、什么时候采集的"。

## 协议选择指南

| 场景 | 推荐 |
|------|------|
| 简单、低成本、非实时 | Modbus TCP |
| Siemens 生态 | Profinet |
| 极致速度和同步（运动控制） | EtherCAT |
| 不同品牌设备互联 | Modbus（通用）或 OPC UA（高级） |
| 连接云端/IT 系统 | MQTT 或 OPC UA |
| Rockwell 生态 | Ethernet/IP |
