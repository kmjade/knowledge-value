# 07-生态与认证

> 不是"自称支持 Matter"就行——产品必须通过 CSA 认证才能打上 Matter 标识。认证确保设备真正符合规范、真正能互操作。

## CSA (连接标准联盟)

| 信息 | 内容 |
|------|------|
| 原名 | Zigbee Alliance |
| 2002 | 成立 |
| 2021 | 更名为 CSA，发布 Matter |
| 成员 | 550+ 公司 |
| 董事会 | Apple, Google, Amazon, Samsung, IKEA, NXP, Schneider 等 |

### 会员级别

| 级别 | 参与度 | 年费 |
|------|--------|------|
| **Promoter** | 董事会级——制定方向 | 最高 |
| **Participant** | 参与工作组和测试 | 中等 ($20K) |
| **Associate** | 访问规范、认证 | 较低 ($7K) |
| **Adopter** | 仅认证 | 最低 (免费?待确认) |

## 认证流程

```
会员申请 → 自测试 (Test Harness) → 送授权实验室 (ATL) → 
符合性声明 → CSA 审核 → 获得认证 → 量产写入 DAC → 上市
```

### 认证测试内容

| 测试 | 验证什么 |
|------|----------|
| 协议一致性 | 设备的行为符合 Matter 规范 |
| 互操作性 | 能和不同平台的 Controller 配网和控制 |
| 安全 | DAC 证书正确、加密通信 |
| 网络 | Thread/WiFi 配网成功 |

## Matter 标识

认证通过后可在产品和包装上使用：

```
┌──────────────────┐
│    [Matter 标识]  │
│     Matter        │
└──────────────────┘
```

对消费者的承诺：
- 简单：扫码添加
- 互操作：跨平台工作
- 可靠：本地控制
- 安全：加密保护
