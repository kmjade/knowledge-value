# 04-仿真与验证

> 仿真是数字孪生的核心能力——"如果这样做会怎样？" 能够安全地在数字世界中测试，而不影响运行的物理资产。

## 从离线仿真到实时仿真

| 类型 | 运行方式 | 速度 | 用途 |
|------|----------|------|------|
| **离线仿真** | 手动输入参数，运行数小时-数天 | 慢 | 设计验证 |
| **在线仿真** | 自动获取实时数据 | 实时 | 预测和优化 |
| **加速仿真** | 比物理时间快 100-1000× | 极快 | 快速探索参数空间 |
| **嵌入式仿真** | 运行在边缘设备上 | 实时 | 实时控制辅助 |

## 多物理场仿真

真实世界是耦合的——热影响结构，流体影响振动：

| 物理场 | 方法 | 典型工具 |
|--------|------|----------|
| **结构 (FEA)** | 有限元分析 | Ansys Mechanical, Abaqus |
| **流体 (CFD)** | 计算流体力学 | Ansys Fluent, OpenFOAM |
| **热** | 热传导、对流、辐射 | 同上 |
| **电磁** | 麦克斯韦方程数值解 | Ansys Maxwell, CST |
| **声学** | 波动方程 | Actran, VA One |

> 多物理场的挑战：同时求解所有物理场计算量太大。实际做法是先做单向耦合（流体→结构），只有必要时才做双向耦合。

## 降阶模型 (ROM)

全阶模型 (FOM) 可能需要数小时才能跑一个工况。降阶模型将计算时间压缩到毫秒级。

| 方法 | 原理 |
|------|------|
| **SVD/POD** (本征正交分解) | 提取最主要的"模态"，丢弃不重要的 |
| **神经网络 ROM** | 用 FOM 数据训练一个快速的神经网络替代 |
| **响应面** | 在设计空间中拟合一个简单的多项式曲面 |

```
FOM (3小时/工况) → 生成训练数据 → 训练 ROM → ROM (1ms/工况)
```

> ROM 的精度用原 FOM 验证。关键在于确保 ROM 在未见过的新工况下仍然可靠——这是最难的。

## 验证与验证 (V&V)

| 概念 | 含义 | 问题 |
|------|------|------|
| **验证 (Verification)** | 模型数学上是否正确？ | "我们解对了方程吗？" |
| **确认 (Validation)** | 模型是否代表真实世界？ | "我们解对了方程吗？" |

### 验证的实用方法

| 方法 | 说明 |
|------|------|
| 基准测试 | 用已知解析解的算例测试 |
| 网格独立性 | 加密网格→结果变化 <1% |
| 与实测对比 | 孪生的预测 vs 传感器实测 |
| 残差监控 | 仿真迭代的误差是否收敛 |

## 实时仿真的挑战

| 挑战 | 解决 |
|------|------|
| 速度 | ROM、GPU 加速、1D 替代 3D |
| 数据同化 | 卡尔曼滤波——将实测数据融合进仿真 |
| 不确定性量化 | 不是给一个答案，而是给概率分布 |
