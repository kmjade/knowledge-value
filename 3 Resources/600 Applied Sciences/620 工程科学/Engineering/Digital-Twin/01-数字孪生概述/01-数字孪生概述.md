# 01-数字孪生概述

> 数字孪生是物理资产、过程或系统的虚拟表示——通过传感器数据持续同步，能够模拟、预测和优化其物理对应物的行为。

## 核心三要素

```
        物理实体
           ↙  ↘
    数据流(实时同步)    反馈控制(优化指令)
           ↘  ↙
        数字孪生
```

| 要素 | 说明 |
|------|------|
| **物理实体** | 真实世界中的设备、产线、建筑或系统 |
| **数字模型** | 几何、物理、行为、规则的虚拟表示 |
| **数据连接** | 双向数据流——物理→数字(感知)、数字→物理(控制) |

## 演进史

| 阶段 | 时期 | 特征 |
|------|------|------|
| **数字模型** | 1980s-90s | CAD、CAE——静态的、离线的 |
| **数字影子** | 2000s-10s | 单向数据流——PLC→数据库→显示 |
| **数字孪生** | 2015s- | 双向、实时、全生命周期 |

### 关键里程碑

| 年份 | 事件 |
|------|------|
| 2002 | Michael Grieves (密歇根大学) 在 PLM 课程中首次提出数字孪生概念 |
| 2010 | NASA 在《技术路线图》中正式定义数字孪生 |
| 2015 | GE 推出 Predix 平台——工业数字孪生商业化 |
| 2017 | Gartner 将数字孪生列入十大战略技术趋势 |
| 2020s | 数字孪生从航空航天/国防扩散到制造、建筑、医疗 |

## 数字孪生的类型

| 类型 | 范围 | 例子 |
|------|------|------|
| **组件孪生** | 单个零件/传感器 | 轴承、阀门、涡轮叶片 |
| **资产孪生** | 一台设备 | 泵、发动机、机器人 |
| **系统孪生** | 多个资产协同 | 整条产线、发电机组 |
| **过程孪生** | 生产/业务流程 | 从原材料到成品的全过程 |

## 数字线程 (Digital Thread)

数字孪生是"点"，数字线程是"线"——连接产品全生命周期的数据流：

```
设计 → 制造 → 测试 → 使用 → 维护 → 报废
  ↑_____数据持续流动_____↑
```

> 数字线程确保"设计时的孪生"→"制造时的孪生"→"运行时的孪生"是一个连贯的整体，而不是三个孤岛。
