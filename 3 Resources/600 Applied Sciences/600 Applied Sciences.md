# 应用科学 (Applied Sciences)

> **UDC 6**: Applied Sciences. Medicine. Technology
> 应用科学是将基础科学的发现转化为解决实际问题的技术体系——基础科学回答"为什么"，应用科学回答"怎么做"。

如果基础科学是绘制地图（发现自然规律），应用科学就是建造道路（将规律转化为技术），工程则是建造城市（将技术集成为可用的产品/系统）。三者不是三个隔离的箱子，而是一个**从发现到落地的连续光谱**。

## 基础科学 → 应用科学 → 工程

```
基础科学                应用科学                 工程实践
(Basic Science)    (Applied Sciences)      (Engineering)
     ↓                    ↓                      ↓
   物理学            →   材料科学          →   芯片制造
   化学              →   化学工程          →   制药工艺
   生物学            →   生物技术          →   CRISPR 疗法
   数学              →   控制论/运筹学     →   自动驾驶
   计算机科学        →   AI/软件工程       →   ChatGPT
```

| 维度 | 基础科学 | 应用科学 | 工程 |
|------|----------|----------|------|
| 问题 | "电子如何运动？" | "如何用电子传输信息？" | "设计一款手机芯片" |
| 目标 | 理解自然规律 | 将规律转化为技术原理 | 构建可用的产品/系统 |
| 时间尺度 | 数十年 | 数年-十数年 | 数月-数年 |
| 成果 | 论文、理论 | 专利、原型 | 产品、标准、基础设施 |
| 评判标准 | 真/假 | 可行/不可行 | 可靠/不可靠 |

> **关键洞察**: TRL (Technology Readiness Level) 衡量一项技术在光谱上的位置。基础科学在 TRL 1-3，应用科学在 TRL 3-7，工程在 TRL 7-9。大多数"技术突破"新闻死在 TRL 4-6——"死亡谷"。

`Engineering/` 是本库中**最深、最活跃**的子库：

```
Engineering/
├── 01-工程学概述/         ← 约束优化/安全系数/工程伦理
├── 02-土木工程/           ← 结构/岩土/交通/水利
├── 03-机械工程/           ← 热力学/热机/制造工艺
├── 04-电气与电子工程/     ← 电力/芯片/信号处理
├── 05-化学工程/           ← 单元操作/催化/过程安全
├── 06-航空航天工程/       ← 空气动力学/推进/火箭方程

│   └── Industry-5/        ←  工业5.0深度（人机协作/可持续/韧性）
│       └── Cobot/         ←   协作机器人专项
├── 09-工程创新与未来/     ← 数字孪生/生成式设计/可持续
├── Automation/            ← PLC/运动控制/SCADA/工业协议
├── Digital-Twin/          ← 数字孪生技术栈深度
├── ERP/                   ← 企业资源计划
```

> 工程学子库的核心观点：**工程不是科学的"应用"——工程有自己的认识论。** 工程师在科学理论不完备的情况下建造了蒸汽机、飞机和芯片。工程知识包含大量"这是这么做的"默会知识，它们不在任何教科书的方程里。

## 技术成熟度 (TRL)：理解"这技术能用了么"

NASA 提出的 TRL 是评估一项技术离实际应用有多远的标准框架：

| TRL | 阶段 | 说明 | 在哪一层 |
|-----|------|------|----------|
| 1 | 基础原理 | 在论文里 | 基础科学 |
| 2 | 技术概念 | 有应用设想 | 基础科学 |
| 3 | 概念验证 | 实验室验证 | **应用科学** ← |
| 4 | 组件验证 | 在实验室环境中 | **应用科学** |
| 5 | 系统验证 | 在模拟真实环境中 | **应用科学** |
| 6 | 原型演示 | 在真实环境中 | **应用科学** |
| 7 | 实际环境演示 | 接近最终形态 | 工程 ← |
| 8 | 系统完成 | 通过测试和认证 | 工程 |
| 9 | 实际应用 | 在真实任务中运行 | 工程 |

> **"死亡谷"在 TRL 4-6**——实验室可行 ≠ 量产可行 ≠ 有市场需求。绝大多数"改变世界的技术突破"死在这个阶段。

## 在三层模型中的位置

应用科学是三层知识模型中**连接理论的科学层和操作的学科层的桥梁**：

| 层 | 应用科学中的体现 |
|----|------------------|
| **科学层** | 应用科学不产生新的基础理论，但**重新配置科学知识的优先级**——哪些理论值得投入工程资源去实现？这个选择本身就是知识判断 |
| **知识层** | 应用科学的核心产品是**技术知识**——专利、工艺参数、设计规范、最佳实践。这类知识的特点是：它必须"管用"，而不只是"正确" |
| **学科层** | 应用科学塑造了现代大学的应用学科（工程院、医学院、农学院），以及工业界的 R&D 部门。TRL 框架本身就是学科层的产物 |

> **认识论特征**: 应用科学暴露了一个被忽视的知识类型——**技术知识**（technological knowledge）。它不同于科学知识（命题式的"知道什么"）和默会知识（体验式的"知道怎么做"），而是两者的混合：它需要科学原理的正确性，但也需要工程直觉的判断力。"这个材料在 500°C 还能用"——既需要热力学的理论框架，也需要材料工程师对晶界行为的直觉。
