### 2.2 兩種組織模式的共存

```
模式 A：通用概述章節 (01-13)
  → 用於泛 600 主題（生物科技、能源、環境…）
  → 單一 .md 文件，非完整 KB
  → 提供概念性概述，不深入細節

模式 B：DDC 子知識庫 (610, 620, 689…)
  → 用於有明確 DDC 編號的獨立學科
  → 完整 KB 結構（14+ 文件）
  → 可包含深度子 KB
```

### 2.3 MOC 設計

600 級 MOC 作為所有子 KB 的統一中樞：

```markdown
# 600 Applied Sciences

| 610 醫學健康 | 619.1 中醫學 | 620 工程科學 | ... | 690 建築科學 |

### 活躍子領域
| [[689 业余手工/08-電子DIY\|🔌 電子DIY]] | 智能小車、UGV、RDK |

## 3. DDC 子 KB 結構規範

### 3.1 標準結構（重述）

每個 DDC 子 KB 遵循 vault 級標準：

```
[DDD] [名稱]/
├── [DDD] [名稱].md          MOC 主入口
├── 00-MOCs/                 總覽 + 學習路徑
│   ├── [名稱]知識庫總覽.md
│   └── [名稱]學習路徑.md
├── 01-[章節].md             9 個核心章節
├── ...
├── 09-[章節].md
├── 99-資源收集/             資源 + FAQ
│   ├── [名稱]資源收集.md
│   ├── [名稱]常見問題.md
│   └── ram/                原始素材存檔
└── [子 KB]/                可選深度子 KB
```

### 3.2 600 特有約定

| 約定 | 說明 | 範例 |
|------|------|------|
| **目錄命名** | `DDD 英文+繁體` | `610 医学健康` |
| **技術內容** | 繁體中文 + English 術語 | `BPU 推理加速` |
| **程式碼區塊** | 使用 fenced code block + 語言標記 | ` ```cpp ` |
| **硬體規格** | 使用表格呈現 | 電機參數、腳位定義 |
| **外部來源** | `source:` frontmatter | Waveshare Wiki、RDK Docs |

## 4. 深度子 KB 模式

### 4.1 何時使用深度子 KB

| 條件 | 說明 |
|------|------|
| 子主題有 **50+** 文件 | 獨立 MOC 管理 |
| 子主題有多個**獨立專案** | 每個專案有自己的文件樹 |
| 子主題有**外部文檔源** | 需要從外部 Wiki 持續同步 |
| 子主題有**完整的教程體系** | 獨立的學習路徑 |

### 4.3 層級深度規則

| 層級 | 深度 | 範例 | MOC 類型 |
|------|:---:|------|---------|
| L0 | 600 | `600 Applied Sciences/` | 領域 MOC |
| L1 | DDC | `689 业余手工/` | DDC KB MOC |
| L2 | 子 KB | `08-電子DIY/` | 子 KB MOC |
| L3 | 專案 | `智能小车/` | 專案 MOC (Dataview) |

## 5. LLM-Wiki 整合層

### 5.1 600 中的 LLM-Wiki 應用

LLM-Wiki 模式在 600 領域的兩種用法：

| 用法 | 場景 | 範例 |
|------|------|------|
| **Wiki 子庫** | 持續積累的研究主題 | 中醫學經典研究、AI 演算法 |
| **嵌入 ram/** | KB 內的原始素材管理 | `99-資源收集/ram/Book/` |

### 5.2 ram/ 原始素材目錄

```
[DDC KB]/99-資源收集/ram/
├── Book/               PDF 書籍、電子書
│   ├── MT4编程手册技术大全.pdf
│   ├── 日本蜡烛图教程.pdf
│   └── ...
├── articles/           網頁剪藏、文章
├── papers/             學術論文
├── economic-calendar/  結構化數據
└── [其他來源]/
```

### 5.3 來源追溯

所有從外部整合的內容必須標註來源：

```yaml
---
source: https://www.waveshare.net/wiki/UGV01
source: https://d-robotics.github.io/rdk_doc/Robot_development/tros
---
```

或在內文中：

```markdown
> 來源：[UGV01 Wiki](https://www.waveshare.net/wiki/UGV01)
```

## 6. raw/ 原始素材管理

### 6.1 素材分類

| 類型 | 目錄 | 格式 | 處理方式 |
|------|------|------|---------|
| PDF 書籍 | `ram/Book/` | .pdf | 保持原樣，不轉換 |
| 網頁文檔 | `ram/` 或直接整合 | .md | AI 結構化重寫 |
| 程式碼 | 嵌入章節內容 | code block | 格式保留 |
| 圖片 | Obsidian attachments | .png/.jpg | 自動管理 |
| 數據 | `ram/` 子目錄 | .md/.csv | 結構化存儲 |

### 6.2 素材生命周期

```
外部來源 (URL/PDF)
    │
    ▼
ram/ (原始存檔，保留原格式)
    │
    ▼ AI 處理
結構化 KB 章節 (.md，繁體中文)
    │
    ▼
合併到對應 DDC KB
    │
    ▼ Git 提交
版本控制 → 可追溯、可回滾
```

### 7.1 工作流總覽

```

    │
    ▼

           ▼

           ▼

│  - 更新 MOC 導航             │
│  - git add → commit → push   │

```

## 8. 內容類型與格式標準

### 8.1 600 領域內容類型

| 類型 | 格式 | 範例 |
|------|------|------|
| **教程** | 步驟式 + 程式碼 | Arduino 入門、MQL4 EA 開發 |
| **參考手冊** | 表格 + 指令速查 | JSON 指令集、MT4 快捷鍵 |
| **硬體規格** | 規格表 + 腳位圖 | 電機參數、ESP32 腳位 |
| **對比指南** | 雙欄/多欄對比表 | UGV01 vs UGV02、MT4 vs MT5 |
| **學習路徑** | Phase 階梯 | Phase1→Phase6 |
| **FAQ** | Q&A 格式 | 常見問題集中管理 |

### 8.2 程式碼範例標準

```cpp
// 必須包含：
// 1. 語言標記 (```cpp, ```python, ```bash)
// 2. 註解（繁體中文）
// 3. 完整可執行（非片段）

//+------------------------------------------------------------------+
//| 簡單移動平均交叉 EA                                              |
//+------------------------------------------------------------------+
input int FastMA = 10;     // 快線週期
input int SlowMA = 30;     // 慢線週期

void OnTick() {
   // 獲取 MA 值
   double fast = iMA(NULL, 0, FastMA, 0, MODE_SMA, PRICE_CLOSE, 0);
   double slow = iMA(NULL, 0, SlowMA, 0, MODE_SMA, PRICE_CLOSE, 0);
   // ...
}
```

|------|---|---|---|
| **重點** | 值 | 值 | 值 |
| 說明 | 文字 | 文字 | 文字 |
