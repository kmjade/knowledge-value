# 2. 通信任务——遥控器SBUS通信

> [!summary] 概述
> 本文介绍 STM32 通过解析 SBUS 信号实现遥控小车通信的方法，包括 SBUS 协议格式、信号解析代码实现以及遥控器连接状态处理。

---

## 一、遥控器通信原理简介

要实现一个遥控小车当然要有一个遥控器了，目前市面上常用的航模遥控器基本都是 2.4G 无线信号，从遥控器到小车的链路是这样的：

```
┌──────────┐      2.4G无线       ┌──────────┐      SBUS/PPM/PWM      ┌──────────┐
│  遥控器   │ ──────────────────→ │   接收机  │ ─────────────────────→ │  控制板   │
│          │                     │          │                        │          │
└──────────┘                     └──────────┘                        └──────────┘
```

![通信任务——遥控器SBUS通信](https://i-blog.csdnimg.cn/blog_migrate/7fdb724eba91f0fd70ce35fb7fa1e051.png)

### 接收机输出信号类型

接收机出来的信号一般有三种：

| 信号类型 | 说明 | 优缺点 |
|----------|------|--------|
| **SBUS 信号** | 一路串行信号，接串口按协议解析 | 高效省资源，**推荐使用** |
| **PPM 信号** | 一路信号，多路 PWM 混合成一帧 | 解析麻烦，浪费资源 |
| **PWM 信号** | 多路 PWM 接口表示多个信号 | 接线复杂，占用 IO |

> [!important] 推荐方案
> 本系列所有机器人都使用 SBUS 信号，推荐大家使用 SBUS 信号，解析简单且高效省资源。

## 二、SBUS 信号解析

### 2.1 SBUS 信号简介

**SBUS（Serial-BUS）** 是一种串口通信协议，广泛应用于航模遥控器（接收机）中。只用一根信号线就能传输多达 16 通道的数据，比多路 PWM 捕获高效且省资源。

#### （1）串口配置

| 参数 | 值 |
|------|-----|
| **波特率** | 100k bps |
| **数据位** | 8 位 |
| **停止位** | 2 位 |
| **校验位** | 偶校验（EVEN） |
| **流控** | 无 |
| **帧长度** | 25 字节 |

#### （2）协议格式

每帧数据 25 字节，结构如下：

| 字节 | 内容 | 说明 |
|------|------|------|
| Byte 0 | 0x0F | 帧头 |
| Byte 1-22 | 通道数据 | 16 个通道（ch1-ch16），每个通道 11 bit |
| Byte 23 | Flag | 标志位，表示遥控器通讯状态 |
| Byte 24 | 0x00 | 帧尾 |

**Flag 位说明：**

| Flag 值 | 说明 |
|---------|------|
| **0x00** | 遥控器正常连接（乐迪 AT9S） |
| **0xC0** | 遥控器断开 |

> [!tip] 失控保护
> 可以通过查询 Flag 位来采取失控保护措施。

#### （3）数据范围

不同遥控器的 SBUS 数据范围可能不同：

| 遥控器 | 范围 | 中值 |
|--------|------|------|
| **乐迪 AT9S** | 300 - 1700 | 1000 |
| **数传设备** | 341 - 1707 | 1024 |
| **标准 PWM** | 1000 - 2000 | 1500 |

> [!warning] 注意
> **一定要用串口调试助手把数据读出来看看范围是多少**，不然后面会出问题。

#### （4）SBUS 的负逻辑

> [!danger] 重要警告
> 接收机接板子**必须加硬件反相器**！因为 SBUS 的信号是采用负逻辑，也就是电平相反。

**为什么不能用软件取反？**

软件里面只能操作数据位（串口配置里面的数据位 8 位），无法操作停止位、校验位等！

**硬件反相器电路：**

```
接收机 SBUS ──→ [反相器] ──→ STM32 UART RX
                    │
                   GND
```

![硬件反相器电路](https://i-blog.csdnimg.cn/blog_migrate/45de567073ce793316023d189250e1dd.png)

如果自己画板子，加一个简单的反相器电路即可。

### 2.2 SBUS 数据结构定义

```c
#ifndef __SBUS_H
#define __SBUS_H
#include "sys.h"

#define SBUS_FRAME_SIZE		25
#define SBUS_INPUT_CHANNELS	16

// 定义sbus信号的最小值、最大值、中值、死区以及希望转换成PWM值的范围（1000-2000）
#define SBUS_RANGE_MIN 300.0f
#define SBUS_RANGE_MAX 1700.0f
#define SBUS_TARGET_MIN 1000.0f
#define SBUS_TARGET_MAX 2000.0f
#define DEAD_RANGE_MIN 960          // 死区
#define DEAD_RANGE_MAX 1040
#define SBUS_RANGE_MIDDLE 1000.0f
#define SBUS_CONNECT_FLAG 0x00

// 低速与高速模式，这里用一个二段开关控制速度档位
#define LOW_SPEED 0
#define HIGH_SPEED 1

// 定义四个摇杆与拨动开关的功能
#define YAW 1
#define THROTTLE 2
#define PITCH 3
#define ROLL 4
#define SPEED_MODE 6

extern int command[20];			   // 遥控器数据

typedef struct
{
    uint16_t signal[25];
    uint16_t CH1;   // 通道1数值
    uint16_t CH2;   // 通道2数值
    uint16_t CH3;   // 通道3数值
    uint16_t CH4;   // 通道4数值
    uint16_t CH5;   // 通道5数值
    uint16_t CH6;   // 通道6数值
    uint16_t CH7;   // 通道7数值
    uint16_t CH8;   // 通道8数值
    uint16_t CH9;   // 通道9数值
    uint16_t CH10;  // 通道10数值
    uint16_t CH11;  // 通道11数值
    uint16_t CH12;  // 通道12数值
    uint16_t CH13;  // 通道13数值
    uint16_t CH14;  // 通道14数值
    uint16_t CH15;  // 通道15数值
    uint16_t CH16;  // 通道16数值
    uint8_t ConnectState;  // 遥控器与接收器连接状态 0=未连接，1=正常连接
} SBUS_CH_Struct;

extern SBUS_CH_Struct SBUS_CH;

// SBUS信号解析相关函数
u8 update_sbus(u8 *buf);
u16 sbus_to_pwm(u16 sbus_value);
float sbus_to_Range(u16 sbus_value, float p_min, float p_max);

#endif
```

### 2.3 SBUS 信号解析实现

```c
#include "sbus.h"

SBUS_CH_Struct SBUS_CH;
int command[20];  // 遥控器数据

// 将sbus信号转化为通道值
u8 update_sbus(u8 *buf)
{
    int i;
    for (i = 0; i < 25; i++)
        SBUS_CH.signal[i] = buf[i];
    
    if (buf[23] == SBUS_CONNECT_FLAG)
    {
        SBUS_CH.ConnectState = 1;
        
        // 解析16个通道数据，每个通道11位
        SBUS_CH.CH1 = ((int16_t)buf[ 1] >> 0 | ((int16_t)buf[ 2] << 8 )) & 0x07FF;
        SBUS_CH.CH2 = ((int16_t)buf[ 2] >> 3 | ((int16_t)buf[ 3] << 5 )) & 0x07FF;
        SBUS_CH.CH3 = ((int16_t)buf[ 3] >> 6 | ((int16_t)buf[ 4] << 2 ) | (int16_t)buf[ 5] << 10 ) & 0x07FF;
        SBUS_CH.CH4 = ((int16_t)buf[ 5] >> 1 | ((int16_t)buf[ 6] << 7 )) & 0x07FF;
        SBUS_CH.CH5 = ((int16_t)buf[ 6] >> 4 | ((int16_t)buf[ 7] << 4 )) & 0x07FF;
        SBUS_CH.CH6 = ((int16_t)buf[ 7] >> 7 | ((int16_t)buf[ 8] << 1 ) | (int16_t)buf[ 9] << 9 ) & 0x07FF;
        SBUS_CH.CH7 = ((int16_t)buf[ 9] >> 2 | ((int16_t)buf[10] << 6 )) & 0x07FF;
        SBUS_CH.CH8 = ((int16_t)buf[10] >> 5 | ((int16_t)buf[11] << 3 )) & 0x07FF;
        
        SBUS_CH.CH9  = ((int16_t)buf[12] << 0 | ((int16_t)buf[13] << 8 )) & 0x07FF;
        SBUS_CH.CH10 = ((int16_t)buf[13] >> 3 | ((int16_t)buf[14] << 5 )) & 0x07FF;
        SBUS_CH.CH11 = ((int16_t)buf[14] >> 6 | ((int16_t)buf[15] << 2 ) | (int16_t)buf[16] << 10 ) & 0x07FF;
        SBUS_CH.CH12 = ((int16_t)buf[16] >> 1 | ((int16_t)buf[17] << 7 )) & 0x07FF;
        SBUS_CH.CH13 = ((int16_t)buf[17] >> 4 | ((int16_t)buf[18] << 4 )) & 0x07FF;
        SBUS_CH.CH14 = ((int16_t)buf[18] >> 7 | ((int16_t)buf[19] << 1 ) | (int16_t)buf[20] << 9 ) & 0x07FF;
        SBUS_CH.CH15 = ((int16_t)buf[20] >> 2 | ((int16_t)buf[21] << 6 )) & 0x07FF;
        SBUS_CH.CH16 = ((int16_t)buf[21] >> 5 | ((int16_t)buf[22] << 3 )) & 0x07FF;
        
        return 1;
    }
    else
    {
        SBUS_CH.ConnectState = 0;
        return 0;
    }
}

// 将SBUS值转换为PWM值（1000-2000）
u16 sbus_to_pwm(u16 sbus_value)
{
    float pwm;
    pwm = (float)(sbus_value - SBUS_RANGE_MIN) / (SBUS_RANGE_MAX - SBUS_RANGE_MIN) 
          * (SBUS_TARGET_MAX - SBUS_TARGET_MIN) + SBUS_TARGET_MIN;
    return (u16)pwm;
}

// 将SBUS值映射到指定范围
float sbus_to_Range(u16 sbus_value, float p_min, float p_max)
{
    float value;
    value = (float)(sbus_value - SBUS_RANGE_MIN) / (SBUS_RANGE_MAX - SBUS_RANGE_MIN) 
            * (p_max - p_min) + p_min;
    return value;
}
```

## 三、通信任务实现

### 3.1 通道映射说明

SBUS 信号读出来是 300-1700 的数据，可以转换成 1000-2000 的 PWM 值（方便后续处理）。

**遥控器通道分配：**

| 通道 | 功能 | 类型 |
|------|------|------|
| CH1 | YAW（偏航） | 摇杆，连续值 |
| CH2 | THROTTLE（油门） | 摇杆，连续值 |
| CH3 | PITCH（俯仰） | 摇杆，连续值 |
| CH4 | ROLL（横滚） | 摇杆，连续值 |
| CH5-16 | 拨码开关 | 离散值 |

**拨码开关值：**

| 开关类型 | 通道值 | 说明 |
|----------|--------|------|
| **二段开关** | 300 / 1700 | 两个状态 |
| **三段开关** | 300 / 1000 / 1700 | 三个状态 |

### 3.2 通信任务代码

在上一章的工程模板基础上，创建 `sbus.c` 和 `sbus.h` 文件，并在通信任务中处理遥控器数据：

```c
// 通信任务
void communicate_task(void *p_arg)
{
    OS_ERR err;
    CPU_SR_ALLOC();
    
    while (1)
    {
        // 检查遥控器连接状态
        if (SBUS_CH.ConnectState == 1)
        {
            // 遥控器正常连接
            // 解析遥控器数据到command数组
            
            // 摇杆通道（转换为PWM值）
            command[YAW] = sbus_to_pwm(SBUS_CH.CH1);
            command[THROTTLE] = sbus_to_pwm(SBUS_CH.CH2);
            command[PITCH] = sbus_to_pwm(SBUS_CH.CH3);
            command[ROLL] = sbus_to_pwm(SBUS_CH.CH4);
            
            // 拨码开关（判断档位）
            if (SBUS_CH.CH6 < 500)
                command[SPEED_MODE] = LOW_SPEED;
            else
                command[SPEED_MODE] = HIGH_SPEED;
        }
        else
        {
            // 遥控器断开，执行失控保护
            // 停止电机等操作
        }
        
        delay_ms(10);
    }
}
```

### 3.3 串口中断接收

SBUS 信号通过串口接收，建议使用 DMA + 空闲中断的方式：

```c
// UART 接收中断回调
void USART1_IRQHandler(void)
{
    u8 res;
    
    if (USART_GetITStatus(USART1, USART_IT_RXNE) != RESET)
    {
        res = USART_ReceiveData(USART1);
        // 累积接收数据，直到收到25字节
    }
    
    if (USART_GetITStatus(USART1, USART_IT_IDLE) != RESET)
    {
        // 清除空闲中断标志
        USART_ReceiveData(USART1);
        
        // 解析SBUS数据
        update_sbus(sbus_buffer);
    }
}
```

## 五、调试技巧

### 5.1 验证 SBUS 数据

1. 使用串口调试助手连接接收机
2. 配置串口：100k 波特率，8N2E（偶校验）
3. 观察是否收到 25 字节数据帧
4. 摇动摇杆观察数据变化

### 5.2 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 收不到数据 | 波特率错误 | 确认 100k 波特率 |
| 数据乱码 | 无硬件反相器 | 添加反相器电路 |
| 连接状态异常 | Flag 位判断错误 | 用串口助手确认 Flag 值 |
| 通道值不对 | 范围参数错误 | 测量实际范围后修改 |
