# 8. RoboMaster SDK 如何记录日志

> [!info] 本章概述
> 介绍 RoboMaster SDK 的日志配置方法，包括日志等级设置和日志文件的使用。

---

## 8.1 配置日志等级

RoboMaster SDK 的日志等级默认为 `ERROR`，用户可根据自己的需要进行修改。

### 日志等级说明

| 等级 | 说明 | 用途 |
|------|------|------|
| `logging.DEBUG` | 调试信息 | 详细调试 |
| `logging.INFO` | 一般信息 | 常规运行 |
| `logging.WARNING` | 警告信息 | 潜在问题 |
| `logging.ERROR` | 错误信息（默认） | 错误报告 |
| `logging.CRITICAL` | 严重错误 | 严重故障 |

### 设置日志等级

```python
import logging
from robomaster import logger

# 设置为 WARNING 级别
logger.setLevel(logging.WARNING)

# 设置为 INFO 级别
logger.setLevel(logging.INFO)

# 设置为 DEBUG 级别
logger.setLevel(logging.DEBUG)
```

> [!tip] 建议
> - 开发调试阶段：使用 `logging.DEBUG` 或 `logging.INFO`
> - 生产运行阶段：使用 `logging.WARNING` 或 `logging.ERROR`

## 8.2 日志文件的使用

如果用户在使用过程中遇到问题，可以将日志写入文件中，并将日志文件提供给技术支持人员。

### 生成日志文件

在程序最开始添加语句：

```python
import robomaster

# 启用日志文件记录
robomaster.enable_logging_to_file()
```

### 日志文件信息

| 项目 | 说明 |
|------|------|
| **存放路径** | 程序同级目录 |
| **命名格式** | `RoboMasterSDK_YYYYMMDDHHMMSS_log.txt` |
| **示例** | `RoboMasterSDK_20260523150030_log.txt` |

### 提交日志文件

将生成的系统日志文件发送到邮箱：**developer@dji.com**

**邮件模板：**

[请详细描述遇到的问题]

复现步骤：
1. [步骤一]
2. [步骤二]
3. [步骤三]

环境信息：
- 操作系统：[Windows 10 / Ubuntu 20.04 / macOS]

## 完整示例代码

参考例程：`/examples/01_robot/00_logger.py` ([GitHub](https://github.com/dji-sdk/robomaster-sdk))

```python
import robomaster
from robomaster import robot

if __name__ == '__main__':
    # 启用日志文件记录
    robomaster.enable_logging_to_file()
    
    # 如果本地IP 自动获取不正确，手动指定本地IP地址
    # robomaster.config.LOCAL_IP_STR = "192.168.2.20"
    
    ep_robot = robot.Robot()
    
    # 指定连接方式为AP 直连模式
    ep_robot.initialize(conn_type='ap')
    
    version = ep_robot.get_version()
    print("Robot version: {0}".format(version))
    
    ep_robot.close()
```

## 高级日志配置

### 自定义日志处理器

```python
import logging
from robomaster import logger

# 创建文件处理器
file_handler = logging.FileHandler('custom_log.txt')
file_handler.setLevel(logging.DEBUG)

# 创建格式化器
formatter = logging.Formatter(
    '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
file_handler.setFormatter(formatter)

# 添加处理器
logger.addHandler(file_handler)
```

### 同时输出到控制台和文件

```python
import logging
from robomaster import logger

# 控制台处理器
console_handler = logging.StreamHandler()
console_handler.setLevel(logging.INFO)

# 文件处理器
file_handler = logging.FileHandler('robomaster.log')
file_handler.setLevel(logging.DEBUG)

# 设置格式
formatter = logging.Formatter(
    '%(asctime)s - %(levelname)s - %(message)s'
)
console_handler.setFormatter(formatter)
file_handler.setFormatter(formatter)

# 添加处理器
logger.addHandler(console_handler)
logger.addHandler(file_handler)

# 设置总日志等级
logger.setLevel(logging.DEBUG)
```

## 日志等级选择指南

| 场景 | 推荐等级 | 说明 |
|------|----------|------|
| **正常使用** | `ERROR`（默认） | 只记录错误 |
| **调试连接问题** | `INFO` | 查看连接过程 |
| **调试通信问题** | `DEBUG` | 查看详细通信数据 |
| **性能分析** | `WARNING` | 关注潜在问题 |
| **问题排查** | `DEBUG` + 日志文件 | 完整信息记录 |

## 常见问题

### 1. 日志文件未生成

**解决方案：**
- 确认 `enable_logging_to_file()` 在程序最开始调用
- 检查程序运行目录是否有写入权限
- 确认程序正常执行完成

### 2. 日志信息不完整

**解决方案：**
- 降低日志等级（如从 `ERROR` 改为 `DEBUG`）
- 确认日志处理器配置正确

### 3. 日志文件过大

**解决方案：**
- 使用 `RotatingFileHandler` 限制文件大小
- 定期清理历史日志文件

```python
from logging.handlers import RotatingFileHandler

# 最大 1MB，保留 3 个备份
handler = RotatingFileHandler(
    'robomaster.log', 
    maxBytes=1024*1024, 
    backupCount=3
)
```
