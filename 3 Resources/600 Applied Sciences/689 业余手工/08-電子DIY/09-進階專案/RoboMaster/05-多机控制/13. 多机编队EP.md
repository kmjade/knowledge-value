# 13. RoboMaster SDK 多机编队EP

> [!info] 本章概述
> 介绍如何使用 RoboMaster SDK 控制 EP 机器人进行多机编队，包括环境配置、组网连接和示例程序运行。

---

## 13.1 Demo 的环境要求及物料清单

### 编程环境

#### （1）下载 RoboMaster-SDK 压缩包

**下载地址：**
- [GitHub 下载](https://github.com/dji-sdk/RoboMaster-SDK)
- [Gitee 备用下载](https://gitee.com/xitinglin/RoboMaster-SDK)

**下载步骤：**
1. 点击 `Code`
2. 点击 `Download ZIP` 下载压缩包

**压缩包内容：**

| 文件 | 说明 |
|------|------|
| `VisualCppRedist_AIO_20200707.exe` | VC 相关运行库 |
| `visualcppbuildtools_full.exe` | VS Build Tool |
| `examples/` | RoboMaster SDK 示例代码 |
| `lib_install.bat` | 第三方库安装脚本 |

---

#### （2）安装必要的 VC 库

运行 `VisualCppRedist_AIO_20200707.exe`，完成安装。

> [!warning] 注意
> 不安装 VC 库，使用 SDK 会引起运行错误。

---

#### （3）安装 VS Build Tool

运行 `visualcppbuildtools_full.exe`，完成安装。

---

#### （4）Python 环境安装

**版本要求：** Python 3.6.6 - 3.8.9（64 位）

**安装步骤：**

1. 从 [Python 官网](https://www.python.org/downloads/) 下载安装包
2. 确认安装包版本是 **64-bit**
3. 勾选 `Add Python 3.7 to Path`
4. 选择 `Install Now` 进行安装

**验证安装：**

```bash
# 按 Win + R，输入 cmd 打开命令提示符
python --version
# 输出：Python 3.7.8
```

---

#### （5）Python 第三方依赖库安装

**方法一：使用安装脚本**

在 `RoboMaster-SDK-master` 目录中找到 `lib_install.bat`，右键选择 **以管理员身份运行**。

**方法二：手动安装**

以管理员身份运行命令提示符，依次输入：

```bash
pip install robomaster
pip install netaddr
pip install netifaces
pip install myqr
```

## 13.2 EP 组网连接

### 步骤 1：设置路由器组网模式

将每台 EP 机器人设置为 **路由器组网模式**，并将电脑与机器人加入到同一个局域网内。

**切换开关位置：**

将智能中控的连接模式开关切换至 **组网模式**。

---

### 步骤 2：生成二维码

参考示例代码：`/examples/01_robot/05_sta_conn_helper.py`

```python
import time
import robomaster
from robomaster import conn
from MyQR import myqr
from PIL import Image

QRCODE_NAME = "qrcode.png"

if __name__ == '__main__':
    helper = conn.ConnectionHelper()
    info = helper.build_qrcode_string(ssid="RoboMaster_SDK_WIFI", password="12341234")
    myqr.run(words=info)
    time.sleep(1)
    img = Image.open(QRCODE_NAME)
    img.show()
    if helper.wait_for_connection():
        print("Connected!")
    else:
        print("Connect failed!")
```

> [!warning] 重要
> 示例代码中的 `ssid`（路由器名称）和 `password`（路由器密码）需要根据实际的路由器信息进行填写。

### 步骤 3：扫码连接

1. 运行示例代码，会出现二维码图片
2. 按下机器人智能中控上的 **扫码连接按键**
3. 扫描二维码进行组网连接

**运行结果：**

```
Connected!
```

**成功提示：** 机器人的灯效从白色呼吸变为 **青绿色常亮**。

> [!note] 注意
> 一共 6 台 EP 小车需要连接 WiFi，首次逐个连接 6 台 EP 时，连接每台小车都要重复运行示例代码。连接完成后，下次重启小车时会自动连接。

## 示例代码分析

### 6 台 EP 编队示例

**示例文件：** `/examples/15_multi_robot/multi_ep/03_six_ep_demo.py`

**核心结构：**

```python
from multi_robomaster import multi_robot

# SN 列表
robot_sn_list = ['SN1', 'SN2', 'SN3', 'SN4', 'SN5', 'SN6']

# 创建多机对象
multi_ep = multi_robot.MultiEP()
multi_ep.initialize()

# 编号
for i, sn in enumerate(robot_sn_list):
    multi_ep.number_id_by_sn([i, sn])

# 编组
group_all = multi_ep.build_group([0, 1, 2, 3, 4, 5])

# 定义任务函数
def formation_task(robot_group):
    # 编队动作
    robot_group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()

# 执行任务
multi_ep.run([group_all, formation_task])

# 释放资源
multi_ep.close()
```

### 2 台 EP 编队示例

**示例文件：** `/examples/15_multi_robot/multi_ep/02_two_ep_demo.py`

**适用场景：** 只有 2 台 EP 时使用

```python
from multi_robomaster import multi_robot

robot_sn_list = ['SN1', 'SN2']

multi_ep = multi_robot.MultiEP()
multi_ep.initialize()

multi_ep.number_id_by_sn([0, robot_sn_list[0]], [1, robot_sn_list[1]])

# 编组
group_all = multi_ep.build_group([0, 1])

# 定义任务
def move_forward_task(robot_group):
    robot_group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()

def move_backward_task(robot_group):
    robot_group.chassis.move(x=-1, y=0, z=0, xy_speed=0.7).wait_for_completed()

# 分组执行不同任务
group_0 = multi_ep.build_group([0])
group_1 = multi_ep.build_group([1])

multi_ep.run([group_0, move_forward_task], [group_1, move_backward_task])

multi_ep.close()
```

## 编队任务设计模式

### 模式一：同步执行

所有机器人执行相同动作：

```python
def sync_task(robot_group):
    robot_group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()

multi_ep.run([group_all, sync_task])
```

---

### 模式二：分组并行

不同组执行不同动作：

```python
def task_01(robot_group):
    robot_group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()

def task_23(robot_group):
    robot_group.chassis.move(x=-1, y=0, z=0, xy_speed=0.7).wait_for_completed()

group_01 = multi_ep.build_group([0, 1])
group_23 = multi_ep.build_group([2, 3])

multi_ep.run([group_01, task_01], [group_23, task_23])
```

---

### 模式三：顺序执行

动作按顺序执行：

```python
def sequence_tasks(robot_group):
    # 前进
    robot_group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()
    # 左转
    robot_group.chassis.move(x=0, y=0, z=90, z_speed=45).wait_for_completed()
    # 前进
    robot_group.chassis.move(x=1, y=0, z=0, xy_speed=0.7).wait_for_completed()

multi_ep.run([group_all, sequence_tasks])
```

## 常见问题

### 1. 连接失败

**解决方案：**
- 确认路由器模式设置正确
- 确认 SN 号输入正确
- 确认所有机器人在同一局域网

### 2. 动作执行不同步

**解决方案：**
- 确认 `wait_for_completed()` 已调用
- 检查任务函数格式是否正确

### 3. VC 库安装问题

**解决方案：**
- 以管理员身份运行安装程序
- 重启电脑后重试
