# 5. RoboMaster EP UART 接口

> [!info] 本章概述
> 介绍 RoboMaster EP UART 接口的引脚说明、串口配置、Python 编程和明文 SDK 使用方法。

---

## 5.1 介绍

UART 是第三方平台跟 EP 连接的一种方式：

- 用户通过 UART 可以很方便地将搭载在 EP 上的单片机跟 EP 建立连接
- 在单片机上实现交互逻辑
- 使用明文 SDK 和 EP 机器人进行通信
- 实现对 EP 的自动化控制

---

## 5.2 引脚说明

EP 的 UART 接口设置在运动控制器上。

![UART 接口](https://robomaster-dev.readthedocs.io/zh-cn/latest/_images/uart.png)

## 5.3 串口配置

### 默认参数

| 参数 | 值 |
|------|-----|
| **通讯接口** | UART |
| **波特率** | 115200 |
| **数据位** | 8 |
| **停止位** | 1 |

## 5.5 Python 编程示例

### 步骤概述

1. PC 通过串口转 USB 连接到 EP 运动控制器的 UART 接口
2. PC 端开启串口调试助手，选择打开串口对应 COM 口
3. 打开已跟 EP 建立连接设备的官方 App，进入 Python 编程模式
4. 在 Python 编程界面下，写一个简单的程序
5. 使用 `read_string()` 读取串口数据，打印出来后再通过 `write_string()` 进行转发
6. 在串口调试助手上发送一个字符串，看是否能正确收到发出去的字符串

### Python 代码示例

```python
from robomaster import robot

def uart_echo():
    """串口回显示例"""
    ep_robot = robot.Robot()
    ep_robot.initialize(conn_type="ap")
    
    # 获取 UART 对象
    ep_uart = ep_robot.uart
    
    print("等待串口数据...")
    
    try:
        while True:
            # 读取串口字符串
            data = ep_uart.read_string()
            if data:
                print(f"收到: {data}")
                # 回传数据
                ep_uart.write_string(data)
                print(f"发送: {data}")
    except KeyboardInterrupt:
        print("\n停止")
    
    ep_robot.close()

if __name__ == "__main__":
    uart_echo()
```

## 5.6 明文 SDK 示例

### 测试步骤

1. PC 通过串口转 USB 连接到 EP 运动控制器的 UART 接口
2. PC 端开启串口调试助手，选择打开串口对应 COM 口
3. 在串口调试助手上发送字符串 `command;`
4. 若能收到 EP 发回来的 `ok` 表示明文 SDK 解析成功

### 常用明文 SDK 命令

| 命令 | 说明 | 返回 |
|------|------|------|
| `command;` | 进入命令模式 | `ok` |
| `version;` | 查询版本 | `版本信息` |
| `chassis speed x=0.5 y=0 z=0;` | 底盘速度控制 | `ok` |
| `led control comp=all r=255 g=0 b=0 effect=on;` | LED 控制 | `ok` |
| `quit;` | 退出命令模式 | `ok` |

### 明文 SDK 控制示例

```python
import time
import serial

def text_sdk_demo():
    """通过串口发送明文 SDK 命令"""
    # 打开串口
    ser = serial.Serial('COM3', 115200, timeout=1)
    
    # 进入命令模式
    ser.write(b'command;')
    response = ser.readline().decode().strip()
    print(f"进入命令模式: {response}")
    
    # 获取版本
    ser.write(b'version;')
    response = ser.readline().decode().strip()
    print(f"版本: {response}")
    
    # 控制底盘前进
    ser.write(b'chassis move x=1 y=0 z=0 xy_speed=0.7;')
    response = ser.readline().decode().strip()
    print(f"移动: {response}")
    
    time.sleep(3)
    
    # 退出命令模式
    ser.write(b'quit;')
    response = ser.readline().decode().strip()
    print(f"退出: {response}")
    
    ser.close()

if __name__ == "__main__":
    text_sdk_demo()
```

## 5.7 Python API

### UART 类方法

| 方法 | 说明 | 参数 |
|------|------|------|
| `write_string(data)` | 发送字符串 | data: 字符串 |
| `read_string()` | 读取字符串 | 无 |
| `write_bytes(data)` | 发送字节 | data: 字节数据 |
| `read_bytes(size)` | 读取字节 | size: 字节数 |

### 完整示例

```python
import time
from robomaster import robot

def uart_communication():
    """UART 完整通信示例"""
    ep_robot = robot.Robot()
    ep_robot.initialize(conn_type="ap")
    
    ep_uart = ep_robot.uart
    
    # 发送字符串
    ep_uart.write_string("Hello from EP!")
    print("发送: Hello from EP!")
    
    # 接收字符串
    received = ep_uart.read_string()
    print(f"接收: {received}")
    
    # 发送字节数据
    ep_uart.write_bytes(b'\x01\x02\x03')
    print("发送字节: 01 02 03")
    
    # 接收字节数据
    data = ep_uart.read_bytes(10)
    print(f"接收字节: {data.hex()}")
    
    ep_robot.close()

if __name__ == "__main__":
    uart_communication()
```

## 常见问题

### 1. 串口无法打开

**解决方案：**
- 检查 COM 口是否被占用
- 确认波特率设置正确（115200）
- 验证 USB 转串口驱动是否安装

### 2. 数据乱码

**解决方案：**
- 确认波特率、数据位、停止位配置一致
- 检查是否使用正确的编码格式
- 验证硬件连接是否正常

### 3. 通信不稳定

**解决方案：**
- 检查接线是否松动
- 确认电源供电稳定
- 添加适当的通信协议（校验、重传）
