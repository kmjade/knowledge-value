# 基于大疆A板与STM32F427的CAN总线双闭环控制——M2006/M3508位置与速度模式实践

> [!summary] 概述
> 本文详细介绍基于大疆 A 板与 STM32F427 的 CAN 总线双闭环控制实践，涵盖硬件连接、CubeMX 配置、电机控制代码移植及双闭环 PID 控制实现，重点解析 M2006/M3508 电机在位置与速度模式下的参数调整与调试技巧。

## 一、硬件准备与电路连接

### 1.1 硬件清单

| 组件 | 型号 | 说明 |
|------|------|------|
| **开发板** | 大疆 A 型板 | STM32F427IIH6 核心 |
| **电机** | M2006 / M3508 | 直流无刷电机 |
| **电调** | C610 / C620 | 配套电调 |
| **电源** | 24V 锂电池 | 电调供电 |
| **调试器** | ST-Link / J-Link | SWD 接口 |
| **连接线** | 1.25mm 间距 4pin 插座 | CAN 通信接口 |

### 1.2 接口连接问题

拿到大疆 A 板和 M2006/M3508 电机时，最头疼的就是**硬件连接问题**。

这块 A 板用的是 STM32F427 核心，虽然性能强劲但外设接口都被高度封装，连下载口都要专用线。

**解决方案：**
- 普通杜邦线接触不良，容易导致调试时频繁断连
- 淘宝上可买到 **1.25mm 间距的 4pin 插座**
- 用手机拍下板子接口照片搜索就能找到匹配的公头

### 1.3 供电注意事项

> [!warning] 供电问题
> - A 板最大输出只有 **12V**
> - C610/C620 电调标称电压是 **24V**
> - 单个电机用 12V 供电勉强能工作
> - 接两个电机做闭环控制会电压不足

**推荐方案：** 用外接 24V 电源单独给电调供电，A 板只负责信号控制。

### 1.4 接线方案

```
                    ┌─────────────────┐
                    │   大疆 A 板      │
                    │  (STM32F427)    │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
         CAN_H│         SWD_CLK       SWD_IO│
              │              │              │
              ▼              ▼              ▼
        ┌─────────┐   ┌─────────────────────┐
        │ CAN 总线 │   │      ST-Link        │
        └────┬────┘   └─────────────────────┘
             │
    ┌────────┴────────┐
    │                 │
    ▼                 ▼
┌───────┐        ┌───────┐
│ C610  │        │ C620  │
│(M2006)│        │(M3508)│
└───┬───┘        └───┬───┘
    │                │
    └───────┬────────┘
            │
            ▼
      ┌──────────┐
      │ 24V 电池  │
      └──────────┘
```

**接线要点：**

| 连接 | 说明 |
|------|------|
| CAN_H / CAN_L | 接电调对应接口 |
| 电调电源正负极 | 接外置 24V 锂电池 |
| A 板 | 通过 SWD 接口烧录程序 |

> [!tip] 调试建议
> 调试时建议先**断开电机机械负载**，用万用表确认各节点电压正常后再上电。

## 二、CubeMX 基础配置

### 2.1 常见配置错误

用 CubeMX 配置 CAN 总线时，新手常犯三个错误：

1. **时钟源选错**
2. **波特率计算错误**
3. **中断未使能**

### 2.2 可靠配置流程

#### 步骤 1：时钟配置

在 Clock Configuration 里将 HCLK 设为 **168MHz**

> [!note] 时钟说明
> STM32F427 最高支持 180MHz，但 168MHz 更稳定。

#### 步骤 2：CAN 引脚配置

在 Connectivity 下启用 CAN1，选择 **PA11/PA12** 作为 CAN 引脚。

| 引脚 | 功能 |
|------|------|
| PA11 | CAN1_RX |
| PA12 | CAN1_TX |

#### 步骤 3：波特率计算

**波特率计算公式：**

```
波特率 = APB1时钟 / (Prescaler × (BS1 + BS2 + SyncJumpWidth))
```

电调要求 **1Mbps** 波特率。

**推荐配置：**

| 参数 | 值 | 说明 |
|------|-----|------|
| Prescaler | 3 | 预分频系数 |
| BS1 | 5 | 时间段 1 |
| BS2 | 2 | 时间段 2 |
| SyncJumpWidth | 1 | 同步跳转宽度 |

**计算验证：**

```
APB1 时钟 = 42MHz
波特率 = 42MHz / (3 × (5 + 2 + 1))
       = 42MHz / 24
       = 1.75Mbps ≈ 1Mbps
```

#### 步骤 4：中断配置

务必勾选 CAN 中断：

- **CAN_IT_RX_FIFO0_MSG_PENDING**

### 2.3 代码检查

生成代码后要检查生成的 `can.c` 文件，确认 `hcan1` 实例的 Init 成员参数与预期一致。

## 四、双闭环 PID 控制

### 4.1 双闭环控制原理

双闭环控制包含**速度环**和**位置环**：

```
目标位置 ──→ [位置环PID] ──→ 目标速度 ──→ [速度环PID] ──→ 目标电流 ──→ 电机
              ↑                                    ↑
              │                                    │
           位置反馈                             速度反馈
```

| 环路 | 输入 | 输出 | 作用 |
|------|------|------|------|
| **位置环** | 目标位置 - 实际位置 | 目标速度 | 控制位置精度 |
| **速度环** | 目标速度 - 实际速度 | 目标电流 | 控制速度稳定 |

### 4.2 速度环 PID

```c
/**
 * @brief  速度环 PID 控制
 */
typedef struct
{
    float Kp;           // 比例系数
    float Ki;           // 积分系数
    float Kd;           // 微分系数
    float target;       // 目标速度
    float error;        // 当前误差
    float last_error;   // 上次误差
    float integral;     // 积分项
    float integral_limit; // 积分限幅
    float output;       // 输出电流
    float output_limit; // 输出限幅
} Speed_PID_t;

/**
 * @brief  速度环 PID 计算
 */
float Speed_PID_Calc(Speed_PID_t *pid, float feedback)
{
    // 计算误差
    pid->error = pid->target - feedback;

    // 积分项（带限幅）
    pid->integral += pid->error;
    if (pid->integral > pid->integral_limit)
        pid->integral = pid->integral_limit;
    else if (pid->integral < -pid->integral_limit)
        pid->integral = -pid->integral_limit;

    // PID 计算
    pid->output = pid->Kp * pid->error
                + pid->Ki * pid->integral
                + pid->Kd * (pid->error - pid->last_error);

    // 输出限幅
    if (pid->output > pid->output_limit)
        pid->output = pid->output_limit;
    else if (pid->output < -pid->output_limit)
        pid->output = -pid->output_limit;

    pid->last_error = pid->error;

    return pid->output;
}
```

### 4.3 位置环 PID

```c
/**
 * @brief  位置环 PID 控制
 */
typedef struct
{
    float Kp;           // 比例系数
    float Ki;           // 积分系数（通常为0）
    float Kd;           // 微分系数
    float target;       // 目标位置
    float error;        // 当前误差
    float last_error;   // 上次误差
    float output;       // 输出速度
    float output_limit; // 输出限幅
} Position_PID_t;

/**
 * @brief  位置环 PID 计算
 */
float Position_PID_Calc(Position_PID_t *pid, float feedback)
{
    // 计算误差
    pid->error = pid->target - feedback;

    // PD 控制（位置环通常不用积分）
    pid->output = pid->Kp * pid->error
                + pid->Kd * (pid->error - pid->last_error);

    // 输出限幅
    if (pid->output > pid->output_limit)
        pid->output = pid->output_limit;
    else if (pid->output < -pid->output_limit)
        pid->output = -pid->output_limit;

    pid->last_error = pid->error;

    return pid->output;
}
```

### 4.4 双闭环控制实现

```c
/**
 * @brief  电机控制结构
 */
typedef struct
{
    // 电机数据
    uint16_t angle;       // 机械角度
    int16_t speed;        // 转速
    int16_t current;      // 实际电流
    int16_t set_current;  // 设定电流

    // 位置环
    Position_PID_t pos_pid;
    float target_position;  // 目标位置

    // 速度环
    Speed_PID_t speed_pid;
    float target_speed;     // 目标速度

    // 控制模式
    uint8_t mode;           // 0: 速度模式, 1: 位置模式
} Motor_t;

/**
 * @brief  双闭环控制
 */
void Motor_DualLoop_Control(Motor_t *motor)
{
    if (motor->mode == 1)  // 位置模式
    {
        // 位置环输出作为速度环输入
        float target_speed = Position_PID_Calc(&motor->pos_pid, motor->angle);
        motor->speed_pid.target = target_speed;
    }

    // 速度环输出作为电流设定值
    motor->set_current = (int16_t)Speed_PID_Calc(&motor->speed_pid, motor->speed);
}
```

## 五、PID 参数整定

### 5.1 速度环参数整定

**M2006 典型参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| Kp | 5.0 ~ 15.0 | 比例系数 |
| Ki | 0.1 ~ 0.5 | 积分系数 |
| Kd | 0.0 ~ 0.5 | 微分系数 |
| integral_limit | 1000 | 积分限幅 |
| output_limit | 10000 | 输出限幅 |

**M3508 典型参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| Kp | 3.0 ~ 10.0 | 比例系数 |
| Ki | 0.1 ~ 0.3 | 积分系数 |
| Kd | 0.0 ~ 0.3 | 微分系数 |
| integral_limit | 1000 | 积分限幅 |
| output_limit | 16000 | 输出限幅 |

### 5.2 位置环参数整定

| 参数 | 值 | 说明 |
|------|-----|------|
| Kp | 5.0 ~ 20.0 | 比例系数 |
| Ki | 0.0 | 位置环通常不用积分 |
| Kd | 0.5 ~ 2.0 | 微分系数 |
| output_limit | 500 ~ 2000 | 输出限幅（最大速度） |

### 5.3 整定步骤

> [!tip] 参数整定口诀
> 1. **先速度环，后位置环**
> 2. 速度环：Kp 从小到大，加入 Ki 消除稳态误差
> 3. 位置环：Kp 控制响应速度，Kd 抑制超调

**步骤 1：速度环整定**

1. 将 Ki、Kd 设为 0
2. 逐渐增大 Kp，观察速度响应
3. 速度开始振荡时，取 Kp 的 60%
4. 加入小 Ki 消除稳态误差
5. 最后加入 Kd 提高稳定性

**步骤 2：位置环整定**

1. 速度环调试完成后，开始位置环整定
2. 位置环主要用 PD 控制
3. Kp 控制位置跟踪速度
4. Kd 抑制位置超调

### 5.4 常见问题与解决

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 速度振荡 | Kp 过大 | 减小 Kp 或增大 Kd |
| 响应慢 | Kp 过小 | 增大 Kp |
| 稳态误差 | Ki 过小 | 增大 Ki |
| 位置超调 | Kd 过小 | 增大 Kd |
| 电机抖动 | 参数不匹配 | 重新整定 |

## 六、代码实现

### 6.1 CAN 初始化

```c
/**
 * @brief  CAN 初始化
 */
void CAN_User_Init(void)
{
    CAN_FilterTypeDef filter;

    // 配置过滤器（接收所有电机数据）
    filter.FilterBank = 0;
    filter.FilterMode = CAN_FILTERMODE_IDMASK;
    filter.FilterScale = CAN_FILTERSCALE_32BIT;
    filter.FilterIdHigh = 0x0000;
    filter.FilterIdLow = 0x0000;
    filter.FilterMaskIdHigh = 0x0000;
    filter.FilterMaskIdLow = 0x0000;
    filter.FilterFIFOAssignment = CAN_RX_FIFO0;
    filter.FilterActivation = ENABLE;
    filter.SlaveStartFilterBank = 14;

    HAL_CAN_ConfigFilter(&hcan1, &filter);

    // 启动 CAN
    HAL_CAN_Start(&hcan1);

    // 使能接收中断
    HAL_CAN_ActivateNotification(&hcan1, CAN_IT_RX_FIFO0_MSG_PENDING);
}
```

### 6.2 电机数据解析

```c
/* 电机实例 */
Motor_t motor[4];

/**
 * @brief  解析电机反馈数据
 */
void Motor_Decode(uint32_t id, uint8_t *data)
{
    uint8_t index = id - 0x201;

    if (index > 3) return;

    // 解析数据
    motor[index].angle = (data[0] << 8) | data[1];
    motor[index].speed = (int16_t)((data[2] << 8) | data[3]);
    motor[index].current = (int16_t)((data[4] << 8) | data[5]);
}
```

### 6.3 CAN 发送控制命令

```c
/**
 * @brief  发送电机电流控制命令
 */
void Motor_SendCurrent(void)
{
    CAN_TxHeaderTypeDef txHeader;
    uint8_t txData[8];
    uint32_t txMailbox;

    txHeader.StdId = 0x200;
    txHeader.IDE = CAN_ID_STD;
    txHeader.RTR = CAN_RTR_DATA;
    txHeader.DLC = 8;

    // 填充数据
    for (int i = 0; i < 4; i++)
    {
        txData[i * 2] = (motor[i].set_current >> 8) & 0xFF;
        txData[i * 2 + 1] = motor[i].set_current & 0xFF;
    }

    HAL_CAN_AddTxMessage(&hcan1, &txHeader, txData, &txMailbox);
}
```

### 6.4 CAN 接收中断回调

```c
/**
 * @brief  CAN 接收中断回调
 */
void HAL_CAN_RxFifo0MsgPendingCallback(CAN_HandleTypeDef *hcan)
{
    CAN_RxHeaderTypeDef rxHeader;
    uint8_t rxData[8];

    if (HAL_CAN_GetRxMessage(hcan, CAN_RX_FIFO0, &rxHeader, rxData) == HAL_OK)
    {
        if (rxHeader.StdId >= 0x201 && rxHeader.StdId <= 0x204)
        {
            Motor_Decode(rxHeader.StdId, rxData);
        }
    }
}
```

### 6.5 电机控制循环

```c
/**
 * @brief  电机控制循环（1ms 周期）
 */
void Motor_Control_Loop(void)
{
    for (int i = 0; i < 4; i++)
    {
        // 执行双闭环控制
        Motor_DualLoop_Control(&motor[i]);
    }

    // 发送控制命令
    Motor_SendCurrent();
}
```

## 七、调试技巧

### 7.1 调试流程

1. **CAN 通信验证**
   - 检查 CAN 波特率是否正确
   - 用示波器观察 CAN_H/CAN_L 波形
   - 确认能否接收电机反馈数据

2. **开环测试**
   - 发送固定电流值
   - 确认电机能够转动
   - 检查反馈数据是否正确

3. **速度环调试**
   - 先调试速度环
   - 观察速度响应曲线
   - 调整 PID 参数

4. **位置环调试**
   - 速度环稳定后再调试位置环
   - 观察位置跟踪曲线
   - 调整 PD 参数

### 7.2 常用调试工具

| 工具 | 用途 |
|------|------|
| **示波器** | 观察 CAN 波形 |
| **逻辑分析仪** | 分析 CAN 通信 |
| **串口调试** | 打印调试信息 |
| **Keil 调试** | 在线调试变量 |
