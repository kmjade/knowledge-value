# 網絡服務商內網現狀

## 概述

隨著 IPv4 地址枯竭，各大網絡服務商（ISP）紛紛採用內網（CGNAT）方案，這對用戶的網絡體驗產生了重要影響。

## 發展趨勢

### 早期情況

- **移動**：早期就採用大內網，常被其他用戶嘲笑
- **電信**：早期提供公網 IP，用戶體驗較好

### 現狀（2020 年代後）

> [!quote] 觀察
> "忽然發現電信也是大內網了，以前總是嘲笑移動大內網，現在電信和移動差別好像也不大了？"

電信已經開始大規模採用內網方案：
- 與移動的差異逐漸縮小
- 公網 IP 越来越難申請
- 對 P2P、遠程訪問等應用影響增大

## CGNAT 概念

**Carrier-Grade NAT (CGNAT)** 是網絡服務商為解決 IPv4 地址短缺而採用的技術：

- **工作原理**：在服務商側進行 NAT，多個用戶共享公網 IP
- **影響**：
  - 無法從外網直接訪問家庭網絡
  - P2P 連接困難
  - 端口映射失效
  - 遠程桌面等應用受限

## 用戶影響

### 正面影響

- 安全性提高（外部無法直接訪問）
- 服務商可以節省 IPv4 地址資源

### 負面影響

- P2P 下載速度下降
- 遠程訪問變困難
- 無法搭建對外公開的服務
- 網絡遊戲連接問題

## 解決方案

1. **申請公網 IP**：部分服務商仍提供申請渠道
2. **使用 IPv6**：未來的主要解決方案
3. **端口穿透工具**：如 frp、ngrok 等
4. **更換服務商**：尋找仍提供公網 IP 的 ISP

## 未來展望

IPv6 的普及將逐漸解決這個問題，但目前處於過渡期，用戶需要適應內網環境。
