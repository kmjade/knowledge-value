# 企业AI采购决策框架

> [!quote]
> 企业AI采购不仅是技术选择，更是战略决策，需要综合考量数据主权、供应商依赖、战略自主性等多维度因素。

---

## 核心内容

### 1. 数据主权 (Data Sovereignty)

| 考量因素 | 说明 | 重要性 |
|---------|------|-------|
| 数据存储位置 | 数据存放在哪个国家/地区 | 高 |
| 数据处理方式 | 云端处理 vs 本地部署 | 高 |
| 数据所有权 | 谁拥有数据的最终控制权 | 高 |
| 合规要求 | 满足GDPR等法规要求 | 中 |

### 2. 供应商依赖风险 (Vendor Lock-in Risk)

| 风险类型 | 描述 | 缓解策略 |
|---------|------|---------|
| 技术锁定 | 难以迁移到其他平台 | 采用开放标准 |
| 价格锁定 | 价格上涨无处可逃 | 合同条款约束 |
| 功能锁定 | 过度依赖特定功能 | 多供应商策略 |
| 数据锁定 | 数据难以导出 | 数据可移植性 |

### 3. 战略自主性 (Strategic Autonomy)

- **核心问题**：AI供应商是否会影响企业核心业务决策
- **评估维度**：
  - 产品路线图的影响力
  - 技术更新的依赖程度
  - 退出成本与难度

### 4. 技术整合度 (Integration)

| 整合层面 | 考量点 |
|---------|-------|
| 数据层面 | 与现有数据源的兼容性 |
| 系统层面 | 与ERP/CRM等系统的集成 |
| 用户层面 | 与现有工作流的衔接 |
| 开发层面 | API的丰富程度和稳定性 |

### 5. 成本效益分析

```
总拥有成本 (TCO) =
  直接成本（许可费、使用费）+
  间接成本（培训、集成、维护）+
  风险成本（供应商风险、数据风险）
```
