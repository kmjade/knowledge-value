將 LLM-Wiki 模式（raw/ → wiki/ AI 編譯）深度部署到 DDC 600 Applied Sciences 領域，使選定的子知識庫具備：

## 4. 實施計劃 Implementation Plan

### Phase 1: 619.1 中醫學 LLM-Wiki 部署 (P0)

#### Step 1.1: 恢復 TCM 經典素材

```bash
# 從 git 歷史恢復被刪除的素問章節
git checkout d9ab9fa^ -- "3 Resources/600 Applied Sciences Engineering/619.1 中醫學/.../ram/素问/"

# 預期恢復：~80 篇素問章節 .md 文件
```

#### Step 1.2: 創建 CLAUDE.md Schema

```
619.1 中醫學/CLAUDE.md
├── 子庫定義（中醫學經典知識庫）
├── 核心概念域（陰陽五行、臟腑經絡、辨證論治…）
├── 頁面模板（經典章節 → wiki 概念頁映射）
├── 編譯規則（原文→註解→現代應用）
└── 使用命令（/wiki-compile tcm）
```

#### Step 1.3: 建立 wiki/ 目錄結構

```
619.1 中醫學/wiki/
├── index.md           # 中醫知識索引
├── log.md             # 編譯日誌
├── concepts/          # 概念頁
│   ├── 陰陽五行.md
│   ├── 臟腑學說.md
│   ├── 經絡學說.md
│   └── ...
├── entities/          # 實體頁
│   ├── 黃帝內經.md
│   ├── 張仲景.md
│   └── ...
└── sources/           # 來源頁
    ├── source-素問.md
    └── source-靈樞.md
```

#### Step 1.4: 執行首次編譯

```
輸入：619.1 中醫學/.../ram/ 下的經典章節
處理：AI 提取 → 概念歸納 → 關係建立
輸出：wiki/concepts/*.md + wiki/index.md 更新
```

**預估文件數**: ~20 wiki 頁面（概念 10 + 實體 5 + 來源 3 + 索引/日誌 2）

### Phase 2: 689 電子DIY LLM-Wiki 增強 (P2)

#### Step 2.1: 對現有 PDF 素材建立來源索引

```
689/08-電子DIY/99-資源收集/ram/Book/
→ wiki/sources/source-MT4-programming.md
→ wiki/sources/source-trading-psychology.md
```

#### Step 2.2: 可選：為智能小車生態創建 wiki/

```
689/08-電子DIY/智能小車/wiki/
├── concepts/（機器人控制概念）
├── entities/（UGV01、RDK X3、OriginBot…）
└── index.md
```

**預估文件數**: ~10 wiki 頁面（可選，非必須）

### Phase 3: 基礎設施標準化 (P1)

#### Step 3.1: 600 級 CLAUDE.md 模板

創建可復用的子庫 schema 模板，供所有 600 子庫使用：

```
600 Applied Sciences Engineering/CLAUDE.md  ← 領域級模板
```

內容包括：
- 通用 raw/ → wiki/ 目錄結構
- 應用科學領域的頁面模板（硬體規格、算法參數、實驗步驟）
- 編譯質量標準

#### Step 3.2: 編譯工作流文檔

記錄 `/wiki-compile` 在 600 領域的標準操作流程：

```
1. 掃描 [DDC]/raw/ 中 compiled:false 的文件
2. 按內容類型分類（經典原文 / 技術文檔 / 論文…）
3. 提取概念、實體、關係
4. 生成 wiki/ 頁面
5. 更新 index.md + log.md
```

## 7. 風險與取捨

| 風險 | 緩解 |
|------|------|
| Git 恢復 TCM 素材可能衝突 | 先 `git log` 確認確切 commit hash |
| 中醫概念提取質量依賴 AI 理解 | 先編譯 5 個概念作為樣本，確認質量後批量進行 |
| 編譯產物可能與現有 KB 章節重疊 | wiki/ 側重概念關係網絡，KB 章節側重教程性內容 |
| WSL/NTFS 可能再出現文件同步問題 | 所有操作通過 git 驗證 |
