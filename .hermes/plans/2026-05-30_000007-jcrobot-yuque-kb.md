将语雀空间 **极创机器人JCROBOT**（移动机器人底盘）的公开知识沉淀为 Obsidian KB，与 "富福" 智能小车项目形成技术互补。

## 挑战

语雀为客户端渲染 SPA，无法通过 curl 直接获取目录结构和文档内容。需要：

1. **手动浏览** 获取 7 个公开知识库的名称和结构
2. **导出** 或逐篇获取文档内容
3. **转换为** Obsidian Markdown

## 方案

### 方案 A：手动采集（推荐）

由用户浏览 https://www.yuque.com/jichuangjiqirenjcrobot ，提供：
- 7 个公开知识库的名称和文档数量
- 每个知识库的目录结构（或截图）

然后 AI 批量生成 KB 文件。

**步骤**：
1. 用户在浏览器打开语雀空间，列出 7 个公开知识库的名称
2. AI 根据名称规划 DDC 分类和目录结构
3. 逐知识库：用户提供文档标题列表 → AI 生成章节大纲 → 用户补充关键内容 → AI 完善
4. 整合到 vault

### 方案 B：语雀导出

语雀支持知识库导出（Markdown/ZIP 格式）：
1. 进入每个公开知识库
2. 点击「导出」→ 选择 Markdown 格式
3. 下载 ZIP，解压到 vault 对应位置
4. AI 批量处理：格式化 frontmatter、修复 wikilinks、添加交叉引用

**优点**：完整保留原文内容和图片
**缺点**：需要每个知识库单独操作

### 方案 C：浏览器插件辅助

使用语雀浏览器扩展或第三方工具（如 yuque-tools、yuque2md）批量抓取：
1. 安装对应工具
2. 配置 token
3. 批量下载所有公开文档为 Markdown

## 推荐路径

**方案 B（语雀导出）** — 最保真、最省力。

### 执行步骤

1. **用户操作**：在语雀中依次打开 7 个公开知识库，导出为 Markdown ZIP
2. **AI 处理**：
   - 分析导出内容，确定 DDC 分类
   - 创建对应目录结构
   - 批量格式化（frontmatter、tags、aliases）
   - 建立交叉引用和 MOC
   - 与 "富福" 智能小车项目建立双向链接
3. **提交**

## 风险

- **访问限制**：语雀公开空间可能有 IP 限制或反爬
- **图片链接**：导出文件中图片可能是语雀 CDN 链接，离线不可用——需评估是否下载到本地
- **内容版权**：语雀空间属第三方，KB 仅用于个人学习参考
- **内容质量未知**：302 篇文档，部分可能是草稿或过时内容——需人工筛选
