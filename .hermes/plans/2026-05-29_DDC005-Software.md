## Chapter Content Outline

### 01 — Software Engineering Overview
- What is software engineering? (vs programming vs computer science)
- History: 1968 NATO conference → Agile Manifesto (2001) → DevOps era
- Software crisis & Brooks' "No Silver Bullet"
- Software lifecycle models
- Key metrics: quality, productivity, maintainability

### 02 — Programming Paradigms
- Imperative / Procedural (C, Pascal)
- Object-Oriented (Java, C++, Python)
- Functional (Haskell, Clojure, FP in JS/Python)
- Declarative (SQL, HTML, Prolog)
- Reactive / Event-driven
- Multi-paradigm languages

### 03 — Software Architecture
- Monolithic vs Microservices vs SOA
- Layered / Hexagonal / Clean Architecture
- Event-driven & CQRS
- API design: REST, GraphQL, gRPC
- Cloud-native patterns
- Trade-off analysis (CAP theorem, etc.)

### 04 — Development Methodologies
- Waterfall → Agile transition
- Scrum: roles, artifacts, ceremonies
- Kanban: flow-based
- DevOps: CI/CD, IaC, observability
- Platform engineering
- Team topologies

### 05 — Software Testing
- Testing pyramid: unit → integration → e2e
- TDD (Test-Driven Development)
- BDD (Behavior-Driven Development)
- Test automation frameworks
- Performance & security testing
- Mutation testing & property-based testing

### 06 — Version Control & Collaboration
- Git fundamentals: branches, merge strategies, rebase
- GitHub/GitLab workflows: GitFlow, trunk-based
- Code review best practices
- CI/CD pipelines (GitHub Actions, Jenkins)
- Monorepo vs polyrepo
- Conventional commits

### 07 — Software Design Patterns
- GoF 23 patterns (Creational, Structural, Behavioral)
- SOLID principles
- DRY, KISS, YAGNI
- Dependency Injection
- Domain-Driven Design (DDD) intro
- Anti-patterns to avoid

### 08 — Programming Language Ecosystems
- Python: Django, FastAPI, data science stack
- JavaScript/TypeScript: Node.js, React, Next.js
- Go: concurrency model, cloud-native
- Rust: memory safety, systems programming
- Java/JVM: Spring, Kotlin
- Language selection criteria

### 09 — Open Source & Licensing
- FOSS history & philosophy
- License types: MIT, GPL, Apache, BSD
- Contributor workflows (fork → PR → review)
- Community governance models
- Sustainability & funding models
- Inner source
