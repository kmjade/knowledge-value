## 📖 文档概览

本指南详细介绍了本知识库中的模板系统，帮助您高效使用各种类型的模板来创建和管理知识内容。

### 🗂️ 模板目录结构

```
_templates/
├── _template-project.md      - 项目模板
├── _template-areas.md        - 领域模板
├── _template-resource.md      - 资源模板
├── _template-archive.md      - 归档模板
└── _template-zettel.md       - Zettelkasten 笔记模板
```

### 🔑 核心特性

- **PARA 系统支持**：按照 Projects、Areas、Resources、Archives 分类
- **标准化格式**：统一的 YAML frontmatter 和文档结构
- **自动化功能**：集成 Templater、Tasks 等插件自动化功能
- **快速检索**：完善的标签体系和 wikilink 网络

## 🚀 快速开始

### 使用模板的基本步骤

1. **打开命令面板**：使用快捷键 `Ctrl/Cmd + P`
2. **搜索模板**：输入模板名称（如 `_template-project`）
3. **创建新文件**：选择模板创建新笔记
4. **替换占位符**：将模板中的 `{{title}}` 等占位符替换为实际内容

### 文档组织原则

- **按 PARA 系统分类**：Projects、Areas、Resources、Archives
- **标准化格式**：统一的 YAML frontmatter 和结构
- **快速检索**：通过标签和链接建立知识网络

## 📋 模板详解

### 🎯 Project 模板 (`_template-project.md`)

**适用场景：**
- 有明确目标和交付物的工作项目
- 包含时间期限和里程碑的任务
- 需要跟踪进度和优先级的项目

**核心字段：**

| 字段 | 说明 | 填写建议 |
|------|------|---------|
| `title` | 项目名称 | 简洁明确，体现项目核心 |
| `due` | 截止日期 | 必填，格式 YYYY-MM-DD |
| `priority` | 优先级 | high/medium/low |
| `status` | 项目状态 | planning/active/completed/on-hold |

**任务管理示例：**

```markdown
## 任务清单 / Task List

- [ ] 需求分析 📅 2024-01-15 🏷️ #high
- [ ] 原型设计 📅 2024-01-20 🏷️ #medium
- [ ] 开发实施 📅 2024-02-01 🏷️ #high
- [ ] 测试验收 📅 2024-02-15 🏷️ #medium
```

**Tasks 插件识别规则：**
- `📅 日期` = 任务截止日期
- `🏷️ #tag` = 任务优先级标签
- `- [ ]` = 未完成任务
- `- [x]` = 已完成任务

### 🌱 Area 模板 (`_template-areas.md`)

**适用场景：**
- 长期责任领域（健康、财务、职业等）
- 需要持续维护和关注的生活领域
- 没有明确完成时间但需要定期回顾的事项

**核心字段：**

| 字段 | 说明 | 填写建议 |
|------|------|---------|
| `title` | 领域名称 | 简洁明确，如"健康管理" |
| `status` | 领域状态 | active/inactive/archived |
| `importance` | 重要性评级 | high/medium/low |

**定期回顾机制：**

模板已集成完整的回顾结构：

```markdown
## 定期检查 / Regular Reviews

### 月度回顾
**本月重点：**
- [ ] 关键目标1
- [ ] 关键目标2

**完成情况：**
> 上月目标的完成情况记录

### 季度规划
| 季度 | 关键目标 |
|------|---------|
| Q1 | 春季健康计划 |
| Q2 | 职业技能提升 |
| Q3 | 财务规划执行 |
| Q4 | 年度总结反思 |

### 年度总结
**本年度成就：**
> 列出本年度在该领域取得的主要成就和里程碑

**下一年目标：**
> 为下一年设定的具体目标和行动计划
```

### 🗃️ Archive 模板 (`_template-archive.md`)

**适用场景：**
- 已完成或取消的项目
- 过时或不再需要的文档
- 需要保留但不活跃的内容

**核心字段：**

| 字段 | 说明 | 填写建议 |
|------|------|---------|
| `archived` | 归档日期 | 归档时填写，格式 YYYY-MM-DD |
| `archive_reason` | 归档原因 | completed/cancelled/obsolete/merged/moved |

**归档原因选择：**

| 归档原因 | 说明 | 示例 |
|---------|------|------|
| `completed` | 已完成 | 项目目标达成，正常结束 |
| `cancelled` | 已取消 | 项目被主动取消 |
| `obsolete` | 已过时 | 内容不再适用或有效 |
| `merged` | 已合并 | 内容合并到其他文档中 |
| `moved` | 已移动 | 内容转移到其他位置 |

### 🧠 Zettelkasten 笔记模板 (`_template-zettel.md`)

**适用场景：**
- 知识点和概念的碎片化记录
- 产生新想法的临时笔记
- 建立知识网络的核心单元

**卡片类型分类：**

| 卡片类型 | 用途 | 特点 |
|---------|------|------|
| 概念卡片 | 核心概念定义 | 清晰定义，解释含义 |
| 关系卡片 | 连接不同概念 | 建立知识关联 |
| 方法卡片 | 具体操作步骤 | 实用可执行 |
| 问题卡片 | 需要解决的问题 | 引发思考 |
| 启发卡片 | 灵感和想法 | 创新思维 |

**Zettelkasten 原则：**

> **原子性** - 每张卡片应该包含一个独立的知识点
>
> **可链接性** - 卡片之间通过 wikilink 建立关联
>
> **可检索性** - 使用明确的标题和标签便于搜索

### 🔧 YAML Frontmatter 标准

所有模板都包含标准化前元数据：

```yaml
---
title: 文档标题
aliases:
  - 别名1
  - 别名2
tags:
  - tag1
  - tag2
created: 2024-02-14
updated: 2024-02-14
status: active
---
```

**核心字段说明：**
- `title` - 必填，文档主标题
- `aliases` - 可选，便于搜索的别名
- `tags` - 可选，内容标签分类
- `created` - 自动生成，创建日期
- `updated` - 自动更新，最后修改日期
- `status` - 状态管理字段

### 📅 日期格式标准

| 场景 | 格式 | 示例 |
|------|------|------|
| **截止日期** | `YYYY-MM-DD` | `2024-12-31` |
| **创建时间** | `YYYY-MM-DD HH:mm` | `2024-02-14 14:30` |
| **日期范围** | `YYYY-MM-DD 至 YYYY-MM-DD` | `2024-01-01 至 2024-12-31` |

## ❓ 常见问题解答

### Q: 模板中的 Templater 变量不生效？

**A:** 请检查以下问题：

1. **确保正确安装插件**
   - Obsidian Templater 插件已启用
   - 变量语法正确：`<% tp.date.now("YYYY-MM-DD") %>`

2. **检查语法错误**
   - 拼写错误或格式问题
   - 变量名称与插件文档一致

3. **更新依赖**
   - 确保插件版本兼容
   - 重启 Obsidian 后重试

### Q: Dataview 查询结果为空？

**A:** 常见解决方案：

| 问题类型 | 可能原因 | 解决方案 |
|---------|---------|----------|
| 语法错误 | 查询语法拼写错误 | 检查 Dataview 语法 |
| 字段不存在 | 使用了不存在的字段名 | 检查 frontmatter 中的字段名 |
| 路径问题 | 查询路径有误 | 验证文件路径和目录结构 |
| 插件问题 | Dataview 插件配置问题 | 重启 Obsidian 或重装插件 |

### Q: 如何处理大量归档文件？

**A:** 建议流程：

1. **定期清理** - 每月检查一次过期内容
2. **分类归档** - 按类型或日期分组存储
3. **保留链接** - 维护重要的 wikilink 连接
4. **备份策略** - 重要内容在归档前备份
