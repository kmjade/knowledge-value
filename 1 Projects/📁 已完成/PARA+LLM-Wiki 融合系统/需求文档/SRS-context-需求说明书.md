# /context — 会话状态加载 需求说明书

> 基于 [[../SRS 03-功能需求|SRS §3.4]] + [[../设计文档/Skill 3-context|Skill 3 设计]]

**状态**: ✅ 已实施 | **Phase**: 100% (5/5)

---

## 1. 概述

### 1.1 目的

`/context` 是 Claude Code 会话的感知层，在每次会话开始时加载 Vault 当前状态，提供操作建议。

### 1.2 范围

- 五维状态快照（日记/周记/项目/Inbox/历史）
- 三种子模式（quick/projects/inbox/summary）
- 阈值驱动的操作建议

---

## 2. 功能需求

### FR-030: 快速状态概览

| 属性 | 值 |
|------|-----|
| 优先级 | P0 |
| 模式 | `--quick` (默认) |
| 指标 | Inbox 数量、活跃项目、待编译 raw/、Wiki 页面、最近操作 |

### FR-031: 分模块状态查询

| 模式 | 内容 | 优先级 |
|------|------|:------:|
| `--projects` | 活跃项目 + 状态 + 截止 + 下一步 | P1 |
| `--inbox` | 积压详情 + 文件列表 + 主题猜测 | P1 |
| `--summary` | 优先级排序建议 + 阈值触发 | P1 |

### FR-032: 操作建议生成

| 条件 | 阈值 | 建议 |
|------|:----:|------|
| Inbox 积压 | > 5 | 运行 `/triage` |
| Inbox 积压 | > 20 | 🔴 立即分拣 |
| raw/ 未编译 | > 3 | 运行 `/wiki-compile` |
| 距上次 lint | > 7 天 | 运行 `/lint` |

## 3. 非功能需求

| ID | 需求 | 指标 |
|----|------|------|
| NFR-003 | 快速查询响应 | `--quick` < 30 秒 |

## 4. 接口

| 接口 | 说明 |
|------|------|
| 命令 | `/context [--quick\|--projects\|--inbox\|--summary]` |
| 输入 | Vault 文件系统 |
| 输出 | Markdown 状态摘要 |
