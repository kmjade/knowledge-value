> [!info] Inbox 是什么？

> Inbox 是 PARA 系统的第一站，用于快速捕获想法、收集未整理的信息、临时存放待处理项目，避免思维中断，让创作流保持连贯。

---

## 📊 快速概览

| 当前状态 | 目标 | 操作 |
|---------|------|------|
| 📋 待处理笔记数 | < 20 | [[Inbox Dashboard]] 查看仪表盘 |
| 📈 本周转化率 | > 80% | `/para-整理收集` 优化处理流程 |
| ⏱️ 平均处理时间 | < 2 天 | 定期微整理，保持 Inbox 清空 |
| 🔄 本周待办 | > 10 | 每周定期回顾待办事项 |

### 📋 捕获阶段（30秒原则）

| 场景 | 操作 |
|------|------|
| **会议中的灵感** | 按 `Ctrl+Shift+I` 快速捕获想法 |
| **路上的想法** | 按 `Ctrl+Shift+I` 快速记录 |
| **洗澡时的灵感** | 按 `Ctrl+Shift+I` 语音或文字快速记录 |
| **阅读中的观点** | 按 `Ctrl+Shift+I` 快速记录要点 |

### 📋 待处理阶段（定期整理）

1. **每日微整理**（5-10分钟）：
   - 查看 Inbox Dashboard 待处理数量
   - 如果超过 20 个，优先处理
   - 处理低价值内容（2分钟笔记、片段）

2. **每周大整理**（30-60分钟）：
   - 完整本周所有待处理笔记
   - 更新 Para 分类标签
   - 清理已完成项目

3. **每月归档**（30-60分钟）：
   - 将已解决项目移动到 Archives
   - 更新已归档状态

## 📝 最佳实践

### 1. 快速捕获原则
- **保持简单**：只记录核心内容，30秒内完成
- **保持简洁**：使用结构化格式，便于后续整理
- **不纠结格式**：先记录，以后再优化
- **批量处理**：定期整理，不要堆积

### 2. 定期回顾
- **每日睡前**：清空 Inbox 或检查待办事项
- **每周日**：整理本周笔记，更新 PARA 分类
- **每月月结**：深度整理，归档已完成项目

### 3. 自动化建议
- **启用 Templater 自动触发**：文件创建后自动询问分类
- **使用 Claudian 自动分类**：在处理时自动建议 PARA 分类
- **设置定时提醒**：定期整理提醒
