# 富福 v3 — 参照 HiWonder 悬挂履带底盘的优化方案

> **参考产品**：[HiWonder Suspended Shock Absorbing Tracked Chassis](https://www.hiwonder.com/products/suspended-shock-absorbing-tracked-chassis)

> **基准**：[[富福-v1-方案|v1]] → [[富福-v2-优化方案|v2]]

## HiWonder 底盘核心参数

| 参数 | HiWonder 悬挂履带底盘 | 富福 v1 (UGV01 方案) |
|------|:---|:---|
| **尺寸** | 300 × 230 × 124 mm | ~300 × 200 mm |
| **净重** | ~1.2 kg | ~1.5 kg |
| **最大载重** | **5 kg** | ~3 kg |
| **悬挂** | 8 弹簧碳钢拉簧 + 摆臂 + 微型轴承 | 4 轮独立弹簧悬挂 |
| **主体材料** | 铝合金 (阳极氧化) | 铝合金框架 |
| **履带材料** | 工程塑料 (高弹性) | 橡胶履带 |
| **电机** | DC 减速电机 + 编码器, 6–12V | JGB37-520, 12V |
| **减速比** | 1:7.48 | 1:30 |
| **空载转速** | 895 rpm @12V | 300–500 rpm @12V |
| **电机驱动** | 4 通道编码器电机驱动 (标配) | TB6612FNG |
| **层结构** | **双层可拆卸** (高级版) | 单层 |
| **兼容性** | Jetson Nano/Orin, Pi 4B/5, Arduino | Pi + Arduino |
| **参考价** | ~¥1,200–1,500 (高级版) | ~¥600 (UGV01 底盘) |

> **关键洞察**：HiWonder 底盘的价格是 UGV01 的 2 倍，但载重多 2kg、悬挂更专业、双层结构省去了自制扩展板的麻烦。

## 一、底盘方案重新评估

### 1.1 三种底盘对比

| 维度 | A: UGV01 (v1) | B: HiWonder (v3) | C: 全自组 |
|------|:---:|:---:|:---:|
| **底盘价格** | ¥600 | ¥1,200–1,500 | ¥200–400 |
| **载重** | 3 kg | **5 kg** | 视框架而定 |
| **悬挂性能** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| **双层扩展** | ❌ 需自建 | ✅ 标配 | 自建 |
| **编码器电机** | 另购 | ✅ 标配 (4 个) | 另购 |
| **电机驱动** | 另购 | ✅ 标配 (4 通道) | 另购 |
| **组装时间** | 2h | **0.5h** (预装) | 6h+ |
| **ROS 兼容验证** | 自行调通 | ✅ JetTank 已验证 | 自行调通 |

### 1.3 推荐决策

| 场景 | 推荐方案 | 理由 |
|------|:---:|------|
| **预算严格 <¥2,000** | A: UGV01 | 成本可控，自组有趣 |
| **追求性能 + 省时间** | **B: HiWonder** | 悬挂好、载重大、ROS 验证 |
| **极限低成本** | C: 自组 | 但耗费大量时间 |

> 💡 **推荐 B** — HiWonder 底盘的 ¥600 溢价换来的是：更好的悬挂 (8 弹簧 vs 4 弹簧)、2× 载重、预装电机+编码器+驱动、JetTank 验证过的 ROS 兼容性。

## 二、HiWonder 底盘的结构设计精华

### 2.1 8 弹簧独立悬挂系统

```
                  ┌──────────────────────────┐
                  │     上层扩展板 (可拆卸)      │
                  │   Pi 5 / Jetson / 传感器   │
                  ├──────────────────────────┤
                  │     下层底盘主体 (铝合金)    │
                  │                          │
    ┌─────────────┼──────────────────────────┼─────────────┐
    │             │                          │             │
    ▼             ▼                          ▼             ▼
┌───────┐   ┌───────┐                  ┌───────┐   ┌───────┐
│摆臂 1 │   │摆臂 2 │    ...×4 对...    │摆臂 7 │   │摆臂 8 │
│  ┌──┐ │   │  ┌──┐ │                  │  ┌──┐ │   │  ┌──┐ │
│  │轮│ │   │  │轮│ │                  │  │轮│ │   │  │轮│ │
│  └──┘ │   │  └──┘ │                  │  └──┘ │   │  └──┘ │
│   ○   │   │   ○   │   碳钢拉簧        │   ○   │   │   ○   │
│  / \  │   │  / \  │   (每摆臂 1 根)   │  / \  │   │  / \  │
│ /   \ │   │ /   \ │                  │ /   \ │   │ /   \ │
└───────┘   └───────┘                  └───────┘   └───────┘
   履带                                      履带
```

**核心设计要点**：

| 设计元素 | 描述 | 对富福的启发 |
|---------|------|------------|
| **独立摆臂** | 每对负重轮独立摆动，适应不平地形 | 避免 UGV01 的"整体倾斜"问题 |
| **8 通道碳钢拉簧** | 高弹性，疲劳寿命长 | 优于普通弹簧/橡胶悬挂 |
| **微型轴承** | 摆臂轴心用微型轴承减少摩擦 | 悬挂响应更快，无卡滞 |
| **长行程** | 弹簧拉伸行程 ~20mm | 可通过 4cm 障碍物 |

### 2.2 双层可拆卸架构

```
┌────────────────────────────────────────────┐
│              Layer 2: 扩展层                 │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ Pi 5     │  │ 扩展板    │  │ LiDAR    │  │
│  │          │  │ (CAN/IMU) │  │ (顶部)   │  │
│  └──────────┘  └──────────┘  └──────────┘  │
│  ┌──────────┐  ┌──────────┐                │
│  │ Camera   │  │ Arduino  │                │
│  └──────────┘  └──────────┘                │
├────────────────────────────────────────────┤
│        铜柱连接 (可快速拆卸上层)              │
├────────────────────────────────────────────┤
│              Layer 1: 动力层                 │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│  │ 电池      │  │ 电机驱动  │  │ 降压模块  │  │
│  │ 12V 3S   │  │ 4-CH     │  │ 12V→5V   │  │
│  └──────────┘  └──────────┘  └──────────┘  │
├────────────────────────────────────────────┤
│              Chassis: 底盘                   │
│         铝合金框架 + 悬挂系统 + 履带          │
└────────────────────────────────────────────┘
```

**双层架构的优势**：

| 优势 | 说明 |
|------|------|
| **热隔离** | 电池/电机驱动 (发热) 在下层，计算单元在上层 |
| **EMI 隔离** | 大电流走线在下层，信号线在上层 |
| **快速维护** | 拆卸上层铜柱即可访问下层，无需重新走线 |
| **扩展灵活** | 上层可按需更换布局，不影响动力系统 |

> ✅ **直接采纳**：无论最终选 UGV01 还是 HiWonder，都应采用双层铜柱架构。

### 2.3 标配 4 通道编码器电机驱动

HiWonder 底盘标配的驱动板特性：

| 特性 | 描述 | 对比 TB6612 (v1) |
|------|------|:---:|
| **通道数** | 4 通道 (可驱动 4 个电机) | 2 通道 |
| **编码器接口** | 4 路编码器直连 | 需额外接线 |
| **板载稳压** | 内置 5V/3.3V 稳压输出 | 需外置降压 |
| **通信接口** | UART / I2C (推测) | PWM |
| **电流检测** | 可能内置 | 需外置 INA219 |

> ✅ 如果使用 HiWonder 底盘，电机驱动无需自行解决，且天生支持 4 电机 (可扩展为 4 驱履带)。

## 三、悬挂系统深度优化

### 3.1 借鉴 HiWonder 的弹簧摆臂设计

即使使用 UGV01 底盘，也可以借鉴以下原则进行改装：

| 优化项 | UGV01 现状 | HiWonder 启发 | 可行性 |
|--------|:---|------|:--:|
| **摆臂独立化** | 4 轮整体式 | 每个负重轮独立摆臂 | 🟡 需改框架 |
| **弹簧升级** | 普通弹簧 | 碳钢拉簧 (高疲劳寿命) | 🟢 可更换 |
| **轴承化** | 滑动摩擦 | 微型轴承 (滚动摩擦) | 🟢 可加装 |
| **行程增加** | 短行程 | 长行程 (~20mm) | 🟡 受限于框架 |

### 3.2 弹簧选型指南

```
碳钢拉簧参数 (对标 HiWonder):
  线径: 1.2–1.5 mm
  外径: 8–10 mm
  自由长度: 40–50 mm
  最大拉伸: ~25 mm
  弹簧常数: ~0.5 N/mm
  
安装方式:
  一端挂摆臂 → 一端挂底盘框架
  预拉伸 5mm (预载) → 消除间隙
```

### 3.3 履带优化

| HiWonder 履带特性 | v1 橡胶履带 | 优化建议 |
|:---|:---|------|
| 工程塑料 (高弹性) | 橡胶 | 工程塑料更耐磨，雨天不打滑 |
| 内齿驱动 | 摩擦驱动 | 确保驱动轮齿距匹配履带节距 |
| 宽幅面 (~40mm) | 视型号 | 宽幅面 = 更大接地面积 = 更低地压 |

## 四、JetTank 验证过的 ROS 架构

HiWonder 的 JetTank 机器人已完整验证了以下技术栈在本底盘上的可行性：

### 4.1 已验证的硬件组合

```
JetTank = HiWonder 底盘 + Jetson Nano + SLAMTEC A1 LiDAR + 深度相机
         ↓
富福 v3 = HiWonder 底盘 + Pi 5 + YDLIDAR X4 + USB Camera
```

| 组件 | JetTank | 富福 v3 | 兼容性 |
|------|:---|:---|:---:|
| 主控 | Jetson Nano | **Pi 5** | ✅ 更高性能 |
| LiDAR | SLAMTEC A1 | **YDLIDAR X4** | ✅ 驱动已成熟 |
| ROS | ROS1 Melodic | **ROS2 Humble** | ✅ Pi 5 原生支持 |
| SLAM | gmapping | **slam_toolbox** | ✅ 更现代 |
| 导航 | move_base | **Nav2** | ✅ 升级版 |

### 4.2 JetTank 的 URDF 模型可直接参考

HiWonder 官方提供了 JetTank 的完整 URDF 模型，富福可复用其运动学参数：

```xml
<!-- 参考 JetTank URDF 的关键参数 -->
<link name="chassis">
  <mass value="1.2"/>    <!-- 底盘净重 -->
  <box size="0.30 0.23 0.124"/>  <!-- 底盘尺寸 (m) -->
</link>

<joint name="left_track" type="continuous">
  <!-- 履带差速模型 -->
  <parent link="chassis"/>
  <child link="left_track_wheel"/>
</joint>

<!-- 履带宽度 (两履带中心距) = 约 200mm -->
```

### 4.3 已验证的 Nav2 参数

JetTank 的导航参数可作为富福的起点，避免从零调参：

```yaml
# 参考 JetTank 的 Nav2 配置
controller_server:
  ros__parameters:
    controller_plugins: ["FollowPath"]
    FollowPath:
      plugin: "dwb_core::DWBLocalPlanner"
      # 履带差速的关键约束
      max_vel_x: 0.5          # 最高前进速度
      min_vel_x: -0.3         # 最高后退速度
      max_vel_theta: 1.5      # 最高旋转速度
      acc_lim_x: 1.0          # 加速度限制
      acc_lim_theta: 2.0      # 角加速度限制
      
local_costmap:
  ros__parameters:
    track_unknown_space: false
    robot_radius: 0.18        # 机器人外接圆半径
    inflation_radius: 0.25    # 膨胀半径
```

## 六、推荐实施方案

### 方案 A：预算优先 (¥2,068)

```
底盘: UGV01 (保持 v1) + v2 全部软件优化
适用: 预算严格 <¥2,000，享受自组乐趣
风险: 悬挂较弱、需自行调参 Nav2
```

### 方案 B：性能优先 (¥2,520) ⭐ 推荐

```
底盘: HiWonder 高级版 + v2 全部软件优化
适用: 预算可放宽至 ¥2,500，追求更好的悬挂和 ROS 兼容性
优势: JetTank 已验证的硬件栈、双层扩展、5kg 载重
```

### 方案 C：混合方案 (¥2,200)

```
底盘: UGV01 + 自建双层铜柱 + 8 弹簧改装
软件: v2 全部优化
特色: 用 HiWonder 的设计理念改造 UGV01 底盘
```

## 七、如果选 HiWonder 底盘的实施调整

### 7.1 BOM 变化

| 原 v1 BOM 条目 | v3 变动 |
|---------------|---------|
| 履带底盘 + 电机 ×2 + 驱动 | → **HiWonder 高级版 (一站配齐)** |
| TB6612FNG | → 移除 (HiWonder 标配 4-CH 驱动) |
| 杜邦线/电机线 | → 减少 (底盘预装) |
| 铜柱/螺丝 | → 保留 (用于上层扩展) |

### 7.2 接线简化

```
原 v1:
  Arduino → TB6612 → 左电机
                    → 右电机
          编码器 ×2 → Arduino
          
v3 (HiWonder):
  标配驱动板 → 4 个编码器电机 (预接)
  Arduino ↔ 驱动板 (UART/I2C)
  Pi 5 ↔ Arduino (CAN)
```

### 7.3 组装时间节省

| 工序 | v1 (UGV01) | v3 (HiWonder) | 节省 |
|------|:---:|:---:|:--:|
| 底盘组装 | 2h | 0.5h | -1.5h |
| 电机安装 | 1h | 已预装 | -1h |
| 编码器接线 | 1h | 已预装 | -1h |
| 驱动板焊接 | 0.5h | 标配 | -0.5h |
| **合计** | **4.5h** | **0.5h** | **-4h** |
