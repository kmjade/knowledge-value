## 一、CAN 总线架构

### 1.1 为什么升级到 CAN

| 维度 | UART (v1) | CAN (v2) |
|------|:---:|:---:|
| **接线** | 双工 2 线 (TX+RX) | 差分 2 线 (CAN_H+CAN_L) |
| **抗干扰** | ❌ 差 (TTL 电平) | ✅ 强 (差分信号) |
| **多设备** | ❌ 点对点 | ✅ 多主总线 (挂载 N 个节点) |
| **错误检测** | ❌ 无 | ✅ CRC15 + 自动重发 |
| **优先级** | ❌ 无 | ✅ 仲裁机制 (ID 越小优先级越高) |
| **速率** | 115200 bps | 1 Mbps |
| **RoboMaster 对标** | — | ✅ M3508/S1/EP 标准总线 |

### 1.2 v2 通信拓扑

```
│                   CAN Bus 1 Mbps                         │
│                                                          │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌─────────┐ │
│  │ Pi 5     │  │ Arduino  │  │ Motor    │  │ Battery │ │
│  │ MCP2515  │  │ MCP2515  │  │ Driver   │  │ Monitor │ │
│  │ (CAN-1)  │  │ (CAN-2)  │  │ (CAN-3)  │  │ (CAN-4) │ │
│  └──────────┘  └──────────┘  └──────────┘  └─────────┘ │
│       ▲             ▲              ▲              ▲      │
│       │             │              │              │      │

└─────────────────────────────────────────────────────────┘
                                │
                    保留 UART 作为调试/备用通道
```

### 1.3 CAN 帧协议设计

```c
// 参考 RoboMaster CAN 协议格式 (11-bit ID)
#define CAN_ID_MOTOR_CMD    0x200   // 电机控制指令 (类似 M3508)
#define CAN_ID_MOTOR_FB     0x201   // 电机反馈数据
#define CAN_ID_IMU_DATA     0x300   // IMU 姿态数据
#define CAN_ID_BATTERY      0x301   // 电池状态
#define CAN_ID_EMERGENCY    0x001   // 紧急停止 (最高优先级)

// 电机控制帧 (8 bytes)
// Byte 0-1: 左电机目标速度 (int16, rpm)
// Byte 2-3: 右电机目标速度 (int16, rpm)
// Byte 4-5: 左电机目标电流 (int16, mA)  ← 新增 (电流环)
// Byte 6:   控制模式 (0=速度, 1=位置, 2=电流)
// Byte 7:   CRC8

// 电机反馈帧 (8 bytes)
// Byte 0-1: 左电机实际速度
// Byte 2-3: 右电机实际速度
// Byte 4-5: 左编码器累计
// Byte 6-7: 右编码器累计
```

## 二、串级 PID + 前馈控制

### 2.1 RoboMaster 三环结构

```
RoboMaster M3508 三环控制：
  位置环(P) → 速度环(PI) → 电流环(PI) → 电机
      ↑            ↑            ↑
   编码器       编码器微分    电流采样
  
富福 v2 双环映射（DC 电机无电流传感器，用估算）：
  位置环(P) → 速度环(PI) → 电压前馈 → 电机
      ↑            ↑
   编码器       编码器微分
```

### 2.2 串级 PID 实现

```cpp
// Arduino 下位机：串级 PID 控制
struct CascadePID {
    // 外环：位置 → 目标速度
    PID pos_pid;    // Kp=2.0, Ki=0.0, Kd=0.1  (纯 P + 微 D)
    
    // 内环：速度 → PWM 输出
    PID vel_pid;    // Kp=1.5, Ki=0.5, Kd=0.05 (PI + 微 D)
    
    // 前馈项
    float ff_accel;  // 加速度前馈系数
    float ff_friction; // 静摩擦补偿
};

float cascadeUpdate(CascadePID& ctrl, float target_pos, float current_pos, 
                    float current_vel, float dt) {
    // 1. 外环：位置误差 → 期望速度
    float pos_error = target_pos - current_pos;
    float target_vel = pidUpdate(ctrl.pos_pid, pos_error, dt);
    
    // 2. 速度限制
    target_vel = constrain(target_vel, -MAX_VEL, MAX_VEL);
    
    // 3. 内环：速度误差 → PWM
    float vel_error = target_vel - current_vel;
    float output = pidUpdate(ctrl.vel_pid, vel_error, dt);
    
    // 4. 前馈补偿
    output += ctrl.ff_accel * target_vel;           // 加速度前馈
    output += ctrl.ff_friction * sign(target_vel);  // 摩擦补偿
    
    return output;
}
```

### 2.3 前馈项详解

| 前馈项 | 物理意义 | 作用 | 标定方法 |
|--------|---------|------|---------|
| **加速度前馈** | 质量 × 加速度 → 力 | 减少跟踪延迟 | 给定斜坡速度，测量跟踪误差 |
| **摩擦补偿** | 静摩擦力 | 消除启动死区 | 逐步增加 PWM 直到电机开始转动 |
| **电压前馈** | 反电动势补偿 | 高速下维持力矩 | `ff = Kv * target_vel` (Kv 由空载转速推算) |

### 2.4 PID 调参策略 (借鉴 RoboMaster)

```
1. 内环优先：先调速度环 (位置环输出 = 0，手动设目标速度)
2. 由内向外：速度环稳定后，再接入位置环
3. 阶跃测试：给 500 → 1000 rpm 阶跃，观察：
   - 上升时间 < 100ms
   - 超调 < 15%
   - 稳态误差 < 2%
4. 加入前馈：观察跟踪斜坡信号 (三角波) 的延迟
```

**参考初始值** (JGB37-520 电机, 11PPR, 1:30 减速比):

| 控制器 | Kp | Ki | Kd | 积分限幅 |
|--------|:--:|:--:|:--:|:--:|
| 位置环 | 2.0 | 0.0 | 0.1 | — |
| 速度环 | 1.8 | 0.6 | 0.05 | ±150 |
| 加速度前馈 | 0.02 | — | — | — |
| 摩擦补偿 | 25 (PWM) | — | — | — |

## 三、IMU 升级

### 3.1 MPU6050 vs BNO055

| 参数 | MPU6050 (v1) | BNO055 (v2) |
|------|:---:|:---:|
| **自由度** | 6-DOF (Acc+Gyro) | 9-DOF (Acc+Gyro+Mag) |
| **输出** | 原始数据 | 融合后的欧拉角/四元数 |
| **姿态精度** | 依赖软件融合 (漂移大) | 硬件融合 (±1° 动态) |
| **CPU 占用** | 需要 Madgwick/Mahony 滤波 | 芯片内置 Cortex-M0 融合 |
| **采样率** | 最高 200Hz | 最高 100Hz (融合模式) |
| **价格差** | ¥10 | +¥30 |
| **RoboMaster 对标** | ❌ | ✅ 对标板载 IMU 融合 |

### 3.2 安装要求

```
       ┌──────────────────┐
       │  BNO055 模块      │
       │  ┌──────────────┐ │
       │  │ 减震橡胶垫    │ │  ← 隔离电机振动
       │  │ (3M 胶固定)   │ │
       │  └──────────────┘ │
       │  重心位置 (底盘中心) │
       └──────────────────┘
```

> **关键**：IMU 安装在底盘 **几何中心 + 重心位置**，远离电机。RoboMaster EP 的 IMU 同样安装在机身中心，避免角速度测量偏差。

## 四、状态机架构

### 4.1 借鉴 RoboMaster 裁判系统

RoboMaster 竞赛机器人通过裁判系统强制状态切换（待命→战斗→阵亡→复活）。富福 v2 引入类似状态机：

```
                    ┌──────────┐
          上电 ────►│  SAFE    │◄──────────────┐
                    │ 安全模式  │                │
                    │ 电机锁止  │  急停触发      │
                    └────┬─────┘                │
         手动遥控       │  自主导航             │
                    ┌───┴───┐                  │
                    ▼       ▼                  │
              ┌────────┐ ┌────────┐     ┌──────┴─────┐
              │ MANUAL │ │  AUTO  │────►│  E-STOP    │
              │ 遥控   │ │ 自主   │ 碰撞│  紧急停止   │
              └───┬────┘ └───┬────┘     └──────┬─────┘
                  │          │                 │
        低电量    │          │ 低电量           │ 复位
                  ▼          ▼                 ▼
              ┌────────────────┐         ┌──────────┐
              │    CHARGE      │         │  SAFE    │
              │  回充/低功耗    │────────►│  (恢复)  │
              └────────────────┘         └──────────┘
```

### 4.2 状态定义

| 状态 | 触发条件 | 行为 | 退出条件 |
|------|---------|------|---------|
| **SAFE** | 上电/复位 | 电机锁止，传感器工作 | 收到启动指令 |
| **MANUAL** | 手柄/键盘控制 | 遥控行走 | 切换模式/急停/低电量 |
| **AUTO** | Nav2 导航激活 | 自主规划行走 | 到达/碰撞/切换/低电量 |
| **E-STOP** | 急停按钮 | 立即切断电机电源 | 急停释放 + 确认 |
| **CHARGE** | 电池 < 10.8V (3S) | 回程/待机，仅保留通信 | 充电完成/手动恢复 |

### 4.3 ROS2 状态机实现

```python
# fufu_state_machine.py — 使用 smach 或自实现
class FufuStateMachine(Node):
    STATES = ['SAFE', 'MANUAL', 'AUTO', 'ESTOP', 'CHARGE']
    
    def __init__(self):
        self.state = 'SAFE'
        self.create_subscription(BatteryMsg, '/battery', self.battery_cb, 10)
        self.create_subscription(EmergencyMsg, '/estop', self.estop_cb, 10)
        
    def transition(self, new_state):
        if new_state not in self.STATES:
            return False
        
        # 退出当前状态行为
        self.exit_state(self.state)
        
        # 进入新状态行为
        self.enter_state(new_state)
        self.state = new_state
        
        # 发布状态变更
        self.publish_state()
        return True
    
    def enter_state(self, state):
        if state == 'ESTOP':
            self.send_can(CAN_ID_EMERGENCY, [0x01])  # 最高优先级 CAN 帧
            self.motor_disable()
        elif state == 'CHARGE':
            self.send_can(CAN_ID_MOTOR_CMD, [0,0,0,0,0,0,0])  # 停止
            self.set_max_velocity(0.3)  # 限速回程
```

## 十一、关键代码骨架

### 11.1 CAN 总线 Arduino 端

```cpp
// fufu_can.h — 基于 mcp_can 库
#include <mcp_can.h>
#include <SPI.h>

#define CAN_CS_PIN 53  // Arduino Mega
MCP_CAN CAN(CAN_CS_PIN);

struct MotorCmd {
    int16_t left_vel;    // rpm
    int16_t right_vel;
    uint8_t mode;        // 0=速度 1=位置
} __packed;

struct MotorFb {
    int16_t left_vel;
    int16_t right_vel;
    int32_t left_enc;    // 累计脉冲
    int32_t right_enc;
} __packed;

void canSetup() {
    CAN.begin(MCP_ANY, CAN_1000KBPS, MCP_8MHZ);
    CAN.setMode(MCP_NORMAL);
}

void sendMotorCmd(const MotorCmd& cmd) {
    uint8_t buf[8];
    memcpy(buf, &cmd, sizeof(MotorCmd));
    CAN.sendMsgBuf(0x200, 0, 8, buf);  // CAN_ID_MOTOR_CMD
}

bool recvMotorFb(MotorFb& fb) {
    if (CAN.checkReceive() == CAN_MSGAVAIL) {
        uint8_t buf[8]; uint32_t id;
        CAN.readMsgBuf(&id, &buf);
        if (id == 0x201) {
            memcpy(&fb, buf, sizeof(MotorFb));
            return true;
        }
    }
    return false;
}
```
