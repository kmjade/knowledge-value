# `/daily-open` 使用指南

> 每日開啟儀式 — 你的晨間 AI 管家

---

## 快速上手

每天第一次打開 Vault 時，在 Claude Code 中輸入：

```bash
/daily-open
```

AI 會自動：檢查/創建今日日記 → 填充活躍任務 → 顯示 Inbox 狀態 → 回顧昨日。

**建議**：每天早上的第一件事，30 秒完成上下文加載。

---

## 命令參數

| 命令 | 用途 | 適合場景 |
|------|------|----------|
| `/daily-open` | 檢查/創建今日日記並填充上下文 | 每日例行 |
| `/daily-open --dry-run` | 僅預覽，不創建文件 | 首次使用前測試 |

---

## 五步驟詳解

### Step 1：檢查今日日記是否存在 🔍

路徑：`Periodic/daily/YYYY/YYYY-MM-DD.md`

- **已存在** → 跳到 Step 5，輸出摘要
- **不存在** → 繼續創建

> 永遠不會覆蓋已存在的日記，放心反覆運行。

### Step 2：確保目錄存在 📁

自動檢查並創建 `Periodic/daily/YYYY/` 目錄。

### Step 3：從模板創建日記 📝

創建結構化日記，包含 8 個區塊：

```
🌅 今日意圖     — 你今天最重要的一件事（手動填寫）
📋 任務         — AI 自動從活躍專案填充
📅 日程         — 日曆事件（需配置日曆集成）
📥 Inbox        — 待分揀文件數量
📝 工作日誌     — 自由記錄今天的工作
💡 Fleeting Ideas — /triage 自動追加的閃念
🤖 Agent 操作記錄 — Claude Code 操作日誌
🌙 日終回顧     — 睡前回顧（手動填寫）
```

### Step 4：自動填充上下文 🤖

#### 📋 任務區塊

掃描 `1 Projects/` 下所有活躍專案，提取你的任務，分類顯示：

```markdown
## 📋 任務

### 今日截止
- [ ] 提交報告 — 來自 [[專案A]]

### 已過期
- [ ] 回覆郵件 — 來自 [[專案B]]（截止 2026-05-30）

### 無截止日期
- [ ] 整理筆記 — 來自 [[專案C]]
```

若無活躍任務，顯示：`> 今日無活躍任務 🎉`

#### 📥 Inbox 區塊

統計 `0 Inbox/` 待處理文件，列出前 5 個：

```markdown
## 📥 Inbox

Inbox 待處理：**3** 個文件

| 文件 | 日期 |
|------|------|
| LLM原理詳解 | 2026-05-30 |
| 讀書筆記-深度工作 | 2026-05-29 |
| 會議記錄-週會 | 2026-05-28 |

> 使用 `/triage` 進行分揀
```

若 Inbox 為空，顯示：`> Inbox 為空 ✨`

#### 🔄 昨日回顧

讀取昨日日記的 `## 🌙 日終回顧`，自動滾動未完成任務：

```markdown
## 🔄 昨日回顧

> 來自 [[2026-05-30]]

- 完成：3 個任務
- 未完成（已滾動到今日）：1 個
```

### Step 5：輸出摘要 ✅

```markdown
✅ 今日日記已就緒：Periodic/daily/2026/2026-05-31.md

📋 活躍任務：5 個（今日截止 1，已過期 2）
📥 Inbox 待處理：3 個文件
🔄 昨日回顧：有
```

## 典型場景

### 場景一：每天早上

```bash
/daily-open
```

→ 5 秒鐘，日記就緒，任務一目了然。

### 場景二：中午才打開 Vault

```bash
/daily-open
```

→ 日記已存在（可能由日曆插件自動創建），跳到摘要輸出，不覆蓋任何內容。

### 場景三：想看看今天有什麼但不寫

```bash
/daily-open --dry-run
```

→ 預覽今天會創建什麼、有哪些任務，不產生任何文件。

### 場景四：補昨天的日記

手動創建 `Periodic/daily/2026/2026-05-30.md`，然後今天運行 `/daily-open` → 自動讀取昨日回顧並滾動任務。

## FAQ

### Q: 重複運行會覆蓋日記嗎？
**不會。** 如果日記已存在，只輸出摘要，不修改任何內容。

### Q: 可以自訂模板嗎？
模板來自 `_Meta/templates/Periodic/daily-template.md`。如需修改模板結構，編輯該文件即可。

### Q: 為什麼任務區塊是空的？
可能原因：
1. 沒有 `status: active` 的專案
2. 專案中沒有 tasks 相關文件
3. 所有任務都已完成

### Q: 昨日回顧為什麼沒有顯示？
因為昨日日記不存在，或昨日日記中沒有填寫 `## 🌙 日終回顧` 區塊。`/daily-open` 只能讀取你手動填寫的內容。

### Q: 日記會自動歸檔嗎？
日記的 `lifecycle: ephemeral`。在週日運行 `/weekly-review` 時，>14 天的 ephemeral 內容會被移入歸檔。但日記本身（`Periodic/daily/`）是排除在歸檔之外的。

### Q: Inbox 計數為 0 但明明有文件？
`/daily-open` 只統計 `0 Inbox/` 根目錄下的 `.md` 文件。子目錄中的文件不計入。
