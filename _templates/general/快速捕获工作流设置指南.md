# 快速捕获工作流设置指南

> [!info] 本指南帮助你设置完整的快速捕获系统，实现"一键想法到目标位置"

---

## 一、工作流概览

```
瞬间想法/灵感
      ↓ (Ctrl+Shift+U, 30秒)
         ↓
  0 Inbox 临时存储
         ↓
    定期回顾 (Inbox Dashboard)
         ↓
    PARA 分类 (Claudian 自动判断)
         ↓
    目标位置移动 (脚本)
```

---

## 二、QuickAdd 配置

### 2.1 添加快捷键选择

打开 QuickAdd → Manage Choices → Add new choice:

**Choice 配置**：
```json
{
  "id": "ultra-capture",
  "name": "🚀 超级快速捕获",
  "type": "Template",
  "command": false,
  "templatePath": "_templates/general/超快速捕获.md",
  "fileNameFormat": {
    "enabled": true,
    "format": "{{DATE:YYYY-MM-DD}}_{{VALUE}}"
  },
  "folder": {
    "enabled": true,
    "folder": "0 Inbox"
  }
}
```

### 2.2 文件名格式配置

**文件名格式**：`YYYY-MM-DD_{{VALUE}}.md`
- **示例**：`2026-01-28_产品灵感.md`

## 三、Templater 自动分类配置

### 3.1 创建 PARA 判断脚本

在 Templater 设置中添加以下脚本：

**脚本名称**：`PARA自动分类`

```javascript
<%*
// Quick capture PARA classification
const questions = [
  "这个想法需要行动吗？(是/否)",
  "有截止日期吗？(是/否)",
  "这是参考资料吗？(是/否)"
  "这是纯想法吗？(是/否)"
];

let choice = "";

// 决策树逻辑
function determinePARA(answers) {
  // 是否需要行动？
  const needsAction = await tp.system.suggester(["是", "否"], "这个想法需要行动吗？(是/否)??");

  // 有截止日期？
  const hasDeadline = await tp.system.suggester(["是", "否"], "有截止日期吗？(是/否)??");
  // 是参考资料？
  const isReference = await tp.system.suggester(["是", "否"], "这是参考资料吗？(是/否)??");
  // 是纯想法？
  const isPureIdea = await tp.system.suggester(["是", "否"], "这是纯想法吗？(是/否)??");

  // 是否有参考价值？
  const hasValue = await tp.system.suggester(["是", "否"], "这个想法有参考价值吗？(是/否)??");

  // 是否过时？
  const isOutdated = await tp.system.suggester(["是", "否"], "这个想法已经过时了吗？(是/否)??");

  // 简单选决策
  if (needsAction && hasDeadline) {
    choice = "project"; // 有截止日期 + 需要行动 → 项目
  } else if (hasDeadline && !needsAction) {
    choice = "project"; // 有截止日期 + 不需要行动 → 项目（可能是信息收集）
  } else if (isReference) {
    choice = "resources"; // 参考资料 → 资源
  } else if (isPureIdea) {
    choice = "areas"; // 纯想法 → 领域（可能需要沉思）
  } else {
    choice = "areas"; // 有价值但不需要立即行动 → 领域
  }
  }

  return `para: ${choice}`;
}
%>
```

### 3.2 添加文件夹移动脚本

在 Templater 添加以下脚本：

**脚本名称**：`移动到PARA目标`

```javascript
<%*
// Move to PARA destination based on choice
const paraChoice = tp.frontmatter.para;

const destinations = {
  "projects": "1 Projects",
  "areas": "2 Areas",
  "resources": "3 Resources",
  "archives": "4 Archives"
};

// 等待 Templater 脚本执行
const waitForTemplater = 1000; // 毫秒检查一次

setTimeout(async () => {
  while (true) {
    await new Promise(r => setTimeout(r, waitForTemplater));
    // 检查 Templater 是否已准备好
    if (tp.file.getFrontmatter()) {
      // 如果 frontmatter 中没有 para，使用脚本移动
      const paraValue = tp.frontmatter.para || paraChoice;
      const destFolder = destinations[paraValue];
      if (destFolder) {
        const destPath = await tp.app.vault.adapter.getAbstractFileByPath(destFolder);
        if (destPath) {
          await tp.app.workspace.activeLeaf.rename(
            tp.file.path,
            `${destFolder}/${tp.file.basename}`
          );
          tp.notice.success(`已移动到: ${destFolder}`);
          break;
        }
      }
    }
  }
  }
}, 5000); // 5秒后超时
})();

%>
```

## 四、键盘快捷键设置

### 4.1 Obsidian 快捷键配置

在 Obsidian 设置中添加以下热键：

| 功能 | 快捷键 | 说明 |
|------|--------|------|
| 超级快速捕获 | `Ctrl+Shift+U` | 触发超快速模板 |
| 打开 Inbox Dashboard | `Ctrl+Shift+D` | 查看待处理笔记 |
| 创建想法卡 | `Ctrl+Shift+I` | 使用问题卡模板 |
| 创建资源收集 | `Ctrl+Shift+R` | 使用资源收集模板 |
| 打开日记 | `Ctrl+Shift+J` | 快速创建今日日记 |

### 4.2 创建快捷键

1. 进入 `Settings → Hotkeys`
2. 点击 `Create hotkey`
3. 命令输入：`Ctrl+Shift+U`
4. 选择类型：`Template` → `Macro`
5. 选择模板：`🚀 超快速捕获`
6. 点击 `Create`

## 五、使用流程

### 5.1 瞬间捕获（<30秒）

1. **按下** `Ctrl+Shift+U`
2. **输入标题** → 瞬间想法直接输入标题
3. **30秒后自动保存** → 文件自动保存到 `0 Inbox/`

**结果**：快速笔记创建完成，等待后续处理

### 5.2 事后处理

1. **打开** `Ctrl+Shift+D` 打开 Inbox Dashboard
2. **点击"PARA 自动分类" → 或手动分类
3. **按规则移动** 到目标文件夹

---

## 六、故障排除

| 问题 | 解决方案 |
|------|----------|
| 脚本未触发 | 检查 Templater 插件是否启用，脚本名称是否正确 |
| 文件未移动 | 确认 PARA 分类值正确，目标文件夹存在 |
| 快捷键不生效 | 检查 Obsidian 设置中快捷键是否配置 |

---

## 七、进阶技巧

- **批量处理**：每天集中时间处理 Inbox Dashboard 中的待分类笔记
- **标签系统**：利用 `tags` 属性进行筛选，如 `tags:#idea` `tags:#problem`
- **搜索优化**：为常用模板设置别名的 aliases
