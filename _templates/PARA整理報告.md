## 五、PARA 核心原則復習

### 5.1 核心原則

| 原則 | 說明 | 行動建議 |
|------|------|----------|
| **1. 按行價值組織** | 是否需要行動？ | 是 → Projects/Task<br>否 → Areas/Resource<br>否 → Archives/Delete |
| **2. 按目標和期限** | 是否有明確截止日期？ | 是 → 設優先排序<br>否 → 長期跟蹤 |
| **3. 是否有參考價值** | 是否是實用的參考資料？ | 是 → Resources<br>否 → Archives |
| **4. 領域持續責** | 是否需要長期維護？ | 是 → 定期審查<br>否 → 整理歸檔 |

### 5.2 決策樹

```
開始處理
         ↓
    是否需要下一步行動？
         ├─ 是 → 判斷決行動：
         │       ├─ 有目標？→ 是 → 🎯 創建 Project/Task
         │       ├─ 有截止日期？ 是 → 設優先排序
         │       └─ 有參考價值？ 是 → 移入 Resources
         └────────────────────→ 否 → Areas/長期跟蹤
         └───────────────────── 否 → Archives 或刪除
```

**文件命名規範**：
- 英文名稱 + 擬文名稱
- 使用連字符分隔單詞
- 不使用特殊字符
- 保持簡潔

- 英文主標籤 + 中文名稱
- 使用連字符分隔單詞
- 使用斜槓分層（最多3-4層）

```yaml
---
title: 文件名稱
aliases:

para: resource|project|area|archive
tags:
  - #para/resource/tech
  - #topic/networking

  - #zettel/type/permanent
  - #learning/[status]
  - #difficulty/[level]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```
