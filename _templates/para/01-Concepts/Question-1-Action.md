# 任務目標
決定新資訊是否需要行動，判斷是否屬於 Projects 類別

## 問題：需要行動嗎？

### 判斷標準

問自己：**「這個資訊需要我採取具體行動嗎？」**

#### ✅ 是，需要行動 → Projects

**Projects 特徵**：
- 有明確的完成目標
- 需要連續執行
- 有截止日期
- 產出具體成果
- 有具體的任務清單

#### ❌ 否 → 繼續下一個問題

---

## Projects 識別檢查清單

### 核心檢查項目
- [ ] 有明確的完成目標嗎？
- [ ] 需要連續執行多個任務嗎？
- [ ] 有截止日期或時間限制嗎？
- [ ] 會產出具體成果或結果嗎？
- [ ] 完成後就結束嗎？

### 如果以上都是「是」→ Projects

## Projects 範例

### 工作相關
| 內容 | 說明 |
|------|------|
| 完成季度銷售報告 | 有目標（完成報告）、有期限（季度末）、需連續執行 |
| 網站改版專案 | 有目標（網站改版）、有期限、有具體任務 |
| 產品發布準備 | 有目標（產品發布）、有期限、需連續行動 |

### 學習相關
| 內容 | 說明 |
|------|------|
| 通過專業認證考試 | 有目標（通過考試）、有期限（考試日）、需準備 |
| 讀完 20 本書 | 有目標（讀完20本）、有期限、需連續閱讀 |
| 學習 Python 程式設計 | 有目標（掌握Python）、有計劃、需連續學習 |

### 生活相關
| 內容 | 說明 |
|------|------|
| 規劃年度家庭旅行 | 有目標（旅行）、有期限、需連續規劃 |
| 整理居住環境 | 有目標（整理完畢）、有期限、需連續行動 |
| 更新個人履歷 | 有目標（完成履歷）、有期限、需撰寫和更新 |

## 不是 Projects 的範例

### 區別 1：Areas（領域）
| 不是 Projects | 是 Areas | 區別說明 |
|--------------|----------|----------|
| 運動 | 運動習慣維持 | Projects 有完成目標，Areas 沒有 |
| 學習 | 職業技能提升 | Projects 有期限，Areas 是持續的 |
| 健康管理 | 健康狀態維持 | Projects 有具體成果，Areas 是維持標準 |

### 區別 2：Resources（資源）
| 不是 Projects | 是 Resources | 區別說明 |
|--------------|-------------|----------|
| 運動教學影片 | 收集運動技巧 | Projects 需要行動，Resources 是收集 |
| 學習資料 | Python 學習資料 | Projects 有目標，Resources 純收集 |
| 健康資訊 | 健康知識筆記 | Projects 產生成果，Resources 是參考 |

### 區別 3：一般任務
| 內容 | 說明 |
|------|------|
| 買牛奶 | 這是單次任務，不是項目 |
| 回覆郵件 | 這是例行工作，不是項目 |
| 預約牙醫 | 這是單次行動，不是項目 |

**說明**：單次任務不需要建立 Projects，可以直接處理或加入每日任務清單。只有需要連續執行多個任務才能完成的內容才建立 Projects。

## 決策流程

### 收到新資訊時

```
收到資訊
    ↓
問自己：需要行動嗎？
    ↓
    ├─ 是 → 有明確目標和期限？
    │         ├─ 是 → 🎯 Projects
    │         └─ 否 → 可能是 Areas 或 Resources
    │
    └─ 否 → 繼續下一個問題（影響生活領域嗎？）
```

## 練習題目

### 練習 1
**內容**：準備下週的簡報

**分析**：
1. 需要行動嗎？ ✅ 是
2. 有明確目標嗎？ ✅ 是（完成簡報）
3. 有期限嗎？ ✅ 是（下週）
4. 需要連續執行嗎？ ✅ 是（準備、製作、演練）

**答案**：**Projects**

**分類**：Projects/工作/下週簡報準備

---

### 練習 2
**內容**：想學習攝影技巧

**分析**：
1. 需要行動嗎？ ❌ 否（沒有明確目標和期限）
2. 這屬於什麼？ → 可能是 Areas 或 Resources

**答案**：**不是 Projects**

**後續判斷**：
- 想維持攝影習慣？ → Areas/興趣/攝影
- 只是收集資料？ → Resources/興趣/攝影技巧

---

### 練習 3
**內容**：每月 15 號繳交房租

**分析**：
1. 需要行動嗎？ ✅ 是
2. 有明確目標嗎？ ✅ 是（繳交房租）
3. 有期限嗎？ ✅ 是（每月15號）
4. 需要連續執行嗎？ ❌ 否（單次行動）

**答案**：**不是 Projects**

**說明**：這是例行工作，不是需要連續執行多個任務的項目。可以加入每月例行任務清單，或視為 Areas（財務管理）的一部分。

---

### 練習 4
**內容**：開發一個 iOS App

**分析**：
1. 需要行動嗎？ ✅ 是
2. 有明確目標嗎？ ✅ 是（開發App）
3. 有期限嗎？ ✅ 是（可設定目標期限）
4. 需要連續執行嗎？ ✅ 是（設計、開發、測試、上架）

**答案**：**Projects**

**分類**：Projects/工作/iOS App開發

---

### 練習 5
**內容**：收集 Python 學習資源

**分析**：
1. 需要行動嗎？ ❌ 否（純粹收集，沒有明確目標和期限）
2. 這屬於什麼？ → Resources

**答案**：**不是 Projects**

**分類**：Resources/學習/Python學習資源

## 常見陷阱

### ❌ 陷阱 1：所有任務都建立 Projects
**問題**：把買牛奶、回覆郵件等單次任務都當成 Projects

**解決**：
- 單次任務直接處理或加入每日任務清單
- 只有需要連續執行多個任務的內容才建立 Projects

### ❌ 陷阱 2：沒有明確期限也建立 Projects
**問題**：把「想學習 Python」當成 Projects

**解決**：
- 沒有明確期限的內容應視為 Areas（學習領域）或 Resources（學習資源）
- 為「想學習 Python」設定具體目標（如：3個月內完成Python基礎課程）才建立 Projects

### ❌ 陷阱 3：混淆例行工作與 Projects
**問題**：把每月繳交房租、每週運動等例行工作當成 Projects

**解決**：
- 例行工作是持續的，沒有「完成」狀態
- 應歸類到 Areas（如：財務管理、健康管理）
- 可以在 Areas 中設定例行任務清單
