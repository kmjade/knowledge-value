# 任務目標
決定資訊是否值得收集和學習，判斷是否屬於 Resources 類別

## 問題：有興趣學習嗎？

### 判斷標準

問自己：**「這個資訊是我有興趣收集和學習的內容嗎？」**

#### ✅ 是，純粹收集資料 → Resources

**Resources 特徵**：
- 沒有完成壓力
- 純粹收集資訊
- 可自由探索
- 未來可能使用
- 提供靈感和學習機會

#### ❌ 否 → **刪除**

---

## 不是 Resources 的範例

### 區別 1：Projects（項目）
| 不是 Resources | 是 Projects | 區別說明 |
|--------------|-------------|----------|
| Python 資料 | 開發 Python 專案 | Resources 純收集，Projects 需行動 |
| 攝影技巧 | 學習攝影課程 | Resources 無壓力，Projects 有目標 |
| 健康資訊 | 執行健康計劃 | Resources 是參考，Projects 產生成果 |

### 區別 2：Areas（領域）
| 不是 Resources | 是 Areas | 區別說明 |
|--------------|-----------|----------|
| 運動技巧 | 運動習慣維持 | Resources 純收集，Areas 需持續投入 |
| 攝影作品 | 攝影興趣維持 | Resources 是參考，Areas 是生活領域 |
| 財務知識 | 財務管理 | Resources 是知識，Areas 是管理行為 |

### 區別 3：無用資訊（應刪除）
| 內容 | 說明 |
|------|------|
| 過期的優惠券 | 已無用處，應刪除 |
| 不相關的資訊 | 與生活工作無關，應刪除 |
| 重複的內容 | 重複收集，應刪除重複 |

## Resources 管理原則

### 1. 按需收集

**只收真正有興趣的：**
- 問自己：我會真的看這個嗎？
- 問自己：這對我有什麼價值？
- 避免收藏「可能有用」的內容

### 2. 定期整理

**避免資訊過載：**
- 每月檢視 Resources
- 去除重複內容
- 移除過時資料
- 更新連結和標籤

### 3. 建立系統

**快速找到所需資訊：**
- 清晰的資料夾結構
- 使用標籤分類
- 建立索引文件
- 有效的搜尋機制

### 4. 主動使用

**避免資源堆積塵埃：**
- 定期閱讀收集的資料
- 嘗試應用學到的知識
- 做筆記和總結
- 分享給他人

## 常見陷阱

### ❌ 陷阱 1：收集「可能有用」的內容
**問題**：看到任何資訊都覺得「可能有用」，結果堆積大量資料

**解決**：
- 問自己：我會真的看這個嗎？
- 問自己：這對我現在或未來有什麼具體價值？
- 避免「先收集後再想」的心理

### ❌ 陷阱 2：永不使用的資料庫
**問題**：收集大量資料但從不看和使用

**解決**：
- 定期閱讀收集的資料
- 嘗試應用學到的知識
- 如果沒時間看，就不要收集

### ❌ 陷阱 3：資料分類混亂
**問題**：Resources 資料夾混亂，無法快速找到需要的內容

**解決**：
- 建立清晰的資料夾結構
- 使用標籤系統分類
- 定期整理和歸檔
- 建立索引文件
