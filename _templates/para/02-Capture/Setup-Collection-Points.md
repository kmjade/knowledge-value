# 任務目標
建立統一的收集點用於接收新資訊

## 收集點的重要性

### 為什麼需要收集點？
- **即時捕捉**：不要讓好想法和資訊流失
- **統一管理**：所有資訊集中在一個地方
- **後續整理**：有專門時間處理和分類
- **減少壓力**：大腦不需要記住所有資訊

### 收集點原則
- **單一性**：儘量使用一個主要收集點
- **可訪問性**：隨時隨地都能使用
- **便利性**：快速、簡單、不麻煩
- **一致性**：養成使用同一個收集點的習慣

## 設置步驟

### 步驟 1：選擇工具（10 分鐘）

#### 決策清單
- [ ] 我主要使用什麼裝置？（手機/平板/電腦）
- [ ] 我需要什麼功能？（快速記錄/雙向連結/搜尋/同步）
- [ ] 我已經在使用什麼工具？
- [ ] 我希望收集點整合到現有系統嗎？

#### 推薦選擇
**如果你使用 Obsidian：**
- 使用 Obsidian 的收件箱筆記或 Daily Notes
- 利用 QuickAdd 插件快速記錄
- 使用 Dataview 插件管理收件箱

**如果你使用 Notion：**
- 建立一個 Inbox 頁面
- 使用 Notion 的快速添加功能
- 設定資料庫管理收件箱

**如果你使用 Apple：**
- 使用 Apple Notes 作為收集點
- 建立一個名為「Inbox」的資料夾
- 使用 Siri 快速記錄

**如果你使用 Android：**
- 使用 Google Keep 作為收集點
- 建立一個名為「Inbox」的標籤
- 使用 Google Assistant 快速記錄

---

### 步驟 2：建立收集點（10 分鐘）

#### 數位收集點設置

**範例：Obsidian 收集點設置**
```markdown
---
title: 收件箱
created: {{date}}
type: inbox
---

# 📥 收件箱

記錄所有新想法、任務和資訊到這裡。

## 收集日期
{{date}}

## 新增內容

### 想法
-

### 任務
-

### 資訊
-

### 靈感
-

---
```

**範例：Notion 收集點設置**
1. 建立一個名為「📥 收件箱」的頁面
2. 使用資料庫模板記錄：
   - 類型（想法/任務/資訊/靈感）
   - 內容
   - 日期
   - 優先級

**範例：Apple Notes 收集點設置**
1. 建立「Inbox」資料夾
2. 建立一個名為「收件箱」的筆記
3. 使用「📥」前綴識別收件箱筆記

#### 實體收集點設置

**範例：紙本收件本**
1. 使用一個專用的筆記本作為收件本
2. 在封面標示「📥 收件箱」
3. 每天晚上將手寫內容數位化

**範例：便利貼系統**
1. 準備一疊便利貼
2. 貼在書桌或牆上方便取用
3. 定期將便利貼內容轉錄到數位系統

---

### 步驟 3：設定快速記錄方式（10 分鐘）

#### 數位工具快速鍵設置

**Obsidian 快速記錄：**
- 使用 Hotkeys for specific commands 插件
- 設定快速鍵開啟新筆記（如：Ctrl/Cmd + Shift + I）
- 使用 Templates 插件設定收件箱模板

**Notion 快速記錄：**
- 使用 Notion 的快速添加功能
- 設定桌面快捷方式開啟收件箱頁面

**Apple Notes 快速記錄：**
- 使用 Siri：「嘿 Siri，新增筆記到收件箱」
- 使用控制中心的快速開啟

#### 實體工具放置
- 將收件本放在最容易取用的地方
- 在多個地方放置便利貼（書桌、客廳、臥室）
- 攜帶一個小型筆記本隨身

#### 測試項目
- [ ] 記錄一個想法

## 測試驗證

### 測試內容

#### 測試 1：快速記錄
**情境**：突然有一個想法

**測試步驟**：
1. 停止當前工作
2. 開啟收集點
3. 記錄想法
4. 繼續工作

**預期結果**：
- 記錄時間 < 10 秒
- 不打斷當前工作節奏

---

#### 測試 2：隨時使用
**情境**：在不同場景下記錄

**測試場景**：
- [ ] 工作時（電腦前）
- [ ] 通勤時（移動中）
- [ ] 會議時（無法打字）
- [ ] 睡前（床上）

**預期結果**：
- 所有場景都能記錄
- 記錄方式因應場景調整

---

#### 測試 3：定期清理
**情境**：每天清理收件箱

**測試步驟**：
1. 檢查收件箱內容
2. 按照三問測試分類
3. 處理或歸檔
4. 清空收件箱

**預期結果**：
- 收件箱可以清空
- 分類決策清晰
