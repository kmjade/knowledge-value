> [!info] PARA 模板使用指南
> 快速创建符合 PARA 标准的笔记文档

## 概览 / Overview

本指南帮助您正确使用 PARA 模板创建项目、领域、资源和归档文档。

### 模板文件位置

```markdown
_Template/para/
├── _template-project.md     - 项目模板
├── _template-areas.md      - 领域模板
├── _template-resource.md    - 资源模板
├── _template-archive.md     - 归档模板
├── _template-zettel.md      - Zettelkasten 模板
└── para.md                 - 模板选择指南
```

## 使用方法 / Usage Methods

### 方法一：手动复制（推荐初学者）

1. 打开对应的模板文件
2. 全选内容复制到新文件
3. 替换占位符 `{{title}}` 等为实际内容
4. 填写具体信息

### 方法二：使用 Templater 插件（推荐）

1. 安装 [Templater](https://github.com/SilentVoid13/Templater) 插件
2. 配置模板文件夹为 `_Template/`
3. 使用快捷键（默认 `Ctrl/Cmd + P`）打开命令面板
4. 搜索 "Templater: Insert Template"
5. 选择模板文件
6. 输入文件名，内容会自动填充

### 方法三：使用 Obsidian 核心模板功能

1. 右键点击文件 → 选择 "Make a copy"
2. Obsidian 会创建模板副本，无需手动操作

## 各模板详细说明

### Project 模板 (`_template-project.md`)

#### 使用场景
- 有明确截止日期的短期项目
- 需要任务追踪的项目
- 有里程碑和交付物的工作

#### 关键字段说明

| 字段 | 说明 | 填写建议 |
|------|------|---------|
| `title` | 项目名称 | 简洁明确 |
| `due` | 截止日期 | 必填，格式 YYYY-MM-DD |
| `status` | 项目状态 | todo/in-progress/done |
| `priority` | 优先级 | high/medium/low |
| `domain` | 关联领域 | 链接到对应 Area |

#### 任务管理

**推荐使用 Obsidian Tasks 插件进行任务管理：**

```markdown
- [ ] 任务 1 📅 2024-01-15 🏷️ #high
- [ ] 任务 2 📅 2024-02-01 🏷️ #medium
```

Tasks 插件会自动识别：
- `📅 日期` = 任务截止日期
- `🏷️ #tag` = 任务优先级
- `- [ ]` = 未完成任务
- `- [x]` = 已完成任务

### Area 模板 (`_template-areas.md`)

#### 使用场景
- 长期责任领域（健康、财务、职业等）
- 需要持续维护的责任
- 没有明确完成时间的事项

#### 关键字段说明

| 字段 | 说明 | 填写建议 |
|------|------|---------|
| `title` | 领域名称 | 简洁明确 |
| `updated` | 最后更新 | 每次维护后更新 |
| `status` | 领域状态 | active/inactive/archived |
| `importance` | 重要性 | high/medium/low |

#### 定期回顾机制

模板已包含月度、季度、年度回顾结构：

```markdown
**完成情况：**
> 上月目标的完成情况
```

### Resource 模板 (`_template-resource.md`)

#### 使用场景
- 持续感兴趣的话题和材料
- 学习资源（文章、书籍、视频、课程）
- 需要记录个人理解和应用场景

#### 关键字段说明

| 字段 | 说明 | 填写建议 |
|------|------|---------|
| `title` | 资源名称 | 简洁明确 |
| `type` | 资源类型 | article/book/video/podcast/course/tool |
| `status` | 学习状态 | to-read/reading/done/reference |
| `rating` | 资源评分 | ⭐☆☆☆☆ |
| `domain` | 关联领域 | 链接到对应 Area |

#### 资源类型选择

| 类型 | 使用场景 |
|------|---------|
| `article` | 文章、博客、论文、研究报告 |
| `book` | 书籍、电子书 |
| `video` | 教程视频、演讲视频 |
| `podcast` | 播客、音频节目 |
| `course` | 在线课程、教程系列 |
| `tool` | 工具、软件、插件 |

#### 标签策略

建议使用多维度标签系统：

```markdown
# 主题分类
- #AI
- #编程
- #效率
- #健康

# 难度分类
- #beginner
- #intermediate
- #advanced

# 用途分类
- #reference
- #tutorial
- #inspiration
```

### Archive 模板 (`_template-archive.md`)

#### 使用场景
- 项目完成后的归档
- 取消或放弃的内容
- 过时或不再需要的内容
- 合并到其他文档

#### 关键字段说明

| 字段 | 说明 | 填写建议 |
|------|------|---------|
| `title` | 归档名称 | 原项目/内容名称 |
| `archived` | 归档日期 | 归档时日期 |
| `original_category` | 原始分类 | projects/areas/resources |
| `archive_reason` | 归档原因 | completed/cancelled/obsolete/merged/moved |

#### 归档原因选择

| 原因 | 说明 | 使用场景 |
|------|------|---------|
| `completed` | 项目完成 | 成功完成所有目标后归档 |
| `cancelled` | 取消或放弃 | 因各种原因取消的项目 |
| `obsolete` | 内容过时 | 信息已过时不再有用 |
| `merged` | 合并到其他 | 内容已整合到其他文档 |
| `moved` | 移动位置 | 文件移到新位置 |

### Zettelkasten 模板 (`_template-zettel.md`)

#### 使用场景
- 快速记录灵感和想法
- 创建原子化笔记
- 构建知识网络

#### 卡片类型说明

| 类型 | 用途 | 特点 |
|------|------|------|
| `fleeting` | 临时笔记 | 快速记录，后续整理 |
| `literature` | 文献笔记 | 学习内容的记录 |
| `permanent` | 永久笔记 | 标准化的核心思想 |
| `structure` | 结构笔记 | 组织卡片关系 |

#### Zettelkasten 原则

> **原子性** - 每张卡片只记录一个思想
> **自主性** - 卡片可以独立理解
> **链接性** - 通过链接建立知识网络
> **唯一性** - 使用唯一标识符

## 最佳实践 / Best Practices

### 命名规范

| 建议 | 示例 | 不推荐 |
|------|------|------|
| 使用中英文双语标题 | `Project: Website` | `项目网站` |
| 使用简洁描述 | `学习笔记` | `我的学习笔记文件.md` |
| 避免特殊字符 | `笔记关于AI` | `笔记#AI` |
| 使用有意义的前缀 | `2024-01-15_会议` | `会议1.md` |

### Frontmatter 规范

所有模板都包含标准化 YAML frontmatter：

```yaml
---
title: 文件标题
aliases: 别名
para: projects/areas/resources/archives
created: 创建日期
tags: 标签列表
---
```

### 日期格式

- **创建日期**：`YYYY-MM-DD`
- **截止日期**：`YYYY-MM-DD`
- **更新日期**：`YYYY-MM-DD`

### 状态值标准

| 类别 | 值选项 |
|------|---------|
| 项目状态 | todo, in-progress, done, on-hold |
| 领域状态 | active, inactive, archived |
| 资源状态 | to-read, reading, done, reference |
| 归档原因 | completed, cancelled, obsolete, merged, moved |
| 优先级 | high, medium, low |

### 标签使用

**标签命名建议：**
- 使用英文标签便于跨语言搜索
- 使用连字符连接多词：`machine-learning` 而非 `machinelearning`
- 使用斜杠创建层级：`AI/machine-learning/深度学习`
- 避免过长的标签

### Wikilink 使用

| 类型 | 语法 | 示例 |
|------|------|------|
| 普通链接 | `[[文件名]]` | `[[项目文档]]` |
| 带显示文本 | `[[文件名\|显示文本]]` | `[[长文件名\|短名称]]` |
| 链接标题 | `[[文件名#标题]]` | `[[项目文档#项目目标]]` |
| 链接块 | `[[文件名#^块ID]]` | `[[笔记#重要信息]]` |

### Callout 使用

常用 callout 类型：

| 类型 | 语法 | 效果 |
|------|------|------|
| 提示 | `> [!note] 内容` | 📝 蓝色提示框 |
| 信息 | `> [!info] 内容` | ℹ️ 蓝色提示框 |
| 警告 | `> [!warning] 内容` | 🟧 橙色提示框 |
| 成功 | `> [!success] 内容` | 🟩 绿色提示框 |
| 错误 | `> [!error] 内容` | 🔴 红色提示框 |
| 折叠（默认折叠） | `> [!faq]- 内容` | 🔽 灰色提示框，点击展开 |
| 折叠（默认展开） | `> [!faq]+ 内容` | 🔽 灰色提示框，点击折叠 |

## 常见问题 / FAQ

### Q: 如何修改模板？

**A:** 打开 `_Template/para/` 对应的模板文件，直接编辑内容，保存后自动更新到所有使用该模板的新文档。

### Q: Templater 变量不工作？

**A:** 确保：
1. Templater 插件已安装
2. 模板文件夹配置正确（`_Template/`）
3. 变量使用正确语法 `<% tp.date.now("YYYY-MM-DD") %>`

### Q: Dataview 查询出错？

**A:** 常见原因和解决方法：

| 问题 | 原因 | 解决方法 |
|------|------|---------|
| 语法错误 | 拼写错误 | 检查 Dataview 语法 |
| 字段不存在 | 使用了不存在的字段名 | 检查 frontmatter 中的字段名 |
| 查询路径错误 | `FROM ""` | 移除或修改路径 |

### Q: 如何批量创建 PARA 文档？

**A:** 使用以下命令快速创建：

```markdown
/para-整理收集
```
这个命令会自动分析 InBox 内容并建议 PARA 分类。

### Q: 项目完成后如何处理？

**A:** 建议流程：

1. 完成项目回顾 → 总结成果和经验
2. 提取有价值内容 → 创建到 `3 Resources/`
3. 归档项目 → 使用 Archive 模板
4. 更新关联 Area → 添加到对应 Area
