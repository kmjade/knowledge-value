# OpenCode GitLab 集成

> [!info] 概述
> OpenCode 可以与 GitLab 集成，在 GitLab CI/CD 流程中使用 AI 辅助。

## 核心能力

- Issue 分析和处理
- Merge Request 审查
- 自动修复和提交
- CI/CD 流程集成

---

## 配置 GitLab CI

### .gitlab-ci.yml

```yaml
opencode:
  stage: analysis
  image: node:20
  script:
    - npm install -g opencode-ai
    - opencode run "分析代码变更" --model claude-sonnet
  variables:
    ANTHROPIC_API_KEY: $ANTHROPIC_API_KEY
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'
```

---

## Webhook 集成

### 配置步骤

1. 在 GitLab 项目中添加 Webhook
2. URL: `https://your-server/opencode/webhook`
3. 选择触发事件：Issue comments, Merge Request events

### 处理 Webhook

使用 OpenCode serve 模式：

```bash
opencode serve --port 3000
```

---

## GitLab CI 变量

| 变量 | 说明 |
|------|------|
| `ANTHROPIC_API_KEY` | Claude API Key |
| `OPENCODE_MODEL` | 默认模型 |
| `GITLAB_TOKEN` | GitLab Access Token |

## 使用示例

### Merge Request 审查

```yaml
review:
  stage: test
  script:
    - opencode run "审查这个 MR 的代码质量"
  only:
    - merge_requests
```

### Issue 自动处理

```yaml
issue-handler:
  stage: analysis
  script:
    - opencode run "处理 Issue #${CI_ISSUE_ID}"
  only:
    variables:
      - $CI_PIPELINE_SOURCE == "issue"
```

---

## 与 GitHub 集成的差异

| 特性 | GitHub | GitLab |
|------|--------|--------|
| App 集成 | ✅ GitHub App | ⚠️ 需手动配置 |
| Actions | ✅ 原生支持 | ✅ GitLab CI |
| Webhook | ✅ 自动配置 | ⚠️ 需手动配置 |
| Secret 管理 | GitHub Secrets | CI/CD Variables |

---

## 最佳实践

1. 使用 CI/CD Variables 存储 API Key
2. 限制触发条件避免过多调用
3. 添加人工审批步骤
4. 定期检查日志
