# OpenCode Skills 技能系统

> [!info] 概述
> Skills 是 OpenCode 的**能力插件系统**，将 AI 行为从单次提示升级为可复用能力模块。

## 核心概念

### Skills vs 普通 Prompt

| 对比项 | 普通 Prompt | Skills 机制 |
|--------|-------------|-------------|
| 每次都要重新描述 | 是 | 否（只描述一次） |
| 上下文长度占用 | 每次全量塞入 | 渐进式加载 |
| 一致性 | 依赖每次 prompt 质量 | 高（固定 SOP + 模板） |
| 复用性 | 手动复制粘贴 | 自动匹配 / slash 命令 |
| 维护方式 | 改一次 prompt 就要重新发 | 修改 SKILL.md 全局生效 |

### 核心优势

- **按需加载**：只在需要时才把内容塞进上下文
- **渐进式披露**：极大节省 token
- **可复用**：可在多个项目或团队中复用

---

## Skill 核心结构

Skills 的核心就是：**一个文件夹 + 一个 SKILL.md 文件**

```
my-skill/
└── SKILL.md   （唯一必需）
```

---

## SKILL.md 模板

### 最小必填示例

```markdown
---
name: skill-name
description: 说明该 Skill 的功能以及适用场景
---
```

### 完整示例

```markdown
---
name: pdf-processing
description: 从 PDF 中提取文本和表格，填写表单，并合并文档
license: Apache-2.0
metadata:
  author: example-org
  version: "1.0"
---

# PDF 处理

## 使用场景
当需要对 PDF 文件进行操作时使用，例如：

- 提取 PDF 文本或表格数据
- 填写 PDF 表单
- 合并多个 PDF 文件

## 提取文本
- 使用 `pdfplumber` 提取文本型 PDF 内容
- 扫描版 PDF 需配合 OCR 工具

## 填写表单
- 读取 PDF 表单字段
- 按输入数据填充并生成新文件
```

---

## Frontmatter 字段

| 字段 | 必需 | 说明 |
|------|------|------|
| `name` | 是 | Skill 名称，最长 64 字符，只能使用小写字母、数字和 `-` |
| `description` | 是 | Skill 功能说明 |
| `license` | 否 | 许可证 |
| `metadata` | 否 | 元数据（author, version 等） |

---

## Skill 存放位置

| 位置 | 范围 |
|------|------|
| `./.opencode/skills/` | 项目级 |
| `~/.config/opencode/skills/` | 全局级 |

## 使用 Skill

### 自动触发

AI 根据任务自动选择匹配的 Skill。

### 斜杠命令

```
/skill-name
```

### 手动调用

```
@skill-name 执行任务
```

---

## 创建 Skill

### 步骤 1：创建目录

```bash
mkdir -p .opencode/skills/my-skill
```

### 步骤 2：创建 SKILL.md

```bash
touch .opencode/skills/my-skill/SKILL.md
```

### 步骤 3：编写内容

```markdown
---
name: code-review
description: 代码审查技能，检查代码质量、安全性和最佳实践
---

# 代码审查

## 审查维度

### 代码质量
- 代码可读性
- 函数复杂度
- 代码重复

### 安全性
- SQL 注入
- XSS 漏洞
- 敏感信息泄露

### 最佳实践
- 设计模式应用
- 错误处理
- 测试覆盖
```

---

## Skill 最佳实践

### 1. 单一职责

每个 Skill 只做一件事：

```markdown
---
name: git-commit
description: 生成规范的 Git 提交信息
---
```

### 2. 清晰的触发条件

```markdown
## 使用场景
- 需要生成 Git 提交信息时
- 需要规范化提交历史时
```

### 3. 具体的执行步骤

```markdown
## 执行步骤

1. 检查暂存区变更
2. 分析变更内容
3. 生成提交信息
4. 格式：`type(scope): description`
```

## 内置 Skills

OpenCode 内置了一些常用 Skills：

| Skill | 说明 |
|-------|------|
| `init` | 初始化项目配置 |
| `commit` | 生成提交信息 |
| `review` | 代码审查 |
