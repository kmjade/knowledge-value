## 三条架构思路

### 思路一：自下而上 — 夯实基础层
先把 02-04（分类法/叙词表/本体）做扎实，建立受控词表、统一 schema、层级化 MOC。

### 思路二：顶层驱动 — AI 反哺底层
让 LLM 编译层自动补全结构化工作：自动生成 MOC、提取概念关系、审计结构健康度。

### 思路三：以终为始 — 从表达需求反推架构
先定义输出制品（文章/演讲/项目），再从"需要什么知识"反推信息架构。

### 最终建议：三层并进
```
表达层 (Express)  →  outputs/ — 文章、决策、创作
编译层 (LLM-Wiki) →  wiki/ — 概念、实体、关系、MOC
组织层 (PARA)     →  分类体系 + 受控词表 + 元数据标准
```
