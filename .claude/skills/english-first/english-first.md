# English-First Skill

This skill enforces English-first practices for both code development and Obsidian note writing.

## Usage

Call this skill when you want to:
- Create or edit content following English-first principles
- Review existing content for language consistency
- Convert Chinese content to English where appropriate

## Principles

### For Code
- Write all code, comments, and documentation in English
- Use English variable/function names (camelCase or snake_case)
- Commit messages must be in English
- README and documentation files in English

### For Obsidian Notes
- Create notes with English titles and content
- Use English tags instead of Chinese tags
- Frontmatter properties in English (or standard Obsidian format)
- Dataview queries and inline fields in English

## Trigger

This is a manual skill. Use the `/english-first` command to activate it when needed.
