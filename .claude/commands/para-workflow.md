# PARA Workflow

Manage your notes using the PARA method: **P**rojects, **A**reas, **R**esources, **A**rchives.

## Quick Commands

| Command | Action |
|---------|--------|
| `/new-project [name]` | Create new project |
| `/new-area [name]` | Create new area |
| `/new-resource [name]` | Create new resource |
| `/archive-note` | Move note to Archive |

## PARA Structure

```
/
1 Projects/     # Active work with deadlines
2 Areas/        # Ongoing responsibilities
3 Resources/    # Topics of interest
4 Archive/      # Inactive items
```

## What Goes Where?

| Category | Has Deadline? | Ongoing? | Description |
|----------|---------------|----------|-------------|
| Project | Yes | No | Defined outcome, time-bound |
| Area | No | Yes | Life responsibilities |
| Resource | No | No | Reference material |
| Archive | - | - | Inactive items |

## Examples

- **Project**: "Website Redesign Q1"
- **Area**: "Health", "Finance"
- **Resource**: "Machine Learning Notes"
- **Archive**: "Old Project X"
